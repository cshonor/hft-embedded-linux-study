# 7.3 单周期处理器

> **Core Concept:** 一拍走完取指→译码→执行→访存→写回；CPI=1，clk 由最慢指令定  
> **Link Target:** ↔ CSAPP §4.3 SEQ · Ch5 RegFile/ALU · Ch6 指令字段  
> **档位:** 精读

- [x] 已写要点

## 笔记

一条指令在一个时钟周期内做完全部事。

### 7.3.1 单周期数据通路（逐步堆）

典型装配顺序（口述）：

1. 读 **PC** → **指令存储器** 取指  
2. 指令字段 → **寄存器堆** 读出源操作数  
3. 立即数 **零扩展/符号扩展**  
4. **ALU** 运算（或算有效地址）  
5. **数据存储器** 读/写（LDR/STR）  
6. 结果 **写回** 目的寄存器；更新 PC  

所有单元在同一拍内组合连通 —— 最长路径定最小时钟周期。

### 7.3.2 单周期控制

根据 `Op` / `Funct` 等字段生成控制信号：

| 块 | 作用 |
|----|------|
| **主译码器** | 大部分控制信号 |
| **ALU 译码器** | ALU 操作选择 |
| **条件逻辑** | 维护/使用 NZCV 等标志 |

本质是**组合逻辑**控制器（对比多周期的 FSM）。

### 7.3.3 更多指令

扩展通路/控制以支持移位、`CMP` 等 —— 加 MUX、控制行，不改「一拍一条」模型。

### 7.3.4 性能

| 优点 | 缺点 |
|------|------|
| CPI = 1，概念干净 | **Tclk ≥ 最慢指令**（常是 LDR：Mem+ALU+…） |
| 易画、易对照 CSAPP SEQ | 快指令也被迫等慢指令 |

教学价值高；真芯片很少纯单周期。
