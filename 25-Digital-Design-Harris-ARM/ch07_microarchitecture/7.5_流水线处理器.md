## 笔记

重叠执行多条指令 → 提高**吞吐**（单位时间完成的指令数）。

### 7.5.1–7.5.2 数据通路与控制

把单周期切成经典 **5 级**，级间插**流水线寄存器**：

| 级 | 名 | 干什么 |
|----|-----|--------|
| 1 | **F** Fetch | 取指 |
| 2 | **D** Decode | 译码、读寄存器 |
| 3 | **E** Execute | ALU |
| 4 | **M** Memory | 访存 |
| 5 | **W** Writeback | 写回寄存器堆 |

控制信号随指令在流水线寄存器里**逐级传递**（每级用自己那一截控制）。

### 7.5.3 冲突（Hazards）—— 核心

后指令要用的数据/控制，前指令还没准备好。

#### 数据冲突（如 RAW）

| 手段 | 何时 |
|------|------|
| **转发 / 旁路 Forwarding** | 结果已在 E/M 算出，直接喂给后面的 E，不必等写回 |
| **停顿 Stall + 气泡** | 结果还没出来（典型：**LDR** 之后立刻用加载结果）→ 插 bubble |

#### 控制冲突（分支）

下一条 PC 未知 → 可能取错指令。

| 手段 | 直觉 |
|------|------|
| 尽早算分支（在 E） | 缩短「猜错窗口」 |
| **冲刷 Flush** | 丢掉错误取入的指令 |

（高级预测见 7.7；本节先掌握 Flush/Stall 语义。）

### 7.5.4 性能

| | |
|--|--|
| 理想 CPI | ≈ 1 |
| 实际 CPI | **> 1**（stall / flush） |
| 相对单/多周期 | 吞吐通常**远好**；是现代 CPU 教学主模型 |
