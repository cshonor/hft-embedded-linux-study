## 笔记

计算机核心运算模块；面积–速度–功耗在这里第一次系统出现。

### 5.2.1 加法器

| 积木 | 作用 |
|------|------|
| **半加器** | 两位 1bit 相加 → Sum + Carry（无来自低位的进位） |
| **全加器** | A、B、Cin → Sum、Cout |
| **CPA** | 多个全加器级联 → N 位进位传播加法器 |

常见 CPA（速度 ↑、硬件复杂度 ↑）：

| 类型 | 进位方式 | 特点 |
|------|----------|------|
| **行波进位** | 串行传进位 | 简单、慢；教学默认 |
| **先行进位** | 分组提前算进位 | 更快 |
| **前缀加法器** | 树状并行前缀 | 最快量级；**32 位以上**常用 |

HFT / 高频路径：加法器级数直接吃关键路径；选型 = 延迟预算。

### 5.2.2 减法器

补码：`A − B = A + (~B) + 1` → **同一套加法器** + 取反 + Cin=1。

### 5.2.3 比较器

| 类型 | 做法 |
|------|------|
| **相等比较** | 逐位 XNOR，再 AND 聚合 |
| **数量比较** | 做减法，看符号位与溢出，判 `<` / `>` |

### 5.2.4 ALU

多路控制：加 / 减 / 与 / 或 / …；除结果外常输出 **Flags**（ARM 习惯）：

| Flag | 含义 | 用途 |
|------|------|------|
| **N** | Negative | 有符号条件 |
| **Z** | Zero | `beq` / 判零 |
| **C** | Carry | 无符号进位/借位 |
| **V** | oVerflow | 有符号溢出 |

对照 CSAPP/Y86：ZF / SF / OF（无统一的「C」命名习惯时别混）。条件跳转依赖这些标志。

### 5.2.5 移位器与循环移位

| 种类 | 空位怎么填 |
|------|------------|
| **逻辑移位** | 补 0 |
| **算术移位** | 保留符号位（ASR） |
| **循环移位** | 移出位绕回另一端 |

直觉：左移 N 位 ≈ ×2^N；算术右移 ≈ ÷2^N（有符号向 −∞ / 实现相关，先记硬件语义）。

### 5.2.6–5.2.7 乘法与除法

| 运算 | 硬件直觉 | 成本 |
|------|----------|------|
| **乘法** | AND 出部分积 → 加法树/阵列累加 | 面积、延迟都大 |
| **除法** | 阵列除法：反复减 + 移位 | **更贵**；常多周期或不用组合阵列硬做 |

嵌入式：很多 MCU 可选硬件乘除；HFT 路径尽量避开慢除法、用移位/乘法近似。
