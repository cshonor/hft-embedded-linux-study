## 笔记

Cache = 主存常用子集的 **高速小容量**（多为 SRAM）副本。

### 8.3.1 存什么

按 **块（block / line）** 从主存搬入 —— 吃时间局部性 + 空间局部性，降 miss rate。

### 8.3.2 怎么找（映射）

地址拆成：

| 字段 | 作用 |
|------|------|
| **Tag** | 这一块是不是我要的那份 |
| **Set（组号）** | 映射到哪一组 |
| **Offset** | 块内字节/字偏移 |

| 组织 | 规则 | 代价 |
|------|------|------|
| **直接映射** | 每地址只进唯一一组/一路 | 硬件简单；冲突多 |
| **N 路组相联** | 进某组，组内 N 路任选 | 折中常用 |
| **全相联** | 可放任意位置 | 冲突最少；比较器贵 |

### 8.3.3 替换

组满 / 全相联满时：常用 **LRU（最近最少使用）** 决定踢谁（也有近似 LRU）。

### 8.3.4* 高级设计

| 主题 | 要点 |
|------|------|
| **多级 Cache** | L1、L2… 进一步压 AMAT |
| **缺失三原因** | **强制**（第一次）· **容量**（Cache 不够大）· **冲突**（映射撞车） |
| **直写 write-through** | 写 Cache 同时写下级 → 简单、总线忙 |
| **回写 write-back** | 只标脏，换出时写回 → 省带宽、控制复杂 |

### 8.3.5* ARM Cache 演变

从无 Cache → 多级、I/D 分离 Cache（哈佛式 L1 常见）。扫过即可。

### 对你

| 方向 | 带走 |
|------|------|
| **HFT** | 热数据对齐块大小、避免伪共享；miss 代价按 8.2 数量级想 |
| **嵌入式** | 有的 MCU 无 Cache 或只有简单 I-Cache；开 Cache 要管一致性/DMA |
