## 笔记

用巨大但慢的磁盘/SSD 作层次最底，让程序以为有「远大于物理内存」的地址空间；并隔离进程。

### 8.4.1 地址转换

CPU 发 **虚地址 VA** → 硬件/OS 映射为 **物理地址 PA**：

```
VPN（虚页号） → PPN（物理页号）  +  页内偏移（通常直接穿过）
```

### 8.4.2 页表

- 放在**主存**；记录 VPN→PPN、**有效位**等。  
- 每次「按页表老实查」≈ **多一次主存访问** → 太贵。

### 8.4.3 TLB

**TLB（Translation Lookaside Buffer）** = 页表项的专用小 Cache。  
命中则快速出 PPN；缺失才走页表（再可能 page fault）。

访存路径直觉：

```
VA → TLB → PA → Cache →（可能）DRAM
```

### 8.4.4 存储器保护

每进程自己的页表 → **独立地址空间**；只能碰映射到自己的物理页 → 防互相踩踏、非法访问（配合权限位：读/写/执行）。

### 8.4.5* 替换

物理页不够：OS 用近似 LRU 等把页换到 **swap**；代价是磁盘级 MissPenalty。

### 8.4.6* 多级页表

单级页表太大 → **多级**：只为实际用到的虚址范围分配页表页，省物理内存。
