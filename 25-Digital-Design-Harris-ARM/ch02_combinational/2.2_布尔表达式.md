## 笔记

### 2.2.1 术语回顾

设 n 个变量 \(A,B,C\cdots\)：

| | **最小项 \(m_i\)** | **最大项 \(M_i\)** |
|--|-------------------|-------------------|
| 形式 | 每个变量以原/反 **恰好出现一次** 的 **与项** | 每个变量以原/反 **恰好出现一次** 的 **或项** |
| 特点 | **只有一组** 输入使它 = **1** | **只有一组** 输入使它 = **0** |

另：变量的非；乘积项 / 蕴涵项（可少于全部变量）。

---

### 2.2.1b 为何已有最小项，还要发明最大项？

核心疑问：最小项已能挑出「令 F=1」的输入；为何还要最大项专门描述「令 F=0」？

| | 最小项 \(m_i\) | 最大项 \(M_i\) |
|--|---------------|----------------|
| 本质 | **仅**第 i 组输入令 \(m_i=1\)，其余全 0 | **仅**第 i 组输入令 \(M_i=0\)，其余全 1 |
| 组函数 | \(F=\sum m_i\)：挑出所有令 F=1 的项相加 | \(F=\prod M_i\)：挑出所有令 F=0 的项相乘 |

#### 理由 1：0 很少时，最大项写法极短

真值表 **1 少、0 多** → 用 \(\sum m\) 舒服；  
**0 少、1 多** → 用 \(\prod M\) 更省事。

三变量，仅 **110** 使 F=0：

| 写法 | 式子 |
|------|------|
| 最小项 | \(F=\sum m(0,1,2,3,4,5,7)\)（7 项，啰嗦） |
| 最大项 | \(F=\prod M(6)\)（**一项**） |

再例：仅 **ABC=111** 使 F=0 → \(\sum m(0..6)\) vs \(\prod M(7)\)，高下立判。

#### 理由 2：直接通向「或与式」硬件

| 卡诺玩法 | 得到 | 电路骨架 |
|----------|------|----------|
| 圈 **1**（最小项） | 最简 **与或式** | 与门 → 或门 |
| 圈 **0**（最大项） | 最简 **或与式** | 或门 → 与门 |

两种结构门数、延时不一定一样；或门资源多 / 与门稀缺、或要 **低电平有效** 逻辑时，或与可能更合适。  
→ 最大项体系 = 多一条硬件结构路径。

#### 理由 3：取反 / 范式互转方便

\(F=\sum m(S) \iff F=\prod M(\overline{S})\)。  
\(\overline{F}\) 的最小项集合 = F 的最大项下标集合 → 求反函数、SOP↔POS 切换快。

#### 理由 4：低电平有效信号思维对齐

复位、片选等常 **0=有效、1=无效**；需求描述的是「哪些输入让输出变 0」。  
用最大项建模，不必反复取反绕弯。

#### 一句话对比

| 情况 | 优先 |
|------|------|
| **1 少 0 多** | 最小项 \(\sum m\) |
| **0 少 1 多** | 最大项 \(\prod M\) |

最小项主路径 → 与或电路；最大项另开路径 → **或与电路**。

**结构差异速记：** 与或 = 先与后或（多条乘积项并起来）；或与 = 先或后与（多条和项叠与）。门型与级数不同，面积/延时/极性要选更省的那条。

### 2.2.2 最小项、最大项有什么用（核心）

#### 一、标准表达式（范式）——最核心用途

任何逻辑函数都能写成：

| 范式 | 记法 | 结构 |
|------|------|------|
| **最小项之和**（标准与或式 SOP） | \(F=\sum m(\dots)\) | 选中的最小项 **相或** |
| **最大项之积**（标准或与式 POS） | \(F=\prod M(\dots)\) | 选中的最大项 **相与** |

**作用：唯一表示一个逻辑函数。**  
普通与或式写法不唯一（可化简出多种形式）；**范式唯一**。  
→ 判断两式是否相等：都展开成最小项列表，列表相同 = 同一函数。

例（二变量）：\(F=A+B\) → \(\sum m(1,2,3)\)。  
另一式若最小项集合也是 \(\{1,2,3\}\)，功能完全相同。

#### 二、真值表 ↔ 逻辑函数快速转换

| 真值表 | 写法 |
|--------|------|
| 输出为 **1** 的输入组合 | → 对应 **最小项**，相加得 \(F=\sum m(\dots)\) |
| 输出为 **0** 的输入组合 | → 对应 **最大项**，相乘得 \(F=\prod M(\dots)\) |

> 做题最常用：给真值表，立刻写出标准逻辑式。

#### 三、卡诺图化简的基础

卡诺图每个格子编号 = **最小项下标 i**。  
- **圈 1**：合并最小项 → 最简 **与或式**  
- **圈 0**：合并最大项 → 最简 **或与式**  

没有最小项编号体系，卡诺图无法规范使用。→ [2.7](./2.7_卡诺图.md)

#### 四、互补关系（做题高频）

同一下标：\(m_i=\overline{M_i}\)，\(M_i=\overline{m_i}\)。

若 \(F=\sum m(a,b,c)\)，则 \(\overline{F}=\sum m(\text{其余项})\)；  
取反可得 \(F=\prod M(\text{其余项})\)。  
→ 最小项和 ↔ 最大项积 快速互转。

#### 五、工程意义

| 场景 | 作用 |
|------|------|
| **逻辑电路设计** | 标准式统一构造；最小项之和 ↔ **与门 + 或门** |
| **PLA / PROM / ROM** | ROM 译码产生全部 \(m_i\)，再选哪些相或 → 目标函数（输出 = 选中最小项之和） |
| **多输出函数** | 用最小项清单找公共项，减门、共享逻辑 |

#### 六、一句话

**最小项、最大项 = 逻辑函数的「标准身份证」。**  
用途：真值表转式、判相等、卡诺化简、ROM/电路设计、SOP↔POS 互转。

### 2.2.3 与或式 (SOP) / 或与式 (POS)

- **SOP**：真值表中为 1 的最小项 **相或**。  
- **POS**：真值表中为 0 的最大项 **相与**。  

再交给 [2.3](./2.3_布尔代数.md) / [2.7](./2.7_卡诺图.md) 化简。

### 小例巩固（三变量）

真值表输出 1 出现在输入 **001、011** → \(F=\sum m(1,3)\)。

则 \(\overline{F}=\sum m(0,2,4,5,6,7)\)，  
且 \(F=\prod M(0,2,4,5,6,7)\)。

### 2.2.4 对照例题：0 只一处时，最大项碾压

真值表（三变量）：仅 **i=7（ABC=111）** 时 F=0，其余全 1。

| 方式 | 写法 | 长短 |
|------|------|------|
| 最小项之和 | \(F=\sum m(0,1,2,3,4,5,6)\) | **7 项**，很长 |
| 最大项之积 | \(F=\prod M(7)\) | **1 项** |

**翻译 \(M_7\)：** ABC=111；最大项规则 = 输入为 1 写反变量、为 0 写原变量  
→ \(M_7=\overline{A}+\overline{B}+\overline{C}\)，故 \(F=\overline{A}+\overline{B}+\overline{C}\)。

验证：仅 A=B=C=1 时该或式 =0，其余 =1，与真值表一致。

**硬件：** \(F=\overline{A}+\overline{B}+\overline{C}\) 可用 **一个三输入或门**（或与特例）。  
若硬写 7 个最小项 → 7 个三输入与门 + 大或门，成本天差地别。

#### 反过来：1 只一处时，最小项更短

仅 **i=2（ABC=010）** 时 F=1，其余全 0：

| 方式 | 写法 |
|------|------|
| 最小项 | \(F=\sum m(2)=\overline{A}\,B\,\overline{C}\)（**1 项**） |
| 最大项 | \(F=\prod M(0,1,3,4,5,6,7)\)（**7 项**） |

> ⚠️ \(m_2\)：A=0,B=1,C=0 → \(\overline{A}B\overline{C}\)，不是 \(A\overline{B}C\)（那是 \(m_5\)）。

#### 核心再钉

| 真值表形态 | 优先 |
|------------|------|
| **1 少、0 多** | \(\sum m\) |
| **0 少、1 多** | \(\prod M\) |

卡诺图 **圈 0** 走最大项 → 最简或与式，见 [2.7](./2.7_卡诺图.md)。
