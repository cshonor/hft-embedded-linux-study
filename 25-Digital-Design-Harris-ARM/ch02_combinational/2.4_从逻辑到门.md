### 2.4.0 整条设计流水线（贴合 CMOS）

**先钉两条化简路线（看你底层用什么门）：**

| 路线 | 目标门阵列 | 化简到 | 再变换成 |
|------|------------|--------|----------|
| **A（工业最常用）** | **与非门** | 最简 **SOP（积之和）** | 德摩根 → **与非–与非** |
| **B** | **或非门** | 最简 **POS（和之积）** | → **或非–或非** |

❌「和形式最好都转成乘积形式」——不准确。  
✅ 优先最简 SOP，因大量用与非；若底层是或非阵列，则走最简 POS。  
变换目的 = **匹配 CMOS 原生反相门**，不是硬性规定和式必须变乘积式。

```
需求 → 真值表 / 原始布尔式
  → 卡诺/代数 → 最简 SOP 或 最简 POS（按路线选，勿盲目无限拆）
  → 德摩根适配 CMOS：原生只有 与非/或非/非（无原生与/或）
  → 门级网表（每门一套独立 CMOS：PUN+PDN）
  → 画逻辑图 → 版图·流片（连线固化）或 FPGA / 分立接线
```

**例（路线 A）：** \(F=AB+\overline{A}C\)（SOP）  
\(F=\overline{\overline{AB+\overline{A}C}}=\overline{\overline{AB}\cdot\overline{\overline{A}C}}\)  
→ 与非–与非（可含反相得 \(\overline{A}\)）；**每只与非/反相各一套** PUN+PDN，门间金属相连。

**分支（进阶）：** 若整理成单一 \(\overline{(\dots)}\)（如 \(\overline{AB+\overline{A}C}\)），可用 **单级复合 CMOS 门**（一套 PUN+PDN）；课堂主学多门级联。→ [1.7](../ch01_binary/1.7_CMOS晶体管.md)

> 面包板可改线；普通 CMOS 芯片内部连线 **出厂定死**。对比 → [1.7 · 面包板 vs 硅片](../ch01_binary/1.7_CMOS晶体管.md)。

画图本质：**把布尔表达式翻译成硬件电路**。

### 2.4.1 两种表达式 → 两种电路（再接 CMOS）

| 路线 | 化简形态 | 纸上门骨架 | CMOS 适配后 |
|------|----------|------------|-------------|
| **圈 1** | 最简 SOP | 与门→或门 | 德摩根 → **与非阵列** |
| **圈 0** | 最简 POS | 或门→与门 | → **或非阵列** |

**非门（反相器）** 产生 \(\overline{A}\) 等；每个独立门 = 一套独立 CMOS（PUN+PDN）。

**极简四步：** ① 最简 SOP/POS → ② 按门库德摩根成单一类型级联 → ③ 每门一套 CMOS → ④ 原生带非，故必须做适配变换。

### 2.4.2 我们到底在设计什么？

#### 现阶段主目标：组合逻辑

输出 **仅由当前输入** 决定（无记忆）。常见：

| 模块 | 用途 |
|------|------|
| 编码器 / 译码器 | 码制转换 |
| 多路选择器 MUX | 选通一路 |
| **加法器**（半加 / 全加） | 算术核心 |
| 比较器 | 比大小 |
| 奇偶校验 | 检错 |
| 简单控制 | 报警、联锁等 |

通俗例：温度过高 **或** 门非法打开 → 报警响。  
列真值表 → 布尔式 → 画逻辑门 → 可搭实物报警器。

#### 进阶（Ch3 起）：时序逻辑

时序 = **组合逻辑 + 触发器（存储）**。  
触发器本身由门搭成；计数器、寄存器、CPU 运算单元底层大量仍靠组合块。→ [Ch3](../ch03_sequential/README.md)

### 2.4.3 工程落地：原理图之后

纸上逻辑图只是中间步骤，可实现为：

| 路径 | 做法 |
|------|------|
| **分立逻辑芯片** | 买与/或/非 IC，面包板接线（实验课） |
| **FPGA / CPLD** | HDL/原理图进工具，自动映射到片内逻辑单元（现代主流） |

### 2.4.4 考试常考：两套式子选哪张图？

同一函数可有 **最简与或式** 与 **最简或与式** → 两张不同电路图。  
画图前对比：**门更少、输入端更少** 的方案成本更低。  
→ 这也是学最大项、圈 0 的工程意义。→ [2.2](./2.2_布尔表达式.md)

### 2.4.5 绘图规范（提高可读性）

- 输入在左/上，输出在右/下  
- 直线连接，避免乱线  
- **多输出：** 真值表可共用；实现时 **共享** 公共门，少重复  

Logisim 等工具按同样规范画即可。

### 2.4.6 再下一层：布尔变量 → 逻辑门 → MOS（物理实现）

✅ **A、B、C = 输入信号，F = 输出信号**（逻辑层没错）。  
❌ 不能粗说「输入=栅极、输出=漏极」——要区分 **单管** vs **CMOS 门（N+P 成对）**。

#### 单管角色

| 极 | 作用 |
|----|------|
| **G 栅** | 控制端；布尔输入 **最终接到各管栅极**（栅几乎不流电流） |
| **S / D** | 电流通路两端；通/断由栅电平决定 |

> **单独一只 MOS 不能稳定输出完整逻辑电平。** 标准单元几乎都是 **CMOS：PMOS + NMOS 配对**。

#### 例：CMOS 反相器 \(F=\overline{A}\)

| 管 | S | G | D |
|----|---|---|---|
| PMOS | VDD | **A** | 公共输出节点 |
| NMOS | GND | **A** | 公共输出节点 |

- 输入 A → **两管栅极**  
- 输出 F → **两管漏极短接的公共节点**（不是「某一个管子单独的漏极」）

#### 标准 CMOS 门（工业原生 NAND / NOR）

芯片里原生优先做 **与非 / 或非**（再加非得与/或）。拓扑 → [1.7 · 2b](../ch01_binary/1.7_CMOS晶体管.md)。

| 门 | 下 NMOS | 上 PMOS |
|----|---------|---------|
| 与非 \(\overline{AB}\) | **串联** | **并联** |
| 或非 \(\overline{A+B}\) | **并联** | **串联** |

A、B 分接对应管栅；公共漏节点 = F。

> 标准静态 CMOS：**某一个 k 输入门** → k 个 N + k 个 P；  
> ❌ **不是**「电路有几个全局输入就有几对管」。伪 NMOS / 动态例外 → [1.7 · §6](../ch01_binary/1.7_CMOS晶体管.md)。

#### 两层翻译，禁止跳步

```
布尔式（A,B,C,D…）
  → 逻辑门网表（先化简；与/或 用德摩根 → 与非/或非/非）
  → 逐个门展开 CMOS 管（按该门的 k 计管子）
  → 外输入布线到各栅（可扇出）
```

例 \(F=AB+\overline{C}D\)：外部仍 4 信号；转成若干 2 输入与非后，管子按 **门的个数×每门 2N+2P** 累加（可远多于 4 对），不是 4N+4P。

反例：名义 4 输入、电路只用 \(\overline{A+B}\) → 仅 **2N+2P**。

#### 自上而下完整层级

1. 功能需求  
2. 真值表 → 范式 → **布尔式 (A,B,C,F)**  
3. 画逻辑门图（与/或/非）  
4. （可选）德摩根 → 与非/或非网表  
5. **每个门底层 = 若干 PMOS/NMOS**  
6. 输入 A/B/C **布线到各管栅极 G**（一输入可扇出到多管栅）  
7. 输出 F = **漏极互连节点**

#### 易踩误区

| ❌ | ✅ |
|----|-----|
| 输入 = 栅极 | 输入信号 **被布线送到** 各管栅极；一输入可接很多栅（**扇出**） |
| 输出 = 某管漏极 | 输出 = **一组** MOS 漏极相连的公共节点 |
| 逻辑图 = 晶体管图 | **逻辑抽象层** ↔ **门电路翻译层** ↔ **晶体管物理层** |
| 全局 4 输入 ⇒ 4N+4P | 管子数看 **门清单**；k 指 **单门输入脚数** |

#### 与布尔代数的接点

与/或/非运算 = 用 A/B/C 控制各管栅极通断，把节点 F **拉到 VDD(1) 或 GND(0)**。

> **一句话：** 逻辑输入驱动 MOS **栅极**；逻辑输出取自漏极 **互连节点**；CMOS 是上下一对管协同，不是单管「栅入漏出」。

**补钉：** 不能只用 PMOS（或只用 NMOS）搭稳定与/或门——单一类型只能单向拉电平；**关断后的浮空 ≠ 逻辑 0/1**。必须 **P 上拉网 + N 下拉网**。→ [1.7 · §7](../ch01_binary/1.7_CMOS晶体管.md)
