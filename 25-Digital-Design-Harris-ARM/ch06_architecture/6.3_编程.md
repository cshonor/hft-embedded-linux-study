## 笔记

本章核心：高级语言结构如何落到 ARM 汇编。

### 6.3.1 数据处理指令

| 类 | 例子 |
|----|------|
| 逻辑 | `AND` `ORR` `EOR` … |
| 移位 | `LSL` `LSR` `ASR` `ROR` |
| 乘法 | `MUL` |

（与 Ch5 移位器 / ALU 硬件对应。）

### 6.3.2 条件标志

**CPSR** 里保存 ALU 条件标志（接 Ch5.2.4）：

| 位 | 含义 |
|----|------|
| **N** | Negative |
| **Z** | Zero |
| **C** | Carry |
| **V** | oVerflow |

条件执行 / 条件分支的依据。许多数据处理指令可带 `S` 后缀更新标志（如 `ADDS`）。

### 6.3.3–6.3.5 分支 · 条件 · 循环

| 高级语言 | 汇编手段 |
|----------|----------|
| `if` / `else` | 条件执行 + `B` / 条件分支 |
| `switch` / `case` | 比较 + 多路分支（或跳转表） |
| `while` / `for` | 比较 + 回跳分支 |
| 函数调用入口 | `BL`（写 LR）等 |

关键指令直觉：`B` 跳转；`BL` 带链接（保存返回地址到 LR）。

### 6.3.6 存储器

| 指令 | 作用 |
|------|------|
| **LDR** | 加载：Mem → Reg |
| **STR** | 存储：Reg → Mem |

寻址模式（口述三点）：

| 模式 | 直觉 |
|------|------|
| 偏移量 | `base + offset` |
| 预索引 | 先改基址再访存（可写回） |
| 后索引 | 先访存再改基址 |

数组、字符（ASCII）经 LDR/STR + 地址计算访问。ARM 是 **load/store 架构**：算术一般不直接吃内存操作数（对比 6.8 x86）。

### 6.3.7 函数调用

| 约定项 | ARM（典型 AAPCS 直觉） |
|--------|------------------------|
| 参数 | **R0–R3**（更多走栈） |
| 返回值 | **R0**（宽返回再扩） |
| 返回地址 | **LR (R14)**，由 `BL` 写入 |
| 栈 | **SP (R13)** 向下长 |

**被调用者保存 vs 调用者保存**（书中常称受保护 / 不受保护寄存器）：

- 跨调用必须活着的寄存器 → 被调方进函数时压栈、返回前恢复  
- 临时寄存器 → 调用方若还需用，自己先保存  

细节以 AAPCS 为准；嵌入式读反汇编时先认 R0–R3 / LR / SP。
