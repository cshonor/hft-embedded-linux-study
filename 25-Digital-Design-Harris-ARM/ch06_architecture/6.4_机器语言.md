## 笔记

汇编 → 0/1，电路才能执行。经典 ARM：**固定 32 位指令字**（Thumb 见 6.7）。

### 6.4.1–6.4.3 指令格式（三大类）

字段大致包括：`cond`、`op`、`funct`、`Rn`、`Rd` 等（具体位域以书图为准）。

| 大类 | 例子 | 编码里要表达什么 |
|------|------|------------------|
| **数据处理** | ADD、AND | 操作类型、寄存器号、立即/移位 |
| **存储器** | LDR、STR | 基址、偏移、升降址、写回 |
| **分支** | B、BL | 相对偏移、是否写 LR |

`cond`：几乎每条指令可带条件码 → 条件执行（少分支气泡的历史设计点）。

### 6.4.4–6.4.5 寻址与反汇编

四种主模式（口述）：

| 模式 | 用途直觉 |
|------|----------|
| 寄存器 | 操作数在寄存器 |
| 立即 | 常数在指令里 |
| 基址 | 基址寄存器 ± 偏移 → 访存 |
| PC 相对 | 相对当前 PC 取偏移（分支、文字池常相关） |

练习：十六进制机器码 → 拆字段 → 还原助记符（Ch7 译码器吃的就是这些位）。

### 6.4.6 程序存储（Stored Program）

指令当**数据**放在存储器里；处理器循环：

```
Fetch → Decode → Execute …
```

同一硬件跑不同程序 → 通用计算机。第7章把这条环路变成具体数据通路。
