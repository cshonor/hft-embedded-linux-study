# 3.1 指针与数组的根本混淆

## 本节讲什么

语义层最核心规则：**数组 ≠ 指针**。数组名在多数表达式中 **decay** 为首元素指针，但 `sizeof` / `&` 例外；函数形参处 **永远退化为指针**。

## 要点

### decay 与例外

| 语境 | `int arr[10]` 的类型/行为 |
|------|---------------------------|
| 一般表达式 | `int*`（首元素地址） |
| **`sizeof(arr)`** | 整个数组：**40**（10×4） |
| **`&arr`** | `int (*)[10]`，步长 40 |
| **形参 `int arr[]`** | 等价 **`int *arr`** |

```c
int arr[10];
sizeof(arr);     /* 40 */
sizeof(&arr);    /* 8（指针宽度，64 位） */
int *p = arr;
sizeof(p);       /* 8 — 不是缓冲区长度！ */
```

### 内核/嵌入式坑

```c
void copy(int *buf) {
    memcpy(dst, buf, sizeof(buf));  /* 错：得到 8，不是缓冲区字节数 */
}
```

必须 **单独传 `size_t len`** 或用 `sizeof` 仅对 **未 decay 的数组对象**。

### 函数参数

```c
void func(int arr[]);  /* 完全等价 */
void func(int *arr);
/* 函数内 sizeof(arr) == sizeof(int*) */
```
