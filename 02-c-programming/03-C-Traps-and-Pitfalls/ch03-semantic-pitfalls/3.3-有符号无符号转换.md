# 3.3 有符号与无符号隐式转换

## 本节讲什么

**usual arithmetic conversions**：有符号与无符号混合运算时，有符号操作数常 **转为无符号**。比较、循环边界因此 **静默颠倒**。

## 要点

### 比较陷阱

```c
unsigned u = 10;
int i = -1;
if (i < u)   /* 假！i 转为 UINT_MAX 量级 */
```

等价：`if ((unsigned)i < u)` → 巨大无符号数 `< 10` 为假。

### 无符号循环死循环

```c
for (unsigned i = 10; i >= 0; i--)
    ;  /* i 永远 >= 0，永不退出 */
```

无 unsigned **不存在负数**，`i--` 到 0 后再减 → 回绕到 `UINT_MAX`。

### 底层危害

- HFT 遍历数组 `for (size_t i = n; i >= 0; i--)` 卡死
- 驱动 DMA 长度 `int len` 与 `size_t` 混比

### 工程规范

1. **循环计数器类型统一**（全 `size_t` 或全 `int` 且保证非负）
2. 与无符号比较前 **显式 cast** 或改类型
3. 负数场景 **不用 unsigned** 存「可能为负」的量
