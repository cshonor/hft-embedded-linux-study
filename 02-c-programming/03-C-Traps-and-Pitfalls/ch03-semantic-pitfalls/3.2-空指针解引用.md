# 3.2 空指针解引用

## 本节讲什么

`NULL`（地址 0）解引用属于 **未定义行为（UB）**。表现因平台而异：用户态常 **SIGSEGV**；裸机可能 **静默写硬件寄存器**。

## 要点

### 典型错误

```c
char *p = NULL;
*p = 'a';           /* UB */
if (p) *p = 'a';    /* 正确：先检查 */
```

### 平台差异

| 环境 | 行为 |
|------|------|
| Linux 用户态 | 段错误，易调试 |
| MCU 地址 0 映射寄存器 | **无崩溃**，硬件被改 |
| 内核 `NULL` 指针保护 | 故意映射不可访问页，触发 oops |

### 与短路求值

```c
if (p != NULL && *p == 'x')  /* 安全：右侧不执行 */
```

语法见 [ch02 2.1](../ch02-syntactic-pitfalls/2.1-运算符优先级.md) `&&` 短路。

### 工程规范

- 解引用前 **显式判空**
- API 文档标明谁负责分配/释放
- 可选：`assert(p != NULL)` 调试版
