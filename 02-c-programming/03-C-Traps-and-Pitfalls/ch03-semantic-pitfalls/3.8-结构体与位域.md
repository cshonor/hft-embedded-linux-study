# 3.8 结构体与位域

## 本节讲什么

结构体 **`sizeof` 含 padding**；跨机器/协议 **不能直接 memcpy 整 struct** 当 wire format。位域 **signed/unsigned** 混用导致符号解释错误。

## 要点

### 对齐与填充

```c
struct S { char a; int b; };
/* sizeof(S) 常为 8，不是 5 */
```

见 [Expert C ch07](../04-Expert-C-Programming/ch07-adventures-in-memory/) 对齐手算；网络协议用 **packed** 或 **手动序列化**。

### 位域

```c
struct Flags {
    unsigned ok : 1;
    signed   err : 4;  /* 高位可能按符号扩展 */
};
```

### 工程规范

- 协议：**固定宽度类型** + 显式字节序
- 硬件寄存器：对照手册偏移，不假设 `offsetof` 跨编译器相同（除非 `packed` + 固定 ABI）
