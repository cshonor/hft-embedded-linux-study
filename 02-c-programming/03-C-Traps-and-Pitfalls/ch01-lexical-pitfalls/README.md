# 第 1 章 词法「陷阱」

**Lexical Pitfalls** — Andrew Koenig, *C Traps and Pitfalls*

## 本章目标

掌握编译器 **词法分析（lex）** 如何把源码拆成 **token**，从 **字符流** 根源规避歧义，杜绝「看起来合法、拆分逻辑却错了」的隐性 bug。

> 词法 → 语法 → 语义 → 连接 → 库 → 预处理 → 可移植性（全书链路；见 [ch00](../ch00-introduction/)）

## 词法分析是什么

```text
源码字符流  →  [词法分析 lex]  →  token 流  →  [语法分析 parse]  →  ...
                关键字、标识符、
                123、010、'0'、"0"、
                ++、==、/* */
```

本章所有坑：**编译器在切 token 时产生歧义或误读**。

## 速查表

| 陷阱 | 正确习惯 |
|------|----------|
| `if (x = 0)` | `if (0 == x)` 或分两句 |
| 注释屏蔽含 `*/` 的块 | `#if 0` … `#endif` |
| `c == "0"` | `c == '0'` |
| `010` | 写 `10` 或 `0o10` 注释说明八进制 |
| `a+++b` | 自增单独一行，加括号 |
| 宏与关键字同名 | 避免 `#define int` 类恶作剧 |

## 底层开发适用

1. **HFT**：杜绝 `=` 代替 `==`，分支静默颠倒
2. **驱动**：大块注释禁用 `#if 0`；寄存器值用 `'0'` 非 `"0"`
3. **裸机**：八进制地址字面量 `0120` vs `120`

4. `010` 的值是多少？`09` 能否编译？
