# 1.1 等号与相等判断 `=` / `==`

## 本节讲什么

C 在 `if`/`while` 条件里 **允许赋值表达式**，编译器不会把 `=` 当成笔误报错。`x = 0` 与 `x == 0` 是最经典、最危险的 **静默 bug** 之一。

## 要点

### 赋值 vs 比较

```c
if (x = 0)   /* 赋值：x 变为 0，表达式值为 0 → 条件为假 */
if (x == 0)  /* 比较：判断 x 是否为 0 */
```

| 写法 | 效果 |
|------|------|
| `if (x = expr)` | 把 `expr` 赋给 `x`，条件看 **赋值后的值** |
| `if (x == expr)` | 仅比较，**不修改** `x` |

### 典型危害

- **分支永远走错**：本意判断阈值，实际每次把变量清零
- **HFT/策略逻辑颠倒**：无崩溃、无断言，回测与实盘偏差极难查
- **与 ch02 叠加**：`if (a = b > 0)` 实际是 `a = (b > 0)`（[Expert C ch08](../04-Expert-C-Programming/ch08-halloween-vs-christmas/) Halloween 陷阱）

### 工程规避

1. **Yoda 条件**：`if (0 == x)` — 漏写 `=` 成 `if (0 = x)` 编译报错
2. **赋值与判断分离**：先赋值，再 `if (x == 0)`
3. **开启警告**：`-Wall -Wparentheses`，部分场景 `-Wassignments`（若编译器支持）
4. **静态分析**：clang-tidy、`if (x = ...)` 模式
