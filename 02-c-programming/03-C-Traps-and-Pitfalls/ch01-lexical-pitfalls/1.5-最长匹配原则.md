# 1.5 运算符最长匹配（maximal munch）

## 本节讲什么

词法分析器扫描运算符时，采用 **最长匹配**：在多种合法拆法下，**尽可能吃掉更长的 token**。这是 `a+++b`、`x---y` 等行为与直觉不符的根源。

## 要点

### 规则

遇到 `+`、`-`、`<`、`>`、`=` 等时，编译器 **向前看** 组成最长合法运算符：

| 字符序列 | 拆分 | 不是 |
|----------|------|------|
| `a+++b` | `(a++) + b` | `a + (++b)` |
| `x---y` | `(x--) - y` | `x - (--y)` |
| `i+++++j` | 非法（`++` 后不能再 `++` 接 `+` 形成合法 token 链） | — |
| `<<=` | `<<=` 一个 token | `<<` + `=` |

### 与语法/优先级的关系

- **词法** 先定 token 边界（本章）
- **语法/优先级** 再定表达式树（**ch02**）
- 两者叠加：`a+++b` 先被切成 `a++`、`+`、`b`，再按左结合做加法

### 自增混用风险

```c
int a = 1, b = 2;
int c = a+++b;  /* c = (a++) + b → a 变 2, c = 1+2 = 3 */
```

同一表达式多次修改 `a` 的变体（`++a + ++a`）属于 **UB**（[Expert C ch08](../04-Expert-C-Programming/ch08-halloween-vs-christmas/) Christmas 问题）—— **词法合法 ≠ 语义合法**。

### 工程建议

1. 自增 **单独一行**
2. 复杂表达式 **加括号**（帮助人类，不改变词法拆分）
3. 禁止在宏里拼 `a++` 与相邻 `+`
