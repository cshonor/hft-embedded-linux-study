# 1.2 注释不会嵌套

## 本节讲什么

C 只认 **`/*` … `*/` 块注释** 与 **`//` 行注释**（C99+）。块注释 **不能嵌套**：从第一个 `/*` 到 **遇到的第一个 `*/`** 即结束，中间一切（包括另一个 `/*`）都被当作注释内容。

## 要点

### 错误示例

```c
/* 外层开始
   int a;
   /* 内层？ */
   int b;
*/
```

编译器视角：

1. 第一个 `/*` 开始注释
2. 遇到 **第一个** `*/`（「内层？」后面）→ 注释 **结束**
3. 后面的 `int b;` 变成 **正常代码**；再出现的 `*/` → **非法 token**，编译报错

### 实用坑

| 场景 | 后果 |
|------|------|
| 用 `/* */` **整块注释掉** 已含注释的代码 | 提前截断，语法错乱 |
| 驱动里临时屏蔽大段初始化 | 以为全禁用了，实际只禁了一部分 |
| 文档里写 `/* example */` 嵌在外层注释 | 外层被意外关闭 |

### 正确做法

1. **临时屏蔽**：用 `#if 0` … `#endif`（预处理器，可嵌套）
2. **行注释**：逐行 `//`（注意字符串里不要有未闭合的 `/*`）
3. **版本控制**：删代码靠 git，不靠长期大块注释

```c
#if 0
/* 这里可以安全地写 /* 也不会破坏外层 */
int legacy_code(void) { return 0; }
#endif
```
