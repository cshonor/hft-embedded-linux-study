# 1.4 数字字面量歧义

## 本节讲什么

数字在词法阶段的写法直接决定 **进制、类型、值**。`010` 不是十进制 10；后缀决定 `unsigned`/`long`/`float`。

## 要点

### 八进制陷阱

| 写法 | 实际值 |
|------|--------|
| `10` | 十进制 10 |
| `010` | 八进制 → **十进制 8** |
| `09` | **非法**（八进制无数字 9） |

```c
int addr = 010;  /* 常见误读：以为是 10，实为 8 */
```

**硬件/驱动**：数据手册写八进制地址时，源码 `0120` 与 `120` 完全不同。

### 十六进制

`0x` 前缀：`0x10` = 16。与八进制混淆较少，但注意 **无前缀的 0**。

### 浮点

| 写法 | 等价 |
|------|------|
| `.5` | `0.5` |
| `5.` | `5.0` |
| `1e2` | `100.0` |

### 后缀

| 后缀 | 含义 |
|------|------|
| `U` / `u` | `unsigned` |
| `L` / `l` | `long` |
| `LL` / `ll` | `long long` |
| `F` / `f` | `float` |
| `UL` | `unsigned long` |

```c
123U
123L
123.0F
```

无后缀整型：类型为 `int`，超出范围才变 `long`（规则见 K&R / 标准）。
