# 4.7 未定义引用

## 本节讲什么

编译期只需 **声明**；**定义** 必须在某个 `.o` 或库成员中，并在链接命令行 **被拉入**。漏 `.o`、漏 `-lfoo` → `undefined reference to 'symbol'`。

## 要点

```text
main.o: undefined reference to `foo'
```

常见原因：

| 原因 | 修复 |
|------|------|
| 只声明未定义 | 某 `.c` 实现 `foo` |
| 实现 `.o` 未加入链接 | Makefile 加 `foo.o` |
| 实现在 `.a` 但库顺序/扫描问题 | 调整 `-l` 顺序（4.5） |
| C/C++ 混编未 `extern "C"` | 包装 C 接口 |

### 与编译错误区别

- **编译错误**：单文件内语法/类型
- **链接错误**：全程序符号表 **拼不齐**
