# 4.1 外部变量：声明与定义

## 本节讲什么

多文件工程中，**声明（declaration）** 与 **定义（definition）** 混淆是链接阶段第一坑。`extern` 只引入名字；**定义** 才分配存储，且全局对象 **整个程序只能有一处定义**。

## 要点

| | 写法 | 作用 |
|---|------|------|
| **声明** | `extern int val;` | 告诉编译器「别处有定义」，**不** 分配内存 |
| **定义** | `int val;` / `int val = 0;` | **分配** 存储，链接器导出符号 |

### 经典错误

```c
/* bad.h — 每个 #include 的 .c 都生成一份定义 */
int val;
```

链接：`multiple definition of 'val'`。

### 正确模式

```c
/* good.h */
extern int val;

/* good.c（唯一） */
int val = 0;
```
