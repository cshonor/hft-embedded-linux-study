# 4.5 静态库链接顺序

## 本节讲什么

`.a` 是 **`.o` 归档**；链接器按命令行 **从左到右** 扫描，**仅当** 当前未解析符号 **需要** 库中某成员时才提取该 `.o`。顺序错误 → **`undefined reference`**。

## 要点

```bash
# 典型：主程序 .o 在前，库在后
gcc main.o -L. -lfoo -o prog

# 错误示例（老链接器/严格环境）：
gcc -lfoo main.o   # 扫描 libfoo 时尚无 undefined，库可能被跳过
```

### 规则

> **被依赖的库写在依赖它的目标文件之后**；循环依赖时 **重复** `-lfoo`。

### 现代 GCC

`gcc main.c -lm` 常仍能工作（链接器多次扫描），**嵌入式/交叉工具链** 仍可能严格；Makefile 应显式：`OBJS ... -lfoo -lbar`。
