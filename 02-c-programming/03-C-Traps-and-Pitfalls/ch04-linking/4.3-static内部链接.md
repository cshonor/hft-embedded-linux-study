# 4.3 static 与符号可见性

## 本节讲什么

文件作用域 **`static`** 全局变量/函数 → **内部链接（internal linkage）**：符号 **不导出** 给其他 `.o`，仅本翻译单元可见。

## 要点

```c
/* file_a.c */
static int counter;      /* 仅 file_a.o 可见 */
static void helper(void) { }

/* file_b.c */
static int counter;      /* 不同符号，不冲突 */
```

| 链接属性 | 写法 | 其他文件 |
|----------|------|----------|
| **外部链接** | 全局 `int g;` / 非 static 函数 | `extern` 可引用 |
| **内部链接** | `static` 全局 / `static` 函数 | **不可** 访问 |

### 工程场景

- 驱动、内核：**模块私有** 全局与工具函数一律 `static`
- 避免 **同名符号** 在多个 `.o` 冲突
- 与 C++ `anonymous namespace` 类似意图

### 勿与 static 局部混淆

函数内 `static int x` 是 **静态存储期、块作用域**（[ch03](../ch03-semantic-pitfalls/)），仍在本 `.o` 有符号（通常 **local** 符号）。
