# 4.4 跨文件类型不一致（最危险静默 bug）

## 本节讲什么

链接器按 **符号名** 合并引用，**不做 C 类型检查**。定义与 `extern` 声明类型不一致时，编译各自通过，运行 **内存解释完全错误**。

## 要点

```c
/* file1.c */
char msg[] = "hello";

/* file2.c */
extern char *msg;
```

| 符号 | file1 视角 | file2 视角 |
|------|------------|------------|
| `msg` | 数组，内容在 `.rodata`/`.data` | **指针变量**，应存地址 |

file2 把 **字符串首字节** 当 **指针值** 解引用 → 段错误或乱码。

### 同类陷阱

```c
/* file1 */ int buf[100];
/* file2 */ extern char *buf;  /* 同样灾难 */
```

见 [Expert C ch04 demo02](../04-Expert-C-Programming/ch04-arrays-are-not-pointers/demo/demo02_extern/)。

### 为何链接不报错

- 各 `.o` 的符号表只有 **名字 + 地址/段**，无完整 C 类型
- **同一名字** 即可匹配

### 工程规范

- 共享对象 **统一头文件** 声明
- 数组：`extern char msg[];` 或 `#define MSG_LEN` + `extern char msg[MSG_LEN];`
- 禁止 `.c` 之间口头约定类型
