# 4.2 未初始化与已初始化全局变量

## 本节讲什么

全局/静态对象在链接合并后进入 **`.bss`（未初始化）** 或 **`.data`（已初始化）**；行为不同，依赖「默认全零」时要明确哪些对象在 `.bss`。

## 要点

```c
int g;        /* .bss，加载时清零 */
int h = 10;   /* .data，占 ELF 初始值 */
```

| 存储 | 特点 |
|------|------|
| **`.bss`** | ELF 常 NOBITS；运行时分配并 **清零** |
| **`.data`** | 文件中带初值拷贝 |

### 逻辑坑

- 以为「所有全局都会清零」—— **只有未显式初值的全局/静态** 进 `.bss` 清零；`static int x = 0` 也在 `.data`（初值为 0 但仍占 `.data` 条目，实现可优化到 bss）
- 多文件 **不会** 因一个 `.bss` 一个 `.data` 同名冲突—— **同名才是灾难**（4.1、4.6）

### 工具

```bash
readelf -S a.out | grep -E '\.bss|\.data'
nm a.out | grep ' g'
```
