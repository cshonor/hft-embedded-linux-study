# 4.8 头文件保护与循环包含

## 本节讲什么

头文件被多次 `#include` 会 **重复展开** 声明/定义，引发 **重复定义** 或类型冲突。**Include guard** 或 `#pragma once` 必需。

## 要点

```c
#ifndef PROJECT_FOO_H
#define PROJECT_FOO_H
/* 声明、inline、宏 */
#endif
```

### 循环包含

`a.h` → `b.h` → `a.h`：第二次进入 `a.h` 时 guard 已定义，内容跳过—— **前提是有 guard**。

无 guard 时：

- 同一 `struct` 定义两次 → 编译错误
- 头文件里误写 `int g;` → 多 `.o` 多重定义（4.1）

### 最佳实践

1. 头文件 **只放声明**、inline、宏
2. **Include guard** 或 `#pragma once`
3. 减少头文件互相依赖；前置声明 `struct X;`
