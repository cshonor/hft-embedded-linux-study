# 5.6 数学库与隐式转换

## 本节讲什么

`<math.h>` 函数（如 `sqrt` / `sin`）参数与返回值均为 **浮点**（多数为 `double`）。整型实参会 **提升为 double**，返回值若赋给整型会 **截断小数**，精度与范围陷阱在数值代码、嵌入式标定中常见。

## 要点

```c
int x = sqrt(4);        /* 合法：结果为 2.0 → 截断为 2 */
double y = sqrt(2);     /* 正确接收 */

float f = 1.0f;
sqrt(f);                /* float 提升为 double 传入 */
```

### 链接注意

使用 `sqrt` 等需 **链接数学库** `-lm`（部分平台 `gcc` 默认已链，嵌入式/静态链接常需显式）：

```bash
gcc main.c -lm
```

与 [ch04 4.5](../ch04-linking/4.5-静态库链接顺序.md) 库顺序问题叠加。

### 其他陷阱

- `pow(0.1, 2)` 浮点误差 — 勿用 `==` 比较浮点结果
- 大整数开方：先转 `double` 可能 **丢失精度**（>2^53）
