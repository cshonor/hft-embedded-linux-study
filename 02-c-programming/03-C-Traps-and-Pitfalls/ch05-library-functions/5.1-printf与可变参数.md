# 5.1 printf / 可变参数 stdarg 系列

## 本节讲什么

`printf` 及同类可变参数函数 **不做实参类型检查**，仅按格式串从栈/寄存器 **按约定宽度** 取二进制位。格式符与实参不匹配、漏传参数、`%s` 传 NULL —— 编译通常 **零告警**，运行输出乱码或 **直接崩溃**。

## 要点

### 5.1.1 格式符与实参类型不匹配

```c
int a = 10;
printf("%f", a);   /* %f 读 8 字节 double，int 仅 4 字节 → 脏内存 */

long x = 0x123456789ABC;
printf("%d", x);   /* 64 位下截断高位 */
```

| 格式符 | 期望实参类型 |
|--------|--------------|
| `%d` `%i` | `int` |
| `%u` | `unsigned int` |
| `%ld` `%lu` | `long` / `unsigned long` |
| `%lld` `%llu` | `long long` / `unsigned long long` |
| `%f` `%lf` | `double` |
| `%s` | `char *` |
| `%p` | `void *` |

### 5.1.2 漏传参数

```c
printf("%d %s");   /* 从栈读垃圾值，不可预测 */
```

### 5.1.3 默认参数提升（default argument promotions）

可变参数中：

- `float` → 提升为 `double`（故 `%f` 对应 double，无 `%f` 配 float 的独立约定）
- `char` / `short` → 提升为 `int`

写 `printf("%c", ch)` 时 `ch` 已是 `int` 宽度。

### 5.1.4 `%s` 要求非空指针

```c
char *p = NULL;
printf("%s", p);   /* 解引用空指针 → 崩溃 */
```

### 工程规范

- 开启 `-Wformat`（`-Wall` 通常包含），GCC/Clang 能捕获 **部分** 静态格式串错误
- 动态格式串（用户输入）极度危险
- 64 位：`size_t` 用 `%zu`，`ptrdiff_t` 用 `%td`
