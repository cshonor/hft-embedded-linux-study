# 5.3 内存分配 malloc / free

## 本节讲什么

堆内存管理错误 —— 判空遗漏、双重释放、释放栈地址 —— 往往 **延迟爆发**：堆元数据损坏后，后续 `malloc`/`free` 随机卡死或崩溃，极难调试。

## 要点

### 5.3.1 malloc(0) 行为未标准化

实现可能返回 `NULL` 或 **唯一非 NULL 指针**；不能统一当作「分配失败」。

### 5.3.2 失败返回 NULL

```c
int *p = malloc(1000);
*p = 1;   /* p==NULL 时崩溃 */
```

**必须判空**；大分配考虑 `calloc` / 检查溢出 `n * size`。

### 5.3.3 双重 free / free 栈内存

```c
int *p = malloc(4);
free(p);
free(p);        /* 双重 free → 堆损坏 */

int arr[10];
free(arr);      /* 只能 free 堆指针 → UB */
```

释放后置 `p = NULL`，避免悬空指针误用。

### 5.3.4 free(NULL) 合法

`free(NULL)` **无操作**，安全；可简化错误路径。

### 工程规范

- 谁分配谁释放；模块边界用明确 ownership
- 避免 `malloc` 后 `memcpy` 到栈再 `free` 的复杂生命周期混乱
