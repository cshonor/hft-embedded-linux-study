# 5.2 字符串处理函数

## 本节讲什么

`<string.h>` 中 `strcpy` / `strcat` / `strlen` / `strcmp` 等是 **安全重灾区**：多数接口 **不检查缓冲区长度**，溢出、缺 `\0`、NULL 解引用在协议解析、驱动、嵌入式中极常见。

## 要点

### 5.2.1 strcpy — 无长度检查

```c
char buf[16];
strcpy(buf, very_long_str);   /* 覆盖相邻栈/堆 → 漏洞 */
```

**替代**：`strncpy` / `snprintf` / 自研带边界拷贝；Linux 内核常用 `strlcpy`（非标准 C）。

`strncpy` 陷阱：拷贝 `n` 字节后 **若源串 ≥ n，不会补 `\0`**：

```c
char dst[4];
strncpy(dst, "hello", 4);   /* dst = "hell"，无终止符 */
strlen(dst);                /* 越界读 */
```

安全模式：

```c
strncpy(dst, src, sizeof(dst) - 1);
dst[sizeof(dst) - 1] = '\0';
```

### 5.2.2 strcat — 同样无边界

拼接前不校验目标剩余空间；须先 `strlen(dst) + strlen(src) < cap`。

### 5.2.3 strlen — 不含 `\0`

`strlen("abc") == 3`；存放需 **`strlen(src) + 1`** 字节。

经典 off-by-one：

```c
char buf[strlen(src)];   /* 少 1 字节，无法存 '\0' */
```

### 5.2.4 strcmp — NULL 崩溃

```c
strcmp(NULL, "test");    /* 内部解引用 NULL → 崩溃 */
```

调用前判空，或封装安全比较。
