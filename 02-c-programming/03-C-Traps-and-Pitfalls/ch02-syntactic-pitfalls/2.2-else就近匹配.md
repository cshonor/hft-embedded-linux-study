# 2.2 else 就近匹配（悬垂 else）

## 本节讲什么

`else` **永远绑定离它最近、尚未配对的 `if`**，与 **缩进、注释、人类阅读意图** 无关。这是典型的 **人脑逻辑 ≠ 编译器语法** 问题。

## 要点

### 经典悬垂 else

```c
if (a > 0)
    if (b > 0)
        puts("a,b>0");
else
    puts("a<=0");   /* 绑定内层 if(b>0)，不是外层 if(a>0) */
```

等价结构：

```c
if (a > 0) {
    if (b > 0)
        puts("a,b>0");
    else
        puts("a<=0");  /* b<=0 时打印，a<=0 时什么都不做 */
}
```

### 修复

**外层加大括号**，明确作用域：

```c
if (a > 0) {
    if (b > 0)
        puts("a,b>0");
} else {
    puts("a<=0");
}
```

### 工程规范（内核 / HFT / 嵌入式）

1. **所有 `if`/`else`/`for`/`while` 一律写 `{}`**，即使只有一行
2. `else if` 链保持统一缩进，不依赖「看起来属于哪层」
3. 复杂条件提取为 `bool` 变量或函数，减少嵌套
