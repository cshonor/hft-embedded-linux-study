# 2.3 函数声明与调用

## 本节讲什么

函数名在表达式中的含义、**是否带 `()`**、以及 **是否提前声明**，都属于语法层陷阱。漏写括号会 **先调用再传参**；隐式声明会让 **类型完全错位**。

## 要点

### `func` vs `func()`

| 写法 | 含义 |
|------|------|
| `handler` | **函数地址**（函数指针值） |
| `handler()` | **调用**函数，表达式值为返回值 |

```c
signal(SIGINT, handler);    /* 正确：注册处理函数 */
signal(SIGINT, handler()); /* 错：先执行 handler，把返回值传给 signal */
```

同类：`pthread_create(..., thread_func, ...)` 传 **函数名**，不是 `thread_func()`。

### 隐式函数声明（C89 / 老代码）

C89 允许调用 **未声明** 的函数，编译器假设：

```c
/* 未声明 void foo(double x); */
foo(3.14);
/* 旧编译器假定 int foo(); 参数按旧规则传递 → double 截断/错乱 */
```

| 标准 | 规则 |
|------|------|
| **C99/C11** | 隐式 `int` 函数声明 **删除**，未声明调用 → 诊断 |
| **老内核/嵌入式** | 大量 K&R 风格残留 |

现代实践：

```c
#include "foo.h"   /* 或 static 原型在 .c 顶部 */
-Werror=implicit-function-declaration
```
