# 2.1 运算符优先级

## 本节讲什么

词法切完 token 后，**语法分析**按 **优先级与结合性** 把 token 组装成表达式树。记不全优先级、少写括号，程序 **编译通过但逻辑完全跑偏**——本章最高频坑。

## 要点

### 易错：`==` 高于 `&`

位运算 `&` `|` `^` 优先级 **低于** 关系运算 `==` `!=`：

```c
if (x & 0x01 == 1)   /* 等价 if (x & (0x01 == 1)) → if (x & 1) */
if (x & 0x02 == 2)   /* 等价 if (x & (0x02 == 2)) → if (x & 0) 永远假！ */
```

| 错误写法 | 实际解析 | 本意 |
|----------|----------|------|
| `val & mask == 1` | `val & (mask == 1)` | `(val & mask) == 1` |
| `a \| b == 0` | `a \| (b == 0)` | `(a \| b) == 0` |

### 其他常见组合

- **`&&` / `||`** 低于 `==`，高于 `=`
- **`=` 赋值** 极低：`if (a = b > 0)` → `a = (b > 0)`（[ch01 1.1](../ch01-lexical-pitfalls/1.1-等号与相等判断.md) + Expert C Halloween）
- **`.` / `->`** 高于 `*`：`p->member` 先成员后解引用

### 工程规范

> **位运算、混合比较一律手动加括号**，不靠记忆。

```c
if ((x & 0x02) == 0x02)  /* 判断 bit1 */
if ((status & MASK) != 0)
```
