# 6.3 多行宏与分号：if-else 绑定崩坏

## 本节讲什么

多行宏用 `\` 续行；调用处若加分号，或宏内自带多余分号，在 `if` / `else` 上下文中会破坏 **语句绑定**，导致逻辑错误或编译失败。

## 要点

### 经典 if-else 陷阱

```c
#define BAD() foo(); bar()

if (cond)
    BAD();
else
    baz();

/* 展开：
if (cond)
    foo(); bar();    ← else 无法匹配 bar() 前的 if
else
    baz();
*/
```

### 裸 SWAP 与 if

```c
#define SWAP(a,b) int t=a; a=b; b=t;

if (flag)
    SWAP(x,y);
/* if 只控制第一行；或 C 语法不允许 if 后直接跟声明 */
```

### 标准：`do { ... } while (0)`

```c
#define SWAP(a,b) do { int t = (a); (a) = (b); (b) = t; } while (0)
```

可安全用于：

```c
if (flag)
    SWAP(x, y);
else
    x = 0;
```

末尾分号成为 `while(0);` 的空语句，语法合法。
