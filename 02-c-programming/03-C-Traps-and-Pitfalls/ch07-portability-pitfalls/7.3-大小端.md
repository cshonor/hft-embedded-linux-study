# 7.3 大小端（字节序）

## 本节讲什么

多字节整数在内存中的 **字节排列顺序** 因 CPU 而异：x86/x86_64 多为 **小端**（低字节在低地址）；网络协议、部分 PowerPC/ARM 配置为 **大端**。直接把 `uint8_t buf[]` 强转为 `uint32_t*` 读取，跨架构数值颠倒。

## 要点

```c
uint8_t buf[4] = { 0x12, 0x34, 0x56, 0x78 };
uint32_t val = *(uint32_t *)buf;   /* 结果依赖本机字节序 */
```

### 规范

跨设备二进制协议：

```c
#include <arpa/inet.h>   /* POSIX */
uint32_t net = htonl(host);
uint32_t host = ntohl(net);
```

Windows 用 `winsock2.h` 同名函数。Wire format **永远按协议字节序** 逐字段解析，勿 `memcpy` 到结构体指望布局一致。
