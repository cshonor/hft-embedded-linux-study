# 7.1 基础整型宽度不固定

## 本节讲什么

C 标准只保证 **`char ≤ short ≤ int ≤ long ≤ long long`** 及 `sizeof(char)==1`，**不** 规定各类型固定字节数。同一份代码在 32/64 位、MCU 与服务器上，`int`/`long` 宽度可能不同，导致溢出、截断、结构体布局变化。

## 要点

| 平台 | 典型 `int` | 典型 `long` |
|------|------------|-------------|
| 32 位 x86 | 4 | 4 |
| 64 位 Linux LP64 | 4 | **8** |
| 部分 16 位 MCU | **2** | 4 |

```c
long timestamp;   /* 32 位 long 存不下 64 位时间戳 */
```

### 移植规范

跨平台固定位宽 **一律** `<stdint.h>`：

```c
#include <stdint.h>
uint8_t   u8;
int32_t   i32;
uint64_t  ts;
```

`size_t` / `ptrdiff_t` 用于大小与指针差；**不要** 假设 `int` 与指针同宽。
