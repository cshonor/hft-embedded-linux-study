# 7.6 有符号右移未标准化

## 本节讲什么

对 **负整数** 右移，标准 **未规定** 算术右移（高位补符号位）还是逻辑右移（高位补 0）。GCC/Clang 对 `int` 多为算术右移；不可移植代码依赖 `-1 >> 1 == -1` 在陌生编译器上可能失败。

## 要点

```c
int x = -1;
x >> 1;
/* GCC x86：常仍为 -1（算术右移）
   假设逻辑右移：可能变成 INT_MAX 量级正数 */
```

### 规范

需要可移植位操作时：

```c
uint32_t u = (uint32_t)(int32_t)x;
u >>= 1;
```

或显式掩码、无符号类型完成逻辑移位；**避免** 对 signed 类型做依赖符号扩展的 `>>`。
