# 7.2 char 默认 signed / unsigned 差异

## 本节讲什么

`char` 的 **符号性由实现定义**：x86 GCC 多为 `signed char`；部分 ARM 工具链默认 `unsigned char`。用裸 `char` 存二进制字节、串口数据时，跨平台比较 `c < 0`、扩展为 `int` 的结果可能相反。

## 要点

```c
char c = 0x80;
if (c < 0) { }   /* x86 signed char：真；unsigned char 平台：假 */
```

| 类型 | 0x80 作为 int 提升后（典型） |
|------|------------------------------|
| `signed char` | 负数（-128） |
| `unsigned char` | 128 |

### 规范

- 文本字符：可用 `char`
- **二进制字节**：`unsigned char` 或 `uint8_t`
- 需要负数语义：显式 `signed char`

与 [ch05 5.4](../ch05-library-functions/5.4-ctype字符分类.md) `isprint((unsigned char)ch)` 一致。
