# 7.10 预处理器平台宏差异

## 本节讲什么

识别 CPU/OS 的预定义宏 **无单一标准**；直接写死 `__x86_64__` 在 MSVC 上不存在，写 `_WIN32` 在 Linux 上无效。 scattered `#ifdef` 导致维护噩梦。

## 要点

| 编译器/平台 | 常见宏 |
|-------------|--------|
| GCC x86_64 | `__x86_64__`, `__GNUC__` |
| GCC ARM | `__arm__`, `__aarch64__` |
| MSVC | `_MSC_VER`, `_WIN64`, `_WIN32` |
| Linux | `__linux__` |
| Apple | `__APPLE__`, `__MACH__` |

### 规范

集中封装 `platform.h`：

```c
#if defined(_WIN32)
#  define MY_PLATFORM_WIN 1
#elif defined(__linux__)
#  define MY_PLATFORM_LINUX 1
#else
#  error "unsupported platform"
#endif
```

业务代码只 include `platform.h`，不直接散落架构宏。
