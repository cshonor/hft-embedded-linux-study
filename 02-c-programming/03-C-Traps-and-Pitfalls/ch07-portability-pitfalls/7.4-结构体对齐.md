# 7.4 结构体对齐与 padding

## 本节讲什么

编译器按 **对齐要求** 在结构体成员间插入填充字节；规则因 CPU（x86 宽松 vs ARM/MIPS 严格）、编译器、`#pragma pack` 而异。用结构体 **直接映射** 硬件寄存器或网络报文，换平台后偏移错位、未对齐访问崩溃。

## 要点

```c
struct Test {
    char a;
    int  b;
};
/* 常见 LP64：sizeof=8，offsetof(b)=4，a 后填充 3 字节 */
```

### 风险

- `#pragma pack(1)` 取消填充 → ARM 上 `int` 未对齐加载 **SIGBUS**
- 不同编译器 pack 默认值不同 → 二进制不兼容

### 规范

- 硬件寄存器：显式 `uint32_t` 字段 + `static_assert(offsetof(...) == 期望)`
- 网络协议：**逐字节** 解析，或 `memcpy` + `ntoh*` 逐字段
- 序列化格式文档化 padding，不依赖「自然对齐」
