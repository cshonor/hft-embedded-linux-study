# 7.5 指针宽度随架构变化

## 本节讲什么

32 位系统指针 **4 字节**，64 位 **8 字节**。用 `int` 保存指针在 32 位「碰巧能跑」，64 位 **截断高 32 位**，句柄/回调/内核 user pointer 全面失效。

## 要点

```c
void *buf = malloc(64);
int addr = (int)(uintptr_t)buf;   /* 64 位：截断，灾难 */
```

### 规范

```c
#include <stdint.h>
uintptr_t u = (uintptr_t)ptr;
void *p = (void *)u;
```

- 指针 ↔ 整数：**`uintptr_t` / `intptr_t`**
- 不要用 `unsigned long` 代替（LP64 下 long 8 字节但 Windows LLP64 下 4 字节）
