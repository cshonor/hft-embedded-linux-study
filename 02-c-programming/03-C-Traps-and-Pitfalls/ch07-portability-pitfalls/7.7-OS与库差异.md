# 7.7 信号、库函数与 OS 差异

## 本节讲什么

「C 标准库」与 **POSIX / Win32 / RTOS**  API 边界模糊。直接调用 `fork`、`mmap`、`sigaction`、路径分隔符、换行符等 **Linux 专属** 接口，代码无法编译或运行在 Windows、裸机、裁剪 libc 的嵌入式环境。

## 要点

| 能力 | Linux/POSIX | Windows | 裸机/RTOS |
|------|-------------|---------|-----------|
| 进程 `fork` | 有 | 无 | 无 |
| `mmap` | 有 | 不同 API | 无 |
| 换行 | `\n` | `\r\n` 文本模式 | 视终端 |
| 完整 libc | 通常有 | MSVC/UCRT | 常裁剪 |

### 规范

- 可移植层：仅依赖 **C 标准** + 明确抽象的平台头（`my_os.h`）
- 条件编译：`#ifdef _WIN32` / `__linux__` 隔离平台代码
- 嵌入式：确认 `malloc`/`printf` 是否链接、是否线程安全
