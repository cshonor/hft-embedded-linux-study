# 7.9 内存地址空间与 NULL

## 本节讲什么

标准规定 `NULL` 是 **空指针常量**（实现为 `(void*)0` 等）。但 **解引用 NULL 是否崩溃**、地址 0 是否可用于映射，由 **操作系统与 MMU** 决定。在「0 地址可读」的老式系统上通过的 bug，移植到 Linux/x86 才变成 SIGSEGV。

## 要点

```c
int *p = NULL;
*p = 1;   /* 现代 Linux：SIGSEGV；极少数嵌入式：可能不崩 */
```

### 规范

- **从不** 解引用 NULL；不假设「NULL 访问必崩」来做逻辑（防御性编程仍要判空）
- 内核/驱动：用户态指针与 `NULL` 检查是安全边界
- 函数指针、句柄与数据指针统一判空
