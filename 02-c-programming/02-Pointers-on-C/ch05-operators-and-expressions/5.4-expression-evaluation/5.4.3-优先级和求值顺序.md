# 5.4.3 优先级和求值顺序

## 本节讲什么

**优先级**决定谁先结合；**结合性**决定同级从左还是从右；**求值顺序**与副作用 —— **UB 根源**。

## 要点

### 优先级（高层，自高到低）

1. 后缀：`()` `[]` `.` `->` `++` `--`（后置）
2. 一元：`!` `~` `++` `--`（前置）`&` `*` `sizeof`
3. `*` `/` `%`
4. `+` `-`
5. `<<` `>>`
6. `<` `<=` `>` `>=`
7. `==` `!=`
8. `&` → `^` → `|`
9. `&&` → `||`
10. `?:`
11. 赋值 `=` `+=` …
12. 逗号 `,`

### 结合性

- 多数双目：**左结合**（`a+b+c` = `(a+b)+c`）
- 一元、赋值、`?:`：**右结合**（`a=b=c` = `a=(b=c)`）

### 工程规范

**不背表** —— 复杂式子**加括号**。

### 副作用与 UB

**序列点**示例：`;`、`&&`、`||`、`?:`、`,`、函数调用 `()` 内参数求值完成前（C99 前规则更复杂）。

```c
printf("%d", a++ + a++);  /* UB */
arr[i++] = i;             /* UB */
a = a++ + ++a;            /* UB */
```

**规范**：单表达式内对同一变量**最多一次**修改；拆成多语句。

### 短路

`&&` / `||` 左侧决定右侧是否求值 —— 安全空指针模式依赖于此。

## 实操

1. 多自增 UB，gcc vs clang 对比  
2. 移位+位运算拆 32 位为 4 字节  
3. 位掩码 set/clear/test 函数
