# 1.3 编译

## 本节讲什么

**Linux 下完整编译四阶段** + 运行、退出码、**gdb** 入门。对接 DPDK/内核构建的缩影；链接产物 **ELF** 是 ch18 的前置。

## 要点

### 四阶段（本章重中之重）

| 阶段 | 命令 | 产物 | 作用 |
|------|------|------|------|
| 1. 预处理 | `gcc -E main.c -o main.i` | `.i` | 展开 include/宏，删注释 |
| 2. 编译 | `gcc -S main.i -o main.s` | `.s` | 生成汇编 |
| 3. 汇编 | `gcc -c main.s -o main.o` | `.o` | **ELF Relocatable**，符号未解析 |
| 4. 链接 | `gcc main.o -o main` | 可执行 ELF | 解析 `printf` 等符号，合并段 |

一步到位：`gcc main.c -o main` 等价串联四步。

### 链接方式

| 方式 | 命令/特点 |
|------|-----------|
| **动态链接**（默认） | 依赖 `libc.so`，体积小，启动时加载 |
| **静态链接** | `gcc -static main.c -o main`，库代码打进二进制；HFT 离线部署、嵌入式无 libc 时常用 |

链接器工作：解析未定义符号、重定位、合并多个 `.o` 的 **text/data/bss**。

### 运行与退出码

```bash
./main
echo $?          # main 的 return 值
```

### gdb 入门

```bash
gcc -g main.c -o main
gdb ./main
```

| 命令 | 作用 |
|------|------|
| `list` | 看源码 |
| `break main` | 断点 |
| `run` | 运行 |
| `print val` | 打印变量 |
| `print &val` | 打印地址（衔接 `&`） |
| `bt` | 崩溃后看调用栈 |

`-g` 保留调试符号；排查段错误、空指针必备。

### Makefile 模板（工程习惯）

```makefile
CC = gcc
CFLAGS = -Wall -Wextra -g

main: main.o
	$(CC) -o $@ $^

main.o: main.c
	$(CC) $(CFLAGS) -c $<

clean:
	rm -f main main.o main.i main.s
```

可增目标：`prep` → `-E`，`asm` → `-S`，便于教学对比。

## 专属价值（嵌入式 + HFT + DPDK）

1. DPDK/内核模块构建本质同四阶段，看懂复杂 Makefile/meson 不迷茫  
2. 链接产出 **ELF** —— 行情程序、内核镜像、DPDK 应用同格式（**ch18**）  
3. **gdb** 是交易偶发崩溃、驱动内存损坏的核心工具  
4. **静态 vs 动态** 影响启动延迟与部署依赖  

## 实操清单

1. 分别执行 `-E/-S/-c`，查看 `.i` / `.s` / `.o`（`file main.o` 看 ELF relocatable）  
2. 对比 `main` 与 `main-static` 文件大小（`ls -lh`）  
3. gdb 断点 `main`，`print &val` 观察栈上地址  
4. scanf 去掉 `&`，gdb **`bt`** 定位  
5. 仿写循环求和，校验 `scanf` 返回值
