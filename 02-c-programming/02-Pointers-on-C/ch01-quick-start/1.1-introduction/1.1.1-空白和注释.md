# 1.1.1 空白和注释

## 本节讲什么

C **词法**忽略大部分空白；**注释**文档化代码。工业底层项目的格式与注释规范从此处开始。

## 要点

### 空白

- 空格、制表、换行在词法上等同（字符串与预处理除外）
- 自由排版，但**一致性**比风格更重要
- 内核/DPDK：统一缩进、大括号风格，便于 review 与 diff

### 注释

```c
/* 块注释：内核代码传统上仅推荐这种 */
// 行注释：C99+，用户态工程常用

/*
 * 多行说明：函数意图、不变量、返回值
 */
```

| 规范 | 说明 |
|------|------|
| 内核风格 | 倾向 **`/* */`** |
| 用户态/HFT | `//` 与 `/* */` 均可，团队统一即可 |
| 好注释 | 解释**为什么**，而非复述代码 |

### 工程要求（仓库配套）

- 统一缩进、大括号换行
- 公共 API 注释参数、返回值、错误码
- 避免无意义注释（`i++ // increment i`）
