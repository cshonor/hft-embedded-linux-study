# 1.1 简介

## 本节讲什么

全书开篇：用一个**完整但极简**的 C 程序（列宽排序示例）走通「写代码 → 编译 → 运行」全链路。本章是**铺垫章**，不深入语法，只建立**从源码到 CPU 执行**的最小认知。

## 要点

- 目标：最小可运行 C 程序 + 统一 **WSL/Linux** 编译调试流程
- 配套源码均为极简示例，聚焦**程序执行流程**，无复杂语法
- 全书后续指针、内存、结构体都建立在本章「程序如何被构建与启动」之上

## 最简骨架（仓库标准模板）

```c
#include <stdio.h>

int main(void)
{
    printf("Hello Low Latency & Kernel\n");
    return 0;
}
```

| 部分 | 底层含义（HFT/内核视角） |
|------|--------------------------|
| `#include <stdio.h>` | 预处理插入声明；实现于 **libc**，链接时解析 |
| `int main(void)` | 唯一入口；`(void)` 显式无参，工程规范 |
| `printf` | libc 封装，底层走 **write** 等 syscall |
| `return 0` | 退出码；shell **`echo $?`** 可读 |

## 本章程序结构（书中示例）

1. **main** — 控制流程，调用读写与排序
2. **read_column_numbers** — 从 stdin 读入列宽（首次引入 **`&` 取地址**）
3. **rearrange** — 对读入数据重排（熟悉 if/for）

详见各小节；完整逻辑在读完 1.1.1–1.1.5 后串联。
