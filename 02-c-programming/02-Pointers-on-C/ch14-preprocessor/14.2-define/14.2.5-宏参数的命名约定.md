# 14.2.5 宏参数的命名约定

## 本节讲什么

宏与参数的**命名规范** —— 避免与代码标识符冲突、提升可读性。

## 要点

| 约定 | 说明 |
|------|------|
| 宏名 **全大写** | `BURST_SIZE`、`RTE_LOG` |
| 参数加括号使用 | `(x)` 非 `x` |
| 避免短名与字段冲突 | 不用 `i`、`n` 作全局宏参数名 |
| 工程前缀 | `RTE_`、`PKT_` 防命名空间污染 |

```c
/* 内核/DPDK 风格 */
#define RTE_MIN(a, b) \
    __extension__ ({ \
        typeof(a) _a = (a); \
        typeof(b) _b = (b); \
        _a < _b ? _a : _b; \
    })
```

- GNU **statement expression** 可求值一次（读 DPDK 源码时会见到）

### `#undef`

- 局部取消宏定义，避免后续误展开
