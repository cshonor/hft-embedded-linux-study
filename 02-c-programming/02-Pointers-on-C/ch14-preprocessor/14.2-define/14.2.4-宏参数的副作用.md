# 14.2.4 宏参数的副作用

## 本节讲什么

宏参数含**自增、函数调用**时会被**重复展开求值** —— 线上逻辑异常主因。

## 要点

```c
#define MIN(x,y) ((x) < (y) ? (x) : (y))

int a = 1, b = 3;
MIN(a++, b);
/* 展开：((a++) < (b) ? (a++) : (b))  → a 可能自增两次 */
```

### 防御

1. **复杂逻辑改用 `static inline`**
2. 宏内参数只引用**一次**（难保证时用 inline）
3. 调用处不用带副作用表达式

### 对比

```c
min_int(a++, b);   /* a++ 只执行一次 */
```

## 实操

MIN 宏 vs inline min 自增行为对比。
