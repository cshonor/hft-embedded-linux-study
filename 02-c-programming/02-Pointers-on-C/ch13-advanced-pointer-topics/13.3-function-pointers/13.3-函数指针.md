# 13.3 函数指针

## 本节讲什么

**函数指针**：存函数入口地址，typedef 封装，数组化做**分发器** —— DPDK 驱动表、内核 **file_operations** 核心。

## 要点

### 声明与调用

```c
int add(int x, int y) { return x + y; }

int (*calc)(int, int);
calc = add;
int res = calc(3, 5);   /* 等价 add(3, 5) */
```

- 函数名在表达式中**退化为**函数指针（非调用时）

### typedef（强制规范）

```c
typedef void (*pkt_handler_t)(struct rte_mbuf *pkt);

pkt_handler_t handler = process_quote;
handler(mbuf);
```

### 函数指针数组（分发）

```c
pkt_handler_t pkt_dispatch[] = {
    process_heartbeat,
    process_quote,
    process_order,
    process_cancel,
};
pkt_dispatch[pkt->msg_type](mbuf);
```

- 替代多层 if-else → 分支预测友好（HFT）

### 内核 file_operations 风格

```c
struct file_operations fops = {
    .read  = dev_read,
    .write = dev_write,
    .ioctl = dev_ioctl,
};
```

## 陷阱

- 数组 vs 函数指针数组声明混淆 → 跳转非法地址
- 赋值**不同签名**函数 → 栈帧错乱
