# 3.2.3 声明指针

## 本节讲什么

指针声明语法与 **const** 四组合；**restrict** 在性能敏感 API 中的用法。

## 要点

### const 与指针

```c
const int a = 10;
const int *p;         /* 指向 const：*p 不可改，p 可改 */
int *const q = &x;    /* q 不可改，*q 可改 */
const int *const r;   /* 都不可改 */
```

工程：

- 只读报文：`void parse(const uint8_t *pkt, size_t len);`
- 配置表：`const struct cfg g_cfg;` → **.rodata**

### volatile

```c
volatile uint32_t *reg = (volatile uint32_t *)0xFE000000;
*reg = 1;   /* 每次必须访存 */
```

- 硬件寄存器、被 ISR 修改的标志
- 与 **memory barrier** / 原子（C11 `_Atomic`）区分

### restrict

```c
void copy(uint8_t *restrict dst, const uint8_t *restrict src, size_t n);
```

- 承诺 dst/src **不别名**；编译器可优化读写的顺序与寄存器化
