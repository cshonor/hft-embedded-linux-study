# 3.1.2 浮点类型

## 本节讲什么

**float / double / long double** 的精度与存储；HFT 价格处理的行业惯例。

## 要点

| 类型 | 典型 | 说明 |
|------|------|------|
| `float` | 32 bit | ~23 位尾数；精度有限 |
| `double` | 64 bit | ~52 位尾数；一般「双精度」默认 |
| `long double` | 扩展 | 因架构而异；缓存占用大，低延迟路径少用 |

- 字面量：`3.14`、`1e-6`（延迟统计）
- **浮点 ≠ 精确货币**：量化常用 **整型分厘**（fixed-point）存价格，避免舍入误差
- `int`→`double` 常可精确表示；大 `double`→`int` **截断/溢出** UB

## 与 HFT

- 回测/展示可用 `double`
- 撮合、风控核心路径优先 **int64 定点**

## 与后续衔接

**ch05** 浮点运算与比较陷阱（勿用 `==` 判等）。
