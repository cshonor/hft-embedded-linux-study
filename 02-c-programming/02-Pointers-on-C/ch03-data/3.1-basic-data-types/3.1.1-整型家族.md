# 3.1.1 整型家族

## 本节讲什么

按**符号 + 位宽**划分的整型；64 位 Linux 典型宽度如下（**实现定义**，须 `sizeof` 验证）。

## 要点

| 类型 | 典型位宽 | 工程场景 |
|------|----------|----------|
| `char` / `signed char` / `unsigned char` | 8 | 字节流、寄存器、ASCII |
| `short` / `unsigned short` | 16 | 协议字段、端口号 |
| `int` / `unsigned int` | 32 | 计数、索引、掩码 |
| `long` / `unsigned long` | 32 或 64 | 地址（平台相关） |
| `long long` / `unsigned long long` | 64 | 纳秒时间戳、订单 ID |

### 关键规则

1. **符号扩展**：有符号 `char` → `int` 时高位补 1；无符号补 0
2. **整数提升**：`char`/`short` 参与运算先提升为 **`int`**
3. **字面量后缀**：`10U`、`10L`、`10LL`；掩码常带 **`U`** 防溢出

### 隐式转换陷阱

```c
unsigned char a = 200;
if (a > -1)   /* a→int 200；-1 为 int；永远为真 */
```

- 表达式含 **`unsigned`** 时，常转为无符号运算；负数变超大正数

### 定长类型（推荐）

```c
#include <stdint.h>
uint8_t  byte;
uint16_t port;
uint64_t seq;
```

## 实操

`sizeof` 打印各类型；32/64 位对比。

## 与后续衔接

**ch05** 位运算；报文字段解析用定宽无符号类型。
