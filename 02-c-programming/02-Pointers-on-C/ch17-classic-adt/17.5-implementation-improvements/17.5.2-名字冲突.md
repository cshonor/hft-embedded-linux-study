# 17.5.2 名字冲突

## 本节讲什么

ADT **命名空间隔离** —— 前缀、static 内部函数、不透明类型防冲突。

## 要点

```c
/* 模块前缀 */
typedef struct hft_queue hft_queue_t;

hft_queue_t *hft_queue_create(size_t cap);
void hft_queue_destroy(hft_queue_t **q);
```

| 手段 | 说明 |
|------|------|
| **前缀** | `rte_` / `hft_` / 模块名 |
| **static** | `.c` 内辅助函数不导出 |
| **不透明指针** | 不暴露 `struct Queue` 成员 |
| **单头文件 guard** | **ch14** `#pragma once` |

- 避免全局 **`push`/`pop`/`create`** 短名  
- DPDK **rte_* **、内核 **`linux/list.h`** 命名惯例  

## 与后续衔接

大型工程多模块链接；**ch18** 符号可见性。
