# 7.6.2 可变参数的限制

## 本节讲什么

可变参的**类型安全、可移植性、默认提升**限制 —— 《C陷阱与缺陷》库函数陷阱。

## 要点

### 无类型检查

```c
log_msg("%d", 3.14);   /* 未定义：期望 int 给了 double */
```

- 格式串与实参必须人工一致

### 默认参数提升

可变参中 **`float` → `double`**，**char/short → int** 等；`va_arg` 类型须匹配**提升后**类型。

### 无法获知个数

- 靠 **format 串**（`printf`）
- 或 **显式 count 参数**（更安全）

```c
void log_n(int n, ...);   /* 仍须约定每个参数类型 */
```

### 工程规范

| 推荐 | 避免 |
|------|------|
| 固定参数 API | 业务核心路径可变参 |
| `rte_log(level, ...)` 仅日志 | 解析/交易用 `...` |
| 编译器 format 属性 `__attribute__((format(printf,...)))` | 裸 `...` 无检查 |

## 与后续衔接

**ch15**；GCC **`__printf__`** 属性启用 `-Wformat`。
