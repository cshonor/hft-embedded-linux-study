# 7.4 ADT 和黑盒

## 本节讲什么

用**函数接口**隐藏实现细节（**抽象数据类型 ADT**、**黑盒**）。DPDK/HFT 分层与 **ch17** 经典 ADT 的函数观。

## 要点

### 黑盒原则

- 调用者只依赖 **`.h` 原型** 与 documented 行为
- 内部数据结构、算法藏在 **`.c`**（`static` 函数/变量）
- 好处：替换实现、单元测试、减少耦合

### 示例结构

```c
/* queue.h — 对外 API */
typedef struct queue queue_t;
queue_t *queue_create(void);
void queue_destroy(queue_t *q);
int queue_push(queue_t *q, void *item);
void *queue_pop(queue_t *q);
```

```c
/* queue.c — 内部 struct 不暴露 */
struct queue { /* ... */ };
static void helper(void) { /* static 私有 */ }
```

### 与 DPDK/HFT

| 层 | 黑盒接口 |
|----|----------|
| 网络 | `rte_eth_rx_burst` 隐藏 PMD 细节 |
| 策略 | 策略模块只暴露 `on_tick()` |
| 风控 | `risk_check(order*)` 内部规则不可见 |
