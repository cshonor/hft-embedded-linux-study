# 8.2.1 存储顺序

## 本节讲什么

多维数组在内存中**按行平铺**（row-major），无指针间接层。

## 要点

```c
int buf[4][3];
/* 内存顺序： [0][0][0][1][0][2] [1][0]... [3][2] */
```

- 地址递增：`&buf[i][j+1]` 相邻（同行）；换行跳 **`sizeof(int[3])`**
- CPU 缓存：同行访问局部性好 —— HFT 批量数组优势
- 与「指针数组」不同：后者每元素是指针，**不一定连续**

## 与后续衔接

**ch10** 结构体数组；帧缓冲 **stride** 可能 ≥ 行宽。
