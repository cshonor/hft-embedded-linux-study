# 2.2 词法规则

## 本节讲什么

编译器**词法分析**把源码切成 **Token（记号）**。C 有五类基础记号；**最长匹配**原则决定如何切分 —— 《C陷阱与缺陷》ch01 核心。

## 要点：五类 Token

| 类别 | 例子 |
|------|------|
| **标识符** | `count`、`rte_mbuf`、`main` |
| **关键字** | `int`、`static`、`struct`、`if`（约 32 个，不可作标识符） |
| **字面量** | `123`、`0xFF`、`'A'`、`"abc"` |
| **操作符** | `+`、`*`、`&`、`->`、`==`（**ch05** 优先级） |
| **分隔符** | `()` `{}` `,` `;` `.` `#` |

### 最长匹配（贪心）

- 编译器尽量读**最长**合法 Token
- 例：**`a+++b`** → `a` `++` `+` `b`，而非 `a` `+` `++` `b`
- 宏、寄存器位操作表达式易踩坑
