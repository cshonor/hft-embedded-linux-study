# 2.2.4 标识符

## 本节讲什么

**标识符**命名规则、**关键字**、工程命名惯例；词法层区分 Token 的第一关。

## 要点

### 命名规则

- 首字符：**字母**或 **`_`**
- 后续：字母、数字、`_`
- **区分大小写**
- 不能是**关键字**（`int`、`static`、`struct` 等）

### 工程规范（内核/DPDK）

| 对象 | 惯例 |
|------|------|
| 全局变量 | `g_xxx` 或模块前缀 |
| 函数 | 蛇形 `rte_mbuf_refcnt_update` |
| 宏 | 全大写 `RTE_PKTMBUF_HEADROOM` |

### 避坑

- **单下划线 + 大写**、**双下划线开头**：保留给实现，勿用
- 与宏同名：无参数宏可「像函数」调用，难调试

### 关键字

标准 C 约 **32** 个；C99/C11 增加 `inline`、`_Bool` 等（实现相关）。不可作变量名。
