# 2.1.1 翻译

## 本节讲什么

**翻译**：源码 → 预处理 → 编译 → 汇编 → 链接 → 可执行文件。词法分析是编译器**第一步**，完全基于 ch02 词法规则。

## 要点

### 流程（与 ch01 一致）

| 阶段 | 输入 → 输出 | 关键 |
|------|-------------|------|
| 预处理 | `.c` → `.i` | `#include`、宏、删注释 |
| 编译 | `.i` → `.s` | 语法 + 语义分析，生成汇编 |
| 汇编 | `.s` → `.o` | **ELF Relocatable**，符号可能未定义 |
| 链接 | 多 `.o` → 可执行 | 解析 **extern** 符号，**单定义** |

### 翻译单元

- 一个 `.c` 经预处理后 = **一个翻译单元**
- **声明** 在本单元可见；**定义** 全局唯一（除 `static` 内部链接）
- DPDK 多模块：每驱动/库单独 `.o`，最后链接进 `librte_*` 或应用

### 链接报错与本章关系

| 报错 | 常见原因 |
|------|----------|
| `multiple definition` | 同一符号多处**定义** |
| `undefined reference` | 有调用无**定义**，或忘链接 `.o`/库 |
| 修复 | **`extern` 声明** + **一处定义**；`static` 隐藏私有符号 |
