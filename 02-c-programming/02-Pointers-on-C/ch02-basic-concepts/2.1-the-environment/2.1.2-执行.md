# 2.1.2 执行

## 本节讲什么

**执行环境**：OS 加载 ELF，划分虚拟内存段，调用 `main`，顺序执行至 `return`。底层视角衔接 **ch18**。

## 要点

### 加载与入口

1. shell / loader 调用 **`execve`** 映射 ELF
2. 内核分配虚拟地址空间：**text / data / bss / stack / heap**
3. 启动代码（crt）初始化后进入 **`main(int argc, char *argv[])`**（ch01 可写 `void`）
4. **`return n`** → 进程退出码；内核回收资源

### ELF 段与变量（预览）

| 段 | 内容 |
|----|------|
| **`.text`** | 函数机器码（只读） |
| **`.data`** | **已初始化** 全局 / 静态变量 |
| **`.bss`** | **未初始化** 全局 / 静态，加载时**清零** |
| **stack** | 局部 **auto** 变量；**不自动清零** → 脏数据风险 |
| **heap** | `malloc`/`free`（**ch11**） |

### 关键底层细节

- **`.bss` 清零**：未显式初始化的全局/static 为 0
- **栈上 auto**：未赋值则** indeterminate**；报文解析、临时缓冲必须初始化
- 查看段：`readelf -S ./a.out`

### 执行模型

- 默认**顺序执行**语句
- 控制语句（**ch04**）改变流
- `goto` 标签仅函数内（函数作用域）

## 实操

5. **`readelf -S ./a.out`**，对照全局/静态/局部变量存放

## 与后续衔接

**ch03-data** 存储类；**ch06** 指针生命周期；**ch18** 完整运行时。
