# 2.3 程序风格

## 本节讲什么

**可读、可维护、可 review** 的编码习惯。语法正确 ≠ 工程合格；内核/DPDK 风格在本章落地。

## 要点

### 命名

- 见 **2.2.4**：一致前缀、宏大写、避免遮蔽全局
- 函数名动词开头：`read_` `init_` `rte_`；布尔返回 `is_` `has_`

### 格式

- **4 空格**缩进，不混 Tab
- 大括号：K&R / Allman 择一，项目统一
- 一行一事；过长表达式换行

### 注释

- 解释**为什么**、不变量、错误码含义
- 公共 API：参数、返回值、线程安全
- 内核倾向 **`/* */`**

### 防御性习惯

- 禁止 **`if (...);`** 空 body 误用
- 局部与全局**不同名**
- 多文件：**头文件声明，单点定义**
- 栈上大数组 → 静态/堆

### Makefile / 工具

- `-Wall -Wextra` 开警告
- `clang-format` / 内核 `checkpatch` 风格（团队选用）

## 与后续衔接
