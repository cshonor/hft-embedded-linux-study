# 11.1 为什么使用动态内存分配

## 本节讲什么

**栈 / 静态段 / 堆** 三分区对比；可变长度、大容量、跨函数生命周期数据**必须上堆** —— DPDK mempool、订单池的设计出发点。

## 要点

| 区域 | 分配 | 生命周期 | 容量 | 适用 |
|------|------|----------|------|------|
| **栈** | 自动 | 块结束回收 | ~MB 级 | 小临时变量 |
| **静态 .data/.bss** | 编译期 | 进程全程 | 固定 | 全局配置 |
| **堆** | malloc/free | **手动** | GB 级 | 链表、变长 buf |

### 核心结论

- 栈上大数组 → **栈溢出**；VLA 同样危险（**ch08**）
- 动态报文、链表节点、跨函数对象 → **堆**
- DPDK **rte_mempool** 是堆思想的工业优化：预分配、固定块、无锁

### 栈 vs 堆选型（HFT）

| 优先栈 | 强制堆 |
|--------|--------|
| <256B、块内短命 | 运行时变长 |
| 无手动释放 | 跨函数存活 |
| | 单块 >1KB |

## 与后续衔接

**ch12** 堆上链表；**ch18** brk/mmap。
