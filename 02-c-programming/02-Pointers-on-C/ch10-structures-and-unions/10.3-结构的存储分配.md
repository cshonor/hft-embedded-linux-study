# 10.3 结构的存储分配

## 本节讲什么

**内存对齐、padding、sizeof** —— 网络协议解析错位、跨平台移植 bug 的**头号根源**。

## 要点

### 对齐规则（x86 Linux 典型）

1. 成员起始地址 = **自身宽度**的整数倍  
2. struct **总大小** = **最大成员宽度**的整数倍  
3. 不足处编译器插入 **padding**

```c
struct test {
    char a;   /* 1 */
    int  b;   /* 4 → a 后 3 字节 pad */
};
/* sizeof = 8，不是 5 */
```

### 对线上的致命影响

| 问题 | 后果 |
|------|------|
| 线格式无 padding | 直接 cast/赋值字段**全错** |
| 共享内存跨进程 | 32/64 位或 ARM/x86 偏移不一致 |
| 持久化二进制 | fread 进 struct 错位 |

### packed 紧凑布局（协议头）

```c
struct __attribute__((packed)) quote_hdr {
    uint32_t seq;
    uint16_t price;
    uint8_t  pkt_type;
};
```

- GCC/Clang 扩展；MSVC 用 `#pragma pack(1)`
- **风险**：非对齐访存 → ARM 异常；x86 性能下降 → HFT 热路径慎用

### 正确加载线格式

```c
struct __attribute__((packed)) quote_hdr hdr;
memcpy(&hdr, payload, sizeof hdr);
hdr.seq = rte_be_to_cpu_32(hdr.seq);
```

- 或手动偏移解析，不依赖默认 padding

## 实操

`sizeof` 对比普通 struct vs packed；观察 padding。
