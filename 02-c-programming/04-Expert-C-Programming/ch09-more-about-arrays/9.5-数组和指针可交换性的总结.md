# 9.5 数组和指针可交换性的总结

## 本节讲什么

本节是全章 **枢纽**：归纳 **数组与指针在用法上可交换** 的边界，区分易混的 **指针数组** vs **数组指针**，解析 **`main` 的 `argv`**，并介绍 **C99 柔性数组成员（flexible array member）** 与 **`char *buf` 成员** 在 **一次 `malloc` vs 两次 `malloc`**、**栈上变长结构** 上的差异与陷阱。

## 要点

### 可交换性总结表

| 语境 | 数组 | 指针 | 可交换？ |
|------|------|------|----------|
| **`a[i]` / `*(a+i)`** | 名退化后 | `T *a` | ✅ |
| **一维形参** | `int a[]` | `int *a` | ✅（仅形参） |
| **`sizeof`** | 真数组大小 | 指针宽 | ❌ |
| **`&` 取址** | `T(*)[N]` | `T**` | ❌ |
| **`extern` 声明** | `extern T a[]` | `extern T *a` | ❌ |
| **赋值** | 数组名非左值 | `p = &x` | ❌ |
| **二维传参** | `int a[][4]` | `int (*a)[4]` | ✅；**≠ `int**`** |

**一句话**：**下标与一维传参可互换；类型、链接、存储布局不可互换。**

### 指针数组 vs 数组指针（必背）

**[ch03 螺旋](../ch03-analyzing-c-declarations/)** 读法：

```c
char *str_ptr[10];   /* 指针数组：10 个 char*，各指一处 */
char (*str_arr)[20]; /* 数组指针：指向 char[20] 的单个指针 */
```

#### `char str[10][20]` vs `char *str[10]` 对照表

| 维度 | **`char str[10][20]`** 二维字符数组 | **`char *str[10]`** 指针数组 |
|------|-------------------------------------|------------------------------|
| **类型** | `char[10][20]` — 200 字节连续 | `char*[10]` — 10 个指针 |
| **`sizeof`** | `10 * 20 = 200` | `10 * sizeof(char*)`（通常 40/80） |
| **字符串存储** | **内嵌** 在每行 20 字节槽内 | 指针指向 **别处**（字面量 rodata 或堆） |
| **可写性** | `str[i][j]` 可改（栈数组） | 若指字符串字面量 → **只读**，改 `str[i][0]` **SIGSEGV** |
| **行长度** | 每行 **最多 19 字符 + `\0`**（固定 20） | 各行 **可不等长** |
| **内存布局** | 一行接一行，**cache 友好** | 指针连续，**数据可能分散** |
| **初始化** | `{"hi", "bye"}` 拷贝进槽位 | `{"hi", "bye"}` 指针指向只读区 |
| **传参** | 退化为 `char (*)[20]` | 退化为 `char **` |
| **典型用途** | 固定列宽表、嵌入式缓冲 | **`argv`**、字符串指针表 |

见 **demo02_string_arrays**：`str_buf[0][0]='H'` 合法；注释掉的对 `str_ptr[0][0]` 赋值会段错误。

```c
char str_buf[2][8] = {"hello", "c"};   /* 可写副本 */
char *str_ptr[2] = {"hello", "c"};     /* 指向字面量 */
```

### `argv`：指针数组的典型

```c
int main(int argc, char *argv[]) { ... }
/* 等价 */
int main(int argc, char **argv) { ... }
```

- **`argv`** 是 **`char *` 数组**，每个元素指向 **一个参数字符串**（长度各异）。
- **不是** `char argv[][]` — 无法声明「锯齿未知内维」的二维栈数组。
- **`argv[0]`** 通常是程序名；**`argv[argc]`** 为 **NULL**（C 标准保证）。

### C99 柔性数组成员（flexible array member）

```c
struct Msg {
    int len;
    char buf[];   /* C99：柔性成员，不计入 sizeof(struct Msg) */
};
```

**一次 `malloc` 分配头 + 载荷（连续）**：

```c
struct Msg *m = malloc(sizeof(struct Msg) + payload_len);
m->len = payload_len;
memcpy(m->buf, data, payload_len);
free(m);   /* 一次释放 */
```

| 特性 | 说明 |
|------|------|
| **`sizeof(struct Msg)`** | **不含** `buf[]`，仅 `len` 等固定成员 |
| **适用** | **堆分配**；`buf` 紧挨结构体后扩展 |
| **C99 前写法** | `char buf[1]` + `malloc(sizeof+extra-1)`（老技巧） |

见 **demo03_flex_vs_ptr**。

### 对比：`char *buf` 成员（两次 `malloc`）

```c
struct MsgPtr {
    int len;
    char *buf;
};

struct MsgPtr *m = malloc(sizeof *m);
m->buf = malloc(payload_len);
memcpy(m->buf, data, payload_len);
free(m->buf);
free(m);
```

| 维度 | **柔性 `char buf[]`** | **`char *buf`** |
|------|----------------------|-----------------|
| **分配次数** | **1 次** | **2 次** |
| **内存局部性** | 头与数据 **连续** | 头与数据 **可能分离** |
| **释放** | **1 次 `free(m)`** | 先 `free(buf)` 再 `free(m)` |
| **柔性数组栈上** | ❌ **非法/危险** — 无空间 | 指针可指栈缓冲（需管理生命周期） |

### 柔性数组陷阱

| 陷阱 | 说明 |
|------|------|
| **栈上 `struct { int n; char buf[]; } s;`** | **无扩展空间**，`buf` 不可用 → **编译错误或未定义** |
| **`struct Msg arr[10];` 数组 of flex** | C11 起 **允许** 作结构成员数组的特殊规则，但 **不能** `sizeof` 整个数组期望含 flex 空间；通常仍 **仅堆用** |
| **忘记 `+ payload_len`** | 分配不足 → 堆溢出 |
| **与 `char buf[0]` 混用** | 老代码；新代码用 **`[]`**（C99） |

```c
/* ❌ 栈上柔性 — 无意义 */
struct Msg s;   /* sizeof 只有 len */
/* s.buf[0] = 'x';  UB */

/* ✅ 堆上 */
struct Msg *m = malloc(sizeof *m + 64);
```

### 其他可交换场景速查

| API | 数组写法 | 指针写法 |
|-----|----------|----------|
| **遍历** | `for (i=0;i<n;i++) a[i]` | `for (p=a; p<a+n; p++) *p` |
| **字符串** | `char s[] = "hi"` | `char *s = "hi"`（字面量只读） |
| **返回多个值** | 通过 **数组形参** 写出 | 同左 |
