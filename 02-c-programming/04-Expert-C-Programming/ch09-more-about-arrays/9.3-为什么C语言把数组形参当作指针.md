# 9.3 为什么C语言把数组形参当作指针

## 本节讲什么

C 把 **数组形参** 当作 **指针** 传递，是 **效率与 ABI 的历史选择**，不是语法疏漏。本节说明：**一维形参** `int a[]`、`int a[10]`、`int *a` **完全等价**（仅形参列表）；**二维及以上** 必须保留 **除最外层以外的所有维度**；以及为何 **不能把 `extern` 或 `sizeof` 的形参规则** 推广到文件作用域。

## 要点

### 一维形参：三种写法等价

```c
void f(int a[]);
void f(int a[10]);   /* 10 被忽略，不写边界 */
void f(int *a);
/* 三者对编译器完全相同 */
```

**原因（书中观点）**：

1. **传参成本**：若按值复制整个数组，大数组开销不可接受。
2. **历史 ABI**：C 诞生时按 **地址** 传参；保留数组形参语法只是 **糖**，底层仍是 **`T*`**。
3. **边界丢失**：形参处的 `10` **不分配** 10 个元素，**不** 让 `sizeof(a)` 变成 40。

```c
void print(int arr[]) {
    /* sizeof(arr) == sizeof(int*)，不是数组字节数 */
    /* 函数内无法从 arr 本身得知元素个数 — 需另传 n */
}
```

### 与 ch04 / 9.2 的边界

| 规则 | 适用范围 |
|------|----------|
| **`int a[]` ≡ `int *a`** | **仅函数形参** |
| **`extern char arr[]` ≠ `extern char *arr`** | **文件作用域声明** |
| **`sizeof(a)` 为指针宽** | **形参 `a`** |
| **`sizeof(local)` 为数组总大小** | **定义数组的局部/全局** |

**切勿** 因形参等价而在头文件写 `extern int *arr` 对应 `int arr[N]` 定义。

### 二维数组形参：必须保留内维

```c
int m[2][4] = {{1,2,3,4}, {5,6,7,8}};

void ok(int arr[][4]);      /* ✅ */
void ok(int (*arr)[4]);     /* ✅ 数组指针，推荐写法 */
void bad(int **arr);        /* ❌ 类型不匹配 */
void bad(int arr[][]);      /* ❌ 非法：内维不能为空 */
```

**原因**：`m` 退化为 **`int (*)[4]`**（指向含 4 个 int 的行），不是 `int*`。编译器需要 **内维 4** 才能计算 **`arr[i]`** 的偏移：`arr + i` 跳过 **i 行**，每行 **4×sizeof(int)** 字节。

```c
/* arr[i][j] 的地址计算（行优先 row-major） */
/* base + i * COLS + j */
void print_row(int arr[][4], int row) {
    for (int j = 0; j < 4; j++)
        printf("%d ", arr[row][j]);
}
```

见 **demo01_2d_pass**：`print_row(m, 0)` 与 `int (*p)[4] = m` 的步长验证。

### 形参等价表

| 实参类型 | 匹配的形参声明 | 不匹配的形参 |
|----------|----------------|--------------|
| **`int a[10]`** | `int a[]` / `int *a` | — |
| **`int m[2][4]`** | `int m[][4]` / `int (*m)[4]` | `int **m`、`int m[][]` |
| **`char *argv[]`** | `char **argv` / `char *argv[]` | `char argv[][]` |

### 为何 C 不把二维也「全退化成指针」

若 `int m[][4]` 也变成 `int**`：

- 需要 **指针数组 + 各行堆分配** 的布局，与 **栈上连续 `int[2][4]`** 内存不同。
- **`m[i][j]`** 的两次解引用语义与 **单次偏移** 不兼容。
- Fortran 等语言按 **列优先** 或传 **描述符**；C 选择 **最小成本：传首行地址 + 内维常量**。

### 工程实践

1. **一维**：形参写 `int *a` 或 `int a[]`，**另传 `size_t n`**。
2. **二维**：形参写 `int (*a)[COLS]` 或 `int a[][COLS]`，**COLS 用宏/常量**。
3. **头文件**：函数原型与定义 **完全一致**；二维 **写上内维**。
4. **动态「二维」**：`malloc` 行指针 + 每行 `malloc`，类型才是 `int **` — 与栈二维 **不同 API**。

### 与 ch03 声明读法

```c
int (*p)[4];   /* 先 * 绑定 p → 指向 int[4] 的指针 */
int *p[4];     /* 先 [4] 绑定 → 4 个 int* */
```

**[ch03 螺旋规则](../ch03-analyzing-c-declarations/)**：从标识符出发，**`()` 与 `[]` 优先于 `*`**，是读懂形参与 **`argv`** 的关键（**9.5**）。
