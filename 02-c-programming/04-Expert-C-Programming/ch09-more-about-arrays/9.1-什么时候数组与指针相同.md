# 9.1 什么时候数组与指针相同

## 本节讲什么

**ch04** 已确立硬结论：**数组 ≠ 指针**。本章在此基础上追问：**在哪些语境下，数组与指针「表现得像一回事」？** 答案是 **数组名退化（array decay）**：多数表达式中，`T[N]` 的数组名被当作 **`T*`** 使用，与指针一样参与下标、算术、比较、传参。本节系统回顾 **退化场景**，明确 **「相同」指的是可交换的使用方式，不是类型等同**，为 **9.2–9.6** 的混淆点与多维数组打基础。

## 要点

### ch04 结论回顾（必记）

| 维度 | 数组 `T a[N]` | 指针 `T *p` |
|------|---------------|-------------|
| **类型本质** | `T[N]` — 连续 N 个 T | `T*` — 一个地址 |
| **存储** | N×`sizeof(T)` 字节 | 一个指针宽度 |
| **赋值** | **不可** `a = ...` | **可** `p = &x` |
| **`sizeof`** | `N * sizeof(T)` | `sizeof(T*)` |

详见 **[ch04 4.1](../ch04-arrays-are-not-pointers/4.1-数组并非指针.md)**、**[4.5](../ch04-arrays-are-not-pointers/4.5-数组和指针的其他区别.md)**。

### 数组名退化的四种典型场景

| # | 场景 | 示例 | 效果 |
|---|------|------|------|
| **1** | **赋给指针** | `int *p = arr;` | `arr` → `&arr[0]`，类型 `int*` |
| **2** | **函数实参** | `foo(arr);` | 形参收到 `int*`（**9.3** 详解） |
| **3** | **下标 / 指针算术** | `arr[i]`、`arr + 1` | 先 decay，再按 `T*` 运算 |
| **4** | **比较** | `if (arr == p)` | 比较首元素地址 |

```c
int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;           /* 场景 1：退化 */
printf("%d\n", arr[2]); /* 场景 3：等价 *(arr+2) */
foo(arr);               /* 场景 2：传的是地址，不是「整个数组对象」 */
```

**口诀**：除 **sizeof / &arr / 声明语境** 外，**看见数组名，先想它会不会变成指针**。

### 「相同」的精确含义

书中说的 **「数组与指针相同」** 指：

1. **下标等价**：`a[i]` ≡ `*(a + i)` ≡ `i[a]`（**ch04 4.5**、**9.4**）
2. **传参等价**：一维形参 `int a[]` ≡ `int *a`（**仅限形参**，**9.3**）
3. **遍历等价**：用 `for (p = arr; p < arr+N; p++)` 与 `for (i=0; i<N; i++) arr[i]` 可互换

**不指**：`sizeof` 相同、可赋值、跨文件 `extern` 可互换（这些正是 **9.2** 的坑）。

### 不退化的例外（与 ch04 对照）

| 语境 | 行为 | `int a[10]` 示例 |
|------|------|------------------|
| **`sizeof(a)`** | 整个数组字节数 | `40`（假设 int 4 字节） |
| **`&a`** | 类型 `int (*)[10]` | `&a+1` 跳过 **40** 字节 |
| **声明/定义行** | `a` 仍是数组类型 | `int a[10];` |
| **文件作用域 `extern`** | 必须 `extern int a[]`，**不能** `extern int *a` | **ch04 4.2–4.4** |

```c
int arr[10];
printf("%zu %zu\n", sizeof arr, sizeof &arr);
/* sizeof arr == 40；sizeof &arr == sizeof(int(*)[10]) == 指针宽度 */
```

### 一维 vs 多维的预告

- **一维**：`arr` decay 为 `int*`；`arr[i]` 直接可用。
- **多维**：`int m[2][4]` 中 **`m` decay 为 `int (*)[4]`**（指向行的指针），**不是** `int*`（**9.6**）。
- 误把二维数组当 `int**` 传参是经典面试题（**demo01_2d_pass**）。

### 与 ch03 声明螺旋的衔接

读声明时 **`[]` 与 `*`** 的绑定顺序决定类型（**[ch03 3.3](../ch03-analyzing-c-declarations/)**）：

```c
int *p[10];    /* 指针数组：10 个 int* */
int (*p)[10];  /* 数组指针：指向 int[10] */
```

**9.5** 会用 **`char str[10][20]` vs `char *str[10]`** 表把声明与内存布局对齐。
