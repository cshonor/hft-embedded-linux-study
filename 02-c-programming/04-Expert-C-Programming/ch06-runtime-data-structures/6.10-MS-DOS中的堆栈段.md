# 6.10 MS-DOS中的堆栈段

## 本节讲什么

**6.9** 的 Unix **平坦虚拟地址** 模型之前，**Intel x86 实模式 / 16 位 MS-DOS** 用 **段寄存器（CS、DS、SS、ES）** 把逻辑地址 **`段:偏移`** 转成 **20 位物理地址**。本节作 **历史对照**：**near / far**、**堆栈段 SS:SP** 与 **6.5 栈帧** 概念相通但 **硬件寻址完全不同**。

## 要点

### 段:偏移 寻址

```text
  物理地址 = (段寄存器 << 4) + 偏移
  例：SS=0x9000, SP=0x0100 → 0x9000*16+0x0100 = 0x90100
```

| 寄存器 | 典型用途（DOS 时代） |
|--------|----------------------|
| **CS:IP** | **代码** |
| **DS** 或 **SS** | **数据 / 栈**（可重叠设置） |
| **SS:SP** | **栈顶** |
| **ES** | 额外数据段（字符串操作等） |

**6.5** 的 **SP** 仍在，但 **没有 x86-64 的 RBP 链** 那么统一；**16 位** 栈帧更 ** cramped**。

### near 与 far

| 修饰 | 指针大小 | 跨段？ |
|------|----------|--------|
| **near** | 16 位偏移（同段） | 否 |
| **far** | 16 位段 + 16 位偏移 | 是 |

```c
/* 老编译器扩展（示意） */
void near func_near(void);
void far  func_far(void);
```

- **`call near`**：压 **16 位 offset**（或仅跳转同段）
- **`call far`**：压 **CS 与 IP** —— 类似 **跨代码段** 的 **RA**

与 **6.5 RA** 类比：**far call** 保存 **段 + 偏移**，返回需 **`retf`**。

### MS-DOS 内存布局（简化）

```text
  高地址  ┌─────────────┐
          │  PSP / 环境  │
          ├─────────────┤
          │  栈 (SS:SP)  │  ← 向下增长
          ├─────────────┤
          │  堆 (brksize)│  ← 向高地址（与 Unix 概念类似）
          ├─────────────┤
          │  BSS / 数据  │
          ├─────────────┤
          │  代码 (CS)   │
  低地址  └─────────────┘
```

| 对比 **6.9 Unix** | DOS |
|-------------------|-----|
| 单一平坦 **4GB/48-bit** 地址空间 | **640KB  conventional** + 扩展内存（XMS/EMS） |
| **`mmap` + MMU** | 实模式 / 286 保护模式过渡 |
| **ASLR** | 无 |

### 段寄存器与「段 overflow」

- **栈段 SS** 与其它段 **可共享基址** → 栈 **溢出** 可 **覆盖代码或数据**（与 **6.5 / demo04** 精神相同，更易发生）
- **TINY / SMALL / COMPACT / MEDIUM / LARGE / HUGE** 内存模型（老 **Turbo C / MSC**）决定 **指针默认 near/far**
