# 6.1 a.out及其传说

## 本节讲什么

Unix 传统上把编译链接产物叫 **a.out**（**assembler output**，汇编器输出），这个名字比 **ELF** 古老得多。本节追溯 **可执行文件命名** 的历史、**a.out 格式** 与 **ELF** 的关系，并说明：无论后缀是 `a.out`、`main` 还是 `kernel.elf`，本质都是 **带段表与入口地址的二进制映像**——为 **6.2–6.4** 的段布局、载入与运行时启动打名词基础。

## 要点

### a.out 这个名字从哪来

| 时代 | 习惯 | 说明 |
|------|------|------|
| 早期 Unix | `cc foo.c` → 默认 **`a.out`** | 链接器输出，**a**ssembler **out**put |
| 现代 Linux | `gcc -o myprog` | 后缀任意；`file myprog` 看 **ELF** |
| 嵌入式 | `kernel.elf`、`app.bin` | 常配合 **链接脚本**（**ch05 demo04**） |

```bash
gcc hello.c              # 仍可能生成 a.out（取决于 -o）
gcc hello.c -o hello
file a.out hello         # "ELF 64-bit LSB executable"
```

**传说**：新手以为 `a.out` 是某种神秘格式；实际上它只是 **默认文件名**，与 **ELF 内容** 无矛盾——今天 Linux 上的 `a.out` 几乎总是 **ELF**，不是 1970 年代那种原始 a.out 结构。

### 三种「可执行」概念（别混）

| 层次 | 是什么 | 谁关心 |
|------|--------|--------|
| **源文件** | `.c` | 程序员 |
| **目标 / 可执行文件** | `.o`、ELF 可执行、`.so` | 链接器、**readelf**（**5.1**, **6.11**） |
| **进程映像** | 载入内存后的 **段 + 堆 + 栈** | 内核载入器、**pmap**（**6.3**, **6.9**） |

**6.2** 讲文件里的 **段**；**6.3** 讲内核如何把 ELF **映射** 成进程内存。

### a.out 格式 vs ELF

| 维度 | 古典 a.out | 现代 ELF（Linux x86-64 默认） |
|------|------------|-------------------------------|
| 年代 | Bell Labs 早期 Unix | System V ABI、GNU 工具链 |
| 段模型 | 较简单（text / data / bss 等） | **Section** + **Program Header（PT_LOAD）** |
| 工具 | 老系统 `nm`、`size` | **readelf -l**、**objdump -x**（**6.11**） |
| 动态链接 | 后期扩展有限 | **`.dynamic`、GOT/PLT**（**5.2**） |

```bash
readelf -h a.out         # ELF 头：类型、机器、入口点
readelf -l a.out         # Program Headers：哪些段 LOAD 进内存
```

**要点**：书名 / 章节仍写 **「a.out 文件里发生了什么」** 是 **历史惯用语**——读时应理解为 **「可执行映像」**，不必纠结文件名。

### 与 ch05 的衔接

**5.1** 已在 **链接阶段** 合并 `.text`、`.data`、`.bss`（BSS 为 **NOBITS**）。**6.1–6.3** 接续 **链接之后**：磁盘上的 ELF → 进程地址空间。

```text
  .c ──ld──► ELF (a.out) ──execve/loader──► 进程内存布局
              ↑ ch05                    ↑ ch06
```

### 可执行文件里至少有什么

| 内容 | 典型 ELF 节 / 段 | 下一节 |
|------|------------------|--------|
| 机器码 | `.text` | **6.2** |
| 只读常量 | `.rodata` | **6.2** |
| 已初始化数据 | `.data` | **6.2**, **6.3** |
| 未初始化数据 | `.bss`（零初始化） | **6.2**, **6.3** |
| 入口 | **`e_entry`** → 通常 **`_start`**，**不是 `main`** | **6.4** |
