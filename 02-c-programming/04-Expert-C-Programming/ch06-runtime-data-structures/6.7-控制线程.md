# 6.7 控制线程

## 本节讲什么

**多线程** 在同一进程内 **共享** 地址空间的大部分：**`.text`、`.rodata`、`.data`、`.bss`、堆** 共用；**每个线程有独立栈**（及 **TLS** 线程局部存储）。本节说明这对 **6.6 static 局部**、**6.9 堆**、**6.5 栈帧** 的含义，以及 **竞态** 与 **栈大小** 问题。

## 要点

### 一进程多线程：谁共享、谁独立

```text
  进程地址空间
  ┌──────────────────────────────────────┐
  │ .text / .rodata     ← 所有线程共享    │
  │ .data / .bss        ← 共享（需同步）  │
  │ heap (malloc)       ← 共享（需同步）  │
  ├──────────────────────────────────────┤
  │ 线程 1 栈            ← 独立           │
  │ 线程 2 栈            ← 独立           │
  │ TLS (thread-local)   ← 每线程独立      │
  └──────────────────────────────────────┘
```

| 资源 | 共享？ | 注意 |
|------|--------|------|
| **代码 `.text`** | ✓ 只读 | 安全 |
| **全局 / static 变量** | ✓ | **必须** mutex / atomic（**6.6**） |
| **堆对象** | ✓ | **`malloc` 线程安全**（glibc），**指针本身** 要同步 |
| **栈** | ✗ 每线程一份 | **`auto` 局部** 仅本线程可见 |
| **`errno`（传统）** | 实为 **TLS** | 每线程独立值 |

### 栈：每线程独立

- **6.5** 栈帧模型 **对每个线程重复**
- 线程 A 的 **`&local`** 与线程 B **无关**
- 默认 **pthread** 栈 ~ **8MB**（可 **`pthread_attr_setstacksize`**）；**递归过深** 仍 **栈溢出**

```bash
ulimit -s        # 栈大小 KB（shell 主线程；线程库另有默认）
```

### static 局部与多线程

```c
void bump(void) {
    static int count;   /* 全进程唯一副本 */
    count++;            /* 多线程同时 ++ → 数据竞争 */
}
```

**C11 `_Thread_local`** 可把变量放到 **TLS**，每线程一份：

```c
_Thread_local int tls_var;
```

### 堆与线程

| 操作 | 说明 |
|------|------|
| 线程 1 **`malloc`**，线程 2 **`free`** | 合法但需 **约定所有权** |
| 共享指针无锁读写 | **未定义行为** |
| **内存泄漏** | 进程级；任一线程泄漏都占进程资源（**ch07 7.6**） |

### 与 6.8 setjmp 的线程注意

**POSIX**：**`longjmp`** 若跳过 **其他线程** 的栈帧 → 未定义；**`setjmp` 缓冲** 应 **仅在同线程** 使用。

### 内核线程 vs 用户线程（拓展）

| 模型 | 栈 | 切换 |
|------|-----|------|
| **1:1 pthread** | 内核可见独立栈 | 内核调度 |
| **green thread** | 用户态分配 | 库调度（少见） |

**6.13** **内核栈** 与 **用户栈** 分离——内核代码 **不用** 用户栈执行 syscall 路径。
