# 6.12 轻松一下——卡耐基-梅隆大学的编程难题

## 本节讲什么

作者用 **卡耐基-梅隆大学（CMU）** 一道 **编程谜题** 调剂：题目看似简单，却考验对 **C 语义、指针、内存** 的精确理解——与 **6.5–6.6**「你以为知道栈和数据段里有什么」形成幽默对照。本节复述 **谜题精神** 与 **教训**，不依赖标准答案代码。

## 要点

### 谜题类型（本书语境）

CMU 风格难题常问：

| 类型 | 例子方向 |
|------|----------|
| **输出是什么？** | 未定义行为、**序列点**、**static** 累加 |
| **能否编译？** | **语法 vs 扩展** |
| **最少改几行？** | 约束条件下的 hack |

**6.6 demo02** 若改成面试题：「调用 **`counter()` 三次，两列数字各是多少？」—— **`auto` 每次 1**，**`static` 递增**。

### 与运行时结构的联系

谜题常 exploiting：

```text
  你以为的模型          实际模型
  ─────────────        ─────────────
  「局部变量没了」      栈帧复用，内存仍在
  「static 每次清零」   .bss 只初始化一次
  「指针就是地址」      **6.5** 数组、**ch04** 数组≠指针
  「main 是入口」       **6.4 _start**
```

### 经典「陷阱」主题（CMU / 面试同源）

1. **`printf` 格式与类型不匹配** — 栈上读错宽度（**ch02**）
2. **`i++ + i++`** — 未定义行为
3. **返回栈上数组地址** — **6.5** 悬垂
4. **`sizeof` 与 `strlen`** — **ch04**
5. **宏多行副作用** — **ch02 2.2**

### 编程难题的「轻松」用意

Peter van der Linden 用意：

- 提醒 **C 既高效又危险**
- **运行时结构**（本章）不是 akademische 细节——答错谜题 = 不理解 **内存**
- 读 **6.11** 用工具 **验证** 猜测，而非 **凭感觉**

### 自测一道（本章版）

**问**：下列程序可能输出什么？/legal?

```c
#include <stdio.h>
void f(void) {
    static int a;
    int b;
    printf("%d %d\n", a++, b);
}
int main(void) { f(); f(); return 0; }
```

| 变量 | 行为 |
|------|------|
| **`a`** | **0, 1** 然后 **1, 2**…（**static** 在 **.bss**） |
| **`b`** | **未初始化 auto** → **未定义**（可能 0、垃圾、不变） |

—— 这正是 **6.6** + **6.5** 考点。
