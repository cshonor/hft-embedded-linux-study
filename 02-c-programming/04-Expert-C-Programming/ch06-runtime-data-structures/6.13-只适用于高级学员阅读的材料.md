# 6.13 只适用于高级学员阅读的材料

## 本节讲什么

面向已掌握 **6.2–6.11** 的读者：补充 **信号栈（signal stack）**、**alloca**、**VLA（可变长数组）** 在 **栈上的动态占用**，以及 **内核栈** 与用户栈 **分离** 的要点。内容为 **进阶**，面试深问 / 系统编程 / 内核方向可读。

## 要点

### 信号与独立信号栈

| 概念 | 说明 |
|------|------|
| **默认** | 信号 handler 在 **当前线程用户栈** 上运行 |
| **sigaltstack** | 注册 **备用栈**，深度用户栈很浅或 **stack overflow** 时 handler 仍安全 |
| **sigsetjmp / siglongjmp** | 保存 **信号掩码**（**6.8** 增强版） |

```c
#include <signal.h>
stack_t ss;
ss.ss_sp = malloc(SIGSTKSZ);
ss.ss_size = SIGSTKSZ;
ss.ss_flags = 0;
sigaltstack(&ss, NULL);
```

**注意**：handler 内只应调用 **async-signal-safe** 函数（**不是 printf**）。

### alloca：栈上「malloc」

```c
#include <alloca.h>
void foo(size_t n) {
    char *buf = alloca(n);   /* 在 *当前栈帧* 上 bump SP */
    /* 函数 return 时自动「释放」—— 无 free */
}
```

| 对比 | **alloca** | **malloc** |
|------|------------|------------|
| 位置 | **栈**（**6.5** 帧内） | **堆**（**6.9**） |
| 释放 | **return** 即失效 | **`free`** |
| 失败 | 可能 **栈溢出** | **NULL** |
| 可移植 | 谨慎（**GNU 扩展** 常见） |

**危险**：**`n` 很大** 或 **递归 alloca** → **瞬间栈耗尽**。

### VLA（C99 可变长数组，C11 可选）

```c
void bar(int n) {
    int vla[n];        /* sizeof 在运行时；占 *栈* */
    /* C11 后许多编译器仍支持；-std=c11 可能需 -VLA 或扩展 */
}
```

| 特性 | 说明 |
|------|------|
| 存储 | **栈**，随 **6.5** 帧创建/销毁 |
| **`sizeof vla`** | **整个数组**（**ch04** 不退化例外在 VLA 块内） |
| 风险 | **`n` 过大** → 栈溢出；**6.8 longjmp** 跳过销毁 → 泄漏 **无**（栈收回）但 **语义复杂** |

**Linux 内核** 禁用 VLA；用户态教学代码可用 **`malloc(n*sizeof *p)`** 替代。

### 内核栈 vs 用户栈

```text
  用户态 thread                内核态（syscall / 中断）
  ┌─────────────┐              ┌─────────────┐
  │ 用户栈       │              │ 内核栈       │  ← 每线程固定大小（如 8KB–16KB）
  │ (MB 级)      │   syscall    │ (独立映射)   │
  └─────────────┘ ──────────► └─────────────┘
```

| 要点 | 说明 |
|------|------|
| **分离** | 内核 **不** 在用户栈上执行 **内核 C 代码** |
| **大小** | 内核栈 **小** → 内核代码 **禁止大数组局部**、**无 deep recursion** |
| **copy_from_user** | 内核读用户指针需 **验证** 地址在用户映射内（**ch07**） |

**6.7** 用户线程 **1:1** 内核线程时，各有一套 **内核栈**。
