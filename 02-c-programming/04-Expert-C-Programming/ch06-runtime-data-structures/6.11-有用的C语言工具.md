# 6.11 有用的C语言工具

## 本节讲什么

**6.2–6.9** 的抽象需 **工具验证**。本节汇总读 **a.out / 进程映像** 的常用命令：**`size`、`nm`、`objdump -x`、`readelf -l`、`pmap`**，并提及 **`valgrind`** 查 **堆泄漏与越界**（配合 **demo01–04**）。

## 要点

### 工具速查表

| 工具 | 典型命令 | 看什么 |
|------|----------|--------|
| **size** | `size a.out` | **text / data / bss** 字节摘要（**5.1**, **6.2**） |
| **nm** | `nm -C a.out` | **符号**：T 代码、D 数据、B bss、U 未定义（**5.1**） |
| **objdump** | `objdump -x a.out` | ELF 头、段、符号、**入口** 一览 |
| **readelf** | `readelf -l a.out` | **Program Headers**、**PT_LOAD**（**6.3**） |
| **readelf** | `readelf -S a.out` | **Section** 表（**ch05**） |
| **pmap** | `pmap -x $$` | 运行中 **映射 + 权限 + RSS** |
| **/proc** | `cat /proc/self/maps` | 同 pmap，脚本友好 |
| **ldd** | `ldd a.out` | 动态库依赖（**5.2**） |
| **valgrind** | `valgrind --leak-check=full ./demo` | 泄漏、非法读写（**6.9**） |

### size：段大小三列

```bash
size demo01
#    text    data     bss     dec     hex filename
```

- **text**：**.text**（常含 rodata 合并视工具而定）
- **data**：**.data**
- **bss**：**.bss**（**NOBITS** 大小）

对比 **demo02** 增加 **`static_cnt`** 后 **bss** 列变化。

### nm：找 main 与 _start

```bash
nm a.out | egrep ' _start| main| static_cnt'
# 0000000000400560 T main
# 00000000004003e0 T _start
```

| 符号字母 | 含义 |
|----------|------|
| **T/t** | **.text**（全局 / static 函数） |
| **D/d** | **.data** |
| **B/b** | **.bss** |
| **R/r** | **.rodata**（视版本） |
| **U** | 未定义，需链接 |

### objdump -x：综合头信息

```bash
objdump -x a.out | head -40
# START ADDRESS 0x...  ← 入口（_start 附近）
# Program Header / Section 摘要
```

与 **`readelf -h`** 互补；**反汇编** 用 **`objdump -d`**（看 **6.5 call/ret**）。

### readelf -l：载入视角

```bash
readelf -l a.out
# LOAD  R E  → .text / rodata
# LOAD  RW   → .data + bss 零填充区
```

直接对应 **6.3 mmap** 步骤。

### pmap / maps：运行时布局

```bash
./demo01 &
PID=$!
pmap -x $PID
cat /proc/$PID/maps
```

关注：

- **`r-xp`** 可执行映射（**.text**）
- **`rw-p`** 数据、**heap**、**栈**
- **[heap]**、**[stack]** 标注（若内核提供）

验证 **6.2**：**栈地址 > 堆 > 数据 > 代码**（定性）。

### valgrind（Memcheck）

```bash
valgrind --leak-check=full --show-leak-kinds=all ./demo01
# demo01 无泄漏应 "All heap blocks were freed"
```

| 检测 | 对应节 |
|------|--------|
| **Invalid read/write** | **6.5** 越界、**6.9** use-after-free |
| **definitely lost** | **6.9** 泄漏 |
| **Stack overflow** | 大数组局部（**6.13 VLA**） |

**注意**：valgrind 较慢；也可用 **ASan**：**`gcc -fsanitize=address`**。

### 推荐 inspect 工作流（本章 demo）

```bash
cd demo
make

# 6.2 地址
./demo01

# 6.6 static
size demo02 && nm demo02 | grep static

# 6.5 栈
./demo03

# 6.5 溢出
./demo04

# 6.3 / 6.9 映射
readelf -l demo01
pmap -x $(pidof demo01)   # 需进程在跑
```

### 与 ch05 工具链合并

| 阶段 | 工具 |
|------|------|
| **链接前 .o** | **`readelf -S -r`**（**5.1**, **5.5**） |
| **链接后** | **size, nm, readelf -l**（本章） |
| **运行** | **pmap, valgrind, ldd**（**5.2**） |
