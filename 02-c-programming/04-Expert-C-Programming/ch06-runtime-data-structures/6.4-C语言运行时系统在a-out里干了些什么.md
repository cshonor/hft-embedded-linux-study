# 6.4 C语言运行时系统在a.out里干了些什么

## 本节讲什么

**`main` 不是程序入口。** 内核跳到 **`e_entry`**，通常是 **`_start`**（由 **crt0 / crt1** 等启动文件提供）。本节梳理 **C 运行时（CRT）** 在调用 **`main(argc, argv, envp)`** 之前和 **`return`/exit** 之后做了什么：**BSS 再确认**、**堆初始化**、**注册 `atexit`**、**stdio 缓冲**、调用 **`exit`  syscall**。

## 要点

### 入口点：_start 而非 main

```text
  OS loader ──► _start (crt) ──► __libc_start_main ──► main ──► exit
                    │                                      │
                    └─ 初始化 TLS、堆、stdio 等              └─ 刷新缓冲、atexit、_exit
```

| 符号 | 来源 | 作用 |
|------|------|------|
| **`_start`** | **`crt1.o` / `Scrt1.o`**（PIE） | 程序 **真正入口**；调 libc 启动 |
| **`main`** | **你的 `.c`** | C 语义入口 |
| **`exit` / `_exit`** | **libc** | 终止进程 |

```bash
readelf -h a.out | grep Entry
nm a.out | egrep ' _start| main'
# _start 在 .text；main 是后续被调用的普通函数
```

**ch05 demo04** 裸链 **`ENTRY(_start)`** 同理：无 libc 时 **`_start`** 手写汇编跳 C。

### crt0 典型职责（glibc 概念模型）

**6.3** 已由内核 **映射段并清零 bss**；CRT 仍可能：

| 步骤 | 说明 |
|------|------|
| 调整 **栈** | 对齐；处理 **auxv** |
| **TLS** 线程局部存储 | 多线程 / `errno`（**6.7**） |
| **堆初始化** | 设置 **program break** 或 mmap arena（**6.9**） |
| **C++ / ctor** | `.init_array`（若 C++） |
| 构造 **argc/argv/envp** | 传给 **`main`** |
| 调用 **`main`** | 你的代码开始 |
| **`exit(main_ret)`** | 正常返回路径 |

```c
int main(int argc, char *argv[], char *envp[])
{
    return 0;   /* 等价于 exit(0)，会跑 atexit 与 fflush */
}
```

### main 之前堆是否可用

- **静态存储期** 对象（**`.data/.bss`**）在 **`main` 前已有效**（**6.2**）
- **`malloc`** 依赖 CRT 堆初始化——**在 `main` 里调用安全**；在 **`__attribute__((constructor))`** 里需谨慎顺序
- 裸机无 libc：**`_start`** 里显式清 **bss**、设 **sp**，再调 **`main`**（**Embedded**、**ch05 demo04**）

### _start 流程（ASCII）

```text
  _start:
    xor %ebp, %ebp          # 帧指针链起点（ABI）
    调用 __libc_start_main(main, ...)
      ├─ 初始化堆 (sbrk/mmap)
      ├─ 注册 stdin/out/err
      ├─ main(argc, argv, envp)
      ├─ exit(ret)
      └─ 不会返回
```

### exit 与 return 的区别

| 路径 | 行为 |
|------|------|
| **`return 0` from main** | CRT 包装为 **`exit(0)`** |
| **`exit(0)`** | **`atexit` 处理器**、**fflush**、然后 **`_exit`** |
| **`_exit(0)`** | **立即** 内核终止，**不** 跑 atexit |

### 与 6.3 的分界

| 工作 | OS loader | CRT |
|------|-----------|-----|
| mmap **PT_LOAD** | ✓ | |
| **bss 零填充** | ✓（主要） | 可能二次保险（裸机 CRT 全包） |
| **堆 / stdio** | | ✓ |
| 调 **main** | | ✓ |

### 常见误解（面试）

1. **「程序从 main 开始」** → 错；**`_start`**
2. **「return 后回到 OS」** → 经 **`exit`** 清理
3. **「全局变量在 main 第一行才存在」** → **`.data/.bss` 在 main 前已就绪**
