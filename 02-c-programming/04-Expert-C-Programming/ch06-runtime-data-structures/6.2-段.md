# 6.2 段

## 本节讲什么

C 程序在内存里不是「一整块字节」，而是按 **段（segment / section）** 组织：**代码、只读数据、可写全局数据、未初始化数据、堆、栈** 各司其职。本节建立 **Linux 典型进程虚拟地址布局**（低地址 → 高地址），对照 **ch05** 的 ELF 节名，并用 **demo01_memory_layout** 打印各区域地址验证直觉。

## 要点

### 五大内存区域（进程视角）

典型 **Linux x86-64** 用户进程（具体地址因 **ASLR** 每次不同，**相对顺序** 稳定）：

```text
  高地址  ┌─────────────────────────┐
          │         栈 stack         │  ← 向下增长（6.5, 6.9）
          │            ↓             │
          ├─────────────────────────┤
          │      （映射区 / 空洞）    │
          ├─────────────────────────┤
          │         堆 heap          │  ← 向上增长（malloc, 6.9）
          ├─────────────────────────┤
          │   .bss  未初始化全局/静态  │
          ├─────────────────────────┤
          │   .data 已初始化全局/静态  │
          ├─────────────────────────┤
          │   .rodata 只读常量        │
          ├─────────────────────────┤
          │   .text  程序代码         │
  低地址  └─────────────────────────┘
```

| 区域 | 典型内容 | 存储期 | 谁分配 |
|------|----------|--------|--------|
| **`.text`** | 函数机器码 | 程序生命周期 | 链接器 + 载入只读映射 |
| **`.rodata`** | 字符串字面量、`const` 全局 | 程序生命周期 | 同上 |
| **`.data`** | 有初值的全局 / 文件内 `static` | 程序生命周期 | ELF 映像 **拷贝**（**6.3**） |
| **`.bss`** | 无初值的全局 / 文件内 `static` | 程序生命周期 | 载入时 **清零**（**NOBITS**） |
| **堆 heap** | `malloc` / `mmap` 块 | 直到 `free` | 运行时库 / 内核 |
| **栈 stack** | 局部变量、返回地址、调用链 | 函数调用期间 | 编译器 + CPU **rsp/rbp** |

### ELF 节与运行时区域的对应

| C 源码 | ELF 节 | 运行时 |
|--------|--------|--------|
| `int g = 1;` | **`.data`** | 可写数据段 |
| `int g;` | **`.bss`** | 可写数据段（零初始化） |
| `static int s;`（文件作用域） | **`.bss`** | 同进程内持久 |
| `void foo(void){...}` | **`.text`** | 可执行、通常只读 |
| `"hello"` | **`.rodata`** | 只读 |
| `int x;`（函数内 **auto**） | （无 ELF 节） | **栈**（**6.5**, **6.6**） |
| `static int cnt;`（函数内） | **`.bss` / `.data`** | **不在栈上**（**6.6**, **demo02**） |

### demo01 地址模式（定性）

**demo01_memory_layout** 打印：

- **`main` / 函数指针** → **`.text`** 附近（低地址区）
- **`ro_msg`** → **`.rodata`**
- **`global_init`、`static_init`** → **`.data`**
- **`global_uninit`、`static_uninit`、`file_static`** → **`.bss`**
- **`malloc` 返回** → **堆**
- **`&stack_var`** → **栈**（通常最高）

**注意**：ASLR 下绝对地址每次运行变化；比较 **相对高低** 与 **同区域聚类** 即可。

### 与 ch05 段表一致

**5.1** 已在 **`.o`** 级别区分：

```bash
readelf -S main.o | egrep '\.(text|data|bss|rodata)'
size main.o    # text / data / bss 三列摘要（6.11）
```

链接后 **可执行文件** 仍保留这些节，载入器按 **Program Header** 映射（**6.3**）。

### 堆与栈不是 ELF 里的「节」

| 对象 | 在 ELF 文件中？ | 何时出现 |
|------|-----------------|----------|
| `.text/.data/.bss` | **有**（或 bss 为 NOBITS） | 链接时确定 **大小/内容** |
| **堆** | **无** | **`_start` / crt** 初始化 brk 或 mmap  arena（**6.4**, **6.9**） |
| **栈** | **无** | 内核为进程建 **栈映射**，`rsp` 指向栈顶 |

### 权限与安全（预告）

- **`.text` / `.rodata`**：**读 + 执行 / 只读**，防止代码被改写
- **`.data` / `.bss` / 堆 / 栈**：可读写，**不可执行**（NX / W^X，**6.3**）
- 把数据当函数跳转 → **SIGSEGV**（**ch07 7.7** 总线 / 段错误）
