# 6.9 UNIX中的堆栈段

## 本节讲什么

Unix 进程里 **栈向下长、堆向上长**，中间是 **可扩展的数据/映射区**。本节讲 **栈** 的来源与 **`ulimit`**，**堆** 如何通过 **`brk/sbrk`** 与 **`mmap`** 服务 **`malloc`**，以及 **泄漏**、**free 后野指针** 与 **6.2 五区域图** 的关系。

## 要点

### 栈（stack）

| 特性 | 说明 |
|------|------|
| **增长方向** | **向低地址**（x86-64 **`sub rsp`**） |
| **内容** | **6.5** 活动记录、参数、RA、局部变量 |
| **大小** | 主线程 **`ulimit -s`**；pthread 默认栈另设 |
| **分配方式** | 编译器 **`sub rsp`**；**无** `malloc 栈` API |

```bash
ulimit -s unlimited   # 谨慎：仅调试
```

**demo03**：连续调用 **main→A→B→C**，局部变量地址 **递减**。

### 堆（heap）

| 特性 | 说明 |
|------|------|
| **增长方向** | **向高地址**（与栈 **相向**） |
| **API** | **`malloc` / `free` / `realloc`**；底层 **libc** |
| **初始化** | **6.4 crt** 后可用 |

#### brk / sbrk（传统 program break）

```text
  .data/.bss 结束 ──► program break (brk) ──► 堆在此向上扩
```

- **`sbrk(n)`**：移动 break，扩大 **连续** 堆 arena
- 小分配常从 **brk 区** 切分；大块可能 **mmap** 单独映射

#### mmap（大块与 modern allocator）

| 路径 | 典型用途 |
|------|----------|
| **brk 区** | 中小块、**glibc malloc** 主 arena |
| **匿名 mmap** | 大块（超过阈值）、**独立映射** |
| **`mmap` 文件** | 共享库、内存映射 I/O（**6.3**） |

```bash
# 观察堆映射（运行 demo01 时另开终端）
cat /proc/$(pidof demo01)/maps | grep heap
pmap -x $(pidof demo01)
```

### 栈 vs 堆 对比表

| | **栈** | **堆** |
|---|--------|--------|
| 管理 | 编译器 + CPU | **malloc** / OS |
| 速度 | 极快（指针 bump） | 较慢（分配器逻辑） |
| 大小 | 有限（MB 级） | 大（受 **vm.overcommit** 等限制） |
| 生命周期 | 函数作用域 | 直到 **`free`** |
| 碎片 | 无（LIFO 帧） | 可能 **外部碎片** |

### 内存泄漏（leak）

```c
void leak(void) {
    char *p = malloc(1024);
    /* 忘记 free(p); p 离开作用域但堆块仍占用 */
}
```

| 现象 | 工具 |
|------|------|
| 进程 **RSS** 持续涨 | **valgrind --leak-check=full**（**6.11**） |
| 仍持有指针但不再用 | 静态分析 / ASan |

**6.7**：泄漏占 **进程** 资源，影响 **所有线程**。

### free 后的野指针（wild / dangling pointer）

```c
int *p = malloc(sizeof(int));
free(p);
*p = 1;        /* 未定义：use-after-free */
printf("%d", *p);  /* 可能崩溃或读到垃圾 */
```

| 错误 | 后果 |
|------|------|
| **use-after-free** | 堆块可能已分配给他处 |
| **double free** | 堆元数据破坏，**crash** |
| **返回局部栈指针** | **栈版** 悬垂（**6.5**） |

**最佳实践**：**`free` 后置 `p = NULL`**（仅防 **误用** 一次，非万能）。

### 栈与堆「相撞」？

经典教科书图：**栈↓ 堆↑** 中间留空。现代 Linux：

- **ASLR** + **mmap 随机** 使 **栈与堆之间常有大空洞**
- 堆大块 **独立映射**，不一定紧挨 **brk**
- 仍可能 **VM 耗尽** → **`malloc` 返回 NULL**
