# 6.5 当函数被调用时发生了什么：过程活动记录

## 本节讲什么

每次函数调用，编译器在 **栈** 上构造 **活动记录（activation record）**，也称 **栈帧（stack frame）**：保存 **返回地址（RA）**、**上一帧基址（saved RBP）**、**参数**、**局部变量**。本节用 **A → B → C** 调用链（**demo03_stack_frame**）观察地址，并说明 **缓冲区溢出** 如何覆盖 **RA** 或相邻变量（**demo04_buffer_overflow**）。

## 要点

### x86-64 System V 栈帧布局（典型）

**栈向低地址增长**（**6.9**）。一次 **`call foo`** 后，**`foo` 的栈帧** 大致（简化，省略对齐）：

```text
  高地址
        ┌─────────────────────┐
        │  调用者栈帧 ...       │
        ├─────────────────────┤
        │  参数 n, ...         │  ← 可能部分在寄存器（rdi, rsi...）
        ├─────────────────────┤
        │  返回地址 RA         │  ← call 压入；ret 弹出到 RIP
        ├─────────────────────┤
        │  saved RBP (旧rbp)   │  ← push rbp; mov rbp, rsp
        ├─────────────────────┤
        │  局部变量、临时区     │  ← sub rsp, N
  低地址 └─────────────────────┘  ← 当前 RSP
```

| 成分 | 作用 |
|------|------|
| **参数** | 被调函数读取；x86-64 前 6 个整型参数常用寄存器 |
| **RA（Return Address）** | **`ret`** 跳回调用点；被篡改 → 控制流劫持 |
| **saved RBP** | 形成 **帧指针链**，便于调试器回溯 |
| **局部变量** | **`auto`** 对象；离开函数即 **逻辑失效** |

```bash
cd demo/demo03_stack_frame && make && ./demo03
# main → A → B → C：栈地址递减，体现调用深度
```

### A → B → C 调用链（demo03）

**demo03_stack_frame**：

```text
  main(&main_local)
    └─ A(&a_local)
         └─ B(&b_local)
              └─ C(&c_local)
```

- **`&main_local`** 地址 **最高**（先分配的栈帧外层）
- 进入 **C** 时 **`&c_local`** **最低**（栈顶附近）
- 印证：**后调用的函数栈帧更「低地址」**

用 GDB：

```bash
gcc -g demo03_stack_frame/main.c -o demo03
gdb -batch -ex 'break C' -ex run -ex 'bt' -ex 'info frame' ./demo03
```

### call / ret 与 RA

```text
  call foo   →  push 下一条指令地址；jmp foo
  ... foo 内 ...
  ret        →  pop 到 RIP（跳回 RA）
```

**RA 在栈上** → 经典攻击：超长 **`strcpy`** 覆盖 RA 指向 **shellcode** 或 **ROP**（现代 **NX + ASLR + canary** 缓解）。

### 缓冲区溢出（demo04）

**demo04_buffer_overflow** 故意 **`memset(buf, 'A', 16)`** 而 **`buf[8]`**，覆盖 **栈上相邻的 `auth`**：

```c
volatile int auth = 0;
char buf[8];
memset(buf, 'A', 16);   /* 越界改写 auth */
```

| 模式 | 后果 |
|------|------|
| 覆盖 **相邻局部变量** | 逻辑漏洞（本 demo） |
| 覆盖 **RA** | 可能任意代码执行 |
| **`-fstack-protector`** | 检测 canary，**abort** |

```bash
cd demo/demo04_buffer_overflow && make
./demo04                    # 默认可能 abort（保护开启）
gcc -fno-stack-protector ... # 教学对比（慎用）
```

**6.3** **W^X**：栈常 **不可执行**，故攻击常改 **RA 指向已有代码** 而非栈上 shellcode。

### 活动记录与递归

| 特性 | 说明 |
|------|------|
| 每次调用 **新栈帧** | 递归深度受 **栈大小** 限制 |
| 局部 **`auto`** | 仅在本帧有效；指针 **不可** 返回指向局部（悬垂） |
| **alloca / VLA** | 在本帧 **动态扩栈**（**6.13**） |

### 与 6.6 对比

| 存储 | 位置 | 函数返回后 |
|------|------|------------|
| **`int local;`** | **栈帧** | 内存可被复用，**不可再引用** |
| **`static int s;`** | **`.data/.bss`** | **仍有效**（**6.6**） |
