# 6.3 操作系统在a.out文件里干了些什么

## 本节讲什么

链接器生成 ELF 后，**程序还未运行**。本节描述 **内核载入器（loader）** 在 **`execve`** 路径上做什么：**解析 ELF**、**mmap 映射段**、设置 **页权限**、从文件 **拷贝 `.data`**、为 **`.bss` 分配并清零**、启用 **ASLR**，最后把控制权交给 **用户态入口 `_start`**（**6.4**）。

## 要点

### execve 简化流程

```text
  shell: execve("/bin/ls", argv, envp)
           │
           ▼
  ① 读 ELF 头，校验魔数、架构
  ② 解析 Program Headers (readelf -l)
  ③ 对每个 PT_LOAD：mmap 到虚拟地址（+ ASLR 偏移）
  ④ 按权限设置页：R / W / X
  ⑤ 从文件拷贝 p_filesz 字节到映射区（.data 等）
  ⑥ bss：p_memsz > p_filesz 部分在内存中 zero-fill
  ⑦ 建立栈、传递 argv/envp/auxv
  ⑧ 动态链接：若有 PT_INTERP，先跑 ld.so（5.2）
  ⑨ 跳转到 e_entry（通常 _start，不是 main）
```

```bash
readelf -l ./a.out
# LOAD 段：VirtAddr, FileSiz, MemSiz, Flags (R E / RW)
```

### mmap 与段映射

| Program Header 字段 | 含义 |
|---------------------|------|
| **p_vaddr** | 建议虚拟地址（PIE 下为相对基址 + 重定位） |
| **p_filesz** | 磁盘上占用字节（**.bss 在文件中常为 0**） |
| **p_memsz** | 内存中占用（≥ filesz，多出的即 **bss 零填充**） |
| **p_flags** | **PF_R / PF_W / PF_X** |

典型映射：

| 段内容 | 权限 | 说明 |
|--------|------|------|
| **`.text` + 常合并的 `.rodata`** | **R-X** 或 **R--** + **R-X** | 代码只读执行 |
| **`.data` / `.bss` 所在 LOAD** | **RW-** | 可写数据 |

```bash
pmap -x $$          # 看当前 shell 映射（6.11）
cat /proc/self/maps # 更细：权限 r-xp / rw-p
```

### 拷贝 .data 与清零 .bss

**5.1** 要点：**`.bss` 在 ELF 里常标 NOBITS**，**不占文件字节**，但 **占内存**。

```text
  文件:  [ .data 有初值字节 ][ (bss 无文件内容) ]
  内存:  [ .data 拷贝       ][ .bss 零填充到 p_memsz ]
```

示例：

```c
int g_init = 42;   /* .data：映像里带 42 */
int g_bss;         /* .bss：载入后必为 0 */
```

**demo01** 打印 `global_uninit` 所在区域即 **bss**；若载入器未清零，C 语义被破坏——故 **OS 必须** 完成 zero-fill。

### ASLR（Address Space Layout Randomization）

| 对象 | 是否随机化（典型 Linux） |
|------|---------------------------|
| 可执行 **PIE** | 基址随机 |
| **共享库** `.so` | 随机 |
| **栈** | 随机 |
| **堆** | 随机（mmap 分配时） |
| **vdso** | 固定或部分随机 |

目的：提高 **缓冲区溢出** 等攻击难度（**6.5**, **demo04** 仍可在 **相邻局部变量** 上演示逻辑错误）。

```bash
for i in 1 2 3; do ./demo01 | grep stack; done
# stack 地址常变化
```

### 页权限与 W^X

现代内核倾向 **可写页不可执行**、**可执行页不可写**：

- 栈溢出写 shellcode 到栈上 **直接执行** → 常被 **NX** 拦截
- **ROP** 等攻击转向 **已有代码片段**（超出本节）

**6.5** 的 **demo04** 演示 **覆盖相邻栈变量**；覆盖 **返回地址（RA）** 是经典攻击面，**`-fstack-protector`** 会在 RA 前放 canary。

### 动态链接载入（与 5.2 衔接）

若 ELF 含 **`PT_INTERP`**（如 `/lib64/ld-linux-x86-64.so.2`）：

1. 内核映射 **动态链接器**
2. **ld.so** 加载 **`NEEDED`** 共享库、重定位 **GOT/PLT**
3. 再跳 **`_start`**

静态链接（**`gcc -static`**）则 **无 ld.so**，**crt 与 libc** 均在同一映像。

### 与 ch05 的分工

| 阶段 | 负责 | 产物 |
|------|------|------|
| **链接 ld** | 合并 `.o`、符号解析、重定位 | 磁盘 ELF |
| **载入 execve** | 映射、权限、bss 清零 | 进程地址空间 |
| **crt _start** | 堆初始化、调 `main` | C 程序逻辑开始（**6.4**） |
