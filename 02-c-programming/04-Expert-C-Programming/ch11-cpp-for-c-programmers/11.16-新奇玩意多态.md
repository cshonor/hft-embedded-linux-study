# 11.16 新奇玩意——多态

## 本节讲什么

本书用 **「新奇玩意（a novel approach）」** 指：**把 vptr 放进对象内存布局** 这一 C++ 设计 — 对 C 程序员 **直观又陌生**。本节通过 **内存视角** 巩固 **11.14 解释**：**有/无 virtual** 时 **`sizeof` 差异**、**对象地址与 vptr 地址**、**单继承/多继承** 下 **vptr 个数**。建议配合 **demo03_c_vs_cpp_polymorph** 打印地址验证。

## 要点

### 有 virtual → 对象「开头是指针」

```cpp
class NoVirt { int x; };
class WithVirt { virtual void f(); int x; };

/* 典型 64-bit:
   sizeof(NoVirt)    == 4 (或 8 对齐)
   sizeof(WithVirt)    == 16 (vptr 8 + int 4 + padding)
*/
```

```text
WithVirt 对象 @ 0x1000:
0x1000: [ vptr = 0x4000 ]  → 指向 WithVirt vtable
0x1008: [ x = 42        ]
```

**「新奇」之处**：C struct **第一个字段通常是数据**；C++ 多态对象 **第一个字段常是 vptr** — **ABI 约定**，非标准硬性文字但 **事实 de facto**（Itanium C++ ABI）。

### 打印布局（实验思路）

```cpp
WithVirt obj;
void **slot = reinterpret_cast<void **>(&obj);
std::cout << "object: " << &obj << "\n";
std::cout << "vptr:   " << *slot << "\n";
```

**demo03_c_vs_cpp_polymorph** 对比：

```text
C:  obj->ops  在对象内或外单独存储
C++: *(void**)obj == vtable 地址
```

### 继承下的 vptr

**单继承**（常见）：

```text
Derived 对象 = [ vptr | Base 数据 | Derived 数据 ]
              一个 vptr，Derived vtable 含 Base virtual 槽
```

**多继承（11.9）**：

```text
可能:
[ vptr_B1 | B1 数据 | vptr_B2 | B2 数据 | D 数据 ]
```

**多个 vptr** — **`static_cast`/`dynamic_cast`** 可能 **调整指针**（**偏移 thunks**）。

### 无虚函数 — 与 C 布局对齐

```cpp
struct Pod { int a, b; };           /* C 兼容 */
class  PodClass { int a, b; };      /* 无 virtual → 通常同布局 */
```

**与 C 互操作**、**DPDK mbuf 扩展** 等要求 **无 vptr** — **禁止对 POD 加 virtual**。

### vtable 存放位置

- **通常在 .rodata**（只读）
- **每类一份** — 不随对象数增长
- **vptr  per object** — **N 个对象 = N 个 vptr**，**1 个 vtable**

### 性能直觉

```text
直接调用:  call FixedAddress
virtual:   mov rax, [obj]      ; vptr
           call [rax + off]    ; 间接
```

**热路径**（包转发）→ **避免 virtual** 或 **final + 去虚化**；**冷路径**（插件架构）→ **virtual 合理**。

### 与 ch06/ch07 运行时布局

- **ch06**：栈帧、**`argv`**
- **ch07**：对齐、padding — **vptr 引发对齐填充**
- **ch11**：**OOP 在对象头加 indirection**
