# 11.11 C++ 如何进行操作符重载

## 本节讲什么

C++ 允许把 **运算符**（**`+`、`-`、`<<`、`[]`** 等） **重载为用户定义类型** 的函数，使 **UDT 用法接近内置类型**。语法上 **`operator@`** 是 **特殊函数名**，参与 **普通重载规则（11.10）**；常配合 **友元（11.5）** 实现 **对称二元运算**。本书态度：**适度使用** — 算术/IO 直观；**`operator=`** 等须谨慎（自赋值、深拷贝）。

## 要点

### 成员 vs 友元形式

```cpp
class Vec2 {
public:
    Vec2(double x, double y) : x_(x), y_(y) {}
    Vec2 operator+(const Vec2 &o) const {
        return Vec2(x_ + o.x_, y_ + o.y_);
    }
    friend std::ostream &operator<<(std::ostream &os, const Vec2 &v);
private:
    double x_, y_;
};

std::ostream &operator<<(std::ostream &os, const Vec2 &v) {
    return os << '(' << v.x_ << ',' << v.y_ << ')';
}
```

**二元运算符**：

| 形式 | 左操作数 |
|------|----------|
| **成员** | **`this`**（左） |
| **友元/自由** | 任意一侧可隐式转换 |

**`a + b`** → **`a.operator+(b)`** 或 **`operator+(a, b)`**。

### 常见可重载运算符

| 类别 | 运算符 | 注意 |
|------|--------|------|
| **算术** | `+ - * /` | 返回 **新对象** 或 const |
| **复合赋值** | `+= -=` | 通常 **成员**，改 `this` |
| **比较** | `== !=` | C++20 可 **`<=>`** |
| **下标** | `[]` | 须 **一对 const/非 const** |
| **调用** | `()` | **仿函数** functor |
| **IO** | `<< >>` | 几乎总是 **友元** |

### 不可重载（语言保留）

**`::` `.` `.*` `?:` `sizeof` `typeid`** 等 — **不能** 重载。

### 与 C 的对比

| C | C++ |
|---|-----|
| **`point_add(&a, &b, &out)`** | **`c = a + b`** |
| **`strcmp(s1, s2) == 0`** | **`s1 == s2`**（**`string`**） |
| **`fprintf(f, "%d", x)`** | **`f << x`**（**11.12**） |

**可读性** 提升；**滥用**（如 **`operator+` 做字符串拼接业务逻辑**）则 **难调试**。

### 赋值与拷贝（与 11.4 呼应）

```cpp
String &operator=(const String &other) {
    if (this == &other) return *this;   /* 自赋值 */
    delete[] data_;
    /* 深拷贝 ... */
    return *this;
}
```

**Rule of Three/Five**：若自定义 **dtor / 拷贝 ctor / `operator=`** 之一，通常 **三者都要**（或 **`=default`/`=delete`**）。

### IO 运算符与 `printf`（11.12）

```cpp
std::cout << "x=" << x << '\n';
```

**类型安全** — 无需 **`%d`/`%f` 格式串** 不匹配；**性能** 上早期 C++ stream 较慢，现代已大幅改善。
