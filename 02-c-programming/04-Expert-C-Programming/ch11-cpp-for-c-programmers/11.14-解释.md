# 11.14 解释

## 本节讲什么

本节是 **11.13 多态** 的 **机制详解**：编译器如何为 **含 `virtual` 的类** 生成 **vtable**，对象如何在 **构造时** 写入 **vptr**，以及 **动态调用** 在汇编级近似 **`(*(this->vptr[n]))(this, args...)`**。把 C++ 多态 **去神秘化** — 它就是 **结构化的函数指针表**，与 **ch10 10.7**、**11.15 内核 ops** 同构，只是 **编译器代劳**。

## 要点

### vtable 与 vptr

```text
类 Circle（有 virtual draw）:

Circle vtable (只读,  per-class):
┌──────────────────┐
│ &Circle::draw    │
│ &Circle::~Circle │
└──────────────────┘

Circle 对象 (per-instance):
┌──────────┬─────────┬───────┐
│ vptr ────┼─→ vtable│ r_    │ ...
└──────────┴─────────┴───────┘
   ↑
  通常位于对象布局最前（Itanium ABI 等）
```

| 术语 | 粒度 | 存储 |
|------|------|------|
| **vtable** | **每个多态类一份** | **只读** 段（.rodata 类似） |
| **vptr** | **每个对象一份** | 对象 **首地址** 常见 |

### 动态调用步骤

```cpp
Shape *p = new Circle();
p->draw();
```

1. 加载 **`p`** 指向的对象
2. 读对象 **前若干字节** 的 **vptr**
3. 在 vtable 中 **查 `draw` 槽位**（编译期固定偏移）
4. **间接跳转** 到 **`Circle::draw`**, **`this`** = 对象地址

**与 C 对比**：

```c
ops->draw(container);   /* ops 来自对象或类描述符 */
/* C++: vptr 来自对象，vtable 来自类 */
```

### 构造 / 析构期间 vptr 变化

```text
构造 Circle:
  Shape 子对象 ctor → Shape vptr
  Circle ctor 完成 → Circle vptr

析构 Circle:
  ~Circle() 开始 → 可能仍 Circle vptr
  ~Shape()      → Shape vptr（为何 ctor 里 virtual 行为异常）
```

**理解**：**vptr 随「当前构造阶段的有效类型」** 更新 — **11.13**「ctor 中 virtual 调派生无效」的根源。

### 单继承布局（典型）

```cpp
class Base  { virtual void f(); int b; };
class Child : public Base { virtual void g(); int c; };
```

```text
Child 对象:
[ vptr | b | c ]   /* Base 子对象在前，含 vptr */
```

**多继承（11.9）**：可能 **多个 vptr** — 布局 **编译器相关**，**跨 DLL ABI 须小心**。

### 与 C 函数指针表对照

| | **C `struct ops`** | **C++ virtual** |
|--|---------------------|-----------------|
| **表位置** | 常 **全局模板** 或 **每实例 ops*** | **每类 vtable** + **每对象 vptr** |
| **注册** | **手动赋值** | **编译器自动** |
| **类型** | **`void*` / 不透明 ctx** | **`this` 强类型** |
| **可读性** | 内核源码常见 | 应用 OOP 常见 |

**Linux `file_operations`** 详见 **11.15**。

### 调试技巧

- **`sizeof(Derived)`** 含 **vptr**（通常 **+指针宽**）
- **无 virtual 的类** — **无 vptr**，与 C struct **布局更接近**
- **反汇编** 看 **`mov rax, [obj]; call [rax+offset]`** 模式
