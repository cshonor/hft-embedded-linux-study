# 11.8 继承——复用已经定义的操作

## 本节讲什么

**继承（inheritance）** 让 **派生类** 获得 **基类** 的成员与行为，在 **不修改基类源码** 的前提下 **扩展或特化**。**`public` 继承** 表达 **「is-a」**（圆是图形的特例）。C 里用 **结构体嵌入 + 手动转发** 模拟；C++ 用 **语法 + 访问控制** 固化。本节还涉及 **构造/析构顺序** 与 **函数覆盖（override）** — 多态的前奏（**11.13**）。

## 要点

### 基本继承语法

```cpp
class Shape {
public:
    void move(int dx, int dy) { x_ += dx; y_ += dy; }
    virtual void draw() { /* 默认 */ }
protected:
    int x_ = 0, y_ = 0;
};

class Circle : public Shape {
public:
    void draw() override { /* 画圆 — 11.13 需 virtual */ }
    void set_radius(int r) { r_ = r; }
private:
    int r_ = 1;
};
```

**`Circle`** 自动拥有 **`x_`, `y_`, `move`** — **代码复用**。

### 构造与析构顺序

```text
构造: Base → Derived
析构: Derived → Base
```

```cpp
Circle c;   /* 先 Shape 子对象 ctor，再 Circle 成员 */
```

**规则**：派生 ctor **必须**（直接或委托）初始化基类；基类 **无默认 ctor** 时 **显式 `Circle(int r) : Shape(), r_(r) {}`**。

### 覆盖 vs 隐藏

| 术语 | 条件 | 绑定 |
|------|------|------|
| **覆盖 override** | 基类 **`virtual`** + 签名相同 | **运行时**（**11.13**） |
| **隐藏 hide** | 派生同名但 **无 virtual** 或参数不同 | **编译期** 按静态类型 |

```cpp
Shape *p = &circle;
p->draw();   /* virtual → Circle::draw；非 virtual → Shape::draw */
```

### C 模拟继承

```c
typedef struct {
    Shape base;   /* 嵌入 — 类似继承子对象 */
    int radius;
} Circle;

void circle_draw(void *self) {
    Circle *c = (Circle *)self;
    /* ... */
}
```

Linux **`container_of`** 从 **`base`** 反推 **`Circle*`** — C 惯用法（**11.15**）。C++ **编译器自动做偏移**。

### 继承的代价

- **对象更大**：含 **基类子对象**
- **布局复杂**：多继承时 **多个基类子对象**（**11.9**）
- **耦合**：基类改 **protected** 接口影响所有派生类

**组合优于继承**：若只是 **「有一个」** 而非 **「是一个」**，用 **成员对象** 而非继承。

### 虚析构（与 11.4 呼应）

```cpp
class Base {
public:
    virtual ~Base() = default;
};
```

**多态删除** **`Base* p = new Derived; delete p;`** 必须 **virtual ~Base()**，否则 **UB/泄漏**。
