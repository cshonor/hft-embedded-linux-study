# 11.12 C++ 的输入/输出 (I/O)

## 本节讲什么

C++ **标准 IO** 基于 **`iostream`**：**`std::cin` / `std::cout` / `std::cerr`** 与 **流插入/提取运算符 `<<` / `>>`**。C 的 **`printf` / `scanf`** 仍可在 C++ 中使用（**`<cstdio>`**），但 **类型不安全**（格式串与参数不匹配 → UB）。本节对比两套 IO，并说明 **与 C 库混用** 时的 **缓冲同步** 问题。

## 要点

### 基本用法

```cpp
#include <iostream>
#include <string>

int x;
std::string name;

std::cout << "enter name and age: ";
std::cin >> name >> x;
std::cout << name << " is " << x << " years old\n";
```

| C | C++ |
|---|-----|
| **`printf("%s %d", name, x)`** | **`cout << name << ' ' << x`** |
| **`scanf("%d", &x)`** | **`cin >> x`** |
| **`fgets(buf, N, stdin)`** | **`std::getline(cin, line)`** |

### 类型安全对比

```c
/* C — 格式错 → UB */
printf("%d", 3.14);        /* 错用 %d 打印 double */
```

```cpp
/* C++ — 编译期/运行时类型匹配 */
std::cout << 3.14;         /* 自动选 overload */
```

**ch08** 软件规则：**printf 格式串是「契约」** — C++ stream **把契约交给类型系统**。

### 缓冲与混用陷阱

| 问题 | 说明 |
|------|------|
| **`printf` + `cout` 交错** | 可能 **缓冲顺序** 与预期不符 |
| **`cin >>` 后 `getline`** | 留下 **换行符** — 需 **`cin.ignore()`** |
| **同步** | **`std::ios::sync_with_stdio(false)`** 可解耦 C/C++ 缓冲（性能） |

**建议**：同一模块 **选一套** 为主；内核/嵌入式常 **只用 `printf` 风格** 或 **无 IO**。

### 文件 IO

```cpp
#include <fstream>
std::ifstream in("data.txt");
std::ofstream out("out.txt");
std::string line;
while (std::getline(in, line))
    out << line << '\n';
```

C 对应 **`fopen` / `fgets` / `fclose`** — **ch07** 文件抽象在 C++ 中可 **RAII 封装**（析构自动 **`close`**）。

### 与 STL / string（11.17）

```cpp
std::string s;
std::getline(std::cin, s);   /* 整行，含空格 */
```

替代 C **`char buf[N]` + `fgets`** — **无固定上限**（仍须防滥用内存）。

### 工程选型

| 场景 | 推荐 |
|------|------|
| 应用 / 竞赛 / 日志 | **`iostream`** 或 **fmt/spdlog** |
| 内核 / DPDK | **无 iostream**（**全局 ctor、体积**） |
| 与 C 库同链接 | **`<cstdio>`** + 注意 **`extern "C"`** |

### STL IO 与 C 库对照（节选）

见 **11.17** 完整表：**`printf` ↔ `cout`**、**`fgets` ↔ `getline`**。
