# 11.5 访问控制

## 本节讲什么

C++ 用 **`public` / `private` / `protected`** 在 **编译期** 强制 **谁能访问谁**。C 只能靠 **命名约定**（`_hidden`）或 **不透明指针** 做「软封装」；C++ 把 **接口与实现的分界** 写进语言。**`protected`** 专为 **继承（11.8）** 设计：派生类可访问基类 **`protected`** 成员，外部仍不可。本节是 **封装（11.3）** 的语法保障。

## 要点

### 三级访问

| 说明符 | 类内 | 派生类 | 外部代码 |
|--------|------|--------|----------|
| **`public`** | ✅ | ✅ | ✅ |
| **`protected`** | ✅ | ✅ | ❌ |
| **`private`** | ✅ | ❌ | ❌ |

```cpp
class Base {
protected:
    int prot_ = 0;    /* 子类可用，外部不可 */
private:
    int priv_ = 0;    /* 仅 Base 成员/友元 */
public:
    int pub_ = 0;     /* 接口 */
};

class Derived : public Base {
    void f() { prot_ = 1; /* OK */  /* priv_ = 1; // 编译错误 */ }
};
```

### 继承方式与访问级别

| 继承写法 | 基类 `public` 成员在派生类中 |
|----------|------------------------------|
| **`class D : public B`** | **`public`** |
| **`class D : protected B`** | **`protected`** |
| **`class D : private B`** | **`private`** |

**常用**：**`public` 继承** 表达 **「is-a」**；**`private` 继承** 表达 **「用实现」**（较少见）。

### `class` vs `struct` 再访

```cpp
struct S { int x; };           /* 默认 public */
class  C { int x; };           /* 默认 private */
```

与 C 互操作时常用 **`struct`** 暴露 POD 字段；业务类型用 **`class`** 默认私有。

### 友元（friend）— 有意破坏封装

```cpp
class Matrix {
    friend Matrix operator*(const Matrix &a, const Matrix &b);
private:
    double data_[4];
};
```

**友元函数/类** 可访问 **`private`** — 用于 **运算符重载（11.11）** 等需对称访问的场景。**不要滥用**。

### 与 C 模块边界的对比

| C | C++ |
|---|-----|
| **`static` 函数** — 翻译单元内 | **`private` 成员** — 类型内 |
| **不透明 `struct`** — 无完整定义 | **`class` + 前向声明** — pimpl 惯用法 |
| 头文件暴露全部字段 | **头文件只暴露 public 接口** |

**pimpl 惯用法**（降低编译依赖）：

```cpp
/* header */
class Widget {
public:
    Widget();
    ~Widget();
    void draw();
private:
    struct Impl;
    Impl *pimpl_;
};
```

### 访问控制 ≠ 内存安全

**`private`** 只防 **编译期误用**，不加密内存；**指针/布局 hack** 仍可越界（**ch07**）。内核/DPDK 更依赖 **约定 + review** 而非 **`private`**。
