# 11.7 如何调用成员函数

## 本节讲什么

**`obj.method(args)`** 在机器级近似 **`method(&obj, args)`** — 编译器隐式传入 **`this` 指针**，指向 **当前对象**。理解 **`this`** 是把 C 的 **「第一个参数 `self`/`ctx`」** 习惯 **形式化** 的关键一步；也是理解 **多态（11.13–11.16）** 中 **vptr 位置** 的基础。本节还对比 **静态成员函数**（无 `this`）与 **普通成员函数**。

## 要点

### `this` 指针

```cpp
class Counter {
public:
    void inc() { ++value_; }           /* 等价 ++this->value_ */
    int  get() const { return value_; }
private:
    int value_ = 0;
};

Counter c;
c.inc();   /* 编译器生成 Counter::inc(&c) */
```

| 概念 | 说明 |
|------|------|
| **`this` 类型** | **`T*`**（非 const 成员）或 **`const T*`**（const 成员） |
| **谁分配 `this`** | 调用方（栈/堆对象地址） |
| **C 类比** | `void counter_inc(Counter *self)` |

### 成员函数调用机制（无虚函数）

```text
对象内存布局（无 virtual）:
┌─────────────────┐
│  value_         │  ← this 指向此处
│  ...其他成员    │
└─────────────────┘

call c.inc():
  1. 取 c 的地址 → this
  2. 跳转 Counter::inc
  3. 通过 this 访问 value_
```

**与 C 函数表区别**：非虚成员 **编译期绑定** 固定地址；**`virtual`** 才走 vtable（**11.13**）。

### 静态成员 vs 实例成员

```cpp
class IdGen {
public:
    static int next() { return ++seq_; }  /* 无 this */
    int id() const { return id_; }
private:
    static int seq_;
    int id_;
};
```

| 种类 | `this` | 访问实例成员 | 调用方式 |
|------|--------|--------------|----------|
| **实例成员** | ✅ | ✅ | **`obj.f()`** |
| **静态成员** | ❌ | ❌（无实例） | **`Class::f()`** |

### `const` 成员函数

```cpp
int size() const { return len_; }   /* this 为 const T* — 不可改成员 */
```

**承诺**：对 **`const` 对象** 可调用；**不修改** 逻辑状态（**mutable** 例外，了解即可）。

### C 程序员常见迁移

| C 模式 | C++ |
|--------|-----|
| `ops->write(ctx, buf, n)` | `obj.write(buf, n)` |
| **`container_of`** 从嵌入成员找 struct | 继承 + **`this`** 自然指向完整对象 |
| 函数表 **`self` 为 `void*`** | **模板 / 虚函数** 恢复类型安全（**11.17**） |

**demo02_ref_vs_ptr**：C 用 **`int **` 让被调函数改指针；C++ 用 **`int&`** 或 **返回值** 更清晰（见 **c-to-cpp-migration-checklist.md §3**）。

### 成员函数指针（了解）

```cpp
void (Counter::*pmf)() = &Counter::inc;
(c.*pmf)();                    /* 通过对象调用 */
(Counter::*pmf)();             /* 错误 — 仍需对象 */
```

实际工程 **极少用**；**`std::function` / 虚函数** 更常见。
