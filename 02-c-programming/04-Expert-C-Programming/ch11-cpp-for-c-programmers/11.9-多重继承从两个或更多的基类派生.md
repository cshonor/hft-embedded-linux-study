# 11.9 多重继承——从两个或更多的基类派生

## 本节讲什么

C++ 允许 **从多个基类派生**：**`class D : public B1, public B2`**。这带来 **更强的复用**，也带来 **布局、歧义、菱形继承** 等复杂度 — 本书以 **坦诚警告** 的笔调介绍：**能用单继承 + 组合解决就不要多继承**。C 程序员可把它类比为 **一个 struct 里嵌多个基类子结构**，但 C++ 还要处理 **同名成员冲突** 与 **vtable 数量**。

## 要点

### 多重继承语法

```cpp
class Printable {
public:
    virtual void print() const = 0;
};

class Serializable {
public:
    virtual void serialize() const = 0;
};

class Document : public Printable, public Serializable {
public:
    void print() const override { /* ... */ }
    void serialize() const override { /* ... */ }
};
```

**对象布局**（简化）：

```text
Document 对象:
┌──────────────┬──────────────┬─────────────┐
│ Printable    │ Serializable │ Document    │
│ 子对象(vptr?)│ 子对象       │ 自有成员    │
└──────────────┴──────────────┴─────────────┘
```

**多虚基类** → 可能 **多个 vptr**（**11.16** 布局因编译器/ABI 而异）。

### 菱形继承问题

```text
        Base
       /    \
   Left    Right
       \    /
        Join
```

**`Join`** 含 **两份 `Base` 子对象** → **`Join::Base` 歧义**。

**解决**：**虚继承 `virtual public Base`**

```cpp
class Left  : virtual public Base { };
class Right : virtual public Base { };
class Join  : public Left, public Right { };
```

**代价**：**更复杂的对象布局** 与 **运行时偏移计算** — 本书观点：**复杂，且多数场景不必**。

### 名字歧义与作用域解析

```cpp
class A { public: void f(); };
class B { public: void f(); };
class C : public A, public B {
    void g() {
        A::f();   /* 显式指定 */
        B::f();
    }
};
```

**编译器不会** 自动合并同名函数。

### 与 C / 内核的对比

| C++ 多继承 | C / 内核 |
|------------|----------|
| 语言级 **is-a** 多重 | **单继承嵌入链** + **`container_of`** |
| 虚基类 | **手动** 保证唯一基类实例 |
| 多个 vptr | **一个 `ops` 表** 或 **多表**（**11.15**） |

**Linux 内核** 几乎不用 C++ 多继承；**接口类 + 组合** 或 **C 表** 更常见。

### 实用建议（Expert C 立场）

1. **默认单继承** + **成员组合**
2. **多继承** 主要用于 **Mixin 式接口类**（全 **`virtual` 纯虚**、无数据）
3. **避免** 菱形；若必须，**虚继承** 并 **文档化 ABI**
4. **DPDK/嵌入式**：**零多继承** — 可预测布局优先
