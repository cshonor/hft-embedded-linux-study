# 11.1 初识 OOP

## 本节讲什么

全书 **ch1–ch10** 在 C 层面建立了声明、链接、内存、指针与函数表等底层能力；**ch11** 在此基础上回答：**「你懂 C，C++ 难在哪？」** 本节先给出 **OOP（面向对象编程）** 的三块拼图——**抽象、封装、继承/多态**——并强调 C++ 与 C 的 **血缘关系**：C++ 早期几乎就是 **「带类的 C」**，绝大多数 C 程序可经 **C 编译器子集** 或 **`extern "C"`** 与 C++ 混编。同时预告 C++ **更严格** 的规则（**`void*`、链接、`const`** 等），这些「陷阱」在 **11.17** 集中对照。

## 要点

### OOP 三要素（本章路线图）

| 概念 | 一句话 | 对应小节 |
|------|--------|----------|
| **抽象** | 只暴露「是什么、能做什么」，隐藏实现 | **11.2** |
| **封装** | 数据 + 操作绑在同一类型里 | **11.3–11.5** |
| **继承 / 多态** | 复用已有类型；运行时选具体实现 | **11.8–11.16** |

C 程序员已有对应物：**`struct` + 函数指针表**（**10.7**, **11.15** 的 **`file_operations`**）；C++ 用 **`class` + `virtual`** 把模式 **语言化**。

### C 与 C++：兼容与差异

| 维度 | C | C++ |
|------|---|-----|
| **语法超集** | — | 几乎包含 C；**合法 C 往往是合法 C++**（少数例外） |
| **`.c` vs `.cpp`** | C 编译器 | C++ 编译器；**链接阶段 name mangling** 不同 |
| **混编** | 调用 C++ 需 C 包装 | **`extern "C"`** 导出 C 链接名（**11.17**, **demo01_extern_c**） |
| **严格性** | `void*` 隐式转 `T*` | **必须显式 cast** |
| **bool** | 用 `int` 0/1 | 内置 **`bool` / `true` / `false`** |

```cpp
/* C++ 中仍可写「C 风格」——但不推荐长期保留 */
#include <cstdio>
int main() {
    printf("still works\n");   /* C 库在 C++ 中可用 */
    return 0;
}
```

### 从 C 思维到 OOP 思维

**C 常见模式**（**ch10 10.7**）：

```c
typedef struct {
    int (*draw)(void *self);
} ShapeOps;
```

**C++ 等价方向**（细节见 **11.4, 11.13**）：

```cpp
class Shape {
public:
    virtual void draw() = 0;   /* 纯虚 → 接口类 */
    virtual ~Shape() = default;
};
```

**关键转变**：把 **`self` 指针 + 函数表** 交给编译器生成 **vtable / vptr**（**11.14–11.16**）。

### 严格性陷阱预览（C 程序员易踩）

1. **`void*`**：C 里 `int *p = malloc(...)` 常见；C++ 中 **`malloc` 返回值需 cast**（更推荐 **`new`**，**11.17**）
2. **链接**：C++ **name mangling** — 同名函数因参数不同而符号不同；与 C 库链接必须用 **`extern "C"`**
3. **`const` 全局**：C 中 **`const int x = 1`** 往往 **外部链接**；C++ 默认 **内部链接**（**11.17**）
4. **未初始化**：C++ 对 **成员、局部对象** 的初始化规则比 C **更细**（**ctor**, **11.4**）
