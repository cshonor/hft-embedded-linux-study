# 11.4 展示一些类——用户定义类型享有和预定义类型一样的权限

## 本节讲什么

C++ 的 **用户定义类型（UDT）** 与 **`int`、`double`** 一样，可以 **在栈上声明、传参、返回、放数组、作结构成员**。本节通过完整 **`class`** 示例引入 **构造函数（ctor）** 与 **析构函数（dtor）** — 对象 **创建时自动初始化、销毁时自动清理**，这是 C **`malloc`/`free` + 手动 init** 的 **RAII** 雏形。理解 ctor/dtor 是后续 **继承、虚析构、多态（11.8, 11.13）** 的前提。

## 要点

### 最小完整类示例

```cpp
class String {
public:
    String(const char *s);     /* 构造函数 */
    ~String();                 /* 析构函数 */
    size_t length() const;
private:
    char *data_;
    size_t len_;
};

String::String(const char *s) {
    len_ = std::strlen(s);
    data_ = new char[len_ + 1];
    std::strcpy(data_, s);
}

String::~String() {
    delete[] data_;            /* 与 new[] 配对 */
}
```

**栈上使用** 与内置类型无异：

```cpp
void foo() {
    String greeting("hello");  /* ctor */
    /* ... */
}                              /* dtor 自动调用 */
```

对比 C（**ch07, ch10**）：须 **`create_string` + `destroy_string`** 且易漏 **`free`**。

### ctor / dtor 规则速查

| 事件 | 触发 | 典型职责 |
|------|------|----------|
| **默认 ctor** | `T obj;` | 成员默认初始化 |
| **带参 ctor** | `T obj(args);` | 分配资源、建立不变量 |
| **拷贝 ctor** | `T b(a);` | 深拷贝（若管理资源） |
| **dtor** | 作用域结束 / `delete` | 释放资源、逆序清理 |

**口诀**：**谁 `new`，谁 `delete`**；**谁打开，谁关闭** — 放进 dtor（**demo04_new_delete**）。

### 用户定义类型「与 int 同级」的含义

```cpp
String a("hi");
String b = a;              /* 拷贝 — 需正确定义或 =delete */
String arr[3];             /* 数组 of 对象 — 每元素 ctor/dtor */
void f(String s);          /* 传值 — 拷贝 ctor */
String make();             /* 返回值 — RVO/NRVO 优化 */
```

C 里 **`struct`** 也可栈分配，但 **无自动构造析构**；C++ **保证** 对象生命周期与作用域绑定（除非 **`new`**）。

### 资源管理：从 C 到 RAII

| C（ch10） | C++ |
|-----------|-----|
| `p = malloc(...); ... free(p);` | **`new` / `delete`** 或 **`vector`** |
| 中途 `return` 易泄漏 | **dtor 必跑**（栈对象） |
| `FILE *f = fopen; fclose(f);` | 可封装为 **`File` 类** 或 **`unique_ptr` + deleter** |

**禁止混用**：**`new` 配 `free`**、**`malloc` 配 `delete`** — 未定义行为（**11.17**, **demo04_new_delete**）。

### 虚析构预告（11.13）

```cpp
class Base {
public:
    virtual ~Base() = default;   /* 多态基类必须有虚 dtor */
};
```

若基类 dtor **非 virtual**，通过 **`Base*`** **`delete` 派生对象** 会 **只调基类 dtor** → 泄漏派生资源。
