# 11.3 封装——把相关的类型、数据和函数组合在一起

## 本节讲什么

**封装（encapsulation）** 把 **数据** 与 **操作该数据的函数** 绑进 **同一个用户定义类型**（通常是 **`class`**）。C 里常见 **`struct Point { int x, y; }`** + 全局函数 **`point_move(&p, dx, dy)`**；C++ 写成 **`p.move(dx, dy)`**，编译器隐式传入 **`this`**（**11.7**）。封装 + **访问控制（11.5）** 构成 **不变量（invariant）** 的防火墙：外部无法绕过 **`push`** 直接改 **`top_`** 破坏栈语义。

## 要点

### C 风格 vs C++ 封装

| C | C++ |
|---|-----|
| `struct Account { double bal; };` | `class Account { ... };` |
| `account_deposit(&a, 100);` | `a.deposit(100);` |
| 谁都能改 `a.bal` | **`private`** 仅成员/友元可改 |

```cpp
class Account {
public:
    void deposit(double amount) {
        if (amount > 0)
            balance_ += amount;
    }
    double balance() const { return balance_; }
private:
    double balance_ = 0.0;   /* 不变量：余额非负由接口保证 */
};
```

### `class` 与 `struct` 的唯一默认差异

在 C++ 中：

| 关键字 | 默认访问 | 典型用途 |
|--------|----------|----------|
| **`struct`** | **`public`** | POD、与 C 互操作、trait 标签 |
| **`class`** | **`private`** | OOP 类型、不变量封装 |

```cpp
struct CCompat { int x; };      /* 默认 public — 类似 C struct */
class  Encapsulated { int x; }; /* 默认 private */
```

**成员布局** 与 C **`struct`** 规则兼容（无 **`virtual`** 时），便于 **ABI 互操作**（**11.17 `extern "C"`**）。

### 封装带来的工程收益

1. **局部化修改**：换内部表示（数组→`vector`）不影响调用方
2. **强制入口**：所有状态变更经 **成员函数**，便于断言/日志
3. **文档即类型**：`Account` 比「两个 double + 若干函数」更易读

### 与 ch05/ch06 的链接

- **ch05**：符号可见性 — **`static` 文件内函数** 类似 **`private` 实现细节**
- **ch06**：运行时布局 — 封装 **不消除** 内存布局，只是 **语法上不可直接访问**
- **ch10**：函数指针表 — C 驱动把「方法」放在 **`struct ops`** 外；C++ 收进 **vtable**（**11.15–11.16**）

### 封装边界：何时仍用 C 风格

| 场景 | 建议 |
|------|------|
| Linux **`file_operations`** | 保留 C 表 + **`container_of`**（**11.15**） |
| DPDK / 热路径 | 无 **`virtual`**、无堆 |
| 应用层业务对象 | **`class` 封装 + RAII** |

见 **[c-to-cpp-migration-checklist.md](./c-to-cpp-migration-checklist.md) §6–7**。
