# 11.13 多态——运行时绑定

## 本节讲什么

**多态（polymorphism）**：**同一接口**，**不同具体类型** 在 **运行时** 执行 **不同实现**。**`virtual`** 成员函数 + **基类指针/引用** 触发 **动态绑定** — 编译器生成 **vtable（虚函数表）** 与对象内 **vptr（虚表指针）**。C 程序员已在 **ch10 10.7** 与 **11.15** 见过 **函数指针表**；C++ 把表 **自动化**，代价是 **内存与间接跳转**。

## 要点

### `virtual` 基本模型

```cpp
class Shape {
public:
    virtual void draw() const { /* 默认 */ }
    virtual ~Shape() = default;
};

class Circle : public Shape {
public:
    void draw() const override { /* 画圆 */ }
};

void render(const Shape &s) {
    s.draw();   /* 运行时决定 Circle::draw 或 Shape::draw */
}

Circle c;
Shape *p = &c;
p->draw();      /* Circle::draw — 即使静态类型是 Shape* */
```

**无 `virtual`** → **编译期绑定**（**切片 slicing** 风险：派生部分被截断）。

### 编译期 vs 运行时绑定

| | **静态绑定** | **动态绑定** |
|--|--------------|--------------|
| **关键词** | 普通成员 / 重载 | **`virtual`** |
| **决定时机** | 编译期 | 运行期（**vptr 查表**） |
| **C 类比** | 直接函数调用 | **`ops->fn(ctx)`** |

### 必要条件

1. 基类函数声明 **`virtual`**
2. 通过 **指针或引用** 调用（**非 `Shape s = circle; s.draw()`** 值语义 — **切片**）
3. 多态基类 **虚析构**（**11.4, 11.8**）

### 纯虚与抽象类

```cpp
class Shape {
public:
    virtual void draw() const = 0;   /* 纯虚 — 无定义 */
    virtual ~Shape() = default;
};
/* Shape s;  // 错误 — 不能实例化抽象类 */
```

**接口类** — 类似 C 的 **只含函数指针的 ops 表**，无 **ops 指针** 的实例。

### 对象切片（slicing）

```cpp
Circle c;
Shape s = c;        /* 只拷贝 Shape 子对象 — 丢失 Circle 部分 */
s.draw();           /* Shape::draw — 非多态 */
```

**应使用** **`Shape&`** 或 **`Shape*`** 或 **`std::unique_ptr<Shape>`**。

### 性能与内核语境

| 代价 | 说明 |
|------|------|
| **空间** | 每对象 **vptr**（**11.14–11.16**） |
| **时间** | 间接调用 + **可能阻止内联** |
| **确定性** | 分支预测失败 |

**Linux 内核、DPDK** 多用 **C 函数表**（**11.15**）— **显式、可审计**。**demo03_c_vs_cpp_polymorph** 并排对比。

### ctor 与 virtual

**构造函数中调 virtual** → 绑定 **当前正在构造的类** 版本（**非最终派生**）— **设计陷阱**。
