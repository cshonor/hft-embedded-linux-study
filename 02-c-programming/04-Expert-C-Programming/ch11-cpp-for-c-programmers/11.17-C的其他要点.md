# 11.17 C++ 的其他要点

## 本节讲什么

本节是 **C 程序员 C++ 速查大全**：**`const` / `constexpr`**、**引用 vs 指针**、**`new`/`delete` vs `malloc`/`free`**、**命名空间**、**`inline` / `bool`**、**模板 vs `void*`**、**STL 与 C 库对照**、**`extern "C"` 与 name mangling**、**`const` 内部链接**、**`void*` 严格性** 等。与 **[c-to-cpp-migration-checklist.md](./c-to-cpp-migration-checklist.md)** 互为 **详略版**；实验见 **demo01–demo04**。

## 要点

### `const` / `constexpr` vs `#define`（ch02/ch08）

| C | C++ | 原因 |
|---|-----|------|
| `#define MAX 128` | `constexpr int MAX = 128;` | 类型、作用域、可调试 |
| `#define SQR(x) ((x)*(x))` | `inline` 或模板 | 无 **重复求值/优先级** UB |
| `#define PI 3.14` | `constexpr double PI = 3.14;` | 浮点类型 |

```cpp
const int *p;       /* 指向常量 — 同 C */
int *const p;       /* 指针常量 — 同 C，ch10 10.4 */
int get() const;    /* 成员函数承诺不改对象 — C 无 */
```

**全局 `const`**：

```cpp
const int x = 1;    /* C++: 默认 internal linkage */
extern const int x = 1;  /* 若他 TU 需要链接 */
```

C 中 **`const int x = 1`** 常 **外部链接** — **混编 ODR 陷阱**。

### 引用 vs 二级指针

| C | C++ |
|---|-----|
| `void f(int **pp)` 改调用方指针 | `void f(int *&p)` |
| 可能 `NULL` | **引用必须绑定对象** |
| `*(*pp) = 1` | `*p = 1` |

```cpp
void attach(std::string *&slot, std::string *heap) {
    slot = heap;   /* 改外部指针变量 — 类似 int** */
}
void increment(int &n) { ++n; }   /* 改值 — 无需解引用语法 */
```

见 **demo02_ref_vs_ptr**、**checklist §3**。

### `new` / `delete` vs `malloc` / `free`

| | **`malloc/free`** | **`new/delete`** |
|--|-------------------|------------------|
| **调用** | **`malloc(n*sizeof(T))`** | **`new T` / `new T[n]`** |
| **构造/析构** | ❌ 不调用 | ✅ **自动 ctor/dtor** |
| **失败** | **`NULL`** | **抛 **`bad_alloc`**（可 **`nothrow`**） |
| **配对** | **`free`** | **`delete` / `delete[]`** |
| **混用** | — | **❌ UB** |

```cpp
int *a = new int(42);
delete a;

int *arr = new int[10];
delete[] arr;   /* 必须 [] 配对数组 new */
```

**现代 C++**：优先 **`vector` / `unique_ptr`** — **RAII**（**11.4**）。见 **demo04_new_delete**。

### 命名空间 `namespace`

```cpp
namespace net {
    void init();
}
net::init();

using net::init;           /* 引入单个名字 */
using namespace std;       /* 慎用 — 污染全局 */
```

**解决 C 全局命名 **污染****；头文件常用 **`namespace project { ... }`**。

### `inline` 与 `bool`

```cpp
inline int max(int a, int b) { return a > b ? a : b; }

bool ok = true;
if (ok) { /* ... */ }
```

**`bool`** 非 **`int` 0/1** — 与 C99 **`_Bool`** 类似；**`<cstdbool>`** 兼容 C。

### 模板 vs `void*`（ch10 10.7）

| | **`void*`** | **模板 `template<typename T>`** |
|--|-------------|----------------------------------|
| **类型检查** | 运行时/人工 cast | **编译期** |
| **代码生成** | 一份函数 | **每类型实例化** |
| **性能** | 无法内联具体类型 | 可 **特化/内联** |
| **二进制** | 小 | 可能 **膨胀** |

```cpp
template<typename T>
T max(T a, T b) { return a > b ? a : b; }

/* 对比 C */
#define MAX(a,b) ((a)>(b)?(a):(b))   /* 宏坑 — ch02 */
void *max_void(void *a, void *b, int type_tag);  /* 易错 */
```

**内核/DPDK**：仍用 **`void*`** + 宏；**应用层**：**模板 + STL**。

### STL vs C 标准库对照表

| 需求 | **C** | **C++ STL** |
|------|-------|-------------|
| 动态数组 | `malloc` + `realloc` | **`std::vector<T>`** |
| 字符串 | `char[]` + `strcpy` | **`std::string`** |
| 关联表 | `qsort` + 手写 | **`std::map` / `unordered_map`** |
| 链表 | 手动 `struct node` | **`std::list`** |
| 栈/队列 | 数组 + 下标 | **`stack` / `queue`** |
| 排序 | **`qsort`** | **`std::sort`** |
| 查找 | **`bsearch`** | **`find` / `lower_bound`** |
| 输出 | **`printf`** | **`cout` / `format`** |
| 输入 | **`scanf` / `fgets`** | **`cin` / `getline`** |
| 内存 | **`malloc/free`** | **分配器 + 智能指针** |
| 断言 | **`assert`** | **`assert` + `<cassert>`** |

**`<cxxx>` 头**：**`#include <cstdio>`** — C 库在 **`std::`** 命名空间 **也有** 全局兼容（实现定义）。

### `extern "C"` 与 name mangling

C++ **重载** → 符号如 **`_Z3fooi`**、**`_Z3food`**；C **无 mangling** → **`foo`**。

```cpp
/* C++ 编译单元 */
extern "C" {
    void c_api_init(void);
    int  c_api_send(const char *buf, int len);
}

/* 或 */
extern "C" void c_api_init(void);
```

```cpp
/* 头文件供 C/C++ 共用 */
#ifdef __cplusplus
extern "C" {
#endif
void my_init(void);
#ifdef __cplusplus
}
#endif
```

**规则**：

- **C 调 C++**：C++ 侧 **`extern "C"`** 导出
- **C++ 调 C**：声明 **`extern "C"`** 告诉 C++ **用 C 链接名**
- **`extern "C"` 函数不能重载**

见 **demo01_extern_c**、**ch05 链接**。

### `void*` 严格性（C++ 更严）

| 操作 | **C** | **C++** |
|------|-------|---------|
| **`void* → T*`** | 隐式 ✅ | **显式 cast** |
| **解引用 `*vp`** | ❌ | ❌ |
| **`vp++`** | ❌（GNU 扩展有时允许） | ❌ |
| **`malloc` → `int*`** | 隐式 | **需 cast**（用 **`new`** 更好） |

```cpp
void *vp = malloc(sizeof(int));
int *ip = static_cast<int*>(vp);   /* C++ */
free(vp);   /* 与 new 不混用 */
```

**ch10 demo04_void_ptr** 规则在 C++ 中 **更硬**。

### 快速对照：C 严格性陷阱清单

1. **`void*` 隐式转 typed pointer** — C++ 禁止
2. **`const` 全局默认链接** — C++ internal
3. **`.c` 与 `.cpp` 混链** — mangling / **`extern "C"`**
4. **`malloc`/`free` 与 `new`/`delete` 混用** — UB
5. **`typedef` 函数指针** vs **成员函数指针** — 不同 ABI
6. **未 **`virtual` 的 dtor** + **`delete` 基类指针** — 泄漏
