# 11.15 C++ 如何表现多态

## 本节讲什么

**C 没有 `virtual`**，但 **处处是多态**：**函数指针表** 让 **同一调用** 在 **运行时** 跳到 **不同驱动实现**。本书以 Linux 内核 **`struct file_operations`** 为 **教科书级范例** — 与 C++ **vtable** **同构**。理解 C 表驱动 = 理解 **为何 C++ 要 vptr**；也理解 **内核为何坚持 C**（**显式、零隐藏成本**）。

## 要点

### `file_operations` 模式

```c
struct file_operations {
    struct module *owner;
    ssize_t (*read)(struct file *, char __user *, size_t, loff_t *);
    ssize_t (*write)(struct file *, const char __user *, size_t, loff_t *);
    int (*open)(struct inode *, struct file *);
    int (*release)(struct inode *, struct file *);
    /* ... */
};

struct file {
    struct file_operations *f_op;
    /* ... */
};

/* 调用路径 */
f->f_op->read(f, buf, count, pos);
```

| C 元素 | C++ 对应 |
|--------|----------|
| **`f_op` 指针** | **vptr**（指向 vtable） |
| **`file_operations` 表** | **vtable**（每类一份） |
| **`read` 槽位** | **virtual read()** 条目 |
| **`struct file *`** | **`this`** |
| **驱动私有 `struct`** | **派生类成员** |

### 嵌入与 `container_of`

```c
struct my_dev {
    struct cdev cdev;      /* 嵌入「基类」 */
    void *private_data;
};

/* 从 cdev 指针找回 my_dev — 内核宏 container_of */
```

**类比 C++ 继承**：**基类子对象** 在 **派生对象前缀**；C++ **编译器自动偏移**，C 用 **宏**。

### 注册 ops — 手动 vtable 绑定

```c
static const struct file_operations my_fops = {
    .owner   = THIS_MODULE,
    .read    = my_read,
    .write   = my_write,
    .open    = my_open,
    .release = my_release,
};

/* 注册 */
filp->f_op = &my_fops;
```

C++ **等价概念**：

```cpp
class MyDev : public FileOps {
public:
    ssize_t read(...) override;
};
/* vptr 在 MyDev 构造时自动 = MyDev vtable */
```

### C 表驱动 vs C++ virtual

| 维度 | **C ops 表** | **C++ virtual** |
|------|--------------|-----------------|
| **谁写表** | 程序员 **静态初始化** | 编译器 **生成** |
| **灵活性** | 可 **运行时换表** | vptr **通常固定类** |
| **类型安全** | **`void*` / 强转** | **编译期检查** |
| **体积/热路径** | **无 vptr** 若表在类级 | 每对象 **vptr** |
| **审计** | 内核 **grep fops** | 需 **反汇编/ABI** |

### 何时保留 C 表（checklist §7）

- **内核 / 驱动 / 固件**
- **禁止异常、RTTI、全局 ctor**
- **需要运行时替换 ops**（**security_module** 等）
