# 11.10 重载——作用于不同类型的同一操作具有相同的名字

## 本节讲什么

**函数重载（overloading）**：**同名函数** 因 **参数列表不同**（个数或类型）而 **共存**；编译器在 **调用点** 按 **最佳匹配** 选择版本。**C 不支持** — 须 **`print_int` / `print_double`** 或 **`void*` + 类型标签**。重载与 **name mangling** 直接相关：**符号名编码参数类型**（**11.17 `extern "C"`** 必须关闭 mangling 才能与 C 链接）。

## 要点

### 基本重载

```cpp
void print(int x)    { std::cout << x; }
void print(double x) { std::cout << x; }
void print(const char *s) { std::cout << s; }

print(42);      /* print(int) */
print(3.14);    /* print(double) */
print("hi");    /* print(const char*) */
```

**返回值类型 alone 不能** 区分重载 — **必须参数不同**。

### 重载决议（简化）

编译器按顺序 **淘汰** 不合法候选，再选 **最精确转换**：

| 优先级（高→低） | 示例 |
|-----------------|------|
| **精确匹配** | `int` → `int` |
| **提升** | `float` → `double` |
| **标准转换** | `int` → `double` |
| **用户定义转换** | 构造函数 / 转换运算符 |
| **可变参 `...`** | 最差 |

**二义性** → **编译错误**（C 里往往 **静默错调**）。

### 与 C 的对比

```c
/* C — 无重载 */
void draw_int(int x);
void draw_point(int x, int y);
/* 或 */
void draw(void *obj, enum type t);
```

```cpp
/* C++ */
void draw(int x);
void draw(int x, int y);
void draw(const Shape &s);
```

**类型安全**：编译期检查；C 的 **`void* + enum`** 易 **传错标签**。

### 重载 vs 覆盖（易混）

| | **重载 overload** | **覆盖 override** |
|--|-------------------|-------------------|
| **范围** | **同一作用域** 多个函数 | **基类/派生类** 同名 **`virtual`** |
| **绑定** | **编译期** | **运行时**（**11.13**） |
| **参数** | **必须不同** | **通常相同** |

### 重载与链接 / `extern "C"`

```cpp
extern "C" void foo(int);   /* C 链接 — 无 mangling，不能重载 foo */
void foo(int);
void foo(double);           /* C++ mangling: _Z3fooi, _Z3food */
```

**C 可调用的接口** 只能暴露 **无重载** 的 **`extern "C"`** 包装（**demo01_extern_c**）。

### 成员函数重载

```cpp
class Buffer {
public:
    void append(char c);
    void append(const char *s);
    void append(const Buffer &other);
};
```

**`this` 不参与** 重载区分（所有实例成员隐含 **`this`**）。

###  pitfalls

- **`typedef`** 别名 **不** 产生新类型用于重载（**`typedef int Integer;`** 与 **`int`** 同一 overload set）
- **默认参数** 可能造成 **与另一 overload 冲突**
- 跨 **`extern "C"`** 边界 **不要** 依赖重载
