# 4.1 数组并非指针

## 本节讲什么

本章标题即结论：**数组（array）和指针（pointer）是两种不同的对象类型**，不能混为一谈。初学者常以为「数组名就是指针」，因为多数表达式里数组名会 **自动退化为（decay）** 指向首元素的指针——但这只是 **使用规则**，不改变 **类型本质**。本节建立正确心智模型，并列出 **不退化** 的例外场景（`sizeof`、`&arr`、声明语境），为 **4.2–4.4** 的跨文件崩溃案例和 **4.5** 的细节对比打基础。

## 要点

### 核心结论

| 概念 | 数组 | 指针 |
|------|------|------|
| **是什么** | 连续存储的 **同类型元素序列** | 存放 **地址** 的标量对象 |
| **声明** | `int a[10]` — a 的类型是 **int[10]** | `int *p` — p 的类型是 **int\*** |
| **内存** | 分配 **10×sizeof(int)** 字节 | 通常只分配 **一个指针宽度** |
| **能否重赋值** | **不能** `a = ...` | **能** `p = &x` |

```c
int a[10];
int *p = a;     /* 合法：a 在此语境退化为 int*，赋给 p */
/* a = p;       非法：数组名不是左值，不可赋值 */
```

### 数组名的退化（decay）

在 **大多数表达式** 中，若数组名不是下列例外之一，编译器将其 **替换** 为「指向首元素的指针」，类型由 `T[N]` 变为 `T*`：

```c
int arr[5];
int *p = arr;           /* arr → &arr[0]，类型 int* */
printf("%d\n", arr[2]); /* 先 decay 再下标，等价 *(arr+2) */
if (arr == p) { }       /* 比较的是地址 */
```

**退化不等于「数组是指针」**：退化是 **临时的类型转换规则**；对象在内存里仍是数组，声明里的类型仍是数组。

### 不退化的三种（必记）例外

| 语境 | 行为 | 示例（`int a[10]`，假设 int 4 字节） |
|------|------|--------------------------------------|
| **`sizeof(a)`** | 求 **整个数组** 字节数 | `sizeof(a) == 40` |
| **`&a`** | 类型为 **指向数组的指针** `int (*)[10]` | `&a + 1` 跳过 40 字节（见 **4.5**） |
| **声明语句中** | `a` 仍是数组名，描述类型而非取值 | `extern int a[];` 声明的是数组，不是指针 |

```c
void foo(int arr[]);   /* 形参处也「像声明」——见 4.5：等价 int *arr */
```

**口诀**：只有 **sizeof** 和 **取地址 &** 能「看见」数组的真实尺寸；其余多数场合看见的是 **首元素地址**。

### 与 `sizeof` 相关的常见误解

```c
int local[10];
void test(int arr[]) {
    sizeof(arr);  /* 指针宽度！通常 4 或 8，不是 40 */
}
```

- 在 **定义数组** 的作用域内：`sizeof(数组名)` = 元素个数 × 元素大小（**demo01_sizeof**）。
- 在 **函数形参** 里：`arr[]` 已退化为指针，`sizeof(arr)` = `sizeof(int*)` —— **无法** 用 sizeof 反推数组长度。

### 类型层面的区分（衔接 ch03）

用 **3.3 螺旋规则** 读声明，主体类型一目了然：

| 声明 | 主体 | 含义 |
|------|------|------|
| `char a[10]` | **a 是数组** | 10 个 char |
| `char *p` | **p 是指针** | 指向 char |
| `char (*pa)[10]` | **pa 是指针** | 指向含 10 个 char 的数组 |

**4.2** 的经典 bug 正是把 `char a[]` 的定义与 `extern char *a` 的声明混为一谈——编译器/linker 不报错，运行却崩溃。

### 何时「看起来一样」

| 场景 | 说明 |
|------|------|
| 下标访问 | `a[i]` 与 `*(a+i)` 在 a 退化后一致（**4.5**） |
| 传给函数 | 数组实参退化为指针（**4.5**，**ch09 9.3** 深入） |
| 赋给指针 | `p = a` 合法 |

**何时绝不一样**：跨文件 **声明/定义匹配**（**4.3–4.4**）、`sizeof`、`&` 运算、存储布局（**4.5**）。
