# 4.3 什么是声明，什么是定义

## 本节讲什么

**4.2** 的崩溃根源之一是：在 A 文件 **定义** 了数组，却在 B 文件用 **错误类型的声明** 引用它。要修复这类问题，必须先分清 C 里 **声明（declaration）** 与 **定义（definition）**——前者告诉编译器「名字与类型」，后者 **分配存储** 并通常 **初始化**。本节厘清概念、存储类说明符，以及「只声明不定义」的 `extern` 用法。

## 要点

### 声明 vs 定义

| | **声明** | **定义** |
|---|----------|----------|
| **作用** | 引入 **名字** 和 **类型**，供编译器做类型检查 | 为名字 **分配存储**（或函数体），是 **实体** |
| **出现次数** | 可 **多次**（同一作用域内类型须一致） | 整个程序中 **一次**（函数、非 extern 变量） |
| **是否占存储** | **否**（仅符号表信息） | **是** |
| **典型关键词** | `extern`（仅声明）、函数原型 | 无 `extern` 的变量；带函数体的函数 |

```c
/* 声明：告诉编译器 arr 的类型是 char 数组，不在此 TU 分配存储 */
extern char arr[];

/* 定义：分配存储并初始化 */
char arr[] = "hello";
```

### 变量：声明与定义示例

```c
extern int count;        /* 声明 */
int count = 0;           /* 定义 + 初始化 */

extern char buffer[];    /* 声明：不完整数组类型 OK */
char buffer[1024];       /* 定义：长度在此确定 */

int x;                   /* 定义：未显式初始化 → 静态存储归零 */
```

**规则**：

- 带 **`extern`** 且 **无初始化** → 通常是声明（不在此文件分配空间）。
- 无 `extern` 或有 **初始化器** → 定义。

### 函数：声明与定义

```c
/* 声明（原型） */
void print_first(void);

/* 定义 */
void print_first(void) {
    printf("...\n");
}
```

函数 **定义** 也是 **声明**（同时引入名字）。**4.2 demo** 中 `arr_def.c` 定义 `print_first`，`use_*.c` 只有原型——链接时合并。

### `extern` 的精确含义

```c
extern int g;     /* 声明：g 的定义在别的 TU */
int g = 42;       /* 定义：在此 TU 分配 g */
```

| 写法 | 含义 |
|------|------|
| `extern int g;` | 仅声明，不分配 |
| `int g;`（文件作用域） | 定义，静态存储，初值 0 |
| `extern int g = 1;` | C 中仍视为 **定义**（有初始化） |

**数组 extern**：

```c
extern char arr[];    /* OK：长度未知，类型为不完整数组 */
extern char arr[10];  /* OK：长度已知，须与定义兼容 */
```

定义侧 `char arr[] = "hello"` 时，声明侧 **不能** 写成 `extern char *arr`——类型不是同一声明链（**4.4**）。

### 声明链（declaration chain）

C 要求：同一标识符的多次声明必须 **兼容**，最终与 **唯一定义** 一致。

```text
header.h:   extern char msg[];
file_a.c:   char msg[] = "error";    /* 定义 */
file_b.c:   #include "header.h"     /* 声明 */
```

若 `file_b.c` 写 `extern char *msg`，则与定义 **不兼容**——编译器 **各自 TU 内** 可能不报错，链接也不查类型（**4.2**）。

### 不完整类型（incomplete type）

```c
extern char arr[];   /* 元素类型已知，长度未知 — 不完整数组类型 */
```

- **可以** 用于 `extern` 声明（长度由定义 TU 提供）。
- **不可以** 用于局部自动数组 `char arr[];`（缺长度且无初始化）。
- `sizeof(arr)` 在 **不完整 extern 声明** 处 **非法**（长度未知）。

### 与存储类的关系

| 存储类 | 常见用途 |
|--------|----------|
| `extern` | 引用他处定义 |
| `static`（文件作用域） | 内部链接，仅本 TU 可见 |
| （默认） | 外部链接，全局定义 |

**4.4** 强调：匹配的不只是 `extern`，而是 **完整声明符**（`*` vs `[]` 不能互换）。

### 常见误区

| 误区 | 正解 |
|------|------|
| 「头文件里写的就是定义」 | 头文件应放 **声明**；定义通常在一个 `.c` |
| 「`extern char *arr` 和 `extern char arr[]` 差不多」 | 完全不同类型，链接后运行才暴露 |
| 「声明了 `extern int n` 就不用再定义」 | 整个程序中 **必须有一处定义** |
