# 4.6 轻松一下——回文的乐趣

## 本节讲什么

本章技术密度高，作者用 **回文（palindrome）** 作 **轻松一下（Light Relief）**：正读反读相同（或忽略空格标点后相同）的短语。本节记录书中趣闻，并用 **与数组/指针相关** 的视角看「对称字符串」——如何用下标从两端向中间比较，而不修改原串。纯趣味 + 一点可运行的 C 思路，无新语法负担。

## 要点

### 什么是回文

**回文**：从左读到右与从右读到左 **相同**（常忽略大小写、空格、标点）。

| 类型 | 例子 |
|------|------|
| 单词 | **radar**、**level**、**rotor** |
| 句子 | **A man, a plan, a canal: Panama** |
| 中文 | **上海自来水来自海上**（忽略标点语境） |
| 数字 | **12321** |

书中列举此类例子，意在 **换脑**——前面刚学完数组与指针的「震惊事实」，这里用语文游戏放松。

### 与 C 字符串的朴素联系

判断 C 字符串是否回文，典型写法用到 **4.5** 的知识：

```c
#include <ctype.h>
#include <string.h>

int is_palindrome(const char *s)
{
    size_t i, j;

    if (s == NULL)
        return 0;
    j = strlen(s);
    if (j == 0)
        return 1;
    j--;
    for (i = 0; i < j; i++, j--) {
        while (s[i] && !isalnum((unsigned char)s[i]))
            i++;
        while (j > i && !isalnum((unsigned char)s[j]))
            j--;
        if (tolower((unsigned char)s[i]) !=
            tolower((unsigned char)s[j]))
            return 0;
    }
    return 1;
}
```

要点：

- 参数 **`const char *s`**：只读遍历，不假定 `s` 是数组还是指针——**4.5** 中二者在下标访问上可互换。
- **`s[i]`** 与 **`*(s+i)`** 等价；也可用 **`i[s]`** 作恶作剧（**4.5**，勿用于正式代码）。
- 若串在 **`char arr[]`** 中定义，传 `arr` 时 **退化为 `char*`** 传入函数。

### 双指针从两端夹逼

```text
  "racecar"
   ↑     ↑
   i     j   比较 s[i] 与 s[j]，向中间移动
```

与 **数组连续存储** 一致：下标 `i`、`j` 是对 **同一底层 char 序列** 的偏移，不涉及「数组名 vs 指针」的跨文件陷阱。

### 可选练习（自学）

1. 写 `int palindrome_arr(const char a[])`，仅用下标，不用 `strlen`（调用方传长度）。
2. 对比 **`char s[] = "aba"`** 与 **`char *s = "aba"`** 传入上述函数——形参侧行为相同。
3. 思考：若 **错误** 地对全局 `char arr[]` 使用 `extern char *arr`（**4.2**），回文函数会在 **第一次解引用** 时就崩溃，根本走不到比较逻辑。
