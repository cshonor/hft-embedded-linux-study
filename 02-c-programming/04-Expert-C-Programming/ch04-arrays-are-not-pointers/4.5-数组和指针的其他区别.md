# 4.5 数组和指针的其他区别

## 本节讲什么

在 **4.1** 确立「数组 ≠ 指针」、**4.2–4.4** 解决跨文件匹配之后，本节汇总 **运行时与语义层面** 的其它关键差异：**存储布局**、`sizeof` 行为、`arr+1` 与 `&arr+1` 的指针算术、**函数形参退化**、下标与解引用的等价关系，以及 **`2[arr]`** 为何合法。配合 **demo01_sizeof**、**demo03_subscript** 验证。

## 要点

### 四项本质差异（必背表）

| 维度 | 数组 `int a[10]` | 指针 `int *p` |
|------|------------------|---------------|
| **存储** | **10 个 int** 连续存放 | 仅 **一个地址**（通常 4/8 字节） |
| **`sizeof`** | `10 * sizeof(int)`（**40**） | `sizeof(int*)`（**4/8**） |
| **`&` 运算** | `&a` 类型 **`int (*)[10]`** | `&p` 类型 **`int **` |
| **赋值** | **`a = ...` 非法** | **`p = &x` 合法** |

```c
int a[10];
int *p = a;

sizeof(a);   /* 40 — 数组总大小 */
sizeof(p);   /* 4 或 8 — 指针本身大小 */

/* a = p;     编译错误 */
p = a;       /* OK：a 退化为 int* */
```

**demo01_sizeof** 还会展示：函数内 `sizeof(arr)` 当形参为 `int arr[]` 时 **只剩指针大小**。

### 存储布局示意

```text
数组 a[3]（假设 int 4 字节）:
  地址:  1000   1004   1008
         [ a0 ] [ a1 ] [ a2 ]

指针 p（指向 a[0]）:
  地址:  2000
         [ 1000 ]  ← 只存首元素地址
```

数组 **是** 数据；指针 **指向** 数据。`char arr[] = "hi"` 把字符串 **拷贝进数组**；`char *p = "hi"` 让 `p` 指向 **只读字面量** 区域。

### 指针算术：`arr+1` vs `&arr+1`

对 `int local[10]`（`sizeof(int)==4`）：

| 表达式 | 类型 | 地址变化 | 含义 |
|--------|------|----------|------|
| `local`（退化） | `int*` | — | 首元素地址，如 `0x1000` |
| `local + 1` | `int*` | **+4** | 下一个 **元素** |
| `&local` | `int (*)[10]` | — | **整个数组** 的地址（值常等于首元素地址） |
| `&local + 1` | `int (*)[10]` | **+40** | 跳过 **整个数组** 对象 |

```c
printf("local=%p  &local=%p\n", (void*)local, (void*)&local);
/* 数值常相同，类型不同 */
printf("local+1=%p  &local+1=%p\n",
       (void*)(local+1), (void*)(&local+1));
/* local+1 跳 1 元素；&local+1 跳 10 元素 — demo01 */
```

**记忆**：`&arr` 的类型是「指向 **数组对象** 的指针」，步长是 **数组总字节数**。

### 函数形参：数组退化为指针

```c
void foo(int arr[]);   /* 等价于 void foo(int *arr); */
void bar(int *arr);    /* 完全相同的原型 */
```

C 标准规定：**函数参数里的数组类型会被调整为指针类型**。

| 位置 | `int a[10]` 的含义 |
|------|-------------------|
| 文件/块作用域 **定义** | 真正的 **int[10] 数组** |
| **函数形参** | 调整为 **`int*`** |

因此：

```c
void test(int arr[]) {
    sizeof(arr);  /* sizeof(int*)，不是 10*sizeof(int) */
}
```

**不能** 在函数内用 `sizeof` 求「传入数组元素个数」——须 **额外传长度** 或 **传指针+size 参数**（**ch09 9.3** 详述设计原因）。

**重要**：此等价 **仅适用于形参**；**4.4** 的文件作用域 `extern` **不适用**。

### 形参等价写法对照

| 写法 | 是否同一原型 |
|------|--------------|
| `void f(int a[])` | ✓ |
| `void f(int a[10])` | ✓（长度被 **忽略**） |
| `void f(int *a)` | ✓ |
| `void f(int a[static 10])` | C99：特殊，仍与上不同语义（少见） |

```c
/* 以下声明完全兼容 */
void f(int *);
void f(int[]);
void f(int[9999]);  /* 9999 被忽略，仍是指针形参 */
```

### 下标与解引用：`a[i] == *(a+i)`

C 定义下标运算符为：

```text
a[i]  ≡  *(a + i)
```

其中 `a` 在表达式中通常 **先退化为指针**，再加 `i`（按元素类型缩放）。

```c
int arr[] = {10, 20, 30};
int *p = arr;

arr[1];      /* 20 */
*(arr + 1);  /* 20 */
p[1];        /* 20 */
1[arr];      /* 20 — 见下 */
```

**demo03_subscript** 打印四者相等。

### 为什么 `2[arr]` 合法

由定义：

```text
a[b]  =  *(a + b)  =  *(b + a)  =  b[a]
```

因此：

```c
arr[2]  ==  2[arr];   /* 都等价 *(arr+2) */
```

**可读性为零**，标准允许，面试常考，**生产代码勿用**。

### 数组名不是左值（不可赋值）

```c
int a[5], b[5];
/* a = b;   非法：数组名不是 modifiable lvalue */
int *p = a;  /* 合法：退化后赋给指针 */
```

指针是 **可修改左值**；数组名在表达式中是 **地址常量**（退化结果），不能作为赋值左端。

### 退化发生的典型场景（小结）

| 场景 | 数组名行为 |
|------|------------|
| 赋给指针、传函数实参 | 退化为 `T*` |
| 下标、`+`、`-` 运算 | 先退化再算 |
| **`sizeof(a)`** | **不退化** |
| **`&a`** | **不退化**，得 `T (*)[N]` |
| **声明/定义行** | 类型仍是数组 |
