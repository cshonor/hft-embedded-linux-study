# 4.2 我的代码为什么无法运行

## 本节讲什么

全书最著名的 **跨文件 bug** 之一：一个 `.c` 里 **定义** `char arr[] = "hello"`，另一个 `.c` 里 **声明** `extern char *arr`，编译、链接 **都能通过**，运行却在访问 `arr` 时 **段错误（SIGSEGV）** 或输出乱码。本节用内存布局逐步解释 **为何编译器/linker 抓不出来**，以及 **错在哪里**。

## 要点

### 问题代码结构

**文件 A（定义）** — `arr_def.c`：

```c
char arr[] = "hello";
```

**文件 B（错误声明）** — `use_wrong.c`：

```c
extern char *arr;   /* 错！定义侧是数组，不是指针变量 */

int main(void) {
    printf("%s\n", arr);  /* 崩溃或乱码 */
}
```

**文件 B（正确声明）** — `use_correct.c`：

```c
extern char arr[];  /* 对：与 char arr[] = "hello" 类型一致 */
```

对应 **demo02_extern**：`make demo02_correct` 正常；`make demo02_wrong` 可能 SIGSEGV。

### 内存里实际有什么

定义 `char arr[] = "hello"` 时：

```text
地址低 ──────────────────────────────► 高
arr:  'h' 'e' 'l' 'l' 'o' '\0'   ← 6 字节连续存储，arr 是这块区域的名字
```

若错误地当作 `char *arr` 理解，编译器认为：

```text
arr 是一个指针变量（4/8 字节），其「值」应是某个地址
```

但链接器把 **符号 arr 的地址** 绑到 **字符 'h' 所在位置**（数据段里数组首字节），**并没有** 单独的指针槽存放「指向 hello 的地址」。

### 崩溃机制（逐步）

1. `use_wrong.c` 中 `extern char *arr` 告诉编译器：`arr` 是 **指针变量**，`*arr` 是从 **arr 里读出的地址** 再解引用。
2. 链接时符号 `arr` 解析为 **数组首元素的字节地址**，例如 `0x601040`。
3. 访问 `*arr` 或 `printf("%s", arr)` 时，编译器生成的代码会：
   - 从地址 `0x601040` **读取 4/8 字节** 当作「指针值」；
   - 那些字节实际是字符 **`'h' 'e' 'l' 'l'`**（小端下可能是 `0x006c6c6568` 一类）；
   - 再把这个「假地址」解引用 → **访问非法地址 → SIGSEGV**。

```text
正确模型：arr 就是 'h' 的地址，arr[0]=='h'，arr 作为整体是 char[6]

错误模型：arr 是指针变量 → 先读 arr 的内容（把 'hell' 当指针）→ 再解引用 → 崩溃
```

### 为何编译器和 linker 都不报错

| 阶段 | 为何放过 |
|------|----------|
| **编译** | 各 TU 单独编译；`char*` 与 `char[]` 在 **表达式** 里都能用 `[]`、都能传给 `printf("%s")` |
| **链接** | 只核对 **符号名** `arr` 是否存在、是否唯一；**不检查** 两侧 C 类型是否一致（C 链接 **无类型**） |
| **运行** | 类型误解体现在 **错误的机器指令**（多读一次内存） |

这是 **C 的类型系统漏洞**：链接层不携带完整类型信息（与 C++ name mangling 不同）。

### 正确写法与原则

```c
/* 定义（一个 .c） */
char arr[] = "hello";

/* 声明（其他 .c / .h）—— 类型必须匹配 */
extern char arr[];     /* 推荐：可省略长度 */
extern char arr[6];    /* 也可写全长度，须 ≥ 实际定义长度 */
```

**禁止**：

```c
extern char *arr;      /* 与 char arr[] 定义不匹配 */
```

**头文件惯例**：

```c
/* header.h */
extern char arr[];     /* 不是 extern char *arr */
```

### 与 `char *p = "hello"` 的对比

| 写法 | 存储 | 类型 | 能否修改内容 |
|------|------|------|--------------|
| `char arr[] = "hello"` | 数组在 **可写** 数据段（或栈若局部） | `char[6]` | 可改 `arr[0]`（字符串字面量初始化后数组本身可写） |
| `char *p = "hello"` | `p` 存 **字面量地址**；字面量常在 **只读** 段 | `char*` | 改 `*p` 可能 **未定义行为** |

跨文件时：**定义是什么类型，extern 就必须声明同一类型**——数组对数组，指针对指针。

### 调试线索

| 现象 | 可能原因 |
|------|----------|
| 链接成功、一访问全局 `arr` 就崩 | `extern char *` vs `char[]` 不匹配 |
| `printf("%p", arr)` 合理但 `printf("%s", arr)` 崩 | 把数组首地址当成了「指针变量里存的值」 |
| 仅首字符相关代码「碰巧」能跑 | 误把 ASCII `'h'`(0x68) 当小地址解引用，有时不立即 crash |
