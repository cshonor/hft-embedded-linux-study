# 7.7 总线错误

## 本节讲什么

Unix 上进程崩溃常见两类信号：**SIGSEGV（段错误）** 与 **SIGBUS（总线错误）**。本节对比二者触发条件，重点说明 **严格对齐架构（如 SPARC）** 上 **未对齐访问** 如何产生 **总线错误**，而 **x86/x86-64** 往往 **容忍** 未对齐（仅性能损失）——联系 **7.1 对齐**、**7.2 packed**、**7.3 页权限**。

## 要点

### SIGSEGV vs SIGBUS 速查

| 信号 | 全称 | 常见原因 | 典型平台 |
|------|------|----------|----------|
| **SIGSEGV** | Segmentation violation | **无效 VA**、**权限违例**（写 rodata、执行 NX）、**NULL 解引用** | 全平台 |
| **SIGBUS** | Bus error | **未对齐访问**（严格 CPU）、**mmap 文件大小变化**、**某些硬件错误** | **SPARC**、旧 **ARM**、部分 **MIPS**；x86 较少 |

```text
  访问类型                    x86-64          SPARC (严格对齐)
  ─────────────────────────────────────────────────────────
  读 NULL                     SIGSEGV         SIGSEGV
  写 .rodata                  SIGSEGV         SIGSEGV
  int 从地址 0x1001 加载       慢，但通常成功    SIGBUS
  packed struct 未对齐成员     通常成功         可能 SIGBUS
```

### 段错误（SIGSEGV）典型场景

| 场景 | 机制 |
|------|------|
| **`NULL` 解引用** | 低地址 **未映射**（**7.3**） |
| **写只读页** | **`p1[0]='H'`** on **`const char *`**（**demo03**） |
| **栈溢出触 guard page** | 越过 **栈映射** 边界 |
| **野指针 / UAF** | 页已 **munmap** 或权限改变（**7.6**） |
| **执行数据区** | **NX** → 不可执行（**ch06 6.3**） |

```bash
# 故意触发（仅学习）
# NULL: echo 'int main(){*(int*)0=1;}' | gcc -x c - && ./a.out
# rodata: 见 demo03 取消注释 p1[0]='H'
```

### 总线错误（SIGBUS）典型场景

| 场景 | 说明 |
|------|------|
| **未对齐 load/store** | **`(int*)0x1001`** 在 **SPARC** |
| **`mmap` 文件缩短** | 访问映射尾外 **已不存在** 的文件页 |
| **某些 `madvise(MADV_DONTNEED)` 后 touch** | 平台相关 |

**7.2 `__attribute__((packed))`** 在 x86 上「能用」的代码，移植到 **严格对齐 CPU** 可能变 **SIGBUS**——**序列化不应依赖 packed 指针强转**。

### x86 的特殊性（7.1 衔接）

- **8086 起** 部分未对齐需 **两次总线周期**，但 **不 trap**
- **现代 x86-64**：未对齐 **跨 Cache 行** 可能更慢；**AVX** 要求 **栈 16/32 B 对齐**
- **`#AC` 对齐检查异常**：仅在 **特定模式**（如 **SMAP/AC flag** 部分场景）常见用户态仍 **宽容**

**面试对比**：「为什么 Linux 上 **Bus error** 少见？」→ **x86 硬件容忍未对齐** + 多数 bug 表现为 **SIGSEGV**（无效地址更常见）。

### 与虚拟内存、Cache 的关系

- **SIGSEGV**：MMU **页表项无效** 或 **R/W/X 位拒绝** → **page fault** 不可修复 → 信号
- **SIGBUS**：地址 **在映射内合法**，但 **硬件无法完成传输**（对齐、总线协议）
- **7.4**：未对齐访问 **跨 Cache 行** → 性能差，x86 上 **不一定是 SIGBUS**

### 调试手段

| 方法 | 作用 |
|------|------|
| **`gdb ./prog` → `run` → `bt`** | 崩溃栈 |
| **`dmesg` / `core dump`** | 内核日志、**`ulimit -c unlimited`** |
| **`-fsanitize=address`** | 越界、UAF **先于** 随机 SIGSEGV 报告 |
| **`readelf -l` / `pmap`** | 确认映射范围与权限 |
