# 7.1 Intel 80x86系列

## 本节讲什么

Intel **80x86** 系列从 **8086** 到现代 **x86-64** 的演进，是理解 PC 内存模型与 **对齐（alignment）** 硬件根源的起点。本节梳理各代 CPU 的寻址能力、段寄存器历史，以及 **char / short / int / 指针** 在 x86 上的典型对齐规则——这些规则直接决定 **7.2** 结构体填充与 **7.7** 总线/段错误的差异。

## 要点

### 80x86 系列简史

| 代际 | 代表 | 数据总线 | 寻址空间 | 关键特征 |
|------|------|----------|----------|----------|
| **8086/8088** | IBM PC | 16/8 bit | **1 MB**（20 位） | **段:偏移** 寻址；**8088** 8 位外部总线 |
| **80286** | AT | 16 bit | **16 MB** | 保护模式雏形；仍保留实模式 |
| **80386** | 32 位 PC | 32 bit | **4 GB** | **32 位平坦寻址**；分页 MMU（**7.3**） |
| **80486 / Pentium** | — | — | 4 GB | 集成 **FPU**、**Cache**（**7.4**） |
| **Pentium Pro+** | — | — | 4 GB+ | **超标量**、更大 Cache 层次 |
| **x86-64（AMD64）** | 现代 PC/服务器 | 64 bit | **48 bit VA**（典型） | **更多寄存器**、**RIP 相对寻址**、仍兼容 32 位模式 |

**历史脉络**：8086 的 **段:偏移** 是为在 16 位寄存器里拼出 20 位地址；386 引入 **分页** 后，OS 普遍采用 **平坦模型（flat model）**——程序看到的几乎是连续虚拟地址，段机制退居幕后（**7.2**）。

### 对齐的硬件原因

CPU 与内存/Cache 以 **字（word）** 为单位高效传输。若数据跨越对齐边界：

- **x86/x86-64**：多数未对齐访问 **能完成**（可能拆成两次访问、略慢），极端情况 **#AC** 对齐检查（若启用）
- **SPARC / ARM（旧）/ MIPS 等**：**严格对齐**，未对齐 → **总线错误 SIGBUS**（**7.7**）

**对齐规则（典型 GCC x86-64 ABI）**：

| 类型 | 大小 | 对齐要求 | 说明 |
|------|------|----------|------|
| **`char`** | 1 | **1** | 任意地址 |
| **`short`** | 2 | **2** | 地址 % 2 == 0 |
| **`int`** | 4 | **4** | 地址 % 4 == 0 |
| **`long` / 指针** | 8 | **8** | 地址 % 8 == 0 |
| **`double`** | 8 | **8** | 同上 |
| **`long double`** | 16 | **16** | SSE/AVX 栈对齐相关 |

**结构体整体对齐** = 成员中 **最大对齐值**；**sizeof** 向上取整到该倍数（**7.2** 手算 **16 字节** 示例）。

### 8086 段寻址（与 7.2 衔接）

```text
  物理地址 = (段寄存器 << 4) + 偏移   （20 位，1 MB）
  例：CS=0x1234, IP=0x0050 → 0x12390
```

- **CS**（代码）、**DS/ES/SS**（数据/附加/栈）各 64 KB 窗口，可重叠
- 386+ **保护模式 + 分页** 后，用户程序几乎不再直接操作段寄存器；Linux 用 **平坦段**（基址 0、限长 4G）

### 与 ch06 内存布局的关系

**ch06 6.2** 从 **进程视角** 描述 **`.text/.rodata/.data/.bss`/堆/栈**；本节从 **CPU 硬件** 解释 **为何** 这些区域需要按页映射、为何指针必须对齐访问多字节类型。**6.10** MS-DOS 的 **near/far** 指针正是 8086 段模型的 C 语言残留。
