# 7.5 数据段和堆

## 本节讲什么

**数据段**（**`.data` / `.bss`**）存放 **进程生命周期** 的全局与静态对象；**堆（heap）** 由 **`malloc` / `free`** 动态管理，底层依赖 **`brk` / `mmap`**（**ch06 6.9**）。本节深入 **malloc 块元数据**、**返回栈上局部数组指针** 的未定义行为，以及 **`const` 三种写法** 与 **`.rodata`** 的关系（**demo02 / demo03**）。

## 要点

### 数据段回顾（ch06 6.2）

| 段 | 内容 | 载入 |
|----|------|------|
| **`.data`** | 有初值全局/静态 | ELF **拷贝** |
| **`.bss`** | 零初值全局/静态 | **NOBITS 清零** |
| **`.rodata`** | 字符串字面量、`const` 全局 | **只读映射** |

函数内 **`static int x`** → **`.data/.bss`**，**不是栈**（**6.6 demo02**）。

### 堆的增长：brk 与 mmap

```text
  低地址 ──► .text .rodata .data .bss ──► [brk] ──► ... ──► [mmap 区] ──► 栈
                                              ↑
                                    program break（堆顶）
```

| 机制 | 典型用途 |
|------|----------|
| **`brk` / `sbrk`** | 小堆扩展：抬高 **program break** |
| **`mmap`** | 大块分配、**arena** 扩展、**>128KB** 级阈值（glibc 实现相关） |
| **释放** | **`free`** 还回 **arena**；不一定立刻 **munmap**（**7.6** 泄漏 vs 缓存） |

**观察**：`pmap -x $$`、**`/proc/self/maps`** 看 **[heap]** 映射。

### malloc 块元数据（概念模型）

glibc **ptmalloc** 等实现：每个用户块前（或邻接）有 **chunk header**（大小、状态位、链表指针等）。

```text
  ┌──────────────────┬─────────────────────────┐
  │ chunk metadata   │  user data (return ptr) │
  │ (size, flags...) │  你 malloc 得到的地址     │
  └──────────────────┴─────────────────────────┘
```

| 要点 | 说明 |
|------|------|
| **对齐** | 返回指针通常 **16 B 对齐**（x86-64） |
| **最小块** | 过小请求也占 **最小 chunk + 头** |
| **footer / 合并** | 相邻 **free chunk** 可 **coalesce** |
| **heap overflow** | 越界写 **覆盖下一 chunk 头** → **`free` 崩溃**（**7.6**） |

**不要** 依赖具体 header 布局——实现私有、版本会变。

### 栈上局部数组返回：悬垂指针（demo02）

```c
static char *get_bad_ptr(void)
{
    char buf[8] = "123456";
    return buf;   /* UB：栈帧销毁后 buf 失效 */
}
```

| 现象 | 原因 |
|------|------|
| 乱码 / 段错误 | **`main` 栈帧覆盖** 原 **`get_bad_ptr` 栈帧** |
| 「看似正常」 | 内存尚未被覆写——**更危险** |

**正确做法**：

- **`static` 缓冲**（注意重入/线程安全）
- **调用者提供 buffer**
- **`malloc` + 调用者 `free`**
- **C99 柔性数组 / 堆上 struct**

**demo02_stack_dangle**：

```bash
cd demo/demo02_stack_dangle && make && ./demo02
```

与 **ch06 6.5** 栈帧、**6.8 setjmp** 后局部失效同类。

### const 三种形式（demo03）

```c
const char *p1 = "hello";      /* ① 指向常量的指针 */
char *const p2 = ...;          /* ② 指针本身是 const */
const char *const p3 = "...";  /* ③ 二者皆 const */
```

| 写法 | 指针可变？ | 所指内容可变？ | 典型 |
|------|------------|----------------|------|
| **`const char *p`** | ✅ | ❌（**`.rodata`**） | 字符串遍历 |
| **`char *const p`** | ❌ | ✅（若指向可写区） | 固定缓冲区 cursor |
| **`const char *const p`** | ❌ | ❌ | 只读表项 |

**记忆**：**`const` 修饰其 **左侧** 类型；左侧无类型则修饰 **右侧**。

```c
const char *p1 = "hello .rodata";
/* p1 = "other";     OK */
/* p1[0] = 'H';      SIGSEGV — 字符串在 .rodata */

char *const p2 = (char *const)"also rodata";
/* p2 = ...;         编译错误 */
/* p2[0] = 'H';      SIGSEGV */
```

**demo03_const_rodata**：

```bash
cd demo/demo03_const_rodata && make && ./demo03
```

**对比 ch06 6.2**：**`"hello"`** 在 **`.rodata`**；**`char buf[] = "hello"`** 在 **栈** 上 **可写副本**。

### 堆 vs 栈 vs 数据段选型

| 存储 | 生命周期 | 线程 | 典型场景 |
|------|----------|------|----------|
| **栈 auto** | 函数调用 | 每线程栈 | 小对象、RAII 风格（C 里注意 scope） |
| **`.data/.bss`** | 进程 | 共享需同步 | 全局配置、**static 局部计数** |
| **堆 malloc** | 直到 **free** | 共享需同步 | 运行时大小未知、大对象 |
