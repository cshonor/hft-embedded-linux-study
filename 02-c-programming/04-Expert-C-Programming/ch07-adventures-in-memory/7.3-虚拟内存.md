# 7.3 虚拟内存

## 本节讲什么

**虚拟内存（Virtual Memory, VM）** 让每个进程拥有 **独立、巨大、连续** 的地址空间 illusion，由 **MMU（内存管理单元）** 通过 **页表** 把 **虚拟地址（VA）** 翻译成 **物理地址（PA）**。本节覆盖 **页故障（page fault）**、**ASLR**、内核 **kernel.elf** 的链接虚拟地址，以及 VM 如何实现 **进程隔离**——衔接 **ch06 6.3** 载入与 **ch05 kernel.lds** 布局。

## 要点

### VA vs PA

| 概念 | 符号 | 谁使用 | 典型范围（x86-64 Linux） |
|------|------|--------|--------------------------|
| **虚拟地址** | **VA** | 程序、`&var`、指针 | 用户空间低半区（约 **0 ~ 128 TB** 量级设计，实际用子集） |
| **物理地址** | **PA** | 内存控制器、DMA | 机器 **RAM** 实际行列地址 |
| **映射** | 页表项 | **MMU 硬件** | **4 KB** 页（常见；还有 **2 MB / 1 GB** 大页） |

```text
  程序:  ptr = 0x7fff12340000  (VA)
           │
           ▼ MMU 查页表（CR3 → 多级页表）
  内存:  物理帧 0x1a3f000       (PA)
```

**每个进程** 有 **独立页表** → 相同 VA 在不同进程可映射到 **不同 PA** → **隔离**。

### MMU 与页表（定性）

**x86-64 四级页表**（简化）：

```text
  VA 拆分: [ PGD | PUD | PMD | PTE | offset ]
            └── 每级 9 bit ──┘      └── 12 bit (4KB) ──┘

  CR3 寄存器 → 页表基址
  TLB：页表项缓存，加速重复翻译
```

| 事件 | 说明 |
|------|------|
| **TLB hit** | 硬件直接 VA→PA，极快 |
| **TLB miss** | 走页表 walk（软件或硬件 walker） |
| **页不在内存** | **缺页异常** → 内核 **page fault** 处理 |

### 页故障（Page Fault）类型

| 类型 | 常见原因 | 内核动作 |
|------|----------|----------|
| **Major fault** | 页尚未载入（**demand paging**） | 从 **swap/文件** 读入或 **零页** 分配 |
| **Minor fault** | 页在内存但 TLB 未缓存 | 更新 TLB |
| **Protection fault** | 写只读页、执行 NX 页、越界 | **SIGSEGV**（**7.7**） |
| **COW（写时复制）** | `fork` 后父子共享只读映射，一方写入 | 复制物理页、更新页表 |

**Demand paging**：`malloc` 返回的 VA 可能尚未触达物理 RAM，**首次写入** 才分配帧（**lazy allocation**）。

### ASLR（Address Space Layout Randomization）

- **目的**：攻击者难以猜测 **栈 / 堆 / mmap / .so** 基址
- **影响**：**demo01**、**ch06 demo01** 每次运行绝对地址 **变化**；**相对顺序** 仍稳定
- **关闭**（仅调试）：`echo 0 | sudo tee /proc/sys/kernel/randomize_va_space`
- **与 ch06 6.3**：载入器 **mmap PT_LOAD** 时加入随机 **slide**

### 进程隔离与安全

| 机制 | 效果 |
|------|------|
| **独立页表** | 进程 A 无法直接读写进程 B 的 PA |
| **页权限 R/W/X** | **`.text` RX**、**`.rodata` R**、**栈 RW 非 X**（W^X） |
| **用户/内核态** | 用户 VA 无法访问内核映射（**syscall** 网关） |
| **NULL 页保护** | 低地址不映射 → **`NULL` 解引用** → **SIGSEGV** |

### kernel.elf 与 kernel.lds（ch05 衔接）

**用户进程** 由 **execve + ELF PT_LOAD** 载入；**裸机/内核** 常用 **链接脚本** 指定 **链接 VA**：

```text
  /* ch05 demo04 kernel.lds 片段 */
  SECTIONS {
      . = 0x80000;          /* 链接地址：内核期望被载入的 VA */
      .text : { _stext = .; *(.text) _etext = .; }
      ...
      .bss  : { _sbss = .; *(.bss) _ebss = .; }
  }
```

| 符号 | 含义 |
|------|------|
| **`_stext` / `_etext`** | 代码段 **VA** 边界 |
| **`_sbss` / `_ebss`** | BSS **VA** 边界；启动代码 **清零** |
| **`ENTRY(_start)`** | 入口 **VA**（非 `main`） |

**bootloader** 把内核映像放到 **PA**，并设页表使 **VA 0x80000** 等可访问——**链接 VA ≠ 物理放置地址**，除非 **identity map** 启动阶段。

```bash
# ch05 demo04
cd ../ch05-thinking-of-linking/demo/demo04_linker_script
make && readelf -l kernel.elf
```

### 与 ch06 载入器对照

```text
  用户 ELF ──execve──► 内核 loader (6.3)
                         mmap PT_LOAD @ 随机 VA + 权限
                         bss NOBITS 零填充
                         建页表项 → CR3

  内核 ELF ──boot──►  临时 identity map → 启用分页 → 高半内核 VA
```

**工具**：`pmap -x PID`、`/proc/PID/maps`、`readelf -l`。
