# 7.2 Intel 80x86内存模型以及它的工作原理

## 本节讲什么

现代 x86 Linux 使用 **平坦虚拟地址空间**：程序看到的是从 0 到最大 VA 的连续字节流，底层由 **MMU 页表**（**7.3**）映射到物理页。本节聚焦 **结构体布局**：编译器如何插入 **填充字节（padding）**、**`__attribute__((packed))`** 的代价、**`offsetof`** 的用途，以及 **按内存布局直接序列化** 为何会在跨平台/跨编译器时出错。

## 要点

### 段模型 → 平坦模型

| 时代 | 程序员看到的地址 | OS/硬件角色 |
|------|------------------|-------------|
| **8086 实模式** | **段:偏移** → 1 MB 物理 | 段寄存器 + 偏移 |
| **80386+ 保护模式** | **32/64 位线性 VA** | **GDT/LDT** 设平坦段 + **页表** |
| **Linux 用户进程** | **`malloc` 返回的指针** 即 VA | **ASLR** 随机化基址（**7.3**） |

**平坦模型**下：`char *p` 加 1 即下一字节；**不再有** near/far 语义（**ch06 6.10** 对比 DOS）。

### 结构体填充：Test1 手算（16 字节）

**demo01_struct_align** 中的典型布局（**x86-64 GCC**）：

```c
struct Test1 {
    char a;    /* 1 B */
    int b;     /* 4 B，需 4 对齐 */
    short c;   /* 2 B，需 2 对齐 */
};
```

```text
  偏移   内容
  0      a (1 B)
  1–3    [padding]  ← b 需从 4 的倍数开始
  4–7    b (4 B)
  8–9    c (2 B)
  10–15  [padding]  ← sizeof 向上取整到 max(1,4,2)=4 的倍数 → 16
  ────────────────
  sizeof(Test1) = 16  （不是 1+4+2=7）
```

**规则摘要**：

1. 每个成员偏移 = 向上取整到 **该类型对齐值**
2. 结构体 **总大小** = 向上取整到 **所有成员最大对齐值**
3. **数组/嵌套 struct** 同样适用

```bash
cd demo/demo01_struct_align && make && ./demo01
# 观察 offsetof(a)=0, b=4, c=8, sizeof=16
```

### Test2 与 Test2_packed 对比

```c
struct Test2 { char a; int b; };           /* 典型 8 B（1 + 3 pad + 4） */
struct Test2_packed __attribute__((packed)) { char a; int b; };  /* 5 B */
```

| | **默认对齐** | **`packed`** |
|---|-------------|--------------|
| **sizeof** | 8 | 5 |
| **b 的偏移** | 4 | 1 |
| **访问 b** | 单次对齐 load | 可能 **未对齐 load**（x86 慢；SPARC **SIGBUS**） |
| **跨平台二进制** | 仍不可 portable | 更不可 portable |

### `__attribute__((packed))` 慎用场景

- **网络协议 / 硬件寄存器映射**：需要与文档字节级一致时 **可能** 合理
- **性能敏感热路径**：未对齐访问拖慢 CPU，破坏 **向量化**
- **可移植性**：不同 ABI / `#pragma pack` 行为不一致
- **未定义行为边缘**：对 **packed** 成员取地址再按 **`int *`** 解引用 → 可能 UB
- **替代方案**：显式 **`uint8_t buf[]` + 手工 unpack**；或 **序列化层** 与内存布局解耦

### `offsetof` 与标准布局

```c
#include <stddef.h>
size_t off = offsetof(struct Test1, b);  /* 编译期常量 */
```

- **用途**：内核/驱动 offsetof 算容器内成员指针；**Flex 数组** 前导长度
- **C11 `_Static_assert`**：`offsetof` 校验协议偏移
- **注意**：**`offsetof` 对位域** 有限制；位域布局 **实现定义**

### 序列化与对齐陷阱

**错误做法**：

```c
write(fd, &struct_val, sizeof(struct_val));  /* 危险 */
```

| 陷阱 | 后果 |
|------|------|
| **padding 字节未初始化** | 泄漏栈/堆垃圾；校验和不一致 |
| **32/64 位 `long`/指针 大小不同** | 跨架构解析错乱 |
| **endian 不同** | 大端/小端反转 |
| **编译器选项 `-fpack-struct`** | 全局改变 ABI，与库不兼容 |

**正确做法**：定义 **wire format**（固定宽度类型 + 显式字节序转换），**不要** 假设内存布局 == 磁盘布局。

### 与 ch06 / ch05 的交叉

- **ch06 6.2**：全局/静态在 **`.data/.bss`**，局部在 **栈**——布局与 **struct 成员布局** 是不同层次
- **ch05 5.1**：链接器合并 **`.data/.bss`** 节；**不决定** struct 内 padding（编译器决定）
- **ch08**：类型与 **求值顺序** 影响表达式中的临时对象布局（只读预告）
