# 7.6 内存泄漏

## 本节讲什么

**内存泄漏（memory leak）** 指已分配堆内存 **失去所有可达指针** 却 **未 `free`**，进程 **RSS 持续增长**、长期服务 **OOM**。本节还覆盖 **悬垂指针（dangling pointer）**、**double free**、**堆溢出破坏 malloc 元数据**——与 **valgrind / ASan** 检测对应（**ch06 6.11**）。

## 要点

### 典型泄漏模式

```c
void leak(void) {
    char *p = malloc(1024);
    /* 忘记 free(p); p 出作用域后块永不可达 */
}

void leak_loop(void) {
    while (1) {
        char *p = malloc(4096);
        /* 只保留最后一次？或全部丢失 */
    }
}
```

| 类型 | 特征 |
|------|------|
| **绝对泄漏** | 无任何指针指向块 |
| **相对泄漏** | 容器只增不减（**vector 永不 erase**） |
| **「伪泄漏」** | **atexit** 前不释放 OS 回收——调试期仍应 **`free`** |

**长期 daemon / 内核模块（kmalloc 不 kfree）** 危害最大；短进程退出时 OS **回收全部映射**，但 **valgrind 仍报错**。

### valgrind 使用（ch06 6.11）

```bash
valgrind --leak-check=full --show-leak-kinds=all ./your_program
```

| 报告 | 含义 |
|------|------|
| **definitely lost** | 明确泄漏 |
| **indirectly lost** | 通过另一泄漏块间接丢失 |
| **possibly lost** | 仅内部指针指向块内（**intptr 技巧** 等） |
| **still reachable** | 程序结束仍可达（**全局链表** 等） |

**现代替代**：**AddressSanitizer**（`-fsanitize=address`）、**LeakSanitizer**（`-fsanitize=leak`）——更快，需重编译。

### 悬垂指针（Use-After-Free）

```c
char *p = malloc(16);
free(p);
strcpy(p, "oops");   /* UB：wild pointer / UAF */
```

| 阶段 | 状态 |
|------|------|
| **`free` 后** | **`p` 值未变**（dangling），块可能进 **free list** |
| **再 `malloc`** | 同址 **复用** → 写坏新对象 |
| **再 `free(p)`** | **double free** → 元数据链表损坏 |

**demo02** 是 **栈** 悬垂；本节强调 **堆** UAF——**valgrind** 报 **Invalid read/write of size N**。

### double free

```c
free(p);
free(p);   /* 崩溃或堆结构破坏，后续 malloc 随机失败 */
```

- **原因**：同一 chunk **两次 unlink**
- **利用**：历史 **堆溢出漏洞** 攻击链一环（现代 **safe-linking** 等缓解）
- **预防**：**`free` 后置 `p = NULL`**（**仅防 double free**，不防 UAF）

### 堆溢出与元数据损坏（7.5 衔接）

```c
char *p = malloc(16);
strcpy(p, "this string is way too long");  /* 越过 user 区 */
free(p);   /* 可能 abort: corrupted double-linked list */
```

**越界写** 可能覆盖：

- **下一 chunk 的 size / prev_size**
- **free list fd/bk 指针**

**症状**：**`malloc`/`free` 随机 segfault**、**malloc(): memory corruption**——用 **ASan** 定位第一次越界写。

### 防御性编码习惯

- **谁分配谁释放**；成对 **`malloc/free`**
- **RAII 思维**（C 里 **`goto cleanup`** 单出口释放）
- **避免裸指针所有权模糊**；文档化 **borrow vs own**
- **`strdup` → free**；**`getline` → free**
- 长期运行：**定期 RSS 监控**、**heap profiling**（**malloc_stats**, **jemalloc prof**）
