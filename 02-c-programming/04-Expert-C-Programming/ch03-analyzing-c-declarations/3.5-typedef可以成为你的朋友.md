# 3.5 typedef 可以成为你的朋友

## 本节讲什么

`typedef` 把 **声明符 + 基本类型** 打包成 **新类型名**，让复杂声明可读、可复用。但 typedef **不是宏**：替换规则、const 绑定、多变量声明行为与 `#define` 不同。本节重点讲 **typedef 的正确用法** 与 **PSTR 经典陷阱**——`typedef char *PSTR; PSTR a, b;` 中 **a、b 都是指针**，而 `char *a, b` 中 **只有 a 是指针**。

## 要点

### typedef 做了什么

```c
typedef unsigned long ulong;
typedef int (*CompareFn)(const void *, const void *);
typedef char (*RowPtr)[80];   /* 指向含 80 char 的数组的指针 */
```

- 编译器在符号表增加 **类型别名**，不是新对象  
- 别名与 `int`、`struct foo` 一样参与声明解析  
- 可读性：回调、函数指针、数组指针在 API 中几乎必用 typedef（**demo02_func_ptr**）

### PSTR 陷阱（必考）

```c
typedef char *PSTR;

PSTR a, b;
/* 等价于：char *a; char *b;  ← a、b 都是 char* */

char *a, b;
/* 等价于：char *a; char b;    ← 只有 a 是指针，b 是 char */
```

| 写法 | `a` | `b` |
|------|-----|-----|
| `char *a, b` | `char*` | `char` |
| `PSTR a, b`（`typedef char *PSTR`） | `char*` | `char*` |

**原因**：`typedef` 定义的是 **完整类型** `PSTR`；`PSTR a, b` 中每个声明符都独立获得该类型。而 `char *a, b` 里 `*` 只属于 declarator `a`。

**工程建议**：

```c
typedef char *PSTR;
PSTR a;
PSTR b;          /* 清晰，避免与 char* 一行多变量混淆 */

/* 或 */
typedef char PSTR_unit;  /* 若本意是「字符串单元类型 char」——别用 PSTR 表示指针 */
```

### const 与 typedef 的绑定（demo03）

```c
typedef char *Str;
const Str s = "hello";
```

- `const Str s` → `s` **本身** 不能改指向（`Str` = `char*`，const 贴在外层指针）  
- **不是** `const char*`：仍可通过 `s[0] = 'H'` 修改所指内容（除非指向字符串字面量则 UB）

若要 **指向内容不可改**：

```c
typedef const char *CStr;
CStr s = "hello";   /* *s 不可改 */
```

**规则**：`const`/`volatile` 出现在 typedef **之后**，修饰的是 **typedef 展开后的整体**，而非「替换成 const char*」那么简单——需按 **3.2 声明符** 展开再看。

### typedef 作为「朋友」的场景

| 场景 | 示例 |
|------|------|
| 函数指针 | `typedef void (*SignalHandler)(int);` |
| 数组指针 | `typedef char (*Line)[256];`（**demo01_arr_ptr**） |
| 回调注册 | `typedef int (*ioctl_fn)(struct file *, unsigned cmd, unsigned long arg);` |
| 跨平台整数 | `typedef uint32_t u32;` |

复杂声明 **先 typedef 命名中间层**，再写业务代码——维护成本远低于一行怪物声明。

### typedef 不是银弹

| 问题 | 说明 |
|------|------|
| 隐藏指针 | `PSTR` 看不出是指针，易误用 `sizeof(PSTR)` |
| 与宏混用 | `#define` 文本替换会破坏 typedef 语义（**3.6**） |
| 重复 typedef struct | `typedef struct foo { } foo;` 三处 `foo` 含义不同（**3.7**） |
| C++ 兼容 | typedef 与 `using`、模板交互另论；纯 C 系统代码仍以 typedef 为主 |
