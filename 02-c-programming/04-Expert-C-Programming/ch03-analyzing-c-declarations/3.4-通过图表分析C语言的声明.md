# 3.4 通过图表分析 C 语言的声明

## 本节讲什么

**图表法** 与 **3.3 螺旋规则** 等价：把声明画成 **以 identifier 为中心的框图**，箭头表示指针，方框表示数组/函数，最终落到基本类型。适合纸笔推演、白板面试、复杂声明分步验证。本节给出画法规范与两个完整示例。

## 要点

### 图表元素约定

| 符号 | 含义 |
|------|------|
| **方框 + 名字** | 当前正在解析的 identifier |
| **→** 或 **`*`** | 指针：「指向」右侧类型 |
| **`[N]`** | 数组：「含 N 个元素，元素类型为……」 |
| **`(params)`** | 函数：「接受 params，返回……」 |
| **括号 `( )`** | 先解析内部，再与外部连接 |

**方向**：从 **identifier 向外扩**，与螺旋规则 **顺时针** 一致。

### 示例 1：`char *str[10]`

```text
        str
         |
      [ 10 ]          ← str 是数组，10 个元素
         |
        char *         ← 每个元素是指向 char 的指针
```

读法：**str 是一个数组，数组有 10 个元素，每个元素是 char 的指针**。

对比 `char (*p)[10]`：

```text
         p
         |
         *            ← p 是指针
         |
      [ 10 ]          ← 指向的对象是含 10 个 char 的数组
         |
        char
```

读法：**p 是指针，指向含 10 个 char 的数组**。

### 示例 2：`int (*func)(int, char)`

```text
        func
         |
         *            ← func 是指针
         |
    ( int, char )     ← 指向「函数：参数 int,char」
         |
        int           ← 该函数返回 int
```

读法：**func 是指针，指向接受 (int, char) 且返回 int 的函数**。

调用方式：`(*func)(42, 'x');` 或 C 习惯写法 `func(42, 'x');`。

### 示例 3：多层 `char *(*(*x())[])()`

分步（与 **3.8** 同步）：

```text
Step A — 从 x 开始：
    x 是函数 ()，返回 ???

Step B — 返回部分 *(*...)[]()：
    返回指针 → 指向数组 [] → 元素是指针 * → 指向函数 () → 返回 char *

合并：x 是函数，返回「指向数组的指针」；数组元素是「指向函数的指针」；
      那些函数接受 void，返回 char*。
```

图表法价值：**每展开一层画一层**，避免一步读到晕。

### 图表法 vs 螺旋规则

| 方法 | 优点 | 适用 |
|------|------|------|
| **螺旋规则** | 无需纸笔，口算快 | 中等复杂度、面试口述 |
| **图表法** | 嵌套深时不易错 | `signal`、`x()` 级怪物声明 |
| **typedef 拆解** | 可读性最好 | 生产代码、API 设计（**3.5**） |

### 实战步骤（推荐流程）

1. 标出 **identifier**  
2. 画框，按 **3.3 优先级** 向外连 `[ ]`、`( )`、`*`  
3. 最外层写 **存储类**（通常不影响类型结构）  
4. 用一句话 **从 identifier 向外** 读完整句  
5. 用 **demo** 或 `typeof`/编译器 `-Wall` 验证（若写变量定义）  

### 常见错误

| 错误 | 纠正 |
|------|------|
| 从 `int` 开始向右读 | 必须从 **identifier** 开始 |
| 忽略括号 | `(*p)` 改变 `*` 与 `[]` 绑定顺序 |
| 把 `[]` 当成表达式下标 | 声明里 `[]` 表 **数组维度** |
| 函数与函数指针混淆 | 有 `*` 在函数名旁 → 函数 **指针**；无 `*` → 函数本身 |
