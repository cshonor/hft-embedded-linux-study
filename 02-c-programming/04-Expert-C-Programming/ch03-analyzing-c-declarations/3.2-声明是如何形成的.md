# 3.2 声明是如何形成的

## 本节讲什么

任意 C 声明都可拆成 **三个部分**：**存储类说明符**、**基本类型**、**声明符（declarator）**。声明符携带名字，并通过 `*`、`[]`、`()` 递归描述「这个名字是什么」。本节建立拆解模板，为 **3.3 螺旋规则** 提供「从哪开始、绕哪转」的语法基础。

## 要点

### 声明的三段式

```c
[存储类]  基本类型  声明符;
```

| 部分 | 常见内容 | 作用 |
|------|----------|------|
| **存储类** | `auto`、`register`、`static`、`extern`、`typedef` | 链接性、存储期；`typedef` 时整行定义别名而非对象 |
| **基本类型** | `int`、`char`、`void`、`struct foo`、`enum bar`、typedef 名 | 声明符最终「指向/返回/元素」的底层类型 |
| **声明符** | 标识符 + `*` / `[]` / `()` 组合 | 把基本类型变成具体对象类型 |

示例：

```c
static unsigned long int *p;
/* ^^^^^^ 存储类
          ^^^^^^^^^^^^^^^^^ 基本类型
                            ^ 声明符：*p → p 是指针，指向 unsigned long int */
```

### 声明符是递归结构

文法上，声明符 = **指针声明符** | **直接声明符**，可嵌套：

```text
direct-declarator:
    identifier
    ( declarator )          ← 括号改变绑定顺序
    direct-declarator [ ]   ← 数组
    direct-declarator ( )   ← 函数

pointer:
    * type-qualifiers? pointer?
```

因此 `char (*p)[10]` 中：`p` 是 **direct-declarator**，外层 `(*p)` 用括号让 `*` 先绑定到 `p`，再 `[10]` 表示指向数组。

### 多个声明符共享前缀

```c
unsigned long int *p, **pp, arr[10];
```

- `p`：`unsigned long int` 的指针  
- `pp`：指向「`unsigned long int` 指针」的指针  
- `arr`：**不是指针**，是含 10 个 `unsigned long int` 的数组  

**规则**：逗号分隔的每个声明符 **独立** 与左侧「基本类型 + 已读的指针/数组/函数后缀」组合；`*` 只作用于紧挨它的那个名字。

### 存储类与 typedef 的特殊行

```c
typedef int *IntPtr;     /* 定义类型名 IntPtr，不是变量 */
extern int count;        /* 声明对象，可能在其他 TU 定义 */
static void helper(void); /* 内部链接函数 */
```

`typedef` 行中，「存储类」位置被 `typedef` 占用，**声明符里的名字变成类型名**（详见 **3.5–3.7**）。

### 类型限定符

`const`、`volatile`、`restrict` 可出现在基本类型或指针各层：

```c
const int *p;    /* 指向 const int：不能通过 p 改 *p */
int * const p;   /* p 本身 const：不能改 p 指向谁 */
const int * const p; /* 两者都 const */
```

解析时：限定符随 **螺旋规则** 扫到 `*` 时挂到对应指针层（**demo03_const_typedef**）。

### 与表达式的对比

| 上下文 | `*p` | `p[i]` | `f()` |
|--------|------|--------|-------|
| **声明** | p 是指针 | p 是数组（或函数参数退化为指针） | p 是函数 |
| **表达式** | 解引用 | 下标 | 调用 |

同一符号，声明读 **类型结构**，表达式读 **运算**——**ch02 2.3** 的符号过载根源。
