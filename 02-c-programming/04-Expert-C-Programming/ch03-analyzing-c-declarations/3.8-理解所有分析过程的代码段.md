# 3.8 理解所有分析过程的代码段

## 本节讲什么

全书 **声明分析** 的实战章：对书中 **每一道** 经典声明，用 **3.3 螺旋规则** 逐步口播解析，并给出 **typedef 改写** 与 **调用方式**。涵盖基础对比、`x()` 级嵌套、**`a[10]` 与括号位置** 的致命差异，以及面试圣杯 **`signal`**。建议 **先遮答案自推，再对照**。

## 要点

### 基础四组（必会）

#### 1. `char *str[10]`

```text
str 是数组 → 10 个元素 → 每个元素是 char 的指针
```

**中文**：str 是 **含 10 个元素的数组**，元素类型为 **指向 char 的指针**（指针数组）。

```c
char *s = str[0];   /* 第 0 个元素是 char* */
```

#### 2. `char (*p)[10]`

```text
p 是指针 → 指向 → 含 10 个 char 的数组
```

**中文**：p 是 **指针**，指向 **含 10 个 char 的数组**（数组指针）。

```c
char arr[10];
p = &arr;           /* 数组指针取整个数组地址 */
(*p)[0] = 'a';      /* demo01_arr_ptr 同类 */
```

| 声明 | 主体 | 对比 |
|------|------|------|
| `char *str[10]` | str **是数组** | 元素为 `char*` |
| `char (*p)[10]` | p **是指针** | 指向 `char[10]` |

#### 3. `int (*func)(int, char)`

```text
func 是指针 → 指向 → 函数(int, char) → 返回 int
```

**中文**：func 是 **函数指针**，指向 **接受 (int, char)、返回 int** 的函数。

```c
int example(int a, char c) { return a + c; }
func = example;
func(1, 'x');       /* 或 (*func)(1, 'x'); — demo02_func_ptr */
```

typedef：

```c
typedef int (*Func)(int, char);
Func func;
```

#### 4. `int *f(int)` vs `int (*fp)(int)`

| 声明 | 解析 |
|------|------|
| `int *f(int)` | f **是函数**，参数 int，**返回 int\*** |
| `int (*fp)(int)` | fp **是指针**，指向 **接受 int、返回 int 的函数** |

**记忆**：`*` 贴名字且被括号包住 → 函数 **指针**；`*` 在返回类型侧、名字旁无括号 → **返回指针的函数**。

### 进阶：`char *(*(*x())[])()`

**从 identifier `x` 顺时针：**

| 步 | 遇到 | 对 x 说 |
|----|------|---------|
| 1 | `x` | x 是 …… |
| 2 | `( )` | x **是函数**，参数列表 **空** |
| 3 | 向外 | 函数 **返回** …… |
| 4 | `*` | …… **指针** |
| 5 | `[ ]` | …… 指向 **数组**（维度未写 = 不完整/取决于上下文） |
| 6 | 元素 `*` | 数组元素是 **指针** |
| 7 | `( )` | 指向 **函数**，参数空 |
| 8 | 返回 | 函数 **返回 char \*** |

**完整中文**：

> **x 是一个函数**；调用时不带参数；**返回** 一个 **指针**，该指针指向 **数组**；数组 **元素** 是指向 **函数** 的指针；那些函数 **无参数**，**返回 char\***。

**typedef 分层（推荐）：**

```c
typedef char *(*InnerFn)(void);
typedef InnerFn InnerFnArray[];      /* 元素为 InnerFn 的数组 */
typedef InnerFnArray *InnerArrPtr;
typedef InnerArrPtr (*OuterFn)(void);

OuterFn x;
```

### 经典对比：`int *(*a[10])()` vs `int (*(*a)[10])()`

两声明 **仅括号位置不同**，语义 **天差地别**。

#### A. `int *(*a[10])()`

```text
a 是数组[10] → 每个元素是指针 * → 指向函数() → 返回 int*
```

**中文**：a 是 **含 10 个元素的数组**；每个元素是 **函数指针**；所指函数 **无参**，**返回 int\***。

```c
/* a[i] 是函数指针；a[i]() 返回 int* */
```

#### B. `int (*(*a)[10])()`

```text
a 是指针 * → 指向数组[10] → 元素是指针 * → 指向函数() → 返回 int
```

**中文**：a 是 **指针**；指向 **含 10 个元素的数组**；数组 **元素** 是 **函数指针**；所指函数 **无参**，**返回 int**（注意：返回 **int**，不是 int*）。

```text
对比记忆：
  *a[10]  → [] 先绑 a → 「指针的数组」再 * → 函数指针数组
  (*a)[10] → * 先绑 a → 「指向数组的指针」，数组里再放函数指针
```

| 声明 | a 本身 | 第一层 |
|------|--------|--------|
| `int *(*a[10])()` | **数组** `[10]` | 元素：返回 `int*` 的函数指针 |
| `int (*(*a)[10])()` | **指针** | 指向 `[10]` 数组；元素：返回 `int` 的函数指针 |

### 圣杯：`void (*signal(int sig, void (*handler)(int)))(int)`

**分两段读**：外层是 **声明** `signal`；内层 `void (*handler)(int)` 是 **参数类型**。

#### Part 1 — 参数 `void (*handler)(int)`

> handler 是 **指针**，指向 **接受 int、返回 void** 的函数（标准信号处理回调）。

#### Part 2 — 整体 `signal(...)`

从 **identifier `signal`** 出发：

| 步 | 构造 | 含义 |
|----|------|------|
| 1 | `signal` | signal 是 …… |
| 2 | `(int sig, void (*handler)(int))` | signal **是函数**，两参数 |
| 3 | 外层 `(int)` | 该函数 **返回** …… |
| 4 | `*` | …… **函数指针** |
| 5 | `(int)` | 指向的函数 **接受 int**，**返回 void** |

**完整中文**：

> **signal 是一个函数**：  
> - 参数 1：`int sig`（信号编号）  
> - 参数 2：`void (*handler)(int)`（新的处理函数指针）  
> - **返回值**：**函数指针**，指向 **接受 int、返回 void** 的函数（即 **先前的** 处理函数）

**typedef 写法（可读性）：**

```c
typedef void (*SigHandler)(int);

SigHandler signal(int sig, SigHandler handler);
/* signal 返回 SigHandler */
```

**使用示例：**

```c
SigHandler old = signal(SIGINT, my_handler);
/* old 保存旧 handler；my_handler 签名必须为 void my_handler(int) */
```

**面试口述模板**：「signal 是函数；参数是 sig 和 handler；handler 本身是指向 void(int) 的指针；signal 返回 void(int) 的函数指针。」

### 推演习惯（考场/面试）

1. **圈 identifier**  
2. **括号由内向外** 标注「函数 / 指针 / 数组」  
3. **函数** 写清 **参数** 与 **返回**  
4. **数组** 写清 **长度** 与 **元素类型**  
5. 最后用 **typedef 一层层折叠** 验证是否顺口
