# 3.7 typedef struct foo { ... foo; } 的含义

## 本节讲什么

C 语法允许 **结构体标签（tag）** 与 **typedef 类型名** 同名，写成 `typedef struct foo { ... } foo;`。一行里 **三个 `foo` 角色不同**：结构体标签、`typedef` 别名、成员访问时的类型。本节厘清 **tag vs typedef 名**、与 C++ `struct foo` 差异、以及 **匿名 struct + typedef** 的惯用法。

## 要点

### 一行三角色

```c
typedef struct foo {
    int   id;
    char  name[32];
} foo;
/*      ^^^ tag          ^^^ typedef 名（类型别名） */
```

| 位置 | `foo` 的角色 | 用途 |
|------|--------------|------|
| `struct foo {` | **结构体标签（tag）** | `struct foo x;` 合法（C 中） |
| `} foo;` | **typedef 引入的类型名** | `foo x;` 合法，无需写 `struct` |
| 成员函数式写法 | 类型名 | `foo *p;`、`sizeof(foo)` |

**效果**：既可用 **K&R 风格** `struct foo a;`，也可用 **typedef 简写** `foo a;`——在头文件中极常见（Linux 内核、glib、SDK）。

### 与「仅 tag、无 typedef」对比

```c
struct bar {
    int x;
};
struct bar a;      /* 必须带 struct */
/* bar b;           错误：bar 不是类型名，只是 tag */
```

```c
typedef struct bar {
    int x;
} bar_t;
bar_t a;           /* 推荐：类型名与 tag 分离，更清晰 */
struct bar b;      /* 仍可用 tag */
```

**现代风格建议**：typedef 名用 **`foo_t`** 或 **`Foo`**，与 tag `struct foo` 区分，避免三个同名 `foo` 初学者晕头转向。

### 自引用与不完整类型

```c
typedef struct node {
    int         data;
    struct node *next;   /* 必须用 struct node*：此时 typedef node 尚未完成 */
} node;
```

- 结构体 **内部** 自引用：标准写法 **`struct tag *`**  
- C 允许指向不完整类型的指针  
- typedef 名在 `}` 之后才完全生效

### 匿名 struct + typedef

```c
typedef struct {
    int x, y;
} Point;
/* 无 tag，仅 Point 可用；不能写 struct ??? */
```

| 写法 | tag | 自引用 | 典型场景 |
|------|-----|--------|----------|
| `typedef struct foo { } foo;` | 有 | 内部用 `struct foo*` | 传统头文件 |
| `typedef struct { } Point;` | 无 | 需前向声明技巧 | 简单 POD |
| `struct foo { };` 无 typedef | 有 | 必须 `struct foo` | 减少命名空间污染 |

### 与 C++ 的对比（阅读混合代码时）

- C++ 中 `struct foo { }; foo x;` **无需 typedef**——`foo` 自动进类型命名空间  
- C 中 **必须** `typedef` 或 **`struct foo`** 前缀  
- 移植头文件时注意 `#ifdef __cplusplus` 包装

### 易错点

| 错误 | 说明 |
|------|------|
| `sizeof(struct foo)` vs `sizeof(foo)` | typedef 后二者相同 |
| 重复定义 | 同一 TU 不能两次定义同一 tag |
| 与 `enum`/`union` 同名 | 同一作用域 tag 命名空间共享，别撞名 |
| 以为 `foo` 是变量 | 行末 `foo` 是 **类型名**，不是实例 |

### 内核/API 示例（概念）

```c
typedef struct file_operations {
    struct module *owner;
    ssize_t (*read)(struct file *, char __user *, size_t, loff_t *);
    /* ... */
} file_operations;
```

读回调声明 = **3.3 螺旋规则** + **3.5 typedef 命名** 的组合——读懂头文件的基础。
