# 3.3 优先级规则

## 本节讲什么

解析 C 声明的 **唯一通用算法**——**螺旋规则（The Spiral Rule，也称 Right-Left Rule）**：从 **标识符** 出发，**顺时针** 沿声明符移动，按 **`[]` 与 `()` 优先于 `*`** 的顺序读，直到整行声明被消费完毕。本节给出四步流程与优先级表，是 **3.4 图表法** 和 **3.8 全部例题** 的理论基础。

## 要点

### 优先级（绑定顺序）

| 优先级 | 构造 | 含义（读到它时对 identifier 说什么） |
|--------|------|--------------------------------------|
| **最高** | `( )` 括号 | 改变扫描顺序：先解析括号 **内** 的 declarator |
| **高** | `[ N ]` | identifier **是**（或 **元素为**）含 N 个元素的数组 |
| **高** | `( params )` | identifier **是** 接受 params 的 **函数** |
| **低** | `*` | identifier **是指针**，指向……（继续扫得到的类型） |

记忆：**括号和方括号、函数括号像「胶水」，比星号贴得更紧**。

```c
char *str[10];    /* [] 先绑定 str → str 是数组，* 修饰元素 → char 的指针数组 */
char (*p)[10];    /* () 强制 * 先绑定 p → p 是指针，[] 修饰所指 → 指向 char[10] */
```

### 螺旋规则四步

以 `char *(*next)();` 为例（书中经典）：

**Step 0** — 找到 **identifier**（如 `next`）。

**Step 1** — 默认：`next` **是** ……（待补全的类型）。

**Step 2** — **顺时针** 移动，先处理 **括号内**（若有），再处理 **紧贴 identifier 的** `[]`、`()`，最后处理 `*`：

```text
char * ( * next ) ( ) ;
              ↑
         从 next 开始
```

- 遇 `( * next )`：括号内 → `next` **是指针**  
- 遇 `( )`：`next` **是函数**，参数列表空  
- 继续向外：函数 **返回** ……  
- 遇 `*`：返回 **指针**，指向 ……  
- 遇 `char`：…… **char**

**Step 3** — 用英语（或中文）串起来：

> `next` 是一个 **函数**，**返回** 指向 **char** 的 **指针**。

等价 typedef 写法：

```c
typedef char *(*NextFn)(void);
NextFn next;
```

### 四步速查表

| 步骤 | 动作 |
|------|------|
| 1 | 写名字：`X 是 ……` |
| 2 | 顺时针：括号内优先；`[]` → 数组；`( )` 带参数 → 函数；`*` → 指针 |
| 3 | 函数：读 **返回类型**；数组：读 **元素类型**；指针：读 **指向类型** |
| 4 | 重复直到基本类型（`int`、`char`、`void`、typedef 名） |

### 常见对比（必背）

| 声明 | 解析结果 |
|------|----------|
| `int *p` | p 是指向 int 的指针 |
| `int arr[10]` | arr 是含 10 个 int 的数组 |
| `int *p[10]` | p 是含 10 个「int 指针」的数组 |
| `int (*p)[10]` | p 是指向「含 10 个 int 的数组」的指针 |
| `int f(int)` | f 是函数，接受 int，返回 int |
| `int (*fp)(int)` | fp 是指向「接受 int 返回 int 的函数」的指针 |
| `int *f(int)` | f 是函数，接受 int，**返回 int*** |

### 与 C 运算符优先级的区别

**不要混淆**：表达式里 `[]`、`()`、`*` 的优先级与 **声明符绑定规则** 相关但不相同。声明解析 **只用螺旋规则**，不要套「乘号比加减优先」那套。

### 练习建议

1. 先 **遮掉存储类和 typedef**，只练 declarator  
2. 每遇一层，用 **中文半句** 标注（「是指针」「是数组」「是函数，返回……」）  
3. 与 **3.4 框图** 对照：每个框 = 螺旋上的一站
