# 5.5 产生链接器报告文件

## 本节讲什么

链接问题（符号缺失、段重叠、意外链入巨大 `.o`）靠 **猜** 很低效。本节介绍三类 **链接器报告**：**Map 文件**（`-Wl,-Map=`）、**重定位表**（**readelf -r**）、**动态重定位**（**objdump -R**）。与 **5.1 inspect**、**demo04 ld -T** 配合，能 **看见** 最终可执行文件里 **谁占哪段、谁引用谁**。

## 要点

### 为什么需要报告

| 问题 | 报告能帮什么 |
|------|--------------|
| `undefined reference` | Map 里看 **哪些 .o 已链入**、缺哪符号 |
| 二进制意外变大 | Map 里 **按 .o 排序的段大小** |
| 插桩是否生效 | Map / **nm** 看 `malloc` 定义来自哪个文件 |
| 内核 / 裸机布局 | **kernel.lds** + Map 对照 **`_stext` / `_ebss`** |

### Map 文件：`-Wl,-Map=`

```bash
gcc main.c -Wl,-Map=link.map -o main
# 或
gcc main.o -L. -ladd -Wl,-Map=link.map -o demo_static
```

**link.map** 典型内容（因 ld 版本略有差异）：

```text
Memory Configuration
Linker script and memory map
 .text           0x...   0x...   main.o
                              libadd.a(add.o)
 .data           ...
 .bss            ...
```

**阅读技巧**：

1. 搜 **`main.o`**、**`libadd.a`** —— 确认 **5.3** 按需提取是否发生；
2. 看 **`.bss` 汇总** —— 未初始化全局总大小；
3. 找 **`LOAD`** / **`AT>`** —— 段在文件与内存中的偏移；
4. 内核链接：**`ld -T kernel.lds -Map=kernel.map ...`**（**demo04**）。

```bash
cd demo/demo04_linker_script
make
grep -E '_stext|_ebss|\.text|\.bss' kernel.map
```

### 重定位：`readelf -r`

针对 **未链接或部分链接** 的 `.o` / `.so`：

```bash
readelf -r main.o
```

输出 **Relocation section**，例如：

```text
Offset   Info   Type           Sym.Value  Sym. Name
00000010 000008 R_X86_64_PC32    ...        puts
```

含义：在 `.text` 偏移 `0x10` 处，需用 **`puts` 符号** 做 **PC 相对** 重定位。

**demo01_four_stage**：

```bash
make inspect
# nm main.o
# readelf -S main.o
# readelf -r main.o
```

| 命令 | 对象 |
|------|------|
| **readelf -S** | 段表（**.text .data .bss**） |
| **readelf -r** | **重定位** 条目（链接前） |
| **readelf -s** | 符号表（与 **nm** 类似） |

### 动态重定位：`objdump -R`

对已链接的 **动态可执行文件 / .so**：

```bash
objdump -R ./main_dyn
# 或
readelf -r ./main_dyn    # 动态段 reloc 也可
```

显示 **`.got.plt`** 等处的 **RUNTIME** 重定位项 —— 与 **5.2 lazy binding** 对应。

| 工具 | 典型输入 | 看什么 |
|------|----------|--------|
| **readelf -r file.o** | 目标文件 | 链接前待填坑位 |
| **objdump -R a.out** | 动态可执行 | 运行时 PLT/GOT 解析 |
| **-Wl,-Map=** | 链接后 | 全局布局与成员来源 |

### 与 `nm` 对照

```bash
nm -n main              # 按地址排序符号
nm -u main.o            # 仅 undefined
nm -A libadd.a          # 归档内每个 .o 的符号
```

**Map + nm** 组合流程（排错模板）：

```text
1. 编译到 .o，nm -u 看 undefined
2. readelf -r 看重定位依赖
3. 链接 -Wl,-Map=，在 map 中确认库成员是否被提取
4. nm 最终二进制，确认符号 T/D/B 段归属
```

### 链接脚本与 Map（demo04）

**kernel.lds** 定义：

```text
ENTRY(_start)
_stext / _etext  — .text
_sbss  / _ebss   — .bss（NOBITS 区域边界）
```

```bash
ld -T kernel.lds -Map=kernel.map start.o -o kernel.elf
readelf -S kernel.elf
```

C 中 **`extern char _sbss, _ebss;`** 可计算 BSS 长度 `(uintptr_t)&_ebss - (uintptr_t)&_sbss`（教学；正式内核用宏）。

### 其它有用选项（了解）

```bash
gcc -Wl,--print-map          # map 打到 stderr
gcc -Wl,--trace-symbol=foo   # 追踪符号 foo 被谁解析
gcc -Wl,--cref                 # 交叉引用列表（部分 ld 版本）
```
