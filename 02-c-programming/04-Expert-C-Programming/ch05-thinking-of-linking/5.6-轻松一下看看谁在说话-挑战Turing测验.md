# 5.6 轻松一下——看看谁在说话：挑战 Turing 测验

## 本节讲什么

本章 **Light Relief**：作者提到 **图灵测验（Turing Test）** —— 评判者通过 **文本对话** 判断对方是人还是机器。书中趣闻用于 **换脑**：刚啃完链接、符号、Map 文件，用哲学梗放松。本节简述测验背景、与 **「程序 / 链接 / 符号」** 的松散类比，并点出 **C 程序员版幽默** —— 无新语法，不考操作。

## 要点

### 图灵测验是什么

**Alan Turing**（1950 论文 *Computing Machinery and Intelligence*）提出：**模仿游戏** —— 若评判者在看不见对方的情况下，无法可靠区分 **人** 与 **机器** 的文本回答，则可认为机器具有智能（操作定义，非哲学终判）。

```text
  评判者 ←──文本──→ 房间 A（人？）
                 └──文本──→ 房间 B（程序？）
```

关键：**仅看对话内容**，不看外观或声音。

### 与「链接」的松散类比（书中幽默精神）

程序员常开玩笑说：

| 对话现象 | 冷笑话式解读 |
|----------|--------------|
| 回答 **完美符合手册** | 像 **标准库文档** —— 规整、无个性 |
| 回答 **含糊但自信** | 像 **未定义行为** —— 听起来对，跑起来错 |
| 回答 **`undefined reference`** | 明显是 **链接器** 在说话，不是人 |
| 回答 **强符号覆盖弱符号** | 像辩论里 **声音大的人赢**（**5.1 demo03**） |

这些 **不是** 图灵原意，而是 Expert C 风格：**用梗巩固记忆** —— 你刚学的 **符号解析**，也可以当成「谁在提供这个答案（定义）」。

### 「看看谁在说话」

标题双关：

1. **图灵测验**：谁在说话 —— 人还是 AI？
2. **调试链接**：**crash 时谁在说话** —— 是你的 `main.o`、错误的 **`extern char *arr`**（**ch04 4.2**），还是 **LD_PRELOAD 的 malloc**（**5.4**）？

```bash
nm a.out | grep ' malloc'    # T 符号来自哪段、哪个目标？
ldd a.out                    # 运行时「说话」的共享库有哪些？
```

### 历史上的趣闻（文化背景）

- 早期聊天程序 **ELIZA**（1960s）用模式匹配假装心理治疗师，已能让部分人 **误以为** 是人 —— 说明 **表面像人 ≠ 真理解**。
- 现代大模型在部分盲测中表现强劲，但 **图灵测验是否仍有效** 学界仍有争议 —— 与本章技术无关，作 **背景知识**。

### 程序员自检清单（轻松版）

若有人声称「这段 C 肯定能链过」，你可以像 **评判者** 一样问：

1. **声明和定义是同一类型吗？**（**4.4**）还是只有 **符号名** 相同？
2. **`-lm` 在 `main.o` 后面吗？**（**5.3**）
3. **BSS 零初始化** 你算进逻辑了吗？（**5.1 demo01**）
4. **`LD_PRELOAD` 有没有换过 libc？**（**5.4**）
5. **Map 文件里** 真的链进了你以为的那个 `.o` 吗？（**5.5**）

答不上来 —— 可能不是「人话」，是 **链接器在跟你玩 Turing 测验**。
