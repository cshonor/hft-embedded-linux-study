# 1.5 今日之ANSI C

## 本节讲什么

**ANSI C**（1989，ISO 称 **C90/C89**）是 C 第一次正式标准化，把 K&R 时代的惯例与编译器实践写进规范。核心变化：**函数原型**、**`void`**、**`const`**、**标准库** 清单、**实现定义/未定义行为** 分类。现代 `-std=c11/c17/c23` 均在此基础上演进；读内核/DPDK/嵌入式代码时，「符合哪个标准」决定你能用哪些语法。

## 要点

### 标准演进简表

| 标准 | 年份 | 别名 | 里程碑 |
|------|------|------|--------|
| **C89/C90** | 1989/1990 | ANSI C | 原型、`const`、`void`、标准库 |
| C99 | 1999 | ISO C99 | `//` 注释、`long long`、VLA、`<stdint.h>` |
| C11 | 2011 | ISO C11 | `_Generic`、`_Atomic`、threads（可选） |
| C17/C18 | 2018 | 缺陷修正，无大特性 | |
| C23 | 2024 | `#embed`、attributes 等 | 逐步普及中 |

### C89 核心语言特性

**1. 函数原型（prototype）**

```c
double sqrt(double x);           /* 声明 */
int printf(const char *fmt, ...); /* varargs 用 ... */

void noop(void);                 /* 无参必须用 void，不是空括号 */
```

**2. `void` 类型**

```c
void exit(int status);    /* 无返回值 */
void *malloc(size_t n);   /* 通用指针 */
```

- 空括号 `()` 在 C89 中 **不是**「无参」，而是 **旧式声明**（参数未指定）——仍与 K&R 兼容

**3. `const`**

```c
const int *p;       /* 指向常量的指针：*p 不可改 */
int *const p;       /* 指针本身是常量 */
const char *msg = "hello";
```

- 增强接口契约；字符串字面量类型在 C89 为 `char *`，C++ 及良好 C 实践用 `const char *`

**4. 标准库（hosted 环境）**

| 头文件 | 主要功能 |
|--------|----------|
| `<stdio.h>` | I/O、`FILE` |
| `<stdlib.h>` | `malloc`/`free`/`exit`/`atoi` |
| `<string.h>` | `memcpy`/`strlen` |
| `<math.h>` | 数学函数 |
| `<stddef.h>` | `size_t`、`NULL`、`offsetof` |
| `<limits.h>` / `<float.h>` | 实现限制与浮点属性 |

- **freestanding** 实现只需提供 `<stddef.h>`、`<stdint.h>`（C99 起）等最小集；**无 `malloc`/stdio**

### 与 K&R 的关键差异

```c
/* K&R：危险 */
int func();

/* ANSI：明确 */
int func(int a, int b);
void func(void);
```

```c
/* K&R：省略返回类型 → int */
foo() { return 1; }

/* ANSI/C99+：必须显式返回类型 */
int foo(void) { return 1; }
```

### 可移植性实践

```c
#include <stdint.h>   /* C99：int32_t 等固定宽度类型 */
#include <stdbool.h>  /* C99：bool/true/false */

#if __STDC_VERSION__ >= 199901L
/* C99 及以上可用特性 */
#endif
```

- Linux 内核用 **GNU C 扩展 + 自有风格**，不是严格 ISO C
- DPDK 指定 GCC/Clang 版本与 `-std=c11` 等标志

### 编译示例

```bash
gcc -std=c89 -pedantic -Wall demo/demo01_kr_vs_ansi.c -o demo01
gcc -std=c11 -Wall demo/demo01_kr_vs_ansi.c -o demo01_c11
echo | gcc -E -dM -std=c11 - | grep __STDC_VERSION__
```
