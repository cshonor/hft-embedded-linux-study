# 1.9 阅读ANSI C标准，寻找乐趣和裨益

## 本节讲什么

Peter van der Linden 建议：把 ISO C 标准当 **参考答案** 而非教材——遇到「编译器为什么这么报」「这算不算 UB」时查原文。本节给出 **打开标准、定位条款、读 footnote、对照编译器** 的实用方法；读标准一次枯燥，长期收益是 **少踩 UB、面试降维打击**。

## 要点

### 获取标准文本

| 来源 | 说明 |
|------|------|
| **ISO/IEC 9899** | 官方 PDF（付费）；C89/C99/C11/C17/C23 |
| **WG14 草案（N 系列）** | 如 open-std.org 上 **C99/C11 draft**，与正式版极接近，学习够用 |
| **cppreference.com** | 条款摘要 + 示例；**引用时仍以标准为准** |
| **Compiler 文档** | GCC **Manual → C Extensions / Implementation-defined** |

```bash
# 本地用 ripgrep 搜草案 PDF 转文本（若有）
rg "undefined behavior" c11-n1570.txt
rg "implementation-defined" c11-n1570.txt | head
```

### 推荐阅读顺序（非从头到尾）

1. **第 4 章**：Hosted/Freestanding、程序执行模型

3. **第 6.7**：声明、原型、链接
4. **第 7 章**：你常用的库函数（`malloc`、`printf`、`memcpy`）
5. **Annex J.2**：UB 示例速查
6. **Annex C**：Translation limits

### 如何定位一条规则

**场景：`a[i] = i++` 是否合法？**

1. 查索引 **「sequence points」** / **「sequenced before」**（C11 用语）
2. 读 **6.5 表达式** 中关于 **副作用与求值顺序**
3. 结论：未 sequenced 的副作用 → **undefined behavior**

**场景：`char` 默认 signed 吗？**

1. 查 **5.2.4.2.1** `Character types`
2. 标注 **implementation-defined**
3. 查 GCC：`-fsigned-char` / `-funsigned-char`

### 标准文本的阅读技巧

| 技巧 | 作用 |
|------|------|
| 区分 **shall / shall not / may** | shall=必须；may=可选 |
| 读 **footnote** | 常含 UB 或历史说明 |
| 看 **EXAMPLE** | 非 normative，但帮助理解 |
| 对照 **DR（Defect Report）** | 标准勘误；cppreference 有时标注 |
| 条款号 **§6.5.2.3** | 跨版本条款号可能变，以版本为准 |

### 与编译器交叉验证

```c
/* 实验：UB 时编译器常无警告 */
int f(int *p) {
    return *p;   /* p==NULL → UB；开启 -Wnull-dereference（Clang） */
}
```

```bash
gcc -std=c11 -Wall -Wextra -O2 test.c
clang -std=c11 -Wall -Wextra -fsanitize=undefined test.c
```

- **标准说 UB** + **Sanitizer 报错** → 理解加深
- **标准说 UB** + **编译通过** → 正常；不能因「能编过」认为合法

### 「乐趣和裨益」实例

**裨益 1：面试**

> 「`memcpy` 重叠区域用什么？」→ 标准：`memmove`；重叠 `memcpy` 是 **UB**。

**裨益 2：读内核**

> `list_for_each` 宏依赖 GCC 扩展；查标准知 **ISO C 没有 statement expression**。

**裨益 3：避免「安静改变」**

> 升级 `-std=c11` 后旧代码行为变——标准 **Annex J** + **1.10** 预警。
