# 1.2 C语言的早期体验

## 本节讲什么

C 的许多「怪癖」不是设计失误，而是 **1970 年代 PDP-11 硬件与编译器能力** 的直接映射。本节梳理数组下标从 0 开始、类型与硬件对应、`auto` 存储类、**数组到指针的衰变**、**float 提升**、`register` 提示等早期体验——它们至今仍是 C 面试与 UB 讨论的高频来源。

## 要点

### 数组下标从 0 开始

- C 继承自 B：**指针 + 偏移** 模型，`a[i]` 等价于 `*(a + i)`
- 若下标从 1 开始，每次访问需多一次减法；从 0 开始代码生成更直接
- 与数学下标习惯不同，但符合 **地址算术** 语义

### 类型映射硬件

| C 类型 | 典型硬件映射 | 注意 |
|--------|--------------|------|
| `char` | 8 位字节 | `CHAR_BIT` 至少 8；`signed/unsigned char` 实现定义 |
| `int` | 机器「自然字长」 | 16/32/64 位随平台变；**可移植代码用 `<stdint.h>`** |
| `short` / `long` | 更短/更长整数 | 仅保证相对关系，不保证绝对位数 |
| `float` / `double` | IEEE 754 浮点 | 运算中常 **提升为 double**（见下） |

### `auto` 存储类

- 局部变量默认就是 `auto`：**栈上分配**，块结束即销毁
- 显式写 `auto` 几乎只在 K&R 时代常见；现代代码通常省略
- 对比：`static`（静态存储期）、`register`（寄存器提示）

### 数组衰变（array decay）

```c
void f(int arr[]);   /* 形参实际类型是 int *，不是数组 */
int a[10];
f(a);                /* a 衰变为指向首元素的指针 */
sizeof(a);           /* 整个数组：40（假设 int 4 字节）*/
/* 传参后 sizeof 在 f 内对参数是 指针大小 */
```

- 数组名在大多数表达式中 **衰变为指向首元素的指针**
- **Expert C ch04** 专论「数组 ≠ 指针」；衰变是混淆根源

### float 提升（default argument promotion）

```c
float f;
double g = f + 1.0;   /* f 先提升为 double 再运算 */
printf("%f", f);      /* varargs 中 float 也会提升为 double */
```

- K&R / ANSI 前：表达式中 `float` 常 **提升为 `double`**
- 影响 **可变参数**（`printf` 等）与混合精度运算；**C99 后部分规则有调整**

### `register` 建议

```c
register int counter;   /* 仅 hint；编译器可忽略 */
```

- 请求编译器尽量放入寄存器，**不保证**
- 取 `register` 变量地址在 C89 中非法；现代编译器优化已很强，少用
- 嵌入式/DPDK 热路径有时仍见，但 **volatile**、**restrict**（C99）更常见

### 编译器优先的连锁效应

```text
硬件字长/寻址方式 → 类型宽度与对齐
    ↓
简单 parser → 数组衰变、隐式转换
    ↓
小内存编译器 → 宏而非 inline、stdio 外置库
```

## 与后续衔接

- **1.4**：K&R 时代无原型，实参按 **default promotion** 传递，类型错误静默发生
- **Expert C ch02**：隐式转换、整型提升、有符号溢出等「语言特性」
- **Expert C ch04/ch09/ch10**：数组衰变 vs 数组语义、指针算术
- **K&R ch5**：指针与数组；**Embedded C ch05**：栈上 `auto` 局部与栈帧
- **内核**：内核少用浮点；DPDK 用固定宽度类型（`rte_*`）规避 `int` 宽度问题
