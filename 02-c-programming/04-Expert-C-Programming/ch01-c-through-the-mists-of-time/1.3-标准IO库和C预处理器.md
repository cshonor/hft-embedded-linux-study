# 1.3 标准I/O库和C预处理器

## 本节讲什么

C 语言故意保持 **极小内核**：输入输出不在语法里，而在 **标准 I/O 库（stdio）** 中；条件编译与文本替换则交给 **C 预处理器（cpp）**。本节理解 `#include` / `#define` / `#ifdef` 的设计意图，以及宏 **纯文本替换** 带来的经典陷阱——这与编译器优先、跨平台移植策略一脉相承。

## 要点

### stdio 库：语言外的 I/O 层

| 设计选择 | 原因 |
|----------|------|
| I/O 不进语言语法 | 内核/嵌入式可无 stdio（**freestanding**） |
| `FILE *` 抽象 | 统一文件、终端、管道；缓冲由库管理 |
| `printf`/`scanf` 家族 | 格式化 I/O；与 **varargs**、float 提升绑定 |
| 缓冲 I/O | 减少系统调用；`setbuf`/`fflush` 控制行为 |

```c
#include <stdio.h>

int main(void)
{
    FILE *fp = fopen("data.txt", "r");
    if (fp) {
        int c;
        while ((c = fgetc(fp)) != EOF)
            putchar(c);
        fclose(fp);
    }
    return 0;
}
```

- **hosted 环境**：必须有标准库；**freestanding**（内核、裸机）通常只提供 `memcpy` 等子集
- Linux 用户态 `printf` 最终经 **write 系统调用**；内核用 `printk` 而非 stdio

### 预处理器：编译前的文本工具

```c
#include <stdio.h>          /* 字面插入头文件内容 */
#include "local.h"          /* 先搜当前目录，再系统路径 */

#define PI 3.14159
#define MAX(a, b) ((a) > (b) ? (a) : (b))   /* 参数加括号 */

#ifdef DEBUG
#  define LOG(x) fprintf(stderr, x)
#else
#  define LOG(x) ((void)0)
#endif

#ifndef MYHDR_H
#define MYHDR_H
/* 头文件守卫，防重复包含 */
#endif
```

| 指令 | 作用 |
|------|------|
| `#include` | 文本包含 |
| `#define` / `#undef` | 宏定义/取消 |
| `#ifdef` / `#ifndef` / `#else` / `#endif` | 条件编译 |
| `#if` / `#elif` | 表达式条件（C99 起更强大） |
| `#pragma` | 实现定义/编译器扩展（见 **1.11**） |
| `#error` / `#warning` | 编译期报错/警告 |

### 宏陷阱（必考）

**1. 缺少括号 — 运算符优先级**

```c
#define BAD_SQUARE(x) (x * x)
#define SQUARE(x)     ((x) * (x))

int a = 3;
BAD_SQUARE(a + 1);   /* 展开: a+1*a+1 → 3+3+1 = 7 */
SQUARE(a + 1);       /* 展开: ((a+1)*(a+1)) → 16 */
```

**2. 多语句宏**

```c
#define INC_TWICE(x)  x++; x++   /* 危险：无 do-while(0) 包裹 */
/* 正确写法：*/
#define INC_TWICE_SAFE(x) do { (x)++; (x)++; } while (0)
```

**3. 副作用重复求值**

```c
#define MAX(a, b) ((a) > (b) ? (a) : (b))
int i = 0;
MAX(i++, j++);   /* i/j 可能被增减多次 → 未定义行为 */
/* 应改用 inline 函数或 GCC statement expression */
```

**4. 与类型系统无关**

- 宏不做类型检查；**ANSI 原型 + inline/static inline（C99）** 更安全


### `#define` vs `const` / `enum`

| 方式 | 特点 |
|------|------|
| `#define` | 编译前替换；无类型；可 `#ifdef` |
| `const` | 有类型；调试器可见；C 中仍可能当 compile-time 常量 |
| `enum` | 整型常量；作用域规则与普通标识符相同 |
