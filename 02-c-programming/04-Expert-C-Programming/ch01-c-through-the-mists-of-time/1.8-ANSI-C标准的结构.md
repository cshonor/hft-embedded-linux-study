# 1.8 ANSI C标准的结构

## 本节讲什么

ISO C 标准不是教程，而是一份 **契约**：规定编译器、标准库、程序三者边界。本节拆解标准文档的 **层次结构**——语法约束、语义规则、标准库描述、**implementation-defined** 清单、**undefined behavior** 条款——为 **1.9** 直接读标准打地图。

## 要点

### 标准文档主要部分（以 C11 为例）

| 部分 | 内容 |
|------|------|
| **1–4 章 引言与约定** | 术语、符合性、翻译/执行环境 |
| **5 章 语法（Grammar）** | 形式语法；与语义分离 |
| **6 章 约束（Constraints）** | **编译期必须诊断** 的规则（如类型不匹配） |
| **6 章 语义（Semantics）** | 合法程序的含义 |
| **7 章 库** | 每个头文件：宏、类型、函数、**前置条件/UB** |
| **Annex C** | Translation limits |
| **Annex J**（C99 起） | **Library introduction**、常见 UB 汇总 |
| **索引** | 按术语查条款号 |

```text
标准文档
├── 语言（第 5–6 章）
│   ├── Constraints → 违反则编译失败（符合实现须诊断）
│   └── Semantics   → Defined / Impl-defined / Undefined
├── 标准库（第 7 章）
│   └── 每个函数：参数、返回值、错误、UB（如 NULL 指针）
└── 附录
    ├── Annex C：编译限制
    └── Annex J：UB 示例索引
```

### 三层「规则强度」

**1. 诊断规则（Constraints）**

```c
int x = "hello";     /* 约束违反：编译器必须报错 */
return 1.0;          /* int 函数返回 double：约束/警告 */
```

**2. 语义 — 完全定义**

```c
/* 6.5.6 数组下标：定义良好 */
int a[5];
a[4] = 0;            /* OK */
```

**3. 语义 — 灰色与黑色**

| 类型 | 标准表述 | 程序员应对 |
|------|----------|------------|
| Implementation-defined | 「实现应文档化行为」 | 查编译器手册 |
| Unspecified | 「合法但未指定哪一种」 | 不依赖顺序 |
| Undefined | 「无限制」 | **绝对避免** |

### 标准库章节的读法

以 `strcpy` 为例，标准会写：

- `#include <string.h>`
- 原型：`char *strcpy(char *restrict s1, const char *s2);`
- **Preconditions**：`s1` 指向足够空间；否则 **undefined**
- **Returns**：`s1`

→ 库函数 UB 与语言 UB **同等重要**（`printf` 格式串、`free` 二次释放等）。

### Implementation-defined 在标准中的位置

- 散落在语义条款中，以 **「which is implementation-defined」** 标注
- 例如：`sizeof(int)`、`CHAR_BIT`、`malloc(0)` 的行为
- **Annex L**（C11）等汇总 **标准库实现定义** 行为

### Undefined behavior 在标准中的位置

- 几乎每章都有 **「undefined behavior」** 脚注
- C99 起 **Annex J.2** 列出 **UB 示例**（非 exhaustive）
- 编译器对 UB 的利用：**优化假设 UB 不发生** → 「删代码」类 bug

```c
int len = strlen(s);
if (len > 0 && s[len] != '\0') { /* 若 len==strlen，s[len] 已是 '\0' 或越界 UB */
    /* 编译器可能认为 UB 路径不存在，整段优化掉 */
}
```

### Hosted vs Freestanding 在标准中的划分

| 环境 | 必须提供的头文件（C11） |
|------|-------------------------|
| **Freestanding** | `<float.h>`, `<iso646.h>`, `<limits.h>`, `<stdalign.h>`, `<stdarg.h>`, `<stdbool.h>`, `<stddef.h>`, `<stdint.h>`, `<stdnoreturn.h>` |
| **Hosted** | 上述 + 完整标准库（stdio, stdlib, string, …） |

- 内核/DPDK 多数按 **freestanding + 自有库** 理解
