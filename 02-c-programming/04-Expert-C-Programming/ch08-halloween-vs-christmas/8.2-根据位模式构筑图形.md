# 8.2 根据位模式构筑图形

## 本节讲什么

书中用 **ASCII 位图**（把字符矩阵当作像素）展示：**同一块内存的位模式，在不同解释下可以是图形、整数或指令**。这直接依赖 **按位与 `&`**、**移位 `<<` `>>`** 和 **掩码（mask）**——而 **`&` 的优先级低于 `==`**，引出了本章第一个经典 **「万圣节问题」** 的前奏：程序员以为在测 **某位是否为 1**，实际却在写 **`val & (mask == 1)`**。

## 要点

### 位模式与图形

```c
/* 假设每个 char 是一行 8 像素，1 表示亮 */
unsigned char row = 0b10110100;  /* 或 0xB4 */
if (row & 0x80) putchar('#'); else putchar('.');
```

- **硬件视角**：内存是一串 **bit**；**`&` 掩码** 提取字段，**`|`** 合并， **`^`** 翻转， **`<<` `>>`** 对齐。
- **软件视角**：必须 **加括号** 表达意图；不能依赖「读起来像数学」的直觉。

### 优先级陷阱：`val & mask == 1`（万圣节问题之一）

**`==` 优先级高于 `&`**（见 **8.8** 表、**[cheatsheet](./operator-precedence-cheatsheet.md)**）：

```c
/* 意图：测试 val 的某位（mask）是否为 1 */
if (val & mask == 1)   /* ❌ 实际：(val & (mask == 1)) */
    ...

/* mask == 1 为 0 或 1；val & 0/1 几乎永远不是「测位」 */
```

| 表达式 | 解析 | 常见后果 |
|--------|------|----------|
| **`val & mask == 1`** | **`val & (mask == 1)`** | 逻辑完全偏离 |
| **`(val & mask) == 1`** | 测 **mask 覆盖的位** 是否等于 1 | ✅ 意图 |
| **`val & mask`** | 非零即真（测 **任意被置位**） | 常用写法 |

**万圣节问题（完整版，8.8）**——**`=` 与 `==` 混淆**：

```c
if ((a = b) == 0)   /* ✅ 赋值 b 给 a，再比较是否为 0 */
    ...

if (a = b == 0)     /* ❌ 解析：a = (b == 0)；a 只能是 0 或 1 */
    ...
```

**工程规则 — Yoda 条件**（把常量在左，防手滑成赋值）：

```c
if (0 == a) { ... }           /* 写成 if (0 = a) 编译失败 */
if (1 == (val & mask)) { ... } /* 测位 */
```

### 位运算速查

| 运算 | 含义 | 典型用途 |
|------|------|----------|
| **`x & mask`** | 保留 mask 为 1 的位 | 取字段、测位 |
| **`x \| flags`** | 置位 | 打开选项 |
| **`x & ~mask`** | 清位 | 关闭选项 |
| **`x ^ mask`** | 翻转 mask 位 | 切换标志 |
| **`x << n` / `>> n`** | 移位 | 乘除 2ⁿ、对齐 |

**注意**：**`>>` 对有符号负数** 可能是 **算术右移**（实现定义/ UB 边界，**C11** 对负值右移有明确规定）；位图代码优先 **`unsigned`**。

### 与 `031` 八进制笑话的联系（8.1）

位常量 **`0377`**（八进制）= **`0xFF`**（十六进制）= **255**——再次说明：**前导 `0` 改变数值解释**。写掩码时用 **`0xFF`** 或 **`(1u << 7) - 1`**，避免 Portzebie 式混乱。
