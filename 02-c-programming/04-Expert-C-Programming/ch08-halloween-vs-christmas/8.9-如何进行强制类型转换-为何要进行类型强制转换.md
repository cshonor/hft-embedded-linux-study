# 8.9 如何进行强制类型转换，为何要进行类型强制转换

## 本节讲什么

**强制转换（cast）** `(type)expr` 告诉编译器：**按另一种类型解释位模式或做数值转换**。本节讲 **何时必须 cast**、**何时 cast 是 UB 或 portability 陷阱**、以及 **cast 与运算符优先级** 的交互——**`(int)*p++` 解析为 `((int)(*p))++` 还是 `(int)((*p)++`？** 理解 **8.3 隐式转换** 与 **8.8 优先级** 后，cast 是 **程序员主动接管的「类型在等待中变化」**。

## 要点

### 何时需要 cast

| 场景 | 示例 | 说明 |
|------|------|------|
| **显式窄化/符号变化** | **`(uint8_t)x`** | 文档化 **只取低 8 位** |
| **void* 还原** | **`(struct foo*)p`** | **`malloc` 返回** 在 C 中 **不需 cast**；C++ 需要 |
| **算术混合** | **`(double)num / den`** | 强制 **浮点除法** |
| **接口/旧 API** | **`(void*)cb`** | 回调 **擦除类型** — 尽量 **少** |
| **位 reinterpret** | **`(unsigned)(signed)x`** | **实现定义** 部分场景 |

### 危险 cast（与 ch07 内存）

```c
float f = 1.0f;
int *ip = (int *)&f;      /* 严格别名 violation → UB（除非 char* 字节访问） */
*(int *)&f = 0;           /* 类型双关 — 勿在生产代码使用 */

struct { int a; } s;
int *p = (int *)&s;       /* 可能违反对齐/别名 */
```

**合法字节视角**：**`unsigned char *cp = (unsigned char *)&obj`** — **memcpy** 语义，**ch07** 序列化。

### cast 与优先级

**Cast 属于一元运算符，优先级高、结合性右**：

```c
(int)*p++        /* (int)(*(p++))  — 先 p++ 再解引用再转 int */
(int)(*p)++      /* 非法：不能对 cast 结果 ++（非 lvalue） */

(void)(printf("hi\n"), 0);  /* 逗号表达式 cast 成 void 吞掉值 */
```

**与 8.2 位运算**：

```c
(unsigned)val & mask   /* ((unsigned)val) & mask */
(unsigned)(val & mask) /* 先 & 再转 unsigned — 不同！ */
```

**永远加括号** 表达 **cast 作用范围**。

### 隐式 vs 显式（8.3 对照）

| | **隐式** | **显式 cast** |
|--|----------|---------------|
| 触发 | **整型提升、赋值转换** | 程序员 **`(type)`** |
| 可见性 | 易忽略 | 代码里 **显眼** |
| 责任 | 编译器 | **程序员保证合法** |

**原型（8.4）** 失败处常靠 **cast  silence warning** — **掩盖问题**：

```c
foo((int)ptr);   /* 若 foo 期望 int 而传指针 — 灾难 */
```

### 工程规则

- **cast 不是万能胶**；优先 **改声明/改 API**
- **禁止** **`(type)ptr`** 解 **不对齐** 对象（**ch07 SIGBUS**）
- **`const`  cast away** — **`(char*)const_ptr`** 仅 **与只读 API 对接** 时用；写入 **UB**
- 用 **`static_cast` 等价思维**：**能否用更安全的 helper 代替？**（C 无 static_cast，靠 **函数**）
