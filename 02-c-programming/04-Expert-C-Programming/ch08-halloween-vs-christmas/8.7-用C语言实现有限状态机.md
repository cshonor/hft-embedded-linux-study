# 8.7 用 C 语言实现有限状态机

## 本节讲什么

**有限状态机（FSM）** 用 **状态 + 输入 → 下一状态/动作** 描述解析器、协议、词法分析。本节展示 C 中两种主流写法：**`switch` 状态变量** 与 **表驱动（state table）**——清晰、可测、无 **goto  spaghetti**（除非刻意 **Duff's device** 优化）。FSM 代码 **大量依赖 `==`、`&&`、括号**；是 **8.8 优先级** 与 **8.6 逐字符输入** 的 **正面工程范例**：**规则写清楚，比硬件状态机还啰嗦，但可维护**。

## 要点

### 状态变量 + `switch`

```c
typedef enum { START, IN_NUM, IN_ID, DONE } State;

State st = START;
int ch;

while (st != DONE && (ch = getchar()) != EOF) {
    switch (st) {
    case START:
        if (isalpha(ch))      st = IN_ID;
        else if (isdigit(ch)) st = IN_NUM;
        else if (ch == ' ')     /* 保持 START */;
        else                    st = DONE;
        break;
    case IN_NUM:
        if (!isdigit(ch)) { ungetc(ch, stdin); st = DONE; }
        break;
    case IN_ID:
        if (!isalnum(ch)) { ungetc(ch, stdin); st = DONE; }
        break;
    default:
        st = DONE;
        break;
    }
}
```

**要点**：

- **`switch` 贯穿（fall-through）** 必须 **`break`** 或 **注释 `/* FALLTHROUGH */`**（**ch02**）
- 条件用 **`&&` 短路**（**8.8**）： **`while (st != DONE && (ch = getchar()) != EOF)`** — 注意 **`=` 在括号内** 是 **赋值**，不是 Halloween 陷阱

### 表驱动 FSM

```c
typedef void (*Action)(int ch);

typedef struct {
    State next;
    Action act;
} Transition;

/* current_state][input_class] → Transition */
static const Transition table[STATE_COUNT][CLASS_COUNT];
```

| 方式 | 优点 | 缺点 |
|------|------|------|
| **`switch`** | 直观、易 debug | 状态多时臃肿 |
| **表驱动** | 数据与逻辑分离、易可视化 | 间接调用、表大小 |

### 与 8.8 运算符纪律

FSM 条件 **反面教材**（勿写）：

```c
if (st & READY == 1)     /* == 高于 & → st & (READY==1) */
if (a = ch == 'x')       /* Halloween：a 只能是 0/1 */
```

**正面写法**：

```c
if ((st & READY) != 0)
if (ch == 'x') { a = 1; } else { a = 0; }
```

### 测试策略

- **表驱动**：对 **`table[s][c]`** 单元测试， **不依赖 `getchar`**
- **输入串**： **`const char *p = "123 abc";`** 循环 **` *p++`**
- **覆盖 `ungetc` 路径**：输入 **边界字符** 后 **断言状态回退**
