# 8.6 不需要按回车键就能得到一个字符

## 本节讲什么

标准 **`getchar()`** 在 **行缓冲（canonical）** 终端上 **必须读到换行才返回**——与 **FSM（8.7）** 逐字符解析的需求冲突。本节介绍 **终端 raw/cbreak 模式**（ **`termios`** ）、**`getchar`/`getc` 返回 `int` 而非 `char`**（ **`EOF` 与 0xFF`** ）、以及 **`ungetc`** 把字符 **推回流** 的技巧。这是 **原型正确（8.4）仍可能写错** 的典型案例：**类型用错一位，逻辑全毁**。

## 要点

### 行缓冲 vs 即时输入

| 模式 | 行为 | 典型场景 |
|------|------|----------|
| **Canonical（规范）** | 编辑到 **Enter** 才提交一行 | shell、普通文本输入 |
| **Cbreak** | 关闭行编辑， **按键即达程序** | vi、游戏 |
| **Raw** | 关闭 **回显、信号字符处理** 等 | 全屏 TUI |

**POSIX `termios` 骨架**（Linux / WSL / macOS）：

```c
#include <termios.h>
#include <unistd.h>

static struct termios saved;

void tty_raw(int enable)
{
    struct termios t;
    tcgetattr(STDIN_FILENO, &t);
    if (enable) {
        saved = t;
        t.c_lflag &= ~(ICANON | ECHO);  /* 关规范模式与回显 */
        t.c_cc[VMIN] = 1;
        t.c_cc[VTIME] = 0;
        tcsetattr(STDIN_FILENO, TCSAFLUSH, &t);
    } else {
        tcsetattr(STDIN_FILENO, TCSAFLUSH, &saved);
    }
}
```

**Windows 注意**：原生 Win32 用 **`SetConsoleMode`**；**WSL** 可用 **`termios`**。**跨平台** 库：**ncurses**、**readline**（行编辑专用）。

### `getchar()` 返回 `int`，不是 `char`

```c
int ch = getchar();   /* ✅ */
if (ch == EOF) ...

char c = getchar();   /* ❌ 若字节 0xFF，与 EOF 混淆（实现相关） */
```

| 值 | 含义 |
|----|------|
| **`0`–`255`** | **`unsigned char` 字节**（提升为 **非负 `int`** ） |
| **`EOF`（通常 -1）** | 流结束或错误 |

**8.3/8.5 联动**： **`char` 可能 signed**， **`0xFF` → -1**，与 **`EOF`** 相等 → 循环提前结束。

### `ungetc`：解析器「偷看」下一字符

```c
int ch = getchar();
if (isdigit(ch)) {
    /* 读多了，退回 */
    ungetc(ch, stdin);
}
```

- **标准保证**：每流 **至少一次** **`ungetc`**（实现常更多）
- **与 FSM**：状态机在 **不消耗** 字符的情况下 **试探** 下一输入

### 与 8.7 FSM 的配合

```text
  按键 → getchar (int) → FSM switch / 表驱动
              ↑ ungetc 回退
```

**工程规则**：

- 交互程序 **退出前恢复终端**（ **`atexit` / signal handler** 里 **`tty_raw(0)`**）
- **Ctrl+C** 在 raw 下常需 **自行处理 `SIGINT`**
- 单元测试 FSM 用 **字符串缓冲 + 指针** 代替 **`getchar`**
