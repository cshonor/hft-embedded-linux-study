# 8.8 软件比硬件更困难

## 本节讲什么

书名双关的 **技术内核** 在本节集中爆发：**硬件位模式** 遵循布尔代数，**可预测**；**C 表达式** 还要叠加 **运算符优先级、结合性、求值顺序、序列点、隐式转换**——组合出 **同一段代码不同编译器/优化级别不同结果** 的 **未定义行为（UB）**。本节是全章 **最重要** 的小节：**完整优先级表**、**Christmas 问题**、**Halloween 问题**、**短路 `&&` `||`**、**逗号运算符陷阱**、**函数实参中的多次修改**、**C11 sequenced-before** 与旧 **sequence point** 术语对照，以及 **工程防御规则**。

## 要点

### 运算符优先级与结合性（概览）

**从高到低**（完整表见 **[operator-precedence-cheatsheet.md](./operator-precedence-cheatsheet.md)** 与 **README**）：

| 级别 | 运算符 | 结合性 | ⚠️ 易混 |
|------|--------|--------|---------|
| 1 | **`()` `[` `->` `.`** | 左 | **`.` 与 `*`** |
| 2 | **`!` `~` `++` `--` `+` `-` `*` `&` `(type)` `sizeof`** | **右** | **`++` 后缀 vs 前缀** |
| 3 | **`*` `/` `%`** | 左 | |
| 4 | **`+` `-`** | 左 | |
| 5 | **`<<` `>>`** | 左 | |
| 6 | **`<` `<=` `>` `>=`** | 左 | |
| 7 | **`==` `!=`** | 左 | **⚠️ `==` vs `&`** |
| 8 | **`&`** | 左 | **⚠️ `&` vs `&&`** |
| 9 | **`^`** | 左 | |
| 10 | **`|`** | 左 | |
| 11 | **`&&`** | 左 | 短路 |
| 12 | **`||`** | 左 | 短路 |
| 13 | **`?:`** | 右 | |
| 14 | **`=` `+=` …** | **右** | **⚠️ `=` vs `==`** |
| 15 | **`,`** | 左 | **⚠️ 逗号 vs 实参逗号** |

**括号** 不受优先级限制；**怀疑时就加括号**——可读性 > 炫技。

### Halloween 问题：`= ` 与 `==`

```c
if ((a = b) == 0)   /* ✅ 赋值后比较 */
    ...

if (a = b == 0)     /* ❌ 等价 a = (b == 0)；a ∈ {0,1} */
    ...
```

**Yoda 条件**：**`if (0 == a)`** — 手滑 **`0 = a`** 编译失败。

**与 8.2 位运算**：**`if (1 == (val & mask))`**。

### Christmas 问题：同一标量多次修改 → UB

```c
i = i++ + ++i;   /* ❌ UB：同一表达式对 i 修改不止一次且无序列点 */
```

**C11 表述**：**`i++` 与 `++i` 的值计算** 彼此 **未 sequenced**；对 **`i` 的赋值** 亦与它们 **未 sequenced** → **UB**。

**类似 UB**：

```c
a[i++] = i;           /* UB */
f(i++, ++i);          /* UB：实参求值顺序未指定 */
x = x++ + 1;          /* UB */
```

**合法写法**（拆语句，每行 **一个** 副作用）：

```c
int t = i;
i = i + 1;
i = t + i;
```

### 序列点 vs sequenced-before（C11）

| 术语 | 时代 | 含义 |
|------|------|------|
| **Sequence point** | C89/C99 教学用语 | 求值中的 **完全结束点**； **`&&` `||` `?:` 首操作数后**、**分号 `;` 前** 等 |
| **Sequenced before / after** | **C11 起** 精确定义 | 若 **A sequenced before B**，则 **A 的副作用对 B 可见** |
| **Unsequenced** | C11 | 同一对象 **无顺序** 的读写 → **UB** |
| **Indeterminately sequenced** | C11 | 函数实参各参数之间 |

**面试一句话**：**C11 用 sequenced-before 取代「序列点」口语**； **`i=i++ + ++i` 在 C99/C11 都是 UB**。

**仍有序列点语义的例子**：

```c
if (p && *p) ...      /* *p 仅在 p 非 0 时求值 — 短路 */
a ? b : c;             /* 仅求值 b 或 c 之一 */
i = 0, j = 1;          /* 逗号：左侧 i=0  sequenced before  j=1 */
```

### 短路求值 `&&` 与 `||`

```c
if (ptr != NULL && ptr->len > 0)   /* ptr 空则不求值 ->len */
if (fd < 0 || (n = read(...)) < 0)
```

| 运算符 | 左侧为假/真时 | 右侧 |
|--------|---------------|------|
| **`&&`** | 左 **假** → 整个假 | **不求值** |
| **`||`** | 左 **真** → 整个真 | **不求值** |

**⚠️ `&` 无短路**：**`ptr & ptr->len`** 在 **`ptr==NULL`** 时仍 **求值右操作数** → **崩溃**。

**demo04_short_circuit**（README）：对比 **`&&` vs `&`** 对 **空指针** 的行为。

### 逗号运算符 `,` vs 函数实参逗号

**逗号运算符**（优先级 **最低之一**）：

```c
i = (j = 1, k = 2, j + k);   /* i == 3；逗号表达式值为最后一项 */
for (i = 0, j = n; i < j; i++, j--)  /* for 头里的逗号合法 */
```

**函数调用**：

```c
f(a++, b++);   /* 这里的逗号 **不是** 逗号运算符；是语法分隔符 */
               /* a++ 与 b++ 求值顺序 **未指定** → 若共享状态则危险 */
```

| | **逗号运算符 `,`** | **实参列表逗号** |
|--|-------------------|------------------|
| 语义 | **先左后右**，值为 **右操作数** | 分隔实参 |
| 顺序 | **左 sequenced before 右** | 各实参 **indeterminately sequenced** |
| 典型坑 | **`if (a=1, b=2)`** 条件为 **2**（真） | **`f(i++, ++i)` UB** |

### 后缀 `++` 与优先级

```c
*p++     /* *(p++)：先取 p 旧值所指，再 p++ */
(*p)++   /* 对 *p 自增 */
*++p     /* 先 ++p 再解引用 */
```

**`++` 后缀优先级高于 `*`**，但 **结合性** 决定 **后缀 ++ 绑定最近操作数**。

### 工程规则（务必遵守）

1. **一行一副作用**：禁止 **`i=i++ + ++i`**、**`a[i++]=i`**
2. **比较用 `==`，赋值单独一行**；防手滑用 **Yoda**：**`0 == x`**
3. **位测用括号**：**`(val & mask) == 1`**
4. **逻辑条件用 `&&`/`||`**，位合并用 **`&`/`|`**
5. **函数实参** 不含 **同一变量的多次 `++/--`**
6. **宏参数** 若含副作用， **括号 + 避免多次求值**（**ch02 `MAX`**）
7. **开启 `-Wall -Wsequence-point -Wparentheses`**（GCC）

### 031 与 25 的再提醒（8.1）

**优先级/进制** 双坑：**软件规则** 叠加 **字面量规则** → **「分不清万圣节与圣诞节」** 是 **温和版**；**Christmas UB** 是 **致命版**。
