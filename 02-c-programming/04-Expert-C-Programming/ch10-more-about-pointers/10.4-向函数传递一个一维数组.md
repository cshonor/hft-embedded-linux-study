# 10.4 向函数传递一个一维数组

## 本节讲什么

一维数组作实参时 **数组名退化为 `T*`**（**9.1, 9.3**），形参 **`int a[]` 与 `int *a` 等价**——但 **边界长度丢失**，函数内 **`sizeof(a)` 是指针宽**。本节系统整理 **`const` 与指针的四/八种组合**、**`typedef` 与 `const` 的陷阱**，以及 **`char*` vs `char[]`** 在 rodata 上的差异；为 **10.5 多维传参** 与 **10.7 函数指针** 打基础。

## 要点

### 数组 decay 传参（复习）

```c
void foo(int a[]);   /* 等价 void foo(int *a); */
void bar(int *a);

int arr[10];
foo(arr);   /* 传 &arr[0]，不是「整个数组对象」 */
```

| 语境 | `int arr[10]` 在 **`foo` 内** |
|------|-------------------------------|
| **`a[i]`、`a+1`** | ✅ 正常 |
| **`sizeof a`** | **指针宽**（8/4），**不是 40** |
| **需要长度** | 另传 **`int n`** 或约定哨兵 |

详见 **[ch04 4.1](../ch04-arrays-are-not-pointers/4.1-数组并非指针.md)**、**[9.3](../ch09-more-about-arrays/9.3-为什么C语言把数组形参当作指针.md)**。

### `const` 与指针：四种基本组合

**读法**：`const` **修饰它左边的东西**；左边没有则修饰右边。

```c
int a = 1, b = 2;

const int *p1 = &a;        /* 指向常量的指针：*p1 不可改，p1 可改指向 */
int *const p2 = &a;        /* 常量指针：p2 不可改指向，*p2 可改 */
const int *const p3 = &a;  /* 二者都不可改 */
int *p4 = &a;              /* 普通指针 */
```

| 声明 | 改指向 `p = &b` | 改值 `*p = 0` | 口语 |
|------|-----------------|---------------|------|
| **`const int *p`** | ✅ | ❌ | 指向常量的指针 |
| **`int *const p`** | ❌ | ✅ | 常量指针 |
| **`const int *const p`** | ❌ | ❌ | 都常量 |
| **`int *p`** | ✅ | ✅ | 普通 |

见 **demo01_const_ptr**。

### 形参中的 `const`（四种传参）

```c
void f1(const int *p);       /* 不修改 *p；可传 int* 或 const int* */
void f2(int *const p);       /* 极少写形参 — const 指针形参无意义（拷贝） */
void f3(const int *const p); /* 不修改 *p */
void f4(int *p);             /* 可能修改 *p */
```

**实用规则**：

- **只读遍历**：`void print(const int *arr, size_t n);`
- **要改元素**：`void scale(int *arr, size_t n);`
- **形参 `int *const`** 几乎不用 — **`const` 修饰的是拷贝进来的指针副本**

### 扩展：`const` 与多级指针

```c
void g1(const char **pp);    /* *pp 是 const char* — 不能改 *pp 指向的内容？ 
                                实际：pp 可改，*pp 指向 char const */
void g2(char * const *pp);   /* 少见 */
void g3(const char * const *pp);
```

**`argv` 只读场景**：`int main(int argc, const char *argv[])` — 承诺不改字符串内容。

### `typedef` 与 `const` 陷阱

```c
typedef char *Str;
const Str s = "hello";   /* 不是 const char*！是 char* const s */
/* s 不能改指向；*s 仍可改 — 与直觉相反 */
```

**原因**：`typedef` 后 **`const Str`** = **`char *const`**，不是 **`const char*`**。

**正确写法**：

```c
typedef const char *CStr;
CStr s = "hello";        /* 指向只读 */

/* 或不用 typedef */
const char *s = "hello";
```

见 **[ch03 3.5](../ch03-analyzing-c-declarations/3.5-typedef可以成为你的朋友.md)**、**[3.7](../ch03-analyzing-c-declarations/3.7-typedef-struct-foo-foo-的含义.md)**。

### `char*` vs `char[]` 与 rodata

```c
char *p = "hello";           /* p 指 rodata；*p 修改 → UB/SIGSEGV */
char buf[] = "hello";        /* buf 是栈数组；内容从 rodata 拷贝，可写 */

p[0] = 'H';   /* 危险 */
buf[0] = 'H'; /* OK */
```

| 维度 | **`char *p = "..."`** | **`char buf[] = "..."`** |
|------|------------------------|---------------------------|
| **类型** | 指针 | 数组 |
| **存储** | 指针在栈/静态，串在 **rodata** | **整个串在栈** |
| **`sizeof`** | 指针宽 | 串长+1 |
| **传参 decay** | `char*` | `char*`（仍 decay） |
| **可写** | ❌（字面量） | ✅ |

**Iliffe 表** `char *lines[] = {"a","b"}` 各行指 **只读区**（**10.2**, **9.5**）。

### 数组形参写法对照

```c
void a(int arr[]);
void b(int arr[10]);   /* 10 被忽略 — 仍是指针形参 */
void c(int *arr);
/* 三者等价 */
```
