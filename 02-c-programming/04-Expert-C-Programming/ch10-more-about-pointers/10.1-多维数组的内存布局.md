# 10.1 多维数组的内存布局

## 本节讲什么

**ch09 9.6** 已建立：`int[M][N]` 在物理上是 **行优先（row-major）** 的 **一块连续 int**。本节从 **指针视角** 深化：对比 **栈上二维数组** 与 **`int**` 动态矩阵** 的 **内存拓扑**；强调 **指针类型决定 `+1` 步长（stride）**——同一地址，用 `int*` 还是 `int (*)[N]` 看，步长完全不同。这是 **10.2–10.5** 区分 Iliffe 向量、锯齿数组、传参写法的基础。

## 要点

### 连续二维：`int[M][N]`

```c
int arr[2][3] = {
    {1, 2, 3},
    {4, 5, 6},
};
```

**物理布局**（无「行指针表」）：

```text
地址增长 →  1  2  3  4  5  6
            └ row0 ┘ └ row1 ┘
```

- **`arr[i][j]`** 偏移：`i * N + j`（**N 必须已知** 才能编译期算步长）
- **Cache 友好**：一行内元素相邻，按行扫描最优

详见 **[ch09 9.6](../ch09-more-about-arrays/9.6-C语言的多维数组.md)**。

### 分散二维：`int **`（锯齿 / 指针向量）

```c
int **m = malloc(rows * sizeof *m);
for (int r = 0; r < rows; r++)
    m[r] = malloc(cols * sizeof **m);
```

**物理布局**：

```text
m → [ ptr0 ][ ptr1 ][ ptr2 ]     ← 指针表（可能连续）
         ↓       ↓       ↓
       row0    row1    row2      ← 各行 malloc，通常不相邻
```

- **两次（或 N+1 次）分配**；行与行之间 **无保证连续**
- **`m[r][c]`** 语法相同，但 **`m+1` 步长是指针宽**，**`m[r]+1` 步长是 `sizeof(int)`**
- 与 **`int[M][N]` 不能混用传参**（**10.5**）

见 **demo05_jagged_vs_2d** 打印的 `&arr[1][0] - &arr[0][0]`（连续）vs `m[0]` 与 `m[1]` 地址（通常不相邻）。

### 指针类型决定步长（核心）

对 **同一块连续内存** `int data[2][3]`：

| 指针类型 | 声明示例 | `p + 1` 跳过 |
|----------|----------|--------------|
| **`int *`** | `int *p = &data[0][0]` | **4 B**（一个 int） |
| **`int (*)[3]`** | `int (*row)[3] = data` | **12 B**（一整行） |
| **`int (*)[2][3]`** | `int (*block)[2][3] = &data` | **24 B**（整块） |

```c
int arr[2][3];
int *ep = &arr[0][0];      /* ep+1 → 同行下一列 */
int (*rp)[3] = arr;        /* rp+1 → 下一行首 */
```

**口诀**：**`[]` 在 `*` 右边 → 指针数组；`(*)` 包 `[N]` → 数组指针**。读法见 **[ch03 3.3](../ch03-analyzing-c-declarations/3.3-优先级规则.md)**。

### 与 ch04 / ch09 的对照

| 对象 | 类型本质 | 传参退化 |
|------|----------|----------|
| **`int a[N]`** | 连续 N 个 int | `int*`（**ch04**, **9.3**） |
| **`int m[R][C]`** | 连续 R×C 个 int | `int (*)[C]`（**9.6**） |
| **`int **m`** | 指针表 + 分散行 | `int**` |

**ch04 硬结论**：数组 ≠ 指针；**ch09** 讲 decay 何时发生；**本章** 讲 **选对指针类型** 才能正确遍历与传参。

### 内存布局对比表

| 维度 | **`int[M][N]` 栈/静态** | **`int **` 动态锯齿** |
|------|-------------------------|----------------------|
| **数据连续** | ✅ 整块 | ❌ 仅行内连续 |
| **分配次数** | 0（栈）或 1（静态） | 1 + M 次 `malloc` |
| **传参类型** | `int (*)[N]` | `int **` |
| **`sizeof` 行** | 编译期知 N | 需额外存 `cols` |
| **释放** | 自动/不需 | 逐行 `free` + `free(m)` |
