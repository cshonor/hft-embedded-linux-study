# 10.3 在锯齿状数组上使用指针

## 本节讲什么

**锯齿数组（jagged array）** 指行宽可不等、各行 **分开分配** 的二维结构，在 C 中通常实现为 **`int **`**：先 **`malloc` 指针表**，再 **逐行 `malloc` 数据**。本节对比 **锯齿 `int **`** 与 **栈上 `int[M][N]`** 的分配、遍历、传参与释放，避免 **10.5** 中最常见的 **`int **` 误接 `int[][N]`** 错误。

## 要点

### 标准构造模式

```c
int rows = 3, cols = 4;

int **m = malloc(rows * sizeof *m);
if (!m) return -1;

for (int r = 0; r < rows; r++) {
    m[r] = malloc(cols * sizeof **m);
    if (!m[r]) { /* 回滚已分配行 */ }
}
```

**释放**（顺序与分配相反）：

```c
for (int r = 0; r < rows; r++)
    free(m[r]);
free(m);
```

见 **demo05_jagged_vs_2d**、**demo03_double_ptr**（`char **` 出参是同一「通过指针的指针改调用方指针」模式）。

### 锯齿 vs 真二维数组

| 维度 | **`int[M][N]`** | **`int **` 锯齿** |
|------|-----------------|-------------------|
| **内存** | 一块连续 M×N | 指针表 + M 块行缓冲 |
| **行相邻** | ✅ | ❌（通常） |
| **行宽** | 固定 N | 可各行不同（变长锯齿） |
| **类型** | `int[M][N]` → `int (*)[N]` | `int **` |
| **传参** | `void f(int a[][N], int rows)` | `void g(int **m, int rows, int cols)` |
| **Cache** | 优 | 行内优，跨行可能 miss |

```c
/* 变长行示例：每行长度 = r+1 */
for (int r = 0; r < rows; r++) {
    m[r] = malloc((r + 1) * sizeof **m);
    /* ... */
}
```

### 遍历与指针算术

```c
/* 下标 — 两种布局语法相同 */
for (int r = 0; r < rows; r++)
    for (int c = 0; c < cols; c++)
        sum += m[r][c];

/* 行指针 */
int *row = m[r];      /* 第 r 行首元素 */
row[c] = 0;

/* 勿混淆 */
m + 1;    /* 下一个「行指针」槽，步长 sizeof(int*) */
m[r] + 1; /* 同行下一 int，步长 sizeof(int) */
```

### 一次 `malloc` 模拟连续（进阶）

若需 **锯齿的灵活性** 又想要 **连续块**（少一次 indirection）：

```c
int *block = malloc(rows * cols * sizeof *block);
int **m = malloc(rows * sizeof *m);
for (int r = 0; r < rows; r++)
    m[r] = block + r * cols;
/* 一次 free(block); free(m); */
```

这是 **Iliffe 表 + 单块数据** 的折中，仍传 **`int **`**，但与 **`int[M][N]`** 布局接近。

### 常见错误

| 错误 | 后果 |
|------|------|
| **`f(int[M][N])` 传 `int **`** | 按连续行算偏移 → **错数据 / 崩溃** |
| **只 `free(m)` 不 free 各行** | 内存泄漏 |
| **栈上 `int*` 数组假扮 `int**`** | 类型勉强可传，语义仍非锯齿 |
| **忘记传 `rows`/`cols`** | 函数无法安全遍历（同 **9.3** 一维丢边界） |
