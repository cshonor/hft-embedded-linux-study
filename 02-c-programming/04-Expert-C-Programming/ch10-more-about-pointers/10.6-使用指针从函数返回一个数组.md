# 10.6 使用指针从函数返回一个数组

## 本节讲什么

C **不能** 按值返回数组（**`return arr;`** 非法）。惯用法是 **返回指向数组首元素的指针**——但必须保证 **被指向内存在调用返回后仍然有效**。本节区分 **栈上局部数组（悬垂指针 / UB）**、**静态 / 全局存储**、**堆上 `malloc`** 三种方案，并与 **ch07 内存**、**10.8 野指针** 衔接。

## 要点

### 非法与危险模式

```c
/* 编译错误 — 不能返回数组类型 */
/* int[10] make(void) { int a[10]; return a; } */

/* 危险 — 返回栈上局部数组地址 */
int *bad(void)
{
    int local[10] = {0};
    return local;   /* 悬垂指针：函数返回后栈帧销毁 → UB */
}
```

**症状**：有时「还能用」、有时崩溃、有时被覆盖 — **不可依赖**。

### 三种合法策略

| 策略 | 示例 | 生命周期 | 注意 |
|------|------|----------|------|
| **静态 / 全局** | `static int buf[N]; return buf;` | 程序全程 | **不可重入**；多线程不安全 |
| **堆 `malloc`** | `int *p = malloc(...); return p;` | 直到 **`free`** | **调用方负责释放** |
| **调用方提供缓冲** | `void fill(int *out, size_t n)` | 由调用方控制 | **最清晰 API** |

```c
/* 1. 静态 — 简单但非重入 */
char *weekday(int i)
{
    static char *days[] = {"Sun","Mon","Tue","Wed","Thu","Fri","Sat"};
    if (i >= 0 && i < 7)
        return days[i];
    return "???";
}

/* 2. 堆 — 调用方 free */
int *make_array(size_t n)
{
    int *p = malloc(n * sizeof *p);
    if (p)
        for (size_t i = 0; i < n; i++)
            p[i] = (int)i;
    return p;
}

/* 3. 出参 — 推荐 */
int fill_buffer(int *out, size_t cap)
{
    if (!out || cap == 0) return -1;
    for (size_t i = 0; i < cap; i++)
        out[i] = (int)i;
    return 0;
}
```

### 返回指针 vs 返回数组类型

```c
int *f(void);           /* 返回 int* — OK */
int (*f)[10](void);     /* 返回指向 int[10] 的指针 — 少见但合法 */
```

**返回二维**：通常 **`int **`**（锯齿）或 **单块 `malloc` + Iliffe 表**（**10.3**），或 **返回 `struct` 封装指针+长度**。

### 与字符串字面量

```c
const char *msg(void)
{
    return "OK";   /* 指向 rodata — 合法，只读 */
}
```

**不是** 「返回栈数组」；字面量寿命 **整个程序**（**10.4** **`char*`**）。

### 常见面试陷阱

```c
char *danger(void)
{
    char buf[] = "hello";  /* 栈数组 */
    return buf;            /* 悬垂 — 与 return local 同类 */
}

char *safe_copy(void)
{
    char *p = malloc(6);
    if (p) strcpy(p, "hello");
    return p;              /* OK — 调用方 free */
}
```
