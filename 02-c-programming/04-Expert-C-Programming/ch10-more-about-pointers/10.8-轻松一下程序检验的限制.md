# 10.8 轻松一下——程序检验的限制

## 本节讲什么

书中以 **「程序检验（program verification）」** 的轶事作 **轻松一下**：再严格的数学证明或静态分析，也难以穷尽 **C 指针** 在运行时的全部胡作非为。本节在轶事之外做 **全章收束**：回顾 **野指针**、**NULL**、**`free` 栈/重复释放**、**悬垂指针** 与 **10.6 返回栈数组** 的关联，强调 **正确性最终靠程序员纪律 + 测试 + 工具辅助**，而非单一 silver bullet。

## 要点

### 程序检验的局限（书中要点）

| 期望 | 现实 |
|------|------|
| **形式化证明** 程序无错 | 规格本身可能错；环境假设难全 |
| **静态分析** 抓住所有 bug | **指针别名**、**未定义行为** 仍易漏 |
| **编译通过 = 安全** | C 大量 **UB** 编译器不报错 |
| **「跑过了」= 正确** | **悬垂指针** 可能「碰巧」能读 |

**与 ch08 8.8 呼应**：软件规则（语言语义）之难 — 硬件简单，**指针 + UB** 让人类与工具都头疼。

### 全章指针纪律清单

#### 1. 野指针（wild pointer）

**未初始化** 或 **已失效** 的指针仍被解引用。

```c
int *p;          /* 野 — 未初始化 */
*p = 1;          /* UB */

int *q = malloc(10);
free(q);
*q = 2;          /* 野 — use-after-free */
```

**对策**：定义时 **`= NULL`**；**`free` 后立即 `p = NULL`**。

#### 2. NULL 指针

```c
int *p = NULL;
if (p) { *p = 1; }   /* 跳过 — 安全 */
/* *p = 1;  ← 通常 SIGSEGV，标准允许 UB */
```

**`malloc` 失败** 返回 **NULL** — 必查（**10.7**, **ch07**）。

#### 3. 悬垂指针（dangling pointer）

| 来源 | 示例 |
|------|------|
| **返回栈地址** | **10.6** `return local;` |
| **`free` 后继续用** | **use-after-free** |
| **指向已销毁对象** | 内部静态被覆盖（重入） |

#### 4. `free` 错误

```c
int stack[10];
free(stack);        /* 错 — 非 malloc 内存 */

free(p);
free(p);            /* 双重释放 — 堆损坏 */

free(p);
/* 忘记 p = NULL，后续 if(p) 仍可能误用已释放块 */
```

**只 `free` 一次**；**只 `free` `malloc/calloc/realloc` 得到的指针**。

#### 5. 类型与布局混用

- **`int **` 接 **`int[M][N]`**（**10.5**）
- **`extern char *` vs `char[]`**（**ch04 4.2**）
- **修改字符串字面量**（**10.4**）

### 工具能做什么、不能做什么

| 工具 | 能帮忙 | 不能替代 |
|------|--------|----------|
| **编译器 `-Wall`** | 部分可疑转换、未初始化 | 全部 UB |
| **Valgrind / ASan** | 越界、UAF、泄漏 | 逻辑错误、数据竞争全覆盖 |
| **静态分析** | 空指针路径 | 复杂别名 |
| **形式化方法** | 小内核、协议 | 大型任意 C 工程 |

**结论**：**检验是辅助**；**理解内存模型**（**ch07**、**本章**）才是根本。
