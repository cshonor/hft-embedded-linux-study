# 2.4 少做之过

## 本节讲什么

**Sins of Subtraction（少做之过）**：C **故意省略** 现代语言常内置的检查与抽象——**无数组边界**、**C89 无内建 bool**、**字符串无长度**、**隐式整型转换（尤其 signed/unsigned）**。这些「不做」换取了接近硬件的速度与极简编译器，却把安全责任 **完全推给程序员**；系统编程与嵌入式领域必须正面理解这些缺口。

## 要点

### 1. 无数组边界检查

```c
int a[10];
a[10] = 0;      /* 越界写 — 未定义行为 */
a[-1] = 0;      /* 同样 UB */
```

| C 的做法 | 后果 |
|----------|------|
| 不生成边界检查指令 | 性能接近手写汇编；内核/DPDK 依赖于此 |
| 越界 = UB | 编译器可假设「下标永远合法」做激进优化 |
| 不保证 crash | 可能 **静默覆盖** 栈上其他变量、函数返回地址 |

**防御**：

- 显式检查下标；使用 **`sizeof(a)/sizeof(a[0])`** 算长度（仅对 **真实数组** 有效，指针不行——**ch04**）
- C99 **VLA**、C11 `_Static_assert`；C++ 边界 **`std::vector`**
- 工具：**ASan**、**UBSan**（`-fsanitize=address,undefined`）
- 内核：**KASAN**；Embedded C **5.7** 内存踩踏监测

### 2. 没有 bool（C89）与「真值」语义

C89 用 **`int` 表示逻辑值**：非零为真，零为假。

```c
/* C89 风格 */
if (flag) { ... }

/* C99 起 */
#include <stdbool.h>
bool ok = true;
```

| 时期 | 类型 | 注意 |
|------|------|------|
| C89 | `int` / 自定义 `typedef` | `if (p = malloc(...))` 等惯用法 |
| C99+ | `_Bool` / `bool` | 与 `int` 可互转；**`<stdbool.h>`** |
| 陷阱 | `bool`  promoted 到 `int` | 与 **可变参**、**位域** 混用时读标准 |

**少做之过**：长期无专用布尔类型，加剧 **`=` 与 `==`** 混淆（**2.3**）。

### 3. C 字符串：无长度，以 `\0` 结尾

```c
char s[] = "hello";   /* 6 字节：'h'..'o' + '\0' */
strlen(s);            /* 扫描直到 \0 — O(n) */
strcpy(dst, src);     /* 不检查 dst 容量 → 经典溢出 */
```

| 缺失 | 影响 |
|------|------|
| 不存储长度 | 不能 O(1) 知长度；二进制数据不能直接用 C 字符串 |
| NUL 终止 | 中间不能含 `\0`；漏写终止符 → 越界读 |
| 库函数 `strcpy`/`strcat` | **不** 带 bounds → 改用 **`strncpy`/`snprintf`**（仍需谨慎） |

**现代替代**：POSIX **`strnlen`**、C11 **`bounds-checking` 附录**（可选）、C++ **`std::string_view`**。内核自研 **`strscpy`** 等安全 API。

### 4. 隐式转换：signed 与 unsigned

当 **有符号与无符号** 同处一个表达式，C 倾向于 **把有符号转为无符号** 再运算。

```c
unsigned u = 5;
int i = -1;

if (u > i)   /* i 转为 unsigned → 巨大正数 → 条件为假 */
    puts("u > i");
else
    puts("u <= i");   /* 实际走这里 */
```

**Demo**：`demo/demo02_sign_unsigned.c`。

| 场景 | 典型坑 |
|------|--------|
| 比较 `u > i` | 负 `int` → 大 `unsigned` |
| `unsigned` 减法下溢 | `0u - 1u` → `UINT_MAX`，循环永不结束 |
| 混合赋值 | `-1` 赋给 `unsigned` → 全 1 位模式 |
| 移位 | 负值或超位宽移位 → UB 或 impl-defined |

**整型提升（integer promotion）**：

```c
char c1, c2;
c1 = c2 + c3;   /* c2、c3 先提升为 int 再相加，再截断赋回 char — 可能溢出 */
```

见 **ch01 1.2** float 提升；**ch08** 整型与浮点转换。

**防御**：

- 比较前 **显式 cast** 到同一类型，并注释意图
- 无符号仅用于 **位掩码 / 模运算**；循环计数慎用 `unsigned`
- 编译选项 **`-Wsign-compare -Wconversion`**
- 可移植整数：**`<stdint.h>`**（Embedded C **ch07**）

### 5. 四类「少做」叠加效应

```text
无边界 + 无长度字符串 + 隐式转换 + 无 bool
    ↓
一次 off-by-one 或符号错误
    ↓
缓冲区溢出 / 死循环 / 安全漏洞
    ↓
「在别的编译器上能跑」的 UB 地雷
```

**Heartbleed** 类漏洞与 **长度字段缺失** 的协议解析，都可追溯到「C 少做 + 程序员假设过多」。

### 小结：少做之过清单

| 语言未提供 | 程序员必须 |
|------------|------------|
| 数组边界 | 自行检查；用 Sanitizer |
| bool（C89） | 约定 0/非 0；C99 用 `<stdbool.h>` |
| 字符串长度 | 传递 `size_t n`；用 `snprintf` |
| 安全算术 | 显式类型；避免 signed/unsigned 混比 |

## 与后续衔接

- **ch04–ch09**：数组、指针、字符串 — 全建立在「无边界 + 衰变」之上
- **ch07**：内存冒险、泄漏、越界与虚拟内存
- **2.5**：少数「少做」其实是无法撤回的历史错误（trigraphs）
- **ch01 1.6**：越界、有符号溢出均属 UB 分类
- **Embedded C ch05–ch07**：栈/堆、指针、枚举与整型宽度
- **DPDK/内核**：大量 `uint32_t`、显式长度字段、位域 — 对「少做之过」的工程回应
