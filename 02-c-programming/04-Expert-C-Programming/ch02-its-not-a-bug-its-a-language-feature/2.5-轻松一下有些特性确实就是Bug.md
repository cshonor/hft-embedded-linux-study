# 2.5 轻松一下——有些特性确实就是Bug

## 本节讲什么

并非所有怪癖都能用「历史妥协」优雅开脱——有些特性是 **设计失误或标准化事故**，因 **已有代码依赖** 或 **无法从语法中抠除** 而长期留存。本节用轻松笔调列举 **三字符组（trigraphs）**、**C89 前没有 `//` 注释**、**悬空 else** 等案例，并区分：**可学规则避坑** vs **只能指望编译器默认关闭/警告** 的真 Bug 级特性。

## 要点

### 1. 三字符组（Trigraphs）——标准里的「幽灵」

为适配 **键盘缺少 `#[]{}|^~`** 的老式终端，C 引入 **三字符组** 替换：

| 三字符组 | 替换为 |
|----------|--------|
| `??=` | `#` |
| `??/` | `\` |
| `??(' | `[` |
| `??)` | `]` |
| `??<' | `{` |
| `??>` | `}` |
| `??!` | `\|` |
| `??-` | `~` |

```c
/* 程序员写： */
printf("What??!??!");
/* 可能被预处理成含 | 和 ! 的 token 序列 — 字符串内也可能受影响（视阶段而定） */
```

**为何像 Bug**：意图是 **可移植输入**，结果却是 **源码里看不见的二义性**；现代编辑器与 ASCII 键盘普及后 **毫无益处**。

**现状**：

- C23 起 trigraphs **移除**（长期 deprecated）
- GCC/Clang 默认通常 **不启用**；`-trigraphs` 才打开
- **教训**：别在代码里写 `??` 序列开玩笑

### 2. `//` 注释：在「老 C」里不是语言特性

| 标准 | `//` 单行注释 |
|------|----------------|
| K&R / C89 | **不支持** — 只有 `/* */` |
| C99 起 | 标准化 |
| C++ | 一直支持 |

```c
/* C89 合法 */
int x = 1; /* 注释 */

// 这在 C89 中是语法错误（或部分编译器当扩展）
// int y = 2;
```

**少做之过 + 真混乱**：混合 C/C++ 项目、老旧嵌入式编译器若锁 **`-std=c89`**，`//` 会导致 **整文件编译失败**。GNU 扩展曾提前提供 `//`，造成 **「在我机器上能跑」** 的移植坑。

**习惯**：需 C89 的公共头文件 **只用 `/* */`**；`.c` 若允许 C99 再用 `//`。

### 3. 悬空 else（Dangling else）

```c
if (a)
    if (b)
        x = 1;
else        /* 绑定内层 if 还是外层 if？ */
    x = 2;
```

C 规则：**else 与最近 unmatched 的 if 配对** → 上面 `else` 属于 **`if (b)`**。

| 写法 | 效果 |
|------|------|
| 默认 | `else` 挂内层 `if` |
| 意图挂外层 | 外层 `if` 加 **大括号** |

```c
if (a) {
    if (b)
        x = 1;
} else
    x = 2;
```

不是解析器 Bug，但 **语言设计易误导**；属于 **2.3 误做之过** 的「人类可读性缺失」。

### 4. 其他「真·Bug 味」特性

| 特性 | 问题 | 应对 |
|------|------|------|
| **digraphs** `<:` `:>` 等 | 类似 trigraph，输入兼容 | 几乎不见；别写 |
| **`#` 与宏换行** | 反斜杠续行易错 | 宏用工具生成或 inline |
| **空宏** `#define X` | 替换为空，残留空格 | 用 `#ifdef` 而非空定义 |
| **typedef 重名**（旧代码） | 作用域规则曾更松 | 遵循 C89 后规则 |

### 5. 特性 vs 真 Bug：如何判断

```text
能写清楚标准语义、且可稳定推导 → 语言特性（即使讨厌）
标准都说不清的优化空间 → UB（2.1、ch01 1.6）
标准承认是败笔但不能删 → 2.5 本节（trigraphs）
编译器默认关闭 + 文档警告 deprecated → 等标准删除
```

**轻松一下的严肃结论**：读 Expert C 不是为了嘲笑 C，而是 **在写内核、协议栈、DPDK 热路径之前**，知道哪些地雷是 **规则**、哪些是 **标准也后悔的遗产**。
