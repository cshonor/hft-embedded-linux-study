# 2.3 误做之过

## 本节讲什么

**Sins of Omission（误做之过）**：van der Linden 在此节重点写 **符号过载（overloaded symbols）**——同一记号的含义依赖上下文，编译器 **不会** 替你在 `=` 与 `==`、`*` 与 `.` 之间做「意图推断」。另有 **`static` 双重语义**、运算符优先级陷阱、**非 void 函数缺 return** 等。这类错误代码 **常能编译通过**，却在运行时或优化后产生离奇结果。

## 要点

### 1. `static` 的双重含义

| 位置 | 含义 | 存储期 / 链接 |
|------|------|----------------|
| **文件作用域** `static int x;` | 内部链接 | 整个程序生命周期；**仅本翻译单元可见** |
| **块作用域** `static int cnt;` | 无链接属性变化 | **静态存储期**：只初始化一次，下次进入块保留值 |

```c
void counter(void)
{
    static int n = 0;   /* 只初始化一次 */
    n++;
    printf("%d\n", n);
}
/* 多次调用 counter → 1, 2, 3, ... */
```

**Demo**：`demo/demo04_static_dual.c` — 对比文件 `static` 与块内 `static`。

**误做之过**：同一个关键字表示 **链接限制** 与 **持久局部变量** 两件不同的事；读代码必须看 **声明位置**。

### 2. 运算符优先级：`*` vs `.` / `->`

结构体指针访问：

```c
struct foo { int x; } s, *p = &s;

(*p).x = 1;    /* 正确：先解引用再取成员 */
*p.x = 1;      /* 错误：等价于 *(p.x)，把 . 当更高优先级 */
p->x = 1;      /* 推荐：-> 专为指针设计 */
```

**记忆**：`.` 与 `->` 优先级 **高于** 一元 `*`；想对解引用结果取成员 **必须加括号**。

### 3. `if` 中的 `=`、`==`、`&`

```c
if (a = b)       /* 赋值：b 的值赋给 a，条件为 b 是否非零 */
if (a == b)      /* 比较：相等为真 */
if (a & b)       /* 按位与：结果非零为真 */
```

| 写法 | 常见意图 | 实际 |
|------|----------|------|
| `if (x = 0)` | 想判断等于 0 | **赋值 0**，条件恒为假；编译器 `-Wparentheses` 可能警告 |
| `if (x == 0)` | 判断等于 0 | 正确 |
| `if (flags & MASK)` | 测试位 | 正确；别写成 `if (flags = MASK)` |

**防御**：惯用 **`if (0 == x)`**（Yoda 条件，现代较少用）；更常见 **`if (x == 0)`** + 开启 `-Wall`；赋值进 if 时写 **`if ((ptr = get()) != NULL)`** 并加括号。

**优先级链**（节选）：

```text
高 → 低：  ()  []  ->  .  →  !  *  &  →  ==  !=  →  &  →  |  →  &&  →  ||  →  = 
```

`==` 高于 `&` 吗？ **不** — `&`（按位与）优先级 **低于** `==`：

```c
if (a & MASK == 0)   /* 解析为 a & (MASK == 0)，通常不是本意 */
if ((a & MASK) == 0) /* 正确 */
```

### 4. 符号过载：`*`、`&`、`[]`

同一符号，多种含义：

| 符号 | 可能含义 |
|------|----------|
| `*` | 乘法、指针解引用、声明指针 |
| `&` | 按位与、取地址 |
| `[]` | 下标、声明数组、函数参数「伪数组」 |

```c
int *p;           /* 声明：p 是指向 int 的指针 */
*p = 3;           /* 解引用 */
int a = 2 * 3;    /* 乘法 */

int i = 5;
int *q = &i;      /* 取地址 */
flags = flags & MASK;  /* 按位与 */
```

**声明读法** 需右左规则（**ch03** 整章）：`int *p` 中 `*` 绑定 `p` 而非 `int`。

```c
int* a, b;   /* a 是指针，b 是 int — 易误读为两个指针 */
```

### 5. 非 void 函数缺少 return

```c
int pick(int x)
{
    if (x > 0)
        return 1;
    /* x <= 0 时执行流到达函数末尾 — 未定义行为（C99 起） */
}
```

| 标准 | 行为 |
|------|------|
| C89 | 可能返回「垃圾值」，部分编译器静默 |
| C99+ | ** reaching end of non-void function → UB ** |
| 优化 | 编译器可假设「不会到达末尾」，删除「看似可达」的代码（**ch01 1.6 UB**） |

**防御**：所有路径 `return`；或用 `-Wreturn-type`；`switch` 加 `default` 与 `return`。

### 6. 与宏的交叉（衔接 2.2）

宏展开 **不** 改变 C 运算符规则：

```c
#define SET_BIT(reg, bit) reg = reg | (1 << bit)
if (SET_BIT(x, 3))   /* 展开后 if 里是赋值，不是比较 */
```

复杂逻辑 **不要用多语句裸宏**；用 `do { ... } while (0)` 或 `static inline` 函数。

### 小结：误做之过清单

| 陷阱 | 根源 | 习惯 |
|------|------|------|
| `static` 双义 | 关键字复用 | 看声明作用域 |
| `*p.member` | `.` 优先于 `*` | 用 `->` 或 `(*p).member` |
| `=` vs `==` | 视觉相似 | `-Wall`、括号、分离赋值与判断 |
| `&` / `==` 优先级 | 记错优先级 | 位运算比较加括号 |
| `*` `&` `[]` 过载 | 声明与表达式共用符号 | **ch03** 声明螺旋 |
| 缺 return | 语言不强制所有路径 | 编译警告当错误 |
