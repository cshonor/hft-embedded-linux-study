# 2.6 参考文献

## 本节讲什么

本章讨论的 Fortran 事故、三类「之过」与 C 标准怪癖，在下列文献中都有 **一手或权威二手** 论述。本节不是空书单，而是 **按主题索引**——读标准原文、历史论文与经典教材，可对照 **2.1–2.5** 各节深化。

## 要点

### 语言设计与历史

| 文献 | 作者 / 来源 | 与本章关系 |
|------|-------------|------------|
| *The Development of the C Language* | Dennis M. Ritchie, 1993 | C 从 B/BCPL 演进；**为何保留数组衰变、宏、弱检查** |
| *The Evolution of the Unix Time-sharing System* | Ritchie, 1984 | Unix 与 C 共生；系统编程需求塑造「少做之过」 |
| *Why Pascal is Not My Favorite Programming Language* | Brian W. Kernighan, 1981 | 对比 Pascal 强类型与 C 实用主义；**语言特性 vs 安全** 论战背景 |
| *Expert C Programming* ch.2 | Peter van der Linden, 1994 | 本章直接来源：Fortran DO、三类之过、trigraphs |

### C 标准与正式定义

| 文献 | 说明 | 查阅主题 |
|------|------|----------|
| **ISO/IEC 9899:1990 (C89/C90)** | 首个 ANSI/ISO C | 隐式 `int` 删除前时代；`//` 未进标准 |
| **ISO/IEC 9899:1999 (C99)** | | `bool`、`<stdint.h>`、`//` 注释、UB 条款更严 |
| **ISO/IEC 9899:2011 (C11)** | | `_Static_assert`、可选 bounds checking annex |
| **WG14 公开草案** | [open-std.org/jtc1/sc22/wg14](https://www.open-std.org/jtc1/sc22/wg14/) | 在线 N1256/N1570 等；索引 **undefined behavior**、**constraints** |
| *Rationale for International Standard—Programming Languages—C* | ANSI X3.159-1989 Rationale | 标准化 **为何保留 K&R 行为**（quiet change 根源，**ch01 1.10**） |

**标准内建议检索**（C11 草案编号示例）：

- **6.2.5** Types — `_Bool`、整数转换秩
- **6.5** Expressions — 整型提升、有符号/无符号转换
- **6.7.4** Function definitions — `return` 与 UB
- **6.8.4.1** The `switch` statement — fall-through 语义
- **J.2** Undefined behavior — 清单式索引

### 建议阅读顺序

1. **2.1 背景**：Ritchie *Development of the C Language*（短文，必读）
2. **2.2–2.4 对照**：Koenig *C Traps and Pitfalls* 或 Harbison & Steele 相关章
3. **查条款**：WG14 C11 草案 + Rationale 对应节
4. **写代码验证**：本章 **demo/** + `-Wall -fsanitize=undefined`
