# 2.8.4 大小端字节序

## 本节讲什么

**Big / Little Endian** —— 多字节在内存中的排布；串口、网络、Flash 必考。

## 要点

```c
uint32_t v = 0x12345678;
/* 小端(x86/ARM LE): 低地址存 0x78
   大端(网络/DSP):   低地址存 0x12 */
```

| 端序 | 低地址字节 |
|------|------------|
| **小端** | 低位字节 |
| **大端** | 高位字节 |

### 检测代码

```c
static int is_little_endian(void)
{
    uint32_t x = 0x01020304;
    return *(uint8_t *)&x == 0x04;
}
```

见 **`demo/demo01_endian.c`**。

### gdb 验证

```bash
gcc -g demo/demo01_endian.c -o demo01_endian
gdb ./demo01_endian
(gdb) break main
(gdb) run
(gdb) x/4xb &v
```

### 工程坑

- 网络协议 **大端** → `htonl/ntohl`  
- 跨 MCU 通信须约定端序  
- 联合体/view 解析错端 → 数值全错  

## 与后续衔接

**ch07** 数据存储；**ch03** ARM 字节访问。
