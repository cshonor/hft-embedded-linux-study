# 2.4.2 一级 Cache 和二级 Cache

## 本节讲什么

**L1 / L2 / L3 的分工、容量与延迟** —— 理解绑核、NUMA 与 per-core Cache 拓扑。

## 要点

| 层级 | 典型 | 特点 |
|------|------|------|
| **L1I/L1D** | 32KB×2 | 分指令/数据，1–4 cycles |
| **L2** | 256KB–1MB | 每核或簇共享 |
| **L3** | 8–64MB | 全片共享，末级 Cache |

- **Harvard 味道**：L1I 与 L1D 分离（**2.3.2**）  
- **Inclusive**：上级含下级副本，一致性简单；**Exclusive** 省容量  
- DPDK：`lscpu` 看 Cache 大小；大页 + 预取减少 L3 争用

## 与后续衔接

**2.4.3** 无 Cache 处理器；**2.6.2** 多核共享 L3 与互连瓶颈。
