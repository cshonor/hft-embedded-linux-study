# 2.4.1 Cache 的工作原理

## 本节讲什么

**局部性原理**与 Cache 命中/失效 —— 数组性能、HFT/DPDK  cache line 优化基础。

## 要点

### 存储金字塔（快到慢）

```
寄存器 → L1 → L2 → L3 → RAM → Flash → 磁盘
  ns      │              μs      ms
```

### 局部性

| 类型 | 含义 | 代码示例 |
|------|------|----------|
| **时间** | 刚用过的还会再用 | 循环内变量 |
| **空间** | 相邻地址会一起用 | **数组顺序扫描** |

### Cache 行

- 主流 **64B/行**；一次载入整行  
- **miss** → 访问 RAM，延迟跳变  
- **抖动**：工作集 > Cache → 反复驱逐  

### 嵌入式优化

- 顺序遍历 `arr[i]`，避免随机跳地址  
- 结构体数组优于指针数组（空间局部性）  
- 多线程：**伪共享** — 同 cache line 不同变量（DPDK padding）
