# 2.5.5 乱序执行

## 本节讲什么

**OoO（Out-of-Order）** —— 在保持 **程序语义** 前提下重排指令，填流水线 bubble。

## 要点

```
保留站 → 执行单元就绪即发射 → 重排序缓冲(ROB) → 按序提交
```

- **提交（commit）** 仍按程序顺序；异常/精确中断点可恢复  
- 隐藏 **Cache miss、长延迟乘除** 带来的停顿  
- 对程序员：**单线程内存模型** 仍复杂（需 memory barrier）；多核见 **2.6**

- DPDK 无锁队列仍依赖 **原子 + barrier**；不能假设源码顺序 = 执行顺序
