# 5.4.2 uC/OS 的堆内存管理

## 本节讲什么

μC/OS-II/III 在 RTOS 内核内提供 **固定分区** 或 **可变堆**（`OSMem*` / `OSHeap*`），在 **禁止调度或关中断** 临界区内操作空闲链表，避免多任务竞态。

## 要点

### 两种策略

| 方式 | 特点 |
|------|------|
| **内存分区** | 等大小块，O(1) 分配，无外部碎片 |
| **堆管理** | 类似 first-fit，可合并，有碎片 |

- 配置：`OS_MEM_EN`、`OS_MAX_MEM_PART`；应用先 `OSMemCreate` 再 `OSMemGet`/`OSMemPut`
- 任务栈由内核 **静态数组** 或用户缓冲创建，与堆区分离

### 与裸机/Linux

- 介于 **5.4.1** 与 **5.4.3**：有任务切换但无 MMU 页表（常见 MCU）
- 分配失败返回 `NULL`，须检查；**禁止 ISR 里 malloc**（除非专用 ISR-safe 池）

### 调试

- 栈检查：`OSTaskStkChk` 看剩余栈字
- 堆泄漏：RTOS 层无 Valgrind，靠 **分配计数** 或 watermark
