# 5.4.3 Linux 堆内存管理

## 本节讲什么

glibc **ptmalloc2**：多 **arena** 减锁竞争；小块用 **main arena 的 brk 区**，大块 **`mmap` 直接映射**；每个 chunk 带 **prev_size/size 标志位** 与边界标记。

## 要点

### chunk 结构（概念）

```
[ prev_size | size | user data ... | padding ]
     ↑ 仅当前块 free 且前块 free 时有效
size 低位：P=上一块在用, M=mmap块, A=main arena
```

- **top chunk**：堆顶剩余；`malloc` 不足则 `brk()` 或 `mmap`
- **fastbin / tcache**：小空闲块缓存，加速复用
- **double free** → 链表破坏 → `malloc(): double free detected` / corruption

```bash
# 观察进程堆
cat /proc/self/maps | grep heap
pmap -x $$
```

### 工具

```bash
malloc_info(0, stdout);       /* 程序内 */
MALLOC_TRACE=./log mtrace ./prog
valgrind --leak-check=full ./demo04_heap_leak
```

### 内核 / DPDK

- **kmalloc**：slab 分配器，按对象大小选 cache；`kfree` 配对
- **vmalloc**：非连续物理页，内核虚拟连续
- **DPDK**：启动时 `rte_eal_init` mmap hugepage，运行时 **`rte_malloc`/`rte_free`**，避免 glibc 锁
