# 5.4.5 实现自己的堆管理器

## 本节讲什么

在 **固定字节数组** 上实现 **first-fit / best-fit** 空闲链表：`malloc` 分割块、`free` 合并相邻空闲块。理解 glibc chunk 的简化版，适合裸机、Bootloader、内核 early alloc。

## 要点

### 最小设计

```c
typedef struct block {
    size_t size;           /* 不含头 */
    int free;
    struct block *next;
} block_t;

static char heap[HEAP_SIZE];
static block_t *free_list;
```

- 对齐：8/16 字节；用户指针指向 **payload**，头在下方
- **合并**：`free` 时检查 `prev/next` 物理相邻且 free → 减碎片
- **realloc**：能原地扩大则扩，否则拷贝

### 必查错误

- 重复 `free`、释放非堆指针、越界写破坏 `size` 字段
- 用 **canary** 或 magic 校验块完整性

```bash
gcc -g -fsanitize=address -o myheap test_myheap.c myheap.c
./myheap
```

### 对比

| | 自研 | glibc |
|--|------|-------|
| 体积 | 可控 | 大 |
| 线程安全 | 需自加锁 | arena+mutex |
| 调试 | 简单 | mtrace/valgrind |

- **Linux 内核** slab 更复杂但思想类似：按对象大小分级缓存
