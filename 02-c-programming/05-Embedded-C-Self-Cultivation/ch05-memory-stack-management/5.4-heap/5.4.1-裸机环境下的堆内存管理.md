# 5.4.1 裸机环境下的堆内存管理

## 本节讲什么

无 OS 时没有 `sbrk`/`mmap`，堆是链接脚本划出的 **`.heap` 区间**（或 `_end` 到 RAM 顶）。分配器在 **固定物理/虚拟地址** 上维护空闲链表；与 **5.2 Linux MMU** 对比：裸机常 **无保护**，越界直接踩硬件。

## 要点

### 典型布局

```
Flash: .text .rodata
RAM:   .data .bss → heap → stack（相向生长，中间留 guard）
```

- 链接脚本：`__heap_start` / `__heap_end`；`_sbrk` 在 `__heap_start` 上 bump
- **newlib** `_sbrk`：向链接器符号要下一字节

```c
extern char __heap_start;
static char *heap_ptr = &__heap_start;
void *_sbrk(int incr) { char *prev = heap_ptr; heap_ptr += incr; return prev; }
```

### 简单 first-fit

- 空闲块头：`size + next` 指针；`malloc` 遍历分割，`free` 合并相邻
- **碎片**：频繁不等长 alloc/free 产生外碎片
- **嵌入式**：常改用 **固定块内存池**（网络 mbuf、DMA 缓冲区）

### 与 Linux

| 裸机 | Linux |
|------|-------|
| 物理地址固定 | VMA + 页对齐 |
| 无 double-free 检测 | glibc 可能 abort |
| 堆栈相撞 → 静默损坏 | 通常 SIGSEGV |
