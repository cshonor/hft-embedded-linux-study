# 5.7.5 内存检测神器：Valgrind

## 本节讲什么

**Valgrind Memcheck** 在 **虚拟 CPU** 上跑程序，拦截每次内存访问，报告 **非法读写、未初始化使用、泄漏、double free**。开发期标配；ARM Linux 可用，嵌入式裸机不可用。

## 要点

### 常用命令

```bash
valgrind --leak-check=full --show-leak-kinds=all ./demo04_heap_leak
valgrind --track-origins=yes ./prog          # 未初始化值来源
valgrind --tool=memcheck --vgdb=yes ./prog   # 配合 gdb
valgrind --help
```

### 报告解读

```
Invalid read of size 4          → 越界读
Use of uninitialised value    → 未初始化
Invalid free / double free    → 释放错误
definitely lost               → 确定泄漏
```

### 与 ASAN 对比

| | Valgrind | ASAN |
|--|----------|------|
| 原理 | 动态二进制翻译 | 编译插桩 |
| 速度 | 很慢 | 约 2x 慢 |
| 需重编译 | 否 | 是 |
| 内核 | 否 | KASAN（内核） |

```bash
gcc -fsanitize=address,undefined -g -O0 -o prog prog.c
./prog
# 内核模块：CONFIG_KASAN=y
```

### DPDK 注意

- 用户态 DPDK 可在 **非 PMD 测试代码** 上 Valgrind；性能路径/hugepage 行为可能误报，需 `--suppressions`
