# 5.7.2 使用 core dump 调试段错误

## 本节讲什么

进程因 **SIGSEGV/SIGABRT** 等异常终止时，内核可写 **core 文件**（进程内存快照）。用 **GDB 加载可执行文件 + core** 在崩溃点 **bt**、查变量、反汇编。

## 要点

### 开启 core

```bash
ulimit -c unlimited
cat /proc/sys/kernel/core_pattern    # core 路径模板
# systemd: /var/lib/systemd/coredump/ 或当前目录 core

gcc -g -O0 -o crash crash.c
./crash                            # SIGSEGV
gdb ./crash core                   # 或 core.crash.PID
(gdb) bt
(gdb) bt full
(gdb) frame 0
(gdb) list
(gdb) info registers
(gdb) x/16wx $sp
```

### 无 core 时

```bash
gdb ./crash
(gdb) run
# 崩溃后直接 bt
(gdb) bt
```

### 技巧

- 发布版保留 **未 strip** 符号或单独 **debuginfo** 包
- **ARM 交叉**：`arm-linux-gnueabihf-gdb` + `set sysroot`
- 内核：**kdump/vmcore** + `crash` 工具，思路相同

| 信号 | 常见原因 |
|------|----------|
| SIGSEGV | 非法地址读写 |
| SIGABRT | assert、heap corruption |
