# 5.7.4 内存踩踏监测：mprotect

## 本节讲什么

用 **`mprotect`** 在缓冲区两侧映射 **PROT_NONE 保护页**，越界访问立即 **SIGSEGV**，比静默损坏易定位。glibc **guard malloc**、调试分配器、部分 **Electric Fence** 思路同源。

## 要点

```c
#include <sys/mman.h>
size_t page = sysconf(_SC_PAGESIZE);
void *base = mmap(NULL, 3*page, PROT_READ|PROT_WRITE,
                  MAP_PRIVATE|MAP_ANONYMOUS, -1, 0);
mprotect(base, page, PROT_NONE);                    /* 左 guard */
mprotect((char*)base + 2*page, page, PROT_NONE);  /* 右 guard */
char *buf = (char*)base + page;
/* buf 仅一页；越界 → SIGSEGV */
```

```bash
gcc -g -o guard guard.c
gdb ./guard
(gdb) run
# 越界时 SIGSEGV，bt 指向越界语句
```

### 其它手段

| 手段 | 说明 |
|------|------|
| `-fstack-protector` | 栈 canary |
| ASAN | 编译器插 redzone |
| `mprotect` | 手动 guard page |
| 内核 **KASAN** | 内核态 ASAN 同类 |

- 开销大，多用于 **测试构建**；生产用代码审查 + 边界 API
- **DPDK debug**：rte_malloc 带 cookie 检查
