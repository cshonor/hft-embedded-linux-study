# 5.7.3 什么是内存踩踏

## 本节讲什么

**内存踩踏（memory stomp）**：写入 **合法对象边界之外**，破坏相邻对象、堆 metadata、返回地址。读踩踏可能 SIGSEGV；写踩踏可能 **延迟崩溃**（corruption 后才 abort）。

## 要点

### 典型场景

```c
char buf[8];
strcpy(buf, "0123456789");     /* 越界写 */
int *p = malloc(16);
free(p);
*p = 1;                        /* use-after-free 写 */
free(p);                       /* double free */
```

- **堆**：覆盖相邻 chunk 的 `size` → `malloc` 后续崩溃
- **栈**：覆盖 saved LR（**5.3.6**）
- **数组下标**：`a[-1]`、`a[n]` 在循环 off-by-one

```bash
valgrind --tool=memcheck ./prog
# Invalid write of size 1 at 0x...
gcc -fsanitize=address -g -O0 -o prog prog.c
```

### 内核

- **buffer overrun** in `copy_from_user` 长度检查不严
- **SLUB RED_ZONE** / `CONFIG_DEBUG_PAGEALLOC` 辅助检测
