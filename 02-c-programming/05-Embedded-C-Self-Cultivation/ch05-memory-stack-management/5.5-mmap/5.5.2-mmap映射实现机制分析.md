# 5.5.2 mmap 映射实现机制分析

## 本节讲什么

用户态 `mmap()` → 内核 **`do_mmap`/`mmap_region`**：在进程 **mm_struct** 找空洞建 **VMA**，`mmap` 文件则挂 **file->f_mapping**，匿名则只 reserve 虚拟区；**首次访问** 触发缺页分配物理页或读入页缓存。

## 要点

### 流程

1. 找合适虚拟地址（`mmap_base` 向下或 `hint`）
2. 创建 VMA（`vm_area_struct`：start/end/prot/flags/file）
3. 读/写缺页 → **filemap_fault** 或 **匿名零页**
4. `munmap` 拆 VMA、释放页表项

```bash
cat /proc/self/maps
# 地址范围 权限 offset dev inode pathname
```

### 与 brk

| brk | mmap |
|-----|------|
| 堆顶连续 bump | 任意空洞 |
| 小对象 heap | 大块、文件、设备 |
| 单 main arena 延伸 | 独立 VMA |

- **TLB shootdown**：多核修改页表需 IPI
- **内核模块**：`ioremap` 把 **物理 MMIO** 映射到内核虚拟（非用户 mmap）
