# 5.5.3 把设备映射到内存

## 本节讲什么

**MMIO**：外设寄存器映射到物理地址，CPU **load/store** 即读写硬件。Linux 用户态通过 **`/dev/uio`、`/dev/mem` 或驱动 mmap** 把物理窗口映射进进程，指针访问替代 `in/out`（x86）或总线周期。

## 要点

### 用户态 UIO 示例（概念）

```c
int fd = open("/dev/uio0", O_RDWR);
void *regs = mmap(NULL, map_size, PROT_READ|PROT_WRITE,
                  MAP_SHARED, fd, 0);
volatile uint32_t *ctrl = (uint32_t *)((char *)regs + offset);
*ctrl = 1;   /* 写寄存器 */
```

```bash
ls -l /dev/uio* /dev/mem
# 需 root 或 udev 规则；优先用内核驱动抽象，勿裸 mmap /dev/mem
```

### 裸机 vs Linux

| 裸机 | Linux |
|------|-------|
| `#define REG (*(volatile uint32_t*)0x40001000)` | 驱动 `ioremap` + 用户 mmap |
| 物理固定 | 虚拟 + 驱动管理权限 |

- **volatile**：防止编译器优化掉对 MMIO 的访问
- **DPDK**：`rte_memzone`、`pci_map_device` 映射 BAR 到用户态做 PMD 轮询
- **缓存一致性**：DMA 缓冲区常需 **uncached** 或 explicit flush
