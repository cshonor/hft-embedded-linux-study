# 5.6.3 内存泄漏检测：MTrace

## 本节讲什么

glibc **mtrace** 钩住 `malloc/free`，把分配记录写入日志，程序结束或 `muntrace()` 时输出 **未释放块** 及调用栈（需 `-rdynamic` 才有符号）。

## 要点

```c
#include <mcheck.h>
int main(void) {
    mtrace();          /* 或环境变量 MALLOC_TRACE=file */
    ...
    muntrace();
}
```

```bash
gcc -g -rdynamic -o leak leak.c
export MALLOC_TRACE=./mtrace.log
./leak
muntrace-leaks ./mtrace.log    # 部分发行版
# 或 mtrace ./leak $MALLOC_TRACE 解析
```

| 工具 | 特点 |
|------|------|
| mtrace | 轻量、glibc 专用、开销小 |
| Valgrind | 全面、慢 10–50x |
| ASAN leak | `-fsanitize=leak` |

- 生产环境常用 **抽样 + 离线分析**；开发期 Valgrind/ASAN
- **内核 kmemleak**：扫描未引用 kmalloc 块（需配置 `CONFIG_KMEMLEAK`）
