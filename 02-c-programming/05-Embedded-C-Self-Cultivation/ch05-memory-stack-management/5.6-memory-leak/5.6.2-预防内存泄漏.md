# 5.6.2 预防内存泄漏

## 本节讲什么

从 **设计习惯** 上保证每条 `malloc` 路径都有对应 `free`：单一出口 cleanup、结构体析构函数模式、避免隐式所有权转移混乱。

## 要点

### 实践

```c
int foo(void) {
    char *p = malloc(n);
    if (!p) return -1;
    if (err) { free(p); return -1; }   /* 每出口释放 */
    ...
    free(p);
    return 0;
}
```

- **配对 API**：`fopen/fclose`、`mmap/munmap`、`rte_malloc/rte_free`
- **容器销毁函数**：`list_destroy()` 内部遍历 free
- 禁用或封装裸 `malloc`；用 **静态分析**（clang-tidy `clang-analyzer-*`）

### 内核 / 嵌入式

- **kmalloc 必须 kfree**；错误路径 `goto err_free`
- 裸机池：**固定生命周期** 对象不 free，重启时整块回收
- **DPDK mempool**：预分配 mbuf，用完 `rte_pktmbuf_free` 回池，非传统 free

```bash
cppcheck --enable=all *.c
```
