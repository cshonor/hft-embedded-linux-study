# 5.6 内存泄漏与防范

## 本节讲什么

**内存泄漏**：已分配堆块 **失去所有指针** 且未 `free`，进程 RSS 单调涨。长期运行的服务器、嵌入式长期任务、DPDK 数据面均需 **配对分配/释放** 与工具检测。

## 要点

- 典型：`malloc` 后丢指针、异常路径未 `free`、循环里重复 alloc
- 检测：**Valgrind**、**mtrace**、ASAN leak、内核 **kmemleak**
- 预防：谁分配谁释放、RAII 思想（C 里 goto cleanup）、池化复用

```bash
cd demo && valgrind --leak-check=full ./demo04_heap_leak
```
