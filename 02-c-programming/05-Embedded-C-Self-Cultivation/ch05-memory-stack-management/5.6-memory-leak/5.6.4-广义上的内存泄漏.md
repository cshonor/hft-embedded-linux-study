# 5.6.4 广义上的内存泄漏

## 本节讲什么

除 **堆块永不释放** 外，「广义泄漏」包括：**文件描述符**、**mmap 未 munmap**、**线程/锁未销毁**、**内核引用计数泄漏**、**池化对象未归还**——资源总量有限，表现同为系统变慢或 OOM。

## 要点

### 类型

| 资源 | 症状 |
|------|------|
| FD | `Too many open files` |
| mmap | VMA 数量涨，`/proc/maps` 变长 |
| 线程 | pid/max 限制 |
| 内核模块引用 | `rmmod` 失败 |

```bash
ls /proc/self/fd | wc -l
cat /proc/self/maps | wc -l
lsof -p $$
```

### 与堆泄漏区别

- 进程退出 OS **回收** 用户堆，但 **泄漏期间** 仍占 RAM
- **嵌入式无 exit**：广义泄漏 = 致命
- **DPDK**：mbuf 未还 pool → 等价包缓冲区泄漏，最终无法收包

### 检测

- `valgrind --track-fds=yes`
- 内核：`/proc/meminfo`、`slabtop`、kmemleak
