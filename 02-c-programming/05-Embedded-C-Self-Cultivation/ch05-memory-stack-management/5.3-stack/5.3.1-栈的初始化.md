# 5.3.1 栈的初始化

## 本节讲什么

进程/线程创建时，内核在用户地址空间顶部（或 pthread 属性指定处）映射 **栈区** 并设置初始 **SP**。裸机则由启动代码在链接脚本定义的 `__stack_top` 写入 SP。

## 要点

### Linux 进程栈

- `execve` 后内核映射 `[stack]`，`argc/argv/envp` 与 **辅助向量** 压栈
- 主线程 SP 指向栈顶；`pthread_create` 为每线程分配独立栈（默认 ~8MB，可 `pthread_attr_setstacksize`）

```bash
cat /proc/self/maps | grep stack
ulimit -a | grep stack
```

### 裸机 / RTOS

```asm
/* startup：SP 指向 RAM 栈顶 */
ldr sp, =__stack_top
```

- uC/OS 等：任务控制块记录 **栈底/栈顶**，创建任务时 SP 指向栈顶
- 嵌入式：**静态定栈** 或 `_sstack`/`_estack` 链接符号；栈水位用填充图案 `0xDEADBEEF` 检测

### 与 x86/ARM 差异

- **ARM AAPCS**：SP 8 字节对齐；函数入口常 `push {fp, lr}` 建立帧链
- 内核每个 task 有 **内核栈**（独立），与用户栈分开
