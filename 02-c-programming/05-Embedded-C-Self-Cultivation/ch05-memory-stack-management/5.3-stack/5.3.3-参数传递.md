# 5.3.3 参数传递

## 本节讲什么

**AAPCS** 规定：前 4 个整型/指针参数用 **r0–r3**，第 5 个起 **入栈**；浮点另有 VFP 规则。返回值 **r0**（64 位用 r0+r1）。结构体过大则传指针或按规则拆寄存器。

## 要点

### 寄存器 vs 栈

```c
void foo(int a, int b, int c, int d, int e) { /* e 在栈上 */ }
```

```bash
arm-linux-gnueabihf-gcc -S -O0 -o - foo.c   # 看 r0–r3 与 [sp, #offset]
# x86_64 不同：rdi, rsi, rdx, rcx, r8, r9 后入栈
```

| 类型 | ARM 32（AAPCS） |
|------|-----------------|
| int/pointer | r0–r3，余者栈 |
| 返回值 | r0 |
| 被调者保存 | r4–r11, fp |
| 调用者保存 | r0–r3, lr |

### 数组与指针

- 数组作实参 **退化为指针**，传地址（4 字节），不拷贝整数组
- **内核** 拷贝用户缓冲区：`copy_from_user()`，避免直接解引用用户指针

### GDB

```bash
(gdb) break foo
(gdb) run
(gdb) info args
(gdb) print/x $r0 $r1 $r2 $r3
(gdb) x/4wx $sp    # 栈上第 5 个参数
```
