# 5.3.2 函数调用

## 本节讲什么

函数调用 = **保存现场 + 跳转 + 被调方建帧 + 返回恢复**。ARM 经典序列：**`push {fp, lr}` → `mov fp, sp` → 减 SP 留局部 → `bl` 写 LR → 返回 `ldmfd sp!, {fp, pc}`**。本节是全书栈分析的 **核心**，配合 GDB 单步与 `x` 看内存。

## 要点

### ARM 栈帧（A32 示例）

```c
int callee(int a) {
    int local = a + 1;
    return local;
}
int caller(void) {
    return callee(42);
}
```

典型汇编骨架：

```asm
caller:
    push   {fp, lr}          @ 保存旧 fp、返回地址到栈
    mov    fp, sp            @ fp 指向本帧基址
    sub    sp, sp, #8        @ 给局部变量腾空间
    mov    r0, #42
    bl     callee            @ lr = 下一条指令地址
    mov    sp, fp
    pop    {fp, pc}          @ 等价 ldmfd sp!, {fp, pc}

callee:
    push   {fp, lr}
    mov    fp, sp
    sub    sp, sp, #4        @ int local
    ...
    mov    sp, fp
    pop    {fp, pc}          @ pc=lr，返回 caller
```

- **`bl`**：Branch with Link，目标地址进 **LR (r14)**
- **`ldmfd sp!, {fp, pc}`**：从栈弹 fp 和 pc（pc←原 lr），**FD=Full Descending** 栈向低址增
- **`stmfd sp!, {…}`** 与 push 同类：先减 SP 再存

### GDB 实战

```bash
gcc -g -O0 -fno-omit-frame-pointer -o demo demo02_stack_frame.c
gdb ./demo02_stack_frame
(gdb) break recurse
(gdb) run
(gdb) bt                    # 回溯：#0 recurse #1 recurse ... #N main
(gdb) info registers sp fp lr pc
(gdb) x/16wx $sp            # 以字查看栈内存
(gdb) x/8i $pc              # 反汇编当前指令
(gdb) disassemble recurse
(gdb) frame 2               # 切换到第 2 帧
(gdb) info locals
```

`demo02` 递归时每个 `frame` 局部变量地址递减 → 直观看到栈向下生长。

### 栈帧布局（概念）

```
高地址
  [caller 的 fp][caller 的 lr]
  [callee 局部变量...]
  [多余参数...]        ← fp 常指向 caller sp 快照
低地址 ← SP
```

- **FP 链**：沿 `fp → 上一帧 fp` 可手动回溯（`-fomit-frame-pointer` 时 `bt` 变难）
- **内核**：`dump_stack()` / `unwind` 依赖 ORC/ftrace 或帧指针

### DPDK / 性能注意

- 热点路径避免深递归与大栈帧；DPDK 常 **禁用递归**、用 per-lcore 预分配
- `-O2` 可能省略 FP；调栈问题时用 **`-O0 -fno-omit-frame-pointer`**
