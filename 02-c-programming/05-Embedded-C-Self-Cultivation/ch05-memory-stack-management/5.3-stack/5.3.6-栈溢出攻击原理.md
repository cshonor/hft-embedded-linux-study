# 5.3.6 栈溢出攻击原理

## 本节讲什么

**栈溢出**：写越界覆盖相邻栈内容，典型目标是 **saved LR/返回地址** 或 **栈 canary**。历史上用于注入 shellcode；现代有 **NX、ASLR、Stack Protector**。嵌入式无 MMU 时同样会因栈越界踩坏相邻 RAM。

## 要点

### 成因

```c
void vuln(void) {
    char buf[64];
    gets(buf);              /* 无边界检查 → 覆盖 fp/lr */
}
```

- 递归过深、超大局部数组 → **合法但危险**，触发 SIGSEGV
- **demo03**（见 demo/README）：无限递归或超大数组，用 GDB 观察

```bash
gcc -g -O0 -fstack-protector-all -o vuln vuln.c
ulimit -s 4096
gdb ./vuln
(gdb) run
(gdb) bt
(gdb) info registers sp lr
```

### 防护

| 机制 | 作用 |
|------|------|
| `-fstack-protector` | 帧入口插入 canary，返回前校验 |
| NX / W^X | 栈不可执行 |
| ASLR | 栈/库基址随机 |
| **ASAN** | 编译期插桩，检测越界（`-fsanitize=address`） |

```bash
gcc -fsanitize=address -g -O0 -o vuln_asan vuln.c
./vuln_asan
```

### 嵌入式规则

- **禁止** `gets/strcpy/sprintf` 无界 API；用 `snprintf`、显式长度
- 为每任务 **实测栈水位**（填充字 + 运行时检查）
- 内核栈仅 **8KB/16KB** 级，内核代码严禁大栈数组
