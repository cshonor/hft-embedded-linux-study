# 5.3.5 栈与作用域

## 本节讲什么

**自动变量**（局部非 static）在函数每次调用时在 **栈** 上分配，离开作用域即失效。**静态局部** 存 **`.bss`/`.data`**，只初始化一次，生命周期贯穿整个程序——与 **ch04 BSS** 机制相同。

## 要点

### 存储类别对比

| 声明 | 存储位置 | 生命周期 |
|------|----------|----------|
| 局部 `int x` | 栈 | 函数调用期间 |
| `static int c`（函数内） | .bss/.data | 程序全程 |
| 全局 | .bss/.data | 程序全程 |

```bash
cd demo && make demo05_static && ./demo05_static
# static=1,2,3 累加；local 每次为 1
size demo05_static
nm demo05_static | grep counter
```

### demo05 行为

- `static int c`：第一次为 0，之后保留上次值
- `int local`：每次调用重新分配（值未定义除非赋值），打印恒为 1

### 嵌入式注意

- 大数组放栈 → **栈溢出**；改 `static`、全局或堆
- ISR 中避免大栈帧；RTOS 任务栈需按 **最坏调用深度** 估算
- **DPDK**：per-lcore 变量用 `RTE_PER_LCORE` 或 TLS，避免栈上大对象

```bash
readelf -S demo05_static | grep -E 'bss|data'
```
