# 5.1 程序运行的「马甲」：进程

## 本节讲什么

**进程** 是操作系统给正在运行的程序穿上的「马甲」：同一 ELF 可多次 `exec`，每次得到独立 **PID、虚拟地址空间、文件描述符表、堆/栈**。用户态看到的 `.text/.rodata/.data/.bss/heap/stack` 五区，在 Linux 里由 **VMA** 映射到物理页；与 **ch04** ELF 段一一对应。

## 要点

### 五段与进程地址空间

| 区域 | ELF/来源 | 典型内容 |
|------|----------|----------|
| `.text` | PT_LOAD | 机器码、`main` |
| `.rodata` | PT_LOAD | 字符串字面量、只读常量 |
| `.data` | PT_LOAD | 已初始化全局/静态 |
| `.bss` | PT_LOAD（NOBITS） | 未初始化全局/静态，加载时清零 |
| heap | `brk`/`mmap` | `malloc` arena |
| stack | 内核映射 | 局部变量、返回地址 |

```bash
cd demo && make demo01_memory_zone && ./demo01_memory_zone
size demo01_memory_zone
readelf -S demo01_memory_zone | grep -E 'text|rodata|data|bss'
cat /proc/self/maps    # 运行中观察各段
```

### 进程 vs 程序

- **程序**：磁盘上的 ELF；**进程**：内存中的实例 + 内核 `task_struct`
- 多进程共享 **代码页**（CoW）；堆/栈/私有数据各自独立
- 内核/DPDK：内核无传统 `malloc` 用户堆，用 `kmalloc`/`vmalloc`；DPDK 用 **hugepage mmap** 预分配 pktmbuf 池
