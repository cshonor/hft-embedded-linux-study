# 5.2 Linux 环境下的内存管理

## 本节讲什么

Linux 用 **MMU + 页表** 为每个进程提供独立虚拟地址；内核以 **VMA（vm_area_struct）** 描述一段连续虚拟区间及其权限。用户态五段 + mmap 区都体现在 `/proc/<pid>/maps`；与裸机「物理地址直连」形成对比。

## 要点

### VMA 与 maps

```bash
./demo01_memory_zone &
PID=$!
cat /proc/$PID/maps
# 典型行：r-xp .text  r--p .rodata  rw-p .data/.bss  rw-p [heap]  rw-p [stack]
pmap -x $PID
```

| 权限 | 含义 |
|------|------|
| `r-x` | 代码 |
| `r--` | 只读数据 |
| `rw-` | 可读写数据/堆/栈 |

### 与 ch04 ELF 的对应

- `execve` 按 **Program Header** 建立 LOAD 段 VMA
- **BSS** 并入最后一 LOAD，占虚拟页、文件内 `memsz > filesz` 部分为零
- **ASLR**：栈、heap、mmap 基址随机化

### 裸机 vs Linux MMU

| | 裸机 | Linux 进程 |
|--|------|------------|
| 地址 | 常物理/固定链接 | 虚拟，经 MMU |
| 保护 | 无或 MPU | 页级 R/W/X |
| 扩展 | 链接脚本定界 | `brk`/`mmap` |

- **内核**：`copy_from_user`/`get_user` 访问用户 VMA；缺页走 `do_page_fault`
- **DPDK**：绕过内核协议栈，仍依赖 **mmap hugepage** 拿连续物理内存
