# 1.2.2 使用 gcc 编译 C 源程序

## 本节讲什么

**GCC 四阶段**与高频参数 —— ch04 前置；交叉编译入门。

## 要点

### 四阶段

```bash
gcc -E test.c -o test.i      # 预处理
gcc -S test.i -o test.s      # 编译 → 汇编
gcc -c test.s -o test.o      # 汇编 → 目标文件
gcc test.o -o test           # 链接 → 可执行文件
```

### 常用参数

| 参数 | 作用 |
|------|------|
| **-Wall -Wextra** | 全开警告（强制） |
| **-std=c11** | 语言标准 |
| **-O0 / -O2** | 调试 / 发布 |
| **-g** | 调试符号（gdb） |
| **-c** | 只编译不链接 |

```bash
gcc -Wall -Wextra -std=c11 -g -O0 main.c -o app
```

### 交叉编译

```bash
arm-linux-gnueabihf-gcc -g main.c -o app.arm
file app.arm
```

- 通过 **buildenv.sh** 设置 `CROSS_COMPILE`（**1.6**）

## 实操

对比 **-O0** 与 **-O2** 的 `objdump -d` 差异（**1.4**）。
