# 1.5 GDB 源码级调试

## 本节讲什么

**gdb** 断点、单步、看内存、backtrace、**core** —— 定位空指针/栈溢出。

## 要点

### 基本流程

```bash
gcc -g -O0 -Wall main.c -o app
gdb ./app
```

```
(gdb) break main
(gdb) run
(gdb) next          # 单步跳过
(gdb) step          # 单步进入
(gdb) print p
(gdb) print *ptr
(gdb) x/16xb &buf   # 16 字节十六进制
(gdb) disassemble main
(gdb) bt            # 调用栈
(gdb) quit
```

### core 转储

```bash
ulimit -c unlimited
./app    # 崩溃生成 core
gdb ./app core
(gdb) bt
```

| 命令 | 作用 |
|------|------|
| **break** | 断点 |
| **x** | 检查内存 |
| **bt** | 栈回溯 |

- 编译须 **-g**；发布 **-O2** 仍可调试但变量可能被优化掉
