# 1.6 环境脚本 buildenv.sh

## 本节讲什么

**`source buildenv.sh`** 在当前 Shell 注入 **PROJECT_ROOT、CC、CFLAGS** 与快捷函数 —— 隔离项目编译环境。

## 要点

### source vs bash

```bash
source ../devenv/buildenv.sh   # 当前 shell 生效
bash ../devenv/buildenv.sh     # 子 shell，变量丢失
```

### 脚本内容（仓库已提供）

路径：`05-Embedded-C-Self-Cultivation/devenv/buildenv.sh`

| 导出 | 作用 |
|------|------|
| **PROJECT_ROOT** | 工程根目录 |
| **CC / CROSS_COMPILE** | 本地或交叉 gcc |
| **CFLAGS** | -Wall -Wextra -std=c11 -g |
| **build / clean** | 封装 make |

```bash
cd 05-Embedded-C-Self-Cultivation
source devenv/buildenv.sh
build ch01-tools-of-the-trade/demo
```

### 永久加载（可选）

```bash
echo 'source ~/path/to/devenv/buildenv.sh' >> ~/.bashrc
```

### 交叉编译

```bash
export CROSS_COMPILE=arm-linux-gnueabihf-
source devenv/buildenv.sh
echo $CC    # arm-linux-gnueabihf-gcc
```
