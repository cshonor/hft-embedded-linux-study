# 7.9.2 数组名的本质

## 本节讲什么

数组名是**非指针常量**：表示整个数组，在表达式中通常**衰变**为指向首元素的指针，但三个例外下数组名仍是「数组类型」：`sizeof`、取址 `&`、初始化时。传参衰变是 `sizeof` 陷阱与 `ARRAY_SIZE` 失效的根源。

## 要点

```c
char buf[16];
char *p = buf;        /* decay: buf → &buf[0] */

&buf;                 /* 类型 char (*)[16] */
buf;                  /* decay 后类型 char *，值常等于 &buf[0] */

sizeof buf;           /* 16 */
void f(char a[16]) { sizeof a; }  /* 8，形参是指针 */
```

| 表达式 | 类型（典型） | 值 |
|--------|--------------|-----|
| `buf` | `char *`（decay） | 首元素地址 |
| `&buf` | `char (*)[16]` | 数组地址 |
| `&buf[0]` | `char *` | 首元素地址 |

```c
#define ARRAY_SIZE(a) (sizeof(a) / sizeof((a)[0]))
/* 仅用于实体数组，指针参数无效 */

int *hp = malloc(n * sizeof *hp);
/* ARRAY_SIZE(hp) 错误！ */
```

```bash
./demo01   # 观察 buf、p、&buf 打印
```

```gdb
(gdb) print buf
(gdb) print &buf
(gdb) print sizeof(buf)
```

- **字符串字面量**：类型 `char[N]`，decay 为 `char *`，存储只读
- **static 数组形参**：`void f(char a[static 16])` C99 优化提示
- **VLA**：运行时大小，仍 decay
