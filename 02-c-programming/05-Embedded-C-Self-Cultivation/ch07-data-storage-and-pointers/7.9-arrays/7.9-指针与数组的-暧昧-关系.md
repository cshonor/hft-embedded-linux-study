# 7.9 指针与数组的「暧昧」关系

## 本节讲什么

数组名在多数表达式中**衰变为**指向首元素的指针，但数组不是指针：不可赋值、不可 `sizeof` 与指针混淆。这一「暧昧」导致 `sizeof`、传参、指针算术的经典面试题与真实 bug。本节总览差异，细节见 **7.9.1–7.9.3** 与 **demo01**。

## 要点

```c
char buf[16] = "hello";
char *p = buf;

buf == p;              /* 真，首元素地址 */
/* buf = p; */         /* 编译错误：数组名非左值可赋值 */

sizeof buf;            /* 16 */
sizeof p;              /* 8（64 位指针） */
```

| 维度 | 数组 `T a[N]` | 指针 `T *p` |
|------|---------------|-------------|
| 存储 | 连续 N 个 T | 一个地址 |
| 赋值 | 不可整体赋值 | 可改指向 |
| sizeof | `N * sizeof(T)` | `sizeof(T *)` |
| 传参 | 衰变为 `T *` | 仍是 `T *` |

```c
void pass(char arr[16]) {
    sizeof arr;   /* 指针大小！仅形参处 */
}
```

```bash
cd demo && gcc demo01_array_ptr.c -o demo01 && ./demo01
```

- **`&buf`**：类型 `char (*)[16]`，整个数组的地址（与 `buf` 值常相等但类型不同）
- **多维数组**：decay 为指向首行指针，非 `T **` → **7.11.3**
- **内核** `ARRAY_SIZE` 仅用于真实数组
