# 7.4 Linux内核中的size_t类型

## 本节讲什么

`size_t` 是 C 标准定义的**无符号整数类型**，表示对象大小与数组下标，宽度与 `sizeof` 结果一致（32 位机常为 32 位，64 位机为 64 位）。内核中几乎所有长度、计数、`memcpy`/`strlen` 返回值都用 `size_t`；与 `int` 混用是比较与循环 bug 的重灾区。

## 要点

```c
#include <stddef.h>   /* size_t */
#include <string.h>

size_t n = strlen(buf);
void *p = malloc(n + 1);   /* n+1 前检查溢出，7.1.3 */

/* 内核 */
size_t copy_to_user(void __user *to, const void *from, size_t n);
```

| API | 返回/参数中的 size_t |
|-----|----------------------|
| `sizeof` | `size_t` |
| `strlen` / `strnlen` | `size_t` |
| `malloc` / `kmalloc` | 请求 `size_t` |
| `read` / `write` | `count` 常为 `size_t` |

**经典陷阱**：

```c
int len = -1;
size_t u = 10;
if (len < u) { }  /* false：len 转为巨大 size_t */

for (int i = 0; i < strlen(s); i++)  /* 每次 O(n)，且 i 类型不佳 */
```

**相关类型**：

| 类型 | 用途 |
|------|------|
| `ssize_t` | 有符号大小/错误码（`-1`） |
| `ptrdiff_t` | 指针相减 |
| `off_t` | 文件偏移 |

```bash
grep -r 'typedef.*size_t' /usr/include 2>/dev/null | head
```

- **内核 `kmalloc(size, gfp)`**：`size` 过大返回 NULL；注意 `SIZE_MAX` 边界
- **DPDK** 类似用 `size_t` 与 `uint32_t` 描述 buffer length（依 API）
- **格式化**：`%zu` 打印 `size_t`；`%zd` 给 `ssize_t`
