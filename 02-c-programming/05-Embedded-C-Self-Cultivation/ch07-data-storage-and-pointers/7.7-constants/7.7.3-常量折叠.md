# 7.7.3 常量折叠

## 本节讲什么

**常量折叠**是编译器在编译期计算常量表达式的优化：如 `3 + 5` 直接变为 `8`，数组大小、`case` 标签、`enum` 初值依赖此特性。`__builtin_constant_p`（ch06 **6.11.4**）让宏区分编译期常量与运行期值。理解折叠有助于写高效宏与静态断言。

## 要点

```c
#define SIZE (1024 * 4)
char buf[SIZE];              /* SIZE 编译期可知 */

int n = 10;
char bad[n];                 /* VLA，C99；C11 可选，非折叠 */

_Static_assert(sizeof(int) >= 4, "int too small");
```

```c
/* 折叠 vs 运行期 */
enum { MASK = (1u << 3) | (1u << 5) };   /* 编译期 0x28 */

int shift(int x) { return 1 << x; }      /* 运行期 */
```

| 工具 | 用途 |
|------|------|
| `gcc -E` | 查看宏展开后是否仍为常量 |
| `__builtin_constant_p(x)` | 宏分支选择 |
| `static_assert` / `_Static_assert` | 编译期约束 |

```bash
echo 'int a[2+3];' | gcc -c -xc - -o /dev/null -Wall
gcc -E const.c | less    # 查看折叠结果
```

```c
/* ch06 风格 */
#define IS_CONST(x) __builtin_constant_p(x)
#define OPTIMIZED_MAX(a, b) \
    (__builtin_constant_p((a) & (b)) ? ((a) > (b) ? (a) : (b)) : max_func(a, b))
```

- **未折叠副作用**：`#define SQR(x) ((x)*(x))` 中 `x` 有副作用会重复求值
- **内核 `ARRAY_SIZE(arr)`**：要求编译期已知长度，指针非数组时陷阱 → **7.9**
