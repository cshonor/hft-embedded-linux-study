# 7.5.3 使用typedef需要注意的地方

## 本节讲什么

typedef 不改变类型本质，却常让人误以为创建了「新类型」。与 `const`、指针、数组组合时，**展开规则**与直觉不符；函数指针 typedef 漏写括号会导致编译通过但语义错误。本节列出高频陷阱。

## 要点

**const 陷阱**：

```c
typedef int *intptr_t;
const intptr_t p;   /* const 修饰的是 intptr_t，即 int *const p */
/* 等价 int *const p — 指针不可改，指向的 int 可变 */

typedef int const *intptr_const_t;
intptr_const_t q;   /* const int * — 指向常量 */
```

**指针数组 vs 数组指针**（详见 **7.9.3**）：

```c
typedef char *str_t;
str_t tab[10];      /* char *tab[10] 指针数组 */

typedef char str_arr_t[10];
str_arr_t *p;       /* 指向「含 10 char 的数组」的指针 */
```

**函数指针**：

```c
typedef int handler_t(void);   /* 错误：handler_t 是返回 int 的函数，非指针 */
typedef int (*handler_t)(void); /* 正确 */
```

| 问题 | 后果 |
|------|------|
| 隐藏 `*` | 不知道变量是否指针 |
| 重复 typedef 同一底层 | 类型兼容但阅读混乱 |
| `#define ptr_t char *` | 宏展开与 const 组合灾难 |

```bash
gcc -Wpedantic -Wtypedef-types wrong.c
```

- **opaque struct**：`typedef struct foo foo_t;` 前向声明与完整定义需一致
- **勿 typedef 基本 int 为 `status`**：与有符号/无符号混用更隐蔽
