# 7.5.2 使用typedef的优势

## 本节讲什么

在大型 C 工程（内核、DPDK、固件）中，typedef 的核心价值是**语义命名**与**平台换型单点修改**。好的别名让 API 自解释；坏的使用仅缩短打字。本节列举嵌入式实践中 typedef 的真正收益。

## 要点

| 优势 | 说明 |
|------|------|
| 可读性 | `gpio_irq_handler_t` 比 `void (*)(int)` 清晰 |
| 移植 | `typedef uint32_t reg_val_t;` 换平台改一处 |
| 封装 | 头文件暴露 `struct device *`，隐藏成员 |
| 函数指针表 | `calc_fn table[]` → **demo04** jump table |
| 与工具链 | 部分静态分析对命名类型更友好 |

```c
/* 驱动状态机 */
typedef enum {
    DRV_IDLE,
    DRV_PROBE,
    DRV_READY,
} drv_state_t;

typedef int (*probe_fn_t)(struct platform_device *pdev);
```

```c
/* DPDK 风格（示意） */
typedef uint16_t port_id_t;
typedef struct rte_mbuf pkt_buf_t;
```

- **文档即类型**：`dma_addr_t` 暗示不宜与普通整数混算
- **减少 void\***：opaque `handle_t` 比裸 void* 稍安全（仍靠约定）
- **不过度**：基本 `int`/`char *` 无需再 typedef
