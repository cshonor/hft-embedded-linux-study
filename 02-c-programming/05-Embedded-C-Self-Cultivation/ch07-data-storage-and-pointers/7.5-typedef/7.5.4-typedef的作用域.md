# 7.5.4 typedef的作用域

## 本节讲什么

typedef 遵循与普通声明相同的**作用域规则**：块内、文件内、`static` 限制链接属性。头文件中 typedef 会随 `#include` 传播到所有翻译单元；重复或冲突的 typedef 会导致难以诊断的编译错误。理解作用域有助于组织驱动私有类型与公共 API。

## 要点

```c
/* file.c */
typedef int file_local_t;        /* 文件作用域 */

static typedef int bad;          /* 非法：typedef 不能与 static 这样写 */
static int x;
typedef int static_alias_t;      /* 类型名全局可见，不 static */

void foo(void) {
    typedef double block_t;      /* 仅 foo 内 */
    block_t v = 1.0;
}
```

**头文件**：

```c
/* public.h */
#ifndef PUBLIC_H
#define PUBLIC_H
typedef struct device device_t;
#endif
```

| 场景 | 建议 |
|------|------|
| 驱动内部状态 | `.c` 文件内 typedef，不写入公共头 |
| 跨模块 API | 单一权威头文件定义 |
| 内核子系统 | `include/linux/types.h` 集中 |

- **重复 typedef**：C11 允许相同别名同一类型重复；不同底层类型重复定义报错
- **命名空间**：C 无 namespace；前缀 `drv_`、`rte_` 防冲突
- **C++ 链接**：`extern "C"` 块内 typedef 与 C 共享
