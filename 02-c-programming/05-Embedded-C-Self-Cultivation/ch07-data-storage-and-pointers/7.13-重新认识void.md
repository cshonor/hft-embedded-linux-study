# 7.13 重新认识void

## 本节讲什么

`void` 表示**无类型**：不能声明 `void` 变量；`void *` 是**通用指针**，可保存任意对象地址，解引用前须转为具体类型。`void` 函数无返回值。`restrict`（C99）修饰指针，承诺无别名以助优化。三者常与 `memcpy`、回调、MMIO 封装一起出现。

## 要点

```c
void noop(void) { }              /* 无返回值 */

void *p = malloc(64);            /* 任意类型地址 */
int *ip = p;                     /* 转回，须对齐且知对象类型 */
free(p);

/* 不声明 void x; — 非法 */
```

| 话题 | 规则 |
|------|------|
| `void *` 算术 | 非法，须先 cast |
| `void *` 比较 | 可与任意对象指针比较 |
| `memcpy` | `void *` 目的，`const void *` 源 |

**restrict**（优化契约）：

```c
void sum(size_t n, const int *restrict src, int *restrict dst) {
    for (size_t i = 0; i < n; i++)
        dst[i] += src[i];   /* 编译器假设 src/dst 不重叠 */
}
/* 内核热路径；重叠须用 memmove */
```

**volatile**（回顾 **demo05**）：

```c
volatile uint32_t *reg = (volatile uint32_t *)base;
*reg = val;   /* 每次真实访问设备 */
```

```c
/* MMIO 宏模式 demo07 */
#define REG32(addr)  (*(volatile uint32_t *)(addr))
#define BIT(n)       (1u << (n))
REG32(GPIO_BASE + 0x04) |= BIT(0);
```

- **`void **` 少见**：通用容器用 `void *` + 类型标签
- **函数指针与 `void *`**：POSIX `dlsym` 返回后 cast，C 标准严格说不保证
- **C++**：`void *` 不能隐式转 `T *`；C 可以
