# 7.1.4 数据类型转换

## 本节讲什么

C 在赋值、运算、传参时大量发生**隐式转换**；强制转换 `(type)` 可压制警告但掩盖逻辑错误。指针与整数、不同层级指针、有符号/无符号之间的转换是内核与驱动调试的高频坑。本节归纳转换规则与嵌入式中的安全写法。

## 要点

**整数提升**：`char`/`short` 参与运算先提升为 `int`。

** usual arithmetic conversions **：较窄操作数向较宽对齐；有符号与无符号混合时，有符号常转为无符号。

```c
int i = -1;
unsigned u = 1;
if (i < u) { }   /* false：i 转为很大的 unsigned */

uintptr_t addr = (uintptr_t)ptr;   /* 指针 ↔ 整数，用于打印/物理地址 */
void *p = (void *)(uintptr_t)0x40001000;  /* MMIO 映射基址，demo07 */
```

**指针转换陷阱**：

| 转换 | 风险 |
|------|------|
| `int *` → `float *` | 严格别名违反，UB |
| `void *` → 任意 `T *` | 需保证对齐与对象有效 |
| 函数指针 ↔ 数据指针 | C 标准不要求可互转 |
| 截断 `uintptr_t` → `uint32_t` | 64 位系统丢高位 |

```c
/* 协议解析：避免未对齐 + 别名 */
uint32_t load_be32(const void *src) {
    uint32_t v;
    memcpy(&v, src, 4);   /* 再 cpu_to_be32 / ntohl */
    return ntohl(v);
}
```

```bash
gcc -Wconversion -Wsign-conversion -Wcast-align file.c
```

- **严格别名规则**：通过不兼容类型指针读同一内存 UB；`char *`/`memcpy`/`union`（谨慎）可穿透
- **restrict**（C99）提示无别名以优化 → **7.13**
- **野指针** 常源于错误 cast 或已释放对象 → **demo06**
