# 7.1.3 数据溢出

## 本节讲什么

运算结果超出类型可表示范围时发生**溢出**。有符号整数溢出在 C 中是**未定义行为（UB）**；无符号溢出按模 2^宽度回绕。内核计时、环形缓冲区索引、长度累加若忽视溢出，会导致安全漏洞与难以复现的故障。

## 要点

```c
uint8_t  u = 255; u++;           /* 0，定义明确 */
int8_t   s = 127; s++;           /* 有符号溢出 → UB */
unsigned a = 0, b = 1;
if (a - b < 10) { }              /* 真！无符号下溢为大数 */
```

| 类型 | 溢出 |
|------|------|
| `signed` | UB，编译器可假设不发生并优化掉「保护代码」 |
| `unsigned` | 模运算，常用于环形下标 |
| 浮点 | 无穷/NaN，一般不用于 IRQ 路径 |

```c
/* 安全累加长度（内核常见模式） */
if (len > SIZE_MAX - need)
    return -ENOMEM;
total += need;

/* 环形缓冲区 */
#define RING_SIZE 256
unsigned head, tail;
unsigned next = (head + 1) % RING_SIZE;  /* 或 & (RING_SIZE-1) 当 SIZE 为 2 幂 */
```

```bash
gcc -ftrapv -O0 overflow_test.c   # 部分平台运行时捕获有符号溢出（调试）
gcc -fsanitize=undefined ./a.out  # UBSan
```

- **宽度提升**：运算前先 cast 到更宽无符号再判断
- **常量折叠** 可在编译期发现部分溢出 → **7.7.3**
- **时间戳回绕**：`jiffies`、毫秒计数用无符号减法技巧 `(now - start) < timeout`
