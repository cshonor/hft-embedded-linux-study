# 7.1.1 大端模式与小端模式

## 本节讲什么

多字节整数在内存中的**字节排列顺序**称为字节序（Endianness）。网络协议、引导加载、跨 CPU 通信必须统一约定；本机是小端还是大端决定你如何解析 `uint32_t` 与如何写 MMIO 寄存器。Linux 提供 `cpu_to_be32` / `le32_to_cpu` 等宏，DPDK 有 `rte_be_to_cpu_16` 等。

## 要点

**定义**：低地址存**最高有效字节** → 大端（Big-Endian）；低地址存**最低有效字节** → 小端（Little-Endian）。

```
uint32_t 0x12345678 在小端内存（低地址在上）:
  78 56 34 12

同值在大端:
  12 34 56 78
```

```c
#include <stdint.h>
#include <stdio.h>

union {
    uint32_t u;
    uint8_t  b[4];
} x = { .u = 0x12345678 };

int is_little(void) {
    return x.b[0] == 0x78;
}
```

| 场景 | 惯例 |
|------|------|
| TCP/IP、以太网帧 | **网络字节序 = 大端** |
| x86 / AArch64 常见 | **小端** |
| 设备寄存器手册 | 按芯片文档，常为大端或固定字段 |

```c
#include <arpa/inet.h>   /* htons, ntohl — 用户态 */
/* 内核: #include <linux/byteorder/generic.h> */
uint32_t net = htonl(host_val);
```

```bash
# 查看本机字节序
printf '\x01\x00' | hexdump -C   # 配合 uint16_t 探测
lscpu | grep -i endian
```

- **位域 + 字节序**：协议解析用定宽类型 + 显式移位，勿依赖位域布局
- **未对齐访问**：ARM 上可能 fault；用 `memcpy` 到对齐变量再转换
