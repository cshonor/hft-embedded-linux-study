# 7.1.2 有符号数和无符号数

## 本节讲什么

同一组比特可用**补码有符号**或**无符号**解释，结果截然不同。嵌入式与内核中混用 `int`/`unsigned` 是 bug 温床：比较、移位、数组下标、位掩码都受影响。本节建立「何时用无符号、何时避免有符号比较」的判断标准。

## 要点

**补码**（典型 `int`）：最高位为符号位；`-1` 的 32 位表示为 `0xFFFFFFFF`。

```c
int8_t  s = -1;
uint8_t u = 255;
/* (int)s == (int)u 可能为真，但 s 与 u 直接比较语义不同 */
```

| 陷阱 | 示例 | 后果 |
|------|------|------|
| 有符号 vs 无符号比较 | `if (i < strlen(s))` | `i` 为负时转大正数 |
| 右移 | `-1 >> 1` | 算术右移仍 -1（实现定义历史问题，C 现多为算术） |
| 下标 | `size_t len; for (int i=0; i<len; i++)` | `i` 永不为负但比较仍可能异常 |
| 位掩码 | `1 << 31` on `int` | 有符号溢出 UB |

```c
/* 内核风格：长度、计数、地址偏移用 size_t / unsigned long */
size_t count;
for (size_t i = 0; i < count; i++) { /* ... */ }

/* 位域掩码 */
#define FLAG_MASK 0xFFu
uint32_t reg = val & FLAG_MASK;
```

- **字符** `char` 是否 signed 由实现定义 → 协议字节用 `uint8_t` / `unsigned char`
- **枚举** 底层常为 `int`，与无符号混用时注意 **7.6**
- **GPIO/寄存器位** 一律无符号 + 显式掩码
