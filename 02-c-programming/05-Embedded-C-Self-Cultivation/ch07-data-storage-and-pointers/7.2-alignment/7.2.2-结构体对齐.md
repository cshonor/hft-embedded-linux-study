# 7.2.2 结构体对齐

## 本节讲什么

结构体成员顺序决定**填充（padding）**位置与总 `sizeof`。手算布局是阅读内核 `struct sk_buff`、`struct device` 与网络协议头的基本功。本节给出对齐规则、手算步骤及与 `offsetof` 的对应关系。

## 要点

**规则**：

1. 每个成员偏移 ≡ 0 (mod min(成员大小, 默认对齐))
2. 结构体总大小 ≡ 0 (mod 最大成员对齐)
3. `packed` 忽略 1–2，成员紧密排列

```c
struct normal {
    uint8_t  type;   /* 0 */
    /* pad 3 */
    uint32_t value;  /* 4 */
    uint16_t flags;  /* 8 */
    /* pad 2 → sizeof = 12 */
};
```

手算 **demo03** 布局：

| 成员 | normal offset | packed offset |
|------|---------------|---------------|
| type | 0 | 0 |
| value | 4 | 1 |
| flags | 8 | 5 |
| sizeof | 12 | 7 |

```c
#include <stddef.h>
printf("value @ %zu\n", offsetof(struct normal, value));

/* 调整顺序可减少 padding */
struct tuned {
    uint32_t value;
    uint16_t flags;
    uint8_t  type;
    /* sizeof 可能仍为 8（尾部 padding） */
};
```

```bash
gcc demo03_struct_offset.c -o demo03 && ./demo03
```

- **数组成员**：按元素类型对齐，占 `N * sizeof(elem)`
- **嵌套结构体**：子结构体对齐系数参与父结构体
- **位域**：布局实现相关，跨平台协议慎用
