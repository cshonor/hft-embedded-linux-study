# 7.2.3 联合体对齐

## 本节讲什么

**联合体（union）** 所有成员共享同一起始地址，`sizeof` 等于**最大成员的大小**（含对齐填充）。常用于协议复用、浮点位域视图、寄存器别名访问。对齐规则与结构体相同，但无成员间顺序 padding，只有尾部对齐填充。

## 要点

```c
union u {
    uint32_t i;
    uint8_t  b[4];
    float    f;
};  /* sizeof = 4（典型），所有成员 offset 0 */
```

| 对比 | struct | union |
|------|--------|-------|
| 存储 | 成员依次排列 | 重叠 |
| sizeof | 各成员之和 + padding | max(成员) |
| 初始化 | 各成员独立 | 写一成员覆盖其余 |

```c
/* 检测字节序（另一种写法） */
union { uint16_t u; uint8_t c[2]; } endian = { .u = 0x0102 };
/* c[0]==1 → 大端 */

/* 寄存器视图（需 volatile，demo05/demo07） */
union reg {
    uint32_t raw;
    struct { uint32_t en:1; uint32_t mode:3; } bits;
};
```

```c
volatile union reg *mmio = (volatile union reg *)0x40001000;
mmio->raw = 0x1;           /* 整字写 */
mmio->bits.en = 1;         /* 位域：实现定义，硬件常用宏代替 */
```

- **类型双关（type punning）**：通过 union 读另一类型，C99+ 合法但有严格别名注意；内核常用 `memcpy`
- **对齐**：union 对齐系数 = 最大成员对齐要求
- **未初始化读**：读最近写入的成员类型之外的语义依赖实现
