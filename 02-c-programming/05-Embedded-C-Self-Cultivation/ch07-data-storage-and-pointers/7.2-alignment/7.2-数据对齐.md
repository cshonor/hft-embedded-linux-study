# 7.2 数据对齐

## 本节讲什么

CPU 访问内存时通常要求数据位于**自然对齐**地址（`N` 字节类型从 `N` 的倍数地址开始）。对齐影响结构体 `sizeof`、填充字节、DMA 描述符布局与原子操作可行性。ch06 的 `aligned`/`packed` 属性建立在本节规则之上。

## 要点

```
对齐要求（典型 x86_64 / AArch64）:
  char   → 1
  short  → 2
  int    → 4
  double → 8
  指针   → 8（64 位）
```

```c
struct example {
    char c;    /* offset 0 */
    /* 3 bytes padding */
    int  i;    /* offset 4 */
};  /* sizeof 常为 8，非 5 */
```

| 概念 | 含义 |
|------|------|
| 对齐系数 | 成员偏移必须是其大小的倍数 |
| 结构体对齐 | 整体 `sizeof` 为最大成员对齐的倍数 |
| `#pragma pack` / `packed` | 取消或压缩填充 → ch06 **6.7.4** |

```bash
# 查看布局
gcc -fdump-record-layouts -c struct.c 2>&1 | less
pahole -C example vmlinux   # 内核镜像
```

- **未对齐访问**：x86 可容忍但慢；部分 ARM/MIPS 直接 **alignment fault**
- **`alignas` / `__attribute__((aligned(n)))`**：强制更高对齐，利于 SIMD / 缓存行
- **堆分配**：`malloc` 返回最大基本对齐；更大用 `aligned_alloc` / `posix_memalign`
