# 7.3 数据的可移植性

## 本节讲什么

同一份 C 源码在不同 CPU、编译器、ABI 下，类型宽度、`int`/`long`/指针关系、对齐与字节序可能不同。**可移植**代码不依赖实现细节，而用标准定宽类型与显式转换。内核通过 `arch/`、`include/linux/types.h` 抽象平台差异；DPDK 通过 `rte_*` 类型与编译目标固定环境。

## 要点

| 不可移植假设 | 可移植替代 |
|--------------|------------|
| `int` 32 位 | `int32_t` |
| `long` 与指针同宽 | `intptr_t` / `uintptr_t` |
| `enum` 1 字节 | 显式 `uint8_t` 字段 |
| 结构体无 padding | `packed` + 文档化 或 序列化函数 |

```c
#include <stdint.h>
#include <stddef.h>

/* POSIX / Linux 内核风格 */
typedef uint8_t  u8;
typedef uint32_t u32;
typedef uintptr_t dma_addr_t;  /* 平台相关，头文件封装 */
```

**ABI 影响**：

- 结构体成员对齐、调用约定、参数传递
- `bool`/`wchar_t` 宽度
- 位域顺序与填充

```bash
# 32 位 vs 64 位对比 sizeof
gcc -m32 -dM -E - < /dev/null | grep SIZEOF
gcc -m64 -dM -E - < /dev/null | grep SIZEOF
```

- **`limits.h` / `stdint.h`**：编译期常量 `UINT32_MAX`、`SIZE_MAX`
- **序列化**：网络/磁盘格式用**大端 + 定宽**，与主机布局脱钩
- **条件编译**：`#if __SIZEOF_POINTER__ == 8` 仅作最后手段
