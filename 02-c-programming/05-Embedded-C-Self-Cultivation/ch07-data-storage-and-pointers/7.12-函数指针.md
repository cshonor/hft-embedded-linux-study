# 7.12 函数指针

## 本节讲什么

**函数指针**保存函数入口地址，实现回调、虚表、**跳转表（jump table）** 与运行时策略切换。内核 `file_operations`、IRQ handler、DPDK burst API 均依赖函数指针。用 typedef 声明签名是工程惯例；**demo04** 用 `enum op_t` 索引分发表。

## 要点

```c
typedef int (*calc_fn)(int, int);

static int add(int a, int b) { return a + b; }
static int sub(int a, int b) { return a - b; }

calc_fn fp = add;
fp(3, 4);                    /* 7 */
calc_fn table[] = { add, sub };
table[0](10, 3);
```

**demo04 jump table**：

```c
typedef enum { OP_ADD, OP_SUB, OP_MUL, OP_DIV, OP_COUNT } op_t;

static calc_fn jump_table[OP_COUNT] = {
    [OP_ADD] = add,
    [OP_SUB] = sub,
    [OP_MUL] = mul,
    [OP_DIV] = div_safe,
};

static int dispatch(op_t op, int x, int y) {
    if (op >= OP_COUNT || !jump_table[op])
        return 0;
    return jump_table[op](x, y);
}
```

| 模式 | 用途 |
|------|------|
| 回调 | `qsort` 比较函数、`request_irq` |
| ops 表 | `struct file_operations` |
| 状态机 | `handlers[state]()` |
| 替代 switch | 密集 op 码分派 |

```c
/* 内核 file_operations 片段（示意） */
struct file_operations {
    ssize_t (*read)(struct file *, char __user *, size_t, loff_t *);
    ssize_t (*write)(struct file *, const char __user *, size_t, loff_t *);
};
```

- **函数名 decay**：`add` 与 `&add` 同义
- **NULL 槽**：分发表可留空，调用前检查
- **与数据指针互转**：标准不要求，勿玩弄
