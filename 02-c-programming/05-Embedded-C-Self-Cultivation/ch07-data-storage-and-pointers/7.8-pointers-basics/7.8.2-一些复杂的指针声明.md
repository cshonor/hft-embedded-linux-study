# 7.8.2 一些复杂的指针声明

## 本节讲什么

C 声明从**标识符**出发，**右左法则**（螺旋法则）向内向外解析：先与右边结合，再与左边。掌握此法可读懂 `int *(*fp)(int *)`、`char *argv[]` 及内核函数指针表，避免 typedef 误用。

## 要点

**右左法则步骤**：

1. 找标识符
2. 右侧优先绑定 `[]` `()`，左侧绑定 `*`
3. 用括号突破默认顺序

```c
int *p;              /* p 是指针，指向 int */
int *f(void);        /* f 是函数，返回 int * */
int (*fp)(void);     /* fp 是指针，指向函数 void→int */
int *(*arr[10]);     /* arr 是数组，元素为 int * 指针 */
int (*parr)[10];     /* parr 是指针，指向含 10 int 的数组 */
char *(*cvp)(int);   /* cvp 指向 char*(int) 的函数 */
```

| 声明 | 读法 |
|------|------|
| `char *str[10]` | 10 个 `char *`（指针数组） |
| `char (*str)[10]` | 指向 10 字符数组的指针 |
| `int (*sig)(int)` | 函数指针 |
| `int *const cp` | 常量指针 |
| `const int *pc` | 指向常量的指针 |

```c
/* typedef 拆解 */
typedef int (*calc_fn)(int, int);
calc_fn table[4];    /* 函数指针数组 → demo04 */

typedef char *string_t;
string_t *lines;     /* char ** */
```

```bash
# 在线练习：用 cdecl 解析
# echo 'int *(*fp)(int *)' | cdecl
```

- **`const` 位置**：`const int *` vs `int *const` → **7.7**
- **函数声明不加括号**即函数，加括号才是函数指针
- **返回指针的函数** vs **指向函数的指针** 最易混淆
