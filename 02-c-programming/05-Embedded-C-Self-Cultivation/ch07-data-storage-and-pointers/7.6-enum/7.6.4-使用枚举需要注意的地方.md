# 7.6.4 使用枚举需要注意的地方

## 本节讲什么

枚举提升可读性，但不能替代**输入校验**与**完整性检查**。从用户态、寄存器、网络包读入的值可能落在枚举定义之外；`switch` 缺少 `default` 会导致静默错误。本节汇总驱动开发中的 enum 防守式用法。

## 要点

| 陷阱 | 对策 |
|------|------|
| 未定义枚举值 | `default:` 分支日志 + 错误码 |
| 用作位掩码 | 勿用连续 enum，改用 `#define FL_FOO BIT(0)` |
| sizeof 假设 | 协议字段用 `uint8_t` 存，不假设 enum 1 字节 |
| 跨模块数值依赖 | 公共头文件单一来源 |

```c
int handle_op(op_t op) {
    if (op < 0 || op >= OP_COUNT)
        return -EINVAL;
    return jump_table[op](a, b);
}

switch (st) {
case DRV_IDLE:
case DRV_ACTIVE:
    break;
default:
    pr_err("bad state %d\n", st);
    return -EIO;
}
```

```c
/* 不要 */
enum flags { F_READ = 1, F_WRITE = 2, F_RDWR = 3 }; /* 非位标志 */

/* 要 */
#define F_READ  BIT(0)
#define F_WRITE BIT(1)
```

- **编译器 `-Wswitch`**：遗漏 case 告警（依赖 enum 范围分析）
- **JSON/配置解析**：字符串 → enum 映射表，非法串拒绝
- **与函数指针表**：下标必须 `< OP_COUNT` → **demo04**
