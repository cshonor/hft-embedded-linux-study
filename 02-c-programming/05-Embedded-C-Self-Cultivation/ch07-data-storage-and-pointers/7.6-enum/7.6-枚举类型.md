# 7.6 枚举类型

## 本节讲什么

`enum` 为一组命名整数常量提供类型与作用域，比 `#define` 更利于调试器与编译器检查。驱动状态机、错误码、操作码（**demo04** `op_t`）广泛使用。本节概述枚举语法、底层表示及与内核 `enum` 宏扩展的关系。

## 要点

```c
enum color { RED, GREEN, BLUE };   /* RED=0 起递增 */
enum { MAX_BUF = 256 };            /* 匿名枚举当常量 */

enum gpio_dir {
    GPIO_IN  = 0,
    GPIO_OUT = 1,
};
```

| 对比 `#define` | enum 优势 |
|----------------|-----------|
| 调试显示符号名 | 符号表保留（未 strip） |
| 类型检查较弱 | 与 `int` 兼容但有 distinct 检查（编译器扩展） |
| 无作用域 | 遵循块/文件作用域 |

```c
/* 指定底层类型（C23 / GCC 扩展） */
enum op_t : uint8_t {
    OP_ADD, OP_SUB, OP_MUL, OP_DIV, OP_COUNT
};
```

- **底层类型**：标准 C 前为 `int`；嵌入式协议字段常显式 `uint8_t` 成员存枚举值
- **计数哨兵**：`OP_COUNT` 作数组大小 → **demo04** jump table
- **内核**：大量 `enum foo` + `FOO_MAX` 模式
