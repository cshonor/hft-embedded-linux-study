# 7.6.3 Linux内核中的枚举类型

## 本节讲什么

内核子系统用枚举表达**电源管理事件**、**总线类型**、**网络协议族**等，常与 `switch`、tracepoint、sysfs 字符串转换配合。部分头文件用宏生成枚举与字符串表，保证调试输出与 UAPI 一致。读 `include/linux/` 时需识别「纯文档 enum」与「ABI 稳定 enum」。

## 要点

```c
/* 示意：设备总线类型 */
enum bus_type {
    BUS_TYPE_PLATFORM,
    BUS_TYPE_PCI,
    BUS_TYPE_AMBA,
    /* ... */
};

/* PM 消息 */
typedef enum {
    PM_EVENT_SUSPEND,
    PM_EVENT_RESUME,
    PM_EVENT_FREEZE,
} pm_event_t;
```

**常见模式**：

```c
#define ENUM_STRING(x) [x] = #x
static const char *const names[] = {
    ENUM_STRING(PM_EVENT_SUSPEND),
    ENUM_STRING(PM_EVENT_RESUME),
};
```

| 场景 | 内核实践 |
|------|----------|
| 驱动 probe | `enum probe_state` 内部状态 |
| ioctl | 用户可见值固定，变更需兼容 |
| trace | `TRACE_DEFINE_ENUM()` |

```bash
grep -r 'typedef enum' include/linux/ | head -5
```

- **UAPI**：`include/uapi/linux/` 中枚举值不可随意改号
- **设备树 / ACPI**：字符串与 enum 双向映射函数
- **错误码**：常为负 `int` 宏，非 enum（`EINVAL` 等）
