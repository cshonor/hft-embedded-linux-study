# 3.7.1 重新认识编译器

## 本节讲什么

`gcc` 对 `.S` 文件：**预处理器 + 汇编器 + 链接器** 一体调度；理解谁生成指令、谁展开伪指令，才能正确读 `.s` 与 `-S` 输出。

## 要点

```bash
# 仅汇编
arm-linux-gnueabihf-gcc -c foo.S -o foo.o
# C → 汇编清单
arm-linux-gnueabihf-gcc -S -O0 foo.c -o foo.s
# 反汇编含源码
arm-linux-gnueabihf-objdump -dS foo.o
```

- **编译器**生成指令调度与寻址；**汇编器**解析伪操作与符号
- `-O2` 可能合并/消除变量，对照学习建议 `-O0 -g`
- 内核 `CONFIG_CC_OPTIMIZE_FOR_SIZE` 影响生成 Thumb/ARM 混合
