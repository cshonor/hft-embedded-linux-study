# 3.7.7 汇编代码分析实战

## 本节讲什么

用 **`objdump -dS`**、**`gdb disassemble`** 对照 C 与 ARM 汇编，验证 AAPCS、寻址与优化——读内核/DPDK 热点的必备技能。

## 要点

```bash
cd demo
make clean && make
# 本地 x86 时 objdump 显示对应 ISA；ARM 交叉：
# arm-linux-gnueabihf-gcc -c -O0 -g add_asm.S demo03_main.c
# arm-linux-gnueabihf-objdump -dS demo03.elf

objdump -d add_asm.o
objdump -d -M reg-names-raw demo03   # 若已链接
gdb -batch -ex "disassemble add_asm" demo03
```

### 读 `add_asm` 清单

```arm
add_asm:
    add r0, r0, r1    @ r0 为返回值
    bx  lr
```

### 读 C `main` 调用链（`-O0 -g`）

- `mov` 装载参数到 **r0, r1**
- `bl add_asm` → **lr** 保存返回点
- `str` 保存返回值到局部变量

### 练习检查单

1. 指出 **栈指针** 在函数入口是否调整  
2. 标出 **callee-saved** 是否被修改  
3. `ldr [fp, #-N]` 对应哪个局部变量或参数  

- 内核 `ftrace`/`perf` 回溯依赖同一套帧分析
