# 3.9 异常与中断汇编

## 本节讲什么

异常/中断发生时 CPU **硬件自动切换模式** 并跳向量；汇编负责 **保存上下文、调用 C handler、恢复返回**——内核 `entry-armv.S`、GIC 中断与 **ch10** 多任务的前置。

## 要点

### ARM32 异常类型

| 向量 | 触发 |
|------|------|
| Reset | 上电 |
| Undefined | 非法指令 |
| SWI/SVC | 系统调用 |
| Prefetch Abort | 取指_fault |
| Data Abort | 数据_fault |
| IRQ / FIQ | 外设中断 |

### 上下文保存（典型 IRQ 入口）

```arm
irq_handler:
    sub lr, lr, #4              @ 修正返回地址
    stmfd sp!, {r0-r12, lr}     @ 保存通用寄存器
    mrs r0, spsr                @ 保存程序状态
    stmfd sp!, {r0}
    bl  do_irq_handler          @ C 处理函数
    ldmfd sp!, {r0}
    msr spsr_cf, r0
    ldmfd sp!, {r0-r12, pc}^    @ ^ 恢复 CPSR 并返回
```

- **SPSR** 保存进入异常前的 CPSR；返回须恢复
- **LR_irq** 为异常专属链接寄存器（banked）
- 内核还需保存 **r13_svc** 等，避免重入错乱

### 与 C 的边界

- 汇编：最少保存、清中断源、栈对齐（AAPCS 8 字节）
- C `do_irq_handler()`：设备逻辑；`request_irq()` 注册
- DPDK 用户态常用 **UIO/vfio** 轮询，不走内核 IRQ 路径，但 SoC 启动仍靠同类向量

### AArch64 简述

- 异常进 EL1/EL2；`stp x0, x1, [sp, #-16]!` 成对保存
- `eret` 从 ELR/SPSR 返回（**3.8**）
