# 3.1 ARM体系结构

## 本节讲什么

ARM 是 **RISC load-store** 架构：运算只在寄存器间进行，访存须通过 `LDR`/`STR`。本章以 **ARM32（AArch32）** 为主（`arm-linux-gnueabihf` 工具链），并对比 **AArch64**（**3.8**）。理解 **7 种处理器模式**、**CPSR/SPSR** 与 **R0–R15**，是读内核启动代码、DPDK 绑核汇编和异常现场的基础。

## 要点

### RISC 与 load-store

- **CISC**：内存操作数可直接参与运算（如 x86 `add [mem], eax`）
- **ARM**：`add r0, r1, r2` 只在寄存器间运算；读写内存必须 `ldr`/`str`
- 好处：指令规整、流水线友好；代价：更多显式 load/store

### ARM32 vs AArch64

| 维度 | ARM32 (AArch32) | AArch64 |
|------|-----------------|---------|
| 通用寄存器 | R0–R15，32 位 | X0–X30，64 位 |
| 指令集 | ARM / Thumb-2 | 单一 A64，**无 Thumb** |
| 模式 | 7 种处理器模式 | EL0–EL3 异常级别 |
| Linux 用户态 | `arm-linux-gnueabihf` | `aarch64-linux-gnu` |

### 7 种处理器模式（ARM32）

| 模式 | 缩写 | 典型用途 |
|------|------|----------|
| User | usr | 应用程序、DPDK 用户态轮询 |
| FIQ | fiq | 快速中断（旧平台） |
| IRQ | irq | 普通外设中断 |
| Supervisor | svc | 系统调用、Bootloader |
| Abort | abt | 数据/指令预取异常 |
| Undefined | und | 未定义指令 |
| System | sys | 特权，与用户共享寄存器 |

模式切换时部分寄存器 **banked**（如 FIQ 的 R8–R14）；User 模式无法直接改 CPSR。

### CPSR / SPSR

- **CPSR**（Current Program Status Register）：N Z C V 标志位 + **T**（Thumb）+ **模式位**
- **SPSR**（Saved PSR）：异常/中断入口时硬件保存旧 CPSR，返回用 `subs pc, lr, #4` 恢复
- 内核 `entry-armv.S`、中断向量表都围绕 CPSR/SPSR 现场切换

### R0–R15 与别名

| 寄存器 | 别名 | 作用 |
|--------|------|------|
| R0–R12 | — | 通用；**AAPCS**：R0–R3 传参/返回值，R4–R11 被调者保存 |
| R13 | **SP** | 栈指针；每种模式可有独立 SP |
| R14 | **LR** | 链接寄存器；`BL` 保存返回地址 |
| R15 | **PC** | 程序计数器；ARM 状态 PC 低 2 位为 0 |

```arm
mov r0, #3        @ 第一个参数
mov r1, #5        @ 第二个参数
bl  add_asm        @ LR = 返回地址，结果在 r0
```

### 与 x86 的差异（嵌入式/DPDK 视角）

- 条件执行：ARM32 几乎所有指令可带条件码（**3.2.6**），减少分支
- 对齐：未对齐访问可能触发 Abort；DPDK `rte_memcpy` 等会考虑对齐
- 端序：多数 ARM SoC 小端；网络字节序仍须 `htonl`
