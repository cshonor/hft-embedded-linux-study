# 3.8 AArch64拓展

## 本节讲什么

**AArch64**（ARMv8+ 64 位执行状态）是服务器、树莓派 3/4、Ampere 等 Linux/DPDK 主流平台。与 ARM32 相比：固定 64 位指令、**31 个 64 位通用寄存器**、**无 Thumb**、异常级别 EL0–EL3。

## 要点

### 寄存器 X0–X30

| 寄存器 | 别名 | 用途 |
|--------|------|------|
| X0–X7 | — | 参数 / 返回值（可传 8 个） |
| X8 | — | 间接结果 / syscall 号 |
| X9–X15 | — | 临时（调用者保存） |
| X16–X17 | IP0/IP1 | 跳板 / PLT |
| X18 | — | 平台保留（内核 per-CPU） |
| X19–X28 | — | 被调者保存 |
| X29 | **FP** | 帧指针 |
| X30 | **LR** | 链接寄存器 |
| — | **SP** | 栈指针（独立编码） |
| — | **PC** | 独立，非通用寄存器 |

- 访问低 32 位写 **Wn**（零扩展）；如 `W0` 为 `X0` 低半部
- **无 CPSR 条件执行全场**：条件分支用显式 `b.eq` 等

### 与 ARM32 对照

```arm
@ ARM32
mov r0, #1
bl  func
bx  lr

// AArch64
mov x0, #1
bl  func
ret                          @ 等价 bx lr
```

```arm
ldr x0, [x1, #8]             @ 64 位加载
add x2, x0, x1, lsl #3
```

### 无 Thumb

- AArch64 **仅 A64 指令集**，无 16/32 位 Thumb 混编；简化取指与工具链
- 32 位用户态程序在 AArch64 CPU 上通过 **AArch32 兼容**（可选，独立工具链 `arm-linux-gnueabihf`）

### 工具链与内核

```bash
aarch64-linux-gnu-gcc -c foo.S
aarch64-linux-gnu-objdump -d foo.o
```

- Linux 内核 `arch/arm64/`、`entry.S` 用 ELx 切换
- DPDK 在 ARM64 上 mbuf/ring 热点多为 NEON + 64 位 load pair
