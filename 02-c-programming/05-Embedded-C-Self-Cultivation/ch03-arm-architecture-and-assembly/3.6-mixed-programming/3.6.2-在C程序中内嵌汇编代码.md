# 3.6.2 在C程序中内嵌汇编代码

## 本节讲什么

GCC **`asm` / `__asm__`** 把 ARM 指令嵌入 C；**`volatile`** 禁止优化删除/重排；**约束** 描述操作数与寄存器对应——内核 `read_cpuid`、`wfe`、`dmb` 的实现方式。

## 要点

```c
int add_inline(int a, int b)
{
    int sum;
    asm volatile (
        "add %0, %1, %2"
        : "=r" (sum)           /* 输出 */
        : "r" (a), "r" (b)     /* 输入 */
        : "cc"                 /* 破坏标志位 */
    );
    return sum;
}
```

| 要素 | 含义 |
|------|------|
| `volatile` | 有副作用，勿删勿挪 |
| `"=r"` | 写寄存器输出 |
| `"r"` | 读寄存器输入 |
| `"cc"` / `"memory"` |  clobber 标志 / 内存 |

```c
/* 内核风格：内存屏障 */
#define barrier() asm volatile("" ::: "memory")
```

- 多语句用 `\n\t` 连接；勿随意写 clobber，否则优化错误
- DPDK `rte_rmb()` 等平台屏障在 ARM 上常为 `dmb ish` 内联汇编
