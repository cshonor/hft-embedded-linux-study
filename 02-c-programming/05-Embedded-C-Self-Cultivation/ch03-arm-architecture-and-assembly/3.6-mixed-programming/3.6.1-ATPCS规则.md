# 3.6.1 ATPCS规则

## 本节讲什么

**ATPCS**（ARM-Thumb Procedure Call Standard），GNU/Linux 上即 **AAPCS**：规定函数调用时 **寄存器分工、栈方向、栈帧布局**——混合编程的核心契约。

## 要点

### 寄存器约定

| 寄存器 | 角色 |
|--------|------|
| R0–R3 | 参数 / 返回值（≤4 字）；多余参数入栈 |
| R4–R11 | 被调者保存（callee-saved） |
| R12 (IP) | 临时 / 跳板 |
| R13 (SP) | 栈指针，**满递减**（向下生长） |
| R14 (LR) | 返回地址 |
| R15 (PC) | 程序计数器 |

### 栈帧示例：`int add(int a, int b)`（C 编译）

```arm
add:
    push    {fp, lr}        @ 保存帧指针与返回地址
    add     fp, sp, #4      @ fp 指向旧 sp
    sub     sp, sp, #8      @ 局部空间（若需要）
    str     r0, [fp, #-8]   @ 存参数 a（编译器可选）
    str     r1, [fp, #-12]  @ 存参数 b
    ldr     r2, [fp, #-8]
    ldr     r3, [fp, #-12]
    add     r0, r2, r3        @ 返回值 → r0
    sub     sp, fp, #4
    pop     {fp, pc}        @ 恢复并返回
```

### 手写 `add_asm` 极简版（demo）

```arm
add_asm:
    add r0, r0, r1    @ 参数已在 r0,r1；结果 r0
    bx  lr            @ 调用者保存 r0-r3，我们不碰 r4-r11
```

- **leaf 函数**可不建立帧指针，不碰 callee-saved 即可 `bx lr`
- 内核 `__switch_to` 等必须完整保存 callee-saved + SP
