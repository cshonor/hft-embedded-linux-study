# 3.6 C语言和汇编语言混合编程

## 本节讲什么

同一工程内 **C 调汇编**、**汇编调 C**、**C 内嵌汇编** 三种路径；须共同遵守 **AAPCS/ATPCS** 调用约定，否则参数、返回值、栈帧错乱——内核 `asm/` 与驱动里极常见。

## 要点

| 方式 | 场景 | 小节 |
|------|------|------|
| C → 汇编函数 | 热点、平台原语 | **3.6.1** + demo |
| C 内嵌汇编 | 读协处理器、屏障 | **3.6.2** |
| 汇编 → C | 启动代码调 `main` | **3.6.3** |

```c
// demo03_main.c
extern int add_asm(int a, int b);
sum = add_asm(3, 5);
```

```arm
@ add_asm.S — AAPCS：r0,r1 入参，r0 返回
add r0, r0, r1
bx  lr
```

- 交叉编译：`CROSS_COMPILE=arm-linux-gnueabihf- make`（**demo/Makefile**）
- DPDK `rte_pause()`、内核 `barrier()` 等为内联汇编封装
