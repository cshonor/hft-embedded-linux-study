# 3.3 ARM寻址方式

## 本节讲什么

ARM load/store 的地址表达式决定 `[...]` 如何计算。七种寻址覆盖从纯寄存器到 PC 相对访问，是读懂 `ldr`/`str` 反汇编和写内联汇编的前提。

## 要点

| 方式 | 形式示例 | 小节 |
|------|----------|------|
| 寄存器 | `mov r0, r1` | **3.3.1** |
| 立即数 | `mov r0, #1` | **3.3.2** |
| 寄存器偏移 | `[r1, r2]` `[r1, r2, lsl #2]` | **3.3.3** |
| 寄存器间接 | `[r1]` | **3.3.4** |
| 基址 | `[r1, #4]` `[r1], #4` | **3.3.5** |
| 多寄存器 | `{r0-r3}` | **3.3.6** |
| 相对寻址 | `[pc, #offset]` | **3.3.7** |

```arm
ldr r0, [r1, r2, lsl #2]   @ 数组下标：base + index*4
ldr r1, =my_var            @ 伪指令，见 3.4
```

- 偏移寄存器可选移位，适合结构体字段、环形缓冲区索引
- 内核链表遍历：`ldr r0, [r0]` 取 `next` 指针
