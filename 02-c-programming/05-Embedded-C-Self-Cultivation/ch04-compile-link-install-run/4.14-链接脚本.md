# 4.14 链接脚本

## 本节讲什么

**链接脚本（Linker Script）** 用 `MEMORY` 与 `SECTIONS` 告诉 `ld`：**各段放入哪块物理存储、链接顺序、VMA/LMA**。裸机/嵌入式把 **`.text/.rodata` 放 Flash**、**`.data/.bss` 放 RAM** 全靠脚本；Linux 内核 `vmlinux.lds`、U-Boot `u-boot.lds` 同理（**4.11**、**4.12**）。

## 要点

### 最小示例（ARM 裸机概念）

```ld
MEMORY
{
    FLASH (rx)  : ORIGIN = 0x08000000, LENGTH = 512K
    RAM   (rwx) : ORIGIN = 0x20000000, LENGTH = 128K
}

SECTIONS
{
    .text : {
        KEEP(*(.isr_vector))
        *(.text*)
        *(.rodata*)
    } > FLASH

    .data : AT (ADDR(.text) + SIZEOF(.text)) {
        _sdata = .;
        *(.data*)
        _edata = .;
    } > RAM

    .bss : {
        _sbss = .;
        *(.bss*) *(COMMON)
        _ebss = .;
    } > RAM
}
```

- **`>`** 目标内存；**`AT`** 指定 LMA（Flash 中存放位置）
- startup 用 `_sdata/_edata` 拷贝、`_sbss/_ebss` 清零（**4.6.2**）

### 使用方式

```bash
arm-none-eabi-gcc -nostdlib -T linker.ld -o firmware.elf startup.o main.o
arm-none-eabi-readelf -l firmware.elf
arm-none-eabi-objcopy -O binary firmware.elf firmware.bin
arm-none-eabi-nm firmware.elf | grep -E '_sdata|_sbss|main'
```

### 常用命令行

```bash
gcc -Wl,-T,linker.ld -Wl,-Map,out.map main.o
# 或
ld -T linker.ld -o out.elf *.o
```

### 与用户态默认脚本

- `ld` 默认脚本：`ld --verbose` 查看
- 一般 Linux 程序不显式写脚本；嵌入式几乎必须自定义
