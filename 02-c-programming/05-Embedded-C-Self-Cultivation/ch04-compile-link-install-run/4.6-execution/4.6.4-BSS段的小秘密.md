# 4.6.4 BSS段的小秘密

## 本节讲什么

**BSS（Block Started by Symbol）** 存放 **未初始化全局/静态变量**，在 ELF 中 **不占文件磁盘空间**（`SHT_NOBITS`），加载时由 loader 或启动代码 **清零**。`static int buf[1024];` 若未显式初始化，全为 0——这是 C 标准保证，靠 BSS 机制实现。

## 要点

### 三段对比

| 段 | 初值 | 文件占用 |
|----|------|----------|
| `.data` | 有（如 `int x=1`） | 占 |
| `.bss` | 视为 0 | **不占**（仅记录大小） |
| `.text` | 机器码 | 占 |

```bash
gcc -c foo.c
size foo.o                       # bss 列
readelf -S foo.o | grep -E 'data|bss'
```

### 示例

```c
int uninit;                      // 进 .bss
int inited = 42;                 // 进 .data
static char buf[4096];           // .bss
```

```bash
nm foo.o | grep -E 'uninit|inited|buf'
# B uninit / D inited / b buf
```

### 谁清零

- **Linux**：`execve` 映射 BSS 页为 **零页**（CoW）
- **裸机**：startup 汇编对 `[__bss_start, __bss_end)` 循环写 0（**4.6.2**）

```bash
readelf -l prog | grep LOAD       # BSS 常合并在最后一 LOAD
objdump -h prog
```

### 常见误区

- BSS 过大占 **运行期 RAM**，不占镜像体积——嵌入式要关注 RAM 预算
- `char *p;` 全局未初始化为 NULL（0），局部变量 **不在 BSS**，值不确定
