# 4.6.1 操作系统环境下的程序运行

## 本节讲什么

Linux 运行 ELF 可执行文件时，内核解析 **程序头（Program Header）**，将 `PT_LOAD` 段映射到进程虚拟地址空间，**物理页** 按需分配；`ld-linux.so` 处理动态段、重定位与初始化，再跳转到用户入口。

## 要点

### execve 装载

```bash
readelf -l ./prog
# LOAD 段：/bin 或 /lib 文件偏移 → 虚拟地址 VMA
strace -e execve,mmap ./prog 2>&1 | head -30
```

### 虚拟内存布局（典型 x86-64）

```
高地址  栈 ↓
        mmap / 堆 ↑
        共享库 .so
        .bss / .data / .rodata / .text
低地址
```

```bash
cat /proc/self/maps              # 运行中查看
pmap -x $(pidof prog)
```

### 动态链接器

- `.interp` 段指定 `/lib64/ld-linux-x86-64.so.2`
- `ld.so` 读取 `.dynamic`：`NEEDED`、`REL`、`INIT`

```bash
readelf -d ./prog
readelf --dyn-syms ./prog | head
```

### ASLR 与 PIE

```bash
gcc -pie -fPIE -o prog main.c
readelf -h prog | grep Type       # DYN（位置无关可执行）
```
