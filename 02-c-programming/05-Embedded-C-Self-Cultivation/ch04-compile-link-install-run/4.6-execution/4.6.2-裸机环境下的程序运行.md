# 4.6.2 裸机环境下的程序运行

## 本节讲什么

无 OS 时，CPU 复位后从 **向量表/Reset Handler** 开始，启动代码负责 **初始化时钟/内存控制器**、把 **`.data` 从 Flash 拷到 RAM**、**清零 `.bss`**、设置栈指针，再跳转到 `main`（或永不返回的内核式循环）。布局由 **链接脚本**（**4.14**）与 startup `.S` 决定。

## 要点

### 与 Linux 运行的差异

| 项目 | Linux | 裸机 |
|------|-------|------|
| 装载 | `execve` + loader | 烧录 Flash，复位直接执行 |
| 库 | `ld.so` | 通常静态链接 |
| BSS | loader 清零 | startup 汇编循环清零 |
| 入口 | `_start`（CRT） | `Reset_Handler` / `_start` |

### 典型启动顺序

1. 设置 SP（栈）
2. 拷贝 `.data`（LMA → VMA）
3. 清零 `.bss`
4. 调用 `SystemInit()`、`.ctors`
5. `bl main`

### 链接脚本关联

```bash
arm-none-eabi-gcc -T linker.ld -nostdlib -o firmware.elf startup.o main.o
arm-none-eabi-objcopy -O binary firmware.elf firmware.bin
readelf -l firmware.elf
```

### 查看段地址

```bash
nm firmware.elf | grep -E '_start|main'
objdump -d firmware.elf | head
```
