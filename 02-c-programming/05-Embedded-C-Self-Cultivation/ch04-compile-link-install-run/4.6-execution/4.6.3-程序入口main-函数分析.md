# 4.6.3 程序入口main()函数分析

## 本节讲什么

C 程序的 **`main` 不是 ELF 入口**。`readelf -h` 的 **Entry point** 指向 **`_start`**（C 运行时 CRT），由它准备 `argc/argv/envp`、调用全局构造函数，再调用 `main`，最后 `exit`。理解这条链对 `gdb` 调试、构造/析构顺序和嵌入式 `nostdlib` 至关重要。

## 要点

### 入口地址

```bash
readelf -h ./prog | grep 'Entry'
# 典型：0x... 指向 _start，非 main
nm ./prog | grep -E '_start|main'
```

### 调用链（glibc）

```
_start (crt1.o)
  → __libc_start_main(main, argc, argv, ...)
      → __libc_csu_init()   # .ctors 全局构造
      → main(argc, argv, envp)
      → exit(ret)
```

```bash
gcc -o prog main.c
objdump -d prog | grep -A5 '<_start>'
ldd prog                         # 依赖 libc
```

### argc/argv 从哪来

- 内核在栈上压入 `argc、argv[]、envp[]`（**4.6.1**）
- `_start` 取栈顶传给 `__libc_start_main`

### 无 libc / 裸机

```bash
arm-none-eabi-gcc -nostdlib -e main ...   # 入口可直接指定 main（需自写 startup）
```

- 内核：`start_kernel()` 替代 `main`；DPDK：`main` → `rte_eal_init()` → 业务

```bash
gdb ./prog
(gdb) break __libc_start_main
(gdb) run
```
