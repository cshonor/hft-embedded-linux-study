# 4.5 程序的安装

## 本节讲什么

「安装」是把 **构建产物**（可执行文件、库、配置、文档）按约定布局复制到目标系统目录（如 `/usr/bin`、`/usr/lib`），并更新缓存/包数据库。与 **4.4 链接** 不同：安装不改变 ELF 内容，只解决 **交付与路径**；运行期 loader 靠 `RPATH`、`LD_LIBRARY_PATH`、`ldconfig` 找到 `.so`。

## 要点

### 安装 vs 链接 vs 运行

| 阶段 | 产物位置 | 谁参与 |
|------|----------|--------|
| 链接 | 构建目录 `build/prog` | `ld` |
| 安装 | `/usr/local/bin/prog` | `make install`、包管理器 |
| 运行 | 进程地址空间 | `execve`、loader（**4.6**） |

### 典型前缀（FHS）

```
/usr/bin/        用户命令
/usr/lib/        共享库
/etc/            配置
/usr/share/      文档/数据
```

### 手工安装示例

```bash
gcc -o mytool main.c
sudo install -m 755 mytool /usr/local/bin/
sudo install -m 644 libmy.so /usr/local/lib/
sudo ldconfig                    # 更新动态库缓存
```

### 与包管理

- **4.5.2** `dpkg`/源码包；**4.5.3** `apt`；**4.5.4** Windows 安装包
- DPDK：`ninja install` 装 PMD 与头文件；内核：`make modules_install`
