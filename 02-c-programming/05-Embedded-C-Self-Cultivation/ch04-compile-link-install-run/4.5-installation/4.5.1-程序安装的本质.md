# 4.5.1 程序安装的本质

## 本节讲什么

安装的本质是 **文件部署 + 元数据登记**：把 ELF 与资源放到标准路径，让 shell、`ld.so`、服务管理器能找到它们。不改变程序逻辑，但决定 **动态库能否加载**、**配置是否生效**、**多版本能否共存**。

## 要点

### 安装做了什么

1. **复制/链接** 二进制与 `.so` 到 `bindir/libdir`
2. **权限**（`chmod`）、属主（`chown`）
3. **库缓存** `ldconfig` 扫描 `/etc/ld.so.conf.d`
4. **包元数据**（deb 的 control、文件清单）

```bash
make prefix=/usr/local install
DESTDIR=/tmp/stage make install   # 打包用暂存根
tree /tmp/stage/usr/local
```

### 动态库路径

```bash
readelf -d ./prog | grep -E 'RPATH|RUNPATH'
ldd ./prog
LD_LIBRARY_PATH=/opt/lib ./prog   # 临时覆盖
```

### 静态 vs 动态安装差异

| 类型 | 安装内容 | 运行依赖 |
|------|----------|----------|
| 静态链接 | 单个可执行文件 | 无外部 `.so` |
| 动态链接 | 可执行 + `.so` | 须安装库或设 RPATH |

```bash
gcc -static -o prog main.c
ldd prog                          # not a dynamic executable
```

### 嵌入式视角

- 无包管理：整镜像烧录（U-Boot + kernel + rootfs）
- **4.14** 链接脚本决定 Flash 布局，与 PC「安装」类比为「烧录布局」
