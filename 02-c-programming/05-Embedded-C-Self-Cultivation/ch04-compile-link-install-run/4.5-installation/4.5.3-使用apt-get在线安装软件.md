# 4.5.3 使用apt-get在线安装软件

## 本节讲什么

`apt` 从软件源下载 **预构建 .deb**，自动解决依赖并调用 `dpkg` 安装到系统路径。用户拿到的是已链接好的 ELF 与 `.so`，无需本地 **gcc/ld**（开发包装 `-dev` 才含头文件与静态库）。

## 要点

### 常用命令

```bash
sudo apt update
sudo apt install build-essential    # gcc、make、libc-dev
apt search dpdk
apt show libdpdk-dev
dpkg -L libdpdk-dev | head          # 已安装文件列表
```

### 依赖解析

- `control` 中 `Depends: libc6 (>= 2.34)` 等
- `apt` 拉取主包 + 依赖库；动态链接程序运行需对应 `.so` 版本

```bash
ldd /usr/bin/some_tool
apt-file search libfoo.so.1         # 查哪个包提供某库
```

### 与源码编译对比

| 方式 | 优点 | 缺点 |
|------|------|------|
| apt 二进制包 | 快、依赖自动 | 版本受发行版限制 |
| 源码 make install | 可定制 `-D`/优化 | 需自行处理依赖 |

### 开发工作流

```bash
sudo apt install gcc gdb binutils    # ch01 工具链
gcc -o hello hello.c
readelf -h hello
```
