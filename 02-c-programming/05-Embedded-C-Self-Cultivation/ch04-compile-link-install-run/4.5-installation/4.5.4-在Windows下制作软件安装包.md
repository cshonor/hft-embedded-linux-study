# 4.5.4 在Windows下制作软件安装包

## 本节讲什么

Windows 上 C/C++ 构建常用 **MSVC/MinGW**，产物为 `.exe`/`.dll`（对应 Linux 可执行/`.so`）。「安装包」由 **NSIS、WiX、Inno Setup** 等把二进制、DLL、运行时（VC++ Redistributable）打包；概念上等同 Linux 的 **文件部署 + 注册表/PATH**，与 **4.4 链接** 阶段分离。

## 要点

### 产物对比

| Linux | Windows |
|-------|---------|
| ELF 可执行 | `.exe`（PE 格式） |
| `.so` | `.dll` |
| `ld.so` | 加载器 + `LoadLibrary` |
| `/usr/lib` | `C:\Program Files\...\` 或 PATH |

```bash
# MinGW 交叉（在 Linux 上）
x86_64-w64-mingw32-gcc -o app.exe main.c
```

### 安装包内容

1. `.exe` 主程序（已 **link** 完成）
2. 依赖 `.dll`（或静态链接 CRT）
3. 快捷方式、卸载项、注册表（可选）

### 与 ELF  toolchain 的差异

- Windows 用 **PE/COFF**，工具为 `dumpbin`、`link.exe`（非 `readelf/nm`，但概念平行）
- WSL 内仍可用 **gcc/readelf** 学 Linux ELF（本书主线）

### 跨平台开发提示

- CMake `install()` 抽象安装路径
- DPDK/内核主线在 Linux；Windows 多为工具链或文档环境

## 与后续衔接

- **4.5.1** 安装本质相同：部署 + 让 loader 找到 DLL
- Linux 主线继续 **4.6** `exec`、**4.8** 动态链接
- 嵌入式 Windows IoT 仍建议掌握 ELF/Linux 链路
