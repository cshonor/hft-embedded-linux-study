# 4.5.2 在Linux下制作软件安装包

## 本节讲什么

Linux 常用 **dpkg（.deb）** 或 **RPM** 分发：把 `make install` 的文件清单、依赖、脚本打包，由 `dpkg/apt` 解压到根文件系统并触发 `postinst`。开源项目常提供 **autotools/cmake + checkinstall** 或正式 `debian/` 目录。

## 要点

### 基本流程

```bash
./configure --prefix=/usr
make -j$(nproc)
sudo make install               # 或 DESTDIR 暂存
```

### DESTDIR 打包

```bash
make DESTDIR=/tmp/pkg install
find /tmp/pkg -type f
# 再用 fpm/checkinstall 生成 .deb
```

### checkinstall 示例

```bash
sudo checkinstall --pkgname=mytool --pkgversion=1.0
# 跟踪 make install 写入的文件生成 deb
```

### deb 结构（概念）

- `DEBIAN/control`：包名、版本、依赖（`libc6` 等）
- `usr/bin/mytool`：实际文件
- 安装后：`ldconfig` 在 `postinst` 中调用

```bash
dpkg -L mytool                    # 列出安装文件
dpkg -s mytool                    # 包信息
```

### 与编译链接的关系

- 构建阶段仍用 **gcc/ld**（**4.1–4.4**）
- 打包只收录 `usr/` 下产物；`-dev` 包另含头文件与 `.a`

## 与后续衔接

- **4.5.3** `apt` 在线安装；**4.8.4** 包依赖 `libc.so.6`
- DPDK：发行版 `dpkg` 或自编译 `ninja install`
- 内核：`make deb-pkg` 生成内核 deb
