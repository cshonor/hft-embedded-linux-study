# 4.1 从源程序到二进制文件

## 本节讲什么

一条 C 源文件从 `hello.c` 到可执行文件，经历 **预处理 → 编译 → 汇编 → 链接** 四阶段。GCC 用 `-E/-S/-c` 停在中间产物，最后用 `ld`（经 `gcc` 驱动）生成 ELF 可执行文件。理解这条流水线，是读 `.i/.s/.o`、排查链接错误和做嵌入式交叉编译的前提。

## 要点

### 四阶段与 gcc 选项

| 阶段 | 输入 | 输出 | gcc 选项 |
|------|------|------|----------|
| 预处理 | `.c` | `.i` | `-E` |
| 编译 | `.i` | `.s` | `-S` |
| 汇编 | `.s` | `.o` | `-c` |
| 链接 | `.o` + 库 | 可执行文件 / `.so` | 无 `-c` |

```bash
gcc -E hello.c -o hello.i      # 预处理
gcc -S hello.c -o hello.s      # 生成汇编
gcc -c hello.c -o hello.o       # 生成目标文件
gcc hello.o -o hello           # 链接
```

### ELF 文件类型

| 后缀 | 类型 | 说明 |
|------|------|------|
| `.o` | `ET_REL` 可重定位 | 含 `.text/.data/.bss`、符号表、重定位表 |
| `.a` | 静态库归档 | `ar` 打包多个 `.o` |
| `.so` | `ET_DYN` 共享库 | 运行期动态加载 |
| 无后缀可执行 | `ET_EXEC` 或 PIE `ET_DYN` | 含程序头、入口地址 |

```bash
readelf -h hello.o hello       # 查看 ELF 头：Type、Machine、Entry
file hello.o hello libfoo.so   # 快速识别文件类型
```

### 常见段（Section）

- **`.text`**：机器码
- **`.data`**：已初始化全局/静态变量
- **`.bss`**：未初始化全局/静态（不占文件空间，运行期清零）
- **`.rodata`**：只读常量
- **`.symtab` / `.strtab`**：符号表（**4.3.3**）
- **`.rel.text`** 等：重定位表（**4.4.3**）

```bash
readelf -S hello.o             # 段头表
objdump -h hello.o             # 段摘要
size hello.o                   # .text/.data/.bss 大小
```
