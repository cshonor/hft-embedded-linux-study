# 4.4.3 重定位

## 本节讲什么

符号决议完成后，链接器遍历各输入的 **重定位表**，按公式把 **占位地址** 改成最终 VMA 下的正确值，写回输出文件的 `.text/.data`。编译期未知的跨文件地址、全局指针、GOT 项都在此或加载期修补。

## 要点

### 重定位公式（概念）

对每条重定位项：`*addr = S + A + P`

- **S**：符号最终值（地址）
- **A**：加数（addend）
- **P**：被修补位置的段内偏移
- 具体因 **重定位类型** 而异（相对/绝对/PC 相对）

```bash
readelf -r foo.o                 # 链接前
readelf -r prog                  # 可执行文件通常为空（已修补）
objdump -d prog | grep call      # 已是正确目标
```

### 典型类型（x86-64）

| 类型 | 场景 |
|------|------|
| `R_X86_64_PC32` | 相对跳转 `call` |
| `R_X86_64_64` | 64 位绝对地址 |
| `R_X86_64_GOTPCREL` | 经 GOT 访问（**4.8**） |

### 链接前后对比

```bash
gcc -c main.c
objdump -d main.o                # call 目标可能为 0 或占位
gcc main.o -o main
objdump -d main                  # call 指向 libc 或 PLT
```

### 加载时重定位

- **PIE + ASLR**：内核 loader 映射到随机基址，再做 **动态重定位**（`.rela.dyn`）
- **内核/U-Boot**：自拷贝与重定位（**4.11**、**4.12**）

```bash
readelf -d ./prog | grep RELA
ldd ./prog
```
