# 4.4.1 分段组装

## 本节讲什么

链接第一步：把各输入文件的 **同名段** 按规则拼接成输出文件的一个段，并为每段分配 **虚拟地址（VMA）** 和 **加载地址（LMA）**。可执行文件的段布局决定运行期内存映像；裸机由链接脚本指定 Flash/RAM 位置（**4.14**）。

## 要点

### 段合并规则

- 所有 `.text` → 输出 `.text`（按链接顺序或脚本排序）
- `.data`、`.bss`、`.rodata` 同理
- **COMDAT / 弱符号** 去重（C++ 内联、模板）

```bash
gcc -c a.c b.c
readelf -S a.o b.o
gcc a.o b.o -o ab
readelf -S ab                 # 段合并后总大小
```

### VMA 与 LMA

- **VMA**：运行时的虚拟地址（Linux 用户态由 loader 建立映射）
- **LMA**：加载到内存/Flash 的物理地址（嵌入式常 VMA≠LMA：代码在 Flash 运行或需拷贝到 RAM）

```bash
readelf -l ab                  # Program Headers：LOAD 段 VMA/Align
objdump -h ab
```

### 链接顺序影响

```bash
gcc -o prog main.o libfoo.a libbar.a   # 从左到右解析符号
# 静库放后面；循环依赖可： -lfoo -lbar -lfoo
```

### 查看布局

```bash
gcc -Wl,-Map=prog.map,-cref main.o foo.o
grep '\.text' prog.map
size prog                       # text/data/bss 汇总
```
