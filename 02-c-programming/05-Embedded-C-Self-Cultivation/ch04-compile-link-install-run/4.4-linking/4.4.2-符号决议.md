# 4.4.2 符号决议

## 本节讲什么

链接器扫描全部符号，为每个 **未定义引用（U）** 找到 **唯一定义**，或报错。规则涉及强/弱符号、COMMON 段、静库成员按需抽取。本节是理解 `undefined reference` 与 `multiple definition` 的核心。

## 要点

### 强符号 vs 弱符号

| 类型 | 示例 | 冲突时 |
|------|------|--------|
| 强 | 普通全局定义 `int x=1;` | 不能有两个强定义 |
| 弱 | `__attribute__((weak))` 或 `int x;`（ tentative） | 强覆盖弱 |

```c
// a.c
int val = 10;           // 强
// b.c
int val;                // 弱 COMMON，链接合并到一处
```

### 常见错误

**undefined reference to `foo'`**

```bash
gcc main.o -o main
# main.o 中 U foo，但未链接定义 foo 的 .o 或库
gcc main.o foo.o -o main    # 补全
nm main.o | grep foo
```

**multiple definition of `bar'`**

```bash
# bar 在两个 .o 都有强定义
nm a.o b.o | grep bar
# 改 static、或只保留一处定义、或用 weak
```

### 静库符号抽取

`ar` 归档的 `.a` **按成员整包尝试**：从左到右，仅当某 `.o` 能解析当前未定义符号时才装入（**4.7**）。

```bash
gcc main.o -L. -lfoo -lbar
nm -A libfoo.a | grep ' T '
```

### C/C++ 注意

- `extern "C"` 避免名字改编
- 内联/`static` 函数不导出符号
- 查看：

```bash
nm -C prog | less
c++filt _Z3foov
```
