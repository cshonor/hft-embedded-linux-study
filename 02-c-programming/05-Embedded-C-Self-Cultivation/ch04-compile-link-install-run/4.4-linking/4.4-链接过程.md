# 4.4 链接过程

## 本节讲什么

链接把多个 `.o` 和库拼成最终 ELF，分三步：**分段组装**（合并 `.text/.data` 等）→ **符号决议**（匹配 `U` 与 `T/D/B`）→ **重定位**（修补代码中的地址引用）。静态链接在构建期完成；动态链接部分推迟到加载/运行（**4.8**）。

## 要点

### 链接器角色

- 输入：`.o`、`.a`、`.so`（仅记录依赖）、链接脚本（**4.14**）
- 输出：可执行文件、`ET_DYN` PIE、或 `.so`
- 工具：`ld`（常由 `gcc/g++` 调用）

```bash
gcc foo.o bar.o -o prog
ld -o prog foo.o bar.o -lc          # 显式调用
gcc -Wl,-Map=prog.map foo.o bar.o   # 生成链接映射
```

### 三步概览

| 步骤 | 小节 | 核心 |
|------|------|------|
| 分段组装 | **4.4.1** | 同名段合并、分配 VMA |
| 符号决议 | **4.4.2** | 解析 undefined、处理冲突 |
| 重定位 | **4.4.3** | 按最终地址写指令/数据 |

### 静/动态链接选择

```bash
gcc -static -o prog foo.o bar.o     # 全静态，体积大
gcc -shared -fPIC -o libfoo.so foo.c
gcc -o prog main.o -L. -lfoo        # 动态链接 libfoo
```

## 与后续衔接

- **4.4.1–4.4.3** 各步细节；**4.7** 静态库；**4.8** 动态链接
