# 4.11 Linux内核编译和启动分析

## 本节讲什么

Linux 内核由数千 `.o` 经 **特殊链接脚本** `vmlinux.lds` 链成 **`vmlinux`**，再压缩为 `bzImage`/`Image`。启动时 Bootloader 载入内核，入口 **汇编 startup** 建立页表、解压、**自重定位**，最后进入 `start_kernel()`——是 **4.4 链接 + 4.6 运行 + 4.12 重定位** 的大尺度实例。

## 要点

### 内核构建（简）

```bash
cd linux
make defconfig && make -j$(nproc)
readelf -h vmlinux | grep Entry
nm vmlinux | grep start_kernel
```

### 链接脚本

- `arch/x86/kernel/vmlinux.lds.S`：定义内核虚拟地址布局（`__start`、`__init` 等段）
- 与用户态 **4.14** 同属 `SECTIONS` 思想，规模更大

```bash
readelf -S vmlinux | grep -E 'text|init|data'
objdump -d vmlinux | head
```

### 启动链（x86 概念）

```
Bootloader → 实模式/保护模式入口
  → head_32/head_64 汇编
  → decompress_kernel（若压缩）
  → start_kernel()
      → setup_arch、mm_init、rest_init
      → kernel_init → 用户态 init
```

### 与 DPDK

- 内核配置 **Hugepage、IOMMU、VFIO** 影响 DPDK 运行环境
- `uname -r` 与 `linux-headers` 版本须与模块构建匹配（**4.10**）
