# 4.3.1 从C文件到汇编文件

## 本节讲什么

编译器核心输出是 **汇编语言**（`.s`），而非直接生成机器码。`gcc -S` 可单独停在汇编阶段，便于对照 C 语句与指令、理解优化与调用约定（**ch03 AAPCS**）。

## 要点

### 生成汇编

```bash
gcc -S hello.c -o hello.s
gcc -S -O2 hello.c -o hello.O2.s    # 对比优化级别
gcc -S -fverbose-asm hello.c        # 汇编中带 C 行号注释
```

### 典型汇编结构（x86-64）

```asm
    .file   "hello.c"
    .text
    .globl  main
main:
    pushq   %rbp
    movq    %rsp, %rbp
    ...
    call    puts
    popq    %rbp
    ret
```

- **`.globl symbol`**：导出全局符号供链接
- **`call foo`**：若 `foo` 在其他 `.o`，生成 **重定位项**（地址先填 0 或占位）

### 阅读要点

- 函数序言/尾声：建立栈帧（**ch05**）
- 参数传递：x86-64 用寄存器（rdi, rsi…）；ARM 用 r0–r3
- 未定义符号：`call` 外部函数时出现 `R_X86_64_PLT32` 等重定位类型

```bash
objdump -d hello.o              # 若已 -c
objdump -S hello.o              # 混排 C 源码（需 -g 编译）
```

### 优化影响

- `-O0`：变量多在栈上，便于调试
- `-O2`：内联、寄存器分配、死代码消除；汇编行数可能变少或重排
