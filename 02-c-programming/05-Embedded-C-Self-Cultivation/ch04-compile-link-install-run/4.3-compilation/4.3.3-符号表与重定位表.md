# 4.3.3 符号表与重定位表

## 本节讲什么

每个 `.o` 携带 **符号表**（有哪些函数/变量、定义还是引用）和 **重定位表**（哪些指令地址待修补）。链接器靠这两张表完成 **符号决议** 与 **重定位**（**4.4**）。读 `nm`/`readelf -s` 是排查 `undefined reference` 的必备技能。

## 要点

### 符号表（.symtab）

| nm 列 | 含义 |
|-------|------|
| `T/t` | 代码段已定义（全局/局部） |
| `D/d` | 已初始化数据 |
| `B/b` | BSS 未初始化数据 |
| `U` | 未定义，需其他模块提供 |
| `C` | 公共符号（已废弃，现代链接器少见） |

```bash
nm -C foo.o                    # C++ demangle
readelf -s foo.o               # 完整符号：Bind、Type、Ndx
readelf --syms foo.o | grep UND
```

### 重定位表（.rel.*）

每条重定位记录：**偏移**、**类型**、**符号索引**、**加数**。

```bash
readelf -r foo.o
# Offset   Info    Type             Sym.Value  Sym.Name
# 0000000a 00000a R_X86_64_PC32      00000000  puts
```

含义：`.text` 偏移 `0xa` 处的 `call` 需相对 `puts` 的最终地址修补。

### ELF 段与符号关系

```bash
readelf -S foo.o
# [Nr] Name    Type     ...
#  1   .text   PROGBITS ...
#  7   .symtab SYMTAB   ...
#  9   .rel.text REL     ...
```

### .o / .a / .so / 可执行文件对比

| 文件 | 符号表 | 重定位 |
|------|--------|--------|
| `.o` | 完整 | 有 `.rel.*` |
| `.a` | 各成员 `.o` 归档 | 成员内仍有 |
| `.so` | 有（动态符号 `.dynsym`） | 链接时/加载时 |
| 可执行 | 可 strip 掉 | 通常已修补完 |

```bash
ar t libfoo.a && nm libfoo.a
readelf -d libfoo.so             # 动态段
```
