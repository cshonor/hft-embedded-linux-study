# 4.3.2 汇编过程

## 本节讲什么

**汇编器 `as`** 将助记符 `.s` 翻译成机器码，写入 `.o` 的 `.text/.data` 等段，并生成 **符号表** 与 **重定位表**。每条未解析的地址引用（跨文件调用、全局变量地址）都记录为重定位项，等链接器填正。

## 要点

### 汇编器输入输出

```bash
as -o hello.o hello.s           # 显式调用
gcc -c hello.c                  # 内部调用 as
```

### 段与符号

汇编伪操作决定段归属：

```asm
.section .text
.globl  func
func:
    ret

.section .data
.globl  counter
counter:
    .long 0
```

```bash
nm hello.o
# T func    — .text 中已定义（局部 Text）
# D counter — .data 中已定义（Data）
# U puts    — 未定义，待链接（Undefined）
```

### 重定位项诞生

对 `call puts@PLT` 或 `movabs $msg, %rdi`，若 `puts`/`msg` 地址未知，`as` 在 `.rel.text` 记录：

```bash
readelf -r hello.o              # 重定位表
objdump -r hello.o              # 另一种查看方式
```

常见类型（x86-64）：`R_X86_64_PC32`（相对调用）、`R_X86_64_64`（绝对地址）。

### 与编译器协作

- 编译器负责生成正确伪操作（`.cfi_*` 栈展开信息等）
- 手写汇编（**ch03**）须自己声明 `.global`、段对齐

```bash
gcc -c add_asm.S -o add_asm.o
readelf -S add_asm.o
```
