# 4.3 程序的编译

## 本节讲什么

「编译」在狭义上指 **C → 汇编**（`gcc -S`），广义上常把 **汇编 → .o** 也算入编译阶段。编译器为每个翻译单元生成独立 `.o`，**外部符号先留空**，由链接器后续决议（**4.4**）。单文件编译快、可并行，是大型内核/DPDK 工程的基础。

## 要点

### 编译器做什么

- 词法/语法/语义分析、优化
- 生成目标体系结构的汇编（x86/ARM 等）
- 输出 **ELF 可重定位文件** `.o`，含机器码 + 符号表 + 重定位项

```bash
gcc -c foo.c -o foo.o
gcc -c bar.c -o bar.o
gcc foo.o bar.o -o prog        # 链接阶段
```

### 与汇编器的分工

| 工具 | 输入 | 输出 |
|------|------|------|
| `cc1`（编译器前端） | `.c/.i` | `.s` |
| `as`（汇编器） | `.s` | `.o` |

`gcc -c` 一次完成两步；`gcc -S` 只到汇编。

### 并行与依赖

```bash
make -j$(nproc)               # 多 .c 并行 -c
gcc -MMD -MP -c foo.c         # 同时生成 foo.d 依赖文件
```

### 查看产物

```bash
file foo.o
readelf -S foo.o
nm foo.o
objdump -d foo.o
```
