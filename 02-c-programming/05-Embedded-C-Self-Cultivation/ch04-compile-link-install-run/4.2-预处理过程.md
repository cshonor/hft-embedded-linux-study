# 4.2 预处理过程

## 本节讲什么

预处理在 **编译之前** 完成：展开 `#include`、处理 `#define` 宏、条件编译 `#if/#ifdef`、删除注释。输出仍是 C 源码（`.i`），但已是「单文件、无宏」的展开结果。内核/DPDK 大量用条件编译区分架构与特性。

## 要点

### 预处理任务

1. **头文件展开**：`#include <stdio.h>` 递归插入头文件内容
2. **宏替换**：`#define MAX(a,b) ((a)>(b)?(a):(b))` 文本替换（注意副作用）
3. **条件编译**：`#ifdef __KERNEL__`、`#if RTE_ARCH_X86` 等
4. **行号标记**：`#line` 保留错误定位
5. **删除注释**与多余空白

### 常用指令

```c
#define PI 3.14
#undef  PI
#ifdef DEBUG
#pragma once          // 防重复包含（非标准但广泛支持）
#endif
```

### 实战命令

```bash
gcc -E hello.c -o hello.i           # 完整展开
gcc -E hello.c | less               # 浏览（很长）
gcc -E -dM hello.c                  # 仅列出宏定义
gcc -E hello.c -D DEBUG=1           # 命令行注入宏
```

### 调试技巧

- 链接错误常源于 **宏展开后** 的类型/声明不一致
- 内核 `make V=1` 可看到实际 `-D` 选项
- 用 `-save-temps` 保留 `.i/.s/.o` 对照：

```bash
gcc -save-temps -c foo.c
ls foo.i foo.s foo.o
```

## 与后续衔接
