# 4.7 链接静态库

## 本节讲什么

**静态库** `.a` 是多个 `.o` 的 **归档**（`ar`），链接时链接器 **按需抽取** 成员 `.o` 并入可执行文件，代码成为二进制一部分，**运行期无 `.so` 依赖**。适合嵌入式、内核部分场景；缺点是体积与升级成本。

## 要点

### 制作与使用

```bash
gcc -c foo.c bar.c
ar rcs libmy.a foo.o bar.o
ar t libmy.a                     # 列出成员
nm libmy.a
gcc main.o -L. -lmy -o prog      # -lmy 即 libmy.a
```

### 链接器搜索规则

- `-L/path` 指定目录
- `-lname` → `libname.a` 或 `libname.so`（默认先 so，可用 `-static` 强制静库）
- **从左到右**：未解析符号才拉入 `.a` 中对应 `.o`

```bash
gcc -static -o prog main.o -L. -lmy
ldd prog                         # not a dynamic executable
```

### 与符号决议（**4.4.2**）

- 静库中未被引用的 `.o` **不会** 进入最终 ELF（减小体积）
- 循环依赖：`gcc ... -lfoo -lbar -lfoo`

### 常见错误

```bash
# undefined reference — 库顺序错或未 -l
gcc main.o -lmy -o prog          # 错：main.o 在后更好
gcc main.o -o prog -L. -lmy

# multiple definition — 同一符号在多个 .o 且都被拉入
nm libmy.a | grep duplicate_sym
```

### 查看最终是否编入

```bash
nm prog | grep my_func
readelf -s prog | grep my_func
```
