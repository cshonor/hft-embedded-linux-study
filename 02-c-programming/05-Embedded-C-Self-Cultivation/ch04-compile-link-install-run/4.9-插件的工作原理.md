# 4.9 插件的工作原理

## 本节讲什么

**插件** 本质是运行期用 **`dlopen/dlsym/dlclose`** 装入的 **共享库**，主程序在编译期不必链接插件，通过函数指针调用。Firefox/Nginx 模块、DPDK **PMD 动态驱动** 同属此模式。

## 要点

### 基本 API

```c
void *h = dlopen("./libplugin.so", RTLD_LAZY);
int (*init)(void) = dlsym(h, "plugin_init");
init();
dlclose(h);
```

```bash
gcc -fPIC -shared -o libplugin.so plugin.c
gcc -o host host.c -ldl
LD_LIBRARY_PATH=. ./host
```

### 与 4.8 的关系

- 插件必须是 **PIC 的 `.so`**
- 符号须 **导出**（`nm -D libplugin.so`）
- 延迟绑定同样适用（**4.8.3**）

```bash
readelf -d libplugin.so
ldd libplugin.so
```

### 常见约定

- 固定入口符号：`plugin_register`、`rte_pmd_init` 等
- 版本协商结构体，避免 ABI 不匹配

### 排错

```bash
dlerror()                        # 代码中打印
LD_DEBUG=bindings ./host       # 观察加载
```
