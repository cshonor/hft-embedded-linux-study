# 4.8.1 与地址无关的代码

## 本节讲什么

**PIC（Position Independent Code）** 使 `.so` 文本段可映射到 **任意虚拟地址**（ASLR），指令中不出现绝对地址，而通过 **GOT/PC 相对** 访问数据与函数。编译加 `-fPIC`（x86-64 常隐含于 `-shared`）。

## 要点

### 为何需要 PIC

- 共享库只加载 **一份物理代码**，多进程映射到 **不同基址**
- 非 PIC：重定位会修改 `.text`（**Copy-on-Write**，失去共享优势）

```bash
gcc -fPIC -c foo.c -o foo_pic.o
gcc -fno-PIC -c foo.c -o foo_nop.o
readelf -r foo_pic.o foo_nop.o
```

### 访问全局变量（x86-64）

- 非 PIC：`movabs $sym, %rax`（绝对地址）
- PIC：`mov sym@GOTPCREL(%rip), %rax` 经 GOT 间接加载

```bash
gcc -fPIC -S -O0 foo.c -o -
objdump -d foo_pic.o | grep -i got
```

### 编译选项

```bash
gcc -fPIC -shared -o libfoo.so foo.c
gcc -fPIE -pie -o prog main.c    # 可执行也可 PIE
readelf -h libfoo.so | grep Type  # DYN
```

### ARM 注记

- ARM/ARM64 通常默认 PIC；`adr` + GOT 访问（**ch03**）
