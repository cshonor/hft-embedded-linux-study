# 4.8.3 延迟绑定

## 本节讲什么

**延迟绑定（Lazy Binding）** 通过 **PLT（Procedure Linkage Table）** 实现：首次 `call printf@plt` 才由 `ld.so` 解析真实地址并写入 GOT，后续调用直达目标。加快启动、避免链接未用函数。可用 `LD_BIND_NOW` 关闭。

## 要点

### PLT 桩（概念）

```
call printf@plt  →  PLT[n]  →  首次：跳 ld.so 解析器
                              →  之后：GOT[n] 已是 printf 地址
```

```bash
gcc -o prog main.c
objdump -d prog | grep -A3 '<printf@plt>'
readelf -r prog | grep JUMP_SLOT
```

### 相关节

```bash
readelf -S prog | grep -E 'plt|got'
readelf -d prog | grep PLTGOT
```

- `.plt`：桩代码
- `.got.plt`：函数地址槽
- `.rela.plt`：`R_X86_64_JUMP_SLOT` 重定位

### 强制立即绑定

```bash
LD_BIND_NOW=1 ./prog
# 或链接： gcc -Wl,-z,now ...
readelf -d prog | grep BIND_NOW
```

### 调试

```bash
ltrace ./prog                    # 观察首次解析
gdb ./prog
(gdb) break printf
(gdb) run
```
