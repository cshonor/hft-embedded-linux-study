# 4.10 Linux内核模块运行机制

## 本节讲什么

**内核模块**（`.ko`）是运行期由 `insmod` 装入内核空间的 **可重定位 ELF**，与用户态 `.so` 类似但无 `ld.so`：由内核 `module.c` 负责分配内存、重定位、解析符号、注册导出表。读 LKD 驱动、DPDK 内核态 PMD 依赖的「装入」概念与此相通。

## 要点

### 与用户态动态链接对比

| | 用户态 `.so` | 内核模块 `.ko` |
|---|--------------|----------------|
| 加载器 | `ld-linux.so` | `kernel/module.c` |
| 入口 | `_init` / 构造函数 | `module_init()` 宏 |
| 符号 | `dlsym` | `/proc/kallsyms`、导出表 |
| 卸载 | `dlclose` | `rmmod` |

### 构建与加载

```bash
make -C /lib/modules/$(uname -r)/build M=$PWD modules
file hello.ko
readelf -S hello.ko | head
sudo insmod hello.ko
lsmod | grep hello
dmesg | tail
sudo rmmod hello
```

### 模块 ELF 特征

```bash
readelf -s hello.ko | grep -E 'init|exit'
nm hello.ko
# module_init → __init 段；GPL 许可证符号等
```

### 内核钩子（DPDK/网络）

- `igb_uio`、`vfio-pci` 等以模块形式提供用户态映射
- 网络驱动 `insmod` 后注册 `net_device`；与用户态 DPDK poll 配合
