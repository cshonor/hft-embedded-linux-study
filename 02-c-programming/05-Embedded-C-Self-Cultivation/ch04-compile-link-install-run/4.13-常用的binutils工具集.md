# 4.13 常用的binutils工具集

## 本节讲什么

**GNU Binutils** 贯穿编译链接全流程：`as`、`ld`、`ar`、`nm`、`objdump`、`readelf`、`strip` 等。掌握它们可在 **不运行程序** 的情况下检查 ELF 段、符号、重定位——排查 **4.4.2** 链接错误与 **4.8** 动态库问题的基本功。

## 要点

### 工具速查

| 工具 | 用途 |
|------|------|
| `gcc -E/-S/-c` | 预处理/汇编/编译（驱动） |
| `as` | `.s` → `.o` |
| `ld` | 链接（`gcc` 封装） |
| `ar` | 静库 `rcs` / 列表 `t` |
| `nm` | 符号表 |
| `readelf` | ELF 头、段、动态段、重定位 |
| `objdump` | 反汇编、段、`-r` 重定位 |
| `strip` | 删除符号/debug |
| `size` |段大小汇总 |

### 实战组合

```bash
# 四阶段产物
gcc -save-temps -c demo.c
ls demo.i demo.s demo.o

# ELF 结构
readelf -hWa demo.o
readelf -S demo.o
readelf -r demo.o

# 符号
nm -C demo.o
nm -D libdemo.so              # 动态符号

# 反汇编
objdump -d -S demo.o
objdump -R ./prog              # 动态重定位

# 静库
ar tv libdemo.a
nm libdemo.a

# 可执行
readelf -l ./prog
ldd ./prog
file demo.o libdemo.a libdemo.so ./prog
```

### 与 demo/ 目录

```bash
cd ch04-compile-link-install-run/demo
make demo01
readelf -S demo01.o demo01
nm demo01.o
```
