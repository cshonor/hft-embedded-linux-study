# 9.8.1 跨平台设计

## 本节讲什么

固件常需**同一套业务逻辑**跑在 MCU、Linux 用户态或 PC 仿真环境。跨平台不是 `#ifdef` 堆满全文件，而是 **platform 抽象层 + weak 默认 + 链接时选择实现**。本节给出 HAL 划分、主机开发流程，并衔接 **ch06 weak** 与 **demo04_weak**。

## 要点

**platform 层职责**：

```
app/           — 业务，零 #ifdef OS
middleware/    — 协议，零寄存器
driver/        — 逻辑设备（UART 状态机）
platform/      — 时间、GPIO、mutex、内存、日志底层
    ├── platform_weak.c    /* PC 仿真 stub */
    ├── stm32/platform.c
    └── linux/platform.c
```

| API | STM32 | Linux | PC weak |
|-----|-------|-------|---------|
| `platform_msleep()` | SysTick | `usleep` | `usleep` |
| `platform_get_tick()` | `HAL_GetTick` | `clock_gettime` | `clock_gettime` |
| `platform_mutex_*()` | RTOS / 空 | `pthread` | 空 stub |

**weak 符号模式**（**ch06 6.9**、**demo04_weak**）：

```c
/* platform_weak.c */
__attribute__((weak)) uint32_t platform_get_tick(void)
{
    return 0;  /* 仿真 */
}

/* platform_stm32.c — 强符号覆盖 */
uint32_t platform_get_tick(void)
{
    return HAL_GetTick();
}
```

- Makefile 通过 `OBJS` 选择链接哪个 platform `.o`
- CMake：`target_sources(platform PRIVATE stm32/platform.c)` 按 option 切换

**条件编译的收敛**：

```c
/* ❌ app/main.c 散落 */
#ifdef STM32
  HAL_Init();
#elif LINUX
  ...
#endif

/* ✅ platform_init() 一处 */
err_t platform_init(void);  /* 各平台 .c 各自实现 */
```

- **`config.h` / CMake `option`**：只控制**编译哪些模块**，不控制业务 if/else
- 允许 `#ifdef` 的地方：`platform/`、`arch/`、汇编

**PC 仿真工作流**：

```bash
# 默认 weak platform，gcc 本机编译
make TARGET=host
./app.elf

# 交叉
make TARGET=stm32 CROSS_COMPILE=arm-none-eabi-
```

- app + middleware + driver 逻辑在 PC 上跑单元测试
- 硬件相关仅 platform/driver 边界替换

**CMake 交叉编译**（**demo03_cmake**）：

```cmake
set(CMAKE_SYSTEM_NAME Generic)
set(CMAKE_C_COMPILER arm-none-eabi-gcc)
add_executable(firmware app/main.c)
target_link_libraries(firmware mod_uart platform_stm32)
```
