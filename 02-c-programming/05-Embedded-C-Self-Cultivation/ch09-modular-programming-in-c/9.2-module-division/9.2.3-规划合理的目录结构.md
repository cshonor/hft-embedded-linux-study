# 9.2.3 规划合理的目录结构

## 本节讲什么

模块划分定稿后，需要**可执行的目录树**承载编译、测试与协作。本节给出嵌入式 C 项目的标准目录模板、命名约定、`.gitignore` 与 `config.h` 放置策略，并与 **demo01_minimal**、**demo02_make** 的 layout 一一对应。

## 要点

**推荐项目骨架**：

```
firmware/
├── Makefile                 # 顶层入口
├── CMakeLists.txt           # 可选，与 Makefile 并存
├── config.h                 # 全局功能开关（或 include/config.h）
├── .gitignore
├── app/
│   ├── main.c
│   └── app_logic.c
├── middleware/              # 可选
│   └── mqtt/
│       ├── mqtt.c
│       └── mqtt.h
├── driver/
│   └── mod_uart/
│       ├── mod_uart.c
│       ├── mod_uart.h
│       └── Makefile         # 单模块构建 → libmod_uart.a
├── platform/
│   ├── platform_weak.c      # demo04_weak
│   └── stm32/
│       └── platform_stm32.c
├── utils/
│   ├── ringbuf.c
│   └── ringbuf.h
├── common/                  # demo05
│   ├── log.h
│   ├── err.h
│   └── err.c
├── include/                 # 对外统一头文件（install 目标）
├── build/                   # 生成物，不入库
└── out/                     # install 输出
    ├── lib/
    └── include/
```

**单模块目录规范**（`mod_uart` 为例，**demo02_make**）：

```
driver/mod_uart/
├── mod_uart.h      # 公开 API
├── mod_uart.c      # 实现
├── mod_uart_internal.h  # 可选，仅本目录 .c 使用
└── Makefile
```

子模块 Makefile 核心：

```makefile
SRCS = mod_uart.c
OBJS = $(SRCS:.c=.o)
TARGET = libmod_uart.a

all: $(TARGET)
$(TARGET): $(OBJS)
	$(AR) rcs $@ $^
install:
	cp mod_uart.h ../../out/include/
	cp $(TARGET) ../../out/lib/
```

**命名约定**：

| 类型 | 约定 | 示例 |
|------|------|------|
| 模块前缀 | `mod_` 或领域名 | `mod_uart`、`ringbuf` |
| 头文件 guard | `MOD_UART_H` / 路径式 | `DRIVER_MOD_UART_H` |
| 静态库 | `lib<name>.a` | `libmod_uart.a` |
| 内部头 | `*_internal.h` | 禁止 app include |

**config.h 与 .gitignore**：

```c
/* config.h — 编译期特性开关 */
#define CONFIG_UART_BAUD 115200
#define CONFIG_LOG_ENABLE 1
#define CONFIG_FEATURE_OTA 0
```

`.gitignore` 至少忽略：

```
build/
out/
*.o
*.a
*.elf
*.bin
.vscode/
```

- `config.h` 可提交模板 `config.h.in`，本地 `config.h` 加入 ignore（若含密钥/路径）
- **demo05** 的 `LOG_ENABLE` 与 `config.h` 配合，统一日志开关

**include 路径策略**（**9.4.7**）：

- 公开头：` -Iinclude` 或 `-Idriver/mod_uart`
- 禁止 `-I.` 全局递归，防止 `#include "secret_internal.h"`
- 模块内用 `#include "mod_uart_internal.h"`；跨模块用 `#include <mod_uart.h>`

**规模与深度**：

| 项目规模 | 建议 |
|----------|------|
| < 5 个 `.c` | 扁平 `src/` + 一个 Makefile |
| 5–30 个 `.c` | 按层分目录 + 子 Makefile |
| > 30 个 `.c` | CMake + `add_subdirectory`；CI 分模块编译 |
