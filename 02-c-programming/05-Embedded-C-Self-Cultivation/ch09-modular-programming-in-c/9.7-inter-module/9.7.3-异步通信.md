# 9.7.3 异步通信

## 本节讲什么

当生产者与消费者**不在同一调用栈**（中断 vs 主循环、多任务、不同模块线程）时，同步回调不够，需要**异步消息**：队列、事件标志、mailbox。本节介绍嵌入式常见的轻量异步模式，为 **ch10 并发与 OS 抽象** 铺垫，并说明与模块化边界的配合。

## 要点

**典型场景**：

```
UART ISR ──► ringbuf ──► parser 任务 ──► app 逻辑
                ▲
           driver 模块    middleware      app 模块
```

| 机制 | 复杂度 | 适用 |
|------|--------|------|
| **volatile 标志 + 轮询** | 低 | 单 bit 事件，主循环快 |
| **环形队列** | 中 | 字节流/固定大小消息 |
| **消息队列（OS）** | 中–高 | FreeRTOS `xQueue`、POSIX mq |
| **event bus** | 高 | 多订阅、类型化事件 |

**环形缓冲模块边界**（utils 层）：

```c
/* ringbuf.h — 无 driver 依赖 */
typedef struct ringbuf ringbuf_t;
ringbuf_t *ringbuf_create(size_t cap);
err_t ringbuf_push(ringbuf_t *rb, uint8_t byte);
err_t ringbuf_pop(ringbuf_t *rb, uint8_t *out);
```

```c
/* uart_isr: 仅 push */
void UART_IRQHandler(void)
{
    uint8_t b = UART->DR;
    ringbuf_push(g_rx_rb, b);  /* ISR 内极简 */
}

/* main loop: pop + 解析 */
while (ringbuf_pop(g_rx_rb, &b) == ERR_OK)
    parser_feed(b);
```

**消息 struct 设计**：

```c
typedef enum { MSG_SAMPLE, MSG_ALARM, MSG_SHUTDOWN } msg_type_t;

typedef struct {
    msg_type_t type;
    union {
        float temp;
        int   code;
    } u;
} msg_t;
```

- 队列元素固定大小 → 无动态分配，适合 MCU
- 模块间只传递 **消息副本或指针**（所有权文档化）

**与回调结合**：

```
ISR → queue_push(msg) → worker 任务 pop → 调 app 注册回调
```

- 回调仍在任务上下文执行，可 LOG、malloc
- **9.7.2** 同步回调的异步升级版

**模块化要点**：

| 原则 | 做法 |
|------|------|
| 队列归属 | 明确 owner 模块 create/destroy |
| 接口窄 | `post_event()` 隐藏队列实现 |
| 背压 | 队列满返回 `ERR_BUSY`，勿静默丢 |
| 无全局 | 队列句柄经 `init` 注入，非 extern |

**Linux 内核对照**：

- `workqueue`：中断下半部
- `kfifo`：内核环形缓冲
- 用户态模块化同样：ISR 等价物 = signal/epoll 回调
