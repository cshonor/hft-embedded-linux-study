# 9.7 模块间通信

## 本节讲什么

模块划清边界后，仍需**交换数据与控制**。本节总览三种主流方式——**全局变量**、**回调**、**异步消息**——的耦合度与适用场景，给出选型矩阵，并指向 **9.7.1–9.7.3** 的分节详解与 **demo06_callback** 实践。

## 要点

**通信方式光谱**：

```
强耦合 ◄────────────────────────────────────► 弱耦合
  全局变量        回调/ops         消息队列/事件总线
  (9.7.1)          (9.7.2)              (9.7.3)
```

| 方式 | 耦合 | 实时性 | 可测性 | 典型场景 |
|------|------|--------|--------|----------|
| **全局变量** | 高 | 最高 | 差 | 快速原型（不推荐量产） |
| **回调/ops** | 中 | 高 | 好（可 mock） | 驱动框架、中断通知 |
| **异步队列** | 低 | 中 | 好 | 任务调度、ISR→线程 |

**选型决策**：

```
数据是否共享 mutable 状态？
  ├─ 是 → 能否收进单一模块 static？
  │         ├─ 是 → 模块 API 读写（首选）
  │         └─ 否 → 回调或消息，避免 extern 全局
  └─ 否 → 函数调用传参 / const 配置

是否跨中断/线程边界？
  ├─ 是 → 9.7.3 异步 + 队列
  └─ 否 → 9.7.2 同步回调或直接调用

是否需要运行时替换实现？
  └─ 是 → ops 表（ch08）或 weak（demo04_weak）
```

**与解耦主线对照**（**9.5**）：

| 需求 | 推荐 |
|------|------|
| driver → app 上报数据 | 注册回调 `on_sample()` |
| app → driver 发命令 | 公开 API `sensor_start()` |
| 多订阅者 | 事件总线或链表挂回调 |
| 统一错误语义 | `err_t` 返回值（**demo05**） |

**反模式：万能全局 struct**

```c
/* ❌ system_state.h — 人人读写 */
typedef struct {
    int uart_ok;
    int mqtt_connected;
    float last_temp;
    /* ... 50 fields */
} g_sys_t;
extern g_sys_t g_sys;
```

- 任何模块改字段 → 全项目重编
- 竞态无锁（**ch10** 预告）
- 单元测试无法隔离
