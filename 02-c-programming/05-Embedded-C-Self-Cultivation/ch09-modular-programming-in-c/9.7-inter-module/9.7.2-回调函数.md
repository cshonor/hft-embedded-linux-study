# 9.7.2 回调函数

## 本节讲什么

**回调**是 C 实现**依赖倒置**与**事件通知**的主力：上层注册函数指针，下层在适当时机调用，双方仅共享函数签名，不共享 mutable 全局。本节讲解回调注册模式、与 **ch08 ops/vtable** 的关系，并对照 **demo06_callback** 的解耦示范。

## 要点

**基本模式**：

```c
/* sensor.h */
typedef void (*sample_cb_t)(float temp, void *ctx);

err_t sensor_register_cb(sample_cb_t cb, void *ctx);
err_t sensor_start(void);

/* sensor.c */
static sample_cb_t s_cb;
static void *s_ctx;

err_t sensor_register_cb(sample_cb_t cb, void *ctx)
{
    s_cb = cb;
    s_ctx = ctx;
    return ERR_OK;
}

static void sensor_fire_sample(float t)
{
    if (s_cb)
        s_cb(t, s_ctx);
}
```

| 要素 | 说明 |
|------|------|
| **typedef 函数指针** | 签名即契约，放公开 `.h` |
| **ctx 指针** | 避免回调内再查全局；传 `app_state_t *` |
| **NULL 检查** | 未注册时不调用 |
| **注册时机** | `main()` 初始化阶段，早于 `start` |

**回调 vs ops 表**（**ch08**）：

| | 回调 | ops 表 |
|---|------|--------|
| 方向 | 通常下层→上层通知 | 上层→下层调用 |
| 数量 | 常 1–N 个事件 | 一组操作（open/read/write） |
| 示例 | `on_rx_byte()` | `struct uart_ops` |
| 组合 | 驱动 ops + 应用回调 | 完整 HAL |

**demo06_callback 思路**：

```
driver: 收到数据 → 调用已注册 rx_cb(buf, len)
app:    注册 rx_cb = app_handle_rx，ctx = &app_state
```

- app 不 `#include` driver 内部头
- 测试时注册 mock_cb 断言调用次数与参数

**多订阅者**：

```c
struct cb_node {
    sample_cb_t fn;
    void *ctx;
    struct cb_node *next;
};
/* 链表挂多个回调 — 注意 ISR 上下文限制 */
```

- 简单项目单回调足够；复杂用 **9.7.3** 事件队列

**陷阱**：

| 陷阱 | 对策 |
|------|------|
| 回调中阻塞过久 | 仅置标志/入队，耗时工作放任务 |
| ISR 中调用户回调 | 只 `queue_work`，不 printf/malloc |
| 注销后仍被调 | `unregister` + 空指针或代际计数 |
| 循环依赖 | 回调 typedef 放独立 `event.h`，前向声明 |
