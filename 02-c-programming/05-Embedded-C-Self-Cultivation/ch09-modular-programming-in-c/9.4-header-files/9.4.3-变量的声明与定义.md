# 9.4.3 变量的声明与定义

## 本节讲什么

全局变量是模块间共享状态的双刃剑（**9.7.1**）。本节聚焦语法层面：**声明**告诉编译器「符号存在」，**定义**分配存储；头文件中只能出现 `extern` 声明，定义有且仅有一处——违反即链接失败或静默重复。

## 要点

**声明 vs 定义**：

```c
/* 声明 — 不分配存储，可多次出现 */
extern int g_sensor_count;
extern const char *app_version;

/* 定义 — 分配存储，整个程序只能一处 */
int g_sensor_count = 0;
const char *app_version = "1.0.0";
```

| 形式 | 存储分配 | 出现次数 |
|------|----------|----------|
| `extern T x;`（声明） | 否 | 可多次 |
| `T x = val;`（定义） | 是 | **全局仅一次** |
| `static T x;`（文件内定义） | 是 | 每 `.c` 一份，链接不可见 |

**头文件中的正确模式**：

```c
/* stats.h */
extern int g_packet_total;   /* 声明 */

/* stats.c */
int g_packet_total = 0;      /* 唯一定义 */
```

**错误模式**：

```c
/* ❌ config.h — 被多个 .c include */
int g_debug_level = 1;       /* 每个 .c 一份定义 → multiple definition */

/* ✅ 改为 */
extern int g_debug_level;    /* 头文件 */
/* config.c */ int g_debug_level = 1;
```

**const 与 extern**：

```c
/* header */
extern const int MAX_SENSORS;

/* one .c */
const int MAX_SENSORS = 16;
```

- `const` 全局默认 internal 链接（GCC）；跨文件需 `extern`
- 字符串字面量 `extern const char ver[]` 同理

**static 文件内全局**：

```c
static int s_uart_refcount;  /* 仅 mod_uart.c 可见，推荐用于模块私有状态 */
```

- 优先 `static` 模块内状态，而非 `g_` 全局（**9.7.1**）
- 若必须跨模块：集中在一个 `state.c`，API 读写，不暴露变量本身
