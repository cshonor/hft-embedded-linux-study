# 9.4.5 前向引用和前向声明

## 本节讲什么

当类型 A 仅需**指针**引用类型 B 时，不必 `#include "b.h"`——**前向声明**即可打断头文件依赖链，降低编译耦合与循环 include 风险。本节讲解不完整类型、前向声明语法及在模块接口中的典型用法。

## 要点

**前向声明语法**：

```c
struct uart_handle;          /* 不完整 struct */
typedef struct uart_handle uart_handle_t;

void app_process(uart_handle_t *uart);  /* 仅需指针，无需完整定义 */
```

| 需要完整类型 | 前向声明足够 |
|--------------|--------------|
| `sizeof(T)`、`T x;`（按值） | `T *p`、`T **` |
| 访问 `p->field` | 函数参数/返回值中的指针 |
| 继承嵌入 `struct base b;` | 指向 opaque 句柄的 API |

**打破循环依赖**：

```
/* 坏：a.h ↔ b.h 互 include */
a.h: #include "b.h"  →  struct B * 其实只需前向声明
b.h: #include "a.h"
```

```
/* 好 */
a.h:
    struct B;
    void a_use_b(struct B *b);

b.h:
    #include "a.h"   /* 单向 */
    struct B { ... };
```

**回调与 ops 表**（衔接 **ch08**、**9.7.2**）：

```c
/* event.h */
struct event;   /* 前向声明 */
typedef void (*event_cb_t)(struct event *e, void *ctx);

void event_register(event_cb_t cb, void *ctx);
```

- 回调签名里用指针 → 头文件无需 event 完整定义
- 实现文件 `#include "event_internal.h"` 访问字段

**不完整类型的限制**：

```c
struct foo;
struct foo x;        /* ❌ 编译错误：大小未知 */
struct foo *p;       /* ✅ */
void f(struct foo);  /* ❌ 按值传参需要完整类型 */
```

**typedef 前向声明**：

```c
typedef struct uart_handle uart_handle_t;  /* 头文件 */

/* .c 中 */
struct uart_handle {
    int fd;
};
```
