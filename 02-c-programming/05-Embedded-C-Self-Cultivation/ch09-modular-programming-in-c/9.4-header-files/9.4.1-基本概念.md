# 9.4.1 基本概念

## 本节讲什么

头文件（`.h`）通过 `#include` 被**文本插入**到源文件中，形成**翻译单元**——编译器每次只看到一个 `.c` 及其展开后的内容。本节澄清预处理机制、include guard、头文件与源文件的职责分工，为后续声明/定义规则打基础。

## 要点

**`#include` 本质**：

```c
#include "mod_uart.h"   /* 先搜当前目录，再 -I 路径 */
#include <stdio.h>      /* 仅搜系统/ -I 标准路径 */
```

- 预处理是**纯文本替换**，不是链接
- 同一 guard 保护的头在同一翻译单元只展开一次

**include guard 标准写法**：

```c
#ifndef MOD_UART_H
#define MOD_UART_H

/* 声明区 */

#endif /* MOD_UART_H */
```

| 方式 | 优点 | 注意 |
|------|------|------|
| `#ifndef`/`#define` | 可移植、内核标准 | guard 名全局唯一 |
| `#pragma once` | 简洁 | 极少数老编译器；内核较少用 |

**头文件 vs 源文件**：

| 文件 | 放什么 | 不放什么 |
|------|--------|----------|
| **`.h`** | 函数声明、typedef、`enum`、宏、`extern` 变量声明 | 非 inline 函数体、变量定义 |
| **`.c`** | 函数定义、变量定义、`static` 实现 | 通常不被其他模块 include |

**翻译单元与符号**：

```
main.c  +  #include 展开  →  编译  →  main.o
mod_uart.c                  →  编译  →  mod_uart.o
                                    ↘
                              链接 → app.elf
```

- 每个 `.c` **独立编译**；类型不匹配在编译期报错，符号缺失在链接期报错
- 头文件是多个 `.c` 之间的**类型契约**，必须一致（**9.4.6**）

**头文件注释模板**：

```c
/**
 * @file mod_uart.h
 * @brief UART 驱动公开 API
 * @note 线程不安全；多线程请外部加锁
 */
```
