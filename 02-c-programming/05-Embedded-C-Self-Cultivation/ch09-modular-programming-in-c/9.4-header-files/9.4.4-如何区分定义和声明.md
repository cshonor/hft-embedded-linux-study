# 9.4.4 如何区分定义和声明

## 本节讲什么

链接错误往往源于「以为写了声明，实际写了定义」。本节给出函数、变量、类型、宏的**一眼辨识规则**与常见混淆场景，便于 code review 和排查 `multiple definition`。

## 要点

**函数**：

| 代码 | 性质 |
|------|------|
| `void foo(int x);` | **声明** |
| `void foo(int x) { ... }` | **定义** |
| `static void bar(void);` | 声明 |
| `static void bar(void) { }` | 定义（internal 链接） |
| `inline void baz(void) { }` | 定义（inline 规则特殊，**9.4.9**） |

- 头文件放**声明**；**至多一个** `.c` 放带函数体的定义
- `static inline` 在头文件 → 每翻译单元一份，通常 OK

**变量**：

| 代码 | 性质 |
|------|------|
| `extern int n;` | 声明 |
| `int n;` / `int n = 0;` | 定义 |
| `static int s;` | 定义（文件作用域） |

**类型**：

| 代码 | 性质 |
|------|------|
| `struct foo;` | 不完整类型声明（前向声明） |
| `struct foo { int x; };` | 类型定义 |
| `typedef struct foo foo_t;` |  typedef 定义 |

- 结构体**定义**在头文件是允许的（POD 数据包），但会暴露布局
- 不透明模块：`typedef struct foo foo_t;` 在 `.h`，`struct foo { ... };` 仅在 `.c`

**宏**：

```c
#define BUF_SIZE 256   /* 预处理替换，无链接概念 */
```

- 宏在头文件重复定义若值不同 → 编译警告/错误
- 复杂宏放 dedicated `config.h`

**快速决策树**：

```
有函数体 { } ？  → 定义（慎放头文件）
有初始化 = ？    → 变量定义（全局只能一处）
仅有 extern ？  → 声明
struct 有字段？ → 类型定义（考虑是否应移到 .c）
```

**典型混淆案例**：

```c
/* 看起来像声明，实为定义 — 若放头文件灾难 */
int buffer[1024];

/* 正确：头文件 */
extern int buffer[1024];
/* 一处 .c */
int buffer[1024];
```
