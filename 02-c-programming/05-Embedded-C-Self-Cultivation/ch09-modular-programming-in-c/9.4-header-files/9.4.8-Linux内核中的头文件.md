# 9.4.8 Linux内核中的头文件

## 本节讲什么

Linux 内核是 C 模块化编程的**超大规模样板**：数万头文件、严格分层、`headers_install` 导出用户态 API。本节梳理内核 `include/` 目录组织、驱动如何 include、与用户态模块化的对照，帮你从内核源码阅读中提炼可复用的头文件纪律。

## 要点

**内核 include 树（简化）**：

```
include/
├── linux/          # 内核核心 API：list、device、file_operations
├── uapi/           # 用户态可见（系统调用、ioctl 常量）→ headers_install
├── asm-generic/    # 架构无关汇编/内存屏障抽象
├── net/            # 网络子系统
└── ...

drivers/tty/serial/imx.c   # 驱动实现，不 install
```

| 目录 | 谁 include | 是否 export 到用户态 |
|------|------------|---------------------|
| `include/linux/` | 内核各子系统 | 部分经 uapi 镜像 |
| `include/uapi/` | 内核 + glibc | 是 |
| `arch/xxx/include/` | 架构相关 | 部分 |
| 驱动目录内 `.h` | 仅该驱动 | 否 |

**内核头文件纪律**：

1. **API 头 vs 内部头**：`linux/serial_core.h`（框架）vs 驱动内 `static` 辅助
2. **`include/linux/` 不放实现**：函数声明 + `static inline` 小 helper（**9.4.9**）
3. **前向声明减依赖**：`struct device;` 避免 include 整棵设备树
4. **`EXPORT_SYMBOL`**：模块导出符号，等价于用户态 `.a` 公开 API

**与用户态模块化对照**：

| 内核 | 用户态嵌入式 |
|------|--------------|
| `include/linux/mod_uart.h` | `include/mod_uart.h` |
| `drivers/.../imx.c` | `driver/mod_uart/mod_uart.c` |
| `Kconfig CONFIG_SERIAL_IMX` | `config.h CONFIG_UART` |
| `obj-$(CONFIG_...) += imx.o` | Makefile 条件编译 |
| `headers_install` | `make install` → `out/include` |

**读内核驱动的 include 习惯**：

```c
#include <linux/module.h>
#include <linux/platform_device.h>
#include <linux/serial_core.h>   /* 抽象层 */
/* 不包含其他 vendor 驱动私有头 */
```

- 驱动通过 **platform/bus 框架**（**ch08 8.3.3–8.3.4**）注册，不互相 include
- 硬件寄存器定义通常在驱动 `.c` 或同目录 `regs.h`，不进 `include/linux/`

**hooks 与扩展点**：

- 内核用 **ops 表** + **weak default** + **Kprobe/tracepoint** 做扩展
- 用户态可借鉴：ops 接口（**ch08**）+ weak platform（**demo04_weak**）+ 回调（**9.7.2**）
