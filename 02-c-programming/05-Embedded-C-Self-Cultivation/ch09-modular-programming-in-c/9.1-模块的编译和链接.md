# 9.1 模块的编译和链接

## 本节讲什么

C 语言没有语言级「模块」关键字，**一个 `.c` 文件 + 对应 `.h` 头文件** 就是工程中最常见的模块单元。本节从编译链接流程出发，说明符号如何从源文件变成可执行文件，以及多模块工程为何必须理解 **编译单元、符号可见性、增量构建**——这是 **9.2 目录划分**、**9.3 封装**、**demo02_make** 的理论基础。

## 要点

**四步流水线**：

```
源文件 (.c)  →  预处理  →  编译  →  目标文件 (.o)  →  链接  →  可执行文件 / 静态库 (.a)
```

| 阶段 | 输入 | 输出 | 关键动作 |
|------|------|------|----------|
| **预处理** | `.c` + `#include` | 展开后的翻译单元 | 宏替换、条件编译、头文件展开 |
| **编译** | 单个翻译单元 | `.o` / `.obj` | 语法分析、生成机器码；**外部符号留待链接** |
| **汇编** | （通常与编译合并） | 目标文件 | — |
| **链接** | 多个 `.o` + 库 | 可执行文件 / `.a` | 符号解析、重定位、合并段 |

**符号与可见性**（衔接 **ch04 4.8 作用域**）：

| 修饰 | 链接属性 | 其他 `.c` 能否引用 |
|------|----------|-------------------|
| 无修饰全局函数/变量 | **external** | 能（需声明） |
| `static` 文件内 | **internal** | 不能 |
| `extern` 声明 | 引用外部定义 | 依赖别处 `.c` 的定义 |

- **单一定义规则（ODR）**：每个全局符号在整个程序中只能有一处定义；多处定义 → 链接错误 `multiple definition`
- **声明 vs 定义**：`.h` 放声明，**有且仅有一个** `.c` 放定义（**9.4.3–9.4.4**）

**多模块构建模式**（**demo01_minimal**、**demo02_make**）：

```
project/
├── Makefile          # 顶层：协调各子模块
├── app/main.c        # 入口，链接 driver + utils
├── driver/mod_uart.c # 编译 → driver/libmod_uart.a
└── utils/ringbuf.c   # 编译 → utils/libringbuf.a
```

典型 Makefile 片段：

```makefile
# 各子目录独立产出 .a
$(MAKE) -C driver
$(MAKE) -C utils
$(CC) app/main.o driver/libmod_uart.a utils/libringbuf.a -o app.elf
```

- **静态库 `.a`**：本质是 `.o` 归档；链接时按需抽取符号，未引用 `.o` 不进入最终镜像
- **增量构建**：只重编改动过的 `.c` → 对应 `.o` → 重链；头文件膨胀会拖慢编译（**9.4**）
- **`install` 目标**：将 `.a`、公开 `.h` 拷贝到 `out/include`、`out/lib`，供其他工程复用

**CMake 等价**（**demo03_cmake**）：

```cmake
add_library(mod_uart STATIC uart.c)
add_executable(app main.c)
target_link_libraries(app PRIVATE mod_uart utils)
target_include_directories(mod_uart PUBLIC include)
```

- **交叉编译**：`CMAKE_TOOLCHAIN_FILE` 指定工具链；各 `add_library` 产出目标平台 `.a`
- **依赖传递**：`PUBLIC` 头文件路径随库传播；`PRIVATE` 仅本目标可见

**常见链接错误速查**：

| 错误信息 | 原因 |
|----------|------|
| `undefined reference to 'foo'` | 声明了但未链接含定义的 `.o`/`.a` |
| `multiple definition of 'foo'` | 头文件中写了函数/变量定义，被多个 `.c` include |
| `relocation truncated to fit` | 大程序地址空间问题（嵌入式较少见） |
