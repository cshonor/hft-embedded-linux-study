# 9.3 一个模块的封装

## 本节讲什么

「模块」在代码层面的最小闭环是 **一对 `.h`/`.c` 文件**：头文件是**公开契约**，源文件是**私有实现**。本节系统讲解 include guard、不透明类型、`static` 内部符号、配对 API 与错误返回，与 **ch08 8.3.1** 的「类模拟」一脉相承，但更强调**工程边界**与 **demo01_minimal** 的可复制模板。

## 要点

**标准模块模板**：

```
mod_uart.h   — 对外 API：类型声明、函数原型、文档注释
mod_uart.c   — 实现：完整 struct、static 辅助函数、硬件细节
```

**include guard**（三选一，**9.4.1**）：

```c
#ifndef MOD_UART_H
#define MOD_UART_H
/* ... */
#endif
```

或 `#pragma once`（主流编译器支持，内核仍偏好 `#ifndef`）

**不透明类型**（衔接 **8.3.1**）：

```c
/* mod_uart.h — 调用方只见指针，不见字段 */
typedef struct uart_handle uart_handle_t;

uart_handle_t *uart_open(const char *port);
void uart_close(uart_handle_t *h);
int uart_write(uart_handle_t *h, const void *buf, size_t len);
```

```c
/* mod_uart.c — 完整定义仅在此 */
struct uart_handle {
    int fd;
    uint32_t baud;
    /* mutex、stats 等内部状态 */
};
```

| 暴露程度 | 头文件内容 | 适用 |
|----------|------------|------|
| **完全不透明** | 仅 `typedef` 指针 | 驱动、有复杂不变量 |
| **只读字段** | `const` 配置 struct | 初始化参数 |
| **全公开 struct** | 完整定义 | 纯数据 POD、协议包 |

**static 隐藏实现**：

```c
static int uart_hw_init(struct uart_handle *h) { /* ... */ }
static void uart_apply_baud(struct uart_handle *h) { /* ... */ }
```

- 文件级 `static` → 符号不进入 `.o` 的全局表，链接时不可见
- 模块内共享但不对外的声明 → `mod_uart_internal.h`，**禁止** app include

**API 设计 checklist**：

| 项 | 要求 |
|----|------|
| 生命周期 | `open/create` 与 `close/destroy` 配对 |
| 错误码 | 统一 `err_t`（**demo05** `ERR_*`），或返回负 errno |
| 参数 | 指针 + 长度；输出参数用 `out` 后缀或二级指针 |
| 线程 | 文档说明是否线程安全；锁封装在模块内 |
| 命名 | `模块_动作` 前缀，避免全局符号冲突 |

**与 ch08 封装对照**：

| ch08 概念 | 模块化落地 |
|-----------|------------|
| 不透明 `counter_t` | `uart_handle_t` |
| `create/destroy` | `uart_open/uart_close` |
| ops 多态 | `uart_ops` 仅在需替换实现时使用 |
| weak BSP | platform 层提供，driver 不感知板型 |

**反模式**：

- 头文件 `#include "hw_regs.h"` 暴露寄存器给 app
- `.h` 中定义全局变量 `int g_uart_debug = 1;` → 多 `.c` include 后链接失败
- 巨型 `utils.h` 包含 50 个无关函数声明
