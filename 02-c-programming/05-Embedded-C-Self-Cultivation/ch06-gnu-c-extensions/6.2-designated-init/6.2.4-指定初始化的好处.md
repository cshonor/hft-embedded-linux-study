# 6.2.4 指定初始化的好处

## 本节讲什么

指定初始化提升 **可读性、可维护性、Review 安全性**；在大型结构体与配置表中收益最大。对比 positional 初始化，字段增删时 merge conflict 更小。

## 要点

### Positional vs Designated

```c
struct timer_cfg { int ms; int enabled; const char *name; };

/* 差：顺序错 silent bug */
struct timer_cfg bad = { 1, 1000, "watchdog" };  /* ms 与 enabled 反了？ */

/* 好：自文档化 */
struct timer_cfg good = {
    .name    = "watchdog",
    .ms      = 1000,
    .enabled = 1,
};
```

| 收益 | 说明 |
|------|------|
| **自描述** | 读代码无需对照 struct 定义顺序 |
| **稀疏** | 数组/表只写非零项 |
| **演进** | 中间插入新成员不破坏旧初始化 |
| **Review** | diff 一眼看出改了哪个字段 |

```c
/* 内核风格：宏生成表项仍保留 designated */
#define DEVICE_ENTRY(_compat, _data)     { .compatible = _compat, .data = (void *)(_data) }
```

- **缺点**：C++ 早期不兼容；跨编译器写库时需 `#ifdef`
- **DPDK**：静态 `rte_eth_conf` 等同理；热路径结构仍倾向运行时赋值 + 少量 static default
