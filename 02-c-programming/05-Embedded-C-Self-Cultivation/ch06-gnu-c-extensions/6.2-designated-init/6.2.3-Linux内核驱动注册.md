# 6.2.3 Linux内核驱动注册

## 本节讲什么

内核驱动用 **静态常量结构体 + 指定初始化** 向子系统注册：字段多、版本演进快，指定初始化避免「位错列」Bug。典型：`file_operations`、`platform_driver`、`pci_driver`、`of_match_table`。

## 要点

```c
/* 简化示意 — 真实内核头文件更复杂 */
#include <linux/module.h>
#include <linux/fs.h>

static int demo_open(struct inode *inode, struct file *filp) { return 0; }
static ssize_t demo_read(struct file *filp, char __user *buf,
                         size_t count, loff_t *ppos) { return 0; }

static const struct file_operations demo_fops = {
    .owner          = THIS_MODULE,
    .open           = demo_open,
    .read           = demo_read,
    /* 未列出的成员为 NULL / 0 */
};

static int __init demo_init(void)
{
    /* register_chrdev(..., &demo_fops); */
    return 0;
}
module_init(demo_init);
```

```c
/* device tree match — of_device_id */
static const struct of_device_id demo_of_match[] = {
    { .compatible = "vendor,demo-device" },
    { .compatible = "vendor,demo-device-v2" },
    { /* sentinel */ },
};
MODULE_DEVICE_TABLE(of, demo_of_match);
```

- **`.owner = THIS_MODULE`**：模块引用计数
- 新增回调时只加一行，旧驱动不受影响
- U-Boot / DPDK：同样模式（`U_BOOT_DRIVER`、静态 ethdev ops）

```bash
# 在内核树中搜实例
grep -r '\.open\s*=' drivers/ --include='*.c' | head
```
