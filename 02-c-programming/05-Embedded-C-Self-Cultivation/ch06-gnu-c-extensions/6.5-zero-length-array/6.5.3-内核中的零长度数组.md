# 6.5.3 内核中的零长度数组

## 本节讲什么

内核大量使用 FAM / `[0]`：**sk_buff 尾部**、**netlink 属性**、**ioctl 结构**、**BPF map 值**。读 `include/linux/skbuff.h` 等头文件时识别「头 + 变长」模式。

## 要点

```c
/* 概念示意 — 非完整内核定义 */
struct nlattr {
    uint16_t nla_len;
    uint16_t nla_type;
    char     nla_data[0];
};

/* 分配 attr */
static struct nlattr *nla_alloc(uint16_t type, uint16_t dlen)
{
    struct nlattr *na = kmalloc(sizeof(*na) + dlen, GFP_KERNEL);
    if (na) {
        na->nla_type = type;
        na->nla_len  = sizeof(*na) + dlen;
    }
    return na;
}
```

```c
/* skb 线性数据 — 历史上有 headroom/data/tail 与 frags */
/* 变长体现在 data_len、frag_list、paged offset 等组合 */
```

- 内核 coding style 倾向 **`[]` FAM**（C99）而非 `[0]`，视树版本而定
- **用户态拷贝**：`copy_to_user` 需传递 **总长度** `sizeof(hdr)+payload`
- DPDK **mbuf** 用固定 headroom + dynfield 扩展，思想类似「头 + 可变数据区」
