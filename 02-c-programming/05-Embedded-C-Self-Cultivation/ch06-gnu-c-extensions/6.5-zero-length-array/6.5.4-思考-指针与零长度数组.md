# 6.5.4 思考：指针与零长度数组

## 本节讲什么

结构体末尾放 **`char *ptr`** 还是 **`char data[]`**？涉及 **一次 vs 两次分配**、**cache 局部性**、**序列化/on-wire 布局**、**ABI 稳定性**。本节对比决策树。

## 要点

| 方案 | 优点 | 缺点 |
|------|------|------|
| **FAM / [0]** | 一次 alloc；连续内存 |  realloc 变长需 memcpy 新块 |
| **指针成员** | 头大小固定；payload 可独立 realloc | 两次 alloc；指针占 4/8 字节 |
| **固定数组 `char buf[N]`** | 最简单 | 浪费或 N 不够 |

```c
/* 指针版 — 适合 payload 生命周期独立 */
struct msg_ptr {
    uint32_t len;
    uint8_t *data;   /* 另 malloc(len) */
};

/* FAM — 适合 netlink/ioctl 一次拷贝到用户态 */
struct msg_fam {
    uint32_t len;
    uint8_t  data[0];
};
```

```c
/*  wire 格式常要 FAM 思想：头后面紧跟字节 */
/*  不能直接把含指针的结构体 send() 出去 */
```

- **内核 copy_to_user**：FAM 结构 **线性** 更好算长度
- **DPDK**：mbuf 数据区在 pool 中，用 offset 而非 FAM，避免频繁 realloc
- **realloc FAM 对象**：`new = realloc(old, sizeof(*h)+newlen)` 可能移动块，注意悬挂指针
