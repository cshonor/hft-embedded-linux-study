# 6.4.3 Linux内核中的container_of宏

## 本节讲什么

**container_of** 实现「从成员指针得到结构体指针」，支撑 `list_for_each_entry`、`device` 嵌套 `platform_device` 等模式。定义在 `include/linux/container_of.h`（或经 `kernel.h` 间接包含）。

## 要点

```c
#include <stddef.h>

#define container_of(ptr, type, member) ({              \
    void *__mptr = (void *)(ptr);                         \
    ((type *)((char *)__mptr - offsetof(type, member))); \
})
```

使用：

```c
struct task t;
struct list_node *n = &t.link;
struct task *tp = container_of(n, struct task, link);
/* tp == &t */
```

```c
/* 内核链表遍历宏（简化） */
#define list_for_each_entry(pos, head, member)              \
    for (pos = list_first_entry(head, typeof(*pos), member); \
         &pos->member != (head);                            \
         pos = list_entry(pos->member.next, typeof(*pos), member))
```

- **`typeof(*pos)`**：pos 指针类型由编译器从变量推断
- **空指针 / 野指针** 传入 UB；使用前需保证 `ptr` 确实来自某 `type` 实例的成员
- U-Boot、部分 DPDK 内部 list 宏同构
