# 6.4.4 container_of宏实现分析

## 本节讲什么

逐步展开 **container_of** 的指针运算与类型转换，理解为何用 **`char *`** 做字节级减法、**`offsetof`** 如何取得成员偏移，以及内核版本中的 **const 检查** trick。

## 要点

### 布局

```
struct task 首地址 ──► [ id | name[16] | link | ... ]
                              ▲
                              └── ptr 指向 link
offset = offsetof(struct task, link)
task_addr = (char *)ptr - offset
```

```c
/* 标准 C 版（无 typeof） */
#define container_of(ptr, type, member) \
    ((type *)((char *)(ptr) - offsetof(type, member)))
```

```c
/* 内核加强：const 成员指针检查（思路） */
#define container_of(ptr, type, member) ({                      \
    void *__mptr = (void *)(ptr);                               \
    static_assert(__builtin_types_compatible_p(                 \
        typeof(((type *)0)->member), typeof(*(ptr))),           \
        "pointer mismatch");                                   \
    ((type *)((char *)__mptr - offsetof(type, member)));       \
})
```

| 步骤 | 说明 |
|------|------|
| `(void *)(ptr)` | 抹除 const 便于中间变量（内核有 const 版本宏） |
| `(char *)` 减法 | 按 **字节** 回退，与对齐无关 |
| 转 `(type *)` | 得到结构体首址 |

```c
/* 错误用法 */
struct task *bad = container_of(NULL, struct task, link);  /* UB */
```

- **`list_entry`** = `container_of` 别名
- DPDK `RTE_CONTAINER` 宏（若存在）同理，读源码时对照 **demo** 与 **rte_ring** 结构
