# 6.1.2 C语言标准的内容

## 本节讲什么

一份 C 标准通常包含：**语言语法与语义**、**诊断规则**、**标准库**、**可移植性限制** 与 **附录**（常见缺陷、库概要）。理解章节划分有助于查「某特性从哪版标准开始合法」。

## 要点

| 部分 | 内容 |
|------|------|
| **语法/约束** | 声明、表达式、语句、翻译阶段 |
| **语义** | 对象存储期、链接、类型规则 |
| **库** | `<stdio.h>`、`<string.h>`、`<stdint.h>` 等 |
| **附录** | 未指定/未定义/实现定义行为清单 |

**翻译阶段（简化）**：

1. 三字符组 → 源字符
2. 拼接行、注释删除
3. **预处理**（`#include`、`#define`）
4. 字符常量转义
5. 翻译单元
6. 链接

```bash
gcc -E hello.c | less          # 只看预处理结果
gcc -save-temps -c hello.c     # 保留 .i 中间文件
```

- **实现定义（implementation-defined）**：编译器文档必须说明（如 `int` 位宽）
- **未指定（unspecified）**：合法但顺序等由实现决定
