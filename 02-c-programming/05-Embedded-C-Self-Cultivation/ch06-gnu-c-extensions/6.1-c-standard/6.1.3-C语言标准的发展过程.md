# 6.1.3 C语言标准的发展过程

## 本节讲什么

C 从 K&R 方言演进为 **C89 → C99 → C11 → C17 → C23**。每版引入新特性；旧代码常混用「编译器扩展 + 后补进标准的特性」。内核与 DPDK 代码时间跨度大，同一树内可能并存多种写法。

## 要点

| 版本 | 年份 | 代表性新特性 |
|------|------|----------------|
| **C89/C90** | 1989/1990 | 原型、`const`/`volatile` |
| **C99** | 1999 | `//` 注释、`stdint.h`、inline、变长数组 |
| **C11** | 2011 | `_Generic`、`_Atomic`、线程、Unicode |
| **C17** | 2018 | 缺陷修复，少新特性 |
| **C23** | 2024 | `typeof`、 `#embed`、attributes 标准化等 |

**GNU 扩展 vs 标准吸收**：

- **指定初始化** `[i]=`、`{.m=}`：C99 进标准，GNU 更早有
- **灵活数组成员** `[]`：C99 标准；GNU **零长度数组** `[0]` 等价用法
- **语句表达式** `({ })`：仍为 GNU 扩展
- **`__attribute__`**：C23 有 `_Attribute`，语法不同

```bash
gcc -std=c89 -pedantic -c old.c
gcc -std=c11 -c modern.c
```

- Linux 内核 6.x： predominantly **GNU C + gnu11**
- 新项目若需 MSVC：避免语句表达式与 `__attribute__`，用宏/portability 层
