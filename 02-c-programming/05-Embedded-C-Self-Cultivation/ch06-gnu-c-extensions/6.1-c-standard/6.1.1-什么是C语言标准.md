# 6.1.1 什么是C语言标准

## 本节讲什么

**C 语言标准** 是由 ISO/IEC JTC1 SC22 WG14 维护的规范文档，定义合法程序、库接口与实现约束。它不等于「K&R 书」或「某个编译器手册」——同一源码在不同标准模式下可能合法或非法。

## 要点

- **实现（implementation）**：编译器 + 标准库 + 运行环境
- **严格符合（strictly conforming）**：仅使用标准语法与标准头，无未定义扩展依赖
- **Hosted vs Freestanding**：带 `main` 和用户态 libc 为 hosted；内核/Bootloader 常为 freestanding（仅 `<stddef.h>` 等子集）

```c
/* C99 起合法 */
#include <stdint.h>
int main(void) {
    uint32_t x = 0;
    return (int)x;
}
```

```bash
man std    # 部分系统有 standards(7)
gcc -dM -E - < /dev/null | grep __STDC_VERSION__
# __STDC_VERSION__ 199901L → C99；201112L → C11
```

- **未定义行为（UB）**：标准允许编译器任意假设；优化可能「删掉」看似会执行的代码
- 嵌入式读源码：看到 GNU 关键字先查是否被 `-std=gnu*` 打开
