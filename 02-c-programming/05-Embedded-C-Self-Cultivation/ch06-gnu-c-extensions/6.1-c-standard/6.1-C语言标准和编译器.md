# 6.1 C语言标准和编译器

## 本节讲什么

C 语言由 **ISO/IEC 标准** 定义语法与语义，由 **编译器（GCC/Clang）** 实现并常附带 **GNU 扩展**。读 Linux 内核、U-Boot、DPDK 源码前，必须分清「标准 C99/C11」与「`-std=gnu11` 实际启用的扩展」。本章后续所有语法均建立在此认知之上。

## 要点

| 角色 | 说明 |
|------|------|
| **标准** | C89/C99/C11/C17/C23，规定可移植子集 |
| **编译器** | 翻译源码 → 目标文件；`-std=` 选择方言 |
| **扩展** | `__attribute__`、语句表达式、`typeof` 等，非 ISO 必选 |

```bash
gcc --version
gcc -std=c99   -pedantic -Wall -c foo.c   # 严格标准，扩展报警
gcc -std=gnu11 -Wall        -c foo.c     # Linux 内核/DPDK 常用
```

- 内核 Makefile 通常 **` -std=gnu11`**（或 gnu89），并依赖大量扩展
- 移植到 MSVC/IAR 时需逐项替换扩展写法
