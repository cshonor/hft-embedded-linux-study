# 6.1.4 编译器对C语言标准的支持

## 本节讲什么

编译器通过 **`-std=`** 选择语言方言；**默认标准** 随 GCC 主版本变化。`-pedantic` 关闭扩展或对其报警；**`-Wall`** 与标准合规是两回事。交叉编译 ARM 内核时，Kbuild 已替你选好 flags。

## 要点

```bash
# 查看默认与宏
gcc -dM -E -x c /dev/null | grep -E '__STDC_VERSION__|__GNUC__'

# 常用组合
gcc -std=c11   -pedantic -Wall -Wextra -c app.c
gcc -std=gnu11 -Wall -Werror          -c driver.c   # 允许 GNU 扩展
clang -std=gnu11 -Weverything -c driver.c
```

| 选项 | 效果 |
|------|------|
| `-std=c99` / `c11` / `c17` | ISO 模式，扩展受限 |
| `-std=gnu99` / `gnu11` | ISO + GNU 扩展（**默认常为此类**） |
| `-pedantic` | 严格 ISO，GNU 扩展产生 warning |
| `-Wpedantic` | 同上（GCC 4.8+） |

```bash
# 某特性是否可用：试编译
echo 'int x[0];' | gcc -std=c99 -x c -c - 2>&1
echo 'int x[0];' | gcc -std=gnu11 -x c -c - 2>&1
```

- **Clang** 在内核/DPDK 构建中 increasingly 使用，仍目标 gnu 方言
- **IAR/Keil**：部分扩展不可用，需 `#ifdef` 隔离
