# 6.11.5 内建函数：__builtin_expect(exp,c)

## 本节讲什么

**`__builtin_expect(long exp, long c)`** 告知编译器 **exp 最可能等于 c**，用于 **基本块布局**（hot path fall-through）。**不改变** exp 的语义与求值次数；主要影响 **代码排列**，而非直接「加速 CPU 分支预测」。

## 要点

```c
#define likely(x)   __builtin_expect(!!(x), 1)
#define unlikely(x) __builtin_expect(!!(x), 0)

if (likely(ptr != NULL)) {
    use(ptr);
} else {
    return -EINVAL;
}
```

```c
/* 常见误解：不会改变调用次数 */
int v = unlikely expensive();  /* expensive() 仍调用一次 */
```

| 项 | 说明 |
|----|------|
| 参数 | `exp` 任意整型；`c` 为期望值 |
| 效果 | 重组 CFG，hot label 布局 |
| 替代 | **PGO**（Profile-Guided Optimization） |

```bash
gcc -O2 -S -o expect.s expect.c
# 对比加 likely 前后基本块顺序
objdump -d expect | less
```

- **Case range** `case 1 ... 5:`（GNU）与 expect 同属编译器扩展工具箱
- DPDK **`rte_likely`/`rte_unlikely`** 在 `rte_branch_prediction.h`
