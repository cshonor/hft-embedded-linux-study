# 6.6.3 U-boot镜像自复制分析

## 本节讲什么

U-Boot 等 Bootloader 常链接在 **Flash（XIP）**，启动时需把 **代码/数据复制到 RAM** 再执行。通过 **链接脚本符号**（`__rel_dyn_start` 等）与 **section 布局** 定位复制区间；理解 section 属性是读 `arch/arm/lib/crt0.S` / `relocate.S` 的前提。

## 要点

### 典型流程

```
1. CPU 复位，PC 指向 Flash 中 _start
2. 初始化最小 C 环境（栈、时钟）
3. 若 CONFIG_SYS_TEXT_BASE != 链接地址：
   memcpy 代码段、清零 BSS、Fixup 重定位表
4. 跳转到 RAM 中 continue_in_ram
5. board_init_f / board_init_r ...
```

```c
/* 概念：链接脚本导出边界符号 */
extern char __text_start[], __text_end[];
extern char __data_start[], __data_end[];
extern char __bss_start[], __bss_end[];

void relocate_code(void *ram_start)
{
    size_t text_len = __text_end - __text_start;
    memcpy(ram_start, __text_start, text_len);
    /* 更新 GOT、清理 cache 等 */
}
```

```bash
# U-Boot 构建产物
arm-linux-objdump -h u-boot | head -20
arm-linux-nm u-boot | grep __image_copy
```

| 概念 | 说明 |
|------|------|
| **LMA** | Load Memory Address（Flash 中位置） |
| **VMA** | Virtual Memory Address（运行地址） |
| **relocate** | LMA≠VMA 时复制 |

- **`.rel.dyn` / `.rela.dyn`**：动态重定位项，复制后需 fixup
- Linux 内核 ARM **`__pa`/`__va`** 与 early paging 是下一阶段
- DPDK **EAL** 在用户态，不涉及 XIP，但 **hugepage 映射** 同属「把代码/数据放到合适内存」
