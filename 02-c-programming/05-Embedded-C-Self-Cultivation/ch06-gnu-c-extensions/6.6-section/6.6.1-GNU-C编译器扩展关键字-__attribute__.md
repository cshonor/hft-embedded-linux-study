# 6.6.1 GNU C编译器扩展关键字：__attribute__

## 本节讲什么

**`__attribute__((...))`** 是 GCC 附着于 **声明** 的编译指示，不改变 C 类型系统，却影响 **布局、链接、优化、诊断**。可写在类型、变量、函数前后；内核用宏封装成 `__packed`、`__init` 等。

## 要点

### 语法形式

```c
int x __attribute__((aligned(4)));
__attribute__((noreturn)) void panic(void);
void foo(void) __attribute__((section(".text.foo")));
```

```c
/* 多个属性 */
__attribute__((section(".init"), unused, noinline))
static void early_init(void) { }
```

| 常见属性 | 作用 |
|----------|------|
| `packed` | 取消结构体填充 |
| `aligned(n)` | 指定对齐 |
| `section(s)` | 自定义段名 |
| `weak` | 弱符号 |
| `format(printf,n,m)` | 格式检查 |
| `noreturn` | 不返回 |
| `constructor`/`destructor` |  main 前后执行 |
| `unused` | 抑制未使用警告 |
| `deprecated` | 调用时警告 |
| `noinline`/`always_inline` | 内联控制 |

```c
/* 内核封装示例 */
#define __packed __attribute__((packed))
#define __init   __attribute__((section(".init.text")))
#define __exit   __attribute__((section(".exit.text")))
```

```bash
gcc -fdump-tree-optimized -c attr.c   # 观察优化
```

- C23 **`[[attributes]]`** 语法不同，可移植性更好
- Clang 兼容大部分 GNU attribute
