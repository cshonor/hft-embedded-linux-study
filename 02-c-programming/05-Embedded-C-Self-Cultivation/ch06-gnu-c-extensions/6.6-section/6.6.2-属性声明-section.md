# 6.6.2 属性声明：section

## 本节讲什么

**section 属性** 将符号放到指定 **ELF section**；链接器脚本把 `.init.text` 等在启动后 **释放或丢弃**，把 `.boot` 放到 RAM 运行地址。Bootloader 与内核启动代码核心技巧。

## 要点

```c
#include <stdio.h>

static int early_data __attribute__((section(".early.data"))) = 1;

__attribute__((section(".init.text")))
void init_once(void)
{
    printf("init_once\n");
}

__attribute__((section(".boot.text")))
void boot_copy_src(void)
{
    /* 链接脚本可能将此段 LMA 与 VMA 不同 */
}
```

```bash
gcc -c section_demo.c -o section_demo.o
objdump -t section_demo.o | grep -E 'early|init|boot'
readelf -S section_demo.o
```

```ld
/* 链接脚本片段示意 */
.early.data : { *(.early.data) } > SRAM
.init.text  : { *(.init.text)  } > DRAM
```

- **内核 `__init`**：启动完成后 `free_initmem()` 回收 `.init.*`
- **模块 `__init`/`__exit`**：加载/卸载段
- **demo02**：自定义 section 变量，**nm/objdump** 验证

```c
/* 函数指针表放入固定段 — 类似 U-Boot linker list */
struct driver __attribute__((section(".drivers")))
drv_instance = { .name = "demo" };
```
