# 6.7.2 结构体的对齐

## 本节讲什么

结构体布局 = **成员顺序 + 各成员对齐要求 + 尾部 padding**。`sizeof` 可能大于成员 size 之和。读 **wire 协议** 与 **硬件寄存器块** 时必须显式控制；否则 `#pragma pack` / **`packed`** 与 **`aligned`** 混用。

## 要点

```c
struct default_layout {
    char  a;   /* offset 0 */
    /* 3 bytes padding */
    int   b;   /* offset 4 */
    char  c;   /* offset 8 */
    /* 3 bytes tail padding → sizeof = 12 */
};
```

```c
struct hw_reg {
    uint32_t ctrl;
    uint32_t status;
} __attribute__((aligned(4)));
```

规则（简化）：

1. 每个成员 offset % align(member) == 0
2. 结构体 align = max(成员 align)，size 补齐到 align 倍数
3. **`packed`** 强制 align 1（**6.7.4**）

```bash
gcc -fdump-lang-class -c layout.c 2>&1 | less   # 或 pahole
pahole -C layout.o 2>/dev/null || echo "install pahole for detail"
```

```c
/* offsetof 验证 */
#include <stddef.h>
_Static_assert(offsetof(struct default_layout, b) == 4, "");
```

- DPDK **`RTE_CACHE_LINE_SIZE`** 用于 pad 结构体防 false sharing
- 内核 **`struct`** 与 **uapi** 头对 userspace 布局稳定
