# 6.7.4 属性声明：packed

## 本节讲什么

**`__attribute__((packed))`** 取消结构体成员间 **padding**，使 `sizeof` 等于成员紧凑之和（仍可能有位域规则）。用于 **网络协议头、Flash 配置块、寄存器映射**；代价是 **未对齐访问** 与 **性能**。

## 要点

```c
struct eth_hdr_packed {
    uint8_t  dst[6];
    uint8_t  src[6];
    uint16_t ethertype;
} __attribute__((packed));

_Static_assert(sizeof(struct eth_hdr_packed) == 14, "");
```

```c
/* 仅 packed 单个成员 — 较少用 */
struct mix {
    int a;
    short b __attribute__((packed));
};
```

| 风险 | 说明 |
|------|------|
| 未对齐 load/store | ARMv7 需 `__attribute__((aligned(1)))` 或 memcpy |
| 原子性 | packed 内 int 可能跨 cache line |
| ABI | 勿跨模块混用 packed/非 packed 同一 layout |

```bash
gcc -std=gnu11 -o demo01 demo01_packed.c
./demo01
# 对比 packed vs 默认 struct 的 sizeof 与 offsetof
```

```c
/* 安全读 unaligned — 内核/ DPDK 常用 */
uint16_t get_be16(const void *p)
{
    uint16_t v;
    memcpy(&v, p, sizeof(v));
    return rte_be_to_cpu_16(v);
}
```

- **demo01** 直观打印布局
- 能不用 packed 则优先 **手动 padding + static assert**
