# 6.3.4 内核中的语句表达式

## 本节讲什么

Linux 内核头文件大量宏基于语句表达式：`min`/`max`、`clamp`、`container_of`、位域操作、`READ_ONCE` 等。读内核或写 out-of-tree 模块必须能 **展开阅读** 这些宏。

## 要点

```c
/* include/linux/minmax.h 思路 */
#define min(x, y) ({                    \
    typeof(x) min1 = (x);               \
    typeof(y) min2 = (y);               \
    (void) (&min1 == &min2);            /* 类型不一致则编译报错 */ \
    min1 < min2 ? min1 : min2;          \
})
```

```c
/* container_of — 见 6.4.4 */
#define container_of(ptr, type, member) ({          \
    void *__mptr = (void *)(ptr);                   \
    ((type *)(__mptr - offsetof(type, member)));     \
})
```

```c
/* 位操作：设/清/测 */
#define BIT(nr) (1UL << (nr))
#define GENMASK(h, l) (/* ... */)
```

- **`(void) (&min1 == &min2)`**： ingenious 编译期类型检查 trick
- **`READ_ONCE`/`WRITE_ONCE`**：含 `volatile` 与 barrier，包装在宏内
- DPDK **`RTE_MIN`/`RTE_MAX`** 类似，定义在 `rte_common.h`

```bash
# 查看预处理展开
gcc -E -I/path/to/kernel/headers test.c | grep -A5 'min('
```
