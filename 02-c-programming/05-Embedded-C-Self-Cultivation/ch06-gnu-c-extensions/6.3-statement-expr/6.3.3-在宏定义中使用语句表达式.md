# 6.3.3 在宏定义中使用语句表达式

## 本节讲什么

编写「像函数一样返回值」的宏时，语句表达式 + **`typeof`** 是标准套路：临时变量保存参数、只求值一次、避免 side effect。本节给出 **`min`/`max`** 与带 `if` 的安全宏模板。

## 要点

### 错误示范

```c
#define BAD_MAX(a, b) ((a) > (b) ? (a) : (b))
/* BAD_MAX(x++, y++)  → x/y 可能 ++ 两次 */
```

### 推荐写法

```c
#define MAX(a, b) ({                        \
    typeof(a) __max_a = (a);                \
    typeof(b) __max_b = (b);                \
    __max_a > __max_b ? __max_a : __max_b;  \
})
```

```c
/* 带分支的宏 */
#define CLAMP(val, lo, hi) ({               \
    typeof(val) __v = (val);                \
    typeof(lo)  __lo = (lo);                \
    typeof(hi)  __hi = (hi);                \
    __v < __lo ? __lo : (__v > __hi ? __hi : __v); \
})
```

| 技巧 | 目的 |
|------|------|
| `typeof` 临时 | 类型安全、单求值 |
| 唯一变量名 `__max_a` | 避免嵌套宏名冲突（内核用更复杂 token paste） |
| 外层 `({})` | 可用于 `if (MAX(a,b) > 0)` |

```c
/* 语句表达式 + GNU 条件编译 */
#define IS_ERR_OR_NULL(ptr) ({              \
    const void *__p = (ptr);                \
    !__p || (unsigned long)__p >= (unsigned long)-4096UL; \
})
```

- 若宏展开为 **语句** 而非表达式，用 `do { ... } while (0)`（无返回值）
