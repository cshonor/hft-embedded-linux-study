# 6.10.5 思考：内联函数为什么定义在头文件中

## 本节讲什么

**定义放头文件** 是为让 **每个编译单元** 都看到函数体，从而 **就地 inline**。若只在 `.c` 定义，其他 TU 只能 **extern 调用**（除非 LTO）。

## 要点

```c
/* helper.h */
static inline int add(int a, int b) { return a + b; }
```

| 模式 | 说明 |
|------|------|
| **`static inline` in .h** | 最常见 |
| **单 .c + 普通函数** | 无 inline（除非 LTO） |

- DPDK **`static __rte_always_inline`** in **`rte_*.h`**
