# 6.10.2 什么是内联函数

## 本节讲什么

**内联** = 编译器把函数体 **插入调用点**，像宏但 **有类型检查** 与 **单求值**。C99 **`inline` 关键字** 与 **`static inline`** 头文件模式是内核/DPDK 默认写法。

## 要点

```c
static inline uint32_t rol32(uint32_t x, int n)
{
    return (x << n) | (x >> (32 - n));
}
```

```c
/* C99 extern inline 模型（少用，易 ODR 踩坑） */
inline int helper(int x) { return x + 1; }
```

| 形式 | ODR / 链接 |
|------|------------|
| `static inline` in .h | 每 TU 副本，无符号冲突 |
| 普通函数 | 单一定义，`call` 指令 |
| 宏 | 文本替换，无类型 |

- **代码膨胀**：大函数 inline 增大 I-cache 压力
- **函数指针** 取址时通常 **必须** 产生 out-of-line 副本
- DPDK **`static __rte_always_inline`** 强制热路径展开

```bash
gcc -O2 -S inline.c -o inline.s
grep -c 'call' inline.s
```
