# 10.6 存储器接口与映射

## 本节讲什么

MCU/SoC 通过 **总线** 连接 **SRAM、Flash、外设寄存器**；CPU 看到的 **地址空间** 由 **存储器控制器与映射** 决定。本节总览 **10.6.1–10.6.3**，为 **10.7 I/O**、**10.8 寄存器**、**10.9 MMU** 提供硬件地址背景。

## 要点

**典型 SoC 地址 map**（示意）：

```
0x0000_0000  Flash / ROM（XIP 代码）
0x2000_0000  SRAM
0x4000_0000  AHB/APB 外设（**10.7.3**）
0x6000_0000  外部 SDRAM（若有）
0xE000_0000  Cortex-M 私有外设（NVIC, SysTick）
```

**存储器 vs 外设**：

| 类型 | 访问特性 |
|------|----------|
| **SRAM/DRAM** | 读写无 side effect，可 cache |
| **MMIO 寄存器** | 读清/写 1 清、不可 cache（或 strongly ordered） |
| **Flash** | 读像 RAM；写需擦除命令 |

**接口类型**（**10.6.1**）：

- **并行 NOR/NAND**、**SPI/QSPI Flash**、**eMMC/SD**、**DDR 控制器**

**映射方式**（**10.6.2**）：

- **Flat 映射**（MCU）：物理地址 = 程序使用的地址
- **MMU 映射**（Linux）：虚拟 → 物理（**10.9**）

**启动 implications**（**10.6.3**）：

- 复位从 **向量表** 所在 Flash 取 PC
- **链接脚本** 指定 `.text` 在 Flash、`.data` 初值在 Flash 运行时拷贝到 RAM

**与多任务关系**：

- 链接脚本划分 **任务栈、堆、TCB** 在 SRAM 区域
- DMA 缓冲区常需 **对齐、非 cache** 属性（Linux `dma_alloc_coherent`）
