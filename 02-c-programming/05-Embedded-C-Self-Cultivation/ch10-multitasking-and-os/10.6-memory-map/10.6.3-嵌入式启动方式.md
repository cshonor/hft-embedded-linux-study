# 10.6.3 嵌入式启动方式

## 本节讲什么

上电后 **谁执行第一条指令**、**代码从 Flash 还是 RAM 跑**、**如何跳到 Linux 或 RTOS**？本节对比 **XIP、RAM 执行、U-Boot 链式加载**，与 **10.5.3 rootfs**、**demo03 qemu-arm** 环境呼应。

## 要点

**Cortex-M 复位流程**：

```
1. PC = 向量表[1]（Reset_Handler）
2. 初始化 SP（向量表[0]）
3. startup：拷贝 .data、清零 .bss
4. SystemInit：时钟、可选 VTOR 重定位向量表
5. main() 或 os_start()
```

**XIP（eXecute In Place）**：

- `.text` 链接在 Flash 地址；CPU 直接从 Flash 取指
- `.data` 初值在 Flash，启动时 **拷贝到 RAM**
- 优点：省 RAM；缺点：Flash wait state、不能原地改代码

**Load to RAM**：

- Bootloader 把镜像 **拷贝到 SRAM/SDRAM** 再跳转
- 适合 **大程序、加密固件解密后运行**

**多级 Boot（Linux SoC）**：

```
ROM BootROM
  → SPL（片上 SRAM，初始化 DDR）
  → U-Boot（Flash/eMMC）
  → fitImage: kernel + dtb + initramfs
  → Linux
```

**U-Boot 常用**：

```bash
setenv bootcmd 'mmc dev 0; fatload mmc 0:1 ${kernel_addr_r} zImage; bootz'
saveenv
```

**RTOS 与 Linux 同芯片**：

| 核心 | 典型 |
|------|------|
| M4 | FreeRTOS 裸机/RTOS |
| A53 | Linux |
| **RPMsg** | 核间通信 |

**qemu-arm 学习**（本章 **README** 环境）：

```bash
qemu-system-arm -M vexpress-a9 -kernel zImage -dtb vexpress-v2p-ca9.dtb
# 或 -kernel demo.elf 测 RTOS 上下文切换
```

**链接脚本与启动一致**：

- `ENTRY(Reset_Handler)` 与向量表一致
- **栈顶** `_estack` 在 RAM 末尾，留足 **多任务栈**（**10.2.2**）
