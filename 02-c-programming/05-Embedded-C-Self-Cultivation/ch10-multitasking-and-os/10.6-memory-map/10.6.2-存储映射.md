# 10.6.2 存储映射

## 本节讲什么

**存储映射（Memory Map）** 把 **物理地址范围** 分配给各设备与 RAM；软件通过 **指针/基址+偏移** 访问。本节说明 **链接脚本、scatter file、Device Tree、`mmap`** 如何描述映射，衔接 **10.9 虚拟地址**。

## 要点

**MCU 平坦映射**：

```c
#define UART1_BASE  0x40011000
#define UART1_DR    (*(volatile uint32_t *)(UART1_BASE + 0x04))
```

- 编译器与 CPU 使用 **同一数字地址**
- **链接脚本** 定义段放置：

```ld
MEMORY {
  FLASH (rx) : ORIGIN = 0x08000000, LENGTH = 512K
  RAM   (rwx): ORIGIN = 0x20000000, LENGTH = 128K
}
```

**Linux 内核映射**：

- 内核 **线性映射** 或 **HIGHMEM**；用户态见 **虚拟地址**
- **`ioremap(phys, size)`**：内核把 MMIO 物理页映射到内核虚拟地址
- **`mmap(/dev/mem)`** 或 **`mmap` char driver**：用户态映射（需权限，**10.4.2**）

**Device Tree（ARM Linux）**：

```dts
uart0: serial@40011000 {
    compatible = "vendor,uart";
    reg = <0x40011000 0x400>;
    interrupts = <33>;
};
```

- 内核 probe 时 `reg` → `ioremap`；驱动 `devm_ioremap_resource`

**Cache 属性**：

| 区域 | 属性 |
|------|------|
| 普通 RAM | cacheable, write-back |
| 设备寄存器 | **device**, non-cacheable |
| 共享 DMA 缓冲 | **non-cache** 或 software coherency |

- 对 MMIO 用 cache → **总线错误** 或读 stale 值

**MPU 区域**（RTOS）：

- 任务可配置 **仅访问** `0x20000000–0x2001FFFF` RAM，碰 MMIO  fault
