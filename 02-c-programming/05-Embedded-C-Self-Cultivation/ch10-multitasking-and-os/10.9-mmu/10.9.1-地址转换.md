# 10.9.1 地址转换

## 本节讲什么

**虚拟地址（VA）** 经 **页表** 转为 **物理地址（PA）**；TLB 缓存近期映射加速访问。本节介绍 **两级/多级页表、段/页粒度** 与 **地址空间布局**，衔接 **10.6.2** 与 Linux **`/proc/pid/maps`**。

## 要点

**为何需要 VA**：

- 每个进程 **以为** 拥有从 0 开始的连续空间
- **简化链接**：PIE 可加载到任意基址
- **共享库** 映射到多个进程同一物理页

**ARM 32 两级页表示意**：

```
VA [31:22] → L1 项 → 指向 L2 表
VA [21:12] → L2 项 → 物理页框号 + 属性
VA [11:0]  → 页内偏移
```

**常见页大小**：4 KB；ARM 还支持 **1 MB section**（启动早期简化）

**地址空间布局（Linux 32 位用户态典型）**：

```
0x00000000  ──► 代码 text
            ──► heap ↑
            ──► mmap 区（共享库、文件映射）
0xBF000000  ──► 栈 ↓
```

**查看映射**：

```bash
cat /proc/self/maps
```

**RTOS 无完整 MMU**：

- 链接脚本固定 **物理地址**；`0x20000000` 即 RAM
- **MPU**：仅 8 区域，粗粒度保护任务栈

**切换进程/任务**：

- Linux：`switch_mm` 换 **TTBR0** + **ASID** 减 TLB flush
- RTOS MPU：上下文切换时 **重配 MPU 区域**

**与 DMA**：

- DMA 用 **物理地址**；CPU 用 VA → 驱动 `dma_map_single` 得到 bus addr
