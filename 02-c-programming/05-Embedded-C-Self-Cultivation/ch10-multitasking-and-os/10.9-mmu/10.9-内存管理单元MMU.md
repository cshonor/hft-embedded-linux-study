# 10.9 内存管理单元MMU

## 本节讲什么

**MMU（Memory Management Unit）** 实现 **虚拟地址到物理地址** 的转换，并附带 **访问权限、cache 属性**。本节说明 MMU 在 **Linux 进程隔离** 与 **RTOS MPU** 中的角色，对比 **10.4 syscall** 与 **10.10 进程**，是理解现代 OS 的硬件基础。

## 要点

**无 MMU（多数 M0/M3）**：

- 平坦地址；任务共享整个空间 → **野指针全局后果**
- 保护靠 **编码 + MPU（可选）**

**有 MMU（A 核 Linux）**：

```
CPU 读 VA 0x00400000
  → MMU 查 TLB/页表
  → 得 PA 0x8fc00000
  → 检查 AP（用户只读？）→ 违例则 data abort
```

**与多任务关系**：

| | RTOS + MPU | Linux + MMU |
|---|------------|-------------|
| 隔离粒度 | 几个区域/任务 | 每进程完整地址空间 |
| 切换开销 | 改 MPU 寄存器 | 换页表 ASID/TTBR |
| 栈保护 | guard region | guard page + NX |

**页 fault**：

- 合法：按需 **分配物理页**（匿名 mmap grow）
- 非法：`SIGSEGV` 杀进程

**与 **10.4.2** 权限**：

- 用户态不能写内核页 → syscall 必须 **copy_from_user**
- **NX bit**：栈不可执行，缓解 shellcode

**嵌入式 Linux**：

- **mmap** 映射 **/dev/fb**、**dma-buf**
- **10.6.2** 设备内存属性在页表设 Device
