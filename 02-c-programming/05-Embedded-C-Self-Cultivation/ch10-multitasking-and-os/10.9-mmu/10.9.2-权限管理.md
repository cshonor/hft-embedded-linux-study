# 10.9.2 权限管理

## 本节讲什么

页表项中的 **AP（Access Permission）、UXN/PXN** 等位决定 **谁能读写的、能否执行**。本节把 **MMU 权限** 与 **10.4.2 OS 权限**、**用户/内核态** 统一起来。

## 要点

**ARM 页表属性（简化）**：

| 属性 | 含义 |
|------|------|
| **AP** | 特权/用户，读/写组合 |
| **UXN** | 用户态不可执行 |
| **PXN** | 特权不可执行（用户页） |
| **TEX/CB** | cacheable / bufferable |

**典型映射策略**：

```
内核代码：  特权 RX，用户不可访问
内核数据：  特权 RW
用户代码：  用户 RX
用户栈/堆：  用户 RW，NX
MMIO：      特权 RW，Device，无 cache
```

**违规访问**：

- 用户读内核地址 → **data abort** → 内核发 **SIGSEGV**
- 执行栈上代码 → **NX fault**

**syscall 与权限**：

- 用户传指针 → 内核 **`access_ok` + copy_from_user** 验证映射且 **可写**
- 避免 **Time-of-check-time-of-use**；内核直接 copy

**MPU（Cortex-M）对比**：

```c
/* 区域：任务栈 RW，无 execute */
MPU->RBAR = stack_base | VALID;
MPU->RASR = SIZE | AP_FULL | XN;
```

- 任务切换 **重配 MPU**；越界 → **MemManage fault**

**安全扩展**：

- **TrustZone**：安全/非安全世界不同页表
- **SELinux**：在 MMU 之上再加 MAC 策略

**与多任务 bug**：

- RTOS 无 MMU：指针错误 **立即破坏内核**
- Linux：多数越界 **只杀自己进程**
