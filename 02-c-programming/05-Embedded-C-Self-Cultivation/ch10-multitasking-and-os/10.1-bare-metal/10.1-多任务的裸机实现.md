# 10.1 多任务的裸机实现

## 本节讲什么

在没有操作系统的 **裸机（Bare Metal）** 环境中，如何让多个「任务」看似同时运行？本节建立 **并发（Concurrency）** 与 **并行（Parallelism）** 的区分，对比 **前后台（Foreground/Background）** 与 **RTOS 抢占式多任务** 的差异，为 **10.1.1–10.1.3** 的模拟实现与 **demo01_foreground** 奠定概念框架。

## 要点

**并发 vs 并行**：

| 概念 | 含义 | 典型场景 |
|------|------|----------|
| **并发** | 逻辑上「同时」推进多个任务，可能分时复用单核 | 前后台轮询、协作式 RTOS |
| **并行** | 物理上同一时刻多核各跑一任务 | SMP Linux、DPDK 多 lcore |

单核 MCU 上常见的是 **并发**；多核 SoC 才谈 **并行**。

**三种多任务形态**：

```
┌─────────────────────────────────────────────────────────┐
│  前后台（Super Loop）                                     │
│  while(1) { 后台任务(); if(事件) 前台处理(); }            │
├─────────────────────────────────────────────────────────┤
│  协作式多任务（Cooperative）                              │
│  任务主动 yield；调度器在任务返回时切换                     │
├─────────────────────────────────────────────────────────┤
│  抢占式多任务（Preemptive RTOS）                          │
│  SysTick/中断强制切换；高优先级可打断低优先级               │
└─────────────────────────────────────────────────────────┘
```

- **前后台**：最简单，无 TCB、无独立栈；后台长循环会 **饿死** 前台响应（**demo01_foreground** 对比）
- **协作式**：需 **每任务独立栈 + TCB**（**ch08 8.3** 对象封装、**demo02_tcb_coop**）；任务不 yield 则他人无法运行
- **抢占式**：定时器中断触发 **上下文切换**（**10.3**、**demo03_preempt**）；更接近 FreeRTOS/uC/OS 行为

**裸机多任务的最小要素**（衔接 **ch05 栈**、**ch08 TCB**）：

| 要素 | 作用 |
|------|------|
| **任务控制块 TCB** | 保存栈指针、优先级、状态、入口函数 |
| **独立任务栈** | 每任务一份，避免局部变量互相覆盖 |
| **调度器** | 决定下一个运行的任务 |
| **上下文切换** | 保存/恢复 CPU 寄存器与 SP（**ch03 stmfd/ldmfd**） |

**常见陷阱**（全书终章反复出现）：

- **栈溢出**：深递归或大数组在任务栈上 → 破坏相邻任务或 TCB（**ch05 栈金丝雀**）
- **未保护的全局变量**：多任务读写同一变量 → 竞态
- **饿死（Starvation）**：低优先级任务长期得不到 CPU
