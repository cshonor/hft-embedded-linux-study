# 10.1.1 多任务的模拟实现

## 本节讲什么

在单线程、无 OS 的 C 程序里，如何用 **状态机** 或 **`setjmp`/`longjmp`** 模拟「多任务」切换？本节是 **demo02_tcb_coop**、**demo03_preempt**（主机端协作分支）的代码级基础，理解「保存现场、恢复现场」的本质，再过渡到真实 ARM 上下文切换。

## 要点

**方法一：状态机 + 分步执行**

```c
typedef enum { TASK_A_STEP0, TASK_A_STEP1, TASK_B_STEP0 } task_step_t;
static task_step_t step_a, step_b;

void scheduler_tick(void) {
    switch (step_a) { case TASK_A_STEP0: /* ... */ step_a = TASK_A_STEP1; break; }
    switch (step_b) { /* 同理 */ }
}
```

- 优点：无栈切换、可预测、适合极简前后台
- 缺点：任务逻辑被拆成碎片，无法写「自然」的阻塞式代码

**方法二：`setjmp`/`longjmp` 协作切换**（**demo03_preempt** 主机端）

```c
#include <setjmp.h>
jmp_buf task_a_env, task_b_env;

void task_a(void) {
    if (setjmp(task_a_env) == 0) {
        /* 首次进入 */
        while (1) {
            do_work();
            longjmp(task_b_env, 1);  /* 主动让出 */
        }
    }
    /* longjmp 返回点：恢复现场继续 */
}
```

| 对比项 | 状态机 | setjmp |
|--------|--------|--------|
| 栈 | 共用主栈 | 共用主栈（**不能真正并发**） |
| 局部变量 | 需 static 保存 | 跳转间局部变量未定义行为 |
| 教学价值 | 理解分时 | 理解「保存 PC/栈帧」概念 |

**方法三：独立栈 + TCB（正路）**（**demo02_tcb_coop**）

```c
typedef struct {
    void (*entry)(void *arg);
    void *arg;
    uint32_t *stack_top;   /* 栈顶（向下增长） */
    uint32_t priority;
    enum { READY, RUNNING, BLOCKED } state;
} tcb_t;
```

- 启动任务：伪造栈帧，使首次「返回」跳进 `entry`
- 切换时：保存 **R4–R11, SP, LR** 到 TCB；加载下一任务寄存器（**ch03** `stmfd sp!, {r4-r11, lr}` / `ldmfd sp!, {r4-r11, pc}`）

**与真实 RTOS 的差距**：

- 模拟方案通常 **无抢占**、无中断栈分离（**10.3.2**）
- 无 **mutex/semaphore** 时全局变量仍不安全（**10.2.4**、**demo04_semaphore**）
