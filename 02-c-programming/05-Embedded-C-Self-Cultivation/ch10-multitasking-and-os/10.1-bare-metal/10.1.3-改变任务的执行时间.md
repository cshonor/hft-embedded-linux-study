# 10.1.3 改变任务的执行时间

## 本节讲什么

**执行频率** 决定任务被调度的次数；**执行时间（时间片 Time Slice）** 决定单次连续占用 CPU 多久。本节讲解 **固定时间片轮转**、**协作式主动让出** 与 **抢占式时间片耗尽**，是理解 **demo03_preempt** 与 **10.2.1 调度器** 的关键一环。

## 要点

**时间片轮转（Round-Robin）**：

```
Task A ████░░░░  Task B ░░████░░  Task C ░░░░████
       ↑ 时间片到，强制切换
```

- 每个就绪任务分配 `quantum` 个 tick（如 5 ms）
- 时间片用完 → 移到就绪队列尾部 → 切换下一任务
- 同等优先级任务获得 **公平** CPU 份额

**协作式：任务自己决定占用多久**：

```c
void task_comm(void) {
    for (;;) {
        process_uart();
        os_yield();  /* 主动交出剩余时间片 */
    }
}
```

- 某任务若从不 yield → **饿死** 其他任务（协作式最大风险）
- 适合已知短临界、无恶意长循环的嵌入式场景

**抢占式：SysTick 强制切换**（**demo03_preempt** ARM 路径）

```
SysTick_Handler:
    tick++
    if (--current->timeslice == 0) {
        current->timeslice = DEFAULT_QUANTUM;
        os_schedule();          /* PendSV 触发上下文切换 */
    }
```

ARM 上下文切换要点（衔接 **ch03**）：

1. **保存**：`stmfd sp!, {r4-r11, lr}` → 写入当前 TCB
2. **切换 SP**：`sp = next->stack_ptr`
3. **恢复**：`ldmfd sp!, {r4-r11, pc}` → 跳入下一任务

**时间片大小的权衡**：

| 时间片 | 效果 |
|--------|------|
| 过小 | 切换开销占比高，吞吐下降 |
| 过大 | 交互响应差，低优先级等待久 |
| 按任务定制 | 高优先级短片、低优先级长片（高级调度） |

**阻塞 vs 时间片耗尽**：

- **时间片到**：任务仍 **就绪**，只是换到队列后面
- **阻塞**（等 semaphore/queue）：任务进 **BLOCKED**，不占时间片（**10.2.1**、**demo04/05**）
