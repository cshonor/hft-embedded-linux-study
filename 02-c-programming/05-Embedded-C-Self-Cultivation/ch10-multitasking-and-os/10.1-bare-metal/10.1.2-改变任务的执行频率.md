# 10.1.2 改变任务的执行频率

## 本节讲什么

多任务系统中，不同任务往往 **不需要** 以相同速率运行：LED 500 ms 闪一次，传感器 10 ms 采一次。本节说明如何用 **软件 tick 计数**、**时间片比例** 控制各任务 **被调度的频率**，衔接前后台与协作式调度，并为 **10.1.3 时间片** 做铺垫。

## 要点

**SysTick / 定时器 tick**：

- 硬件定时器每 1 ms（或 10 ms）触发一次中断 → `tick_count++`
- 调度器在 tick 或主循环末尾检查：**该任务本轮是否到期**

**按周期调度（Period-based）**：

```c
typedef struct {
    tcb_t base;
    uint32_t period_ms;    /* 期望周期 */
    uint32_t last_run_tick;
} task_period_t;

bool task_due(task_period_t *t, uint32_t now) {
    return (now - t->last_run_tick) >= t->period_ms;
}
```

| 任务 | period_ms | 含义 |
|------|-----------|------|
| 按键扫描 | 5 | 200 Hz |
| 温度采集 | 100 | 10 Hz |
| 日志上传 | 5000 | 0.2 Hz |

**按计数比例调度（协作式主循环）**：

```c
/* demo01_foreground 变体：每 N 次循环才跑一次慢任务 */
if ((loop_cnt % 100) == 0) slow_task();
fast_task();  /* 每圈都跑 */
```

- 前后台里这是 **改变执行频率** 的唯一手段
- 缺点：循环耗时不稳定 → 实际周期漂移

**与优先级的区别**（**10.2.1**）：

| 机制 | 回答的问题 |
|------|------------|
| **频率/周期** | 这个任务 **多久运行一次** |
| **优先级** | 多个任务同时就绪时 **谁先运行** |

两者正交：高优先级任务也可以 period=1000 ms（偶尔运行但要及时响应）。

**FreeRTOS 对应概念**：

- `vTaskDelay()` / `vTaskDelayUntil()` 控制 **阻塞时间**
- 软件定时器 `xTimerCreate` 回调以指定频率执行
