# 10.5 揭开文件系统的神秘面纱

## 本节讲什么

**文件系统** 把块设备上的字节组织成目录、文件与权限，是 Linux 应用访问持久化数据的主路径。本节总览 **VFS、挂载、根文件系统**，为嵌入式 Linux（Buildroot/Yocto）与 **10.4 syscall** 中的 `open/read/write` 建立整体图。

## 要点

**为什么需要文件系统**：

- 裸块设备只有 **扇区/LBA**；应用需要 **路径名、层次目录、并发访问**
- 不同介质：**eMMC、SD、NOR/NAND Flash、UBIFS**

**Linux 分层**：

```
应用 open("/etc/config.ini")
    │
    ▼
VFS（虚拟文件系统）── 统一 inode/dentry/file 抽象
    │
    ├── ext4 模块
    ├── procfs / sysfs（伪文件系统）
    └── devtmpfs（/dev）
    │
    ▼
块层 / MTD ──► 驱动 ──► 硬件（**10.6/10.7**）
```

**RTOS 对比**：

- FreeRTOS + **FatFs** / **LittleFS**：单任务或 mutex 保护，无 VFS 多 FS 并存
- 资源受限 MCU 常用 **裸 Flash 分区表** 而非完整 FS

**嵌入式 Linux 典型布局**：

```
/
├── bin/ lib/      用户态程序与库
├── etc/           配置
├── dev/           设备节点
├── proc/ sys/     内核接口
├── mnt/           挂载点
└── userdata/      可写分区（ext4/ubifs）
```

**与 syscall 关系**（**10.4.4**）：

- 每个进程 **FD 表**；`open` 分配 fd → 指向内核 `struct file`
- `read/write` 经 VFS → 具体 FS → 块 I/O
