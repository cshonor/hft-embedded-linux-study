# 10.5.3 根文件系统

## 本节讲什么

**根文件系统（rootfs）** 是 Linux 启动后挂载的第一个 FS，包含 **`/sbin/init`** 及基本目录。本节介绍 **initramfs、initrd、NFS 网络根、只读 SquashFS + 可写 overlay** 等嵌入式方案，衔接 **10.6.3 启动方式**。

## 要点

**启动链**：

```
Bootloader (U-Boot)
    → 加载 kernel + DTB [+ initramfs]
    → kernel 解压 initramfs 到 rootfs（内存中 cpio）
    → 或 mount 块设备上的 root= 分区
    → 执行 /sbin/init（busybox systemd）
```

**initramfs vs initrd**：

| | initramfs | initrd |
|---|-----------|--------|
| 格式 | cpio 归档，常内嵌 kernel | 旧式磁盘镜像 |
| 位置 | 内存 | 需先当块设备 |
| 用途 | 早期用户态、加载驱动、 pivot_root | 遗留 |

**内核命令行**：

```
root=/dev/mmcblk0p2 rootwait rw
root=/dev/nfs nfsroot=192.168.1.1:/export/nfs,v3
init=/linuxrc
```

**典型嵌入式 rootfs 组成**（Buildroot）：

- **BusyBox**：`sh`, `init`, `mount`, `ls`
- 动态库 `libc.so`
- 模块 `/lib/modules/$(uname -r)/`
- 配置 `/etc/inittab` 或 systemd unit

**只读根 + 可写层**：

```
lower: SquashFS（ROM/eMMC 分区，不可改）
upper: UBIFS/ext4 可写层
merged: overlayfs 挂载到 /
```

- OTA 更新只换 lower 分区；配置在 upper

**与 RTOS 镜像对比**：

| | RTOS 固件 | Linux rootfs |
|---|-----------|--------------|
| 格式 | 单一 `.bin/.elf` | 目录树或 cpio |
| 更新 | 全镜像 | 分区/包管理 |
| 大小 | KB～MB | MB～百 MB |

**调试技巧**：

```bash
# initramfs 内 shell
exec /bin/sh
# 或 init=/bin/sh 单用户
```
