# 10.5.1 什么是文件系统

## 本节讲什么

**文件系统（FS）** 是在存储介质上实现的 **数据组织格式**：如何找文件、存内容、管理空闲块与元数据。本节介绍 **inode、目录项、超级块** 及 **常见嵌入式 FS 选型**。

## 要点

**核心概念**：

| 概念 | 说明 |
|------|------|
| **超级块 superblock** | FS 全局元数据：块大小、总量、magic |
| **inode** | 文件元数据：大小、权限、时间戳、数据块指针 |
| **dentry** | 目录项：文件名 → inode 映射 |
| **数据块 block** | 实际存储内容的单元（4K 常见） |

**路径解析**：

```
/open("/home/user/a.txt")
  → 从根 inode 沿目录树查 dentry
  → 得到 a.txt 的 inode
  → read 按块指针读数据块
```

**常见文件系统**：

| FS | 场景 |
|----|------|
| **ext4** | eMMC/SD，大容量 |
| **UBIFS** | 裸 NAND + UBI 卷 |
| **JFFS2** | 老 NAND |
| **SquashFS** | 只读压缩根 FS |
| **LittleFS / FatFs** | MCU RTOS |

**Flash 特有问题**：

- **擦除块** >> 写入页 → 需 **磨损均衡、日志**
- 裸 MCU 可用 **EasyFlash、KVDB** 代替完整 FS

**权限位**（衔接 **10.4.2**）：

```bash
-rwxr-xr--  1 root root  4096 Jan  1 00:00 app.conf
```

- `rwx` user/group/other；目录 `x` 表示可进入

**VFS 统一接口**：

```c
struct file_operations {
    ssize_t (*read)(struct file *, char __user *, size_t, loff_t *);
    ssize_t (*write)(...);
    int (*open)(...);
};
```

- 具体 FS 注册 ops；`/proc` 无真实块设备
