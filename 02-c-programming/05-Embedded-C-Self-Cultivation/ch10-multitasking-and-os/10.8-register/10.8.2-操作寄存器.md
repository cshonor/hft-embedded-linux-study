# 10.8.2 操作寄存器

## 本节讲什么

把 **MMIO 读写** 封装成 **类型安全、可审查** 的内联函数：读状态、写配置、 **RMW 辅助**、 **延迟与 barrier**。本节给出驱动层推荐写法，避免魔法数字散落。

## 要点

**寄存器结构体映射**（可选）：

```c
typedef struct {
    volatile uint32_t DR;
    volatile uint32_t SR;
    volatile uint32_t CR;
    uint32_t _rsvd;
    volatile uint32_t BRR;
} uart_regs_t;

#define UART1  ((uart_regs_t *)0x40011000)
```

**内联 API**：

```c
static inline uint32_t mmio_read32(uintptr_t addr) {
    return *(volatile uint32_t *)addr;
}

static inline void mmio_write32(uintptr_t addr, uint32_t val) {
    *(volatile uint32_t *)addr = val;
}

static inline void mmio_set_bits(uintptr_t addr, uint32_t mask) {
    mmio_write32(addr, mmio_read32(addr) | mask);
}
```

**写后读确认（可选）**：

- 部分 IP 需要 **write → readback** 确认
- 或 **dsb** 保证写入到达外设：

```c
mmio_write32(UART_BASE + CR, val);
__DSB();
```

**批量初始化表**：

```c
static const struct { uint32_t off; uint32_t val; } uart_init[] = {
    { 0x0C, 0x0000 },  /* CR 复位 */
    { 0x08, 0x0683 },  /* BRR 9600 */
    { 0x0C, UART_CR_UE | UART_CR_TE | UART_CR_RE },
};
for (size_t i = 0; i < ARRAY_SIZE(uart_init); i++)
    mmio_write32(UART_BASE + uart_init[i].off, uart_init[i].val);
```

- 清晰、易 code review；与 **ch09** 配置模块结合

**Linux regmap**：

- 统一 **cache、bus、debugfs**；I2C SPI 尤其有用

**多任务/ISR**：

- 修改 **共享控制寄存器** 的 RMW 段 → **临界区**（**10.2.4**）
- 优先使用 **硬件 SET/CLR 寄存器**
