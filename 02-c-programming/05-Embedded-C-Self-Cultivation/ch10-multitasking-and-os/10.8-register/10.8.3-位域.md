# 10.8.3 位域

## 本节讲什么

C **位域（bit-field）** 可把 struct 成员映射到寄存器各位，读起来像手册表格。本节说明 **语法、对齐/endian 陷阱** 与 **何时用手动掩码更可靠**。

## 要点

**语法示例**：

```c
typedef struct {
    volatile uint32_t enable : 1;
    volatile uint32_t mode   : 2;
    volatile uint32_t        : 5;   /* 保留 */
    volatile uint32_t div    : 8;
} __attribute__((packed)) clk_cr_bits_t;

/* 不推荐：对 MMIO 整 struct 赋值 */
```

**问题 1：实现定义布局**：

- 位域 **顺序、跨字节** 由 ABI 决定
- **不同编译器/优化** 可能与硬件位序不一致

**问题 2：非原子 RMW**：

```c
reg->mode = 2;  /* 可能编译成 读-改-写 整字 */
```

- ISR 与任务并发改不同位域 → 互相覆盖

**问题 3：volatile 位域**：

- C 标准对 **volatile struct 位域** 访问规则模糊
- 行业惯例：**MMIO 用 uint32_t + 宏掩码**（**10.8.1**）

**适用场景**：

| 场景 | 建议 |
|------|------|
| 硬件 MMIO | **宏 + 掩码** |
| 协议打包报文 | 位域或手动 shift（明确 endian） |
| 内核 `linux/bitfield.h` | `GENMASK` 风格 |

**可接受模式：只读解析**：

```c
uint32_t raw = mmio_read32(REG);
unsigned mode = (raw >> 3) & 0x3;   /* 显式，可测 */
```

**手册对照流程**：

1. 画表：位号、名称、RW、复位值
2. `#define REG_FOO_MASK` 与 `SHIFT`
3. 单元测试 **golden value** 与仿真器比对
