# 10.7.3 I/O端口与I/O内存

## 本节讲什么

x86 有独立的 **I/O 端口空间**（`in/out` 指令）；ARM 等 **RISC 统一用内存映射 I/O（MMIO）**。本节说明两种模型、**volatile** 语义与 Linux **`ioread32/iowrite32`**，是 **10.8 寄存器操作** 的直接前提。

## 要点

**两种 I/O 模型**：

| | I/O 端口（x86） | MMIO（ARM/MIPS/RISC-V 常见） |
|---|-----------------|------------------------------|
| 地址空间 | 独立 port 编号 | 占用物理地址 map |
| 指令 | `in/out` | 普通 load/store |
| 示例 | `outb(0x3F8, ch)` | `*(volatile uint32_t*)0x40011000 = v` |

**MMIO 访问模板**（**10.8.2**）：

```c
#define REG32(addr)  (*(volatile uint32_t *)(addr))

static inline void reg_set_bits(uint32_t addr, uint32_t mask) {
    uint32_t v = REG32(addr);
    REG32(addr) = v | mask;
}
```

**为何 `volatile`**：

- 防止编译器 **缓存寄存器读** 或 **删除“多余”写**
- 不保证 **CPU 乱序** vs 外设；多核需 **barrier**（`dmb`/`dsb`）

**Linux 内核 API**：

```c
void __iomem *base = ioremap(0x40011000, SZ_4K);
writel(val, base + 0x04);
u32 sr = readl(base + 0x08);
iounmap(base);
```

- **`__iomem`** 标记，sparse 检查误用

**Cache 与 MMIO**：

- 设备区域标记 **Device-nGnRnE**（ARM）→ 不 cache
- 用户态 **不应** 直接 mmap 随机物理地址（**10.4.2**）

**与 **10.6.2** 关系**：

- SoC 手册 **Memory Map** 表 = 驱动 `#define` 来源
- 同一 IP 在不同芯片 **基址不同** → 用 DT 或 **board.h** 抽象（**ch09 weak**）
