# 10.7.2 外部设备

## 本节讲什么

**外部设备** 通过控制器挂到总线上，由 **寄存器组** 控制。本节介绍 **设备驱动分层**、**轮询 vs 中断**、Linux **字符/块/网络驱动** 与 RTOS **HAL** 的对应关系。

## 要点

**驱动分层**（衔接 **ch09**）：

```
app 任务
  │  os_queue / read()
  ▼
driver/uart.c     ← 模块：init, send, irq_handler
  │  MMIO
  ▼
hardware UART IP
```

**访问模式**：

| 模式 | 适用 |
|------|------|
| **轮询** | 简单、bootloader、短超时 |
| **中断** | 异步事件、省 CPU（**10.3**） |
| **DMA** | 大批量传输，CPU 做别的任务 |

**Linux 设备模型**：

```
应用 open("/dev/ttyS0")
  → 内核 tty 层
  → serial_driver ops
  → platform_driver probe(DT)
  → readl/writel 硬件
```

**RTOS HAL**：

- ST HAL `HAL_UART_Transmit_IT` + 回调
- 与 **demo06_mini_rtos** 的 `bsp/uart` + `kernel/queue` 同构

**设备树描述**（Linux）：

- `compatible` 匹配驱动；`reg` 基址；`interrupts` 线号
- 与裸机 `#define UART_BASE` 信息等价，可自动生成头文件（**pinctrl**）

**并发注意**：

- 同一 UART 多任务打印 → **mutex 或 log queue**（**demo05_queue**）
- ISR 与任务共享 ringbuf → **临界区** 或 **无锁 SPSC**（单生产者单消费者）
