# 10.3.3 中断函数的实现

## 本节讲什么

如何编写 **正确、高效、可维护** 的 ISR？本节给出 **模板、FromISR API、临界区** 与 **常见反模式**，衔接 FreeRTOS/uC/OS 惯例与 **demo04/05** 的中断唤醒任务。

## 要点

**ISR 标准模板**（Cortex-M）：

```c
void UART1_IRQHandler(void) {
    BaseType_t wake = pdFALSE;

    if (UART1->SR & RXNE) {
        uint8_t b = UART1->DR;
        ringbuf_put_from_isr(&rx_rb, b);
        xQueueSendFromISR(uart_q, &b, &wake);
    }
    UART1->SR = /* 清标志 */;

    portYIELD_FROM_ISR(wake);  /* wake==pdTRUE 则立即切换 */
}
```

**FromISR 后缀 API**：

| 任务上下文 | 中断上下文 |
|------------|------------|
| `xQueueSend` | `xQueueSendFromISR` |
| `xSemaphoreGive` | `xSemaphoreGiveFromISR` |
| `vTaskDelay` | **禁止** |

- FromISR 版本 **不阻塞**；通过 `pxHigherPriorityTaskWoken` 告知需切换

**临界区在中断中**：

```c
/* 仅屏蔽部分优先级 */
uint32_t mask = taskENTER_CRITICAL_FROM_ISR();
/* 极短共享变量更新 */
taskEXIT_CRITICAL_FROM_ISR(mask);
```

- 不要与任务侧 `taskENTER_CRITICAL()` 混用错误 API

**Defer 模式（推荐架构）**：

```
ISR ──► ringbuf / flag / queue ──► 专用 worker 任务
                                      │
                                      ├─ 协议解析
                                      ├─ mutex 保护 printf
                                      └─ 网络 upload
```

**反模式**：

| 反模式 | 后果 |
|--------|------|
| ISR 里 `printf` | 死锁、耗时 ms 级 |
| ISR 里 `malloc` | 堆锁、不确定延迟 |
| 清标志顺序错 | 丢中断或重复进 ISR |
| 共享变量无 `volatile`/屏障 | 优化器缓存旧值 |

**uC/OS-III 对应**：

- `OSIntEnter()` / `OSIntExit()` 嵌套计数
- `OSTaskQPost()` 从 ISR 投递消息

**与模块化的关系**（**ch09**）：

- 驱动 `.c` 提供 `xxx_irq_handler()`；BSP 向量表注册
- 应用 **不** 直接写 ISR，通过 **回调/queue** 收事件（**9.7.2**）
