# 10.3.1 中断处理流程

## 本节讲什么

从外设拉高中断线到 **ISR 返回**，CPU 与 NVIC 做了哪些硬件动作？本节梳理 **Cortex-M 中断流程**，说明 **SysTick 触发抢占调度** 与 **PendSV 延迟切换** 的分工，对应 **demo03_preempt** 的 ARM 路径。

## 要点

**Cortex-M 硬件自动流程**：

```
1. 完成当前指令（或异常点）
2. 硬件压栈：xPSR, PC, LR, R12, R3, R2, R1, R0  → 当前 SP（任务或 MSP）
3. 读向量表 → 跳 ISR
4. LR = EXC_RETURN（标记返回用哪个栈、是否 FPU）
5. ISR 执行 ...
6. BX LR  → 硬件弹栈 → 回到被打断代码
```

**向量表**：

```c
/* startup 中 */
void (* const vector_table[])(void) = {
    stack_top,
    Reset_Handler,
    NMI_Handler,
    HardFault_Handler,
    /* ... */
    SysTick_Handler,
    PendSV_Handler,
};
```

**SysTick + PendSV 协作**（FreeRTOS 经典模式，**demo03_preempt**）：

```c
void SysTick_Handler(void) {
    tick_count++;
    if (--current->timeslice == 0) {
        current->timeslice = QUANTUM;
        SCB_ICSR |= PENDSVSET;   /* 仅请求 PendSV，不在此切换 */
    }
}

void PendSV_Handler(void) {
    __asm volatile (
        "mrs r0, psp\n"
        "stmfd r0!, {r4-r11}\n"   /* 保存 callee-saved */
        "str r0, [current]\n"
        /* 选 next，加载 sp，ldmfd ... pc */
    );
}
```

- **为何分两步**：SysTick 优先级可较高保证 tick 准；**上下文切换** 放最低优先级 PendSV，避免嵌套切换混乱

**中断嵌套**：

- 高优先级 ISR 可打断低优先级 ISR
- 每层嵌套多占 **一份硬件压栈帧** → 任务栈或 MSP 深度增加（**10.3.2**）

**中断延迟组成**：

```
延迟 = 当前指令完成 + 更高优先级 ISR + 临界区关中断时间 + 流水线
```

- RTOS 中 `taskENTER_CRITICAL()` 会 **抬 BASEPRI**，屏蔽低于阈值的 IRQ

**Bottom Half / 任务 defer**：

```
UART ISR: 读 FIFO → 入 ringbuf → queue_send_from_isr → 唤醒解析任务
解析任务: 协议状态机、printf（可阻塞）
```
