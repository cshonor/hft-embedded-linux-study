# 10.3 中断

## 本节讲什么

**中断** 是硬件异步打断 CPU 正常流程的机制，也是 **抢占式 RTOS** 的时间基准与外设驱动入口。本节总览中断与多任务的关系：**谁可以打断谁**、**中断里能做什么**，并指向 **10.3.1–10.3.3** 的流程、双栈与 ISR 编码规范。

## 要点

**中断在多任务中的角色**：

| 用途 | 示例 |
|------|------|
| **系统 tick** | SysTick → 时间片、delay |
| **外设事件** | UART RX、DMA 完成、GPIO |
| **软件触发** | PendSV 做上下文切换；SVC 进内核 |

**中断 vs 轮询**：

- 轮询：CPU 空转查标志 → 前后台常见
- 中断：事件驱动 → RTOS 任务可 **阻塞等待**，CPU 跑其他任务

**中断优先级（NVIC）**：

```
高 ── SysTick / 关键外设
中 ── UART、Timer
低 ── PendSV（刻意最低，最后执行切换）
```

- ARM Cortex-M：**数字越小优先级越高**（与部分 MCU 相反，以手册为准）
- PendSV 设最低 → SysTick 里只置位「需要切换」，在 PendSV 里做 **stmfd/ldmfd**（**demo03_preempt**）

**ISR 设计铁律**：

1. **短**：微秒～几十微秒级
2. **不阻塞**：不调 `mutex_lock`、`malloc`、`printf`
3. **Defer 到任务**：ISR 里 `sem_post_from_isr` / `queue_send_from_isr`

**与并发的关系**：

- ISR 可与 **任意任务** 并发 → 共享数据要临界区保护（**10.2.4**）
- ISR 可能 **重入** 同一向量（若未清标志）→ 驱动要可重入或关同类中断
