# 10.3.2 进程栈与中断栈

## 本节讲什么

任务在 **任务栈（PSP）** 上运行函数调用；中断发生时硬件往 **当前栈指针** 压寄存器。本节说明 **MSP vs PSP**、何时需要 **独立中断栈**，以及栈溢出在中断嵌套下的风险，衔接 **10.2.2** 与 **ch05**。

## 要点

**Cortex-M 双栈指针**：

| 栈指针 | 典型用途 |
|--------|----------|
| **MSP** | 复位、异常、内核、无 OS 裸机 |
| **PSP** | RTOS 任务用户栈 |

- Thread 模式 + `CONTROL.SPSEL=1` → 用 **PSP** 作任务栈
- 进入异常 → 硬件用 **当前** SP 压栈（RTOS 任务被打断时用 **PSP**）

**独立中断栈（可选）**：

```
部分 Cortex-A / RTOS 配置：
  任务 ──PSP──► 任务栈（函数调用）
  异常 ──MSP──► 独立 IRQ 栈（仅硬件帧 + ISR 局部变量）
```

好处：

- 任务栈 **不必** 预留「最坏中断嵌套」深度
- ISR 深调用不影响任务栈 canary 边界

**Cortex-M 常见做法**：

- 不单独分 IRQ 栈；在 **任务栈大小** 里预留 `configMINIMAL_STACK_SIZE` + 嵌套裕量
- FreeRTOS：`configISR_STACK_SIZE`（部分移植用于 MSP）

**栈溢出在中断场景**：

```
任务栈已满 ──► 中断压 8 字硬件帧 ──► 溢出到 TCB/邻任务
```

检测（**10.2.2**）：

- 调度前检查 canary
- `HardFault` 时读 `CFSR/STKERR` 位

**Linux 对比**（**10.10**）：

- 每线程 **内核栈** + 用户栈；中断用 **per-CPU IRQ stack**
- 内核栈固定 8K/16K，与用户栈分离

**估算任务栈 checklist**：

1. 最深调用链局部变量之和
2. + 最大中断嵌套 × 硬件帧（~32–64 字节/层）
3. + FPU 扩展帧（若用浮点 ISR）
4. + 对齐 padding；再 ×1.5 安全系数
