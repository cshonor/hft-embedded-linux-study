# 10.4.4 Linux系统调用接口

## 本节讲什么

如何在 Linux 上 **查看、调用、追踪系统调用**？本节介绍 **`syscall()` 直调**、**man 2**、**strace** 与常见编号，帮助从 RTOS 思维过渡到 **10.10 进程/线程** 与 **10.5 文件系统** 的 Linux 实践。

## 要点

**man 页面分区**：

| 节 | 内容 |
|----|------|
| **man 2** | 系统调用 |
| **man 3** | libc 函数 |
| **man 7** | 概述（如 `man 7 syscalls`） |

**直接 syscall 示例**（教学用，生产用 libc）：

```c
#define _GNU_SOURCE
#include <unistd.h>
#include <sys/syscall.h>

long n = syscall(SYS_write, 1, "hi\n", 3);
```

**strace 追踪**：

```bash
strace -f ./demo_app          # 列出实际 syscall
strace -e trace=open,read,write ./demo_app
```

典型输出解读：

```
openat(AT_FDCWD, "/etc/ld.so.cache", O_RDONLY) = 3
mmap(NULL, 8192, PROT_READ|PROT_WRITE, ...) = 0x...
write(1, "hello\n", 6)                  = 6
```

**常见 syscall 与 **10.5/10.10** 映射**：

| syscall | 作用 | 章节 |
|---------|------|------|
| `open/read/write/close` | 文件 I/O | **10.5** |
| `fork/execve/wait` | 进程 | **10.10.1** |
| `clone` | 线程 | **10.10.2** |
| `mmap/munmap/brk` | 内存 | **10.9** |
| `ioctl` | 设备控制 | **10.7** |

**错误码**：

- 失败返回 `-1`，`errno` 为 `EAGAIN`、`EINVAL`、`EACCES` 等
- 内核与用户约定 **负返回值** 转 errno（架构相关）

**与 RTOS 对照**：

| RTOS | Linux |
|------|-------|
| `xTaskCreate` | `clone` + pthread 库 |
| `xQueueSend` | `pipe`/`socket`/`eventfd` |
| `vTaskDelay` | `nanosleep` |
| 无 | 每进程独立 FD 表、VMA |

**性能注意**：

- 高频路径用 **vDSO**（`gettimeofday` 无 syscall）
- DPDK **user-space driver** 避免 `read/write` 网络包
