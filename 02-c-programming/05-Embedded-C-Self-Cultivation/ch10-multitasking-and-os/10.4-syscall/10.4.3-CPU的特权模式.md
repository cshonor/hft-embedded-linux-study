# 10.4.3 CPU的特权模式

## 本节讲什么

CPU **特权级（Privilege Level）** 决定当前代码能否执行敏感指令（访问页表、改时钟、关中断等）。本节对比 **x86 Ring、ARM Exception Level、Cortex-M Handler/Thread 模式**，说明 **syscall/异常** 如何触发级切换。

## 要点

**x86 Ring 模型**：

```
Ring 0 ── 内核
Ring 3 ── 用户程序
syscall / 中断 ──► 硬件切换到 Ring 0，换栈
```

**ARM AArch64 Exception Level**：

| EL | 典型 |
|----|------|
| EL0 | 用户应用 |
| EL1 | OS 内核 |
| EL2 | Hypervisor（KVM） |
| EL3 | Secure Monitor |

**ARM Cortex-M（嵌入式常见）**：

- 无 Ring 名，用 **Thread / Handler** 模式 + **CONTROL.nPRIV**
- **特权 Thread**：可访问 NVIC、部分系统寄存器
- **非特权 Thread**：MPU 限制；**SVC** 进 Handler 提权

**敏感指令示例**：

- 写 **TTBR**（页表基址）、**MSR PRIMASK**、**WFI/WFE**、直接 **MMIO**（若无映射）

**syscall 入口（ARM 32 Linux）**：

```c
/* 用户态 */
mov r7, #__NR_write
svc #0          /* 陷入内核 vector_swi */
```

内核：

1. 保存用户寄存器到 **内核栈**
2. 查 **sys_call_table[r7]**
3. 执行 `sys_write`，`copy_from_user`
4. 恢复寄存器返回用户态

**RTOS 无 EL0/EL1 分离时**：

- 任务与内核同特权 → **一个野指针** 可毁全局
- 工业方案：MPU 分区 + SVC 网关 + **TrustZone**

**与中断统一**：

- syscall、fault、IRQ 都经 **异常向量** → 进入 Handler 模式
- **10.3.1** 硬件压栈与 **syscall** 压栈类似，返回用 **EXC_RETURN**
