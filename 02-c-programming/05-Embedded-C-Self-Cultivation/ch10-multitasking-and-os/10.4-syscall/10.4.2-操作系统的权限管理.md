# 10.4.2 操作系统的权限管理

## 本节讲什么

操作系统通过 **权限管理** 限制程序能访问的资源：哪些内存可读写的、哪些指令可执行、哪些设备节点可打开。本节从 **用户/内核**、**UID/GID**、**MPU/MMU** 角度建立模型，衔接 **10.4.3 特权模式** 与 **10.9 MMU**。

## 要点

**权限的目标**：

- **隔离**：进程 A 不能读进程 B 密码
- **稳定**：用户程序 bug 不拖垮内核
- **安全**：非授权不能操作硬件寄存器

**Linux 用户态权限**：

| 机制 | 作用 |
|------|------|
| **UID/GID** | 文件 rwx、进程身份 |
| **`capability`** | 细分 root 能力（`CAP_NET_RAW` 等） |
| **seccomp** | 限制可用 syscall 集合 |
| **namespace/cgroup** | 容器资源边界 |

```bash
ls -l /dev/mem          # 通常 root 才能 rw
getuid(); open("/etc/shadow", O_RDONLY);  # EACCES
```

**RTOS / 裸机**：

- 通常 **无 UID**；所有任务 **同一特权**（或 TrustZone 分世界）
- **MPU**（Cortex-M）：最多 8 区域，限制任务访问 RAM 范围
- 权限靠 **编码规范 + 模块划分**（**ch09**），而非硬件进程隔离

**内存权限**（**10.9.2**）：

| 区域 | 典型权限 |
|------|----------|
| 代码 `.text` | RX |
| 数据 `.data/.bss` | RW |
| 外设 | 仅内核/驱动 RW |
| 用户栈 | RW，不可执行（NX） |

**从 syscall 视角**：

- 用户传指针 → 内核 **copy_from_user** 验证地址是否在用户映射且可写
- 非法指针 → `EFAULT` 而非内核崩溃

**嵌入式 Linux 启动**：

- 应用以普通用户运行；驱动 **内核模块** 或 **udev** 规则授权设备节点
