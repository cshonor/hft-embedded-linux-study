# 10.10.3 线程池

## 本节讲什么

**线程池** 预先创建固定数量工作线程，从 **任务队列** 取活执行，避免 **频繁 create/join** 开销，并限制 **并发度** 防止资源耗尽。本节衔接 **10.2.4 queue**、**demo05_queue** 与服务器/网关常见架构。

## 要点

**基本结构**：

```
提交者 ──► [任务队列] ◄── worker 线程 1..N
              │ mutex + cond
              └─ 阻塞等待 / 唤醒
```

**伪代码**：

```c
void pool_submit(pool_t *p, void (*fn)(void *), void *arg) {
    task_t t = { fn, arg };
    pthread_mutex_lock(&p->mtx);
    queue_push(&p->q, t);
    pthread_cond_signal(&p->cv);
    pthread_mutex_unlock(&p->mtx);
}

void *worker(void *arg) {
    for (;;) {
        pthread_mutex_lock(&p->mtx);
        while (queue_empty(&p->q))
            pthread_cond_wait(&p->cv, &p->mtx);
        task_t t = queue_pop(&p->q);
        pthread_mutex_unlock(&p->mtx);
        t.fn(t.arg);
    }
}
```

**参数 tuning**：

| 参数 | 考虑 |
|------|------|
| 线程数 | CPU 核数、I/O 等待比例 |
| 队列长度 | 背压、拒绝策略 |
| 栈大小 | 与 **10.2.2** 同理 |

**RTOS 对应**：

- **静态创建** N 个任务，**queue** 投递工作项（**demo05_queue**、**demo06**）
- 无 pthread 开销；任务数编译期固定

**与 **10.2.1** 阻塞**：

- worker 在 **空队列** 上 BLOCKED → 不占 CPU  spin
- 高优先级 **ISR** → `queue_send_from_isr` 唤醒（**10.3.3**）

**反模式**：

- 无界队列 → 内存暴涨
- 线程过多 → 上下文切换开销 > 计算

**Linux 生态**：

- `libpthread` 自实现；或使用 **thread pool** 库、**io_uring** 异步
