# 10.10.1 进程

## 本节讲什么

**进程** 是 Linux 中 **资源分配与隔离** 的基本单位：独立 **虚拟地址空间、文件描述符表、PID**。本节说明 **`fork/exec/wait`**、与 **10.9 MMU**、**10.4 syscall** 的关系，对比 **RTOS 无进程** 模型。

## 要点

**进程拥有什么**：

```
struct task_struct（内核）
  ├── mm_struct     → 页表、VMA（**10.9.1**）
  ├── files_struct  → fd 表
  ├── fs_struct     → cwd, root
  └── pid, 信号, 调度实体 ...
```

**创建流程**：

```c
pid_t pid = fork();
if (pid == 0) {
    execl("/bin/child", "child", NULL);
} else {
    wait(NULL);
}
```

- **`fork`**：复制地址空间（**COW** 优化，写时才复制物理页）
- **`execve`**：替换当前进程镜像为新程序
- **`wait`**：收尸，避免僵尸进程

**与 syscall**（**10.4.4**）：

```
fork  → sys_clone(flags=SIGCHLD)
exec  → sys_execve
exit  → sys_exit_group
```

**隔离意义**：

- 进程 A 越界访问 → **MMU fault**，通常不影响 B
- RTOS 任务：**同地址空间**，错误指针破坏全局（**10.9** 对比）

**嵌入式 Linux**：

- `init`（PID 1）spawn 各 daemon
- **容器** = namespace 限制后的「进程视图」

**与 **10.5** 文件**：

- 每进程独立 **fd**；`open` 返回 int fd
- `fork` 后 fd **共享引用计数**；`exec` 常 `FD_CLOEXEC`
