# 10.10.2 线程

## 本节讲什么

**线程** 共享所属进程的 **地址空间与 FD**，独立 **栈、寄存器、调度**；是 Linux 与 **RTOS 任务** 最接近的概念。本节介绍 **`pthread`、clone 标志**、与 **FreeRTOS 任务** 的对照及 **SMP 绑核**。

## 要点

**线程 vs 进程**：

```
进程 P
 ├── 线程 1（主线程）─ 栈1
 ├── 线程 2         ─ 栈2  ─┐
 └── 线程 3         ─ 栈3   ├─ 共享 heap、全局、代码
                            └─ 需 mutex（**10.2.4**）
```

**pthread 示例**：

```c
void *worker(void *arg) {
    /* 独立栈；共享 g_counter */
    pthread_mutex_lock(&mtx);
    g_counter++;
    pthread_mutex_unlock(&mtx);
    return NULL;
}

pthread_t t;
pthread_create(&t, NULL, worker, NULL);
pthread_join(t, NULL);
```

**内核实现**：

- `pthread_create` → `clone(CLONE_VM | CLONE_FILES | ...)` 
- 与 **fork** 区别：**不复制** mm，共享页表

**RTOS 任务对照**：

| FreeRTOS | Linux pthread |
|----------|---------------|
| `xTaskCreate` | `pthread_create` |
| 任务栈 | `pthread_attr_setstacksize` |
| `vTaskDelay` | `nanosleep` |
| 优先级 0–configMAX | nice/ SCHED_FIFO |
| `vApplicationStackOverflowHook` | guard page + valgrind |

**uC/OS-III / FreeRTOS Hook**：

- `Idle Hook`：低功耗 WFI
- `StackOverflowHook`：调试栈（**10.2.2** canary 互补）

**SMP 与 DPDK**：

```c
/* DPDK：每 lcore 一线程，绑 CPU，无抢占式内核调度干扰 */
rte_eal_init(...);
RTE_LCORE_FOREACH_WORKER(lcore_id) {
    rte_eal_wait_lcore(lcore_id);
}
```

- **并行** 跑在物理核；与 **10.1 并发** 区分

**陷阱**（全书 recurring）：

- **未保护全局** → 数据竞争
- **栈过大** 在进程内仍占虚拟空间
- **线程频繁创建** → 用 **10.10.3 线程池**
