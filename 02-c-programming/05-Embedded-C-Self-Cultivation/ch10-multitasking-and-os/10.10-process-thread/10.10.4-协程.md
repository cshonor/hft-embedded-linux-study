# 10.10.4 协程

## 本节讲什么

**协程（Coroutine）** 是 **用户态** 轻量级执行单元，**主动 yield** 切换，无需内核参与。本节对比 **线程/RTOS 任务**，说明 **适用场景与 C 语言实现思路**（`ucontext` 已废弃，可提 **libco/自建栈**）。

## 要点

**切换成本排序**：

```
协程 yield  <  RTOS 任务切换  <  线程切换  <  进程切换
   用户态          内核/汇编          内核+TLB      fork/mm
```

**协程 vs 线程**：

| | 协程 | 线程 |
|---|------|------|
| 调度 | 用户库 | 内核 |
| 并行 | 单核并发 | 可多核并行 |
| 阻塞 syscall | 阻塞整个线程（需多线程+协程） | 内核调度其他线程 |
| 栈 | 小栈或共享 | 内核分配 |

**典型用途**：

- 高并发 **网络服务器**（C20 前 Nginx 用 **伪协程** + 多进程）
- **状态机** 写异步逻辑像同步代码（async/await 语言内置）

**C 实现思路**（概念）：

1. **独立栈 + 上下文切换**（同 **10.1.1** TCB，但在用户态）
2. **`setjmp/longjmp`** 教学版（限制多）
3. **编译器纤程**（C++20 coroutine，C 需第三方）

**与 RTOS 协作式调度**（**10.2.1**）：

- 协作 RTOS 任务 ≈ **内核态协程 + 独立栈**
- 协程缺 **抢占**；长计算需主动 yield

**与 **10.1** 并发**：

- 单线程 + 多协程 = **并发** 处理多连接
- 多线程 × 每线程多协程 = 常见 **M:N** 模型

**局限**：

- 不能利用多核（除非多线程各跑协程调度器）
- 阻塞式 `read` 会卡住同线程内所有协程
