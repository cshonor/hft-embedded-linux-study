# 10.10 进程、线程和协程

## 本节讲什么

Linux 上 **进程、线程、协程** 是三种不同的 **执行流抽象**；与 **RTOS 任务** 对照，理解 **隔离、切换成本、适用场景**。本节总览 **10.10.1–10.10.5**，收束全书 **并发 vs 并行、syscall、MMU** 主线。

## 要点

**三者对比**：

| | 进程 | 线程 | 协程 |
|---|------|------|------|
| 地址空间 | 独立 | 共享 | 共享（同线程内） |
| 切换成本 | 高（页表、FD） | 中 | 低（用户态） |
| 创建 API | `fork/exec` | `pthread_create` | `ucontext`/语言内置 |
| 阻塞 | 独立 | 阻塞不卡其他线程 | yield 让出 |
| 典型 OS | Linux | Linux/RTOS | Go/async C |

**与 RTOS 任务对照**：

| RTOS 任务 | 更接近 |
|-----------|--------|
| FreeRTOS 任务 | 线程（共享地址空间）+ 抢占 |
| 无 MMU 多任务 | 线程但 **无隔离** |
| Linux 进程 | 独立 MMU + 资源 |

**并行 vs 并发**（**10.1**）：

- **多进程/多线程** 在多核上 → **并行**
- **单核多线程** → **并发**（内核时间片）
- **DPDK lcore**：线程绑核 + 轮询 → **避免内核调度**，追求并行

**同步共性**（**10.2.4**）：

- Linux：`pthread_mutex`, `sem_t`, `eventfd`
- RTOS：`xSemaphore`, `xQueue`, EventGroup
- 问题同源：**死锁、优先级反转、饿死**

**选型速查**：

| 需求 | 选择 |
|------|------|
| 强隔离、独立崩溃 | 多进程 |
| 共享数据多、要 SMP | 多线程 + 锁 |
| 海量连接、I/O 密集 | 协程/异步 |
| 硬实时、确定延迟 | RTOS 任务或 RT 线程 |
