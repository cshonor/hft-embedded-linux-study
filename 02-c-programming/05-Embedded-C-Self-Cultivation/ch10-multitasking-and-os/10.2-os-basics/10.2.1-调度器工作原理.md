# 10.2.1 调度器工作原理

## 本节讲什么

调度器决定 **哪个任务在何时获得 CPU**。本节对比 **协作式与抢占式**、**就绪/阻塞状态**、**优先级与 bitmap 就绪表**，并说明 **blocking** 如何释放 CPU，对应 **demo02_tcb_coop**、**demo03_preempt**、**demo06_mini_rtos** 的核心循环。

## 要点

**任务状态机**：

```
        create
          │
          ▼
    ┌──► READY ◄─── 时间片到/被唤醒
    │      │
    │      │ 被调度器选中
    │      ▼
    │   RUNNING ──► BLOCKED（等 sem/queue/delay）
    │      │              │
    │      │ exit         │ signal/post
    │      ▼              │
    └─ TERMINATED    ──────┘
```

**协作式调度**（**demo02_tcb_coop**）：

```c
void os_schedule(void) {
    tcb_t *next = pick_next_ready();  /* 轮转或优先级 */
    if (next != current)
        context_switch(current, next);
}

void os_yield(void) {
    os_schedule();  /* 仅在主动调用点切换 */
}
```

**抢占式调度**（**demo03_preempt**）：

- **SysTick** 或时间片定时器中断 → 标记 `need_reschedule`
- **PendSV**（ARM）或软中断中执行真正的 `context_switch`，避免在 SysTick 里做重活（**10.3.1**）

**优先级调度 + Ready Bitmap**：

```c
#define MAX_PRIO 32
uint32_t ready_bitmap;           /* bit i = 1 表示优先级 i 有就绪任务 */
tcb_t *ready_list[MAX_PRIO];     /* 每优先级一条链表 */

tcb_t *pick_highest(void) {
    int p = __builtin_clz(ready_bitmap);  /* 找最高位 1 */
    return ready_list[p]->head;
}
```

- O(1) 找最高优先级；同优先级 **时间片轮转**
- **FreeRTOS**：`configMAX_PRIORITIES`；**uC/OS**：0 最高或最低取决于移植

**Blocking 释放 CPU**：

```c
void os_sem_wait(sem_t *s) {
    if (s->count == 0) {
        current->state = BLOCKED;
        current->wait_obj = s;
        ready_bitmap_clear(current->priority);
        os_schedule();   /* 立即切走 */
    }
    s->count--;
}
```

- 阻塞任务 **不参与** 时间片竞争 → 提高 CPU 利用率
- 唤醒：`sem_post` → 任务回 READY → 若优先级更高则 **抢占**（**demo04_semaphore**）

**调度策略对比**：

| 策略 | 延迟可预测性 | 实现复杂度 | 典型 |
|------|--------------|------------|------|
| 协作 | 差（依赖 yield） | 低 | 早期 uC/OS-I |
| 抢占 + 优先级 | 好 | 中 | FreeRTOS |
| 完全公平 CFS | 吞吐优 | 高 | Linux 内核 |

**DPDK 侧记**：用户态 **lcore 绑定** 物理核，轮询模式避免内核调度，是 **并行** 而非 RTOS 式 **并发**——与 MCU RTOS 问题域不同，但「一核一线程」思想相通。
