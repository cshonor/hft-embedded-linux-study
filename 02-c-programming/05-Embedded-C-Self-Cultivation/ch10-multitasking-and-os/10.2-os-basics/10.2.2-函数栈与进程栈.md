# 10.2.2 函数栈与进程栈

## 本节讲什么

**函数调用栈**（单任务内的 call stack）与 **任务/进程栈**（每任务独立栈空间）是不同层次的概念。本节衔接 **ch05 栈布局**、**ch03 ARM 栈指令**，说明 TCB 如何持有栈顶、如何检测 **栈溢出**，以及多任务下 **为何不能共用主栈**。

## 要点

**单任务函数栈**（**ch05**）：

```
高地址
  │  参数、返回地址
  │  保存的 callee-saved 寄存器
  │  局部变量
  ▼ SP   栈向下增长（ARM/AAPCS）
低地址
```

- 每次函数调用 **压栈**；返回 **弹栈**
- 递归过深 → **栈溢出** 覆盖 `.bss`/堆/其他区域

**每任务独立栈**（**demo02_tcb_coop**）：

```c
#define TASK_STACK_SIZE 512
uint32_t task_a_stack[TASK_STACK_SIZE];
uint32_t task_b_stack[TASK_STACK_SIZE];

tcb_t task_a = {
    .stack_top = &task_a_stack[TASK_STACK_SIZE],  /* 栈顶（高地址端） */
    .entry = task_a_fn,
};
```

- 上下文切换只改 **该任务的 SP**；任务 A 的局部变量与 B 完全隔离
- 静态分配栈数组 vs `malloc`：嵌入式优先静态，便于链接脚本定址

**TCB 中与栈相关字段**（**ch08 封装**）：

```c
typedef struct os_task {
    uint32_t *stack_ptr;      /* 当前 SP 快照 */
    uint32_t *stack_limit;    /* 栈底（最低合法地址） */
    uint32_t *stack_base;     /* 栈顶（初始化用） */
    size_t    stack_size;
    /* priority, state, name ... */
} os_task_t;
```

**栈初始化（首次运行）**：

- 在栈顶 **伪造** 异常栈帧：`xPSR, PC=entry, LR=task_exit_trap, R0=arg, ...`
- 首次 `context_switch` **恢复** 后 CPU 从 `entry` 开始执行

**栈溢出与 Canary**（**ch05**）：

```c
#define STACK_CANARY 0xDEADBEEF
void stack_init(uint32_t *base, size_t words) {
    base[0] = STACK_CANARY;
    base[1] = STACK_CANARY;
}

bool stack_check(uint32_t *limit) {
    return limit[0] == STACK_CANARY && limit[1] == STACK_CANARY;
}
```

| 检测方式 | 说明 |
|----------|------|
| **Canary** | 调度前检查魔数 |
| **MPU  guard region** | 栈底设不可访问页（**10.9**） |
| **Hook** | FreeRTOS `configCHECK_FOR_STACK_OVERFLOW` |

**函数栈 vs 任务栈关系**：

- 任务运行时，其 **函数调用栈** 就在该 **任务栈** 空间内生长
- 中断可能使用 **独立中断栈**（**10.3.2**），避免占任务栈深度

**多任务陷阱**：

- 在 **共用主栈** 上做 `setjmp` 切换 → 后运行任务破坏先运行任务栈帧
- 任务栈 **估太小** → 静默破坏 TCB 或邻栈 → 极难调试
