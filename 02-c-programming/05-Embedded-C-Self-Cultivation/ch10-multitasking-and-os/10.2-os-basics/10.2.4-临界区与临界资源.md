# 10.2.4 临界区与临界资源

## 本节讲什么

**临界资源** 同一时刻只允许一个执行流访问；访问它的代码段叫 **临界区（Critical Section）**。本节讲解 **关中断、mutex、semaphore、queue、event group** 的选型，以及 **优先级反转** 与 **死锁**，对应 **demo04_semaphore**、**demo05_queue**。

## 要点

**临界资源示例**：

- 全局环形缓冲区、UART 硬件、共享配置文件、动态内存堆

**保护手段层次**：

```
关中断（最短、单核有效）
    ↓
自旋锁 spinlock（多核短临界，内核/DPDK）
    ↓
mutex（可能阻塞，RTOS/Linux 用户态）
    ↓
semaphore / queue（同步 + 资源计数）
```

**关中断临界区**（单核 MCU）：

```c
uint32_t primask = __get_PRIMASK();
__disable_irq();
/* 临界区：几微秒内完成 */
shared_counter++;
__set_PRIMASK(primask);
```

- 不能关太久 → 影响中断响应与调度 tick
- 多核无效 → 需 spinlock + 内存屏障

**Mutex（互斥量）**（**demo04**）：

```c
os_mutex_lock(&uart_mtx);
uart_write(buf, len);
os_mutex_unlock(&uart_mtx);
```

- **优先级继承**：持锁任务临时提升到等待者优先级，缓解 **优先级反转**
- FreeRTOS：`xSemaphoreCreateMutex()` + `configUSE_MUTEXES`

**Semaphore（信号量）**：

| 类型 | 初值 | 典型用途 |
|------|------|----------|
| 二进制 | 0/1 | 事件通知、同步 |
| 计数 | N | 资源池（N 个缓冲区） |

```c
os_sem_wait(&rx_sem);   /* 无数据则 BLOCKED */
process(packet);
os_sem_post(&tx_sem);
```

**Queue（消息队列）**（**demo05_queue**）：

- 传递 **数据副本** 或指针（需约定生命周期）
- 天然解耦生产者/消费者；阻塞式 `recv` 触发调度（**10.2.1**）

**Event Group（事件组）**：

- 多位标志：`wait(TASK_A_BIT | TASK_B_BIT, AND/OR)`
- 适合「等多条件就绪」；FreeRTOS `EventGroup`

**优先级反转**（经典案例）：

```
L 持 mutex ──► M 抢占 L ──► H 等 mutex 被 M 间接阻塞
```

对策：优先级继承 mutex、避免 L/M 长时间占 CPU

**死锁四条件**（破坏任一即可）：

1. 互斥 2. 持有并等待 3. 不可剥夺 4. 循环等待

- 嵌入式对策：**固定加锁顺序**、**trylock 超时**、减少嵌套锁

**选型速查**：

| 需求 | 选用 |
|------|------|
| 保护短全局变量 | 关中断 / mutex |
| 任务间传结构体 | queue |
| 通知「发生了一件事」 | 二进制 sem / event |
| 限流 N 同类资源 | 计数 sem |
