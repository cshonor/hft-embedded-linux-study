# 10.2 操作系统基本原理

## 本节讲什么

从裸机多任务过渡到 **操作系统（OS）** 与 **RTOS**：内核提供调度、同步、内存抽象，应用通过 API 使用资源而非直接操作硬件。本节总览 **协作 vs 抢占**、**TCB/栈**、**同步原语** 与 **迷你 RTOS 分层**（**ch08 TCB OOP + ch09 模块化**），为 **10.2.1–10.2.4** 展开。

## 要点

**OS 在嵌入式中的角色**：

```
应用任务 ──API──► RTOS 内核 ──► 驱动/HAL ──► 硬件
              │调度│同步│内存│时钟│
```

| 能力 | 裸机 | RTOS | Linux |
|------|------|------|-------|
| 多任务 | 手写 | 内核调度 | 进程/线程 |
| 同步 | 关中断 | mutex/sem/queue | futex/pthread |
| 内存保护 | 无 | 可选 MPU | MMU（**10.9**） |
| 系统调用 | 无 | 可选 SVC | 完整 syscall（**10.4**） |

**调度模型**（**10.2.1** 详述）：

- **协作式**：`os_yield()` 才切换；实现简单，依赖任务自律
- **抢占式**：时钟/中断强制切换；FreeRTOS、uC/OS-III 默认模式
- **优先级 bitmap**：32/256 级优先级 O(1) 找最高就绪任务

**任务与栈**（**10.2.2**、**ch05**）：

- 每任务 **独立栈** + **TCB**（**ch08** 可封装为 `os_task_t` 不透明类型）
- **栈溢出检测**：栈底填 **canary** 魔数 `0xDEADBEEF`，调度前检查（**ch05 5.x**）
- **栈大小估算**：最大调用深度 + 局部变量 + 中断嵌套预留

**同步机制一览**（**10.2.4**、**demo04/05**）：

| 原语 | 用途 |
|------|------|
| **mutex** | 互斥访问资源；可优先级继承 |
| **semaphore** | 计数信号；资源池、同步 |
| **queue** | 任务间传数据；解耦生产者/消费者 |
| **event group** | 多位事件组合等待 |

**经典并发问题**：

- **优先级反转**：低优先级持锁，高优先级阻塞等锁，中优先级抢占低 → 高优先级饿死；对策：**优先级继承 mutex**
- **死锁**：互等锁；对策：固定加锁顺序、超时、避免嵌套

**迷你 RTOS 分层**（**demo06_mini_rtos**、**ch09**）：

```
app/          业务任务
kernel/       调度、TCB、tick
sync/         mutex, sem, queue
port/         上下文切换汇编、SysTick
bsp/          板级初始化
```

**商业 RTOS 钩子**：FreeRTOS `vApplicationStackOverflowHook`、`Idle Hook`；uC/OS `OSTaskCreateHook`——用于调试与低功耗。
