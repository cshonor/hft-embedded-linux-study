# 8.4 Linux内核中的OOP思想：继承

## 本节讲什么

内核大量使用 **struct 嵌套**模拟继承：`platform_device` 内含 `struct device`；`net_device` 扩展通用网络接口。首成员布局保证 **`to_platform_device(dev)`** 等宏能安全向下转型。本节总览继承在内核中的三种形态：**内嵌基类**、**抽象基类 + 纯虚 probe**、**多接口 mixin**。

## 要点

**继承模式对照**：

| 模式 | 内核示例 | C 技巧 |
|------|----------|--------|
| **单继承** | `platform_device.device` | 首成员 + `container_of` |
| **抽象基类** | `struct device` 不可单独使用 | 仅通过子类 `register` |
| **接口分离** | `struct dev_pm_ops` | 独立 ops 指针挂到 device |

```c
struct platform_device {
    const char *name;
    struct device dev;          /* 基类 — 通常放末尾或首部，内核用 dev 字段 */
    u32 num_resources;
    struct resource *resource;
};

#define to_platform_device(dev) \
    container_of(dev, struct platform_device, dev)
```

- **`dev`** 字段名固定，所有 bus 子类均可 `to_*_device(dev)` 转换
- 驱动 `probe` 收到 `struct device *`，第一时间 `to_platform_device` 取子类字段
- 与用户态 **demo02** `UartDev.base` 同构
