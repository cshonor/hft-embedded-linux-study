# 8.4.3 继承与接口

## 本节讲什么

C **不支持多重继承**，但可通过 **多个 ops 指针**（mixin / 接口分离）让对象实现多种能力：通信走 `CommOps`，电源走 `PowerOps`，与主 `DevOps` 并列。Linux **`dev_pm_ops`**、**`dma_ops`** 即独立接口 struct，挂到 `device` 上由子系统分别调用——避免一个巨型 ops 表。

## 要点

**多接口布局**（**demo05_multi_iface**）：

```c
struct CommOps {
    int (*send)(void *ctx, const void *buf, size_t len);
    int (*recv)(void *ctx, void *buf, size_t len);
};

struct PowerOps {
    int (*suspend)(void *ctx);
    int (*resume)(void *ctx);
};

struct SmartSensor {
    struct Device  base;       /* 主继承线 */
    struct CommOps *comm;      /* 接口 1 */
    struct PowerOps *power;    /* 接口 2 */
    void           *ctx;
};

static struct CommOps i2c_comm_ops = { .send = i2c_send, .recv = i2c_recv };
static struct PowerOps generic_pm_ops = { .suspend = pm_suspend, .resume = pm_resume };

/* 初始化 */
sensor.comm  = &i2c_comm_ops;
sensor.power = &generic_pm_ops;
```

| 设计 | 说明 |
|------|------|
| **单一巨型 ops** | 简单驱动 OK；子系统交叉时臃肿 |
| **多 ops 指针** | 各子系统只依赖自己的接口头文件 |
| **首成员 + 多 ops** | 内核 `device` + `driver` + `class` 组合 |

**内核示例**：

```c
struct device {
    struct dev_pm_ops *pm;      /* 电源管理接口 */
    struct dev_dma_ops *dma;    /* DMA 映射接口 */
    /* bus、driver、class 各管一块 */
};
```

- 接口 struct 通常 **只含函数指针**，无数据 — 类似 Java `interface`
- 同一 `PowerOps` 可被多种设备 **共享**（静态全局表）
- 调用方检查：`if (dev->pm && dev->pm->suspend) dev->pm->suspend(dev);`
