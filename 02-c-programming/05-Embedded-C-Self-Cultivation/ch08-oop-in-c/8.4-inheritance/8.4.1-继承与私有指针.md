# 8.4.1 继承与私有指针

## 本节讲什么

除首成员继承外，C 常用 **PIMPL（pointer to implementation）**：公开 struct 只含一个 `void *priv` 或 `struct xxx_private *`，真实字段在 `.c` 堆分配。子类「继承」时 priv 指向含基类状态的结构。内核 **`net_device`** 的 `ml_priv`、`ieee80211_ptr` 等即是把协议私有数据与通用 netdev 分离。

## 要点

**私有指针模式**：

```c
/* public.h */
typedef struct widget widget_t;

widget_t *widget_create(void);
void      widget_destroy(widget_t *w);
int       widget_draw(widget_t *w);

/* widget.c */
struct widget_private {
    int x, y;
    struct theme *theme;   /* 「基类」状态也可放 priv */
};

struct widget {
    struct widget_private *priv;   /* 仅一个指针对外可见 */
};

widget_t *widget_create(void) {
    widget_t *w = kzalloc(sizeof(*w), GFP_KERNEL);
    w->priv = kzalloc(sizeof(*w->priv), GFP_KERNEL);
    return w;
}
```

| 方式 | 优点 | 缺点 |
|------|------|------|
| **首成员继承** | 零间接、可 container_of | 子类布局暴露于同一头文件 |
| **私有指针** | 完全隐藏大小与字段 | 多一次间接；需配对 alloc/free |
| **混合** | 公开基类 + priv 扩展 | 内核 netdev 常见 |

**继承 + priv**：

```c
struct uart_dev {
    struct Device base;           /* 公开继承，框架可见 */
    struct uart_private *priv;    /* 寄存器映射等隐藏 */
};
```

- **`dev_get_drvdata`** 等价于在 `struct device` 上挂 priv
- 用户态 **demo01** 不透明类型是 priv 的极端形式（整个 struct 都隐藏）
- 避免在头文件暴露 `struct xxx_private`，否则封装失效
