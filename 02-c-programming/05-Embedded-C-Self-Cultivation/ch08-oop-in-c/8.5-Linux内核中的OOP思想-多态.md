# 8.5 Linux内核中的OOP思想：多态

## 本节讲什么

**多态**是同一调用点根据对象类型执行不同代码。内核与 DPDK 中无处不在：**VFS** 根据打开的文件类型调用不同 `read`；**网卡驱动**填 `net_device_ops`；**DPDK** 用 `eth_dev_ops` 的 `rx_pkt_burst`/`tx_pkt_burst` 在 poll 模式驱动间切换。本节串联 **8.2.3–8.2.4** 的 ops 表与真实子系统。

## 要点

**VFS：`file_operations` 虚表**：

```c
struct file_operations {
    ssize_t (*read)(struct file *, char __user *, size_t, loff_t *);
    ssize_t (*write)(struct file *, const char __user *, size_t, loff_t *);
    int (*open)(struct inode *, struct file *);
    int (*release)(struct inode *, struct file *);
    /* ioctl, mmap, poll ... */
};

/* 系统调用层 */
static ssize_t vfs_read(struct file *file, ...) {
    if (file->f_op->read)
        return file->f_op->read(file, ...);
    return -EINVAL;
}
```

**字符设备注册**：

```c
static struct file_operations my_fops = {
    .owner = THIS_MODULE,
    .read  = my_read,
    .write = my_write,
    .open  = my_open,
};

cdev_init(&cdev, &my_fops);
cdev_add(&cdev, devno, 1);
```

**DPDK `eth_dev_ops`（示意）**：

```c
struct eth_dev_ops {
    eth_rx_burst_t rx_pkt_burst;   /* 必填 — 纯虚 */
    eth_tx_burst_t tx_pkt_burst;
    eth_dev_configure_t dev_configure;
    eth_dev_start_t dev_start;
    /* stats, promiscuous, ... */
};

/* 应用统一入口 */
nb_rx = port->dev_ops->rx_pkt_burst(rx_queue, pkts, nb_pkts);
```

| 子系统 | ops 结构 | 多态入口 |
|--------|----------|----------|
| **VFS** | `file_operations` | `vfs_read/write/ioctl` |
| **Block** | `block_device_operations` | `submit_bio` |
| **Net** | `net_device_ops` | `ndo_start_xmit` |
| **DPDK eth** | `eth_dev_ops` | `rte_eth_rx_burst` |
| **Platform** | `platform_driver` | `bus probe` |

**与用户态 demo 对应**：

```c
/* demo03 — 与内核同构 */
devices[i]->ops->send(devices[i], buf, len);
```

- ops 表在 **编译期** 填好，**运行期** 仅间接调用 — 无 C++ vtable 指针 per object 开销（通常共享静态 ops）
- **RCU** 替换 ops 需内核同步机制；DPDK 热路径避免运行时改 ops
- **module_get(THIS_MODULE)** 防 ops 执行时模块卸载
