# 8.2 面向对象编程基础

## 本节讲什么

**面向对象编程（OOP）** 用「对象」组织数据与行为，强调**封装、继承、多态**三大特性。C 语言没有 class 关键字，但 Linux 内核、DPDK、RTOS 驱动框架普遍用 **struct + 函数指针 + ops 表** 模拟 OOP。本节是 **8.2.1–8.2.4** 的总览：建立 C 与 C++/Java OOP 概念的映射表，为后续内核源码阅读打底。

## 要点

**C ↔ OOP 概念映射**：

| OOP 概念 | C 语言实现 | 内核/DPDK 示例 |
|----------|------------|----------------|
| **类（class）** | `struct` + 配套 API | `struct net_device` |
| **对象（object）** | struct 变量或 `malloc` 实例 | `struct uart_dev uart0` |
| **成员变量** | struct 字段 | `int baudrate` |
| **成员方法** | 以对象指针为第一参数的函数 | `netdev->ops->ndo_open()` |
| **虚函数 / 多态** | 函数指针 / ops 结构体 | `file_operations` |
| **继承** | **基类 struct 作为首成员** | `UartDev` 内嵌 `Device base` |
| **接口** | 只含函数指针的 struct | `PowerOps`、`CommOps` |

```c
/* 最小「类」骨架 */
struct counter {
    int value;
};

void counter_inc(struct counter *self) {
    self->value++;
}
```

- OOP 解决的是**大规模代码的组织问题**，不是语法糖；C 用约定与 discipline 达到同样目标
- **首成员继承**依赖 C 标准「struct 首地址 = 首成员地址」，可用 `(Device *)&uart_dev` 向上转型
- **ops 表**把行为从数据分离，同一框架代码遍历 heterogeneous 设备列表
