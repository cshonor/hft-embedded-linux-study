# 8.2.1 什么是 OOP

## 本节讲什么

**面向对象编程**以「对象」为中心：每个对象封装**状态（数据）**与**行为（方法）**，对外只暴露有限接口。四大支柱——**封装、抽象、继承、多态**——在 C 驱动框架中均有对应实现。理解 OOP 术语有助于阅读 LKD、`struct device`、DPDK `eth_dev_ops` 等源码，即便你永远不写 C++。

## 要点

**四大特性**：

| 特性 | 含义 | C 模拟 |
|------|------|--------|
| **封装** | 隐藏内部实现，只暴露 API | 不透明 struct + `.c` 内实现 |
| **抽象** | 提取共性，忽略细节 | `DevOps` 只声明 open/read/write |
| **继承** | 子类扩展/复用父类 | 基类 struct 作首成员 |
| **多态** | 同一接口，不同实现 | 函数指针运行时跳转 |

```c
/* 抽象：所有字符设备都能 read/write */
struct CharDevOps {
    int (*read)(void *ctx, char *buf, size_t len);
    int (*write)(void *ctx, const char *buf, size_t len);
};

/* 多态：uart 与 spi 各自填 ops */
static int uart_read(void *ctx, char *buf, size_t len) { /* ... */ }
static int spi_read(void *ctx, char *buf, size_t len) { /* ... */ }

struct CharDevOps uart_ops = { .read = uart_read, .write = uart_write };
struct CharDevOps spi_ops  = { .read = spi_read,  .write = spi_write };

/* 统一调用 */
dev->ops->read(dev->ctx, buf, len);
```

**与面向过程对比**：

- 面向过程：`uart_read()`、`spi_read()` 分散，调用方需 `switch(type)`
- 面向对象：调用方只认 `ops->read`，新增设备类型无需改框架

**C 的局限与对策**：

| 局限 | 对策 |
|------|------|
| 无访问控制 | 命名约定 `_` 前缀 + 不透明类型 |
| 无单继承语法 | 首成员嵌入 + 强制转换 |
| 无多重继承 | **mixin**：多个 ops 指针并列（**demo05**） |
| 无析构自动调用 | 显式 `xxx_destroy()` + 配对 `create` |

- **对象** = 一块内存（struct 实例）+ 与之绑定的行为（函数或 ops）
- 内核中「对象」常为静态全局（如 `platform_device`）或 `kmalloc` 分配
