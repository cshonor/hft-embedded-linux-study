# 8.2.2 类的封装与实例化

## 本节讲什么

**封装**把 struct 内部字段藏起来，头文件只暴露**不透明类型**与 **create / destroy / get / set** 等有限 API。调用方无法直接改 `value`，必须通过 `counter_inc()`，从而维护不变量（如不出现负数计数）。**实例化**即创建对象：栈上静态实例或堆上 `malloc`，对应 C++ 的栈对象与 `new`。

## 要点

**不透明类型模式**（**demo01_encapsulation**）：

```c
/* counter.h — 公开接口 */
typedef struct counter counter_t;

counter_t *counter_create(int initial);
void         counter_destroy(counter_t *c);
int          counter_get(const counter_t *c);
void         counter_inc(counter_t *c);

/* counter.c — 隐藏实现 */
struct counter {
    int value;
};

counter_t *counter_create(int initial) {
    counter_t *c = malloc(sizeof *c);
    if (!c) return NULL;
    c->value = initial;
    return c;
}

void counter_destroy(counter_t *c) {
    free(c);
}
```

| API 角色 | 函数命名惯例 | 说明 |
|----------|--------------|------|
| 构造 | `xxx_create` / `xxx_new` | 分配并初始化，失败返回 NULL |
| 析构 | `xxx_destroy` / `xxx_del` | 释放资源，接受 NULL 安全 |
| 读 | `xxx_get_*` | `const` 对象指针，不修改状态 |
| 写 | `xxx_set_*` / `xxx_inc` | 内部校验后修改 |

**静态 vs 堆实例**（**demo04_lifecycle**）：

```c
/* 静态：生命周期 = 整个程序，零 malloc 开销 */
static counter_t sys_counter = { .value = 0 };

/* 堆：动态数量、运行期创建 */
counter_t *c = counter_create(0);
/* ... */
counter_destroy(c);
```

| 方式 | 优点 | 风险 |
|------|------|------|
| **静态/栈** | 无碎片、确定性 | 不可动态扩容；注意初始化顺序 |
| **堆** | 灵活、可传指针 | 必须配对 destroy；泄漏/双重释放 |

- **`const counter_t *`** 承诺不通过该指针修改对象（仅 get 类 API）
- 头文件**不**暴露 `struct counter { ... }` 完整定义 → 无法在外部 `c->value = 999`
- 内核常用 `kzalloc` + `kfree` 替代 `malloc`/`free`；RTOS 用静态池
