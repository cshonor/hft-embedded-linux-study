# 8.2.3 继承与多态

## 本节讲什么

**继承**让子类型复用父类型字段与行为；C 通过把**基类 struct 作为首成员**模拟单继承。**多态**指通过统一接口调用不同实现——运行时根据对象绑定的 **ops / 函数指针** 跳转。UART 与 SPI 都「是」Device，框架遍历 `Device *` 列表并调用各自的 `send()`，无需 `switch(dev_type)`。

## 要点

**单继承：首成员嵌入**（**demo02_inherit**）：

```c
struct Device {
    const char *name;
    struct DevOps *ops;
};

struct UartDev {
    struct Device base;   /* 必须第一个成员 */
    uint32_t    reg_base;
    int         baudrate;
};

struct SpiDev {
    struct Device base;
    int         cs_pin;
};

/* 向上转型：UartDev* → Device* */
struct UartDev uart = { .base = { .name = "uart0" }, .baudrate = 115200 };
struct Device *dev = (struct Device *)&uart;
```

**多态：DevOps 虚表**（**demo03_polymorphism**）：

```c
struct DevOps {
    int (*init)(struct Device *dev);
    int (*send)(struct Device *dev, const void *buf, size_t len);
    int (*recv)(struct Device *dev, void *buf, size_t len);
};

static struct DevOps uart_ops = {
    .init = uart_init,
    .send = uart_send,
    .recv = uart_recv,
};

static struct DevOps spi_ops = {
    .init = spi_init,
    .send = spi_send,
    .recv = spi_recv,
};

/* 运行时分派 */
int device_send(struct Device *dev, const void *buf, size_t len) {
    if (!dev || !dev->ops || !dev->ops->send)
        return -1;
    return dev->ops->send(dev, buf, len);
}
```

| 步骤 | 操作 |
|------|------|
| 定义基类 | `struct Device` + 公共字段 |
| 定义 ops | `struct DevOps` 函数指针表 |
| 定义子类 | 首成员 `struct Device base` + 私有字段 |
| 绑定 ops | 子类初始化时 `dev->ops = &uart_ops` |
| 多态调用 | `dev->ops->send(dev, ...)` |

**向下转型**需知真实类型或携带类型标签：

```c
struct UartDev *uart = container_of(dev, struct UartDev, base);
/* container_of 见 ch06 6.4 */
```

- 首成员规则保证 `(Device *)&uart_dev` 地址合法，与 C++ 子对象布局一致
- **不要**对非首成员的基类做此 trick；多重「继承」用多个 ops 指针（**8.4.3**、**demo05**）
