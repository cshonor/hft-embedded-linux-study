# 8.2.4 虚函数与纯虚函数

## 本节讲什么

C++ **虚函数**允许子类覆盖父类方法；**纯虚函数**（`= 0`）强制子类实现，父类不可实例化——即**抽象类**。C 中对应物是 **ops 结构体里的函数指针**：有默认 stub 的为「虚函数」，置 `NULL` 且调用前检查的为「纯虚」；整个只定义 ops、不提供具体 send 实现的 Device 框架即抽象类。

## 要点

**虚函数表（vtable）在 C 中的形态**：

```c
struct DevOps {
    int (*init)(struct Device *dev);      /* 可选：可有 weak 默认 */
    int (*send)(struct Device *dev, const void *buf, size_t len);
    int (*recv)(struct Device *dev, void *buf, size_t len);
    void (*shutdown)(struct Device *dev); /* 纯虚：必须子类实现或 NULL */
};
```

| C++ 概念 | C 实现 |
|----------|--------|
| 虚函数 | ops 指针 + 默认 weak 实现 |
| 纯虚函数 | ops 槽为 `NULL`，框架调用前检查 |
| vtable 指针 | `dev->ops` 指向静态 `uart_ops` |
| 抽象类 | 只暴露 API，无具体 `struct XxxDev` 实例化 |

**纯虚：调用前检查**：

```c
int device_shutdown(struct Device *dev) {
    if (!dev->ops || !dev->ops->shutdown) {
        fprintf(stderr, "shutdown not implemented\n");
        return -ENOTSUP;
    }
    dev->ops->shutdown(dev);
    return 0;
}
```

**静态 ops 表 vs 每对象 ops**：

```c
/* 常见：所有 uart 实例共享 uart_ops */
static struct DevOps uart_ops = { .send = uart_send, ... };
uart0.base.ops = &uart_ops;

/* 少见：每对象不同 ops（运行时替换策略） */
dev->ops = condition ? &fast_ops : &safe_ops;
```

- Linux **`file_operations`**：未实现的系统调用在 ops 中为 `NULL`，VFS 返回 `-ENOSYS`
- DPDK **`eth_dev_ops`**：`rx_pkt_burst` 必填，`stats_reset` 可选
- ops 通常 **`const` 静态全局**，放 `.rodata`，多实例共享，节省内存
- 函数指针调用无法内联，热路径可用 **static inline 包装** + 直接调用已知类型
