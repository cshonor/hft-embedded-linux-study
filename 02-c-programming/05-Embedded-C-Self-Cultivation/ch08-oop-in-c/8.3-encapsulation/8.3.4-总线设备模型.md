# 8.3.4 总线设备模型

## 本节讲什么

**总线（bus）**是设备与驱动的匹配枢纽：`struct bus_type` 定义 **match**（是否配对）、**probe**（绑定后初始化）、**remove**（解绑）回调。设备挂 `bus_list`，驱动挂 `driver_list`，内核自动扫描配对——这是 Linux **plug-and-play** 的 OOP 核心：**框架代码不变，新驱动只填 ops**。

## 要点

**总线结构（概念）**：

```c
struct bus_type {
    const char *name;
    int (*match)(struct device *dev, struct device_driver *drv);
    int (*probe)(struct device *dev);
    int (*remove)(struct device *dev);
    struct list_head devices;
    struct list_head drivers;
};
```

**platform 总线典型 match**：

```c
/* 按 name 或 OF compatible 匹配 */
static int platform_match(struct device *dev, struct device_driver *drv) {
    struct platform_device *pdev = to_platform_device(dev);
    return strcmp(pdev->name, drv->name) == 0;
}
```

| 总线 | 典型设备 | match 依据 |
|------|----------|--------------|
| **platform** | SoC 内置 UART | 设备名 / DT `compatible` |
| **pci** | 网卡 | Vendor/Device ID |
| **usb** | U 盘 | VID/PID + interface |
| **i2c** | 传感器 | 7-bit 地址 + OF |

**注册时序**：

```
platform_driver_register(drv)  ─┐
platform_device_register(dev) ─┼→ bus 核心 match → probe
                                ┘
driver_unregister / device_unregister → remove
```

- **`struct device_driver`** 含 `probe`/`remove`，与 **8.2.4 ops** 同类：驱动填函数指针，核心调度
- **sysfs**：`/sys/bus/platform/devices/`、`/sys/bus/platform/drivers/` 反映链表状态
- 模块卸载必须 `remove` 干净，否则 dangling `device`
