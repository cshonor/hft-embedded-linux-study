# 8.3.1 类的C语言模拟实现

## 本节讲什么

用 C 写「类」的**标准套路**：头文件声明不透明类型 + 操作函数；源文件定义完整 struct 并实现 **create / destroy / method**。这与 **8.2.2** 一致，但强调**模块边界**——`.h` 是 public API，`.c` 是 private implementation，对应内核驱动的「对外头文件 vs 内部实现」划分。

## 要点

**文件分工**（**demo01_encapsulation**）：

```
counter.h   — typedef struct counter counter_t; + API 声明
counter.c   — struct counter { int value; }; + 实现
main.c      — 只 #include "counter.h"
```

```c
/* counter.c */
struct counter {
    int value;
    /* 可添加 mutex、统计等，头文件不可见 */
};

counter_t *counter_create(int initial) {
    counter_t *c = kzalloc(sizeof(*c), GFP_KERNEL); /* 内核 */
    /* 或 malloc — 用户态 demo */
    if (!c)
        return NULL;
    c->value = initial;
    return c;
}
```

| 原则 | 说明 |
|------|------|
| **最小暴露** | 头文件零 struct 字段 |
| **配对生命周期** | 每个 `create` 对应 `destroy` |
| **错误码** | 内核返回负 errno；用户态返回 NULL + errno |
| **线程安全** | 若需锁，锁在 struct 内，API 内 acquire/release |

**与 C++ class 对照**：

| C++ | C |
|-----|---|
| `class Counter { private: int v; };` | `struct counter` 仅在 `.c` |
| `Counter(int n);` | `counter_create(n)` |
| `~Counter();` | `counter_destroy(c)` |
| `void inc();` | `counter_inc(c)` |
| `this` | 第一个参数 `counter_t *self` |

- 命名：`module_action` 前缀避免符号冲突（内核 `uart_*` 众多）
- **静态单例**：`static counter_t the_counter` + `counter_get_instance()` 适用于全局唯一对象
