# 8.3.2 链表的抽象与封装

## 本节讲什么

内核**侵入式链表**（intrusive list）把 `struct list_head` 嵌入宿主对象，通过 **`list_entry` / `container_of`** 遍历。链表操作被封装进 `list_add`、`list_del`、`list_for_each_entry` 宏，驱动作者无需手写 `next/prev` 指针逻辑——这是 C 里「容器类」的经典抽象。

## 要点

**嵌入与遍历**：

```c
struct uart_port {
    struct list_head list;   /* 链表节点 — 「成员」 */
    int              port_id;
    void __iomem    *membase;
};

static LIST_HEAD(uart_ports);   /* 链表头 */

/* 注册：封装 add */
void uart_port_register(struct uart_port *port) {
    list_add_tail(&port->list, &uart_ports);
}

/* 遍历：封装 iterator */
struct uart_port *p;
list_for_each_entry(p, &uart_ports, list) {
    pr_info("port %d\n", p->port_id);
}
```

**`container_of` 本质**（**ch06 6.4**）：

```c
#define list_entry(ptr, type, member) \
    container_of(ptr, type, member)

/* ptr 指向 list 成员 → 算出 uart_port 首地址 */
```

| API | 作用 |
|-----|------|
| `INIT_LIST_HEAD` | 初始化空链 |
| `list_add` / `list_add_tail` | 头插 / 尾插 |
| `list_del` | 摘除节点（需再 INIT 或 add） |
| `list_empty` | 判空 |
| `list_for_each_entry_safe` | 遍历中可删除 |

**封装收益**：

- 同一套 list 代码驱动 `task_struct`、`file`、`device` 等所有子系统
- 宿主 struct 可有多个 `list_head` 成员，挂到不同链表（如 `device` 的 `bus_list` 与 `driver_list`）
- **hlist**（单指针哈希链）是变种，用于大哈希表

- 摘除后节点：`list_del` + `INIT_LIST_HEAD` 防 use-after-free
- 用户态 demo 可复刻 mini `list.h` 理解宏展开
