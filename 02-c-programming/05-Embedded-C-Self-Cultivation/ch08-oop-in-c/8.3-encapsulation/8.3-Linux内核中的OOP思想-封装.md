# 8.3 Linux内核中的OOP思想：封装

## 本节讲什么

Linux 内核是 **C 模拟 OOP** 的教科书：**不透明抽象**、**list 封装**、**device/bus 分层**都体现封装思想。本节总览内核如何用「只暴露必要接口」控制复杂度——驱动作者操作 `struct device` 与 `list_head` 钩子，而非直接遍历全局数组；链表实现细节藏在 `list_add()` 内部。

## 要点

**内核封装三层**：

| 层次 | 机制 | 隐藏的内容 |
|------|------|------------|
| **类型封装** | 不完整 struct 声明 | 驱动私有数据布局 |
| **API 封装** | `list_add`/`list_del` | 指针 manip 细节 |
| **子系统封装** | `device_register` | 总线匹配、uevent、sysfs |

```c
/* include/linux/list.h — 公开 */
struct list_head {
    struct list_head *next, *prev;
};

static inline void list_add(struct list_head *new, struct list_head *head);

/* 驱动 embed 链表节点，不直接改 next/prev */
struct my_driver {
    struct list_head node;
    int irq;
};
```

- **`container_of`**（**ch06 6.4**）是从封装节点「解密」宿主对象的唯一推荐路径
- **`dev_set_drvdata` / `dev_get_drvdata`** 把私有指针挂在 `struct device` 上，进一步封装
- 错误示范：驱动直接访问 `bus->devices` 内部数组 → 内核版本升级即 break
