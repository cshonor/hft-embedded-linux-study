# 8.3.3 设备管理模型

## 本节讲什么

Linux **设备模型**用 `struct device` 统一表示物理或逻辑设备：名称、父设备、总线、驱动绑定、电源状态、sysfs 节点。驱动不直接操作全局表，而是通过 **`device_register`**、**`dev_set_name`**、**`dev_set_drvdata`** 等 API 与核心交互——设备对象是对硬件的**封装**。

## 要点

**`struct device` 核心字段（简化）**：

```c
struct device {
    struct device      *parent;
    struct bus_type    *bus;
    struct device_driver *driver;
    struct list_head    bus_list;
    void               *platform_data;  /* 或 dev_get_drvdata */
    /* ... 电源、DMA、of_node 等 */
};
```

| API | 作用 |
|-----|------|
| `device_initialize` | 初始化内嵌 kobject/list |
| `device_add` / `device_register` | 加入层次，触发 uevent |
| `device_unregister` | 反注册 |
| `dev_set_drvdata` / `dev_get_drvdata` | 绑定驱动私有数据 |
| `devm_*` | 设备生命周期托管内存（自动释放） |

**驱动绑定流程**：

```
device_register(dev)
    → 挂到 bus 的设备链表
    → bus_match(dev, driver)?
    → driver.probe(dev)  /* 驱动「构造」完成 */
```

```c
static int my_probe(struct device *dev) {
    struct my_chip *chip = devm_kzalloc(dev, sizeof(*chip), GFP_KERNEL);
    dev_set_drvdata(dev, chip);
    /* 映射 MMIO、注册 irq */
    return 0;
}
```

- **`parent` 指针**形成 `/sys/devices/...` 树形层次（PCI → USB → interface）
- **`platform_data` / DT**：板级数据与设备树通过 `device` 传入，驱动保持与板级解耦
- 用户态类比：**demo02** 中 `Device` 基类 + 设备列表管理
