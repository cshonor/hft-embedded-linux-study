# 6.4 指向结构的指针

## 本节讲什么

**结构体指针** 是底层代码的默认姿势：`**pp->member**` 等价于 **`(*pp).member`**。`.` 优先级 **高于** `*`，故 `*pp.member` 非法。本节串起 [6.2](./6.2-结构与函数.md) 传参、[6.3](./6.3-结构数组.md) 指针遍历，并为 [6.5](./6.5-自引用结构.md) 链表铺路。

**前置**：[6.1](./6.1-结构的基本知识.md)（`.`）、[5.2](../ch05-pointers-and-arrays/5.2-指针与函数参数.md)、[5.4](../ch05-pointers-and-arrays/5.4-地址算术运算.md)、[6.3](./6.3-结构数组.md)。

## 一、核心语法：`->` 箭头运算符

### 基础示例

```c
struct point { int x, y; };
struct point p = {10, 20};
struct point *pp = &p;    /* pp 存 p 的地址 */
```

### 两种等价写法

| 写法 | 说明 |
|------|------|
| **`(*pp).x`** | 先解引用得实体，再 `.` 取成员；**括号必须** |
| **`pp->x`** | 工程标准；`->` 等价 `(*pp).` |

```c
(*pp).x = 30;    /* 合法，冗长 */
pp->x = 30;      /* ✅ 惯用 */
```

- **`->` 左操作数** 必须是 **结构体指针**，不能是实体变量
- 实体变量用 **`.`**：`p.x = 10;`

### 致命语法坑：优先级

```c
*pp.x;    /* ❌ 非法：解析为 *(pp.x) */
```

**`.` 优先级 > `*`**：`pp` 是指针，没有 `pp.x` 这种写法。

```
正确：(*pp).x  或  pp->x
错误：*pp.x、pp.x（pp 为指针时）
```

### `.` vs `->` 对照（必考）

| 操作对象 | 运算符 | 示例 |
|----------|--------|------|
| **结构体实体** | `.` | `p.x`、`r.pt.y` |
| **结构体指针** | `->` | `pp->x`、`pr->width` |

混用（指针用 `.`、实体用 `->`）→ **编译报错**。

## 二、结构体指针作函数参数（承接 6.2）

内核、HFT、UEFI **统一** 结构体指针传参，杜绝整块拷贝：

```c
void sety(struct point *p, int y)
{
    p->y = y;          /* 改外部 p 指向的对象 */
}

void print_pt(const struct point *p)
{
    printf("x=%d,y=%d", p->x, p->y);
    /* p->x = 0;  ❌ const 拦截 */
}
```

| 优势 | 说明 |
|------|------|
| 拷贝 | 仅 **8 字节** 指针，与 struct 大小无关 |
| 修改 | `p->member` 直接写 **调用者内存** |
| `const` | 只读接口，防误改 |

### 工程实例

| 领域 | 典型类型 |
|------|----------|
| **Linux 内核** | `struct sk_buff *`、`struct inode *`、`struct file *` |
| **UEFI** | Protocol 首参 **`This`** — 协议实例 struct 指针 |
| **DPDK / HFT** | 订单、报文缓冲区 **`Order *` / `Quote *`** |

## 三、结构体指针算术（5.4 + 6.3）

`struct T *` 的 **`p++` / `p + n`** 步长 = **`sizeof(struct T)`**（**含 padding**）。

```c
struct point arr[100];
struct point *p;
#define LEN (sizeof(arr) / sizeof(arr[0]))

for (p = arr; p < arr + LEN; p++) {
    p->x += 1;
    p->y += 1;
}
```

| 逻辑 | |
|------|--|
| `p = arr` | 退化，指向首元素 |
| `p++` | 跳过 **一整块** `struct point` |
| `p->x` | 指针访问成员用 **`->`** |

> 手算偏移时必须用 **`sizeof(struct)`**，不能累加成员原始字节和（[6.1](./6.1-结构的基本知识.md) padding）。

## 四、高频考点与运行时风险

### 1. 混用 `.` / `->`

编译期即报错；review 时重点看形参是指针还是实体。

### 2. 空指针解引用

```c
struct point *p = NULL;
p->x = 10;    /* ❌ 段错误 */
```

`->` 前须保证指针 **非 NULL** 且 **指向有效对象**（[5.1](../ch05-pointers-and-arrays/5.1-指针与地址.md)）。

### 3. 野指针 / 悬垂指针

```c
struct point *bad(void)
{
    struct point local = {1, 2};
    return &local;    /* ❌ 返回后 local 销毁 */
}
/* bad()->x 乱码/崩溃 */
```

函数返回后仍用 `->` 访问已销毁栈 struct → UB。

### 4. 未初始化指针

```c
struct point *p;
p->x = 1;     /* ❌ p 垃圾值 */
```

**惯用法**：`struct point *p = NULL;` 或 `p = &obj;` 后再 `->`。

## 五、延伸：6.5 自引用结构体预告

```c
struct node {
    int val;
    struct node *next;   /* 指向同类型 struct → 链表 */
};
```

| 要点 | 说明 |
|------|------|
| `next` | 本节 `struct T *` 的直接应用 |
| `struct node a = b` | **浅拷贝** `next` 地址，不复制链表节点 |
| 遍历 | `for (n = head; n; n = n->next)` 全程 **`->`** |

详见 [6.5](./6.5-自引用结构.md)。

