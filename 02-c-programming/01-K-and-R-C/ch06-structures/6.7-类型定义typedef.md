# 6.7 类型定义 typedef

## 本节讲什么

**`typedef`** 为已有类型起**别名**，简化 `struct`、指针、函数指针等复杂声明；UEFI、Linux 内核用其统一跨平台字长命名。

## 要点

```c
typedef struct {
    int x, y;
} Point;          /* 之后可直接 Point p; */

typedef unsigned long UINT64;   /* UEFI 风格 */
typedef int (*comp_t)(const void *, const void *);
```

- **`typedef` 不创建新类型**，仅编译期别名
- 常与 **`struct`  tag** 并用：`typedef struct node { ... } Node;`
- 匿名 struct + typedef 可省略每次写 `struct`

### 工程规范

| 风格 | 例子 |
|------|------|
| UEFI | `UINTN`、`EFI_STATUS`、`EFI_HANDLE` |
| Linux 内核 | `u32`、`size_t`、`loff_t` |
| HFT | 固定宽度 `int64_t`、`uint32_t`（`<stdint.h>` 亦常用） |

### 与 `#define` 区别

- `typedef` 是类型，受类型检查
- `#define` 是文本替换，无类型安全

## 考点

- 读 typedef 复杂声明：与 5.12 右左法则结合
- **`typedef struct X X`** 自引用链表中的惯用写法

## 与后续衔接

Portable 二进制协议优先 `<stdint.h>` 定长类型；typedef 让内核/固件 API 在 32/64 位间平滑切换。
