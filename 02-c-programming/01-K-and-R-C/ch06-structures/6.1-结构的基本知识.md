# 6.1 结构的基本知识

## 本节讲什么

**`struct`** 是 C 的 **复合数据类型**，将不同基础/复合类型成员 **打包为一个整体**。数组只能存同类型元素；结构体可自由组合 `char` / `int` / `double` / 数组 / 嵌套 `struct` / 指针 —— 报文头、硬件描述符、内核对象、UEFI 协议的基础载体。理解 **内存对齐（padding）** 是本章最大难点。

**前置**：[ch05 指针与地址](../ch05-pointers-and-arrays/5.1-指针与地址.md)（`&`、`*`）、[2.4 声明](../ch02-types-operators-expressions/2.4-声明.md)。

## 一、`struct` 核心定位

| 对比 | 数组 | 结构体 |
|------|------|--------|
| 元素类型 | **同一类型** | **不同类型** 可混排 |
| 典型用途 | 批量同构数据 | 报文头、寄存器、内核对象 |
| 整体赋值 | ❌ 不支持 | ✅ 同类型可 `=` 拷贝 |

```
TCP 头、FIX 行情、ELF 头、task_struct、EFI 描述符 → 全是 struct
```

## 二、基础语法：定义、变量、成员访问

### 1. 结构体模板定义

```c
struct point {
    int x;
    int y;
};
```

- 完整类型名：**`struct point`**（C 中单独写 `point` 不行；C++ 可省略 `struct`）
- 模板定义 **不分配内存**；声明变量时才分配

### 2. 变量定义与初始化

```c
struct point p;                    /* 方式 1：先模板后变量 */
struct point p = {10, 20};         /* 按成员声明顺序 */

struct point {                     /* 方式 2：定义同时创建变量 */
    int x;
    int y;
} p1, p2;

struct point p3 = {.y = 50, .x = 30};   /* C99 指定成员初始化（推荐） */
```

| 初始化 | 特点 |
|--------|------|
| `{10, 20}` | 顺序必须与成员声明一致 |
| `{.x = 30, .y = 50}` | **顺序无关**，可读性高 |

未初始化成员（静态/全局 struct）→ 零初始化（[4.9](../ch04-functions-and-program-structure/4.9-初始化.md)）；局部 auto struct → 成员为垃圾值。

### 3. 成员访问运算符

| 运算符 | 用于 | 示例 |
|--------|------|------|
| **`.`** | **结构体实体变量** | `p.x = 10;` |
| **`->`** | **结构体指针** | `pp->x = 100;`（[6.4](./6.4-指向结构的指针.md)） |

```c
struct point p = {1, 2};
struct point *pp = &p;

p.x = 10;              /* 实体用 . */
pp->x = 100;           /* 指针用 -> */
(*pp).x = 100;         /* 等价，繁琐 */
```

### 4. 结构体整体赋值（独有特性）

同类型结构体支持 **`=` 逐成员拷贝**（数组 **不能** 整体赋值）：

```c
struct point a = {1, 2};
struct point b;
b = a;    /* ✅ x、y 完整复制 */
```

⚠️ **浅拷贝**：若成员含 **指针**，只复制 **地址**，不复制所指堆数据 —— 两结构体共享同一块内存。

## 三、嵌套结构体

成员可以是另一个完整 `struct`，多层访问连续 **`.`**：

```c
struct point {
    int x;
    int y;
};
struct rect {
    struct point pt;
    int width;
    int height;
};

struct rect r = {{10, 20}, 100, 200};
r.pt.x = 50;
```

| 工程典型 | |
|----------|--|
| UEFI | `EFI_MEMORY_DESCRIPTOR` |
| 显卡 | GOP 模式描述符 |
| 内核 | `task_struct` 多层嵌套 |

## 四、核心难点：内存对齐与 padding

### 1. 两条核心规则（System V ABI 教学模型，因平台而异）

1. **成员自身对齐**：成员起始偏移必须是 **自身类型大小** 的整数倍（或 ABI 规定值）

| 类型（x86-64 典型） | 对齐 |
|---------------------|------|
| `char` | 1 |
| `short` | 2 |
| `int` | 4 |
| `long` / `double` / 指针 | 8 |

2. **结构体整体对齐**：`sizeof(struct)` 为 **最大成员对齐粒度** 的整数倍；不足末尾 **补 padding**

### 2. 示例分步计算

```c
struct example {
    char c;    /* offset 0，占 1 字节 */
    /* padding 3 字节：offset 1、2、3 */
    int i;     /* int 需 4 对齐 → offset 4 */
};
```

| 项 | 值 |
|----|-----|
| 成员原始和 | 1 + 4 = **5** |
| 最大对齐粒度 | **4** |
| 总 `sizeof` | 向上取 4 倍数 → **8** |

### 3. 工程风险

网络协议头（TCP/IP、FIX）、硬件寄存器映射：编译器 **默认对齐** 插入 padding，若按「成员紧挨」心算偏移 → **字段错位、静默 bug**。

| 方案 | 说明 |
|------|------|
| `#pragma pack(n)` | 压缩对齐（跨编译器谨慎） |
| C11 `_Alignas` | 手动指定对齐 |
| 协议定义 | 用 `offsetof` 对照规范，勿猜 |

### 4. `offsetof` 工具

```c
#include <stddef.h>

size_t off = offsetof(struct example, i);   /* 结果 4 */
```

手写布局、内核/驱动面试必备；与 [5.4](./5.4-地址算术运算.md) 基址+偏移一脉相承。

## 五、高频考点汇总

1. 手写每个成员 **offset**、**padding**、总 **`sizeof`**
2. **`.`** 用于变量，**`->`** 用于指针，禁止混用
3. 结构体可整体赋值；数组不可
4. 协议/寄存器开发：**对齐 padding 无编译报错，仅运行数据异常**

## 六、底层工程场景

| 场景 | 对齐要求 |
|------|----------|
| **网络报文** | TCP/UDP/FIX/SBE 固定偏移 |
| **文件格式** | ELF、PE、BMP 头 — 常 `#pragma pack(1)` |
| **硬件驱动** | MMIO 寄存器映射，偏移必须匹配手册 |
| **UEFI** | Protocol / Memory 描述符嵌套 struct |
| **内核** | `struct stat`、`task_struct`、设备集合 |

## 背诵要点（精简版）

```
struct 打包不同类型；完整类型名 struct point
. 实体  -> 指针；同类型 struct 可 = 赋值（浅拷贝）
嵌套：r.pt.x；C99 {.x=, .y=}
对齐：成员偏移按类型对齐；总大小按最大粒度补齐
offsetof 查偏移；协议/寄存器忌默认 padding 瞎猜
```
