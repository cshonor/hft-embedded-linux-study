# 6.5 自引用结构

## 本节讲什么

结构体内部包含 **指向自身类型的指针**（不能嵌套同名 struct **实体**），用于 **单向/双向链表**、树等动态结构。是 [6.4](./6.4-指向结构的指针.md) `->` 与 [5.6](./5.6-指针数组以及指向指针的指针.md) `struct node **` 的综合实战；[6.6](./6.6-表查找.md) 符号表、[ch08](../ch08-unix-system-interface/) `malloc` 均建立在此之上。

**前置**：[6.4](./6.4-指向结构的指针.md)、[5.2](../ch05-pointers-and-arrays/5.2-指针与函数参数.md)（二级指针改 `head`）、[6.2](./6.2-结构与函数.md)（浅拷贝）。

## 一、为什么只能存指针，不能存实体

### 合法：自引用指针（链表标准）

```c
struct node {
    int data;
    struct node *next;
};
```

### 非法：嵌套自身实体

```c
struct node {
    int data;
    struct node next;   /* ❌ 编译报错 */
};
```

#### 底层原因

| 写法 | 编译器视角 |
|------|-----------|
| **`struct node *next`** | 指针 **固定 8 字节**（64 位）；此时 `struct node` 为 **不完整类型** 即可声明指针 |
| **`struct node next`** | 内含 **完整自身** → 尺寸无限递归 → **无法 `sizeof`** → 报错 |

```
指针：固定宽 → 不完整类型够用
实体：要算整 struct 大小 → 嵌套自身 → 无穷大
```

## 二、单向链表完整逻辑

### 1. 空链表

```c
struct node *head = NULL;   /* NULL = 无节点 */
```

### 2. 新建节点（堆分配，ch08 详述）

```c
#include <stdlib.h>

struct node *new_node(int val)
{
    struct node *p = malloc(sizeof *p);
    if (p == NULL)
        return NULL;
    p->data = val;
    p->next = NULL;
    return p;
}
```

### 3. 头插法（需 `struct node **` 改 head）

```c
void push(struct node **head, int val)
{
    struct node *n = new_node(val);
    if (n == NULL)
        return;
    n->next = *head;
    *head = n;
}
```

| 为何 `**head` | [5.2](../ch05-pointers-and-arrays/5.2-指针与函数参数.md)：一级指针形参改不了外部 `head` 变量本身 |

### 4. 遍历（迭代，工程标准）

```c
void traverse(struct node *head)
{
    for (struct node *cur = head; cur != NULL; cur = cur->next)
        printf("%d ", cur->data);
}
```

### 5. 释放整条链表

```c
void free_list(struct node *head)
{
    struct node *cur, *tmp;
    for (cur = head; cur != NULL; cur = cur->next) {
        tmp = cur;
        free(tmp);           /* 先保存 next 已在 cur->next，或经典双指针写法 */
    }
}
```

经典写法（防 use-after-free）：

```c
void free_list(struct node *head)
{
    while (head) {
        struct node *tmp = head;
        head = head->next;
        free(tmp);
    }
}
```

### 6. 尾插法（自测参考）

```c
void push_tail(struct node **head, int val)
{
    struct node *n = new_node(val);
    if (n == NULL)
        return;
    if (*head == NULL) {
        *head = n;
        return;
    }
    struct node *cur;
    for (cur = *head; cur->next != NULL; cur = cur->next)
        ;
    cur->next = n;
}
```

## 三、双向链表

```c
struct node {
    int data;
    struct node *prev;
    struct node *next;
};
```

| 优势 | 说明 |
|------|------|
| 双向遍历 | `prev` / `next` |
| O(1) 删当前节点 | 已知节点指针即可 unlink |
| 内核 | Linux **`list_head`** 侵入式双向链表（进阶） |

## 四、关键考点

### 1. 结构体赋值 = 浅拷贝

```c
struct node a = {10, NULL};
struct node b = a;
```

| 项 | 行为 |
|----|------|
| `b.data` | 独立副本，改 `b.data` 不影响 `a.data` |
| `b.next` | 与 `a.next` **同一地址** |
| 风险 | `free` 一处节点，另一处 `next` 变 **野指针**；双重 `free` |

链表节点 **不要** 盲目 `struct node b = a`；应复制 **数据** 或 **新建 malloc 节点**。

### 2. 内存泄漏 / 双重释放

| 错误 | 后果 |
|------|------|
| 只 `free(head)` 不遍历 | 后续节点 **泄漏** |
| 同一指针 `free` 两次 | 堆破坏、崩溃 |
| 释放前未保存 `next` | 后半链 **丢失** |

### 3. 递归 vs 迭代

```c
void print_recur(struct node *p)
{
    if (!p) return;
    printf("%d ", p->data);
    print_recur(p->next);
}
```

| | 递归 | 迭代 |
|---|------|------|
| 深度 | O(链长) | O(1) 栈 |
| 内核/HFT | 栈 ~8KB，长链 **溢出** | **首选** |

见 [4.10](../ch04-functions-and-program-structure/4.10-递归.md)。

## 背诵要点（精简版）

```
只能 struct node *next，不能 struct node next（无穷大小）
head=NULL 空链；push 用 **head 改头指针
遍历 while(cur) cur=cur->next；释放先存 next 再 free
浅拷贝 = 复制指针地址；double-free/野指针
长链用迭代，别递归（栈溢出）
双向 prev+next；内核 list_head
```
