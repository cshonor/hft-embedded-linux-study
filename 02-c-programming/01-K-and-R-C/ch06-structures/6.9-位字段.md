# 6.9 位字段

## 本节讲什么

**位域（bit-fields）**：在结构体中为成员指定**占用的 bit 数**，紧凑打包标志位；常见于**硬件控制/状态寄存器**本地建模。

## 要点

```c
struct flags {
    unsigned int is_valid : 1;
    unsigned int type     : 3;
    unsigned int priority : 4;
    /* 可能仍有 padding 到 int 边界 */
};
```

- 语法：`类型 成员名 : 位数;`
- 类型多为 **`unsigned int`** / **`int`**（实现相关）
- **布局由编译器决定**：位序、是否跨字、signed 位域算术 —— **跨平台不兼容**
- **`&位域成员` 非法**（无独立地址，不能取址）

## 适用场景

| 适合 | 不适合 |
|------|--------|
| 本地 MCU 寄存器模型 | 网络协议 on-wire 格式 |
| 驱动私有状态压缩 | 跨编译器二进制 interchange |
| 无人机电机/中断掩码 | 需严格 ABI 的 API |

## 与手动掩码对比

```c
#define CTRL_ENABLE  (1u << 0)
reg |= CTRL_ENABLE;

/* vs 位域 */
ctrl.enable = 1;
```

- 可移植协议：`uint32_t reg;` + 宏 **`(reg >> n) & mask`**
- 单平台固件：位域可读性更好

## 考点

- 位域 vs 位运算：面试常问优缺点
- 大小与对齐不可手算，需看编译器输出或 `sizeof`

## 与后续衔接

Linux `struct tcp_header` 等用位域但网络栈仍大量显式移位；读 ARM CMSIS、STM32 HAL 寄存器定义到位域与 union 组合。
