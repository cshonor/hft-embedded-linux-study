# 6.3 结构数组

## 本节讲什么

**结构体数组** = 数组，每个元素是 **完整的 `struct` 对象**。批量存储同类复合数据：关键字表、设备列表、内存映射描述符、传感器采样、行情环形缓冲。内存 **整块连续**，元素步长 = **`sizeof(struct)`**（含 padding）。

**前置**：[6.1](./6.1-结构的基本知识.md)、[5.3](./5.3-指针与数组.md)（退化）、[5.4](./5.4-地址算术运算.md)（`p++` 步长）、[5.8](./5.8-指针数组的初始化.md)（`ARRAY_LEN` 宏）。

## 一、核心定义与访问

```c
struct key {
    char *word;
    int count;
} keytab[100];

keytab[i].word = "auto";
keytab[i].count++;
```

| 层级 | 含义 |
|------|------|
| `keytab[i]` | 第 `i` 个完整 **`struct key`** 实体 |
| `.word` / `.count` | 对实体用 **`.`** 访问成员 |

**固定语法**：`数组下标.成员名`（下标得 **实体**，不是指针）。

```
内存：| struct key | struct key | … |  连续，步长 sizeof(struct key)
```

## 二、结构体数组初始化

### 1. 显式逐个初始化

```c
struct point { int x; int y; };

struct point pts[] = {
    {0, 0},
    {1, 2},
    {5, 8}
};   /* 长度由 {} 内元素个数自动推导 */
```

### 2. C99 指定成员初始化（推荐）

```c
struct point pts[3] = {
    {.y = 10, .x = 1},
    {.x = 20, .y = 30}
};   /* 第 3 个元素未写 → 全成员置 0 */
```

### 3. 静态零初始化

```c
static struct key keytab[64];   /* 全局/静态：全成员清零 [4.9] */
```

## 三、数组名退化 & 指针遍历

结构体数组名在表达式中 **退化为指向首元素的指针**：

```c
struct key keytab[100];
struct key *p = keytab;    /* 等价 &keytab[0] */
```

**指针遍历模板**（无下标，底层高频）：

```c
#define TAB_CNT (sizeof(keytab) / sizeof(keytab[0]))

struct key *p;
for (p = keytab; p < keytab + TAB_CNT; p++)
    p->count++;              /* 指针用 -> */
```

| 要点 | 说明 |
|------|------|
| `p++` | 跳过 **一整个 `struct key`**（含 padding） |
| `.` vs `->` | 下标 `keytab[i].count`；指针 `p->count` |
| 合法性 | `p` 与 `keytab + TAB_CNT` 同数组内比较（[5.4](./5.4-地址算术运算.md)） |

## 四、多层组合场景

### 1. 结构体内含数组

```c
struct Sample {
    float imu[3];
    uint64_t ts;
};
struct Sample buf[256];

buf[i].imu[0] = 0.01f;    /* 结构体数组 + 成员数组 */
```

### 2. 作函数参数（退化）

```c
void count_keys(struct key tbl[], int len);
void count_keys(struct key *tbl, int len);   /* 完全等价 */
```

| 特性 | 说明 |
|------|------|
| 实参 | 传 **首元素地址**，零拷贝 |
| `sizeof(tbl)` 在函数内 | **指针大小**（8），不是数组长度 |
| 必须另传 | **`len`** 或调用方 `ARRAY_LEN` |

### 3. 嵌套结构体数组（UEFI 典型）

```c
struct memdesc {
    uint64_t phys_start;
    uint64_t num_pages;
    uint32_t type;
};
struct memdesc map[128];

map[2].phys_start = 0x100000;
```

## 五、`sizeof` 计算规则（考点）

```c
struct key keytab[100];

sizeof(keytab)         /* 100 × sizeof(struct key) */
sizeof(keytab[0])      /* 单个 struct（含 padding） */
sizeof(keytab[0].word) /* sizeof(char*)，通常 8 */
```

```c
#define ARRAY_LEN(arr) (sizeof(arr) / sizeof((arr)[0]))
```

| 位置 | `sizeof(keytab)` |
|------|------------------|
| 定义处（数组类型） | `100 × sizeof(struct key)` |
| 函数形参 `struct key tbl[]` | **8**（退化为指针） |

`#pragma pack` 改变 `sizeof(struct key)` → 改变 **`p++` 步长** 与数组总大小。

## 六、底层工程场景

| 场景 | 例子 |
|------|------|
| **UEFI** | `GetMemoryMap` → `struct memdesc[]` 描述符数组 |
| **HFT** | `Quote quotes[N]` 环形缓冲，连续内存、cache 友好 |
| **飞控** | `Sample buf[FS]` IMU 采样历史 |
| **Linux 内核** | PCI 设备表、中断向量表（概念上 struct 数组） |

## 七、高频 Bug 与踩坑

| 坑 | 后果 |
|----|------|
| 混淆 `.` / `->` | `keytab[i]` 用 `.`；`p` 用 `->` |
| 函数内 `sizeof(tbl)` 当长度 | 得到 8 |
| 手算步长忽略 padding | 遍历错位 |
| `word` 指向字面量后修改 | 段错误（[5.5](./5.5-字符指针与函数.md)） |
| 超大局部 `struct buf[N]` | 栈溢出 → `static` 或 `malloc` |

```c
/* ❌ 栈上 1MB */
struct tick ticks[10000];

/* ✅ 静态区或堆 */
static struct tick ring[N];
struct tick *heap = malloc(n * sizeof *heap);
```

## 背诵要点（精简版）

```
struct arr[N]：连续 N 个完整 struct，步长 sizeof(struct)
访问：arr[i].member；退化 arr → struct*
遍历：for(p=arr; p<arr+LEN; p++) p->member
初始化 {} 列表；C99 {.x=}
sizeof(arr)=N*sizeof(struct)；形参内 sizeof=8
HFT 环形缓冲：连续 struct 数组 cache 友好
```
