# 6.2 结构与函数

## 本节讲什么

结构体作为 **函数参数** 与 **返回值**：C 允许 **按值传递/返回** 整个结构体，但大对象拷贝昂贵。内核、HFT、UEFI 普遍传 **`const struct T *`**，避免整块复制。衔接 [5.2 指针传参](../ch05-pointers-and-arrays/5.2-指针与函数参数.md)、[6.1](./6.1-结构的基本知识.md)。

**前置**：[6.1](./6.1-结构的基本知识.md)（`.` 访问、浅拷贝）、[5.2](../ch05-pointers-and-arrays/5.2-指针与函数参数.md)。

## 一、两种传参模式核心区别

C 支持 **结构体值传递** 与 **结构体指针传递** —— 底层行为、性能、修改能力完全不同。底层高性能开发（内核 / HFT）**统一优先指针传参**。

### 1. 按值传递（拷贝整个对象）

```c
struct point { int x; int y; };

struct point makepoint(int x, int y)
{
    struct point p;
    p.x = x;
    p.y = y;
    return p;          /* 值返回：拷贝一份回调用方 */
}

void printpoint(struct point p)   /* 入参：栈上完整副本 */
{
    printf("%d,%d", p.x, p.y);
    p.x = 999;         /* 只改副本，外部不变 */
}
```

| 特性 | 说明 |
|------|------|
| 调用 | 栈上 **完整复制** 所有成员 |
| 返回 | `return p` 同样 **拷贝** 一份（编译器可能寄存器优化小 struct） |
| 修改 | 函数内改成员 **不影响** 外部原对象 |
| 性能 | 随 `sizeof(struct)` **线性增长** 拷贝带宽、栈压力 |
| 适用 | **极小** struct（2～4 标量：`point`、坐标、二元组） |

### 2. 指针传递（工程标准）

```c
void printpoint(const struct point *p)
{
    printf("%d,%d", p->x, p->y);
    /* p->x = 10;  ❌ const 拦截 */
}

void setpoint(struct point *p, int x, int y)
{
    p->x = x;          /* ✅ 改调用者对象 */
    p->y = y;
}
```

| 特性 | 说明 |
|------|------|
| 拷贝 | 仅 **一个指针**（64 位 8 字节），与 struct 大小无关 |
| 修改 | 去掉 `const` 后 `p->member` 直接写 **外部内存** |
| `const` | 只读接口，编译期防误写 |
| 适用 | **内核、HFT、驱动、UEFI**、大型 struct |

```
C 仍是值传递：传的是指针值的副本，但通过 -> 改的是外部 struct（5.2 一级改所指）
```

## 二、完整对比表

| 传参形式 | 拷贝内容 | 内存开销 | 改外部原 struct | 适用 |
|----------|----------|----------|-----------------|------|
| `f(struct s val)` | 全部成员 | ∝ `sizeof(s)` | ❌ 仅副本 | 微型 struct |
| `f(const struct s *p)` | 指针 8B | 恒定 | `const` 不可改；非 const 可改 | **工程默认** |
| `f(struct s *p)` | 指针 8B | 恒定 | ✅ | 输出参数、原地修改 |

## 三、返回 struct 两种方案

### 1. 值返回 `struct s func()`

```c
struct point makepoint(int x, int y) { ... return p; }
```

| | |
|---|---|
| 适合 | 微型 struct（点、坐标） |
| 优化 | 编译器可能 **寄存器返回**（ABI 相关） |
| 风险 | 大型 struct 高频返回 → 栈拷贝、延迟 |

### 2. 输出参数 `void func(struct s *out)`（高性能首选）

```c
void init_point(struct point *out, int x, int y)
{
    out->x = x;
    out->y = y;
}

struct point p;
init_point(&p, 10, 20);   /* 零拷贝写入，外部已分配 */
```

| | |
|---|---|
| 模式 | 调用方 **先分配**，传 `&p` 原地写 |
| 开销 | **无** struct 体拷贝 |
| HFT | 订单簿、行情块预分配池 + 指针传递 |

## 四、工程示例：`struct tick`（1024 字节）

```c
struct tick { char buf[1024]; /* 行情字段… */ };

/* ❌ 值传递：每次调用拷贝 1024+ 字节 */
void print_tick_bad(struct tick t)
{
    printf("%.8s\n", t.buf);
}

/* ✅ const 指针：固定 8 字节传参 */
void print_tick_ok(const struct tick *t)
{
    printf("%.8s\n", t->buf);
}
```

| 版本 | 单次传参拷贝量 |
|------|----------------|
| `print_tick_bad` | **~1024 B**（+ padding） |
| `print_tick_ok` | **8 B** |

HFT 热路径禁止前者。

## 五、底层工程场景

| 场景 | 写法 |
|------|------|
| **HFT** | `Order *`、`Quote *`；禁止大 struct 值传递 |
| **UEFI** | Protocol 首参 `This` 实例指针，全程无 struct 拷贝 |
| **Linux 驱动** | `struct file *`、`struct device *` 进 read/write/ioctl |
| **系统调用** | `stat`、`sockaddr` — 用户态传 **缓冲区指针**，内核 `copy_from_user` |

## 六、高频易错坑

| 坑 | 后果 |
|----|------|
| 大 struct **循环值传递** | 栈压力、CPU 带宽、延迟暴涨 |
| 只读接口忘 **`const`** | 函数内误改外部数据 |
| 值传递以为改副本能改外面 | 逻辑错误 |
| 含 **堆指针** 成员的值传递 | **浅拷贝** 两份 struct 共享堆块 → 双重 `free`（[6.1](./6.1-结构的基本知识.md)） |

```c
struct node { int *data; };
struct node a = { malloc(...) };
struct node b = a;   /* 值传递/赋值：data 地址相同 */
free(a.data);
free(b.data);        /* ❌ 双重释放 */
```

## 背诵要点（精简版）

```
小 struct 可值传/值返；大 struct 一律 const T* 或 T*
值传 = 整块拷贝，改副本不影响外面
指针传 = 8 字节，-> 改原对象；const 只读
输出参数 init(&p) 零拷贝；HFT/内核禁止大块值传
含指针成员：值传/= 是浅拷贝，慎 double free
```
