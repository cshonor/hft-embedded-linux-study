# 6.8 联合

## 本节讲什么

**`union`**：多个成员**共享同一块内存**，同一时刻只有**一个成员**的值有效；大小至少为最大成员的大小。

## 要点

```c
union u_tag {
    int i;
    float f;
    char str[4];
};

union u_tag u;
u.i = 123;      /* 写入 i 的布局 */
/* 此时读 u.f 依赖实现，类型 punning —— 需谨慎 */
```

- 所有成员**首地址相同**
- **`sizeof(union)`** = 最大成员大小（含对齐到 union 对齐要求）
- 写入一成员后再读另一成员：C99+ 有 **common initial sequence** 等规则；否则可能未定义或实现定义

## 用途

| 场景 | 用法 |
|------|------|
| 二进制转换 | `u32` 与 4×`u8` 同址查看 |
| 可变报文 | 同一缓冲按类型解析不同字段 |
| 寄存器多视图 | 整字写入、位域/字节读出 |
| 省内存 | 互斥状态机若干字段共用存储 |

## 与 struct 对比（考点）

| | struct | union |
|---|--------|--------|
| 内存 | 各成员独立，可能 padding | 成员重叠 |
| 同时有效 | 全部成员各有存储 | 同一时刻通常只认一个 |
| 典型用途 | 记录聚合 | 类型双关、省空间 |

## 注意

- **无内置同步**；多线程同时写不同成员 → 数据竞争
- 网络协议公开格式优先 **显式偏移 + struct**，少用 union 避免可移植性问题
