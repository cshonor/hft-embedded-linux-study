# 6.6 表查找（符号表哈希链表）

## 本节讲什么

K&R **符号表（keyword / identifier lookup）** 综合实战：融合 **结构体、自引用链表、字符指针、堆分配、哈希桶、拉链法查找**。对应 C 预处理器 **`#define` 宏符号表**，也是内核/链接器哈希表的 **最简原型**。ch06 工程化落地样板。

**前置**：[5.5](../ch05-pointers-and-arrays/5.5-字符指针与函数.md)、[5.8](../ch05-pointers-and-arrays/5.8-指针数组的初始化.md)（指针数组）、[6.4](./6.4-指向结构的指针.md)、[6.5](./6.5-自引用结构.md)、[ch08](../ch08-unix-system-interface/) `malloc`（详述）。

## 一、本节定位

```
第 6 章知识点汇总 → 符号表哈希链表
struct + next链 + char* + hashtab[] + hash/lookup/install
```

| 对应 | 场景 |
|------|------|
| **C 预处理** | `#define` 宏名 → 替换文本 |
| **链接器** | ELF 符号表 |
| **内核** | `ksymtab`、设备哈希、`hlist` |

## 二、核心结构：`struct nlist`

```c
struct nlist {
    struct nlist *next;   /* 哈希冲突链（拉链法） */
    char *name;           /* 符号名（标识符/宏名） */
    char *defn;           /* 宏替换文本 */
    int type;             /* 关键字/用户宏/变量等标记 */
};
```

### 成员拆解

| 成员 | 作用 |
|------|------|
| **`next`** | [6.5](./6.5-自引用结构.md) 自引用；同哈希桶冲突节点串成单向链 |
| **`name` / `defn`** | **`char *`** 指向 **堆拷贝** 字符串；长度不一，指针省空间 |
| **`type`** | 区分关键字、用户宏、变量、函数等，查找后分支处理 |

**为何不用 `char name[]` 嵌在 struct 里？** 符号名长短不一；指针 + 堆串更灵活（对比 [5.8](./5.8-指针数组的初始化.md) 指针数组 vs 定宽二维数组）。

**禁止**：`p->name = 外部临时串` 直接挂指针 —— 外部释放后 **野指针**；只读字面量修改 **段错误**。必须 **`malloc`/`strdup` 独立拷贝**。

### 哈希桶数组（表本体）

```c
#define HASHSIZE 101

static struct nlist *hashtab[HASHSIZE];   /* 全局静态，初值全 NULL */
```

| 项 | 说明 |
|----|------|
| `hashtab` | **指针数组**（[5.6](./5.6-指针数组以及指向指针的指针.md)）；每元素是一条冲突链 **头指针** |
| 存储区 | **静态区**，程序全程存在，不反复分配 |
| `HASHSIZE` | 常取 **质数**，减碰撞（教学惯例） |

**为何是指针数组而非 `struct nlist hashtab[]`？** 桶内节点 **个数不定**、**堆上动态增长**；桶数组只存 **链头指针**，节点按需 `malloc`。

## 三、三大核心操作

### 1. 哈希函数 `hash`

把 `name` 映射到 `0 … HASHSIZE-1`：

```c
unsigned hash(char *s)
{
    unsigned val;
    for (val = 0; *s != '\0'; s++)
        val = *s + 31 * val;
    return val % HASHSIZE;
}
```

| 注意 | |
|------|--|
| 必须 **`% HASHSIZE`** | 否则桶下标越界 |
| 冲突 | 不同串可能同桶 → 靠链表 + `strcmp` 区分 |

### 2. 查表 `lookup`

```c
struct nlist *lookup(char *name)
{
    struct nlist *p;
    unsigned h = hash(name);

    for (p = hashtab[h]; p != NULL; p = p->next)
        if (strcmp(name, p->name) == 0)
            return p;
    return NULL;
}
```

| 步骤 | |
|------|--|
| 1 | `hash(name)` 定位桶 |
| 2 | 沿 `next` 扫冲突链 |
| 3 | `strcmp` 比全名 |
| 4 | 找到返节点，否则 `NULL` |

### 3. 插入/更新 `install`

```c
#include <string.h>
#include <stdlib.h>

struct nlist *install(char *name, char *defn, int type)
{
    struct nlist *p;
    unsigned h = hash(name);

    if ((p = lookup(name)) != NULL) {      /* 已存在：更新 */
        free(p->defn);
        p->defn = strdup(defn);
        p->type = type;
        return p;
    }

    p = malloc(sizeof *p);
    if (p == NULL)
        return NULL;
    p->name = strdup(name);
    p->defn = strdup(defn);
    p->type = type;
    p->next = hashtab[h];      /* 头插 */
    hashtab[h] = p;
    return p;
}
```

| 逻辑 | |
|------|--|
| 先 `lookup` | 支持 **覆写**（类比 `#define` 重定义） |
| 新建 | `malloc` 节点 + **`strdup` 串** |
| 挂载 | **头插** `hashtab[h]`（[6.5](./6.5-自引用结构.md) `push` 同构） |

> **`strdup`** = `malloc` + `strcpy`；POSIX 常用。无则手写分配拷贝。

## 四、关键设计要点

| 要点 | 说明 |
|------|------|
| **内存分层** | `hashtab[]` 静态；节点与串在 **堆** |
| **冲突解决** | **拉链法**：同桶 `next` 链 + `strcmp` |
| **字符串独立拷贝** | 不挂外部栈串/字面量 |
| **查找优先插入** | 先查再决定更新或新建 |

## 五、底层系统映射

| 本节概念 | 工程对应 |
|----------|----------|
| 哈希桶 + 冲突链 | 内核 `hlist`、PLT、设备哈希表 |
| `name` / `defn` 键值 | ELF 符号、`ksymtab` |
| `lookup` / `install` | 设备注册、配置 KV、预处理器宏表 |
| 堆动态节点 | HFT 订单簿哈希、策略参数表 |

### HFT 低延迟补充

高频路径 **禁用频繁 `malloc/free`** → **预分配节点池 + 空闲链表** 复用 `struct nlist`，消除分配抖动与碎片（对比 [6.3](./6.3-结构数组.md) 固定数组 ring）。

## 六、销毁整张表 `free_table`（自测参考）

```c
void free_table(void)
{
    int i;
    for (i = 0; i < HASHSIZE; i++) {
        struct nlist *p = hashtab[i];
        while (p) {
            struct nlist *next = p->next;
            free(p->name);
            free(p->defn);
            free(p);
            p = next;
        }
        hashtab[i] = NULL;
    }
}
```

| 顺序 | 先 `free` 串，再节点；遍历每条桶链；最后桶头置 `NULL` |

## 七、高频 Bug 与考点

| Bug | 后果 |
|-----|------|
| `p->name = name` 浅挂外部串 | 野指针 |
| 销毁只 `free(p)` 忘 `name`/`defn` | 泄漏 |
| `hash` 忘 `% HASHSIZE` | 越界崩溃 |
| 插入未更新 `hashtab[h]` | 新节点丢失 |
| 更新 `defn` 前未 `free` 旧串 | double-free |

## 背诵要点（精简版）

```
nlist: next链 + char* name/defn + type
hashtab[HASHSIZE] 静态指针数组，每桶一条链
hash % HASHSIZE → lookup 扫链 strcmp
install: 有则更新 defn；无则 malloc+strdup 头插
串必须堆拷贝；销毁 free 串+节点
HFT: 节点池复用代替 malloc
```
