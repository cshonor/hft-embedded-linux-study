# 4.9 初始化

## 本节讲什么

外部变量、静态变量、局部自动变量的 **初值规则**：谁默认零初始化、谁是未定义行为（UB）、数组与字符串如何初始化；以及 **`.data` / `.bss` / 栈** 在启动阶段的底层分工。直接影响全局配置表、静态缓冲区、内核/裸机启动代码。

**前置**：[4.3 外部变量](./4.3-外部变量.md)、[4.3.1 程序内存布局](./4.3.1-程序内存布局.md)、[4.6 静态变量](./4.6-静态变量.md)、[4.5 头文件](./4.5-头文件.md)。

## 一、三类变量初始化核心规则对比

| 变量类型 | 存储分区 | 未手动初始化默认值 | 初始化时机 |
|--------|--------|------------------|----------|
| **外部全局变量** / **static 静态全局** | `.data` / `.bss`（[4.3.1](./4.3.1-程序内存布局.md) 全局/静态区） | **强制全零**（`int=0`、浮点 `0.0`、`char='\0'`、指针 `NULL`） | `main()` 执行前，CRT 启动代码完成清零/拷贝 |
| **static 局部变量**（函数内 `static`） | `.data` / `.bss` | **强制全零** | 第一次调用该函数前完成；生命周期与程序一致（[4.6 §1](./4.6-静态变量.md)） |
| **普通局部自动变量**（无 `static`） | **栈** stack | **垃圾随机值（UB）** | 进入栈帧时分配，**无**自动清零 |

```
有初值的全局/static  →  .data 段（加载时拷贝预设值）
无初值的全局/static  →  .bss  段（加载器整块清零）
普通局部 auto        →  栈帧（退出后空间可被复用，不保证清零）
```

## 二、分模块详细解读

### 1. 外部 / 静态变量（全局、static 局部）

#### 规则

1. 只要带 **`static`** 或定义在 **函数外**，无显式赋值时，编译器保证 **自动置零**；
2. 所有类型统一 **全比特置 0**，对应各自类型的零值：
   - 整型：`int a;` → `a = 0`
   - 浮点：`static double x;` → `x = 0.0`
   - 字符：`static char c;` → `c = '\0'`
   - 指针：`char *p;` → `p = NULL`
3. 显式初始化示例：

```c
int global_num = 42;              /* 外部全局，有初值 → .data */
static int file_only_buf[1024];   /* 文件级 static，无初值 → .bss，全元素 0 */

void func(void)
{
    static int cnt = 0;           /* 函数内 static：仅首次执行到声明时赋 0，之后保留 */
    cnt++;
}
```

#### 底层内存分段

| 写法 | 段 | 加载时行为 |
|------|-----|-----------|
| `int g = 42;` | **`.data`** | 直接把 `42` 拷进内存 |
| `static int buf[1024];` | **`.bss`** | 加载器 **一次性整块清零**，可执行文件里不必存 1024 个 `0` |

> `.bss` 清零节省镜像体积，是链接器/加载器的工程优化；语义上等价于「未写初值的全局/static 全是 0」。

### 2. 局部自动变量（普通栈变量）

#### 核心风险点

栈内存 **复用**：函数退出后，这块栈空间会被其他调用覆盖，**不会自动清零**。不赋值直接读取属于 **未定义行为（UB）**。

```c
void test(void)
{
    int a;
    printf("%d", a);   /* ❌ 垃圾随机值，UB */
    a = 10;            /* ✅ 使用前手动赋值 */
    int b = 20;        /* ✅ 声明时初始化（推荐） */
}
```

| | **普通局部 `int x;`** | **函数内 `static int x;`** |
|---|----------------------|---------------------------|
| 存储 | 栈 | 全局/静态区 |
| 未初始化 | **垃圾值，UB** | **保证 0** |
| 函数返回后 | 栈帧销毁 | 值保留 |

### 3. 数组初始化语法规则

**1. 初始化列表少于数组长度 → 剩余元素自动置 0**（与静态变量规则一致）

```c
int arr[5] = {1, 2};
/* 等价于 {1, 2, 0, 0, 0} */
```

**2. 省略数组长度 → 编译器按列表元素个数推算长度**

```c
int arr[] = {1, 2, 3};   /* 长度 = 3 */
```

**3. 字符数组 + 字符串字面量 → 自动追加末尾 `\0`**

```c
char str[] = "hello";
/* 实际：h e l l o \0，长度 6 */

char buf[10] = "hi";
/* { 'h','i','\0', 0,0,0,0,0,0,0 } */
```

> 与 [1.9 字符数组](../ch01-introduction/1.9-字符数组.md) 的 `char s[] = "..."` 写法一脉相承；第 5 章会扩展到指针数组、多维数组、结构体数组。

### 4. 初始化时序 & 特殊环境（内核 / 裸机）

**标准操作系统（Linux / Windows）**

```
OS 加载程序 → CRT 启动代码 → 清零 .bss、拷贝 .data → 调用 main()
```

全局 / 静态变量在 **`main` 执行前** 就已经完成初始化。

**裸机、操作系统内核（无 CRT 运行时库）**

- 没有标准 C 运行库的自动清零逻辑；
- 必须 **手动写启动/加载代码** 清空 **`.bss` 段**，否则全局/static 变量会是内存里的 **垃圾值**；
- 这是底层开发、HFT 固件、自制 OS 的 **关键细节** —— 不能想当然依赖「C 保证全零」。

```c
/* 内核启动伪代码示意 */
extern char __bss_start[], __bss_end[];
for (char *p = __bss_start; p < __bss_end; p++)
    *p = 0;
/* 再拷贝 .data，最后才跳 main 或内核入口 */
```

## 三、高频易混考点

| 对比 | 结论 |
|------|------|
| **`static int arr[10];`** vs **`int arr[10];`**（函数内） | `static` 数组在 **`.bss`**，元素默认 **全 0**，生命周期 = 整个程序；普通局部数组在 **栈** 上，元素全是 **垃圾值** |
| 头文件 **`extern int g;`** | `extern` 只是 **声明**，不分配内存、不执行初始化；**定义与初值** 必须放在 **某一个 `.c`**（[4.3](./4.3-外部变量.md)、[4.5](./4.5-头文件.md)） |
| **`const static int x;`** | 静态只读常量，未初始化依然默认 **0**；只读数据常进 **常量/只读段**（[2.3 const](../ch02-types-operators-expressions/2.3-常量.md)） |

## 四、工程实践（量化 / 内核）

1. **全局缓冲区、统计计数** 优先用 **`static`**，靠自动零初始化省去手写 `memset`；
2. 函数内 **大数组** 避免栈分配（栈溢出风险），可改用 **`static` 缓冲区** 或堆（[7.8.5](../ch07-input-and-output/7.8-other-functions/7.8.5-存储管理函数.md)）；
3. **裸机 / 内核** 必须自己实现 **`.bss` 清零**，不能依赖 CRT；
4. **所有局部变量** 声明时同步初始化，杜绝读垃圾值引发的隐蔽逻辑 bug。

| 场景 | 建议 |
|------|------|
| **HFT 策略状态表** | 文件级 `static` 数组/计数器，依赖零初始化 |
| **内核驱动全局缓冲** | 启动代码清 `.bss` + 显式初值放 `.data` |
| **热路径局部标量** | `int n = 0;` 声明即初始化 |

## 背诵要点（精简版）

```
全局 / static（含函数内 static）  →  未写初值 = 0，main 前就绪
普通局部 auto                     →  未写初值 = 垃圾，UB
有初值全局                        →  .data；无初值全局/static → .bss 清零
数组 {1,2} 补零；char[] = "hi" 自动 \0
extern 只声明不初始化；定义放某一个 .c
裸机/内核无 CRT → 手写清 .bss
```
