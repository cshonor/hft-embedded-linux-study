# 4.3.1 程序内存布局（五区总览）

## 本节讲什么

C 程序运行时，进程的虚拟地址空间通常可粗分为 **栈区、堆区、全局/静态存储区、常量存储区、代码区** 五块。这是理解 **局部变量 vs 全局变量**、**`malloc` vs 栈**、**字符串常量放哪** 的总地图；为 [5.1 指针](../ch05-pointers-and-arrays/5.1-指针与地址.md)、[7.8.5 存储管理](../ch07-input-and-output/7.8-other-functions/7.8.5-存储管理函数.md)、[8.7 存储分配](../ch08-unix-system-interface/8.7-实例-存储分配程序.md) 铺路。

**前置**：[4.3 外部变量](./4.3-外部变量.md)、[4.6 静态变量](./4.6-静态变量.md)、[2.4 声明](../ch02-types-operators-expressions/2.4-声明.md)。

## 五区一览

```
高地址
┌─────────────────┐
│      栈区       │  ← 局部变量、函数参数；自动分配/释放
├─────────────────┤
│        ↓        │  （栈向低地址生长）
│        ↑        │  （堆向高地址生长）
├─────────────────┤
│      堆区       │  ← malloc/calloc/realloc 申请；free 释放
├─────────────────┤
│ 全局/静态存储区  │  ← 全局变量、static 变量（局部/文件级）
├─────────────────┤
│   常量存储区    │  ← 字符串字面量、const 全局只读数据等
├─────────────────┤
│     代码区      │  ← 编译后的机器指令（只读）
低地址
```

> 上图是 **教学用概念模型**；真实 OS（ASLR、mmap、线程栈、共享库）布局更复杂，但 **「谁放哪、谁管生命周期」** 的逻辑不变。

| 区域 | 典型内容 | 谁分配 / 谁释放 | K&R 对应 |
|------|----------|-----------------|----------|
| **栈区** | 局部 `auto` 变量、函数 **形参**、返回地址等 | **编译器** 生成代码自动压栈/弹栈；函数返回即失效 | [4.1](./4.1-函数的基本知识.md) 值传递、[2.4](../ch02-types-operators-expressions/2.4-声明.md) |
| **堆区** | `malloc` 返回的块、动态数组、链表节点 | 程序员 **`malloc` / `free`**（或 `calloc`/`realloc`） | [7.8.5](../ch07-input-and-output/7.8-other-functions/7.8.5-存储管理函数.md)、[6.5](../ch06-structures/6.5-自引用结构.md) |
| **全局/静态存储区** | 文件级 **全局变量**、**`static` 局部**、**`static` 全局** | 程序 **加载时** 分配，**进程结束** 才释放 | [4.3](./4.3-外部变量.md)、[4.6](./4.6-静态变量.md) |
| **常量存储区** | **`"hello"`** 等字符串字面量、只读常量 | 加载时放入，运行期 **不可写**（写则未定义行为/崩溃） | [1.9](../ch01-introduction/1.9-字符数组.md)、[5.5](../ch05-pointers-and-arrays/5.5-字符指针与函数.md) |
| **代码区** | 函数体的 **机器指令** | 加载后只读执行 | 编译链接后的 `.text` 段 |

## 各区详解

### 1. 栈区

- **主要放**：函数内的局部变量、调用时的 **参数副本**（C 值传递）、**返回地址** 等 **临时数据**
- **特点**：**自动** 管理 —— 每次 **调用** 压入一块 **栈帧**，**返回** 时弹栈销毁；**无需** `malloc`/`free`
- **生命周期**：限于 **当前这一次调用**；出了作用域或 `return` 后，这块栈帧上的局部变量 **不再有效**（[4.4 块作用域](./4.4-作用域规则.md)）
- **易混**：栈上的 **普通局部** 随 `return` 销毁；**函数内 `static` 局部** 在 **静态区**，**跨调用保留值**（[4.6 §1](./4.6-静态变量.md)）
- **易混**：栈帧 **每调一次建一次、返一次毁一次**；但 **函数本身的机器指令不在栈上**（见 §6）

```c
void foo(int n)    /* n 在栈上 */
{
    int x = 0;     /* x 在栈上 */
}
```

### 2. 堆区

- **主要放**：运行期 **大小不定** 或 **寿命跨函数** 的数据
- **特点**：`malloc`/`calloc`/`realloc` **手动申请**，`free` **手动释放**；不释放 → **内存泄漏**
- **失败**：`malloc` 返回 `NULL`，必须检查
- **与栈对比**：堆块 **不会** 随函数返回自动回收；指针保存着就能一直用（直到 `free`）

```c
int *p = malloc(10 * sizeof(int));  /* 堆上 10 个 int */
if (p == NULL) { /* 处理失败 */ }
free(p);
```

### 3. 全局/静态存储区

- **主要放**：
  - 无 `static` 的 **全局变量**（**外部链接**，可跨文件 `extern`，[4.3 §1.1](./4.3-外部变量.md)）
  - 有 `static` 的 **文件级全局**（**内部链接**，占本区但 **仅本 `.c`**，不算跨文件外部变量）
  - 函数内的 **`static` 局部**（生命周期 = 整个程序，但名字仍只在函数内）
- **特点**：**不在栈上**；程序启动时分配，进程结束前 **一直存在**
- **初值**：未显式初始化 → **清零**（0、`'\0'`、`NULL`）

```c
int g_count;              /* 全局/静态区，初值 0 */
static int file_private;  /* 同上，但仅本文件链接 */

void f(void)
{
    static int call_count; /* 静态区，非栈；保留上次值 */
}
```

### 4. 常量存储区

- **主要放**：
  - **字符串字面量** `"abc"`
  - **`const` 修饰的只读数据**（尤其 **全局/静态 `const`**）
  - 数值字面量有时 **内嵌在代码区** 作立即数，有时也进只读段 —— 取决于编译器优化
- **特点**：**只读**；程序 **运行期间不能修改**（写 → 未定义行为 / 段错误）
- **`#define` 宏常量**：**不进** 常量区 —— 预处理阶段 **名字就被替换掉**，宏名 **本身不占** 运行时内存（[2.3 §0](../ch02-types-operators-expressions/2.3-常量.md)、[1.4](../ch01-introduction/1.4-符号常量.md)）
- **与字符数组对比**：`char a[] = "hi";` 在 **栈**（局部）或 **静态区**（全局），**可修改** `a[0]`

```c
#define MAX 100              /* 预处理替换；无 MAX 这个运行时对象 */

const int limit = 100;       /* 只读；常量区；默认内部链接，仅本 .c */
extern const int max = 200;  /* 唯一定义时可导出；他文件 extern const int max; */
char *p = "hello";           /* p 可改，指向的 "hello" 在常量区；*p='H' 未定义行为 */
char a[] = "hello";          /* a 在栈（局部）或静态区（全局）；a[0]='H' 合法 */
```

### 5. 代码区

- **主要放**：编译器生成的 **机器指令** —— 函数的 **程序段**（各函数体的 `.text`）
- **特点**：**只读、可执行**；程序 **加载时** 就映射进内存，**进程运行期间一直在**
- 调试时「程序计数器 PC 指向代码区某地址」即正在执行哪条指令
- **不会** 随函数调用在栈上 **创建/销毁** —— 调用只是 **跳转到** 代码区里该函数的入口地址执行

### 6. 函数调用：代码区（指令）vs 栈区（栈帧）

**两件事要分开看**：

| | **放哪** | **何时创建/销毁** |
|---|---------|-------------------|
| **函数的执行代码**（机器指令） | **代码区** | 程序加载时就有；**始终存在**，不跟调用次数增减 |
| **这次调用的临时数据**（局部变量、参数、返回地址） | **栈区**（栈帧） | **每次调用** 分配一块；**返回时** 弹栈销毁 |

```
代码区（固定）                    栈区（每次调用变）
┌──────────────────┐             ┌─────────────────┐
│ foo 的机器指令    │             │ foo 第 3 次调用的栈帧 │
│ bar 的机器指令    │  call foo → │  返回地址、n、x   │
│ main 的机器指令   │             ├─────────────────┤
└──────────────────┘             │ foo 第 2 次调用的栈帧 │  ← 已返回则销毁
                                 └─────────────────┘
```

调用 `foo(10)` 时：

1. CPU **跳到代码区** 里 `foo` 的入口执行 —— **不是** 把 `foo` 的代码复制到栈上
2. 栈区 **新压入一帧**，放：实参副本 `n`、局部 `x`、返回到 `main` 的 **返回地址** 等
3. `foo` **return** 后：栈帧 **弹出**；代码区里的 `foo` **仍在**，下次还能再调

**递归**（[4.10](./4.10-递归.md)）：`printd` 的指令在代码区 **只有一份**；但每层递归 **各压一栈帧**（各自的 `n`、返回地址）—— 深度过大 → **栈溢出**，不是代码区被撑爆。

```c
void foo(int n)
{
    int x = 0;     /* x、n、返回地址 → 栈帧（每次调用一份） */
}                  /* foo 的机器码 → 代码区（全程一份） */
```

## 和前面章节怎么串

| 你学过的概念 | 落在哪一区 |
|--------------|------------|
| `main` 里 `int i;` | **栈**（`main` 的栈帧） |
| `foo()` 函数体的机器指令 | **代码区**（加载后固定，不随调用销毁） |
| 调用 `foo(a)` 时的形参副本、返回地址 | **栈**（本次调用的栈帧） |
| `int global_num = 100;`（`a.c`） | **全局/静态区** |
| `static int count;`（函数内） | **全局/静态区**（名字仍局部可见） |
| `printf("%d", global_num);` 用的数据 | 读 **全局/静态区** |
| 以后 `malloc(sizeof(Node))` | **堆** |
| `"K&R"` 传给 `printf("%s", ...)` | 字符串在 **常量区** |
| `#define SIZE 1024` 里的 `SIZE` | **不占** 运行时内存（预处理已替换） |
| `int num = 10;`（`a.c`） | 全局/静态区；`b.c` **`extern int num;`** 可用可改 |
| `static int s_num = 20;`（`a.c`） | 全局/静态区；**仅 `a.c`**，**可改** |
| `static const int sc = 40;`（`a.c`） | **常量区**；**仅 `a.c`**；**不可改、不可 extern** |
| `const int c_num = 30;`（`a.c` 默认） | **常量区**；**仅 `a.c`** |
| `extern const int c_num = 30;` | **常量区**；他文件可 `extern` **只读** 引用 |

## 易混点

| # | 要点 |
|---|------|
| 1 | **局部变量** 默认在 **栈**；**全局 / static** 在 **全局/静态区**，不是「因为写了 static 就进栈」 |
| 2 | **`static` 局部**：占 **静态区**，生命周期像全局，**作用域** 仍只在函数内 |
| 3 | **堆** 必须配对 `free`；**栈** 自动回收 |
| 4 | **字符串字面量** 在 **常量区**；`char[]` 拷贝在栈或静态区 |
| 5 | **存储区**（全局/静态区）≠ **链接**（能否跨文件）：`static` 全局在前者、不在后者 |
| 6 | 五区是 **逻辑划分**；链接脚本里的 `.data` / `.bss` / `.text` / `.rodata` 与之对应，细节见 ch08 |
| 7 | **函数代码** 在 **代码区**（一份）；**局部变量/参数/返回地址** 在 **栈帧**（每次调用一份） |

## HFT / 内核提示

- **HFT 热路径**：少 `malloc`（堆分配慢、碎片）；对象池、预分配、栈上小缓冲
- **内核**：没有用户态 `malloc`；`kmalloc`/`vmalloc` 是另一套分配器；内核栈 per-thread 固定大小，**栈溢出** 致命
- **嵌入式**：栈空间常只有几 KB，大数组勿放局部；全局/static 占 Flash/RAM 需心里有数
