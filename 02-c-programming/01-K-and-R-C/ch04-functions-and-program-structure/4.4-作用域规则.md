# 4.4 作用域规则

## 本节讲什么

**作用域（scope）** 决定名字在 **哪段代码里可见、能用**（编译期规则）。与 **存储期/生命周期**（内存活多久）不同 —— 见 [1.10](../ch01-introduction/1.10-外部变量与作用域.md)。另讲 **链接（linkage）**：符号能否跨 `.c` 文件合并（`extern` / `static`）。

**前置**：[3.1 块与 `{}`](../ch03-control-flow/3.1-语句与程序块.md)、[4.3 外部变量](./4.3-外部变量.md)、[4.6 静态变量](./4.6-静态变量.md)。

## 要点

### 1. 四种作用域（C 名字可见范围）

C 里名字的作用域 **主要分四种**：

| 种类 | 典型对象 | 可见范围 |
|------|----------|----------|
| **① 局部作用域** | 函数内定义的变量 | **该函数 / 花括号内**；出了范围名字失效 |
| **② 文件作用域** | 函数 **外** 定义的外部变量 | **整个 `.c` 文件**（从定义行到文件末） |
| **③ 函数作用域** | **`goto` 标号** | **整个函数** 内有效 |
| **④ 块作用域** | `for` / `if` 等 **`{}` 里** 声明的变量 | **仅该块** 内（块结束名字失效） |

> **作用域** 回答「名字在哪能写」；**生命周期** 回答「内存活多久」—— 二者相关但不相同（如 `static` 局部：块作用域 + 全程存储期）。

---

#### ① 局部作用域

在 **函数内部** 或 **花括号 `{}` 里** 定义的变量，**只在这个范围内** 能用：

```c
void f(void)
{
    int x = 10;        /* 局部：仅 f 内可见 */
    {
        int y = 20;    /* 也在 f 内，更小一层块 */
    }
    /* y 在这里不可见 */
}
```

- 默认 **`auto`** 存储；离开函数通常销毁（栈）
- 不同函数的同名局部变量 **互不影响**

见 [1.10 §1](../ch01-introduction/1.10-外部变量与作用域.md)、[2.4 局部声明](../ch02-types-operators-expressions/2.4-声明.md)。

---

#### ② 文件作用域（分两种情况）

**外部变量**：定义在 **所有函数外面**（文件顶层）。名字在 **本 `.c`** 从定义行到文件末都可见 —— 这是 **文件作用域** 的共同点。

**是否加 `static`** 决定 **别的 `.c` 能不能用**（链接属性），从而分成两种：

| 定义 | 链接 | 本 `.c` | 其他 `.c` |
|------|------|---------|-----------|
| `int num;` | **外部链接** | ✅ 直接用 | ✅ `extern int num;` 后 **整个项目** 都能用 **同一份** `num` |
| `static int num;` | **内部链接** | ✅ 直接用 | ❌ **用不了**，写 `extern` 也链接不到 |

```c
/* === a.c === */
int shared = 0;          /* 无 static：外部链接 */
static int hidden = 0;   /* 有 static：仅 a.c */

void fa(void) { shared++; hidden++; }

/* === b.c === */
extern int shared;       /* OK：shared 作用域「扩展到」整个工程的多文件 */
void fb(void) { shared++; }

extern int hidden;       /* ❌ 链接失败：hidden 被 static 锁在 a.c */
```

**无 `static`（外部链接）**：

- 本文件：文件作用域，任意函数直接访问
- **他文件**：`extern` 声明后，名字在 **那个 `.c` 全文件** 也可写 —— 口语上说，**实际作用域扩展到整个项目的多个文件**（链接后共享 **一份** 对象）
- 全程序 **只能定义一次** —— [4.3](./4.3-外部变量.md)、[4.5](./4.5-头文件.md)

**有 `static`（内部链接）**：

- **作用域仍限于当前 `.c`**，别的文件 **根本链接不到** 这个符号
- 每个 `.c` 可各有自己的 `static int num;`，互不冲突 —— [4.6](./4.6-静态变量.md)

```c
int bufpos = 0;        /* 外部链接：a.c 内文件作用域 + 他文件可 extern */

void append(int c)
{
    bufpos++;
}
```

---

#### ③ 函数作用域

**主要针对 `goto` 标号**：标号从定义点到 **函数末尾** 都有效，**不受 `{}` 块边界限制**（与变量不同）：

```c
void f(int n)
{
    if (n < 0)
        goto cleanup;

    for (int i = 0; i < n; i++) {
        if (i == 5)
            goto cleanup;    /* 可跳到函数内别处标号 */
    }

cleanup:
    printf("done\n");
}
```

- **标号** 不占存储，无类型；**函数作用域** = 标号在 **整个函数** 内可 `goto`
- 变量 **不能** 用函数作用域跨越块 —— 变量遵 **块作用域**
- 详见 [3.8 goto](../ch03-control-flow/3.8-goto语句与标号.md)

---

#### ④ 块作用域

与 **局部作用域** 类似，但更强调 **`{}` 围起来的块**；变量 **只在块内生效**，块结束即 **不可见**：

```c
void g(void)
{
    int a = 1;                 /* 块：整个函数体 */

    for (int i = 0; i < 10; i++) {   /* C99：i 的块作用域 = 仅 for 循环体 */
        int t = i * 2;
        /* i、t 仅在此块内 */
    }
    /* i、t 在这里都未定义 —— 不能写 i */

    if (a > 0) {
        int b = 2;           /* b 仅 if 块内 */
    }
    /* b 不可见 */
}
```

| 对比 | 局部（口语） | 块作用域（精确） |
|------|-------------|-----------------|
| 范围 | 函数里定义的变量 | 任意 **`{}`** 内声明的名字 |
| 典型 | `void f(){ int x; }` | `for (int i=0; …)`、`if { int t; }` |
| 结束 | 出函数或出块 | **出 `{}` 即失效** |

**遮蔽**：内层块同名变量 **遮住** 外层（[3.2 else 就近](../ch03-control-flow/3.2-if-else语句.md) 是另一规则）：

```c
int x = 1;             /* 文件作用域 */
void h(void)
{
    int x = 2;         /* 块作用域，遮住外面的 x */
    printf("%d\n", x); /* 2 */
}
```

---

### 2. 作用域 vs 链接：两句话分清

| | **作用域** | **链接属性** |
|--|------------|--------------|
| **管什么** | 在 **当前 `.c` 文件里**，**哪能看到 / 能写** 这个变量名 | **链接阶段** 把多个 `.c` 合成一个程序时，**同名符号能不能关联到同一份对象** |
| **谁管** | **编译器**（看源码位置） | **链接器**（合并 `.o` 时） |
| **典型问法** | 「这行代码能写 `num` 吗？」 | 「`a.c` 的 `num` 和 `b.c` 的 `extern num` 是同一个吗？」 |

**简单说**：

- **作用域** → 「在当前文件里 **哪能看到** 这个名字」  
  例：`static int num;` 在 **本 `.c` 所有地方** 都能看见；**出了这个 `.c` 就看不见**（别的文件源码里不能合法引用它）。

- **链接属性** → 「多个 `.c` **合并成一个程序** 时，能不能把 **同名变量关联到一起**」  
  - **无 `static` 的外部变量** → **外部链接**：合并时 **能关联**，全程序 **一份** `num`  
  - **`static` 的外部变量** → **内部链接**：合并时 **各用各的**，`a.c` 的 `static num` 与 `b.c` 的 `static num` **互不干扰**

```c
/* a.c */
static int hidden = 1;   /* 作用域：a.c 全文可见；链接：仅 a.c 这一份 */
int shared = 1;          /* 作用域：a.c 全文可见；链接：可与 b.c extern 合并 */

/* b.c */
extern int shared;       /* 作用域：b.c 全文可见该名字；链接：指向 a.c 那份 shared */
/* 不能 extern hidden —— 链接属性不允许 */
```

文件作用域的外部变量因此 **分两种**（§1②）：

1. **`static`** —— 作用域：本 `.c`；链接：**内部**，他文件接不上  
2. **无 `static`** —— 作用域：本 `.c` + 他文件经 `extern` 后也能写名；链接：**外部**，合并时 **关联成一份**

其他作用域（块、函数）的变量 **不涉及跨文件链接** —— 只有 **文件作用域** 的全局/函数名才谈链接。

其他作用域对照：

| 概念 | 问什么 | 例子 |
|------|--------|------|
| **块/局部作用域** | 名字在 **`{}` 内** 能写吗 | `for` 里的 `i` 出了循环不能用 |
| **函数作用域** | **标号** 在函数内哪都能 `goto` 吗 | `cleanup:` 标号 |

### 3. 内部链接 vs 外部链接（编译单元 = 一个 `.c`）

**编译单元（translation unit）** ≈ **一个 `.c` 源文件**（经 `#include` 预处理展开后）：

```
a.c  ──编译──→  a.o  ─┐
b.c  ──编译──→  b.o  ─┼── 链接器 ──→  a.out（最终程序）
c.c  ──编译──→  c.o  ─┘
```

- 编译器 **单独** 把每个 `.c` 编成 **目标文件** `.o`
- **链接器** 再把多个 `.o` **合并** 成一个可执行程序
- 谈「链接属性」时，**编译单元** 指的就是这个 **`.c`（+ 它 include 进来的头文件内容）**

#### 内部链接（internal linkage）

**符号只在当前编译单元内部有效**。别的 `.c` 里就算有 **同名** 符号，也是 **各自独立的个体**，链接时 **不会合并**，**互不影响**。

| 典型 | 写法 |
|------|------|
| 文件级变量 | `static int num;` |
| 文件级 **`const`**（默认） | `const int limit = 1500;` —— **未写 `extern` 时内部链接** |
| 函数 | `static void helper(void);` |

```c
/* a.c */
static int num = 1;      /* 内部链接：只有 a.o 里有这个 num */

/* b.c */
static int num = 2;      /* 另一个 num，属于 b.o —— 与 a.c 无关，不冲突 */
```

#### 外部链接（external linkage）

**链接器合并 `.o` 时，可以把不同编译单元里的同名符号关联到同一份对象**（全程序通常 **只能有一处定义**）。

| 典型 | 写法 |
|------|------|
| 文件级变量 | `int num;`（无 `static`） |
| 跨文件 **`const`** | `extern const int limit = 1500;`（定义）+ 他处 `extern const int limit;` |
| 函数 | `void foo(void);` / `int main(void)` |
| 跨单元引用 | 他处 `extern int num;` |

```c
/* a.c */
int shared = 0;          /* 外部链接：导出给链接器 */

/* b.c */
extern int shared;       /* 声明：链接时与 a.c 的 shared 绑成同一个 */
void f(void) { shared++; }
```

#### 对照表

| | **内部链接** | **外部链接** |
|--|-------------|-------------|
| **谁** | `static` 全局、`const` 全局（**默认**）、**`static` 函数** | 无 `static` 的全局变量、`extern const` 全局、普通函数 |
| **编译单元** | 符号 **只在本 `.c` 的 `.o` 里** | 符号 **可被其他 `.o` 引用** |
| **同名** | 各 `.c` 各有一份，**互不干扰** | 全程序 **一份定义**，多处 `extern` 引用 |
| **口语** | 「各用各的」 | 「合并时关联到一起」 |

局部变量、函数参数 **没有跨文件链接问题** —— 它们不在链接器符号表里争同名。

### 4. `extern`

- **声明**（不是定义）外部变量或函数：符号在 **其他翻译单元** 或本文件别处 **定义**
- 扩展 **链接** 可见性，**不扩大** 文件内的块作用域

```c
/* b.c */
extern int shared;     /* 用 a.c 的定义，b.c 全文件可见该名字 */
```

### 5. `static` 与链接（与 4.6 配合）

| 写法 | 作用域 | 链接 |
|------|--------|------|
| `int x;`（文件顶层） | 文件 | **外部** —— 全程序一份定义 |
| `const int x = 1;`（文件顶层，默认） | 文件 | **内部** —— 仅本 `.c`；常量区、只读 |
| `extern const int x = 1;`（文件顶层定义） | 文件 | **外部** —— 可经头文件跨文件只读共享 |
| `static int x;` | 文件 | **内部** —— 仅本 `.c` |
| `static void f(void);` | 文件 | **内部** —— 仅本 `.c` 可调用 |
| 函数内 `static int s;` | **块**（函数内） | 无跨文件问题 |

## 易混点

| # | 要点 |
|---|------|
| 1 | **四种作用域**：局部、**文件**、**函数**（标号）、**块** |
| 2 | 文件作用域分两种：**`static`→仅本 `.c`**；**无 static→`extern` 后扩到多文件** |
| 3 | **`goto` 标号** = 函数作用域，可跨 `{}`；**变量不行** |
| 4 | 作用域 ≠ 生命周期；`static` 局部块内名、全程存储 |
| 5 | 内层块可 **遮蔽** 外层同名变量 |
| 6 | **作用域**：本 `.c` **哪能见名**；**链接**：合并 `.c` 时同名 **能否关联** |
| 7 | **编译单元** ≈ 一个 `.c`；**内部链接** = 各 `.o` 各用各的；**外部链接** = 链接时合并 |

## HFT / 内核提示

- 缩小块作用域：C99 `for (int i=0; …)` 防 `i` 泄漏到循环外
- 文件作用域 + `static`：驱动 `.c` 私有全局，等价于 C++ 里「翻译单元内 private」
- 读内核：区分「这个名在本文件哪都能见」vs「别的 `.c` 能不能 `extern`」
