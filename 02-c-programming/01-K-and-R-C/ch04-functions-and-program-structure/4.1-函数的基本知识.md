# 4.1 函数的基本知识

## 本节讲什么

C 程序由函数构成：`main` 是入口，其余函数把逻辑拆成可复用模块。掌握 **返回类型写法**、完整语法、**值传递**、**声明 vs 定义**，是多文件工程与内核代码的基础。

**前置**：[1.7 函数](../ch01-introduction/1.7-函数.md)、[2.4 声明](../ch02-types-operators-expressions/2.4-声明.md)。

## 要点

### 1. 返回类型写在函数名 **左边**

C 里函数的 **返回值类型** 写在 **函数名左侧**：

```c
int add(int a, int b)
/*↑ 返回类型   ↑ 函数名   ↑ 形参列表 */
{
    return a + b;
}
```

| 部分 | 位置 | 示例 |
|------|------|------|
| **返回类型** | 函数名 **之前** | `int`、`void`、`float *`、`struct node` |
| **函数名** | 形参列表 **之前** | `add`、`main`、`read_pkt` |
| **形参列表** | `()` 内 | `(int a, int b)`、`(void)` |
| **函数体** | `{ ... }` | 语句与 `return` |

**声明（原型）** 与 **定义** 都遵循同一顺序 —— 返回类型始终在函数名左边：

```c
int add(int a, int b);              /* 声明：末尾分号，无函数体 */

int add(int a, int b)               /* 定义：带函数体 */
{
    return a + b;
}
```

### 2. C89 及以前：省略返回类型 → 隐式 `int`；C99 起必须显式写出

| 标准 | 没写返回类型 | 没写 `return` 的 `int` 函数 |
|------|-------------|---------------------------|
| **C89 及以前** | 默认按 **`int`** 处理 | **仍视为返回 `int`**，但值 **未定义**（寄存器/栈残留，相当于「随机」） |
| **C99 及以后** | **编译错误** —— 必须显式写返回类型 | 非 `void` 函数应有 `return`；落到 `}` 不 `return` → 返回值 **未定义**（UB） |

```c
/* C89 时代允许 —— 编译器当作 int f(void) */
f()
{
    printf("no return\n");
}   /* 调用方仍可能当 int 用返回值，但值是垃圾 —— 严禁 */

/* C99+：下面直接不过编译 */
/* f() { ... }   error: type defaults to 'int' in C99 */
```

**现代规范**：**始终显式写出返回类型**，不要依赖隐式 `int`：

```c
int add(int a, int b);    /* 对 */
void log_msg(const char *s);
/* f() { ... }            C99 起：不写返回类型 → 报错 */
```

`main` 省略返回类型时老编译器默认 `int main()` —— 见 [1.1 入门](../ch01-introduction/1.1-入门.md)；新代码一律 **`int main(void)`** 或 **`int main(int argc, char *argv[])`**。

### 3. `void` 必须自己写在函数名左边 —— 才表示无返回值

C **不会** 因为你不写 `return` 就自动变成「无返回值函数」。要在函数名左边 **明确写 `void`**：

```c
void greet(const char *name)   /* void 在左边 → 无返回值 */
{
    printf("Hello, %s\n", name);
    /* return; 可写可不写；不能 return 1; */
}

int add(int a, int b)          /* int 在左边 → 有返回值 */
{
    return a + b;
}
```

| 左边写的 | 含义 | 调用方能拿返回值吗 |
|----------|------|-------------------|
| **`int` / `float` / 指针等** | 有返回值 | **能** |
| **`void`** | **无返回值** | **不能**（写了 `x = f();` 且 `f` 为 `void` → 编译错误） |

**无返回值 ≠ 省略返回类型**。省略在 C89 里是隐式 `int`，在 C99 里是 **错误**。

### 3.1 别混：`void`（无返回值） vs `void *`（通用指针）

`void` 出现在函数名左边和出现在指针里 **完全是两回事**：

| 写法 | 角色 | 含义 |
|------|------|------|
| **`void` 单独作返回类型** | 函数左边 | **无返回值** —— `void f(void);` |
| **`void *` / `void *p`** | 指针类型 | **通用指针** —— 可指向 **任意类型** 的对象地址 |

```c
void log_msg(const char *s);   /* void：函数不返回值 */

void *malloc(size_t n);        /* void *：返回「指向未知类型的指针」 */
```

**`void *` 的规则**（K&R 后几章、标准库 `malloc`/`read` 常见，详见 [5.1 指针](../ch05-pointers-and-arrays/5.1-指针与地址.md)、[2.4 声明围绕标识符](../ch02-types-operators-expressions/2.4-声明.md)）：

声明读法：**`*p` 是 `void` → `p` 是 void 指针** —— 类型修饰 **围绕标识符** `p`，与 `int *a[10]` 同一套规则。

| 能做什么 | 不能做什么 |
|----------|------------|
| 存放 **任意** 对象指针（`int*`、`char*`、`struct*` 赋给 `void*` 常 **无需强转**） | **不能直接解引用** `*p` —— 编译器不知道读几个字节 |
| 与其他对象指针 **比较**（同地址）、**相等判断** | 不能 `p[i]`、`(*p).x` 等，除非先转型 |
| `printf("%p", p)` 打印地址 | 指针算术 `p++` 在 `void*` 上 **GCC 扩展/不推荐**；先转成具体类型 |

**使用前必须强转成具体类型指针**：

```c
void *raw = malloc(10 * sizeof(int));
int *arr = (int *)raw;       /* 强转回 int* */
arr[0] = 42;                 /* 现在才能解引用 */

char buf[128];
ssize_t n = read(fd, buf, sizeof buf);   /* read 要 void* 缓冲 —— 任意字节块 */
/* char* 可隐式传给 void* 形参 */
```

**记忆**：`void` = 「没有」；`void *` = 「有地址，但 **暂不声明** 指向哪种类型」—— 先存着，用时再 cast。

### 4. 完整形式与 `return`

```c
返回类型 函数名(形参列表)
{
    语句;
    return 表达式;   /* 非 void：应保证有返回值路径 */
}
```

| 返回类型 | `return` 用法 |
|----------|---------------|
| **`int` 等整型/浮点/指针** | `return 表达式;` — 表达式 **转换为** 声明的返回类型 |
| **`void`** | `return;` 或直接落到 `}` —— **无返回值** |

- **`return 表达式;`** 结束函数并带回值
- **非 `void`** 函数执行到 `}` 却 **没有 `return`** → 返回值 **未定义**（C99 起常报 `-Wreturn-type`）；C89 隐式 `int` 函数同理，只是类型被默认成 `int`
- **`return` 表达式会转换为函数返回类型** —— 见 [2.7 窄→宽/宽→窄](../ch02-types-operators-expressions/2.7-类型转换.md)、[4.2](./4.2-返回非整型值的函数.md)：
  - **小转大**（`float`→`double`）一般 **无精度损失**
  - **大转小**（`double`→`float`、`long`→`int`）→ **截断低位/溢出**

```c
int max(int a, int b)
{
    return a > b ? a : b;
}

void greet(const char *name)
{
    printf("Hello, %s\n", name);
    return;          /* 可写可不写 */
}
```

### 5. 值传递（call by value）

C 默认 **按值传递**：形参是实参的 **副本**，函数内改形参 **不影响** 调用者的变量。

```c
void swap_wrong(int x, int y)
{
    int t = x;
    x = y;
    y = t;           /* 只交换了副本 */
}
```

要在函数内修改调用者数据 → 传 **指针**（第 5 章）或 **用返回值** 带回结果。

调用时，实参副本和形参一起落在 **栈帧** 里（[4.3.1 §6](./4.3.1-程序内存布局.md)）；**函数本身的机器指令** 在 **代码区**，不随每次调用创建销毁。

### 6. 函数定义 vs 函数原型（声明）

口语记忆：**定义 = 造零件**，**声明 = 贴标签**。

| 比喻 | 对应 | 做什么 |
|------|------|--------|
| **造零件** | **函数定义** | 把 **所有细节** 写出来，包括 **具体怎么干活**（函数体） |
| **贴标签** | **函数原型（声明）** | 提前告诉编译器：这个零件 **叫什么**、**能接收什么**、**返回什么** |

| | **函数原型（声明）—— 贴标签** | **函数定义 —— 造零件** |
|--|------------------------------|-------------------------|
| **是什么** | 只列出函数的 **模样** | **完整实现**，写明 **具体做什么** |
| **包含** | 返回类型 + 函数名 + 参数列表 | 同上 + **`{ 函数体 }`** |
| **函数体** | **不写** | **必须写** 具体语句 |
| **末尾** | **分号** `;` | **花括号** `{ }` |
| **作用** | 让编译器 **提前有个数**，调用时做检查 | 生成机器码，**实现逻辑** |
| **放哪** | **调用之前**，或 **头文件 `.h`** | 通常 `.c` 里 |
| **次数** | 可 **多次** 声明 | 全程序 **一次** 定义 |

```c
/* 贴标签 —— 原型，无函数体 */
int add(int a, int b);

int main(void)
{
    int m = add(3, 5);   /* 调用：编译器已见过标签 */
    return 0;
}

/* 造零件 —— 定义，写出怎么干活 */
int add(int a, int b)
{
    return a + b;
}
```

**为何必须先贴标签？**

C 编译器 **从上到下** 编译。若 **调用写在定义前面**，编译器还没见过这个函数的 **标签**（原型或完整定义），就不知道参数怎么传、返回值怎么处理 → **报错**（C99 起）。

```c
int main(void)
{
    foo(1, 2);           /* 错：此时编译器还不认识 foo */
}

void foo(int a, int b) { ... }
```

**解决办法**：在调用 **之前** 写原型，或把 **定义** 挪到调用之前（定义本身自带标签信息）。

**头文件**（[4.5](./4.5-头文件.md)）：把一堆「标签」集中放在 `.h`，各 `.c` `#include` —— 多文件共享；**造零件** 留在对应 `.c`。

返回 **`float` / `double` / 指针** 等非整型时，标签上的返回类型 **必须写对** —— 漏写原型老 C 会默认 `int`，浮点结果被截断错乱，见 [4.2](./4.2-返回非整型值的函数.md) §1。

## 易混点

| # | 要点 |
|---|------|
| 1 | 返回类型在函数名 **左边**：`int add(...)`、`void f(...)` |
| 2 | **C89**：省略返回类型 → 隐式 **`int`**，没 `return` 也有「int 返回值」但 **值未定义** |
| 3 | **C99+**：省略返回类型 → **编译报错**；必须显式写 |
| 4 | **无返回值** 须自己写 **`void`** 在左边；不是不写 `return` 就行 |
| 5 | **`void`**（返回类型）≠ **`void *`**（通用指针）；后者解引用前须强转 |
| 6 | 非 `void` 函数应有 **`return` 表达式**；缺 `return` → 垃圾值 |
| 7 | **定义 = 造零件**（有体）；**声明 = 贴标签**（无体，调用前让编译器有个数） |
| 8 | 形参是 **副本** —— 值传递 |

## HFT / 内核提示

- 头文件 `.h`：**返回类型 + 函数名 + 参数列表** 组成 API 契约；改返回类型 = **破坏 ABI**，须版本化
- 热路径：`static inline int clamp(int x, int lo, int hi)` —— 返回类型显式，便于内联
- 禁用隐式 `int`：开 `-Wimplicit-int` / `-Werror=implicit-int`（GCC）在 CI 里拦遗留代码
