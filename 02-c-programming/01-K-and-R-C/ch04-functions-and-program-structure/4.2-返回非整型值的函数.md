# 4.2 返回非整型值的函数

## 本节讲什么

函数不必只返回 `int`：可返回 `float`、`double`、`char`、指针、结构体等。重点是 **`return` 表达式 → 函数返回类型** 的 **隐式转换**（与 [2.7 类型转换](../ch02-types-operators-expressions/2.7-类型转换.md) 一致），以及 **`void`** 表示无返回值。

**前置**：[4.1 函数基本知识](./4.1-函数的基本知识.md)、[2.7 隐式转换](../ch02-types-operators-expressions/2.7-类型转换.md)。

## 要点

### 1. 返回 `float` 等：原型和定义 **都要** 写对返回类型

返回 **非整型**（`float`、`double`、指针等）时，**贴标签（原型）** 和 **造零件（定义）** 的返回类型 **必须一致、必须显式写出** —— 不能省略，不能默认 `int`。

```c
/* 原型（贴标签）—— 返回 float */
float calculate(float x);

/* 定义（造零件）—— 返回类型同样是 float */
float calculate(float x)
{
    return x * 1.5f;
}
```

| 位置 | 必须写什么 |
|------|-----------|
| **原型** | `float calculate(float x);` |
| **定义** | `float calculate(float x) { return x * 1.5f; }` |
| **调用** | `float y = calculate(2.0f);` —— 编译器按 **float 返回值** 处理 |

**两边返回类型不一致** → 编译错误（或严重警告）。**只在一处写 `float`、另一处漏写** → C99 起定义处直接报错；老代码里更隐蔽。

#### 没写原型时：默认当 `int` → 浮点结果被 **截断**（经典错）

C89 / 老编译器在 **调用前没见过原型** 时，会 **隐式假定函数返回 `int`**。若函数 **实际定义** 返回 `float`：

```c
/* 错：调用在前，且无原型 */
int main(void)
{
    float y = calculate(2.0f);   /* 编译器以为 calculate 返回 int！ */
    return 0;
}

float calculate(float x)       /* 实际返回 float */
{
    return x * 1.5f;             /* 2.0f → 3.0f */
}
```

**后果**：

1. 调用处按 **`int` 返回值** 从寄存器/栈 **取值**（通常只取低 32 位或按整型解释）
2. 函数体内实际按 **`float` 位模式** 返回 `3.0f`
3. 调用方把位模式当整数再赋给 `float` → **完全错乱**；即便再窄化，也 **不是 3.0f**

这就是 K&R 时代 **必须写原型** 的核心原因之一：**非 `int` 返回类型写错标签 = 静默错结果**。

**正确做法**：

```c
float calculate(float x);      /* 先贴标签：返回 float */

int main(void)
{
    float y = calculate(2.0f); /* y == 3.0f */
    return 0;
}

float calculate(float x)
{
    return x * 1.5f;
}
```

C99 起：调用前无原型 → **直接报错**；但 **原型返回类型写错**（如原型写 `int`、定义写 `float`）仍可能链接通过、运行错乱 —— **原型与定义必须一致**。

### 2. 其他非整型返回类型

```c
float  atof(char s[]);
double square(double x);
char  *strdup(const char *s);
void   warn(char *msg);
```

**规则相同**：原型和定义的 **返回类型** 都要显式、一致。

### 3. `return` 会隐式转换为声明的返回类型

**规则**：`return 表达式;` 时，表达式类型 **自动转换** 为函数声明的返回类型 —— 与赋值、二元运算中的隐式转换 **同一套规则**（见 [2.7 §2 常规算术转换](../ch02-types-operators-expressions/2.7-类型转换.md)）。

#### 窄 → 宽（小转大）：一般安全

| `return` 表达式 | 函数返回类型 | 结果 |
|-----------------|-------------|------|
| `float` | `double` | **精度保留**，通常无损失 |
| `int` | `long` | 值在 `int` 范围内 → **安全扩宽** |
| `short` | `int` | 整型提升，**安全** |

```c
double read_price(void)
{
    float tick = 123.45f;
    return tick;       /* float → double：小转大，OK */
}
```

#### 宽 → 窄（大转小）：可能精度损失或溢出

| `return` 表达式 | 函数返回类型 | 风险 |
|-----------------|-------------|------|
| `double` | `float` | **有效位截断**，低位丢失 |
| `long` / `long long` | `int` | 超出 `int` 范围 → **溢出** |
| `double` / `float` | `int` | **截断小数**；超大值溢出 |
| `int` | `char` / `short` | 高位截断 |

```c
int truncate_pi(void)
{
    return 3.14;       /* double → int：砍掉小数 → 3，非四舍五入 */
}

int from_float(void)
{
    float x = -2.7f;
    return x;          /* → -2，不是 -3 */
}
```

float narrow(void)
{
    double x = 1.23456789012345;
    return x;          /* double → float：精度损失 */
}

int overflow_risk(void)
{
    long L = 10000000000L;
    return L;          /* long → int：若装不下 → 截断/溢出 */
}
```

编译器对 **宽→窄** 常给 **`-Wconversion`** 警告；工程上应 **让返回类型与表达式匹配**，或 **显式 cast** 并确认范围：

```c
float narrow_explicit(double x)
{
    return (float)x;   /* 意图明确：接受 float 精度 */
}
```

### 4. `void`：无返回值

```c
void warn(char *msg)
{
    fprintf(stderr, "warning: %s\n", msg);
    return;            /* 可省略 */
}
```

- 只能 `return;`，不能 `return 表达式;`
- 调用方不能 `x = warn(...);`

### 5. 其他返回类型注意点

| 类型 | 注意 |
|------|------|
| **指针** | 返回 **局部变量地址** → 悬垂指针（ch05）；`malloc` 返回的堆内存 OK |
| **数组** | **不能** 返回局部数组（数组不能整体按值返回）；返回指针或 `struct` 包装 |
| **结构体** | 按值复制返回，大结构体有拷贝开销 |

```c
double square(double x)
{
    return x * x;      /* 同类型，无转换 */
}
```

### 6. 旧式 K&R 函数定义（了解）

ANSI 原型 `double f(double x)` 是规范写法。遗留代码可见 K&R 风格无参数类型列表的定义 —— 读代码时对照 [4.1](./4.1-函数的基本知识.md)。

## 易混点

| # | 要点 |
|---|------|
| 1 | 返回 `float` 等：**原型 + 定义** 都要写 **`float`**，不能只在一处写 |
| 2 | **无原型** 调用 → 老 C 默认 **`int` 返回**，`float` 函数结果 **截断/错乱** |
| 3 | `return` 表达式 **隐式转换** 为函数返回类型 |
| 4 | **小转大**（`float`→`double`）一般 **无精度损失** |
| 5 | **大转小**（`double`→`float`、`long`→`int`）→ **截断/溢出** |
| 6 | 不能返回局部数组；指针注意生命周期 |

## HFT / 内核提示

- 策略信号：统一 `double` 或 `int64_t` 返回，避免 `float`/`double` 来回窄化
- 驱动错误码：`int` 返回 0/-EINVAL；纯副作用用 `void`
- API 设计：返回类型与实现 **同宽**，减少隐式转换惊喜
