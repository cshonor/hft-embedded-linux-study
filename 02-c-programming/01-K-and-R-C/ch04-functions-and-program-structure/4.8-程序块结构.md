# 4.8 程序块结构（RPN 逆波兰计算器工程解读）

## 本节讲什么

K&R 用 **逆波兰表达式计算器（RPN Calculator）** 作为 **多文件模块化工程** 的完整示范：把 **头文件 + 多源文件分工**、`extern` 全局共享、`static` 模块隔离（[4.3](./4.3-外部变量.md)、[4.5](./4.5-头文件.md)、[4.6](./4.6-静态变量.md)）串成一条可运行的链路。

**前置**：4.1 函数、4.3 外部变量、4.5 头文件、4.6 `static`。

## 一、文件分工对照表

| 文件 | 核心职责 | 关键语法/知识点 |
|------|----------|----------------|
| **`main.c`** | 程序入口主循环：读输入、分发指令、串联各模块 | 调 `getop()` 解析、`push`/`pop` 运算；主业务调度 |
| **`stack.c` + `stack.h`** | 独立 **栈模块**：`push()` 入栈、`pop()` 出栈 | `.h` 放声明，`.c` 放实现；内部用 **`static`** 隔离 |
| **`getop.c`** | **词法解析**：从输入拆数字/运算符 | 只依赖公共头 `calc.h`，**不耦合** 栈内部实现 |
| **`calc.h`** | **全局共享** 头文件 | 常量宏、`extern` 全局声明、统一 `#include` 各模块头 |

```
calc.h  ──include──→  stack.h
   ↑                      ↑
main.c              stack.c
getop.c
```

```bash
gcc -o rpn main.c stack.c getop.c
```

## 二、知识点拆解

### 1. 头文件与源文件分离（`stack.c` / `stack.h`）

- **`stack.h`**：只写 **函数声明、类型、宏** —— **没有** 函数体；其他文件 `#include "stack.h"` 即可调用 `push`/`pop`。
- **`stack.c`**：写 **具体实现**；栈缓冲区、栈顶指针用 **`static`** —— **仅本文件可见**，外部 **不能** 直接改 `buf`/`sp`，只能通过 API 访问。

```c
/* stack.h */
#ifndef STACK_H
#define STACK_H
void push(double val);
double pop(void);
#endif

/* stack.c */
#include "stack.h"

static double buf[100];   /* static：仅 stack.c；K&R 书中常用 val[] + sp */
static int sp = 0;

void push(double val)
{
    if (sp < 100)
        buf[sp++] = val;
}

double pop(void)
{
    if (sp > 0)
        return buf[--sp];
    return 0.0;
}
```

见 [4.5 栈模块示例](./4.5-头文件.md)、[4.6 §2 `static` 全局](./4.6-静态变量.md)。

### 2. `calc.h` 统一全局共享

所有 `.c` 可 `#include "calc.h"`，集中管理 **全工程共用** 内容：

```c
/* calc.h */
#ifndef CALC_H
#define CALC_H

#include <stdio.h>
#include "stack.h"

#define MAXVAL 100          /* 宏常量，预处理替换 */

extern int stack_size;      /* 声明：定义在某一 .c */

int getop(void);
/* NUMBER、'+'. 等 token 常量、共享缓冲 s[] 等 —— 依 K&R 习题逐步加入 */

#endif
```

| `calc.h` 里放什么 | 示例 |
|-------------------|------|
| **常量** | `#define MAXVAL 100` |
| **`extern` 全局声明** | `extern int stack_size;` —— **定义** 在 `stack.c`：`int stack_size = 100;` |
| **统一 include** | `#include "stack.h"` —— 各 `.c` 只 include `calc.h` 即可 |

**全工程一份内存**：任一文件经 `calc.h` 读写 `stack_size`，改一处他处见最新值（[4.5 §2.1](./4.5-头文件.md)）。

### 3. 模块解耦优势（对应 [4.5 §8](./4.5-头文件.md)）

| 优势 | RPN 中的体现 |
|------|----------------|
| **职责隔离** | 改栈逻辑只动 `stack.c`，不动 `getop.c` |
| **编译提速** | 只改 `getop.c` → **只重编 `getop.o`**，再链接已有 `main.o`、`stack.o` |
| **命名冲突隔离** | 各模块内部 `static` 变量/函数，各 `.c` 可各有 `static int count` |
| **复用性** | `stack.h` + `stack.c` 可复制到其他需要栈的项目 |

`getop.c` **只** 通过 `calc.h` 知道 `getop`/`push` 等接口，**不知道** `buf`、`sp` 的存在 —— **信息隐藏**。

### 4. 数据流（完整运行链路）

```
用户输入 "3 4 +"
    │
    ▼
main.c  主循环读字符
    │
    ▼
getop()  拆出 NUMBER(3)、NUMBER(4)、运算符 '+'
    │
    ├─ 数字 ──→ push(3) ──→ push(4)     [stack 模块]
    │
    └─ '+'  ──→ pop()、pop() 运算 ──→ push(结果)
    │
    ▼
EOF 后 pop() 输出最终结果
```

后缀式：`3 4 +` = 先压 3、4，遇 `+` 弹出两数相加再压回 —— 融合 ch03 控制流 + ch04 函数与模块。

## 三、易错点

| # | 错误 | 正确 |
|---|------|------|
| 1 | 在 **`calc.h` 定义** `int num;` | **`.c` 定义一次**，`.h` **`extern` 声明** —— 否则多 `.c` include → **重复定义** |
| 2 | 栈缓冲 `buf`、`sp` **不加 `static`** | **必须 `static`** —— 防他文件直接改栈，破坏封装 |
| 3 | **无 include 守卫** | `#ifndef CALC_H` / `#define` / `#endif` —— 防嵌套 include 重复声明 |
| 4 | 在 `.h` 写 **`static` 函数实现** | 实现放 `.c`；对外 API 只在 `.h` **声明** |

## 四、与前面章节串联

| 知识点 | RPN 中的落点 |
|--------|----------------|
| **`static` 文件级变量** | `stack.c` 里 `static double buf[]`、`static int sp` —— 仅本 `.c` |
| **`extern` + `.h`** | `calc.h` 声明，`stack.c`（或指定 `.c`）定义 —— **全工程一份** |
| **`.h` 声明 / `.c` 实现** | `stack.h` / `stack.c` —— 模块封装 |
| **常量** | `calc.h` 里 `#define`；只读配置可用 `const` / `extern const`（[2.3](../ch02-types-operators-expressions/2.3-常量.md)） |
| **按模块分头文件** | `stack.h` 栈、`calc.h` 公共；类比 [4.5 `net.h`/`file.h`](./4.5-头文件.md) |

## 示例骨架：`main.c`

```c
#include "calc.h"
#include <stdlib.h>

int main(void)
{
    int type;
    double op2;

    while ((type = getop()) != EOF) {
        switch (type) {
        case NUMBER:
            push(atof(/* 共享输入缓冲 s，见书中实现 */));
            break;
        case '+':
            push(pop() + pop());
            break;
        case '*':
            push(pop() * pop());
            break;
        case '-':
            op2 = pop();
            push(pop() - op2);   /* 减除顺序：后弹出的作右操作数 */
            break;
        case '/':
            op2 = pop();
            if (op2 != 0.0)
                push(pop() / op2);
            break;
        case '\n':
            printf("\t%.8g\n", pop());
            break;
        default:
            printf("error: unknown command %c\n", type);
            break;
        }
    }
    return 0;
}
```

> 完整可编译源码随 K&R 习题 **4-3～4-10** 逐步扩展（`getch`/`ungetch`、`math.h`、函数指针表等见 **5.11**）。
