# 4.6 静态变量

## 本节讲什么

**`static`** 可修饰 **文件级变量**、**函数内局部变量**、**函数定义**（再叠 **`const`** 得只读）。从 **变量** 和 **函数** 两个层面，把符号 **本地化** 在本 `.c` 或本函数内，**避免跨文件同名冲突**；函数内 `static` 则额外保留 **跨调用状态**。

**前置**：[4.3 外部变量](./4.3-外部变量.md)、[4.4 作用域规则](./4.4-作用域规则.md)。

## 要点

### 0. 总览：`static` 的四种修饰（一张表记住）

| 修饰对象 | 写法示例 | 主要效果 | 能否被他文件看见 |
|----------|----------|----------|------------------|
| **文件级全局变量** | `static int s_num = 20;` | **仅本 `.c`**；全局/静态区；**本文件内可改** | ❌ `extern` 无效 |
| **文件级 + `const`** | `static const int sc = 40;` | **本文件只读**；**常量区** | ❌ 不能改、不能 `extern` |
| **函数内局部变量** | 函数里 `static int count;` | 作用域 **仅本函数**；存 **静态区**；**返回后不销毁**，下次调用 **保留上次值** | ❌ 不涉及跨文件 |
| **函数** | `static void helper(void);` | **仅本 `.c` 可调用**；防与他文件 **同名函数** 链接冲突 | ❌ 不必写进对外 `.h` |

```
变量层面：  文件 static / static const  →  符号锁在本 .c
           函数内 static               →  跨调用记状态（仍只在本函数可见）

函数层面：  static 函数                →  调用入口锁在本 .c
```

**一句话**：`static` 的核心是 **作用域/链接的本地化控制** —— 变量不让别的 `.c` `extern`；函数不让别的 `.c` 调用；函数内的 `static` 变量则让 **数据活过多次调用**。

**`static` 不只用在文件顶层** —— **函数内部** 也可以写。同一关键字，**放哪、修饰谁** 决定效果：

| 定义位置 | `static` 主要改变什么 |
|----------|----------------------|
| **函数内** | **生命周期 / 存储区**（作用域仍在本函数） |
| **函数外（文件顶层）** | **链接属性**（能否被他文件 `extern`） |

### 1. 函数内的 `static` 局部变量

**`static` 也可以在函数内部定义**。和普通局部变量相比：

| | **普通局部 `int x;`** | **函数内 `static int x;`** |
|---|----------------------|---------------------------|
| **作用域** | 仅本函数 | 仅本函数（**一样**） |
| **存储区** | **栈** | **全局/静态存储区**（[4.3.1](./4.3.1-程序内存布局.md)） |
| **函数返回后** | 栈帧销毁，**值消失** | **值保留**，对象 **不销毁** |
| **下次再调用** | 重新分配，**垃圾值/重新初始化** | **还能保留上次的结果** |
| **初始化** | 每次进入函数时（若写了初值） | **仅首次** 执行到声明时初始化一次 |

```c
int counter(void)
{
    static int count = 0;   /* 静态区；只初始化一次；名字仅 counter 内可见 */
    return ++count;
}

int main(void)
{
    counter();   /* 返回 1 */
    counter();   /* 返回 2 —— count 没随上次返回而清零 */
    counter();   /* 返回 3 */
    return 0;
}
```

**要点**（你的归纳）：

1. **作用域** 仍 **仅限这个函数** —— 别的函数 **不能** 写 `count`。
2. **存储** 在 **全局/静态存储区**，**不是栈** —— 所以函数调用结束后 **值不会消失**。
3. **下次再调用** 同一函数时，读到的是 **上次留下的值**（除非程序结束才释放）。

```c
void demo(void)
{
    int auto_n = 0;         /* 栈：每次调用重建 */
    static int static_n = 0; /* 静态区：全程一份，跨调用保留 */

    auto_n++;
    static_n++;
    /* 返回后 auto_n 没了；static_n 仍是累加后的值 */
}
```

**易混**：函数内 `static` **不改链接**（不会被他文件 `extern`）—— 它解决的是 **跨多次调用保存状态**，不是跨文件共享。跨文件仍看 §2 的文件级 `static`。

### 2. 文件级的 `static` 变量：限制 **跨文件** 作用域（≠ 跨文件外部变量）

**在函数外**（文件顶层）加 `static`，改变的是 **链接属性**，不是「函数返回后值还在」那套（那是 §1 函数内 `static` 局部的事）。表示 **这个变量只能在本 `.c` 里用** —— **内部链接**。它 **存储在全局/静态区**、**静态存储期** 与无 `static` 全局相同，但 **不能被其他文件 `extern` 引用**，因此 **不算** 严格意义上可跨文件共享的 **外部变量**（[4.3 §1.1](./4.3-外部变量.md)）。

#### 对比：`int num` vs `static int num`（在 `a.c` 定义）

```c
/* === a.c：普通外部变量 === */
int num;

/* b.c、c.c 使用前声明即可 */
extern int num;    /* ✅ 能访问 a.c 里的 num */
```

```c
/* === a.c：static 外部变量 === */
static int num;

/* b.c */
extern int num;    /* ❌ 无效：链接器找不到可供外部的 num */
```

| `a.c` 定义 | `b.c` 里 `extern int num;` |
|------------|---------------------------|
| `int num;` | **可以** —— 跨文件共享一份内存 |
| `static int num;` | **不可以** —— `num` 被锁在 `a.c`，别的文件 **就算写 extern 也访问不到** |

#### `static` 能否修改？看有没有 `const`

**`static` 只管存储期/链接，不表示只读**。能不能改值，要看 **`const` 有没有一起出现**：

| 写法 | 在作用域内能否赋值修改 |
|------|------------------------|
| `static int a = 5;` | ✅ **能** —— `a = 10;` 合法（文件级或函数内均可） |
| `static const int a = 5;` | ❌ **不能** —— 只读，常放 **常量区**；`a = 10;` 编译错误 |

```c
/* 文件级 */
static int s_num = 20;
static const int sc_num = 30;

void f(void)
{
    s_num = 21;        /* OK */
    /* sc_num = 31; */ /* 错误：const */

    static int count = 0;
    static const int max = 100;

    count++;           /* OK：static 局部，可改 */
    /* max = 200; */   /* 错误：static const */
}
```

**记忆**：**`static`** → 活多久、谁能跨文件看见；**`const`** → 能不能写。**两者独立**，叠在一起才是「静态存储的只读常量」。

#### `static const` 叠加：常量区 + 仅本文件 + 不可改

加了 **`const`** 的 **`static`** 变量（文件级），存放在 **常量存储区** —— **`const` 的只读属性** 符合常量区规则；**`static`** 再把它 **锁在当前 `.c`**：

| 关键字 | 贡献什么 |
|--------|----------|
| **`const`** | **只读** → 进 **常量区**；运行期 **不能修改** |
| **`static`**（文件级） | **内部链接** → **仅当前文件** 能访问；他文件 **`extern` 无效** |

```c
/* a.c */
static const int sc_num = 40;   /* 常量区；仅 a.c；不可 sc_num=41 */

void f(void) { printf("%d\n", sc_num); }   /* OK：本文件读 */

/* b.c */
extern const int sc_num;        /* ❌ 链接不到 */
/* sc_num = 50; */              /* 即使能见到也不能改 */
```

**两者结合后**：**既不能修改，也不能被其他文件访问** —— 适合放 **本模块私有的只读配置**（表长上限、魔数、版本号），比 `#define` 有类型，又比 `extern const` 更 **不暴露** 给其他 `.c`。

函数内的 **`static const`** 同样进 **常量区**、**不可改**，但 **名字作用域** 在函数内（不涉及跨文件链接问题）。

**记忆**：`static` 加在 **文件级变量定义** 上 = **对外隐藏**，只给 **本翻译单元** 用；与「全程不销毁」不矛盾 —— 生命周期仍是程序全程，只是 **链接范围** 缩小到单文件。

**为什么要这样设计**：多 `.c` 工程里，每个模块常有 `count`、`state`、`buf` 等 **内部状态**；用 **`static` 全局** 可让 **同名变量在不同文件各有一份**，既 **避免链接时的命名冲突**，又 **不污染** 其他文件的命名空间。需要 **跨文件共享** 的才用无 `static` 全局 + `extern`/头文件。

### 3. `static` 函数：仅本源文件可调用

```c
/* a.c */
static void helper(void)   /* 内部链接：只有 a.c 能调用 */
{
    ...
}

void api(void)
{
    helper();              /* OK：同文件 */
}
```

```c
/* b.c */
extern void helper(void);  /* ❌ 即使声明也链接不到 a.c 的 static helper */
void g(void) { helper(); }
```

- 函数 **定义** 前加 `static` → **内部链接**；**只能在当前源文件里被调用**
- **目的**：与文件级 `static` 变量一样 —— **避免和其他 `.c` 里的同名函数产生链接冲突**；实现细节 **不必** 放进对外 `.h`
- Linux 驱动、UEFI 模块中：**对外 API** 少数几个函数，其余 **`static` 辅助函数** 藏在本 `.c`

### 4. 对比表

| 声明 | 存储 | 名字可见范围 | 链接 | 生命周期 |
|------|------|-------------|------|----------|
| 局部自动 `int x` | 栈 | 块内 | — | 调用期间 |
| 局部 `static int x` | 静态区 | 块内 | — | 整个程序 |
| 全局 `int x` | 静态区 | 全程序可 `extern` | **外部** | 整个程序 |
| 全局 `static int x` | 静态区 | **仅本 `.c`** | **内部** | 整个程序；**非** 跨文件外部变量 |
| 全局 `static const int x` | **常量区** | **仅本 `.c`** | **内部** | 整个程序；**只读** |
| 局部 `static const int x` | **常量区** | 块内 | — | 整个程序；**只读** |
| **`static void f(void)`** | 代码区 | **仅本 `.c` 可调用** | **内部** | — |

## 易混点

| # | 要点 |
|---|------|
| 0 | **`static` 四种**：文件变量 / **`static const`** / 函数内变量 / **`static` 函数** —— 变量+函数两层本地化 |
| 1 | **`static` 可在函数内定义** —— 作用域仍在本函数，但存 **静态区**，**跨调用保留值** |
| 2 | **函数内 `static`** 改 **生命周期**；**文件级 `static`** 改 **链接/可见性** |
| 3 | `a.c`、`b.c` 各 `int num;`（无 static）→ 链接 **重复定义** |
| 4 | 一文件 `int num`、另一文件 `static int num` → `extern` 只指向 **非 static** 那份 |
| 5 | **`static` 全局** 的目的之一：**防跨文件命名冲突** —— 各 `.c` 可各有 `static int state;` |
| 6 | 面试常问：**`static` 局部 vs 全局 `static`** —— 前者块作用域，后者文件作用域 |
| 7 | **`static` ≠ 只读**；**`static const`** 才不能改 —— `static int a=5` 在作用域内可赋值 |
| 8 | **文件级 `static const`**：**常量区** + **仅本 `.c`** → **不能改、不能 extern** |

## HFT / 内核提示

- 模块内部状态机：`static` 全局 + `static` 函数封装，**.h 只暴露必要 API**
- 线程安全：`static` 局部计数器在多线程下需加锁
- 内核 `static struct` 单例驱动数据是常见模式
