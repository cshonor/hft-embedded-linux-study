# 4.11 C 预处理器

## 本节讲什么

**预处理器**在编译前处理源文件：`#define`、`#include`、条件编译等。源码 → **预处理** → 编译 → 链接。Linux 内核、UEFI、跨平台代码离不开 `#ifdef`。

## 要点

- 以 `#` 开头的**预处理指令**，不是 C 语句
- 主要能力：
  - **宏定义**（4.11.2）
  - **文件包含**（4.11.1）
  - **条件编译**（4.11.3）
- 宏展开是**纯文本替换**，无类型检查，滥用易出 bug
- 现代 C 可用 `const`、`enum`、`inline` 替代部分宏

## 与后续衔接

内核 `CONFIG_*`、DPDK `RTE_*`、UEFI 平台宏，都建立在条件编译之上；读头文件必先理解预处理阶段发生了什么。
