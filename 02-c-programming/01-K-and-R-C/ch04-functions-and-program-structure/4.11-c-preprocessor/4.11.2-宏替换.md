# 4.11.2 宏替换

## 本节讲什么

**`#define`** 定义符号常量与**带参宏**；预处理器做文本替换。第 1 章已用 `#define YES 1`，本节补充函数式宏与陷阱。

## 要点

### 简单宏

```c
#define MAXLINE 1000
#define PI 3.14159265
```

### 带参宏

```c
#define max(a, b) ((a) > (b) ? (a) : (b))
```

- 参数和整体表达式加 **`()`**，避免优先级与多次求值副作用
- 宏**无类型**；`a` 若带副作用（如 `i++`）可能被求值多次

### 宏 vs 函数

| 宏 | 函数 |
|----|------|
| 文本展开，无调用开销 | 类型检查、单点求值 |
| 无作用域，难调试 | 可断点、可 inline |

### 常用技巧

- **`#define` 续行** `\` 换行
- 字符串化 `#`、连接 `##`（高级用法，内核常见）

## 示例陷阱

```c
#define square(x) ((x) * (x))
square(z + 1)   /* 展开为 ((z + 1) * (z + 1))，正确因括号 */
```

```c
int i = 5;
max(i++, j)     /* 危险：i 可能被递增多次 */
```

## 与后续衔接

`container_of`、`ARRAY_SIZE` 等内核宏；HFT 里可用 `const` + `inline` 替代热路径宏，减少隐蔽 bug。
