# 4.11.3 条件包含

## 本节讲什么

**条件编译**：根据宏是否定义，选择性地包含或排除代码。跨平台、调试开关、可选功能的标配（Linux `CONFIG_*`、UEFI 平台宏）。

## 要点

```c
#ifdef DEBUG
    fprintf(stderr, "debug: %s\n", msg);
#endif

#ifndef MYHDR_H
#define MYHDR_H
/* 头文件内容 */
#endif

#if defined(LINUX) && !defined(__KERNEL__)
    /* ... */
#elif defined(_WIN32)
    /* ... */
#else
    /* ... */
#endif
```

### 常见指令

| 指令 | 含义 |
|------|------|
| `#ifdef` / `#ifndef` | 宏已定义 / 未定义 |
| `#if` / `#elif` / `#else` / `#endif` | 常量表达式条件 |
| `#defined(NAME)` | 在 `#if` 中测试宏是否定义 |

### 典型用途

- **调试日志**：`-DDEBUG` 编译时打开
- **平台差异**：Windows vs Linux 分支
- **内核 Kconfig**：`CONFIG_NET` 决定是否编译网络栈
- **头文件守卫**：`#ifndef HEADER_H`

## 与后续衔接

读任何大型 C 工程，先找 Makefile/CMake 里 `-D` 传递了哪些宏，再读 `#ifdef` 分支，否则容易跟错代码路径。
