# 4.11.1 文件包含

## 本节讲什么

**`#include`** 在预处理阶段把指定文件**原样插入**当前位置，实现声明共享与模块化。

## 要点

```c
#include <stdio.h>    /* 系统/标准库头：编译器搜索系统目录 */
#include "myhdr.h"    /* 本地头：通常先搜当前目录，再搜系统路径 */
```

- `#include` 可嵌套；重复包含同一头会导致重复声明 → 用 **include guard** 或 `#pragma once`
- 尖括号 vs 引号：约定上 **系统 API 用 `<>`**，项目头用 `""`
- 预处理仅做文本替换，**不进行类型检查**

## 工程习惯

- 每个 `.c` 先 include 自己的 `.h`（验证头文件自洽）
- 头文件尽量 **只放声明**，减少编译依赖

## 与后续衔接

内核 `#include <linux/...>`、UEFI `#include <Uefi.h>` 与 user space `#include <stdio.h>` 机制相同，搜索路径由构建系统（Makefile、CMake）指定。
