# 2.8 自增运算符与自减运算符

## 本节讲什么

讲解一元运算符 **`++`（自增）**、**`--`（自减）** 的两种写法：**前置** `++i` / `--i` 与 **后置** `i++` / `i--`。核心区分二者 **表达式返回值** 与 **修改变量的时机**；并强调 **同一表达式内多次修改同一变量** 属于 **未定义行为（UB）** —— 高频面试与生产禁区。

**前置**：[2.4 声明](./2.4-声明.md)、[2.10 赋值](./2.10-赋值运算符与表达式.md)；[1.3 for 语句](../ch01-introduction/1.3-for语句.md) 循环步进。  
**后续**：ch05 指针 `p++`；[2.13 求值次序](./2.13-运算符优先级与结合性.md)；C 陷阱表达式专题。

## 要点

### 一、前置 vs 后置对照（设 `i` 初值为 3）

| 写法 | 执行逻辑 | 表达式整体结果 | 运算后 `i` |
|------|----------|----------------|------------|
| **`++i`** 前置自增 | ① `i` 先 +1<br>② 返回 **新** `i` | 加完之后的值 | `i = i + 1` |
| **`i++`** 后置自增 | ① 先取 **旧** `i` 作结果<br>② `i` 再 +1 | 自增 **之前** 的原值 | `i = i + 1` |
| **`--i`** 前置自减 | ① `i` 先 −1<br>② 返回新值 | 减完之后的值 | `i = i - 1` |
| **`i--`** 后置自减 | ① 先取旧值作结果<br>② `i` 再 −1 | 自减 **之前** 的原值 | `i = i - 1` |

#### 分步拆解

```c
int i = 3;
int a = ++i;
/* ① i = i + 1  →  i = 4
   ② 表达式值为 4，赋给 a
   结果：a = 4，i = 4 */

int b = i++;
/* ① 表达式值为 i 当前值 4
   ② b = 4
   ③ i = i + 1  →  i = 5
   结果：b = 4，i = 5 */
```

自减完全对称：

```c
int j = 5;
int c = --j;   /* c=4, j=4 */
int d = j--;   /* d=4, j=3 */
```

### 二、单独使用时：前置与后置无差别

若 `++i` / `i++` **不参与** 其他运算、不赋值、不嵌套，只是 **单独一条语句**，效果相同 —— 变量都 ±1：

```c
i++;
++i;
/* 单独写，最终 i 都 +1，无差异 */
```

**典型场景**：`for` 循环步进（[1.3](../ch01-introduction/1.3-for语句.md)）

```c
for (i = 0; i < n; i++)    /* 后置 */
for (i = 0; i < n; ++i)    /* 前置 —— 此处等价 */
```

循环第三表达式只求 **副作用**（让 `i` 前进），不关心表达式 **返回值**，故 `i++` 与 `++i` **等价** —— **判断逻辑与执行次数都不变**（见 [3.5 §4](../ch03-control-flow/3.5-while循环与for循环.md)）。

#### 例外：`while` 条件里的 `++` / `--`（与 `for` 步进不同）

`for` 的第三部分在 **循环体之后** 才执行，且返回值 **不参与判断**；**`while (i++ < n)`** 却把自增塞进 **条件表达式** —— **直接改变判断时的 `i`**：

```c
int i = 9;
while (i++ < 10)   /* 用 9 比，能进体；i 变 10 */
while (++i < 10)   /* 先变 10，10<10 假，进不去 */

int j = 0;
while (j++ < 10);   /* 结束时 j == 11 */
while (++j < 10);   /* 结束时 j == 10，次数也不同 */
```

详见 [3.5 while 与 for](../ch03-control-flow/3.5-while循环与for循环.md) §2–§4。工程上更推荐 **`i < n` + 体内 `i++`** 或标准 **`for`**，少在 `while` 条件里写自增。

### 三、同一表达式多次修改同一变量 → UB（严禁）

C 标准 **不规定** 同一表达式内对同一对象的多次修改与读取的求值顺序；不同编译器、优化级别结果可完全不同。**生产代码一律禁止**。

**经典陷阱**：

```c
int i = 0;
a[i++] = i;   /* UB：同时读/写 i，顺序未定义 */
```

**同类违规**：

```c
printf("%d %d", i++, ++i);   /* UB */
int x = i++ + ++i;           /* UB */
a[i] = i++;                  /* UB */
```

| 为何危险 | 说明 |
|----------|------|
| 无标准顺序 | 编译器可任意安排子表达式求值 |
| 优化干扰 | `-O2` 可能与 `-O0` 结果不同 |
| 无诊断 | 常 **编译通过**，运行才暴露 |

**安全写法**：拆成多条语句，一次表达式只改一次：

```c
int idx = i;
i++;
a[idx] = i;

/* 或 */
a[i] = i + 1;
i++;
```

详见 [2.13](./2.13-运算符优先级与结合性.md)：**优先级 ≠ 求值顺序**；`&&` `||` `?:` 有序，多数二元运算左右次序 **不保证**。

### 四、适用范围（C 语言限制）

`++` / `--` 只能作用于 **可修改的左值（modifiable lvalue）**：

| 允许 | 不允许 |
|------|--------|
| 整型变量：`char` `short` `int` `long` `stdint` 定宽 | 常量、`++10` |
| 指针变量（ch05：`p++` 按所指类型跨度偏移） | 纯表达式：`(i+1)++` |
| | 字符串字面量、`"abc"++` |

```c
++10;       /* 非法 */
(i + 1)++;  /* 非法：非左值 */
```

**浮点**：C 标准 **不允许** `double d; d++;`（与 C++ 不同）。

### 五、C 与 C++ 的区别

| | **C** | **C++** |
|---|-------|---------|
| 对象 | 仅内置算术类型、指针 | 可对类、迭代器 **重载** `++` |
| 前置 vs 后置性能 | **无差别**（无重载） | 后置常需 **临时保存旧值**，前置迭代器惯例更高效 |
| 工程习惯 | `for` 里 `i++`/`++i` 随意 | 自定义类型优先 **`++it`** |

学 C 时勿把 C++ 的「前置更高效」套到裸 `int` 循环上 —— 对 `int` 二者生成相同机器码。

### 六、常见正确用法

```c
/* 数组遍历 */
for (int k = 0; k < n; ++k)
    buf[k] = 0;

/* 前置：需要「新值」参与后续表达式 */
if (++retry > MAX_RETRY)
    break;

/* 后置：需要「旧值」作下标/参数 */
buf[i++] = getchar();

/* 指针（ch05） */
for (char *p = s; *p; ++p)
    putchar(*p);
```

**`buf[i++] = c`** 合法：`i++` 与 `i` 的其它修改不在 **同一表达式** 内冲突 —— 仅 `i++` 修改 `i`，右侧无第二次改 `i`。

## 易混点与高频考点

| # | 要点 |
|---|------|
| 1 | **前置**：先改值，表达式返回 **新值** |
| 2 | **后置**：表达式返回 **旧值**，再改 `i` |
| 3 | **单独语句** `i++;` 与 `++i;` **无差别** |
| 4 | **`a[i++] = i`** 等同表达式多改 `i` → **UB** |
| 5 | C 仅 **整数、指针**；浮点、字面量、表达式非法 |
| 6 | C++ 迭代器前置更高效 —— **不适用于 C 的 `int`** |

## HFT / 底层提示

- 热路径循环：`for (size_t i = 0; i < n; ++i)` —— 与 `i++` 等价，风格统一即可
- 解析缓冲区：`p++` 扫字节流比反复 `buf[i]` 更常见（ch05）
- **禁止** 在宏里写 `x++` 且参数传入含副作用表达式（[1.4](../ch01-introduction/1.4-符号常量.md) 宏展开坑）
- 审查代码时搜 `++` 出现在 **同一语句多个** 处 —— 重点查 UB

## 面试题

1. `int i=3; int a=++i; int b=i++;` 最终 `a`、`b`、`i` 各是多少？
2. 为什么 `for (i=0; i<n; i++)` 与 `++i` 写法等价，而 `int a=++i;` 与 `int a=i++;` 不同？
3. `a[i++] = i;` 为什么是未定义行为？应如何改写？
4. `while (i++ < 10)` 与 `while (++i < 10)` 在 `i=9` 时谁还能进循环体？与 `for` 第三部分有何不同？
5. `printf("%d %d", i++, ++i);` 能预测输出吗？
6. C 中 `double d; d++;` 是否合法？指针 `p++` 表示什么（预告 ch05）？
7. C 与 C++ 在 `++` 上的主要区别是什么？
