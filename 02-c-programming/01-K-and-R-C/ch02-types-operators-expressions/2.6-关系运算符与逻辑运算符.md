# 2.6 关系运算符与逻辑运算符

## 本节讲什么

讲解两类运算符：**关系比较**（`<` `>` `<=` `>=` `==` `!=`）与 **逻辑组合**（`&&` `||` `!`）。核心重点是 C 的 **布尔真值规则**（0 为假、非 0 为真）以及 `&&` / `||` 的 **短路求值** —— 分支条件、循环条件、空指针安全判断都依赖本节。

**前置**：[2.4 声明](./2.4-声明.md)、[2.5 算术运算符](./2.5-算术运算符.md)；ch01 `if` / `for` 分支语法。  
**后续**：[2.7 类型转换](./2.7-类型转换.md)、[2.13 优先级](./2.13-运算符优先级与结合性.md)；[2.1.1 C99](./2.1.1-C99标准与演进.md) 中 `_Bool` / `<stdbool.h>`。

## 要点

### 1. 关系运算符（比较大小、相等）

| 运算符 | 含义 |
|--------|------|
| `<` `>` | 小于、大于 |
| `<=` `>=` | 小于等于、大于等于 |
| `==` `!=` | 等于、不等于 |

**结果类型**：`int`，只有 **`1`（真）** 或 **`0`（假）**。

```c
5 > 3;      /* 1 */
10 == 10;   /* 1 */
7 != 7;     /* 0 */
```

#### 布尔真值与 C89 / C99

| 标准 | 布尔表示 |
|------|----------|
| **C89** | 无原生布尔类型；**`0` = 假**，**任意非 0** = 真 |
| **C99** | 新增 `_Bool`；`<stdbool.h>` 提供 `bool`、`true`、`false` |

```c
#include <stdbool.h>

bool ok = (x > 0);   /* 底层仍是整数语义：true→1, false→0 */
```

关系运算 **始终返回 `int` 的 0/1**，与是否使用 `bool` 类型无关。

#### ⚠️ `=` 与 `==` 混淆（高频错误）

```c
if (x = 5)    /* 合法语法：赋值，x 变为 5，表达式值 5 → 永远“真” */
if (x == 5)   /* 正确：判断 x 是否等于 5 */
```

- `=` 是 **赋值**，表达式结果是 **右值**
- `==` 是 **相等比较**，结果是 0 或 1
- 部分编译器对 `if (x = 5)` 会警告；工程上可用 **Yoda 写法** `if (5 == x)` 防手滑（习惯因人而异）

### 2. 逻辑运算符（拼接多个条件）

| 运算符 | 名称 | 目数 |
|--------|------|------|
| `&&` | 逻辑与 | 二元 |
| `\|\|` | 逻辑或 | 二元 |
| `!` | 逻辑非 | 一元 |

#### 统一真假标准

- **`0` = 假**
- **所有非 0 整数、非空指针等 = 真**

指针单独作条件时：**`NULL` / 空指针 = 假**，**有效地址 = 真**（不必写 `!= NULL`，但显式写法更清晰）。

#### `!` 逻辑非

```c
!0;     /* 1 —— 假变真 */
!100;   /* 0 —— 真变假 */
!!x;    /* 归一化为 0/1（少见，了解即可） */
```

#### `&&` 逻辑与 —— **左假短路**

规则：**左操作数为假（0）时，整体为假，右操作数不再求值、不执行**。

```
A && B
├── A == 0  →  结果 0，B 不算
└── A != 0  →  才计算 B，结果由 B 真假决定
```

**考题**：

```c
if (i && f())
/* i 为 0：f() 根本不调用
   i 非 0：才调用 f()，整体真假看 f() 返回值 */
```

**空指针安全**（短路是标准写法）：

```c
if (p != NULL && p->len > 0)
/* p 为 NULL 时左边为假 → 不会执行 p->len，避免段错误 */
```

[1.6 数组](../ch01-introduction/1.6-数组.md) 中 `c >= '0' && c <= '9'` 同理：先排除非法字符，再参与运算。

#### `||` 逻辑或 —— **左真短路**

规则：**左操作数为真（非 0）时，整体为真，右操作数不再求值、不执行**。

```
A || B
├── A != 0  →  结果 1，B 不算
└── A == 0  →  才计算 B，结果由 B 决定
```

**考题**：

```c
if (!i || g())
/* i == 0：!i 为真，整体为真，g() 不调用
   i != 0：!i 为假，才调用 g() */
```

#### 短路与副作用

短路保证：**右侧带副作用的表达式可能被跳过** —— 既是安全特性，也是陷阱：

```c
if (ptr && *ptr++)   /* ptr 为 NULL 时 *ptr++ 不执行 */
if (lock() || retry())  /* lock 成功则 retry 不调 —— 锁顺序要刻意设计 */
```

内核 / 并发代码里，`&&` / `||` 左右顺序若牵涉 **锁、引用计数、IO**，须按依赖关系排列，不能假设两边总会执行。

### 3. 本节涉及优先级（完整表见 2.13）

从高到低（本节相关部分）：

```
!          （逻辑非，一元）
> < >= <=  （关系）
== !=      （相等）
&&         （逻辑与）
||         （逻辑或）
```

```c
a > 0 && b < 10 || !c
/* 等价于 */ ((a > 0) && (b < 10)) || (!c)
```

复杂条件 **建议加括号** 提升可读性，不必死记全表（[2.13](./2.13-运算符优先级与结合性.md) 汇总）。

### 4. 关系 vs 逻辑：结果仍是 0/1

```c
int r1 = (3 < 5);           /* 1 */
int r2 = (1 && 0);          /* 0 */
int r3 = (10 > 5 || 0);     /* 1 */
```

可赋给 `int`、`bool`（C99），也可直接用于 `if` / `while` / `for` 条件。

## 三、易混点与高频考点

| # | 要点 |
|---|------|
| 1 | 比较运算 **只返回 0/1**；C89 无 `true`/`false` 关键字 |
| 2 | **`&&` 左假短路**、**`||` 左真短路** —— 右侧函数、指针访问可能被跳过 |
| 3 | **`p != NULL && p->field`** 是判空标准模式 |
| 4 | **`=` 与 `==`** 混淆导致逻辑错误或“永远为真”的 `if` |
| 5 | 指针、整数都能当条件：**0/NULL 假，其余真** |
| 6 | 短路 ≠ 优先级：先按优先级分组，再在组内短路 |

## HFT / 底层提示

- 热路径条件尽量 **无函数调用**，或把可能失败的检查放 `&&` **左侧**（少做无用计算）
- 解析协议字段：`if (len >= HDR_SZ && buf != NULL)` —— 先长度再指针，或先指针再访问，按依赖排列
- 不要用 `if (ptr)` 与 `if (ptr != NULL)` 混用风格于同一模块；团队统一即可
- 布尔返回值函数配合短路：`if (valid_pkt(hdr) && parse_body(hdr))`

## 面试题

1. C 语言中哪些值算“假”？比较 `5 > 3` 的结果类型和值是什么？
2. `if (x = 3)` 和 `if (x == 3)` 有何区别？前者何时“合法但危险”？
3. `if (i && f())` 中，`i == 0` 时 `f()` 会被调用吗？为什么？
4. `if (!i || g())` 中，`i == 0` 时 `g()` 会被调用吗？
5. 为什么 `if (p != NULL && p->len > 0)` 能避免空指针解引用？请用短路规则解释。
6. `a > 0 && b < 10 || !c` 按默认优先级应如何加括号？
