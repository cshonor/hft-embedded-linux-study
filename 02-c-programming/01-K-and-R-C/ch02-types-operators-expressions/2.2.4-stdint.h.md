# 2.2.4 stdint.h

## 本节讲什么

**C99** 标准头文件 **standard integer types header** —— 定义 **固定宽度** 有符号/无符号整数类型，消除 `int`/`long` 随平台变化带来的溢出与协议解析错误。配套 **`<inttypes.h>`** 打印宏、**边界值宏**、内核/DPDK/HFT 必用类型全景。

选型总览见 [2.2.1](./2.2.1-整型头文件选型总览.md)；三头对比速查：[2.2.2](./2.2.2-limits-stdint-float速查.md)；原生整型范围：[2.2.3 limits.h](./2.2.3-limits.h.md)。

## 一、为什么需要 `stdint.h`？

### 痛点：同样写 `long`，平台不同宽度不同

| 环境 | `int` | `long` |
|------|-------|--------|
| 32 位 | 4 字节 | 4 字节 |
| 64 位 Linux（LP64） | 4 字节 | **8 字节** |
| Win64（LLP64） | 4 字节 | **4 字节** |

跨平台用 `long` 存时间戳、协议字段、文件偏移 → **截断、溢出、解析错位**。

**`stdint.h` 承诺**：`int32_t` **就是** 32 位有符号；`uint64_t` **就是** 64 位无符号 —— **与 CPU 架构无关**（在实现提供该类型的平台上）。

## 二、核心定宽类型（最常用）

### 有符号 `intN_t`

| 类型 | 位数 | 典型范围 |
|------|------|----------|
| **`int8_t`** | 8 | -128 ~ 127 |
| **`int16_t`** | 16 | ±32768 量级 |
| **`int32_t`** | 32 | ±约 21 亿 |
| **`int64_t`** | 64 | ±约 9×10¹⁸ |

### 无符号 `uintN_t`（仅非负）

| 类型 | 位数 | 典型最大值 | 常见用途 |
|------|------|------------|----------|
| **`uint8_t`** | 8 | 255 | **字节**、RGB、报文原始 octet |
| **`uint16_t`** | 16 | 65535 | 端口、长度字段 |
| **`uint32_t`** | 32 | ~42 亿 | IPv4、计数、tick |
| **`uint64_t`** | 64 | 极大 | 时间戳 ns、大计数 |

```c
#include <stdint.h>

uint32_t price_ticks;
uint8_t  raw_pkt[1500];
int64_t  ts_nanos;
```

## 三、特殊配套类型

| 类型 | 头文件 | 含义 |
|------|--------|------|
| **`size_t`** | `<stddef.h>` | `sizeof` 结果、数组长度、`malloc` 大小；无符号；打印 **`%zu`** |
| **`ptrdiff_t`** | `<stddef.h>` | 指针相减差值 |
| **`intptr_t` / `uintptr_t`** | `<stdint.h>` | 能 **完整存放指针值** 的整数；地址算术、与指针互转 |
| **`intmax_t` / `uintmax_t`** | `<stdint.h>` | 当前平台 **最宽** 标准整数 |
| **`int_leastN_t` / `int_fastN_t`** | `<stdint.h>` | 至少 N 位 / 运算最快至少 N 位 |

> `size_t` 不在 `stdint.h` 内定义，但与定宽类型 **天天一起用**；勿用 `%d` 打印。

## 四、边界值宏（最值）

无需手写 `255`、`0x7FFFFFFF`：

| 宏 | 含义 |
|----|------|
| `INT8_MAX` / `INT8_MIN` | `int8_t` 上下限 |
| `UINT8_MAX` | `uint8_t` 上限 |
| `INT32_MAX` / `INT32_MIN` | `int32_t` |
| `UINT32_MAX` | `uint32_t` |
| `UINT64_MAX` | `uint64_t` |

与 **`limits.h` 的 `INT_MAX`** 不同：`INT32_MAX` **恒为 32 位上限**（在提供 `int32_t` 的平台上）；`INT_MAX` 随 **本机 `int` 宽度** 变。见 [速查 §易踩坑](./2.2.2-limits-stdint-float速查.md)。

## 五、`<inttypes.h>` —— `printf` 配套宏

**完整详解**：[inttypes.h 完整讲解](./2.2.5-inttypes.h.md)。

固定宽度整数 **不能** 在所有平台乱用 `%d` / `%ld` / `%lld`；用 **格式宏** 拼接：

| 宏族 | 用途 |
|------|------|
| **`PRId8` `PRId16` `PRId32` `PRId64`** | 有符号十进制 |
| **`PRIu8` `PRIu16` `PRIu32` `PRIu64`** | 无符号十进制 |
| **`PRIx32` `PRIx64`** 等 | 十六进制 |

```c
#include <stdio.h>
#include <stdint.h>
#include <inttypes.h>

int main(void)
{
    int32_t  a = 123456;
    uint64_t b = 9999999999ULL;

    printf("a = %" PRId32 "\n", a);
    printf("b = %" PRIu64 "\n", b);
    return 0;
}
```

**写法**：`"%" PRId32` 宏展开为平台正确的 `d` / `ld` / `lld` 等。详见 [1.1.2 printf](../ch01-introduction/1.1.2-printf格式与安全.md)。

## 六、使用要求

1. **C99 起** 才有 `<stdint.h>`；编译 **`gcc -std=c99`** 或更高（本仓库 demo 常用 `-std=c11`）
2. **推荐包含顺序**：

```c
#include <stdint.h>    /* 定宽类型 + INT32_MAX 等 */
#include <inttypes.h>  /* PRId64 等（依赖 stdint） */
#include <stdio.h>     /* printf */
```

3. C89 工程无此头 —— 现代内核/DPDK/嵌入式默认 C99+

## 七、适用场景（内核 / DPDK / HFT）

| 场景 | 类型示例 |
|------|----------|
| **网络报文解析** | 字段严格 1/2/4 字节 → `uint8_t`/`uint16_t`/`uint32_t` |
| **硬件寄存器** | 位宽固定，禁止裸 `int` |
| **跨平台 ABI** | x86_32 / x86_64 / ARM 数据布局一致 |
| **环形队列 / mbuf** | DPDK `rte_*` 大量 `uint16_t`/`uint32_t` 长度与偏移 |
| **纳秒时间戳** | `int64_t` / `uint64_t` + `PRId64` |

## 九、高频易错点

1. 协议字段用 **`long`** 代替 **`uint32_t`**
2. `printf("%d", (int64_t)x)` —— 用 **`%" PRId64`**
3. 混淆 **`INT_MAX`** 与 **`INT32_MAX`**
4. 以为 **`size_t`** 在 `stdint.h` 里定义
5. 对 **`uint8_t*`** 数据用 **`%s`** 打印（无 NUL 会越界）—— 应用 **`%02x`** 循环或 `hexdump` 思维
