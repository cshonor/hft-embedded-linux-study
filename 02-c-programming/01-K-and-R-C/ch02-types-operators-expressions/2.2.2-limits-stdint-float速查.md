# 2.2.2 limits / stdint / float 速查

## 本节讲什么

**选型入口**见 [2.2.1 整型头文件选型总览](./2.2.1-整型头文件选型总览.md)。本节承接 [2.2 数据类型及长度](./2.2-数据类型及长度.md) 与 [2.3 常量](./2.3-常量.md)，对比三个标准头文件：**各自管什么类型、输出宏还是类型、工程里怎么选**。底层、网络、DPDK、内核、HFT 代码几乎天天碰 `stdint.h`；浮点边界与精度看 `float.h`；查平台原生 `int`/`long` 范围用 `limits.h`。

## 一、核心定位区分

| 头文件 | 核心用途 | 适用数据类型 | 输出内容 |
|--------|----------|--------------|----------|
| **`<limits.h>`** | 获取 **标准原生整型** 取值上下限 | `char` / `short` / `int` / `long` / `long long` | **仅宏**（范围、`CHAR_BIT`） |
| **`<stdint.h>`** | 提供 **固定宽度定长整数** 类型 | `int8_t`～`int64_t`、`uint8_t`～`uint64_t` 等 | **类型定义** + 配套最值宏 |
| **`<float.h>`** | 获取 **浮点型** 精度、极值、误差参数 | `float` / `double` / `long double` | **仅宏**（极值、epsilon、尾数/指数位数） |

**一句话**：`limits.h` 描述 **编译器默认整型有多宽**；`stdint.h` 给你 **恰好 N 位的整型**；`float.h` 只管 **浮点**，与整型头 **无交集**。

## 二、`<limits.h>`：原生整型范围

**只给宏，不新增任何数据类型**，描述当前编译目标下 **原生整型** 的边界（**实现定义**，随平台变）。逐宏表与 `CHAR_MIN` 陷阱见 **[limits.h 完整详解](./2.2.3-limits.h.md)**。

### 关键宏分组

| 系列 | 常用宏 |
|------|--------|
| **char** | `CHAR_BIT`、`SCHAR_MIN`/`SCHAR_MAX`、`UCHAR_MAX`、`CHAR_MIN`/`CHAR_MAX` |
| **short** | `SHRT_MIN`/`SHRT_MAX`、`USHRT_MAX` |
| **int** | `INT_MIN`/`INT_MAX`、`UINT_MAX` |
| **long** | `LONG_MIN`/`LONG_MAX`、`ULONG_MAX` |
| **long long**（C99） | `LLONG_MIN`/`LLONG_MAX`、`ULLONG_MAX` |

- **`CHAR_BIT`**：一个 `char` 多少 **位**（几乎总是 8）
- **`INT_MAX`**：当前平台 **`int` 能表示的最大值** —— 宽度 **不保证** 是 32 位，只保证 ≥ 16 位有效范围

### 短板

1. **无法保证比特宽度**：64 位 Windows `long` = 32 bit，Linux `long` = 64 bit —— 跨平台用 `long` 存大数易翻车
2. **没有** 8/16/32/64 **定宽** 类型名 —— 协议字段不能写 `typedef unsigned int uint32_t` 就完事

### 适用场景

- 用户态业务、单平台工具：查 `INT_MAX` 做溢出判断
- 与 `sizeof(int)` 对照，理解 **当前编译目标** 的原生类型

## 三、`<stdint.h>`：定宽整数（C99，底层/网络必备）

**完整详解**：[stdint.h 完整讲解](./2.2.4-stdint.h.md)。下文为速查摘要。

### 1）基础定长类型（必用）

| 有符号 | 无符号 |
|--------|--------|
| `int8_t` `int16_t` `int32_t` `int64_t` | `uint8_t` `uint16_t` `uint32_t` `uint64_t` |

含义 **严格固定**（若实现提供）：

- `int8_t` — 恰好 8 位有符号
- `uint32_t` — 恰好 32 位无符号

**wire format、报文头、寄存器镜像、跨进程 ABI** 一律用这套。

### 2）配套最值宏（与 limits.h 区分）

`INT8_MAX`、`UINT8_MAX`、`INT32_MIN`、`UINT32_MAX`、`INT64_MAX`、`UINT64_MAX` …

**专门对应定宽类型**，数值 **不随 `int` 宽度变化**（在提供 `int32_t` 的平台上 `INT32_MAX` 恒为 32 位有符号上限）。

### 3）配套辅助类型

| 类型 | 含义 |
|------|------|
| `int_least8_t` 等 | **至少** 8/16/32/64 位（最小满足宽度的类型） |
| `int_fast8_t` 等 | 平台 **运算最快** 的至少 N 位类型 |
| `intptr_t` / `uintptr_t` | 可 **完整存储指针值** 的整数（宽度 = 指针） |
| `size_t` / `ptrdiff_t` | 在 `<stddef.h>`，常与 `stdint` 一起用 |

### 4）`<inttypes.h>` 补充

**完整详解**：[inttypes.h 完整讲解](./2.2.5-inttypes.h.md)。速查摘要：

提供 **`PRId64`、`PRIu64`、`PRIx32`** 等 **printf/scanf 格式宏**，打印 `int64_t`/`uint64_t` 跨平台统一：

```c
#include <inttypes.h>
#include <stdint.h>
#include <stdio.h>

int64_t x = 9223372036854775807LL;
printf("%" PRId64 "\n", x);
```

避免 Win/Linux 上 `%lld` / `%ld` 不一致。

### 与 limits.h 搭配逻辑

| 场景 | 选型 |
|------|------|
| 可移植底层、网络协议、DPDK、内核 | **优先 `stdint.h` 定宽类型** |
| 仅查「本机 `int`/`long` 多大」 | `limits.h` + `sizeof` |

## 四、`<float.h>`：浮点参数

**仅针对浮点**，无整型定义，全部为宏。

### 常用核心宏

| 类别 | 宏 | 含义 |
|------|-----|------|
| **极值** | `FLT_MAX` / `FLT_MIN` | `float` 最大、最小 **正规** 数 |
| | `DBL_MAX` / `DBL_MIN` | `double` |
| | `LDBL_MAX` / `LDBL_MIN` | `long double` |
| **精度** | `FLT_EPSILON`、`DBL_EPSILON` | 1.0 与下一个可表示 `float`/`double` 的最小差（机器 epsilon） |
| **位数** | `FLT_MANT_DIG`、`DBL_MANT_DIG` | 尾数有效位数 |
| | `FLT_MAX_EXP`、`DBL_MAX_EXP` | 指数范围相关 |

### 使用场景

- 浮点比较：`|a - b| < DBL_EPSILON` 量级，而非 `a == b`
- 科学计算、图形、**量化价格边界**、仿真数值校验
- 与 [1.2 `5.0/9.0`](../ch01-introduction/1.2-变量与算术表达式.md)、[2.7 类型转换](./2.7-类型转换.md) 衔接

## 五、三者关键区别总结

| 问题 | 用谁 |
|------|------|
| 原生 `int`/`long` 上下限？ | **`limits.h`** |
| 固定 8/16/32/64 位整型 + 对应 MAX/MIN？ | **`stdint.h`** |
| 浮点极值、epsilon、尾数位？ | **`float.h`** |

### 工程选型

| 层级 | 建议 |
|------|------|
| 用户态业务 | `int` + `limits.h` 常够用 |
| 底层、网络、跨平台、内核、DPDK、HFT | **强制 `stdint.h`** |
| 浮点运算、仿真、数值计算 | 引入 **`float.h`** |

## 六、极简示例

```c
#include <stdio.h>
#include <limits.h>
#include <stdint.h>
#include <float.h>

int main(void)
{
    /* limits.h：原生 int 上限（随平台实现） */
    printf("INT_MAX = %d\n", INT_MAX);

    /* stdint.h：定宽 32 位无符号 */
    uint32_t u32 = 100;
    printf("UINT32_MAX = %u\n", (unsigned)UINT32_MAX);

    /* float.h：float 机器精度 */
    printf("FLT_EPSILON = %e\n", FLT_EPSILON);

    return 0;
}
```

## 七、易踩坑区分

### 1. `INT_MAX` ≠ `INT32_MAX`

| 宏 | 含义 |
|----|------|
| **`INT_MAX`**（limits.h） | 当前平台 **`int` 类型的上限**；常见 32 位系统上常等于 2³¹−1，但标准 **不保证** `int` 就是 32 位 |
| **`INT32_MAX`**（stdint.h） | **严格** 32 位有符号上限 **2147483647**，只要存在 `int32_t` 则恒定 |

### 2. `char` 不能替代 `uint8_t`

- `char` 有无符号 **实现定义**（`CHAR_MIN`/`CHAR_MAX` 与 `signed char` 或 `unsigned char` 对齐方式因编译器而异）
- 网络字节、DMA 缓冲、二进制协议：**必须 `uint8_t` / `unsigned char`**，保证无符号语义一致

### 3. `float.h` 与整型头无交集

查 `INT_MAX` 不要 include `float.h`；查 `FLT_EPSILON` 不要指望 `limits.h`。

### 4. 打印 `uint64_t` / `int64_t`

用 **`<inttypes.h>`** 的 `PRIu64` / `PRId64`，不要硬编码 `%llu`（部分平台会警告或错）。

## HFT / 底层实用

1. **报文字段**：`uint32_t price_ticks;` + `UINT32_MAX` 边界检查
2. **时间戳**：`int64_t` nanos，打印用 `PRId64`
3. **指针与整数**：`uintptr_t`，不用 `unsigned long` 冒充
4. **行情价浮点**：比较用 epsilon；极值用 `DBL_MAX` 防溢出
