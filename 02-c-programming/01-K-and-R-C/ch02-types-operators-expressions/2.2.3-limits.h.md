# 2.2.3 limits.h

## 本节讲什么

专讲 C 标准头文件 **`<limits.h>`**：**无函数、无类型**，只提供 **宏常量**，查询当前编译器/平台下 **标准原生整型** 的取值范围与 `CHAR_BIT`。选型总览见 [2.2.1](./2.2.1-整型头文件选型总览.md)；定宽类型见 [2.2.4 stdint.h](./2.2.4-stdint.h.md)；浮点见 [2.2.2 速查 §float.h](./2.2.2-limits-stdint-float速查.md#四-floath浮点参数)。

三头对比表：[2.2.2 速查](./2.2.2-limits-stdint-float速查.md)。

## 一、基本定义与标准沿革

| 项 | 说明 |
|----|------|
| **是什么** | 标准 C 头文件；**仅宏常量**，定义 **原生整型** `char` / `short` / `int` / `long` / `long long` 的取值范围 |
| **作用** | 替代手写 `0x7FFFFFFF`、`255` 等魔数；**跨平台**查本机 `int`/`long` 上下限 |
| **C89** | 已有 `char`～`long` 相关宏 |
| **C99** | 补充 **`long long`** 系列：`LLONG_MIN` / `LLONG_MAX` / `ULLONG_MAX` |
| **不管** | 浮点（→ `<float.h>`）、定宽 `int32_t`（→ `<stdint.h>`） |

**编译期常量**：所有宏可在 **`#if`**、数组长度（VLA 除外）、`enum` 初始化等 **编译期** 上下文使用。

### 与 `stdint.h` 必分清（配套记忆）

| 头文件 | 针对类型 | 典型用途 |
|--------|----------|----------|
| **`<limits.h>`** | **原生** `int` / `long` / `char`… | 通用业务、本机 `int` 溢出检查 |
| **`<stdint.h>`** | **定宽** `int32_t` / `uint64_t`… | 协议、硬件、DPDK、跨平台 wire format |
| **最值宏** | `INT_MAX`（随平台 `int`） | `INT32_MAX`（严格 32 位）— **互不冲突** |
| **`<inttypes.h>`** | 无类型 | `PRId64` 等 **printf** 占位宏 — [完整详解](./2.2.5-inttypes.h.md) |

定宽详解：[stdint.h 完整讲解](./2.2.4-stdint.h.md)。

## 二、全部标准宏（64 位 Linux / GCC 典型值）

> 下列「典型值」为 **实现定义**，换编译器/ABI 须实测；**宏名** 由标准规定。

### 1. `char` 相关

| 宏 | 含义 | 典型值 |
|----|------|--------|
| `CHAR_BIT` | 1 个 `char` 占用 **比特** 数 | **8** |
| `SCHAR_MIN` | `signed char` 最小值 | **-128** |
| `SCHAR_MAX` | `signed char` 最大值 | **127** |
| `UCHAR_MAX` | `unsigned char` 最大值 | **255** |
| `CHAR_MIN` | **原生 `char`** 最小值（实现定义有无符号） | **-128** 或 **0** |
| `CHAR_MAX` | **原生 `char`** 最大值 | **127** 或 **255** |
| `MB_LEN_MAX` | 多字节字符中单字符最大字节数 | **4**（常见） |

**关键坑**：C **未规定** `char` 默认有符号还是无符号 —— 不同编译器/平台不同。

- 可移植代码：优先 **`signed char` / `unsigned char`** 或 **`int8_t` / `uint8_t`**
- 明确有符号范围：用 **`SCHAR_MIN` / `SCHAR_MAX`**
- 明确无符号上限：用 **`UCHAR_MAX`**
- **少直接依赖** `CHAR_MIN` / `CHAR_MAX` 推断 `char` 语义

### 2. `short`

| 宏 | 含义 | 典型值 |
|----|------|--------|
| `SHRT_MIN` | `short` 最小值 | **-32768** |
| `SHRT_MAX` | `short` 最大值 | **32767** |
| `USHRT_MAX` | `unsigned short` 最大值 | **65535** |

### 3. `int`（主流 64 位平台常为 4 字节）

| 宏 | 含义 | 典型值 |
|----|------|--------|
| `INT_MIN` | `int` 最小值 | **-2147483648** |
| `INT_MAX` | `int` 最大值 | **2147483647** |
| `UINT_MAX` | `unsigned int` 最大值 | **4294967295** |

> **`INT_MAX` ≠ `INT32_MAX`**（后者在 `<stdint.h>`，严格 32 位上限）。见 [易踩坑](./2.2.2-limits-stdint-float速查.md#七易踩坑区分)。

### 4. `long`（平台差异最大）

| 环境 | `sizeof(long)` |
|------|----------------|
| Windows 64 位（LLP64） | **4** |
| Linux / macOS 64 位（LP64） | **8** |

| 宏 | 含义 | Linux 64 典型值 |
|----|------|-----------------|
| `LONG_MIN` | `long` 最小值 | **-9223372036854775808** |
| `LONG_MAX` | `long` 最大值 | **9223372036854775807** |
| `ULONG_MAX` | `unsigned long` 最大值 | **18446744073709551615** |

Windows 64 上 `LONG_MAX` 往往与 **`INT_MAX` 相同**（32 位 `long`）。

### 5. `long long`（C99）

| 宏 | 含义 | 典型值 |
|----|------|--------|
| `LLONG_MIN` | `long long` 最小值 | **-9223372036854775808** |
| `LLONG_MAX` | `long long` 最大值 | **9223372036854775807** |
| `ULLONG_MAX` | `unsigned long long` 最大值 | **18446744073709551615** |

跨平台大整数：**优先 `long long` + `LLONG_MAX`**，勿假设 `long` 宽度。

## 三、核心使用场景

### 1. 数值溢出判断（相加前检查）

```c
#include <limits.h>
#include <stdio.h>

int a = 2000000000;
int b = 2000000000;

if (a > INT_MAX - b) {
    printf("相加溢出\n");
} else {
    printf("%d\n", a + b);
}
```

通用写法（含负数下溢）：

```c
int add_safe(int a, int b)
{
    if (b > 0 && a > INT_MAX - b)
        return -1;   /* 上溢 */
    if (b < 0 && a < INT_MIN - b)
        return -1;   /* 下溢 */
    return a + b;
}
```

HFT / 协议层更常直接用 **`stdint.h` 定宽类型** + 显式位宽检查；`limits.h` 适合围绕 **原生 `int`** 的通用工具代码。

### 2. 通用业务与循环边界

- 不涉及硬件固定位宽、单平台工具：用 `INT_MAX` 做阈值
- 循环、缓冲区上限：**替代硬编码** `2147483647`

### 3. 跨平台兼容：拒绝硬编码魔数

```c
/* 错 */
if (num > 2147483647)

/* 对 */
if (num > INT_MAX)
```

### 4. 位运算与缓冲区

- **`CHAR_BIT`**：1 字节多少位，通用掩码/移位计算
- 与 [2.9 按位运算符](./2.9-按位运算符.md) 衔接

### 5. 无符号类型无 `UINT_MIN`

无符号最小值恒为 **0**；标准 **不提供** `UINT_MIN`、`ULONG_MIN` 等宏，判断下限时直接与 0 比即可。

### 6. 关键注意（易忘）

1. **`int` / `long` 宽度随平台变** —— 32 位 `long` 常 4 字节；64 位 Linux `long` 常 8 字节；**协议/寄存器勿依赖 `LONG_MAX` 当 64 位上限**
2. **网络 / 硬件 / DPDK** → 改用 **`stdint.h`**，不用 `limits.h` 猜位宽
3. **兼容性**：C89 可用主体宏；**`long long` 宏仅 C99+**
4. 打印 `LLONG_MAX` 用 **`%lld`**；`int64_t` 用 **`PRId64`**（`<inttypes.h>`）

## 四、与相关头文件区分

| 头文件 | 管什么 |
|--------|--------|
| **`<limits.h>`** | 标准原生整型 `char`～`long long` 范围 |
| **`<stdint.h>`** | 定宽 `int8_t`～`uint64_t` + `INT32_MAX` 等 |
| **`<inttypes.h>`** | `PRId64` / `PRIu64` 打印定宽类型 |
| **`<float.h>`** | `float` / `double` 极值与 `FLT_EPSILON` |
| **`<unistd.h>`**（POSIX） | **系统资源限制**（路径名长度等），**不是**数据类型范围 |

## 五、完整测试示例

```c
#include <stdio.h>
#include <limits.h>

int main(void)
{
    printf("int最大值：%d\n", INT_MAX);
    printf("单字节无符号最大值：%u\n", UCHAR_MAX);
    printf("long long最大值：%lld\n", LLONG_MAX);
    return 0;
}
```

```bash
gcc -Wall -Wextra -std=c11 -o limits_print limits_print.c && ./limits_print
```

打印 `long long` 用 `%lld`；若变量是 `int64_t`，改用 `<inttypes.h>` 的 `PRId64`。

## 六、内核 / DPDK 开发注意

1. **内核态** 常包含 `<linux/limits.h>` 等内核头，与用户态 `<limits.h>` 命名空间不同；**用户态标准 C** 用 **`<limits.h>`**
2. **协议与 DMA**：字段类型用 **`uint32_t`** 等，范围用 **`UINT32_MAX`**，不单靠 `UINT_MAX`
3. **`long` 陷阱**：Win64 `long` 32 位、Linux64 `long` 64 位 —— 超大计数、时间戳用 **`int64_t` / `long long`**
4. DPDK / 网络栈用户态：与内核交互的定长字段仍以 **`stdint.h`** 为准，`limits.h` 辅助查本机 `int` 行为
