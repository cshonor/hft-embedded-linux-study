# 2.3 常量

## 本节讲什么

讲解 C 语言中的常量机制：**`#define` 宏常量**（预处理文本替换、不占运行时内存）与 **`const` 只读变量 / 字面量**（编译进程序、常放常量存储区）。并分 **整型、浮点、字符、字符串** 四类字面量的语法与存储，衔接 [1.4 `#define`](../ch01-introduction/1.4-符号常量.md)、[1.9 字符数组](../ch01-introduction/1.9-字符数组.md)。

字面量属于编译器 **词法分析** 阶段的 **token**；预处理之后，编译前端按规则解析进制、转义、后缀，翻译成符号表中的数值或只读数据。

### 0. C 里「常量」的两种机制（先分清）

口语里说「常量」常混两类，**处理阶段** 和 **占不占内存** 完全不同：

| | **`#define` 宏常量** | **`const` 只读变量 / 字面量** |
|---|----------------------|------------------------------|
| **是什么** | 预处理 **文本替换** 的名字 | 有类型的 **变量** 或源码里的 **字面量** |
| **处理阶段** | **预处理**（编译前） | **编译 / 链接** |
| **运行时内存** | 宏名 **本身不占** 内存；替换后的值可能变成 **立即数** 或并入只读数据 | **`const` 全局**、**字符串字面量** 等通常进 **常量存储区**（[4.3.1 §4](../ch04-functions-and-program-structure/4.3.1-程序内存布局.md)） |
| **能否修改** | 无「对象」可改 —— 只有替换 | 运行期 **不可写**（写 `const` / 改字面量 → 未定义行为或编译错误） |
| **典型写法** | `#define MAXLINE 1000` | `const int maxline = 1000;`、`"hello"` |

```c
#define BUFSIZE 1024          /* 预处理：BUFSIZE → 1024，宏名不占运行时空间 */

const int max_conn = 64;      /* 只读变量；全局 const 常放 .rodata */
char *msg = "ready";          /* 字面量 "ready" 在常量存储区，不可改 *msg */
```

**记忆**：

1. **`#define`**：编译前 **全文替换**，**不是变量**，没有类型、一般不能取址。
2. **`const` + 字面量**：是 **程序数据**，加载后躺在 **常量区**（或局部 `const` 在栈上但仍 **只读**），运行期间 **不能改**。

### 0.1 函数外部的 `const`：像全局，但默认「文件私有」

在 **函数外**（文件顶层）写 `const`，和 **全局变量** 有相似之处，也有关键区别：

| | **无 `static` 全局 `int g`** | **文件级 `const int limit`**（默认） |
|---|------------------------------|--------------------------------------|
| **存储** | 全局/静态区（可写） | **常量存储区**（只读，[4.3.1 §4](../ch04-functions-and-program-structure/4.3.1-程序内存布局.md)） |
| **作用域** | 本文件全文可见 | 本文件全文可见（**文件作用域**，同全局） |
| **能否改值** | ✅ 运行期可写 | ❌ **只读** |
| **默认链接** | **外部链接** | **内部链接**（C 规则：未写 `extern` 的 `const` 默认内部） |
| **他文件 `extern`** | ✅ `extern int g;` 可引用 | ❌ **不能** 像普通全局那样跨文件引用 |

```c
/* a.c */
int num = 10;                 /* 非 static 全局 → b.c: extern int num; ✅ */
static int s_num = 20;        /* static 全局 → 仅 a.c ✅，b.c extern ❌ */
const int c_num = 30;         /* 常量区，默认内部 → 仅 a.c；不能 c_num=40 */

/* b.c */
extern int num;               /* ✅ 用 a.c 那份，可改 */
/* extern int s_num; */       /* ❌ */
/* extern const int c_num; */ /* ❌ 除非 a.c 定义写成 extern const int c_num = 30; */
```

若 **确实要跨文件共享** 只读常量，`a.c` 定义处须 **`extern const`**：

```c
/* a.c —— 唯一定义 */
extern const int c_num = 30;   /* 外部链接 + 只读；b.c 可 extern 引用，仍不能改值 */

/* b.c */
extern const int c_num;
printf("%d\n", c_num);         /* OK */
/* c_num = 40; */              /* 编译错误 */
```

完整三变量对照见 [4.3 §1.2](../ch04-functions-and-program-structure/4.3-外部变量.md)。

**记忆**：文件级 `const` **像全局**（整文件能用、放常量区），但 **本质是只读的**；且 **默认内部链接**，和 **非 static 可写全局** 不同 —— 他文件 **不能** 单靠 `extern` 接到你没导出的那份。

**`static` 不等于只读**：`static int s_num=20` 在 **`a.c` 内可改**；加了 `const` 才 **不能改**。

**文件级 `static const`**：`const` → **常量区**、只读；`static` → **仅本 `.c`**；**既不能改，也不能被他文件 `extern`**（[4.6 §2](./4.6-静态变量.md)）。

详见 [4.4 §3](../ch04-functions-and-program-structure/4.4-作用域规则.md)、[4.3 §1.1](../ch04-functions-and-program-structure/4.3-外部变量.md)、[1.4](../ch01-introduction/1.4-符号常量.md)。

## 要点（字面量详解）

### 1. 整型常量（整数字面量）

编译器按 **前缀** 自动识别进制：

| 进制 | 写法 | 示例 | 十进制值 |
|------|------|------|----------|
| **十进制** | 无前缀，数字 `0`～`9` | `100`、`256`、`-88` | 本身 |
| **八进制** | 以 **`0`** 开头，仅 `0`～`7` | `017` | **15** |
| **十六进制** | 前缀 **`0x` / `0X`**，`0`～`9`、`a`～`f` | `0xFF`、`0X1A` | **255**、**26** |

**陷阱**：`08`、`09` **非法**（八进制不能含 8、9）。

**后缀修饰类型**（可选）：

| 后缀 | 含义 |
|------|------|
| `U` / `u` | `unsigned` |
| `L` / `l` | `long` |
| `LL` / `ll` | `long long` |
| 可组合 | `0xFFu`、`100LL` |

- 编译期直接翻译成二进制数值，写入目标文件的常量区 / 立即数
- **运行时不可修改** —— 字面量不是左值（C++ 中字符串字面量例外地可当 `const char*` 用）

### 2. 浮点常量（浮点数字面量）

满足 **带小数点** 或 **带指数 `e`/`E`** 任意一条，即为浮点字面量；**默认类型 `double`**：

| 形式 | 示例 | 含义 |
|------|------|------|
| **小数** | `3.14`、`0.5`、`.25`、`100.` | 通常 double |
| **指数** | `1e-2`、`2.5E3` | **0.01**、**2500** |

**后缀区分精度**：

| 后缀 | 类型 |
|------|------|
| `f` / `F` | `float` |
| `l` / `L` | `long double` |
| 无后缀 | `double` |

与 [1.2 `5/9` 陷阱](../ch01-introduction/1.2-变量与算术表达式.md) 呼应：要写浮点运算，字面量至少一侧带 **`.` 或 `e` 或后缀 `f`**。

### 3. 字符常量 `'单引号'`

**字符常量** = 用 **单引号** 括起来的 **单个字符**（含转义写法也算「一个字符」）：

```c
'a'    '5'    '#'    'x'    '\n'    '\0'
```

| 写法 | 含义 | 常见 ASCII 值 |
|------|------|---------------|
| `'a'` | 字母 a | **97** |
| `'5'` | 数字 **字符** 5（≠ 整数 `5`） | **53** |
| `'#'` | 井号 | **35** |
| `'\n'` | 换行（转义字符常量） | **10** |

| 要点 | 说明 |
|------|------|
| **类型陷阱** | 字符常量类型是 **`int`**（integer promotion），**不是** `char` |
| **存储** | 编译期转为对应 **码值**（实现字节集，常见 ASCII/UTF-8 单字节） |
| **与字符串区别** | `'a'` 是一个 `int`；`"a"` 是 **两个** 字符的数组：`{'a','\0'}` |
| **与 `switch`** | `case 'a':` 匹配的是码值 **97**，见 [3.4 switch](../ch03-control-flow/3.4-switch语句.md) |

**转义字符**：

| 类别 | 写法 | 含义 |
|------|------|------|
| 控制转义 | `\n` `\t` `\\` `\"` `\'` `\0` | 换行、制表、反斜杠、双引号、单引号、空字符 |
| **八进制数值** | `\ddd`（1～3 位八进制） | `\141` → `'a'` |
| **十六进制数值** | `\xhh`（1～2 位十六进制） | `\x41` → `'A'` |

```c
'a' == 97;         /* 常见 ASCII */
'5' == 53;         /* 字符 '5' ≠ 整数字面量 5 */
'\n' == 10;        /* 换行符码值 */
'\0' == 0;         /* NUL，字符串结束标记 */
```

HFT / 协议：帧界定、字段分隔常写成 `'\x01'`、`\x00` 等 **数值转义**，比魔数直观。

### 4. 字符串常量 `"双引号"`

```c
"hello"
```

| 要点 | 说明 |
|------|------|
| **底层本质** | **只读 `char` 数组**，编译器 **自动** 在末尾追加 **`'\0'`** |
| **内存布局** | `"hello"` → `{'h','e','l','l','o','\0'}` — 长度 **6** 字节（含 NUL） |
| **存储段** | 通常放在 **只读数据段**（`.rodata`）；**修改** 字面量字符 → **未定义行为** |
| **类型** | 类型为 **`char[N]`** 数组，使用时 **衰变为** `char*` 指向首元素 |

**与 [1.9 字符数组](../ch01-introduction/1.9-字符数组.md) 对照**：

```c
char manual[] = {'h','e','l','l','o','\0'};  /* 可读写数组副本 */
char *p = "hello";                            /* p 指向只读字面量，勿写 *p='H' */
```

| | 字符串字面量 `"..."` | 手动 `char arr[]` |
|---|---------------------|-------------------|
| 可修改 | **否**（UB） | **是** |
| `\0` | 编译器自动补 | 程序员负责 |
| 典型用途 | 格式串、常量消息、符号名 | 输入缓冲、可改写行缓存 |

**相邻字面量自动拼接**（编译期）：

```c
"hello" " world"    /* → "hello world" */
```

多行宏、长格式串常用；仍是 **一个** 只读数组。

## 本节核心考点

0. **`#define` 宏常量** vs **`const`/字面量**：前者预处理替换、不占内存；后者进常量区、运行期只读
0.1 **文件级 `const`**：常量区 + 文件作用域；**默认内部链接**，跨文件须 `extern const` 定义
1. **进制前缀**：无前缀十进制、`0` 八进制、`0x` 十六进制
2. **浮点判定**：有小数点 **或** 指数 `e`/`E` → 浮点；默认 `double`
3. **`'x'` 是 `int`**，不是 `char`；`"x"` 是含 `\0` 的 `char` 数组
4. **数值转义**：八进制 `\ddd`、十六进制 `\xhh`
5. **字符串**：底层 `char[]` + 自动 `\0`，只读；与 1.9 字符数组一体两面

## HFT / 底层开发的用处

1. **协议魔数**：`0xFF`、`'\x01'` 字段分隔 — 字面量比运行时算更清晰
2. **精度后缀**：热路径 `3.14f` 避免隐式 double 运算
3. **只读配置串**：行情 topic 名 `"SHFE.au"` 放 rodata，与可写 buffer 严格区分
4. **八进制陷阱**：老代码 `012` 当十进制 12 读会错 — code review 看前缀

## 与 C++ 的差异

| 主题 | C（本节） | C++ |
|------|-----------|-----|
| 字符串字面量 | `char[]` / `char*` | `const char*`；C++14 起 `""s` 为 `std::string` |
| 字符常量 | `int` | 同为 `char` 提升为 `int` 用于运算 |
| 用户定义字面量 | 无 | `12_km`、`"txt"_s` 等 |
| 修改字面量 | UB | 同样 UB（`const` 语义更严） |
