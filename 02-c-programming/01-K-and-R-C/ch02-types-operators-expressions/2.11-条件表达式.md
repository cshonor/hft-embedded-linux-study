# 2.11 条件表达式

## 本节讲什么

讲解 C **唯一的三元运算符** **`条件 ? 表达式2 : 表达式3`**（`?:`），用单行实现简易分支逻辑。核心特性：**分支短路求值**（只算一支）、两分支结果经 **usual arithmetic conversions**（[2.7](./2.7-类型转换.md)）对齐为公共类型。

**前置**：[2.6 逻辑/关系运算](./2.6-关系运算符与逻辑运算符.md)、[2.7 类型转换](./2.7-类型转换.md)、短路概念（`&&` `||`）。  
**后续**：[2.12 逗号](./2.12-逗号运算符.md)、[2.13 优先级](./2.13-运算符优先级与结合性.md)。

## 要点

### 一、完整执行规则

**语法**：`expr1 ? expr2 : expr3`

| 步骤 | 行为 |
|------|------|
| 1 | 先计算 **`expr1`**（条件） |
| 2 | 若 `expr1 != 0`（真）：**只求值并返回 `expr2`**，`expr3` **完全不计算** |
| 3 | 若 `expr1 == 0`（假）：**只求值并返回 `expr3`**，`expr2` **完全不计算** |

与 `&&` / `||` 同属 **短路机制** —— 未选中的分支可含函数调用、指针解引用，且 **不会执行**。

**短路验证**：

```c
int a = 1;
a ? printf("执行2\n") : printf("执行3\n");
/* a 为真 → 只打印「执行2」，第二个 printf 不运行 */
```

```c
if (p != NULL ? p->len > 0 : 0)
/* p 为 NULL 时不会解引用 p->len —— 但勿滥用，if-else 更清晰 */
```

### 二、最常用极简场景

#### 1. 取最大 / 最小值

```c
int max = a > b ? a : b;
int min = a < b ? a : b;
```

#### 2. 简易绝对值（`int` 雏形，注意 `INT_MIN` 溢出）

```c
int abs_x = x > 0 ? x : -x;
```

#### 3. `printf` 格式化分支

```c
printf("数字：%s\n", x >= 0 ? "正数" : "负数");
```

#### 4. 指针 / 默认值

```c
const char *msg = err ? err_msg : "ok";
size_t n = count ? count : DEFAULT_SZ;
```

#### 5. 内核 / DPDK 宏风格

```c
#define MIN(a, b) ((a) < (b) ? (a) : (b))
/* 宏内 ?: 极常见；参数求值次数仍受 ?: 短路保护（每支只求值一次） */
```

### 三、类型转换规则（联动 2.7）

`expr2` 与 `expr3` 类型不同时，对 **两支结果** 做 **usual arithmetic conversions**，整个 `?:` 表达式的类型为对齐后的 **公共类型**。

```c
unsigned u = 10;
int i = -1;
/* 条件为 1（真）→ 取 i；但公共类型含 unsigned 时 i 转 unsigned */
unsigned res = 1 ? i : u;   /* i=-1 → 巨大 unsigned */
```

| 场景 | 风险 |
|------|------|
| `signed` + `unsigned` 两支 | signed 支可能按 unsigned 解释 |
| 窄类型 + 宽类型 | 提升为宽类型 |
| 指针两支类型不完全兼容 | 须指向 **兼容** 类型，否则编译错误 |

**安全写法**：两支 **显式 cast 到同一类型**：

```c
int64_t v = cond ? (int64_t)a : (int64_t)b;
uint32_t x = ok ? (uint32_t)val : 0u;
```

**指针两支**：

```c
void *p = cond ? (void *)pa : (void *)pb;  /* 须可安全转换 */
```

### 四、优先级与结合性

| 项 | 规则 |
|----|------|
| **优先级** | **很低** —— 低于赋值、算术、移位、关系、位运算、逻辑 `&&` `\|\|`；**高于逗号** `,`（[2.13](./2.13-运算符优先级与结合性.md)） |
| **结合性** | **右结合** → 嵌套时从右分组 |

```c
a ? b : c ? d : e
/* 等价于 */ a ? b : (c ? d : e)
```

**嵌套示例**（可读性差，慎用）：

```c
int grade = score >= 90 ? 'A' : score >= 60 ? 'B' : 'C';
/* 等价于 score >= 90 ? 'A' : (score >= 60 ? 'B' : 'C') */
```

**加括号澄清**：

```c
int m = (a > b) ? a : (c > d ? c : d);
```

与赋值混用：

```c
x = cond ? y : z;   /* 等价 x = (cond ? y : z)，因 ?: 低于 = */
```

### 五、代码风格规范

| | 说明 |
|---|------|
| **优点** | 一行替代简单 `if-else`；宏、内联取值、日志紧凑 |
| **缺点** | 多层嵌套晦涩；调试单步不便；分支含副作用时难读 |
| **建议** | **单层** `?:` 可直接用 |
| | **超过两层** 嵌套改 `if-else` |
| | 复杂逻辑 **不要** 塞进 `printf`、宏实参 |
| | 分支勿写过长表达式 —— 提取临时变量 |

```c
/* 不推荐 */
result = a > b ? (foo() + bar()) : (baz() ? qux() : 0);

/* 推荐 */
if (a > b)
    result = foo() + bar();
else if (baz())
    result = qux();
else
    result = 0;
```

### 六、与 `if-else` 语句对比

| | **`?:` 表达式** | **`if-else` 语句** |
|---|-----------------|-------------------|
| 性质 | **表达式**，有值 | **语句**，无统一“表达式值” |
| 用途 | 赋值、实参、返回值内嵌 | 多语句分支、副作用 |
| 短路 | 仅一支求值 | 仅执行一支 |
| 可读性 | 简单场景好 | 复杂场景更好 |

```c
/* 表达式语境 —— ?: 合适 */
return n > 0 ? n : 0;

/* 多语句 —— if-else */
if (n > 0) {
    log("positive");
    return n;
} else {
    return 0;
}
```

### 七、易混点

1. **`?:` 是表达式**，可嵌在 `=`、`return`、`printf` 实参中；**不是语句**
2. 条件 `expr1` **总会求值**；只有 `expr2`/`expr3` 二选一
3. 两支 **类型须能对齐**；混 signed/unsigned 见 2.7 坑
4. 右结合：`a ? b : c ? d : e` 从 **右** 挂 `?:`
5. `?:` 优先级 **低于** `&&` `||` —— 条件常可直接写关系表达式

```c
/* OK */
x = a && b ? y : z;   /* (a && b) ? y : z */

/* 易读性：加括号 */
x = (a && b) ? y : z;
```

## HFT / 底层提示

- 热路径取 min/max：`latency = t1 < t2 ? t1 : t2;` 常编译为 **条件传送**（cmov），无分支
- 日志级别：`if (lvl) ? log(...) : (void)0` 在宏里常见；注意 **短路** 避免昂贵 `printf`
- 协议字段默认值：`len = hdr->len ? hdr->len : DEFAULT_MTU;`
- 避免嵌套 `?:` 选三种以上状态 —— 用 `switch` 或查表

## 面试题

1. `expr1 ? expr2 : expr3` 中，何时 `expr2`、`expr3` 会被求值？说明短路。
2. `1 ? i : u`（`int i=-1`, `unsigned u=10`）结果类型和值可能是什么？
3. `score >= 90 ? 'A' : score >= 60 ? 'B' : 'C'` 如何结合（左还是右）？
4. `?:` 与 `if-else` 语句各适合什么场景？
5. `a ? b : c ? d : e` 等价于什么加括号形式？
6. 为什么宏 `#define MAX(a,b) ((a)<(b)?(a):(b))` 里要对参数加括号？
