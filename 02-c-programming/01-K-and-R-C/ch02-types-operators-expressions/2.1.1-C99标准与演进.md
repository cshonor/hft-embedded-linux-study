# 2.1.1 C99 标准与演进

## 本节讲什么

**C99 = ISO/IEC 9899:1999**（1999 年第二版国际标准），在 **C89（ANSI C）** 基础上大规模升级，是当今 **Linux 内核、DPDK、嵌入式、HFT** 最主流的 C 方言基线。说明 C99 相对 C89 的核心变化、**5 个新增关键字**（与 [2.1 变量名](./2.1-变量名.md) 衔接）、常用语法特性与编译启用方式。

## 一、C89 vs C99 一句话

| 标准 | 年代 | 定位 |
|------|------|------|
| **C89 / ANSI C** | 1989 | 初代统一标准；语法限制多 |
| **C99** | 1999 | **现代 C** 工业底层标配；实用语法、`<stdint.h>`、`//` 注释 |
| **C11 / C17 / C23** | 后续 | 在 C99 上继续增特性（`_Generic`、线程等） |

**演进脉络**：K&R C → **C89** → **C99** → C11 → C17 → C23  
C99 **承上启下**，几乎所有底层与高性能框架以 C99 为最低共识。

## 二、C99 新增 5 个关键字

C89 原有关键字不变；C99 **新增** 下列保留字 —— **不能单独作标识符**，**可嵌入** 更长名字（见 [2.1 §3](./2.1-变量名.md)）：

| 关键字 | 用途 |
|--------|------|
| **`inline`** | 内联函数，减少调用开销 |
| **`restrict`** | 指针无别名承诺，利于优化（DPDK/内核热点） |
| **`_Bool`** | 布尔基础类型；配 `<stdbool.h>` 用 `bool`/`true`/`false` |
| **`_Complex`** | 复数类型 |
| **`_Imaginary`** | 虚数类型 |

```c
int inline_val;    /* 嵌入：语法允许，工程不推荐 */
inline void foo(void) { }   /* inline 作函数修饰 */

_Bool flag;        /* 或 #include <stdbool.h> 后 bool flag; */
```

**混编**：除 C99 关键字，共用头文件还须避开 **C++ 独有关键字**（`class`、`namespace`、`template`…）。

## 三、相对 C89 的高频新特性

### 1. 语法层面

| 特性 | C89 | C99 |
|------|-----|-----|
| **注释** | 仅 `/* */` | 支持 **`//` 单行注释** |
| **变量声明位置** | 块内变量须在语句 **之前** | **用到再声明**；`for (int i = 0; …)` 合法 |
| **VLA 变长数组** | 无 | `int n = 10; int buf[n];`（长度运行期定；内核部分场景用，栈风险需注意） |
| **指定初始化** | 顺序初始化 | `struct A s = {.y = 10, .x = 1};` |
| **复合字面量** | 无 | `func((struct A){1, 2});` 临时结构体 |

### 2. 数据类型扩展

1. **`long long` / `unsigned long long`** — 64 位整数（见 [2.2](./2.2-数据类型及长度.md)）
2. **`_Bool`** — 布尔；`<stdbool.h>` 提供 `bool`、`true`、`false`
3. **`<stdint.h>`** — `int32_t`、`uint64_t` 等定宽类型（[2.2.1 选型总览](./2.2.1-整型头文件选型总览.md)、[2.2.2 速查](./2.2.2-limits-stdint-float速查.md)）
4. **`_Complex`** — 复数浮点

### 3. 函数与性能

- **`inline`** — 内联定义，减少函数调用（仍受 `static`/链接规则约束，ch04）
- **`restrict`** — 承诺指针是访问对象的 **唯一途径**，编译器可激进优化；误用 → 未定义行为

### 4. 预处理与库

- **可变参数宏**：`#define LOG(fmt, ...) printf(fmt, __VA_ARGS__)`
- 数学库扩展、IEEE 754 浮点、`<tgmath.h>` 类型泛化数学
- **柔性数组成员**：结构体末尾 `int arr[];`（常见内核结构体模式，ch06）

## 四、编译器如何启用

```bash
gcc -std=c99 -Wall -Wextra -o prog prog.c
clang -std=c99 ...
```

| 编译器 | 说明 |
|--------|------|
| **GCC / Clang** | `-std=c99` 明确标准；默认常为 **GNU11**（GNU 扩展 + C11 超集，**兼容多数 C99**） |
| **MSVC** | C99 支持 **不完整**；工业底层多用 GCC/Clang（WSL、Linux、交叉编译） |

本仓库 demo 常用：**`-std=c11`**（C99 超集，向后兼容 C99 特性）。

## 五、与 [2.1 变量名](./2.1-变量名.md) 的关联

1. **5 个 C99 关键字** 不能整词作变量名；`restrict_ptr` 类嵌入合法但不推荐
2. **标识符有效长度**（C99）：**外部链接** 至少 **31** 字符、**内部** 至少 **63** 字符须区分 —— 长名不再像 C89 那样易被截断歧义
3. **`restrict` / `inline`** 与 **`rte_` / `__`** 工程命名并列出现：前者是 **语言修饰符**，后者是 **项目/实现命名空间**

## 六、HFT / 内核 / DPDK 实践

| C99 特性 | 典型场景 |
|----------|----------|
| `<stdint.h>` | 报文字段、时间戳、定长协议 |
| `restrict` | 热循环内指针别名优化 |
| `inline` | 头文件内小函数、`static inline` 工具函数 |
| `//` 注释 | 全工程可读性 |
| `long long` / `int64_t` | 纳秒时间、大计数 |

**注意**：VLA 在 **Linux 内核已移除**；用户态可用但大 VLA 易栈溢出 —— 大缓冲用 `malloc` 或固定数组。
