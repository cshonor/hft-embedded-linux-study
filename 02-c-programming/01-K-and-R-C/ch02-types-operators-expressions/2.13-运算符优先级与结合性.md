# 2.13 运算符优先级与结合性

## 本节讲什么

梳理 C 全部运算符的 **优先级**（谁先分组运算）与 **结合性**（同优先级从哪边结合），并区分两个极易混淆的概念：**优先级/结合性** ≠ **求值顺序**。寄存器掩码、报文解析、复杂条件判断的歧义根源，多在本节。

**前置**：ch02 全部运算符小节（[2.5](./2.5-算术运算符.md)–[2.12](./2.12-逗号运算符.md)）汇总。  
**后续**：[附录 A.7 表达式](../appendix-a-reference-manual/A.7-表达式.md) 权威语法；ch05 指针运算符 `*` `&`；C 陷阱表达式专题。

## 要点

### 一、优先级完整排序（高 → 低，12 级）

数字越小 **优先级越高**，先参与分组；**同级** 看 **结合性**。

| 级 | 运算符 | 说明 |
|----|--------|------|
| **1** | `()` `[]` `.` `->`、后缀 `++` `--` | 调用、下标、成员、后置自增 |
| **2** | `!` `~`、前置 `++` `--`、`+` `-`（一元）、`*` `&`、`(type)` `sizeof` | 逻辑非、取反、前置自增、解引用、取址、强转 |
| **3** | `*` `/` `%` | 乘、除、模 |
| **4** | `+` `-` | 二元加、减 |
| **5** | `<<` `>>` | 移位 |
| **6** | `<` `>` `<=` `>=` | 关系比较 |
| **7** | `==` `!=` | 相等判断 |
| **8** | `&` `^` `\|` | 位与 → 位异或 → 位或（**同级内** 左结合） |
| **9** | `&&` `\|\|` | 逻辑与 → 逻辑或 |
| **10** | `?:` | 三元条件 |
| **11** | `=` `+=` `-=` `*=` `/=` `%=` `&=` `\|=` `^=` `<<=` `>>=` | 赋值与复合赋值 |
| **12** | `,` | 逗号（**全场最低**） |

**速记口诀**：后缀一元 > 前缀一元 > 算术 > 移位 > 关系 > 相等 > 位 > 逻辑 > 三元 > 赋值 > 逗号。

#### 实战易错示例

```c
flags |= val << 2;
/* 移位(5) > 复合或赋值(11)
   → flags |= (val << 2)  ✓ */

if (a & MASK != 0)
/* !=(7) 高于 &(8)？ 错！
   相等(7) 高于 位与(8)
   → if (a & (MASK != 0))  ✗ 几乎永远错 */

if ((a & MASK) != 0)   /* 正确 ✓ */
```

```c
*p++        /* 后缀 ++ 绑定 p：*(p++)，不是 (*p)++ */
a + b << 1  /* << 高于 + → a + (b << 1) */
```

### 二、结合性（同优先级如何分组）

#### 1. 左结合（绝大多数二元运算符）

同优先级时 **从左往右** 分组：

```c
a - b - c     →  (a - b) - c
a / b / c     →  (a / b) / c
a >> 1 >> 2   →  (a >> 1) >> 2
a & b ^ c     →  (a & b) ^ c   /* & 与 ^ 不同级：& 先 */
a & b | c     →  (a & b) | c
a && b || c   →  (a && b) || c
```

#### 2. 右结合（少数）

| 运算符 | 示例 | 分组 |
|--------|------|------|
| **一元** `! ~ ++ -- * & + -` | `*++p` | `*(++p)`（一元从右挂） |
| **三元** `?:` | `a ? b : c ? d : e` | `a ? b : (c ? d : e)` |
| **赋值** `=` `op=` | `a = b = 0` | `a = (b = 0)` |

```c
*&a           →  *(&a)
a ? b : c ? d : e  →  a ? b : (c ? d : e)
x = y = z     →  x = (y = z)
```

**注意**：结合性决定 **括号怎么插**，**不决定** 子表达式谁先 **求值**（见第三节）。

### 三、重中之重：优先级 ≠ 求值顺序

| 概念 | 回答的问题 |
|------|------------|
| **优先级 + 结合性** | 运算 **如何加括号分组** |
| **求值顺序** | 子表达式 **按什么先后执行**（含副作用） |

二者 **独立**。优先级只规定 `a + b * c` 是 `a + (b * c)`，**不规定** `a` 与 `b` 谁先算。

#### 1. 有严格求值顺序保证的（标准规定）

| 运算符 | 顺序规则 |
|--------|----------|
| **`&&`** | 左 → 右；左假则右不求值（[2.6](./2.6-关系运算符与逻辑运算符.md)） |
| **`\|\|`** | 左 → 右；左真则右不求值 |
| **`?:`** | 先 `expr1`；再 **仅** `expr2` 或 `expr3` 之一（[2.11](./2.11-条件表达式.md)） |
| **`,`** | 严格左 → 右，值为最右（[2.12](./2.12-逗号运算符.md)） |

#### 2. 无固定求值顺序（C17 前：未规定；乱写 → UB）

- **普通二元运算** `+` `-` `*` `&` `|` 等：左右操作数谁先求值 **编译器自由决定**
- **函数实参列表**：各实参谁先求值 **不保证**

**严禁** 依赖左右先后：

```c
i++ + i++;           /* UB：同一表达式多次改 i（[2.8](./2.8-自增运算符与自减运算符.md)） */
func(a++, b++);      /* 实参顺序未定义；若同一变量更糟 */
a[i++] = i;          /* UB */
printf("%d %d", i++, ++i);  /* UB */
```

**C17 起**：部分求值顺序有所规定，但 **同一表达式多次修改同一对象** 仍为 UB —— **永远不要写**。

#### 3. 对比小结

```c
/* 优先级：分组明确 */
a + b * c   →  a + (b * c)

/* 求值顺序：a 与 b*c 谁先算？—— 无保证（只要不违反 UB） */

/* 逗号：分组且顺序都有保证 */
(x++, y++)  →  先 x++，再 y++，值为 y++ 结果
```

### 五、底层工程建议（寄存器 / 掩码 / 协议）

1. **不要死记优先级表** —— 复杂位运算、判断 **手动加括号**
2. **位与 + 相等** 必须括号：`if ((val & BIT(0)) != 0)`
3. **移位 + 掩码 + 或赋值** 分步写清：

```c
/* 差：易看错 */
reg |= data << 8 & 0xFF00;

/* 好 */
reg |= (data << 8) & 0xFF00u;
```

4. 超长条件 **拆多行** 或提布尔变量
5. 审查 `grep`：`&.*==`、`<<.*&`、条件里的 `=`

```c
#define BIT(n)  (1u << (n))

if ((status & BIT(3)) != 0)
    handle_ready();

uint32_t field = (reg >> FIELD_SHIFT) & FIELD_MASK;
```

### 六、与类型转换的交互

分组确定后，各子表达式仍受 [2.7](./2.7-类型转换.md) **整型提升**、**usual arithmetic conversions** 约束 —— 优先级 **不解决** signed/unsigned 混算坑。

```c
if (u > i - 1)   /* 分组：u > (i - 1)；仍可能有 unsigned 比较陷阱 */
```

## 速记要点

| # | 要点 |
|---|------|
| 1 | 12 级：后缀 > 一元 > 算术 > 移位 > 关系 > 相等 > 位 > 逻辑 > 三元 > 赋值 > 逗号 |
| 2 | 二元多 **左结合**；一元、三元、赋值 **右结合** |
| 3 | 仅 **`&&` `\|\|` `?:` `,`** 有标准规定的求值/短路顺序 |
| 4 | 二元运算、函数实参 **左右谁先算无保证** |
| 5 | **同一表达式多次改同一变量** → UB |
| 6 | 硬件位操作 **优先括号**，不靠记忆 |

## HFT / 内核提示

- PTE / 标志位：`(pte & PTE_PRESENT) && (pte & PTE_RW)` —— 逻辑与优先级低于位与，常需括号
- DPDK `ol_flags`：`(m->ol_flags & RTE_PKT_RX_VLAN_PKT) != 0`
- 热路径避免嵌套三元 + 位掩码一行写完 —— 利可读性与审查
- Code review checklist：括号、UB、unsigned 比较
