# 2.10 赋值运算符与表达式

## 本节讲什么

讲解基础赋值 **`=`** 与全套 **复合赋值** 运算符。核心两点：**赋值本身是有返回值的完整表达式**；**`a op= b` 与 `a = a op b` 不完全等价**（左侧只求值一次）。并梳理优先级、工程易错陷阱，关联 [2.7 类型转换](./2.7-类型转换.md) 隐式规则。

## 要点

### 一、运算符分类

#### 1. 基础赋值 `=`

把 **右侧** 运算结果存入 **左侧可修改左值**：

```c
int x;
x = 42;
buf[i] = c;
*p = val;
```

#### 2. 复合赋值（算术 + 位运算全套）

| 类别 | 运算符 |
|------|--------|
| **算术** | `+=` `-=` `*=` `/=` `%=` |
| **位运算**（内核/寄存器高频） | `&=` `\|=` `^=` `<<=` `>>=` |

**统一形式**：`a OP= b`

**逻辑近似**：`a = a OP b` —— **但关键区别是左侧 `a` 只求值一次**。

```c
/* 复合赋值：arr 的下标表达式只求值一次 */
arr[i++] |= 1;

/* 若手写展开（错误示范） */
arr[i++] = arr[i++] | 1;   /* i++ 执行两次 → 下标错乱 */
```

| 写法 | 左侧求值次数 |
|------|--------------|
| `a += b` | **1 次** |
| `a = a + b` | **1 次**（`a` 是简单变量时） |
| `arr[i++] op= x` | `i++` **1 次** |
| `arr[i++] = arr[i++] op x` | `i++` **2 次** → 危险 |

底层改寄存器、mbuf 标记、位域大量使用：

```c
reg |= BIT(3);
flags &= ~MASK_ERR;
cnt += pkt_len;
```

### 二、赋值表达式的核心特性

#### 1. 赋值有返回值

整个 `a = b` 表达式的值 = **赋值完成后 `a` 的值**（不是 `b` 的「旧」语义上的独立副本概念，而是左值当前内容）。

```c
int x, y;
y = (x = 5) + 1;   /* x=5，表达式 x=5 值为 5，y=6 */
```

#### 2. 链式赋值（右结合）

```c
int a, b, c;
a = b = c = 10;
/* 结合顺序：a = (b = (c = 10))
   ① c = 10，表达式值 10
   ② b = 10
   ③ a = 10 */
```

**可读性**：生产代码链式过长时建议拆开，调试更直观。

### 三、优先级与结合性

| 项 | 规则 |
|----|------|
| **优先级** | 所有赋值运算符 **极低** —— 低于算术、移位、关系、位运算、逻辑；**仅高于逗号** `,`（[2.13](./2.13-运算符优先级与结合性.md)） |
| **结合性** | **右结合** → 支持 `a = b = c` |

```c
int x = 2 + 3;        /* 先 2+3=5，再 x=5 → x = (2+3) */
flags |= val << 2;      /* 先 val<<2，再 |= → flags |= (val << 2) */
a += b * c;             /* 先 b*c，再 += → a += (b * c) */
```

**记忆**：「先算右边表达式里的运算，最后再赋给左边」。

### 四、高频易错坑

#### 坑 1：条件里误写 `=` 而非 `==`

```c
if (x = y)    /* 合法：赋值，再判 x 是否非 0 —— 经典 bug */
if (x == y)   /* 正确：相等比较 */
```

| 现象 | 说明 |
|------|------|
| `if (x = 0)` | `x` 被清零，条件为假 |
| `if (x = func())` | 故意写法：赋值并检测成功；须注释说明 |

**工程规范**：条件中 **避免赋值**；若必须，写 `if ((ptr = get()) != NULL)` 并加括号/注释。

与 [2.6](./2.6-关系运算符与逻辑运算符.md) 的 Yoda 写法 `if (5 == x)` 可防手滑。

#### 坑 2：复合赋值触发隐式类型转换（联动 2.7）

`a += b` 等价于 `a = a + b`，**完整经历 usual arithmetic conversions**：

```c
unsigned u = 10;
int i = -20;
u += i;   /* i 转为 unsigned；结果巨大正数，非 -10 */
```

```c
uint8_t a = 200;
a += 100; /* 提升为 int 运算，再截断赋回 uint8_t → 44 */
```

**规范**：位标志用 **同类型无符号**；混合符号先 **显式 cast** 再复合赋值。

#### 坑 3：把复合赋值当「保证求值一次」的万能药

左侧若是 **宏** 或 **函数返回引用**，`op=` 只求值一次仍可能因宏展开踩坑（[1.4 宏](../ch01-introduction/1.4-符号常量.md)）。复杂左值优先 **临时变量**：

```c
volatile uint32_t *reg = ...;
uint32_t v = *reg;
v |= MASK;
*reg = v;
```

#### 坑 4：浮点复合赋值精度

`f += 0.1` 累积舍入误差；金融/HFT 计数用 **整数 tick** + `int64_t`，少用 `double +=`。

### 五、使用限制

赋值左侧必须是 **可修改左值（modifiable lvalue）**：

| 允许 | 不允许 |
|------|--------|
| 变量 `x = 1` | 常量 `10 = a` |
| 数组元素 `a[i] = 0` | 表达式 `(x+1) = 2` |
| 指针解引用 `*p = v` | 字符串字面量 `"a" = 'b'` |
| 结构体成员 `s.field = 1`（成员可改） | `const` 对象 |

```c
10 = a;        /* 非法 */
(x + 1) += 2;  /* 非法：非左值 */
```

### 六、复合赋值完整示例

```c
/* 算术 */
sum += delta;
ticks *= 2;
rem %= align;

/* 位运算 —— MMIO */
*gpio_reg |=  (1u << PIN_EN);
*gpio_reg &= ~(1u << PIN_RST);

/* 链式与返回值 */
int err;
if ((err = init_hw()) != 0)
    return err;

/* 解析累加 */
offset += (uint32_t)hdr->len;
```

## 速记要点

| # | 要点 |
|---|------|
| 1 | `a op= b` ≈ `a = a op b`，但 **左值只求值一次** |
| 2 | 赋值是表达式，**有返回值**，支持链式 `a=b=c` |
| 3 | 优先级 **很低**；算术/移位/位运算 **先算**，再赋值 |
| 4 | `if (x=y)` 是经典 bug；条件中慎写赋值 |
| 5 | 复合赋值走 **2.7 隐式转换**；unsigned/signed 混用要谨慎 |

## HFT / 底层提示

- 热路径：`flags |= RX_OK` 单条指令级语义清晰，优于读-改-写三行
- 统计累加：`uint64_t bytes += len;` 注意 `len` 类型一致
- MMIO：**读-改-写** 竞态时须锁或写 1 清 0 硬件语义；`reg |= bit` 非原子时要小心
- 审查：`grep` 条件里的 `=` —— `if (` / `while (` 内赋值
