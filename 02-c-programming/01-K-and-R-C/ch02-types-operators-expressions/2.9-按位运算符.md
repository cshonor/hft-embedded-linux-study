# 2.9 按位运算符

## 本节讲什么

讲解仅对 **整数类型** 生效的 6 个按位运算符：`&` `|` `^` `~` `<<` `>>`。这是硬件寄存器、OS 页表、DPDK 报文标记、裸机系统的 **核心工具**；与 [2.7 类型转换](./2.7-类型转换.md) 联动时，整型提升、有符号/无符号差异会带来大量隐蔽坑。

**前置**：[2.7 类型转换](./2.7-类型转换.md)、[2.2 定宽类型](./2.2-数据类型及长度.md)。  
**后续**：ch05 地址与掩码；内核 PTE、DPDK `rte_*` flags；网络字节序 `htonl`。

## 要点

### 一、6 个按位运算符对照

| 运算符 | 名称 | 核心用途 |
|--------|------|----------|
| **`&`** 二元 | 按位与 | 掩码 **提取**、**清除** 特定位；保留指定 bit |
| **`\|`** 二元 | 按位或 | **置 1**（打开指定 bit） |
| **`^`** 二元 | 按位异或 | **翻转** 指定 bit；两数交换（少用） |
| **`~`** 一元 | 按位取反 | 全 bit 0↔1；配合 `&` **清位** |
| **`<<`** 二元 | 左移 | 低位补 0；等价 × \(2^n\) |
| **`>>`** 二元 | 右移 | 逻辑/算术右移；等价 ÷ \(2^n\)（有符号负数有坑） |

**限制**：操作数须为 **整型**（`char`/`short`/`int`/`long`/`stdint` 定宽）；**不能** 用于 `float`/`double`。

```c
uint32_t x = 0x0F;
uint32_t y = x & 0x03;   /* 0x03 */
uint32_t z = x | 0xF0;   /* 0xFF */
uint32_t w = x ^ 0x0F;   /* 0x00 */
uint32_t t = ~0u;        /* 全 1（注意加 u 避免 ~0 为有符号全 1） */
```

### 二、四大标准位操作模板（工程高频）

设 `MASK_ENABLE = 0x01`、`MASK_READY = 0x02`（通常 `#define` 或 `enum` 单 bit）：

#### 1. 置位（目标 bit → 1，其余不变）

```c
flags |= MASK_ENABLE;
```

#### 2. 清位（目标 bit → 0，其余不变）

```c
flags &= ~MASK_ENABLE;
/* ~MASK 后：目标位为 0，其余为 1；& 保留其它位 */
```

#### 3. 检测某一位是否为 1

```c
if (flags & MASK_READY)
    /* 为真：该位已置 1；勿写 flags & MASK == MASK 除非刻意 */
```

#### 4. 提取指定区间 bit（如高 8 位）

```c
uint16_t word = 0x1234;
uint8_t high8 = (word >> 8) & 0xFF;   /* 0x12 */
uint8_t low8  = word & 0xFF;          /* 0x34 */
```

**多 bit 掩码**：

```c
#define FIELD_MASK  0xF00u
#define FIELD_SHIFT 8
uint32_t field = (reg & FIELD_MASK) >> FIELD_SHIFT;
```

### 三、移位规则与未定义行为（UB）

#### 1. 左移 `<<`

- 低位 **补 0**
- 语义上相当于 × \(2^{\text{shift}}\)（无溢出前提下）
- **风险**：移出位宽的高位 **丢弃**，**无警告**
- **UB**（标准禁止，结果不可预测）：
  - 移位量 **≥ 操作数类型宽度**（如 32 位 `x << 32`）
  - 移位量为 **负数**
  - 左移 **有符号负数** 或移入符号位（C99+ 规则细，安全做法：**用无符号**）

```c
uint32_t x = 1u;
x << 31;    /* OK */
x << 32;    /* UB */
x << -1;    /* UB */
```

#### 2. 右移 `>>` —— 逻辑 vs 算术（重点）

| 操作数类型 | 常见行为 | 可移植性 |
|------------|----------|----------|
| **无符号** `unsigned` / `uint32_t` | **逻辑右移**：高位补 **0** | 标准保证 |
| **有符号非负** | 通常逻辑右移 | 较安全 |
| **有符号负数** | **算术右移**：高位补 **符号位**（常见） | **实现定义**（C 标准不强制唯一） |

```c
int i = -1;        /* 补码：全 1 */
i >> 1;            /* 多数平台 → -1（补 1），实现定义 */

unsigned u = 0x80000000u;
u >> 1;            /* 逻辑右移 → 0x40000000，高位补 0 */
```

**工程建议**：寄存器、协议、位域 **一律 `uint32_t` / `uint8_t`**，右移行为确定。

#### 3. 移位与赋值

```c
flags <<= 1;       /* 左移 1，等价 flags *= 2（无符号语义） */
val >>= 3;         /* 右移 3，等价 val /= 8（无符号确定） */
```

### 四、与 2.7 类型转换联动坑点

#### 1. 整型提升 + `~` 截断

`char`/`short`/`uint8_t` 参与 `~`、`&`、`|` 前先 **提升为 `int`**：

```c
uint8_t flags = 0x0F;
flags &= ~0x01;    /* ~0x01 在 int 上为 0xFFFFFFFE，再 & 后赋回 uint8_t 截断 → OK */

uint8_t v = 0x80;
uint8_t r = ~v;    /* ~v 先提升：~0x80 → 0xFFFFFF7F，截断为 uint8_t → 0x7F */
```

**清位模板** 对窄类型仍安全，因 `&` 后只保留低 8 位；但 **`~` 单独赋给窄类型** 要小心预期。

#### 2. 有符号 / 无符号混合位运算

二元 `&` `|` `^` 遵循 **usual arithmetic conversions**（[2.7](./2.7-类型转换.md)）：

```c
int mask = 0xFF;
unsigned flags = 0x100;
flags & mask;      /* mask 可能转为 unsigned，语义依宽度而定 */
```

**规范**：位操作数 **统一无符号定宽**：

```c
uint32_t flags = ...;
flags &= (uint32_t)~MASK;
```

#### 3. 有符号数左移

```c
int x = 1;
x << 31;           /* 可能 UB（移入符号位） */
uint32_t y = 1u;
y << 31;           /* OK → 0x80000000 */
```

#### 4. 比较位标志时用 `&` 结果当布尔

```c
if (flags & MASK)          /* OK：非 0 为真 */
if ((flags & MASK) == MASK) /* 检测「所有 MASK 位都置 1」 */
```

勿与 [2.6](./2.6-关系运算符与逻辑运算符.md) 混淆：`&` 是 **位与**，`&&` 是 **逻辑与**。

### 五、异或与实用技巧

```c
/* 翻转指定位 */
flags ^= MASK_TOGGLE;

/* 交换两整数（无临时变量，面试题，生产少用） */
a ^= b; b ^= a; a ^= b;

/* 判奇偶 */
if (n & 1)   /* 最低位为 1 → 奇 */

/* 对齐：向上取 2 的幂边界（无分支常用技巧，另章展开） */
```

### 六、底层工程场景

| 场景 | 位运算用法 |
|------|------------|
| **MMIO 寄存器** | `reg \|= BIT_EN;` `reg &= ~BIT_RST;` `if (reg & BIT_RDY)` |
| **页表 PTE** | 提取物理页号：`(pte >> 12) & PFN_MASK` |
| **DPDK mbuf** | `mb->ol_flags & RTE_PKT_RX_VLAN_PKT` |
| **协议头** | 取版本/长度：`(byte0 >> 4) & 0x0F` |
| **权限 bitmap** | `perms & WRITE_BIT` |

```c
volatile uint32_t *gpio = (volatile uint32_t *)0x40000000u;
*gpio |= (1u << 5);              /* 置位 pin 5 */
*gpio &= ~(1u << 3);             /* 清位 pin 3 */
if (*gpio & (1u << 0)) { ... }   /* 读 pin 0 */
```

**`volatile`**：防止编译器优化掉对硬件的重复读/写（与 `const` 不同）。详见 [4.7.1](../ch04-functions-and-program-structure/4.7.1-volatile限定符.md)。

## 易混点与高频考点

| # | 要点 |
|---|------|
| 1 | 仅 **整数**；浮点非法 |
| 2 | **置位 `\|=`、清位 `&= ~MASK`、测试 `& MASK`、提取 `>> &`** |
| 3 | 移位 **≥ 宽度** 或 **负数** → **UB** |
| 4 | **无符号右移** 逻辑补 0（标准保证）；**有符号负数右移** 实现定义 |
| 5 | 窄类型先 **整型提升** 再位运算，`~` 后截断易踩坑 |
| 6 | 寄存器/协议 **优先 `uint32_t`**，少用 `int` 做掩码 |
| 7 | `&` ≠ `&&`；`~` 是一元，优先级见 [2.13](./2.13-运算符优先级与结合性.md) |

## HFT / 内核提示

- 订单标志、连接状态：单字 `uint32_t` + 位掩码，比多个 `bool` 省缓存行
- 网络解析：`ntohs`/`ntohl` **之后** 再 `&`/`>>` 取字段
- 禁止对 **有符号** 价格差做 `>>` 除 2 期望等价除法 —— 用 `/ 2` 或显式无符号
- 审查 `<< n`：`n` 来自变量时 **边界检查** `n < sizeof(type)*8`
