# 2.7 类型转换

## 本节讲什么

讲解 C 两种类型转换机制：**隐式自动转换**（整型提升 integral promotion、常规算术转换 usual arithmetic conversions）与 **显式强制转换** `(类型)表达式`。本节是 OS / 驱动 / HFT 最容易出 **隐蔽 bug** 的章节 —— **无运行时报错**，只会逻辑错乱、死循环、内存访问异常。

**前置**：[2.2 类型与长度](./2.2-数据类型及长度.md)、[2.4 声明](./2.4-声明.md)、[2.6 关系/逻辑运算符](./2.6-关系运算符与逻辑运算符.md)（比较时转换坑最多）、[2.2.4 stdint.h](./2.2.4-stdint.h.md)。  
**后续**：ch05 指针与整数互转；[C 陷阱 7.1](../../03-C-Traps-and-Pitfalls/ch07-portability-pitfalls/7.1-整型宽度.md) 可移植性；网络字节序、溢出检测。

## 要点

### 一、隐式转换（编译器自动，无需写代码）

#### 1. 整型提升 Integral Promotion

**规则**：宽度 **小于 `int`** 的整数类型（`char`、`short`、`unsigned char`、`uint8_t`、`uint16_t` 等）在参与大多数运算、比较、函数传参（默认提升）前，先 **提升为 `int`**（若 `int` 装不下 `unsigned int` 的全部值，则提升为 `unsigned int` —— 少见）。

**触发场景**：`+` `-` `*` `/` `%`、比较 `>` `<` `==`、位运算 `& | ^`、`~`、`!`、`?:` 等。

```c
uint8_t a = 200;
uint8_t b = 100;
int res = a - b;   /* a、b 先提升为 int 再减 → res == 100 */
```

若 **不提升**，`uint8_t` 减法会在 8 位模 256 下溢；提升为 `int` 后按有符号整型算，结果符合直觉。

```c
char c = 'A';
int x = c + 1;     /* c 提升为 int，再与 int 运算 */
```

#### 2. 常规算术转换 Usual Arithmetic Conversions

两个 **不同类型** 的操作数做 **二元算术或比较**（`+ - * / %`、`< > <= >= == !=` 等）时，编译器先把两边对齐到 **公共类型**（common type），再运算。

**对齐顺序（简化记忆）**：

| 优先级 | 规则 |
|--------|------|
| 1 | **浮点优先**：任一侧为 `float`/`double`/`long double` → 另一侧也转浮点（`float` 常升为 `double`） |
| 2 | **整数**：按 **宽度 + 符号类别** 对齐到能容纳两者值的类型 |
| 3 | **同符号**：窄 → 宽（`int` → `long` → `long long`） |
| 4 | **一有 `unsigned`、一有 `signed`**：**signed 按 unsigned 的位模式重新解释**（本节最大坑） |

```c
int i = 3;
double d = 2.5;
double r = i + d;    /* i 隐式转为 double */
```

#### 3. 窄 → 宽 vs 宽 → 窄（小转大 vs 大转小）

| 方向 | 典型例子 | 结果 |
|------|----------|------|
| **窄 → 宽（小转大）** | `float` → `double`；`int` → `long`；`short` → `int` | **一般安全**，有效位通常 **保留** |
| **宽 → 窄（大转小）** | `double` → `float`；`long` → `int`；`int` → `short` | **可能精度损失** —— 截断低位有效数字；超出目标范围 → **溢出/未定义** |

```c
float  f = 3.14f;
double d = f;          /* 小转大：OK，f 精确装入 double */

double x = 1.23456789012345;
float  y = (float)x;   /* 大转小：float 有效位少，低位被截断 */

long   L = 10000000000L;
int    n = (int)L;     /* 大转小：若 int 装不下 → 实现定义/溢出 */
```

**`return` 时同样适用**（[4.1](../ch04-functions-and-program-structure/4.1-函数的基本知识.md)、[4.2](./4.2-返回非整型值的函数.md)）：`return` 表达式 **隐式转换** 为函数声明的返回类型 —— 返回类型比表达式 **窄** 时，与赋值窄化规则一致，可能丢精度。

```c
int truncate_pi(void)
{
    return 3.14;       /* double → int：截断为 3 */
}

float narrow(void)
{
    double x = 1.23456789012345;
    return x;          /* double → float：低位有效数字可能丢失 */
}
```

**工程原则**：函数返回类型与 `return` 表达式 **尽量同宽**；必须窄化时 **显式 cast** 并注释意图，或先检查范围。

#### 3.1 浮点数 → `int`：砍掉小数，**不是四舍五入**

**浮点转整型** = 把 **小数部分直接砍掉**，只留 **整数部分**（C 标准：向零截断 truncate toward zero）。

| 浮点值 | 转成 `int` | 说明 |
|--------|-----------|------|
| `3.9f` | **3** | 小数 `.9` 砍掉，**不是** 四舍五入成 4 |
| `-2.7f` | **-2** | 负数的 `.7` 也砍掉，**不是** -3 |
| `3.0f` | **3** | 无小数部分 |

```c
float a = 3.9f;
float b = -2.7f;
int   i = (int)a;      /* 3 */
int   j = (int)b;      /* -2，不是 -3 */

int k = 3.14;          /* 赋值/return 隐式转换：同样截断 → 3 */
```

| 要点 | 说明 |
|------|------|
| **规则** | **向零取整** —— 正数往下砍、负数往上砍（朝 0 方向） |
| **≠ 四舍五入** | 要四舍五入用 `round()`、`lround()`（`<math.h>`）等 |
| **溢出** | 浮点值 **超过 `int` 能表示的范围** → 结果 **不可控**（实现定义 / 未定义行为） |
| **隐式发生** | `int x = 3.9f;`、`return 3.14;`（`int` 函数）、`int` 形参传 `float` 实参 |

```c
/* 危险：超大浮点转 int */
float huge = 1e20f;
int bad = (int)huge;   /* 超出 INT_MAX → 不可控 */

/* 若要四舍五入 */
#include <math.h>
int rounded = (int)roundf(3.9f);   /* 4 */
```

见 [4.2 返回非整型](../ch04-functions-and-program-structure/4.2-返回非整型值的函数.md)；`return 3.14` 给 `int` 函数是同一规则。

#### 4. 有符号 / 无符号混合 —— 致命坑（必考）

当 **`signed` 与 `unsigned` 同级宽度** 比较或运算时，**signed 操作数转为 unsigned**，按 **无符号** 规则解释位模式。

**经典拆解**：

```c
unsigned u = 1;
int i = -1;

if (u < i)   /* 常为真（1）—— 与直觉相反！ */
    ...
```

执行流程：

1. `i == -1` 与 `unsigned` 运算 → `i` 转为 `unsigned`
2. `-1` 的补码全 1 → 解释为 `unsigned` 时约为 `UINT_MAX`
3. `1 < UINT_MAX` → **真**

| 直觉 | 实际（常见 32 位） |
|------|-------------------|
| `-1 < 1` | `(unsigned)-1` 是巨大正数，`1 < 巨大正数` 为真 |

**衍生：无符号死循环**

```c
unsigned int n = 10;
for (; n >= 0; n--)   /* 死循环：unsigned 永远 ≥ 0 */
    ...
```

`n--` 到 `0` 后再减 → 按模 \(2^{32}\) 回绕到 `UINT_MAX`，条件 `n >= 0` 恒真。

**安全写法**：

```c
for (int n = 10; n >= 0; --n) { ... }

/* 或 */
for (unsigned n = 10; n > 0; --n) { ... }
/* 单独处理 n == 0 */
```

**协议字段比较**（wire 全是无符号）：

```c
uint16_t pkt_len = read_u16(buf);
int max = 1500;
if (pkt_len > max)   /* max 转为 uint16_t/unsigned，负的 max 变巨大正数 */
```

→ 比较前 **统一类型**：`(int)pkt_len > max` 或 `(uint16_t)max` 并保证语义正确。

### 二、显式强制转换 `(目标类型)表达式`

#### 1. 本质

**内存中的二进制位通常不变**（合法转换前提下），**只改变编译器对同一段位的解读方式**。

| 效果 | 说明 |
|------|------|
| **截断** | 宽 → 窄：丢弃高位；`uint32_t` → `uint8_t` 只留低 8 位 |
| **符号翻转** | 同宽度有符号 ↔ 无符号：位不变，正负判断变 |
| **整数 ↔ 指针** | 地址与整数互转；**不**改变硬件地址，只改类型系统（ch05、`uintptr_t`） |
| **浮点 ↔ 整数** | 可能舍入、截断小数；超大整数转浮点可能丢精度 |

```c
uint32_t raw = 0xFFFFFFFFu;
int val = (int)raw;        /* 位不变，解读为 -1（补码） */

double x = 3.9;
int n = (int)x;            /* 3：砍小数，非四舍五入 */

float neg = -2.7f;
int m = (int)neg;          /* -2，不是 -3 */

void *p = (void *)(uintptr_t)0xFFFF800012340000ull;  /* 寄存器映射 */
```

**注意**：非法转换（如错误对齐的指针强转）是 **未定义行为 UB**，编译器不报错。

#### 2. 典型使用场景

| 场景 | 示例 |
|------|------|
| 规避隐式 signed/unsigned 陷阱 | `(int)u` 与 `i` 比较 |
| 寄存器 / DMA 地址 | `(volatile uint32_t *)REG_BASE` |
| 指针与整数互转 | `(uintptr_t)p`、`(void *)addr` |
| 浮点转整数 | `(int)price_tick` |
| 消除警告、匹配 API | `(size_t)len` |

**工程原则**：跨符号、跨宽度运算 **主动显式 cast**，不依赖编译器隐式规则。

### 三、底层工程场景（OS / 驱动 / HFT）

| 场景 | 风险 |
|------|------|
| **硬件寄存器** | `uint8_t`/`uint32_t` 与 `int` 混算触发提升与 unsigned 规则 |
| **DMA 物理地址** | `uint64_t` 强转指针；须 `uintptr_t` + 对齐 |
| **网络报文解析** | 字段全 `uint16_t`/`uint32_t`，与 `int` 比较逻辑反转 |
| **HFT 数值** | `float`/`double`/`int64_t` 混用，隐式转 `double` 或精度丢失 |
| **循环下标** | `size_t`（无符号）与 `int`（有符号）比较 → signed 变 unsigned |

#### 工程规范建议

1. **优先 `stdint.h` 定宽类型**（`uint32_t`、`int64_t`），少用裸 `int`/`long` 表示协议与硬件（[2.2](./2.2-数据类型及长度.md)）
2. **禁止** 无符号与有符号 **直接比较**；统一 cast 到同一符号类别
3. **寄存器 / 地址** 必须显式 cast + `volatile`（按需）；用 `uintptr_t`/`intptr_t`，不用 `unsigned long` 冒充（[2.2.4 stdint.h](./2.2.4-stdint.h.md)）
4. 怀疑转换时 **打印十六进制 + 两侧类型**（`PRIu32`、`PRId32`）

```c
/* 寄存器读改写 —— 显式宽度 + volatile */
volatile uint32_t *reg = (volatile uint32_t *)(uintptr_t)0x40001000u;
uint32_t v = *reg;
*reg = (v | 0x1u) & ~0x2u;

/* 安全比较 */
int32_t delta = (int32_t)a - (int32_t)b;   /* 先统一到够宽的 signed */
```

## 四、隐式 vs 显式对照

| | **隐式转换** | **显式强制转换** |
|---|--------------|------------------|
| 写法 | 无，编译器自动 | `(type)expr` |
| 可见性 | 易忽略，隐蔽 bug | 意图明确 |
| 典型场景 | 整型提升、算术对齐 | 寄存器、指针、刻意截断 |
| 底层建议 | **少依赖** | **主动写** |

## 五、易混点与高频考点

| # | 要点 |
|---|------|
| 1 | 短整型参与运算前 **几乎必先整型提升为 `int`** |
| 2 | **signed + unsigned 混合** → signed 转 unsigned，比较可 **逻辑颠倒** |
| 3 | 强制转换 **不改变位模式**（合法转换下），只改 **类型解释** |
| 4 | `unsigned` 循环 `n >= 0` → **死循环** |
| 5 | 浮点与整数运算 → 整数 **升为浮点** |
| 6 | **浮点→int**：向零截断（`3.9f`→3，`-2.7f`→-2），**非四舍五入**；超范围 **溢出** |
| 7 | **窄→宽** 一般安全；**宽→窄**（含 `return`）→ 截断/溢出 |
| 8 | 截断高位、符号变化、指针强转 —— **无运行时检查** |
| 9 | [2.6](./2.6-关系运算符与逻辑运算符.md) 里每个比较都可能先经历本节规则 |

## HFT / 内核提示

- 订单簿价格差：`int64_t` 全程，避免 `int` 与 `uint32_t` 混比
- DPDK mbuf 长度：`uint16_t data_len` 与常量比较用同类型或显式 `(unsigned)` 两侧
- 内核 `size_t` 与 `int` 比较是经典 bug 源；用 `ssize_t` 或 cast 一致
- 位域、协议头字段：读入后 **先 `ntohs`/`ntohl`** 再与本地类型比较（字节序另章）

1. `uint8_t a=200, b=100; int r=a-b;` 为什么 `r` 是 100 而不是无符号下溢？

3. `for (unsigned n=10; n>=0; n--)` 为何会死循环？
