## 一、四个头文件一句话定位

| 头文件 | 提供什么 | **不提供**什么 | 什么时候用 |
|--------|----------|----------------|------------|
| **`<limits.h>`** | 宏：`INT_MAX`、`LONG_MAX`、`CHAR_BIT`… | 类型、函数 | 查 **本机** `int`/`long`/`char` 能表示多大；单平台业务溢出判断 |
| **`<stdint.h>`** | 类型：`int32_t`、`uint64_t`…；宏：`INT32_MAX`、`UINT64_MAX`… | `printf` 格式串 | **协议、寄存器、跨平台 wire format** —— 字段必须定宽 |
| **`<inttypes.h>`** | 宏：`PRId64`、`PRIu32`…；函数：`strtoimax`… | 新整型（复用 stdint） | **打印/扫描** `int32_t`/`int64_t`；字符串转最大宽整数 |
| **`<float.h>`** | 宏：`FLT_EPSILON`、`DBL_MAX`… | 整型任何东西 | **浮点** 极值、精度、epsilon 比较 |

**配套（不在四头内，但天天见）**：

| 头文件 | 常用项 | 什么时候用 |
|--------|--------|------------|
| **`<stddef.h>`** | `size_t`、`ptrdiff_t`、`NULL` | `sizeof` 结果、`malloc` 长度、数组下标；打印用 **`%zu`** |

```text
limits.h   → 「这台机器上的 int 有多大？」
stdint.h   → 「我就要恰好 32 位无符号。」
inttypes.h → 「怎么 printf 这个 int64_t？」
float.h    → 「double 多精确？能表示多大？」
```

## 二、选型决策（按场景）

```
你要存/传什么数据？
│
├─ 网络报文 / 文件格式 / 寄存器镜像 / 跨进程 ABI
│     → #include <stdint.h>
│     → 类型 uint16_t、uint32_t…
│     → 边界 UINT32_MAX、INT64_MIN…
│     → 要打印再加 #include <inttypes.h>，用 PRIu32、PRId64
│
├─ 本机循环、计数、普通业务（不关心几 bit）
│     → 类型 int、unsigned、long long
│     → 溢出检查 #include <limits.h>，用 INT_MAX、LLONG_MAX
│     → 打印 int 用 %d，long 用 %ld，size_t 用 %zu（stddef）
│
├─ 指针与整数互转、DMA 地址
│     → stdint.h 的 uintptr_t / intptr_t
│
├─ 配置字符串 → 大整数
│     → inttypes.h 的 strtoimax / strtoumax
│
└─ 浮点价格、仿真、科学计算
      → 类型 double / float
      → #include <float.h>，用 DBL_EPSILON、DBL_MAX
```

### 场景对照表

| 场景 | 类型从哪来 | 范围宏 | 打印格式 | include |
|------|------------|--------|----------|---------|
| TCP 端口号 | `uint16_t` | `UINT16_MAX` | `%" PRIu16` | stdint + inttypes |
| 纳秒时间戳 | `int64_t` | `INT64_MAX` | `%" PRId64` | stdint + inttypes |
| `for (int i=0; …)` | `int` | `INT_MAX` | `%d` | limits（按需） |
| `malloc` 长度 | `size_t` | — | `%zu` | stddef |
| 寄存器 32 位 | `uint32_t` | `UINT32_MAX` | `%" PRIx32` | stdint + inttypes |
| 指针存进整数 | `uintptr_t` | — | `%#" PRIxPTR` 或 `%p` | stdint |
| 行情价 `double` | `double` | `DBL_MAX` | `%f` / `%g` | float（按需） |
| 浮点相等判断 | `double` | `DBL_EPSILON` | — | float |

## 三、各头「真正常用」清单（背这些就够开工）

### `<limits.h>` — 原生整型边界

| 宏 | 用途 |
|----|------|
| `CHAR_BIT` | 1 字节几位（几乎总是 8） |
| `INT_MAX` / `INT_MIN` | 本机 `int` 上下限 |
| `UINT_MAX` | 本机 `unsigned int` 上限 |
| `LONG_MAX` / `LONG_MIN` | 本机 `long`（**Win64 与 Linux64 不同！**） |
| `LLONG_MAX` | `long long` 上限（跨平台 64 位常用） |
| `UCHAR_MAX` | 无符号字节 255 |

**方法**：无函数，**全是编译期宏**，可用于 `#if`、`static_assert`（C11）。

```c
#include <limits.h>

if (n > INT_MAX / 2) { /* 乘法溢出预判 */ }
```

### `<stdint.h>` — 定宽类型 + 定宽最值

| 类型（最常用） | 用途 |
|----------------|------|
| `uint8_t` | 字节、原始报文 octet |
| `uint16_t` | 端口、短长度 |
| `uint32_t` | IPv4、计数、寄存器 |
| `uint64_t` | 大时间戳、计数 |
| `int32_t` / `int64_t` | 有符号定宽 |
| `uintptr_t` | 指针 ↔ 整数 |

| 宏（配套） | 用途 |
|------------|------|
| `UINT8_MAX` … `UINT64_MAX` | 无符号定宽上限 |
| `INT32_MAX` / `INT32_MIN` | 有符号 32 位界 |

**方法**：无函数，**类型 typedef + 宏**。

```c
#include <stdint.h>

uint32_t seq;
if (seq == UINT32_MAX) { /* 回绕处理 */ }
```

### `<inttypes.h>` — 打印 / 解析定宽整数

| 宏（打印，最常用） | 用于类型 |
|--------------------|----------|
| `PRId32` / `PRIu32` / `PRIx32` | `int32_t` / `uint32_t` 十进制/十六进制 |
| `PRId64` / `PRIu64` / `PRIX64` | `int64_t` / `uint64_t` |
| `PRIuPTR` / `PRIxPTR` | `uintptr_t` |

| 函数 | 用途 |
|------|------|
| `strtoimax` | 字符串 → 最宽有符号整数 |
| `strtoumax` | 字符串 → 最宽无符号整数 |
| `imaxabs` | `intmax_t` 绝对值 |

**写法**：`printf("%" PRId64 "\n", x);` —— **`PRId64` 必须拆在引号外拼接**。

```c
#include <inttypes.h>   /* 已含 stdint.h */

int64_t ts = 1700000000000000000LL;
printf("ts=%" PRId64 "\n", ts);
```

### `<float.h>` — 浮点参数

| 宏 | 用途 |
|----|------|
| `DBL_EPSILON` / `FLT_EPSILON` | 浮点「机器精度」，做近似相等 |
| `DBL_MAX` / `FLT_MAX` | 浮点最大值，防溢出 |
| `DBL_MANT_DIG` | 尾数有效位数（理解精度上限） |

**方法**：无函数，**全是宏**。

```c
#include <float.h>
#include <math.h>

int double_eq(double a, double b) {
    return fabs(a - b) < DBL_EPSILON;
}
```

## 四、完整工作流示例（协议字段一生）

以「报文里一个 `uint32_t` 长度字段」为例，演示 **类型 → 边界 → 打印 → 解析** 各头如何配合：

```c
#include <stdint.h>
#include <inttypes.h>
#include <stdio.h>
#include <string.h>

/* ① 类型：stdint.h */
typedef struct {
    uint16_t port;
    uint32_t len;
} hdr_t;

/* ② 边界：stdint.h 宏 */
int validate_len(uint32_t len) {
    if (len == 0 || len > UINT32_MAX / 4)  /* 示例上界 */
        return -1;
    return 0;
}

/* ③ 打印：inttypes.h */
void log_hdr(const hdr_t *h) {
    printf("port=%" PRIu16 " len=%" PRIu32 "\n", h->port, h->len);
}

/* ④ 从配置字符串解析大数：inttypes.h 函数 */
void parse_cfg(const char *s) {
    char *end;
    uint64_t v = strtoumax(s, &end, 10);
    (void)end;
    printf("cfg=%" PRIu64 "\n", v);
}
```

**本机业务变量**（非 wire format）可只用 `int` + `limits.h`：

```c
#include <limits.h>
#include <stdio.h>

int main(void) {
    int n = 100;
    if (n < INT_MAX - 1)
        n++;
    printf("%d (INT_MAX=%d)\n", n, INT_MAX);
    return 0;
}
```

## 五、必背易混点

| 误区 | 正确 |
|------|------|
| `INT_MAX` == `INT32_MAX` | **概念不同**：前者随本机 `int` 宽度；后者恒为 32 位上限 |
| 协议字段用 `char` | 用 **`uint8_t`**（`char` 有无符号实现定义） |
| `printf("%lld", int64_t)` | 用 **`%" PRId64`** |
| `sizeof` 用 `%d` | 用 **`%zu`**（`size_t`，`stddef.h`） |
| 查整型范围去 `float.h` | 整型 → limits/stdint；浮点 → float.h |

## 六、底层 / HFT 默认套餐

| 层级 | 默认 include / 类型 |
|------|---------------------|
| **DPDK / 内核 / 驱动 / 协议栈** | `stdint.h` 定宽 + 日志时 `inttypes.h` |
| **用户态工具、单平台** | `int` + 按需 `limits.h` |
| **行情浮点** | `double` + `float.h` epsilon；计数仍用 `int64_t` tick |
| **热路径** | 少 `printf`；类型仍用 `uint32_t` 等保证布局 |
