# 2.2.5 inttypes.h

## 本节讲什么

**C99** 标准头文件，**必须配合 `<stdint.h>` 使用**（`#include <inttypes.h>` 会 **自动包含** `stdint.h`）。两大作用：

1. 为 **`int8_t` / `uint32_t` / `int64_t`** 等定宽整数提供 **`printf` / `scanf` 格式化宏**（`PRId32`、`PRIu64`…）
2. 提供 **`intmax_t` / `uintmax_t`** 及 **`imaxabs`、`strtoimax`** 等跨平台大整数辅助

定宽类型见 [2.2.4 stdint.h](./2.2.4-stdint.h.md)；四头总览：[2.2.1 选型总览](./2.2.1-整型头文件选型总览.md)、[2.2.2 速查](./2.2.2-limits-stdint-float速查.md)。

## 一、为什么不能直接用 `%d` / `%ld`？

| 类型 | 问题 |
|------|------|
| **`int32_t`** | 多数平台 = `int`，`%d` 常能用 —— 但 **标准不保证** 与 `int` 同一 typedef |
| **`int64_t`** | 有的平台是 **`long`**，有的是 **`long long`** |

硬写 `%lld` 或 `%ld` → 部分平台 **警告**、**输出错乱**。

**`inttypes.h` 宏** 在预处理期展开为 **本平台正确** 的转换说明符，**统一源码写法**。

```c
printf("%" PRId64 "\n", (int64_t)x);   /* 跨平台 */
/* 不要写死 printf("%lld\n", x);       除非确定平台 */
```

与 [1.1.2 printf](../ch01-introduction/1.1.2-printf格式与安全.md) 衔接。

## 二、`printf` 输出宏（`PRI` 前缀）

### 有符号十进制 `PRIdN`

| 宏 | 类型 |
|----|------|
| `PRId8` | `int8_t` |
| `PRId16` | `int16_t` |
| `PRId32` | `int32_t` |
| `PRId64` | `int64_t` |

### 无符号十进制 `PRIuN`

`PRIu8` / `PRIu16` / `PRIu32` / `PRIu64`

### 十六进制 `PRIxN` / `PRIXN`

| 宏 | 含义 |
|----|------|
| `PRIx32` | 小写十六进制 `uint32_t` |
| `PRIX64` | **大写**十六进制 `uint64_t` |

### 标准写法（引号拼接）

宏必须拆进格式串：**`"%" PRId32`**，不能写成单个字符串里手写死 `lld`。

```c
#include <stdio.h>
#include <stdint.h>
#include <inttypes.h>

int main(void)
{
    int32_t  a = 12345;
    uint64_t b = 0xdeadbeefULL;

    printf("int32 = %" PRId32 "\n", a);
    printf("uint64 hex = 0x%" PRIX64 "\n", b);
    return 0;
}
```

```bash
gcc -std=c99 -Wall -o pri_demo pri_demo.c && ./pri_demo
```

## 三、`scanf` 输入宏（`SCN` 前缀，较少用）

| 宏 | 用途 |
|----|------|
| `SCNd32` | 读 `int32_t` |
| `SCNu64` | 读 `uint64_t` |
| `SCNx16` | 十六进制读 `uint16_t` |

```c
int64_t val;
if (scanf("%" SCNd64, &val) != 1) { /* 错误处理 */ }
```

生产代码更常 **`strtoimax` / 手动解析协议**，少用 `scanf`。

## 四、最值宏（与 `stdint.h` 重复）

`INT8_MAX`、`UINT64_MAX` 等 **定义在 `<stdint.h>`**；`#include <inttypes.h>` 后 **可直接用** 定宽类型与最值宏，**只写一行** 即可：

```c
#include <inttypes.h>   /* 已间接包含 stdint.h */
```

## 五、辅助函数（C99）

```c
intmax_t  imaxabs(intmax_t j);
intmax_t  strtoimax(const char *nptr, char **endptr, int base);
uintmax_t strtoumax(const char *nptr, char **endptr, int base);
```

| 函数 | 作用 |
|------|------|
| **`imaxabs`** | `intmax_t` 绝对值 |
| **`strtoimax`** | 字符串 → **最大宽有符号** 整数（类似 `strtol`，宽度随平台） |
| **`strtoumax`** | 字符串 → **最大宽无符号** 整数 |

协议解析、配置字符串转大数、工具链常用。`intmax_t` / `uintmax_t` 在 `<stdint.h>` 定义。

> 注意标准函数名是 **`strtoimax` / `strtoumax`**，不是 `imaxstrto`。

打印 `intmax_t` 可用 **`PRIdMAX`** / **`PRIuMAX`**（C99 宏）。

## 六、使用约束与注意事项

| 项 | 说明 |
|----|------|
| **标准** | **C99+**；C89 **无** 此头；`gcc -std=c99` 或更高 |
| **MSVC** | 旧版支持残缺；工业底层多用 **GCC/Clang**（WSL/Linux） |
| **包含顺序** | 随意；`inttypes.h` 内部 `#include <stdint.h>` |
| **普通 `int`/`long`** | **不需要** 本头；用 [limits.h](./2.2.3-limits.h.md) + `%d`/`%ld` |
| **DPDK / 内核 / 报文** | 定宽字段打印 **必用** `PRI*` 宏 |

## 七、四组头文件区分（必背）

| 头文件 | 职责 |
|--------|------|
| **`<stdio.h>`** | `printf`、`fopen`、`stdin`/`stdout` — [B.1](../appendix-b-standard-library/B.1-输入与输出stdio.h.md) |
| **`<limits.h>`** | 原生 `int`/`long` **范围宏** `INT_MAX` — [详解](./2.2.3-limits.h.md) |
| **`<stdint.h>`** | **定宽类型** `int32_t`、`uint8_t` + `INT32_MAX` — [详解](./2.2.4-stdint.h.md) |
| **`<inttypes.h>`** | stdint **配套** `PRId64`/`SCNd64`、`strtoimax` |

```text
协议字段 uint32_t  → stdint.h 定义类型
打印该字段         → inttypes.h 的 PRIu32
判断是否溢出       → stdint.h 的 UINT32_MAX
```

## HFT / DPDK 实践

```c
uint64_t pkt_ts;
printf("[RX] ts=%" PRIu64 " len=%" PRIu16 "\n", pkt_ts, len);
```

热路径仍应 **减少 printf**；日志框架内部统一封装 `PRI*` 宏。
