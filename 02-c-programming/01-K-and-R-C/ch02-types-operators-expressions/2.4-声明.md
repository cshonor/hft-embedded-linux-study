# 2.4 声明

## 本节讲什么

讲解 C 语言 **变量基础声明语法**：`类型 标识符;` / 带初始化；**局部** 与 **全局** 声明的存储位置、生命周期直观差异。本章只讲 **表层规则** —— `static` / `extern` 完整语义、单定义规则、作用域细节留 [ch04](../ch04-functions-and-program-structure/) 与 [1.10 外部变量](../ch01-introduction/1.10-外部变量与作用域.md)。

**前置**：[2.1 变量名](./2.1-变量名.md)、[2.2 类型](./2.2-数据类型及长度.md)。  
**后续**：2.5 起表达式运算；ch04 存储类；[Expert C ch03](../../04-Expert-C-Programming/ch03-analyzing-c-declarations/) 复杂声明。

## 要点

### 1. 基础声明两种格式

**① 仅声明（无初值）**

```c
int a;
double val;
```

**② 声明并初始化**

```c
int num = 10;
float pi = 3.14f;
```

| 形式 | 说明 |
|------|------|
| `类型 标识符;` | 引入名字与类型；局部未初始化 → **垃圾值**；全局/静态 → **0** |
| `类型 标识符 = 常量/表达式;` | 定义 + 赋初值 |

### 2. 一行多变量（逗号分隔）

同类型可 **共用** 前置类型关键字：

```c
int x, y, z;              /* x、y、z 全是 int */
int a = 1, b, c = 5;      /* 混用初始化 */
```

**⚠️ 指针坑**（与 [1.8.1](../ch01-introduction/1.8.1-指针形参与取地址.md) 同源）：

```c
int *p, q;    /* 只有 p 是指针；q 是普通 int */
int *p, *q;   /* 两个都是指针 */
```

逗号 **只分隔变量名**，**不会** 让每个名字都带上 `*`。

### 3. 类型声明 **围绕标识符**（指针 / 数组读法）

C 的声明不是简单「从左到右念类型再念名字」，而是 **以标识符为中心**，`**` `[]` `()` 等 **绑在名字上**，表示「对名字做什么操作得到什么类型」。

**读法**：先找到 **标识符**，问「对它解引用 / 取下标 / 调用，得到什么？」

#### `void *p` —— `p` 是指针，解引用后是 `void`

```c
void *p;
/* 读：*p 是 void → p 是 void 指针（通用指针） */
```

| 步骤 | 含义 |
|------|------|
| 标识符 | `p` |
| 紧邻 `*` | 对 `p` **解引用** → 得到 **`void`** |
| 结论 | `p` 是 **`void *`** |

与 [4.1 `void` vs `void *`](../ch04-functions-and-program-structure/4.1-函数的基本知识.md) 呼应：左边写 `void *` 作返回类型，同样是「返回的那个东西，解引用一层是 void」。

#### `int *a[10]` —— `a` 是数组，元素是 `int` 指针

```c
int *a[10];
/* 读：a[10] 是 int* → a 是含 10 个元素的数组，每个元素是 int 指针 */
```

| 步骤 | 含义 |
|------|------|
| 标识符 | `a` |
| 紧邻 `[10]` | `a[10]` → **数组**，共 10 个元素 |
| 再往前 `int *` | 每个元素类型是 **`int *`** |
| 结论 | **`a` 是指针数组**（不是「指向数组的指针」） |

对比（ch05 [5.12](../ch05-pointers-and-arrays/5.12-复杂声明.md)）：

```c
int *a[10];     /* a：数组，元素 int*  —— [] 先绑 a */
int (*b)[10];   /* b：指针，指向 int[10] —— () 让 * 先绑 b */
```

**为何这么设计**：简单声明 `int x` 与复杂声明 `int *p`、`int a[10]` **同一套规则** —— 都是「修饰符围绕标识符」，避免再发明另一套数组/指针语法。

> 完整螺旋法则、`int (*signal(...))(...)` 见 [5.12](../ch05-pointers-and-arrays/5.12-复杂声明.md)、[Expert C ch03](../../04-Expert-C-Programming/ch03-analyzing-c-declarations/)。

### 4. 局部声明（函数 / 代码块 `{}` 内）

| 项 | 说明 |
|----|------|
| **位置** | `main`、自定义函数、`if`/`for` 的 `{}` 内 |
| **存储类别** | 默认 **自动存储 `auto`**（关键字可省略）→ **栈** |
| **生命周期** | 进入块创建，离开 `}` **销毁** |
| **初值** | 未初始化 → **indeterminate**（垃圾值） |

**C89 vs C99 声明位置**：

| 标准 | 规则 |
|------|------|
| **C89** | 块内所有声明须在 **第一条语句之前** |
| **C99** | **用到再声明**；`for (int i = 0; …)` 合法（[2.1.1 C99](./2.1.1-C99标准与演进.md)） |

```c
void f(void)
{
    int i = 0;
    for (int j = 0; j < 10; ++j)   /* C99：j 仅循环内 */
        i += j;
}
```

### 5. 全局声明（所有函数 **外部**）

| 项 | 说明 |
|----|------|
| **位置** | 文件顶层，任何函数定义之外 |
| **存储** | **静态存储区**（非 `static` 时外部链接，ch04） |
| **生命周期** | 程序启动 → 进程结束 |
| **初值** | 未显式初始化 → **清零**（0、`'\0'`、`NULL`） |
| **作用域** | 本文件从定义行到文件末；他文件 `extern` 引用（ch04 / [1.10](../ch01-introduction/1.10-外部变量与作用域.md)） |

```c
int g_count;           /* 全局，默认 0 */
int max_len = 1024;    /* 全局 + 初值 */
```

### 6. 基础限定符：`const`（本节仅提及）

```c
const int max_len = 1024;   /* 只读：不能 max_len = 2048; */
```

- **`const`** 限制 **修改**，不改变变量仍是 **局部栈** 或 **全局静态** 哪一类
- **文件顶层 `const`**：通常进 **常量区**；C 里 **默认内部链接** —— 他文件不能像普通全局那样 `extern` 引用（见 [2.3 §0.1](./2.3-常量.md)）
- **`volatile`**：变量可能被硬件/异步逻辑修改；**每次使用须从内存读**，编译器不得用寄存器旧值优化（见 [4.7.1](../ch04-functions-and-program-structure/4.7.1-volatile限定符.md)）
- `static` / `extern` / **`register`** —— **ch04** 展开；K&R 本章不深挖

## 二、声明 vs 定义（浅讲，深度后置）

| | **声明（declaration）** | **定义（definition）** |
|---|-------------------------|-------------------------|
| **作用** | 告诉编译器：名字存在、什么类型 | **分配存储**（或函数体） |
| **内存** | 一般不单独分配（`extern`） | **分配** |
| **典型** | `extern int a;` | `int a;` / `int a = 10;` |

```c
extern int errno;     /* 声明：引用别处定义（libc） */
int buf[256];       /* 定义：分配 256 个 int */
```

**本节只建立概念**；**Define once; declare many times**、`extern` 与多文件、ODR 见 [1.10](../ch01-introduction/1.10-外部变量与作用域.md)、ch04、[Expert C ch03](../../04-Expert-C-Programming/ch03-analyzing-c-declarations/)。

> **函数** 同理：原型是声明；带 `{ }` 函数体是定义（[1.7](../ch01-introduction/1.7-函数.md)）。

## 三、局部 vs 全局对照

| 对比 | **局部 `auto`** | **全局（文件作用域）** |
|------|-----------------|------------------------|
| 位置 | `{}` 内 | 函数外 |
| 内存 | 栈 | 静态数据区 |
| 未初始化 | 垃圾值 | **0** |
| 生命周期 | 块内 | 全程 |
| 典型风险 | 未初始化、栈溢出大数组 | 全局耦合（[1.10](../ch01-introduction/1.10-外部变量与作用域.md)） |

## 四、易混点

1. **全局** 不手动初始化 → **默认 0**；**局部 auto** 不初始化 → **垃圾值**
2. **`const`** 只读，不改变全局/局部存储类别
3. **`int *p, q`** 只有 `p` 是指针
4. **`void *p`**：`*p` 是 `void` → `p` 是 void 指针；**`int *a[10]`**：`a` 是数组，元素是 `int*`
5. 不要把 **声明** 与 **#define 宏** 混淆（[1.4](../ch01-introduction/1.4-符号常量.md)）— 宏不分配内存
6. 大数组放 **局部栈** 易溢出 → 全局 / `static` / `malloc`（ch04 / ch08）

## HFT / 底层提示

- 热路径局部变量：**用到再声明**（C99），缩小作用域，利优化
- 协议上下文：倾向 **`struct` + 栈上局部** 或预分配池，少裸全局
- 定宽字段声明：`uint32_t seq;` 而非裸 `int`（[2.2](./2.2-数据类型及长度.md)）
