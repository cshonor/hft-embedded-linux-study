# 2.1 变量名

## 本节讲什么

讲解 C 语言 **标识符（identifier）** 的词法基础 —— **变量名、函数名、宏名、结构体名、typedef 名** 等自定义名称的合法格式、**关键字** 限制、**大小写** 特性，以及工程命名惯例。属于 **词法最底层** 规则：后续所有声明、定义都建立在合法标识符之上。

**前置依赖**：无（ch02 开篇）。  
**后续**： [2.2 类型与长度](./2.2-数据类型及长度.md)、[2.4 声明](./2.4-声明.md)、ch04 存储类、GNU/DPDK/内核命名拓展。

## 要点

### 1. 标识符合法组成规则

| 规则 | 说明 |
|------|------|
| **允许字符** | 英文字母 `a`–`z` / `A`–`Z`、数字 `0`–`9`、下划线 `_` |
| **首字符** | **不能** 以数字开头 |
| **禁止** | `#`、`-`、空格、中文标点等 **非字母数字下划线** 符号 |

```c
int data;        /* 合法 */
int buf_len;     /* 合法 */
int Val100;      /* 合法 */

int 1num;        /* 非法：数字开头 */
int user-name;   /* 非法：含 '-' */
int #temp;       /* 非法 */
```

标识符长度：C99 起 **至少 63 字节** 有效字符须被编译器区分（更早实现可能更短）；工程上 **勿写超长名**。

### 2. 大小写敏感

C **严格区分大小写** —— 大小写不同即 **不同标识符**：

```c
int sum, Sum, SUM;   /* 三个互不相关的变量 */
```

| 惯例（工程） | 示例 |
|--------------|------|
| 普通变量、函数 | **小写 + 下划线**：`buf_len`、`get_line` |
| 宏常量 | **全大写 + 下划线**：`MAXLINE`、`RTE_PKTMBUF_HEADROOM` |
| 结构体 / typedef 类型名 | **首字母大写或统一前缀**：`OrderBook`、`rte_mbuf`（库风格） |

### 3. 关键字（保留字）：单独禁止 vs 嵌入允许

分 **两层** 理解：**完整关键字** 不能当标识符；**嵌入** 在更长名字里 **语法允许**（工程上仍不推荐）。

#### 3.1 单独完整关键字 —— 绝对不能当标识符

`int`、`if`、`return` 等 **整词** 做变量名 → **编译报错**：

```c
int int;      /* 非法 */
int if;       /* 非法 */
int return;   /* 非法 */
```

编译器对关键字做 **完整 token 匹配**，不是子串搜索。

#### 3.2 关键字嵌入标识符 —— 语法合法

只要 **不是单独一个关键字**，作为名字 **一部分** 即可：

```c
int int_val;      /* 内含 int，允许 */
int if_flag;      /* 内含 if，允许 */
int return_buf;   /* 内含 return，允许 */
int num_if;       /* 关键字在末尾，允许 */
```

#### 3.3 下划线包裹关键字 —— 仍合法

```c
int _if;
int if_;
int __int__;
```

语法 **通过**，但 **可读性差**，内核/DPDK 规范一般避免。

#### 3.4 C++ 关键字同理

```c
int class_buf;      /* C/C++ 均可：嵌入允许 */
int namespace_id;   /* 嵌入允许 */
int class;          /* C++ 非法：完整关键字 */
```

#### 3.5 工程建议（语法允许 ≠ 推荐）

| 做法 | 语法 | 工程 |
|------|------|------|
| `int int;` | **禁止** | — |
| `int int_val;` | 允许 | **不推荐** — 易误读、混编歧义 |
| `int max_count;` | 允许 | **推荐** — 见名知意、无关键字子串 |

- **完整清单**：见 [附录 A](../appendix-a-reference-manual/)；**C99 新增 5 个**见 [2.1.1 C99 标准与演进](./2.1.1-C99标准与演进.md)
- 宏名 **禁止** 撞完整关键字或标准库名（[1.4 §5](../ch01-introduction/1.4-符号常量.md)）

**总结**：完整独立关键字 **禁止**；作为子串嵌入 **语法允许，不推荐**。

### 4. 可读性命名 + 底层工程风格

**通用原则**

1. **见名知意**：`max_line_len` 优于 `m`；单字母仅循环 **`i`/`j`/`k`**、数学下标等短作用域
2. **避免** 与标准库 / 常见缩写撞名：`open`、`read`、`index`（POSIX 函数名）
3. **宏全大写 + 前缀**：`HFT_RING_SIZE`，避免 `#define getchar` 劫持标准库（[1.5](../ch01-introduction/1.5-character-io/1.5-字符输入输出.md)）

**内核 / DPDK / HFT 常见风格**

| 前缀/模式 | 含义 | 示例 |
|-----------|------|------|
| **`rte_`** | DPDK 公共 API 前缀 | `rte_mbuf`、`rte_eth_rx_burst` |
| **`__` 开头** | 实现保留 / 内核私有（**业务代码慎用**） | `__kernel_*`；与用户空间 `__` 扩展规则见 GNU 文档 |
| **`static` 文件内** | 本 `.c` 私有符号，不污染全局命名空间（ch04） | `static int dbg_count` |

双下划线、`_` 开头保留规则在 **GNU 扩展 / 可移植性** 章节展开；初学：**自定义标识符用字母/下划线开头，别用 `__` 开头**。

## C 与 C++ 标识符区别

| 主题 | C | C++ |
|------|---|-----|
| 词法规则 | 字母数字下划线、数字非首 | 同左 |
| 关键字数量 | 较少（C11 约 44 个） | **更多**：`class`、`namespace`、`template`、`public`、`private`、`new`、`delete`… |
| `.c` 纯 C 文件 | 避开 **C 关键字** 即可 | — |
| **混编 / 共用头文件** | 变量名须 **同时避开 C 与 C++ 关键字**，防 `.cpp` 编译失败 | 同左 |

```c
/* 在 .h 被 C++ include 时，下面在 C++ 里会炸 */
int class;       /* C 可能过，C++ 不行 */
int template;    /* 同理 */
```

共用头文件用：

```c
#ifdef __cplusplus
extern "C" {
#endif
/* C API 声明 */
#ifdef __cplusplus
}
#endif
```

## HFT / 底层开发的用处

1. **协议字段名** 与 **宏名** 分离：字段 `price_ticks`，上限 `MAX_TICKS`
2. **DPDK 代码** 识别 `rte_` 即库 API，勿自建 `rte_foo` 假官方符号
3. **Code review**：关键字拼写错误、大小写混用（`Int` vs `int`）一眼抓
4. **链接符号**：函数名即链接名，命名冲突见 ch04 / [C 陷阱 ch04](../../03-C-Traps-and-Pitfalls/ch04-linking/)

## 本节高频易错点

1. **`1var`** 数字开头 —— 非法
2. **`sum` 与 `Sum`** 当同一个变量 —— 实为两个
3. 用 **`int`、`return`** 整词当变量名 —— 非法；**`int_val`** 嵌入 —— 合法但不推荐
4. **`#define int long`** 类极端宏 —— 全文件语法崩坏（[C 陷阱 6.5](../../03-C-Traps-and-Pitfalls/ch06-preprocessor/6.5-宏名冲突.md)）
5. C++ 混编头文件里用 **`class`、`new`** 作字段名
