# 2.2 数据类型及长度

## 本节讲什么

内置类型的 **存储宽度** 与 **有符号/无符号** 语义——理解内存、寄存器、协议字段、ABI 对齐的基础。

**核心结论**：

1. **标准 C 只规定最小取值范围与相对大小**，**不固定**各类型字节数
2. 实际占用字节、数值上下限，由 **CPU 架构（32/64 位 x86、ARM）+ 编译器 + OS ABI** 决定

与 [1.2 变量与算术表达式](../ch01-introduction/1.2-变量与算术表达式.md) 的 `int`/`float` 衔接；与 [C 陷阱 ch07 7.1](../../03-C-Traps-and-Pitfalls/ch07-portability-pitfalls/7.1-整型宽度.md) 可移植性专题对照。

**四个标准头怎么选、怎么用** → [2.2.1 整型头文件选型总览](./2.2.1-整型头文件选型总览.md)（`limits.h` / `stdint.h` / `inttypes.h` / `float.h`）。

## 要点

### 1. 基础类型与 CPU 的关联（含存储用途）

| 类型 | 典型字节 | 受 CPU 影响 | 说明 | 存储用途 |
|------|----------|-------------|------|----------|
| **`char`** | **1** | 极小 | 最小可寻址单元；有符号 `-128~127` / 无符号 `0~255`（**实现定义** signed/unsigned） | ① 单个 ASCII 字符（字母、符号）<br>② 小块二进制缓冲、字节流<br>③ 极小范围数值标记（0~255）<br>④ 字符串底层（`char[]` / `char*` 存文本） |
| **`short`** | **≥2**，通常 **2** | 很小 | 标准：`sizeof(short) ≥ sizeof(char)` | ① 范围不大的整数，省内存<br>② 音频采样、屏幕坐标、硬件寄存器短数值<br>③ 嵌入式、大数组小整数场景 |
| **`int`** | **≥2**，现代几乎 **4** | **较大** | 32 位 CPU 普遍 4 字节；64 位 x86/ARM **多数仍 4**；老 16 位机曾见 2 字节 | ① **日常通用整数**：循环、ID、数量、偏移<br>② 普通运算，CPU **原生 int 效率最高**<br>③ 文件大小、下标、函数返回状态码 |
| **`long`** | **≥4** | **很大** | 32 位：**4**；Win64 LLP64：**4**；Linux/macOS LP64：**8** | ① 32 位平台略大范围整数<br>② Linux/macOS 64 位：文件偏移、PID、时间戳<br>⚠️ **跨平台勿当固定 64 位** |
| **`long long`** | **≥8**，通常 **8** | 小 | C99 起；32/64 位平台普遍 **8** 字节 | ① 超大整数：大文件长度、高精度计数、金额<br>② **跨平台固定 64 位**，少踩 `long` 差异<br>③ 加密、哈希、64 位寄存器数据 |
| **`float`** | **4** | 现代 x86/ARM 较统一 | IEEE 754 **单精度** | ① 精度要求一般的小数：游戏坐标、图形、传感器实时数据<br>② 对内存敏感的浮点数组 |
| **`double`** | **8** | 现代 x86/ARM 较统一 | IEEE 754 **双精度** | ① 高精度科学计算、金融、物理仿真<br>② **C 标准库 `<math.h>` 函数默认 `double`**<br>③ 需减少舍入误差的场景 |
| **`void *`** | **4 或 8** | **= CPU 位宽** | 所有对象指针 **大小一致**；**不可直接解引用** | ① 通用指针：`malloc` 返回值、泛型容器<br>② 存任意类型地址再 **cast** 回去<br>③ 判断 32/64 位；内核/驱动通用地址 |

标准只保证：`char ≤ short ≤ int ≤ long ≤ long long`，且 **`sizeof(char) == 1`**。

#### 选型速记

| 需求 | 优先选型 |
|------|----------|
| 日常整数 | **`int`** — CPU 优化最好，无特殊需求首选 |
| **`char` 不只是字符** | 本质 1 字节数；二进制 I/O 靠 `char` / `uint8_t` |
| 跨平台 64 位大数 | **`long long`** 或 **`int64_t`** — 避开 `long` 的 Win/Linux 差异 |
| 浮点 | 省内存 → **`float`**；要精度、少误差 → **`double`** |
| 协议 / wire format | **`stdint.h`** 定宽类型（见 §5） |

#### 指针类型配套用途

与 [1.8.1 指针形参](../ch01-introduction/1.8.1-指针形参与取地址.md) 衔接 —— 指针 **存的都是地址**，**`*` 左侧类型** 决定解引用读宽与语义：

| 指针类型 | 典型用途 |
|----------|----------|
| **`char *`** | 字符串首地址、命令行参数；原始字节流（文件、网络包） |
| **`int *`** | 整型数组批量读写；传参改外部 `int`（交换、多结果） |
| **`float *` / `double *`** | 浮点数组、矩阵、图像像素批量处理 |
| **`long long *`** | 超大数值数组；64 位硬件数据交互 |
| **`void *`** | 类型未知时暂存地址；`malloc`；回调泛型参数 |

### 2. 有符号 / 无符号范围由字节长度推导

字节越多，可表示范围越大（有符号补码 / 无符号模 \(2^n\)）：

| 宽度 | 有符号 `int` 量级 | 无符号量级 |
|------|-------------------|------------|
| 4 字节 | 约 ±21 亿 | 约 42 亿 |
| 8 字节 | 约 ±9×10¹⁸ | 约 1.8×10¹⁹ |

CPU 寄存器宽度影响编译器对「自然整型」的选择，这是 **`int`/`long` 随平台变化** 的根源。

### 3. `sizeof` 与 `<limits.h>`

```c
#include <stdio.h>

int main(void)
{
    printf("sizeof char=%zu short=%zu int=%zu long=%zu long long=%zu void*=%zu\n",
           sizeof(char), sizeof(short), sizeof(int),
           sizeof(long), sizeof(long long), sizeof(void *));
    return 0;
}
```

- **`sizeof(T)`**：编译期常量，类型 **字节数**
- **`<limits.h>`**：`INT_MAX`、`LONG_MAX` 等 **实现定义** 上下限（详见 [2.2.1 选型总览](./2.2.1-整型头文件选型总览.md)、[2.2.2 速查](./2.2.2-limits-stdint-float速查.md)）
- 打印 `sizeof` 用 **`%zu`**（`size_t`），不要用 `%d`

### 4. 直观例子：同一 `long`，不同平台

```c
printf("long size: %zu\n", sizeof(long));
```

| 环境 | 典型输出 |
|------|----------|
| 32 位 Linux | **4** |
| Windows 64 位（LLP64） | **4** |
| 64 位 Ubuntu / WSL（LP64） | **8** |

同一份源码，**`long` 能表示的范围完全不同**——跨平台用裸 `long` 存时间戳、大计数易溢出或截断。

### 5. 固定宽度整数：`stdint.h`（嵌入式 / HFT 必用）

为消除架构差异，C99 提供 **定长类型**：

```c
#include <stdint.h>

uint8_t   u8;
int16_t   i16;
uint32_t  u32;
int64_t   i64;
```

| 类型 | 保证 |
|------|------|
| `int32_t` / `uint32_t` | 恰好 32 位 |
| `int64_t` / `uint64_t` | 恰好 64 位 |

不管 32 还是 64 位 CPU，**wire format、寄存器镜像、跨进程协议** 应优先用这套；`size_t` / `ptrdiff_t` 用于大小与指针差。配套选型与 `PRIu64` 等见 **[2.2.1 整型头文件选型总览](./2.2.1-整型头文件选型总览.md)**、[2.2.4 stdint.h](./2.2.4-stdint.h.md)。

### 6. HFT / 嵌入式实用总结

1. 普通 **`int` / `long` 有平台差异**，底层高性能代码勿假设固定宽度
2. **一律用 `<stdint.h>` 定长类型**，屏蔽 CPU/ABI 差异
3. **`float`/`double`** 在现代 x86/ARM64 上通常 4/8 字节，但仍勿用 `==` 比浮点
4. **指针宽度 = CPU 位宽**；指针与整数互转用 **`uintptr_t`**，不用 `int`（见 C 陷阱 7.5）
5. 字节数据用 **`unsigned char` / `uint8_t`**，不裸用可能 signed 的 `char`

## 与 C++ 的差异

| 主题 | C++ | C |
|------|-----|---|
| 定宽类型 | `<cstdint>` 同 `stdint.h` | `<stdint.h>` |
| `char` 默认符号 | 实现定义 | 实现定义 |
| `long` 宽度 | 同 C，随平台变 | 同左 |
