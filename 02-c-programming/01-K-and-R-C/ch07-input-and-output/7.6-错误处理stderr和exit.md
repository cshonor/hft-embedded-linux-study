# 7.6 错误处理 stderr 和 exit

## 本节讲什么

**`stderr`** 标准错误流与 **`stdout`** 分离；**`exit`** 终止进程；**`ferror`/`clearerr`** 管理流错误。日志分级与调试输出的基础。

## 要点

### stderr

```c
fprintf(stderr, "fatal: cannot open %s\n", path);
```

- 通常**无缓冲**或行缓冲，错误信息及时可见
- 可重定向：`prog 2> err.log`（stdout 仍走 1）
- 正常结果走 stdout，诊断走 stderr —— `grep`、管道不误吞错误

### exit

```c
#include <stdlib.h>
exit(0);    /* 成功 */
exit(1);    /* 失败，返回给 shell */
```

- 调用 **`atexit`** 注册的函数，刷新 stdio 缓冲
- 与 **`return`** from `main` 等价于 `exit(返回值)`

### 流错误

- 读写出错后 **`ferror(fp)`** 为真
- **`clearerr(fp)`** 清除错误与 EOF 标志以便重试
- 不要仅用 **`getchar() == EOF`** 判断错误类型

## 底层场景

| 场景 | 做法 |
|------|------|
| UEFI 固件调试 | 错误码 + 串口/屏幕 Print |
| HFT | 分级日志：trade → 文件，error → stderr |
| 工具链 | 编译器 diagnostic 走 stderr |
