# 7.1 标准输入输出

## 本节讲什么

最底层的字符 I/O：**`getchar()`** / **`putchar()`**，以及 **`EOF`** 结束标记。第 1 章已用 `getchar` 做文件复制；本章系统展开流模型。

## 要点

- **`getchar()`**：从 **stdin** 读一个 `unsigned char` 转为 `int` 返回；无更多数据时返回 **`EOF`**
- **`putchar(c)`**：向 **stdout** 写一个字符
- **字符流抽象**：终端、管道、文件在 C 标准 I/O 层都是**连续字节序列**，区别在是否重定向、`FILE*` 绑定
- **`EOF`**：通常为 `-1`，表示流结束或错误（需结合 `feof`/`ferror` 区分，见 7.5、7.6）
- `getchar`/`putchar` 往往经 **stdio 缓冲**；`fflush(stdout)` 在交互提示后常用

## 示例

```c
#include <stdio.h>

int c;
while ((c = getchar()) != EOF)
    putchar(c);
```

重定向：`prog < in.txt > out.txt` —— 程序不变，stdin/stdout 换绑文件。

## 底层映射

| 场景 | 对应 |
|------|------|
| 串口字符流 | 逐字节 read/write，概念同 getchar |
| UEFI ConOut | 类 putchar 输出到 GOP/串口 |
| HFT 日志 | 热路径少用 stdio，但调试仍靠字符流 |
