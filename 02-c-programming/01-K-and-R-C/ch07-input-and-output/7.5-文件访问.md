# 7.5 文件访问

## 本节讲什么

**`FILE *` 流**：`fopen` / `fclose`、字符 I/O、格式化 I/O、**二进制块读写**、**文件定位**。本章 IO 核心；承接第 6 章 struct 直接读写复合数据。

## 要点

### 标准流

```c
FILE *stdin;   /* 标准输入 */
FILE *stdout;  /* 标准输出 */
FILE *stderr;  /* 标准错误，通常无缓冲 */
```

### fopen 模式

| 模式 | 含义 |
|------|------|
| `r` | 只读 |
| `w` | 写，清空或新建 |
| `a` | 追加 |
| `r+` | 读写，不截断 |
| `w+` | 读写，清空或新建 |
| `a+` | 读+追加 |

- **二进制**：加 **`b`** —— `rb` `wb` `ab` 等，避免 Windows 文本模式下 `\r\n` 转换
- 失败返回 **`NULL`**，须检查

### 关闭与状态

```c
fclose(fp);           /* 刷新缓冲、释放资源 */
feof(fp);             /* 是否因 EOF 停止 */
ferror(fp);           /* 是否出错 */
clearerr(fp);         /* 清除错误与 EOF 标志 */
```

### 字符 I/O

- **`fgetc(fp)`** / **`fputc(c, fp)`** —— 文件版 getchar/putchar
- **`ungetc(c, fp)`** —— 推回一个字符（7.8.3）

### 二进制块读写（用户 7.7）

```c
fwrite(&hdr, sizeof hdr, 1, fp);
fread(buf, size, 1, fp);
```

- 按**字节块**读写，不受换行、编码影响
- 直接读写 **struct** 时注意：**padding、字节序、ABI**（第 6 章）—— 跨设备/跨平台易错

### 文件定位（用户 7.8）

```c
long pos = ftell(fp);
fseek(fp, offset, SEEK_SET);  /* SEEK_CUR, SEEK_END */
rewind(fp);                   /* 等价 fseek(fp, 0L, SEEK_SET) */
```

- 随机访问 ELF 头、镜像分片、历史 tick 文件某偏移
- 大文件用 **`fseeko`/`ftello`**（POSIX，`off_t`）

## 难点

| 坑 | 说明 |
|----|------|
| 文本 vs 二进制 | Windows 文本模式 `\n`↔`\r\n` |
| struct 整块 fread | 对齐/endian 不可移植 |
| EOF vs 错误 | `feof` 仅在读失败后判断；先看返回值 |

## 与后续衔接
