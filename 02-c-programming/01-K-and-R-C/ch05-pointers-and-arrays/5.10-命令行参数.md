# 5.10 命令行参数

## 本节讲什么

**`main(int argc, char *argv[])`**：`argc` 为参数个数，`argv` 为 **指向各参数字符串的指针数组**（退化后等价 **`char **argv`**）。HFT 工具、UEFI 应用、OS loader 用命令行控制启动行为。本节是 [5.6](./5.6-指针数组以及指向指针的指针.md)、[5.8](./5.8-指针数组的初始化.md) 指针数组的 **直接工程应用**。

**前置**：[5.5](./5.5-字符指针与函数.md)、[5.6](./5.6-指针数组以及指向指针的指针.md)、[5.8](./5.8-指针数组的初始化.md)、[5.9](./5.9-指针与多维数组.md)（勿与二维 `char[][N]` 混淆）。

## 一、核心原型与类型本质

```c
int main(int argc, char *argv[]);
```

| 参数 | 类型本质 | 含义 |
|------|----------|------|
| **`argc`** | `int` | 命令行参数 **总个数**（含程序名） |
| **`argv`** | `char *argv[]` → **`char **`** | **指针数组**，每元素 `char *` 指向一个参数字符串 |

```
argv 内存模型（教学用）

argv ──→ [0] ──→ "./hft\0"
         [1] ──→ "-cfg\0"
         [2] ──→ "config.json\0"
         ...
         [argc] = NULL   ← 哨兵
```

### 固定规则（POSIX / C 实现惯例）

| 规则 | 说明 |
|------|------|
| **`argv[0]`** | 程序名，常带路径（`./trader`、`/usr/bin/app`） |
| **`argv[1]` … `argv[argc-1]`** | 用户传入参数 |
| **`argv[argc]`** | **必须为 `NULL`**（哨兵，便于无 `argc` 遍历） |
| 原始类型 | **全是字符串**；数字/开关须 **手动转换** |

### 示例

命令：`./hft -cfg config.json -speed 1000`

| 项 | 值 |
|----|-----|
| `argc` | **5** |
| `argv[0]` | `"./hft"` |
| `argv[1]` | `"-cfg"` |
| `argv[2]` | `"config.json"` |
| `argv[3]` | `"-speed"` |
| `argv[4]` | `"1000"` |
| `argv[5]` | `NULL` |

## 二、两种标准遍历写法

### 方式 1：依靠 `argc`（基础常用）

```c
#include <stdio.h>

int main(int argc, char *argv[])
{
    for (int i = 1; i < argc; i++)
        printf("arg%d: %s\n", i, argv[i]);
    return 0;
}
```

从 **`i = 1`** 开始，跳过 `argv[0]` 程序名。

### 方式 2：依靠末尾 `NULL` 哨兵（纯指针遍历）

```c
int main(int argc, char *argv[])
{
    char **p = argv + 1;    /* 跳过程序名 */
    while (*p != NULL) {
        printf("%s ", *p);
        p++;
    }
    return 0;
}
```

| 方式 | 特点 |
|------|------|
| `argc` 循环 | 直观，不依赖 `argv[argc]==NULL` 也能工作 |
| `NULL` 哨兵 | 纯 **二级指针** 遍历，与 [5.5 `swap`](./5.5-字符指针与函数.md) 同族 |

## 三、数值转换

命令行输入 **全是字符串**，数字参数须转换：

| 函数 | 特点 | 场景 |
|------|------|------|
| **`atoi(s)`** | 转 `int`；失败返回 0，**无法区分错误** | 教学/简单工具 |
| **`strtol(s, &end, base)`** | 转 `long`；可检 `end`、设进制；设 `errno` | **内核 / HFT / 工业首选** |

```c
#include <stdlib.h>
#include <errno.h>

char *end;
errno = 0;
long speed = strtol(argv[4], &end, 10);
if (errno != 0 || *end != '\0' || end == argv[4]) {
    /* 转换失败：非数字、溢出、空串 */
}
```

## 四、扩展：环境变量 `environ`

非 K&R 核心，但 Linux / 服务器工具必备（POSIX）：

```c
extern char **environ;
```

| 项 | 说明 |
|----|------|
| 结构 | 与 `argv` 相同：**`char **` 指针数组，末尾 **`NULL`** |
| 格式 | `KEY=VALUE`，如 `PATH=/bin:/usr/bin` |

```c
char **env = environ;
while (*env != NULL) {
    printf("%s\n", *env);
    env++;
}
```

## 五、底层工程场景

### HFT 高频交易

| 参数 | 用途 |
|------|------|
| `-cfg xxx.json` | 策略配置路径 |
| `-dry-run` | 模拟交易，不真实下单 |
| `-cpu 3` | 绑定 CPU 核，降调度延迟 |
| `-rate 5000` | 报单速率（`strtol` 转换） |

### UEFI Shell / 固件

- `load fs0:\kernel.efi` — 镜像路径
- `-dbg 2` — 日志等级

### OS Bootloader

内核启动参数经 **argv 式** 传递：内存大小、根文件系统、`quiet` 等驱动开关。

## 六、易错考点

| 坑 | 说明 |
|----|------|
| 修改 `argv[i]` 指向的串 | 多在 **只读区**，`argv[1][0]='x'` → 段错误 |
| 忽略 `argv[argc]==NULL` | 方式 2 遍历依赖哨兵 |
| **`argc` 含程序名** | 用户参数从 **`argv[1]`** 起 |
| `char *argv[]` vs `char arr[][N]` | 前者指针数组 / `char**`；后者二维数组 / `char(*)[N]`，**不能混接**（[5.8](./5.8-指针数组的初始化.md)） |
| 裸 `atoi` 无校验 | 非法输入当 0，HFT 配置静默错 |

| 方向 | 内容 |

## 背诵要点（精简版）

```
main(argc, argv[])：argv 是 char** 指针数组
argv[0]=程序名；argv[1..argc-1]=用户参数；argv[argc]=NULL
遍历：for(i=1;i<argc) 或 char**p=argv+1; while(*p)
全是字符串；数字用 strtol 带校验
environ 同 argv 结构；exec 可换整套 argv
```
