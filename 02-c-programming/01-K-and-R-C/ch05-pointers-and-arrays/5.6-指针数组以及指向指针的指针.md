# 5.6 指针数组以及指向指针的指针

## 本节讲什么

全书 **最易混淆考点**：**指针数组** vs **数组指针** vs **二级指针 `**`**。解题万能规则：**`[]`、`()` 优先于 `*`**，先看名字和谁绑定。衔接 [5.5 `swap(char *v[])`](./5.5-字符指针与函数.md)、[5.1 `&p` → `int **`](./5.1-指针与地址.md)、[5.10 `argv`](./5.10-命令行参数.md)。

**前置**：[5.2](./5.2-指针与函数参数.md)、[5.3](./5.3-指针与数组.md)、[5.5](./5.5-字符指针与函数.md)。

## 一、核心区分原则：运算符优先级

```
优先级：[]、()  >  *
```

| 绑定结果 | 含义 | 典型写法 |
|----------|------|----------|
| 名 + `[]` 先结合 | **数组**；`*` 修饰元素 → **指针数组** | `char *argv[]` |
| `(*名)` 先结合 | **指针**；指向整数组 → **数组指针** | `int (*p)[10]` |
| `**` | 指向 `T*` 的指针 → **二级指针** | `char **pp` |

## 二、指针数组 `char *argv[]`

### 1. 定义与解析

```c
char *month[] = {"Jan", "Feb", "Mar"};
/* 等价 char *(month[]); */
```

| 项 | 说明 |
|----|------|
| `month` | **数组**，元素类型 **`char *`** |
| 内存 | 数组里存 **一串地址**，字符串正文在只读区（[5.5](./5.5-字符指针与函数.md)） |
| `month[i]` | 第 `i` 个字符串指针 |
| `month[i][0]` | 第 `i` 个串的首字符（二次解引用） |
| `sizeof(month)` | 元素个数 × `sizeof(char *)`（64 位：3×8=24） |

### 2. 经典场景：`main` 命令行参数

```c
int main(int argc, char *argv[])
```

- 形参 `char *argv[]` 退化为 **`char **argv`**（传首元素地址）；
- `argv[0]`：程序名；
- `argv[1]` … `argv[argc-1]`：参数；
- 标准约定：`argv[argc] == NULL` 作结束标记（实现定义，常见）。

详见 [5.10](./5.10-命令行参数.md)。

### 3. 关键特性

- [5.5 `swap`](./5.5-字符指针与函数.md)：只交换数组内 **指针值**，不拷贝字符串；
- 环境变量 **`char **environ`**：同类结构。

## 三、数组指针 `int (*p)[10]`

### 1. 定义解析

```c
int (*p)[10];
```

- `()` 使 `*p` 先结合 → **`p` 是指针**；
- 指向 **一整个 `int[10]`**，类型 **`int (*)[10]`**（[5.3 §一](./5.3-指针与数组.md) `&arr`）。

### 2. 指针算术（极易踩坑）

```c
int arr[10];
int (*p)[10] = &arr;

p + 1;        /* 跳过整数组：10 × sizeof(int) 字节 */
(*p)[0] = 1;  /* 先解引用得 int[10]，再下标 */
```

### 3. 典型用途：二维数组传参

```c
void mat(int (*row)[5], int rows)
{
    for (int i = 0; i < rows; i++)
        for (int j = 0; j < 5; j++)
            printf("%d ", row[i][j]);
}

int m[3][5];
mat(m, 3);    /* m 退化 → &m[0]，类型 int (*)[5] */
```

`m` 作为二维数组名：首「行」地址，类型为 **指向含 5 个 int 的数组的指针**。

### 指针数组 vs 数组指针 速查表

| 写法 | 本质 | 存什么 / 指什么 | `name+1` 步长 |
|------|------|----------------|---------------|
| `char *arr[]` | **指针数组** | 数组里多个 `char *` | 1 个指针（通常 8 字节） |
| `int (*p)[10]` | **数组指针** | 单个指针 → 整个 `int[10]` | 10 个 `int` 总字节 |
| `char *name[5]` | 指针数组 | 5 个字符串指针 | `sizeof(char*)` |
| `char (*name)[5]` | 数组指针 | 指向 `char[5]` 行 | `5 × sizeof(char)` |

## 四、二级指针 `char **pp`

### 1. 层级拆解

```c
char **pp;
```

| 表达式 | 类型 | 含义 |
|--------|------|------|
| `pp` | `char **` | 存 `char *` 的地址 |
| `*pp` | `char *` | 一级字符串指针 |
| `**pp` | `char` | 单个字符 |

### 2. 两大来源

**① 指针数组传参退化**

```c
int main(int argc, char *argv[]);   /* 形参等价 char **argv */
```

**② 对一级指针取地址**

```c
char *s = "test";
char **pp = &s;    /* [5.1 §二] */
```

### 3. 工程核心用途

**修改外部一级指针**（[5.2](./5.2-指针与函数参数.md)）：

```c
void alloc_buf(char **out)
{
    *out = malloc(64);
}

int main(void)
{
    char *p = NULL;
    alloc_buf(&p);   /* 必须 char **，不能 char * */
}
```

| 场景 | 用法 |
|------|------|
| 输出参数 | `alloc_buf(&p)` |
| 动态字符串表 | `char **table` |
| 环境变量 | `extern char **environ` |

> 为什么一级 `char *` 形参改不了外面？C 值传递，形参是 **指针副本**；要改 **指针变量本身** 须 `char **`。

## 五、识别口诀与典型声明

**括号/中括号优先贴变量名**：

```c
char *argv[];      /* argv[] 数组，元素 char*     → 指针数组 */
int (*p)[10];      /* (*p) 指针，指向 int[10]      → 数组指针 */
int *f(void);      /* f() 函数，返回 int*          → 返回指针的函数 */
int (*fp)(int);    /* (*fp) 指针，指向函数         → 函数指针 [5.11] */
char **pp;         /* 二级指针 */
```

读法模板：

```
char *argv[]  →  argv 是数组，元素是 char *
int (*p)[10]  →  p 是指针，指向 int[10]
```

## 六、底层工程场景

| 场景 | 结构 |
|------|------|
| **`argv` / `environ`** | 指针数组 + `char **` |
| **函数跳转表** | `void (*func_table[])()` — 函数指针数组（[5.11](./5.11-指向函数的指针.md)） |
| **矩阵/图像行** | `int (*row)[W]` 二维数组传参 |
| **内核页表/描述符** | 多层指针间接寻址，语义同二级指针跳转 |

## 七、章节衔接 & Bug 预警

| 方向 | 内容 |
|------|------|
| 前置 | 5.2 传参、5.3 `&arr`、5.5 字符串 `swap` |
| 后置 | 5.7/5.9 多维数组、5.11 函数指针、7.8.5 `malloc` |

| Bug | 后果 |
|-----|------|
| 指针数组当数组指针传参 | 类型不匹配，编译报错或步长错 |
| `**` 漏写一层 | 拿到地址而非字符/数据 |
| 数组指针 `p+1` 当 `int*` 遍历 | 步长过大，越界错位 |
| `alloc_buf(p)` 漏 `&` | 无法改外部指针 |

## 背诵要点（精简版）

```
[] () 优先于 *；名贴[]是数组，(*名)是指针
char *a[]  指针数组，+1 跳一个指针
int (*p)[N] 数组指针，+1 跳 N 个元素
char **  改外部 char* 用 &p；argv 即 char**
一级改 *p 所指；二级 *pp 改外部指针
```
