# 5.1 指针与地址

## 本节讲什么

**指针**是存放 **内存地址** 的变量。`&` 取地址，`*` 解引用访问该地址上的对象。全书最难章节的起点 —— OS、UEFI、HFT、飞控底层代码都建立在此之上。

**前置**：[4.3.1 程序内存布局](../ch04-functions-and-program-structure/4.3.1-程序内存布局.md)、[4.9 初始化](../ch04-functions-and-program-structure/4.9-初始化.md)（未初始化局部 = 垃圾值）、[1.8 参数传值调用](../ch01-introduction/1.8-参数传值调用.md)。

## 一、`int *p;` 到底在说什么？

**不是** 定义一个叫 `*p` 的变量，而是声明：**`p` 的类型是「指向 `int` 的指针」**。

```c
int *p;    /* p 是 int*，不是「*p 这个变量名」 */
```

C 的语法设计里，声明中的 `*` 和表达式中的 `*` **配套**：

| 位置 | `*` 的含义 |
|------|-----------|
| **声明** `int *p;` | `p` **是指针**；`*p` 的类型是 `int` |
| **表达式** `*p = 10;` | **解引用**：访问 `p` 里存的地址上的那个 `int` |

```
声明：  int *p;     →  p 存地址，*p 是 int
使用：  *p = 10;    →  通过 p 找到的 int 被赋值为 10
```

> 读法：**「`p` 是指向 int 的指针」**，不要读成「星 p」是一个变量名。这与 [1.6 §7](../ch01-introduction/1.6-数组.md)「`[]` 贴变量名」同一套声明语法逻辑。

## 二、`&x` 与 `*p`：地址与对象的配对

```c
int x = 10;
int *p = &x;    /* &x：取 x 在内存中的地址，赋给 p */

*p = 20;        /* 解引用：访问 p 所存地址上的 int → x 变为 20 */
```

| 运算符 | 读法 | 结果类型 |
|--------|------|----------|
| **`&x`** | 取 `x` 的地址 | `int *`（指向 x 所在类型的指针） |
| **`*p`** | 取 `p` 所指的对象 | `int`（p 指向的类型） |

```
内存示意（教学用）

  地址 0x1000          地址 0x2000
  ┌─────────┐          ┌─────────┐
  │ x = 20  │  ←────── │ p=0x1000│
  └─────────┘   *p     └─────────┘
       ↑                    ↑
    &x 得到这里          p 存这个地址
```

### `p` 里存什么？类型标注告诉编译器什么？

以 `int x = 10; int *p = &x;` 为例：

1. **`p` 自己也是变量**，在内存里占一块（64 位通常 8 字节），里面存的是 **地址数值** —— 即 `x` 所在位置（上图中 `0x1000`）；
2. **类型写成 `int *`**，不是装饰，而是契约：告诉编译器 **之后用 `*p` 访问时，必须按 `int` 的宽度（通常 4 字节）去读取和解析** 那块地址上的数据；
3. 因此 `*p` 得到的是 **`int` 语义的对象**，不会把同一块内存当成 `char`、`double` 等别的类型乱读。

```
p 这个变量          x 那块内存
┌──────────────┐    ┌────────────────┐
│ 0x00001000   │───→│ 10 (4 字节 int) │
│  (地址数值)   │    └────────────────┘
└──────────────┘
  sizeof(p)=8          *p 按 int 读 4 字节 → 得到 10
```

| 若写成 | `*p` 解引用时编译器行为 |
|--------|------------------------|
| `int *p` | 读 **4 字节**，按 **整型** 解释 |
| `double *p` | 读 **8 字节**，按 **浮点** 解释 |
| `char *p` | 读 **1 字节**，按 **字符** 解释 |

**地址数值本身无类型**（就是一串比特/十六进制数）；**指针类型** 决定解引用时「从该地址起读多长、按什么规则解释」。这就是 §三「宽度相同、语义不同」的根源。

**变量 = 内存里的一块区域 + 类型**；指针 = 存「那块区域从哪开始」的地址值。理解这一点，才能继续读栈/堆布局、页表映射、用户指针 vs 内核指针（ch08）。

### `&p`：取指针自己的地址 → 二级指针 `int **`

`p` **本身也是变量**，当然也能对它取地址：

```c
int x = 10;
int *p = &x;
int **pp = &p;    /* &p：p 自己在内存里的地址；类型是 int ** */
```

| 表达式 | 类型 | 含义 |
|--------|------|------|
| `x` | `int` | 整型对象 |
| `&x` | `int *` | `x` 的地址 → 赋给 `p` |
| `p` | `int *` | 存 `x` 地址的指针变量 |
| `&p` | `int **` | **`p` 这个指针变量自己的地址** → 赋给 `pp` |
| `*p` | `int` | 沿 `p` 找到 `x` |
| `*pp` | `int *` | 沿 `pp` 找到 `p`（即「指针的值」） |
| `**pp` | `int` | 两次解引用 → 最终到达 `x` |

```
三级对象关系（教学用）

  地址 0x1000     地址 0x2000     地址 0x3000
  ┌────────┐      ┌──────────┐     ┌──────────┐
  │ x = 10 │ ←────│ p=0x1000 │ ←───│pp=0x2000 │
  └────────┘  *p  └──────────┘ *pp └──────────┘
                  ↑ &x              ↑ &p
```

**`int **`** = 「指向 `int *` 的指针」，即 **二级指针**。声明里 **两颗 `*`** 表示「多一层指向」；使用时 **一层 `*` 剥一层**：

```c
int x = 10;
int *p = &x;
int **ptr_to_p = &p;   /* 变量名随意，不必叫 pp */

*ptr_to_p;             /* 类型 int *，得到 p（里面存的是 x 的地址） */
**ptr_to_p;            /* 类型 int，得到 x 本身 */
```

| 写法 | 读法 |
|------|------|
| `int **pp;` | `pp` 是二级指针，指向一个 `int *` |
| `pp = &p;` | 把 **一级指针 `p` 的地址** 存进 `pp` |
| `*pp` | 剥一层 → 拿到 `p`（`int *`） |
| `**pp` | 再剥一层 → 拿到 `x`（`int`） |

**变量名不必叠词**：`pp` 只是教材习惯（p 的 pointer），**不是语法要求**。工程里更常见语义化命名：

```c
int **p_to_int;        /* 指向 int* 的指针 */
int **out;             /* 输出参数：把新 malloc 地址写回 *out */
int **row_table;       /* 动态行表 */
```

编译器只看 **类型**（`int **`），不看名字；叫 `pp`、`p2`、`handle` 都可以。

完整用法见 [5.6 指向指针的指针](./5.6-指针数组以及指向指针的指针.md)。

**HFT / 工程里何时需要 `int **`？**

| 场景 | 原因 |
|------|------|
| **在函数里修改调用方的指针** | C 传参是值传递，`p` 副本改不了外面的 `p`；传 `int **pp` 才能改「指针变量本身」（[5.2](./5.2-指针与函数参数.md)） |
| **动态指针数组** | `int **rows` 管理「每行长度不一」或运行时确定行数的二维数据 |
| **`malloc` 返回指针的指针** | 分配器把新块地址写回 `*out` |

```c
void alloc_int(int **out, size_t n)
{
    *out = malloc(n * sizeof **out);   /* 改调用方的指针变量 */
}

int *buf = NULL;
alloc_int(&buf, 100);   /* 传 &buf，类型 int ** */
```

> 口诀：`&` 对谁用，就得到「指向谁类型的指针」—— `&x` → `int *`，`&p` → `int **`。

### 三个最常见疑问（口述澄清）

**① 指针声明和赋值必须同时进行吗？**

**不必须**，但 **解引用前必须先赋好合法地址**。

```c
int *p;          /* ✅ 合法：只声明，此时 p 是垃圾值（局部变量） */
int x = 10;
p = &x;          /* ✅ 合法：稍后赋值 */
*p = 20;         /* ✅ 现在才能安全解引用 */

int *q = &x;     /* ✅ 更推荐：声明时一并初始化 */
int *r = NULL;   /* ✅ 或先标空，用时再赋 */
```

| 写法 | 是否合法 | 注意 |
|------|----------|------|
| `int *p;` 然后 `p = &x;` | ✅ | 赋值前 **不能** `*p` |
| `int *p = &x;` | ✅ | 推荐，一步到位 |
| `int *p;` 然后 `*p = 10;` | ❌ | 未赋值就解引用 → UB |

**规则**：不是「声明赋值必须写一行」，而是「**`*p` 之前，`p` 里必须是合法地址**」。

**② 赋值是不是必须用 `&` 给一个内存地址？**

**本质是给一个合法的地址值**；`&变量` 是最常见的来源，但不是唯一写法。

| 赋值来源 | 示例 | 说明 |
|----------|------|------|
| **取地址 `&`** | `p = &x;` | 指向栈上变量 |
| **堆分配** | `p = malloc(sizeof *p);` | `malloc` 返回的已是地址 |
| **数组名** | `p = arr;` | 数组名退化为首元素地址（[5.3](./5.3-指针与数组.md)） |
| **空指针** | `p = NULL;` | 表示暂不指向对象 |
| **已有指针** | `p = q;` | 复制另一个指针里的地址值 |

```c
int x;
int *p = &x;              /* & 取 x 的地址 */
int *q = malloc(sizeof *q); /* malloc 直接给地址，不需要再 & */
```

**③ 能不能对指针 `p` 自己取地址 `&p`？**

**完全可以**。`p` 也是变量，`&p` 拿到的是 **`p` 自己在内存里的地址**；`int **pp = &p;` 就是 **指向指针的二级指针**，也是后面多级指针用法的基础（[5.6](./5.6-指针数组以及指向指针的指针.md)）。

```c
int x = 10;
int *p = &x;          /* p 是局部变量时，p 自己在栈上 */
int **pp = &p;        /* &p：p 这块栈内存的地址 */

printf("%p\n", (void *)p);    /* p 里存的：x 的地址 */
printf("%p\n", (void *)&p);   /* p 自己的地址（局部 p → 栈上） */
printf("%p\n", (void *)pp);   /* 与 &p 相同 */
```

> `p` 若是 **全局 / static**，`p` 自己在 **静态区**，`&p` 仍是合法地址，只是不在栈上—— **`&` 对谁都能用**，取决于被取地址对象落在哪一区（[4.3.1](../ch04-functions-and-program-structure/4.3.1-程序内存布局.md)）。

## 三、类型约束：宽度相同，语义不同

### 64 位：指针本身都是 8 字节

| 指针类型 | `sizeof`（x86-64 典型） |
|----------|-------------------------|
| `int *` | 8 |
| `double *` | 8 |
| `char *` | 8 |
| `void *` | 8 |

**宽度相同** ≠ **可以混用解引用**。编译器用类型信息决定：

- 解引用时 **读几个字节**（`int` 通常 4，`double` 通常 8）
- 是否满足 **对齐** 要求
- 指针算术 `p+1` 前进多少字节（[5.4](./5.4-地址算术运算.md)）

```c
double d = 3.14;
int *p = (int *)&d;   /* 强转能过编译，但语义错了 */
*p = 0;               /* ❌ 只写 4 字节进 8 字节的 double → 数据损坏 */
```

| 情况 | 后果 |
|------|------|
| 编译器 **类型检查拦住** | 直接报错，相对安全 |
| **强转** 绕过检查 | 对齐错误、部分写入、读错宽度 → **UB** |
| **HFT / 内核** | 表现为 **偶发** 错价、脏数据、极难复现的 crash |

> 量化/HFT 里「同一地址 `int*` 和 `double*` 来回 cast」是经典 **隐蔽 bug** 来源：回测正常、实盘偶发。原则：**指针类型必须与被指对象的真实类型一致**。

## 四、未初始化指针：绝对禁止解引用

普通局部指针 **不会自动清零**（[4.9](./4.9-初始化.md)），和 `int a;` 一样可能是 **垃圾地址**。

```c
void bad(void)
{
    int *p;       /* 垃圾值：不知道指向哪 */
    *p = 10;      /* ❌ 完全未定义；常见结果：段错误 Segmentation fault */
}
```

**必须先让 `p` 指向一块合法、生命周期够用的内存**，再解引用：

| 来源 | 写法 | 存储区 |
|------|------|--------|
| **栈上变量** | `int x; p = &x;` | 栈（[4.3.1](../ch04-functions-and-program-structure/4.3.1-程序内存布局.md)） |
| **堆分配** | `p = malloc(sizeof *p);` | 堆（[7.8.5](../ch07-input-and-output/7.8-other-functions/7.8.5-存储管理函数.md)） |
| **全局/static** | `static int buf[100]; p = buf;` | 静态区 |
| **字符串字面量** | `char *s = "hi";` | 常量区（只读） |

```c
int x;
int *p = &x;      /* ✅ 合法：p 指向栈上的 x */
*p = 10;

int *q = malloc(sizeof(int));
if (q != NULL) {
    *q = 10;      /* ✅ 合法：p 指向堆块 */
    free(q);
}
```

**野指针**：未初始化、已 `free` 仍使用、悬垂引用（指向已销毁栈帧）—— 解引用均属 **UB**，表现从「静默写坏内存」到 **立即 segfault** 都有可能。

## 五、空指针：`NULL` / `nullptr`

**空指针** = 不指向任何有效对象；解引用 `NULL` 通常 **立即崩溃**（比野指针稍「好查」）。

```c
int *p = NULL;        /* C 传统：空指针常量 */
if (p != NULL)
    *p = 10;

/* C++ / 部分现代 C 风格 */
int *q = nullptr;     /* C23 引入；C++ 推荐 */
```

| 写法 | 场景 |
|------|------|
| **`NULL`** | 纯 C 工程、`#include <stddef.h>`，通常是 `(void *)0` |
| **`nullptr`** | C++；C23 可用；语义比 `NULL` 更清晰，避免重载/宏歧义 |

**工程意义**：显式赋 `NULL`/`nullptr`，表示「尚未绑定」或「已释放」；`if (p == NULL) return;` 能在问题发生时 **快速定位**，而不是带着随机地址默默写坏别处内存。

> `if (p)` 与 `if (p != NULL)` 等价（[3.2](../ch03-control-flow/3.2-if-else语句.md)）；**非空 ≠ 合法**—— 野指针、悬垂指针仍可能非 0。

## 六、完整示例

```c
#include <stdio.h>
#include <stdlib.h>

int main(void)
{
    int x = 10;
    int *p = &x;

    printf("x=%d, &x=%p, p=%p\n", x, (void *)&x, (void *)p);
    *p = 20;
    printf("after *p=20: x=%d\n", x);

    int *heap = malloc(sizeof *heap);
    if (heap == NULL)
        return 1;
    *heap = 42;
    free(heap);
    heap = NULL;   /* 释放后置空，防悬垂 */

    return 0;
}
```

`%p` 打印地址时用 `(void *)` 转换，避免类型不匹配告警。

## 七、工程实践（HFT / 内核）

1. **声明即初始化**：`int *p = NULL;` 或 `int *p = &x;`，杜绝裸 `int *p;`
2. **`free` 后立即 `p = NULL`**，双重释放时第二下 `free(NULL)` 安全（no-op）
3. **禁止** 为了省事对指针乱 `cast`；结构体用正确成员类型或 `char *` 做字节视图（`memcpy`）
4. 内核：**用户态指针** 必须先 `copy_from_user` 校验再访问，不能信任地址合法性
5. Code review 警惕：`int *p, q;` —— 只有 `p` 是指针（[1.6 §8](../ch01-introduction/1.6-数组.md)）
