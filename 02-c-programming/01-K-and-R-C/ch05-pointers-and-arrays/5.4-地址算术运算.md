# 5.4 地址算术运算

## 本节讲什么

指针支持 **有限的算术**：加减整数、同类型指针相减/比较。设计初衷 **仅服务连续数组内存模型**；脱离同一数组对象范围的运算是 **未定义行为（UB）**。运算单位是 **「元素」** 而非字节，编译器按 `sizeof(*p)` 自动缩放。

**前置**：[5.3 指针与数组](./5.3-指针与数组.md)（退化、`a[i]` ≡ `*(a+i)`）、[5.1 指针与地址](./5.1-指针与地址.md)。

## 一、核心前提

```
指针算术 ⊂ 连续数组模型
同一数组对象内  →  有定义
跨独立数组      →  UB（不可依赖）
```

| 原则 | 说明 |
|------|------|
| 单位是 **元素** | `p + n` 跳 `n × sizeof(*p)` **字节** |
| 合法范围 | 指向数组某元素，或 **末尾后一位哨兵** |
| 哨兵 | 可比较、可相减；**不能解引用** |

## 二、三类合法地址算术

### 1. 指针 ± 整数（最常用）

语法：`p + n` / `p - n`

- 偏移 `n` 个 **完整元素**；底层字节偏移 = **`n × sizeof(*p)`**
- 允许指向 **最后一个元素的后一位**（哨兵地址）

```c
char       *pc;  pc + 1;  /* +1 字节 */
int        *pi;  pi + 1;  /* +sizeof(int)，通常 4 字节 */
long       *pl;  pl + 1;  /* +8 字节（LP64） */
struct s   *ps;  ps + 1;  /* +sizeof(struct s) */
```

**工程用法：协议头偏移**

```c
uint8_t *pkt = ...;
size_t hdr_len = 14;                    /* 以太网头字节数 */
uint8_t *payload = pkt + hdr_len;       /* uint8_t*：+1 即 +1 字节 */
```

> 类型决定步长：`uint8_t *` 按字节走；`int *` 按 4 字节走——混淆「元素偏移」与「字节偏移」是 HFT/驱动高频 bug。

### 2. 同数组内两同类型指针相减

语法：`q - p`

| 属性 | 值 |
|------|-----|
| 返回类型 | **`ptrdiff_t`**（有符号，`<stddef.h>`） |
| 结果含义 | 两指针间相隔的 **元素个数**（不是字节数） |

```c
int a[10], *p = &a[2], *q = &a[7];
ptrdiff_t cnt = q - p;   /* 5，中间隔 5 个 int */
```

**环形队列算占用**（绕回须另处理）：

```c
int ring[8];
int *head = &ring[2], *tail = &ring[6];
ptrdiff_t used = tail - head;   /* 4 个元素（同数组内合法） */
/* tail 绕回开头时：used = (tail - head + N) % N */
```

### 3. 同类型指针比较 `== != < > <= >=`

**合法条件**：两指针指向 **同一数组对象** 内元素，或 **末尾哨兵地址**。

```c
if (p < q) {
    /* p 在低地址，同数组内比较有定义 */
}
```

逻辑：按 **内存地址数值** 判断前后顺序。

## 三、严格禁止 / UB 清单（高频踩坑）

### 1. 不同数组的指针相减、比较

```c
int a[5], b[5];
int *p = a, *q = b;
ptrdiff_t x = q - p;   /* ❌ UB */
if (p < q) {}          /* ❌ UB */
```

两数组内存 **不保证连续**，差值/大小关系 **无标准定义**。

### 2. 指针 × / ÷ / %

C **语法不支持**，直接编译报错：

```c
p * 2;   /* 编译失败 */
p / 4;
```

### 3. 解引用哨兵地址（末尾后一位）

```c
int a[5], *p = a + 5;   /* ✅ 合法哨兵，可比较 */
*p = 100;               /* ❌ UB，越界读写 */
```

### 4. 偏移到数组前边界之外

```c
int *p = a - 1;         /* 通常 UB（即使不解引用也危险） */
*p = 0;                 /* ❌ 访问未知内存 */
```

## 四、示例完整拆解

```c
int a[10], *p = &a[2], *q = &a[7];
ptrdiff_t n = q - p;   /* n = 5 */
```

**内存视角**（设 `base = &a[0]`）：

| 指针 | 地址 | |
|------|------|--|
| `p = &a[2]` | `base + 2×sizeof(int)` | |
| `q = &a[7]` | `base + 7×sizeof(int)` | |
| 字节差 | `5 × sizeof(int)` | |
| 元素差 `q - p` | **5** | 字节差 ÷ `sizeof(int)` |

```c
/* 哨兵与负下标 */
int *end = a + 10;      /* 末尾后一位，合法指针 */
int last = end[-1];     /* ✅ 等价 a[9]，仍在数组内 */
```

## 五、底层工程落地

| 场景 | 指针算术用法 |
|------|-------------|
| **环形缓冲区**（IO、HFT） | `tail - head` 算占用；绕回取模 |
| **网络协议解析** | `payload = pkt + hdr_len`，零拷贝定位负载 |
| **ELF / 页表 / MMIO** | 基址 + 结构体偏移；或 `uintptr_t` 字节运算 |
| **帧缓冲** | `row + y * stride + x` 定位像素 |

**零拷贝核心**：基指针 + 偏移直达目标字段，避免 `memcpy`。

## 六、与后续衔接 & 关键区分

### 1. `uintptr_t` / `intptr_t`（C99）

指针 ↔ 无符号整数互转，用于 **裸机、寄存器映射** 的 **纯字节地址** 运算：

| 运算 | 步长 |
|------|------|
| `int *p; p + 1` | `sizeof(int)` 字节（通常 4） |
| `uintptr_t u; u + 1` | **1 字节** |

```c
#include <stdint.h>
volatile uint32_t *reg = (uint32_t *)0x40001000;
uintptr_t base = (uintptr_t)reg;
uintptr_t next = base + 4;   /* 字节级 +4，不是 int* +1 */
```

**普通指针算术**（按元素缩放）与 **`uintptr_t` 字节运算** 逻辑分开，不可混用。

### 2. 延伸章节

| 章节 | 内容 |
|------|------|
| **5.6 / 5.9** | 多维数组、二级指针偏移层级 |
| **ch08** | 用户缓冲区边界检查 |

### 3. 高危 Bug 预警

| Bug | 后果 |
|-----|------|
| 硬编码字节数当 `int*` 偏移 | 字段错位 |
| 跨数组 `p < q` / `q - p` | UB，内核/驱动数据错乱 |
| `ptrdiff_t` 为负转 `size_t` | 变成超大正数 |
| 解引用 `buf + len` 哨兵 | 段错误 |

## 背诵要点（精简版）

```
指针算术只服务同一数组对象；跨数组相减/比较 = UB
p+n：跳 n 个元素；步长 = sizeof(*p)
q-p：元素个数，类型 ptrdiff_t（有符号）
哨兵 a+N：可比较，不可 *
禁止 p*2、p/4；禁止 a-1 解引用
uintptr_t +1 是 1 字节；int* +1 是 4 字节
```
