# 5.3 指针与数组

## 本节讲什么

数组与指针 **紧密绑定但非同一概念**。核心定理：**数组名在多数表达式中隐式退化为首元素指针**；`a[i] ≡ *(a + i)`。必须吃透 **退化规则、两大不退化例外、指针算术缩放、`sizeof` 陷阱、传参丢长度** —— HFT / 内核 / DPDK 偏移算错是高频致命 bug。

**前置**：[1.6 数组](../ch01-introduction/1.6-数组.md)、[1.9 字符数组](../ch01-introduction/1.9-字符数组.md)、[5.1](./5.1-指针与地址.md)、[5.2](./5.2-指针与函数参数.md)。

## 一、核心定理：数组名隐式退化规则

定义 `int a[10];`

### 1. 常规表达式：自动退化为首元素指针

- `a` 在赋值、运算、传参时等价于 **`&a[0]`**，类型 **`int *`**
- `a + i`：基地址向后偏移 **`i × sizeof(int)`** 字节
- 下标是语法糖：**`a[i] ≡ *(a + i)`**，编译器完全等同处理

```c
a[3] = *(a + 3) = 3[a];   /* 3[a] 语法合法：*(3+a)，极少手写 */
```

### 2. 两大不退化例外（必考）

数组名 **保留完整数组类型**，不转首元素指针：

| 表达式 | 行为 | 原因 |
|--------|------|------|
| **`sizeof(a)`** | 整个数组字节数 `10 × sizeof(int)` | 若退化只得指针宽 8，与语义冲突 |
| **`&a`** | 整个数组的地址，类型 **`int (*)[10]`** | 需要「数组整体」类型信息 |

**`&a` vs `&a[0]`**：

| | 地址数值 | 类型 | `+1` 步长 |
|---|---------|------|----------|
| `&a[0]` / `a`（退化） | 相同 | `int *` | 1 个 `int`（通常 4 字节） |
| `&a` | 相同 | `int (*)[10]` | **整数组** 10 个 `int`（通常 40 字节） |

### 3. 偏移缩放规则（指针算术核心）

指针 ± 整数时，偏移 **自动按元素类型缩放**（详见 [5.4](./5.4-地址算术运算.md)）：

```c
char  *pc;  pc + 1  →  +1 字节
int   *pi;  pi + 1  →  +sizeof(int) 字节（通常 4）
long  *pl;  pl + 1  →  +8 字节（LP64）
```

这也是 `a[i]` **不用手动乘类型长度** 的底层原理。

## 二、代码示例逐段拆解

```c
int a[10];
int *p = a;           /* 退化，等价 p = &a[0] */

a[3] = 0;
*(p + 3) = 0;         /* 完全等价，同一内存 */

/* 指针遍历（内核/DPDK/HFT 高频写法） */
for (p = a; p < a + 10; p++)
    *p = 0;
```

**遍历逻辑**：

- `a + 10` 指向 **末尾后一位**（合法指针，**不能解引用**）
- `p++` 每次前进一个 `int` 宽度
- `*p` 直接改数组 **原始内存**，无副本拷贝

## 三、数组首地址三种取法

| 方式 | 写法 | 类型 | 建议 |
|------|------|------|------|
| ① 数组名 | `int *p = a;` | `int *` | ✅ 最常用 |
| ② 首元素 | `int *p = &a[0];` | `int *` | ✅ 与 ① 等价 |
| ③ 整个数组 | `int (*p)[10] = &a;` | `int (*)[10]` | ⚠️ `+1` 步长整数组，HFT 遍历慎用 |

## 四、数组 vs 指针 核心对比（笔试/面试高频）

以 `int a[10]` / `int *p` 对比：

| 对比项 | 数组 `int a[10]` | 指针 `int *p` |
|--------|-----------------|---------------|
| 内存 | 连续 10 个 `int` | 一块内存存 **单一地址** |
| `sizeof` | `10 × sizeof(int)` | `sizeof(int *)`（64 位通常 8） |
| 自身赋值 | ❌ `a = p` 非法 | ✅ `p = a` 可改指向 |
| 自增 | `a++` 非法 | `p++` 合法 |
| 初始化 | `int a[10] = {1,2,3};` | `int *p = a;` |
| 取地址 | `&a` → `int (*)[10]` | `&p` → `int **` |
| 传函数 | 退化为 `int *`，**丢长度** | 本来就是 `int *` |

### 关键易错：`char arr[]` vs `char *ptr`（字符串）

```c
char arr[] = "abc";   /* 栈上数组，可改 arr[0]='x'；sizeof(arr)=4 */
char *ptr  = "abc";   /* 指向只读常量区；ptr[0]='x' → 段错误；sizeof(ptr)=8 */
```

见 [1.9 字符数组](../ch01-introduction/1.9-字符数组.md)。

## 五、数组作函数参数（衔接 [5.2](./5.2-指针与函数参数.md)）

两种形参 **完全等价**，编译器均作指针处理：

```c
void func(int arr[], int len);
void func(int *arr, int len);
```

| 特性 | 说明 |
|------|------|
| 实参传入 | 数组名退化，仅传 **首地址**，零拷贝 |
| `sizeof(arr)` 在函数内 | **指针大小**（8），**不是**原长度 |
| 修改元素 | `arr[i]` / `*(arr+i)` 可改 **外部原数组** |
| 返回值 | **不能** 整体返回数组；可返回首指针（注意生命周期） |

```c
#define ARRLEN(a) (sizeof(a) / sizeof((a)[0]))

void double_arr(int *arr, int len)
{
    for (int i = 0; i < len; i++)
        arr[i] *= 2;
}

int a[5];
double_arr(a, ARRLEN(a));   /* 长度在调用方用 sizeof 算 */
```

### 禁止操作

```c
int b[10] = a;              /* ❌ 数组不能整体赋值 */
int *f(void) {
    int a[5];
    return a;               /* ❌ 悬垂指针：栈帧销毁后失效 */
}
```

## 六、底层工程场景（DPDK / 内核 / HFT）

| 场景 | 用法 |
|------|------|
| **DPDK mbuf** | 基址指针 + 偏移访问分片，全程不拷整块 |
| **Linux `sk_buff`** | 数据区指针 + 偏移遍历 eth/ip/tcp 头 |
| **帧缓冲** | `buf + y * width + x` 定位像素 |
| **HFT 行情** | `double *px = book;` 指针遍历档位 |

**零拷贝核心**：传 **首地址 + len/stride**，代替复制整块数组，省内存带宽。

## 七、延伸预告：数组指针 vs 指针数组（[5.6](./5.6-指针数组以及指向指针的指针.md)）

极易混淆，先记速查表：

| 声明 | 读法 | 是什么 |
|------|------|--------|
| `int *a[5]` | `a` 先与 `[5]` 结合 | **指针数组**：5 个 `int *` |
| `int (*a)[5]` | `a` 先与 `*` 结合 | **数组指针**：指向 `int[5]` |
| `char *argv[]` | 同上 | `main` 参数：字符串指针数组 |
| `int (*p)[10] = &arr` | [5.3 §一](./5.3-指针与数组.md) | 整个一维数组的地址 |

**口诀**：**括号优先** —— `()` 贴名是指针，`[]` 贴名是数组。多维展开见 [5.7](./5.7-多维数组.md)、[5.9](./5.9-指针与多维数组.md)。

## 八、Bug 预警

| Bug | 后果 |
|-----|------|
| 函数内 `sizeof(arr)` 当数组长度 | 得到 8，越界或漏处理 |
| 越界：`p < a+10` 写成 `p <= a+10` 后多解引用 | UB |
| 返回局部数组指针 | 悬垂，野指针 |
| `char *p = "lit"; p[0]='X'` | 写只读常量区，崩溃 |
| `&arr` 强转 `int *` 后 `+i` | 步长错误，HFT 读飞数据 |

## 背诵要点（精简版）

```
多数表达式 a → &a[0]（int*）；sizeof(a)、&a 不退化
a[i] == *(a+i)；p+1 步长按元素类型缩放
取首地址用 a 或 &a[0]；&arr 步长整数组
传函数 → int* + len；sizeof(形参)=8
char[] 可改；char* 字面量只读
int *a[5] 指针数组；int (*a)[5] 数组指针
```
