# 5.2 指针与函数参数

## 本节讲什么

C **不存在引用传递、地址传递**，**所有函数参数一律值拷贝**（[1.8](../ch01-introduction/1.8-参数传值调用.md)、[4.1](../ch04-functions-and-program-structure/4.1-函数的基本知识.md)）。传指针时，拷贝的是 **指针里存的地址值**，不是所指对象；函数通过 `*` 解引用即可 **直接操作调用方内存**，无需拷贝大段数据——HFT 追求极致效率、避免内存带宽开销的典型手段。

**前置**：[5.1 指针与地址](./5.1-指针与地址.md)（`&`、`*`、`int **`）、[1.8 传值调用](../ch01-introduction/1.8-参数传值调用.md)。

## 一、核心底层逻辑：C 唯一传参规则——值传递

### 1. 普通变量传参

把变量的 **数值** 复制一份给形参；函数内操作副本，与外部原变量 **完全隔离**，**无法** 修改原值。

```c
void f(int x)
{
    x = 100;   /* 只改形参副本 */
}
```

### 2. 指针变量传参

把 **内存地址** 复制一份给指针形参：

- 形参 `px` 与实参表达式 `&a` **不是同一个指针变量**（栈上两块不同内存）；
- 但二者存的 **地址值相同**，指向 **同一个** 原始对象（无论在栈/堆/静态区）；
- `*px` 沿该地址访问 → **能修改外部变量的真实值**。

```
main 栈帧                    swap 栈帧
┌─────────┐                 ┌─────────┐
│ a = 1   │ ←── 同地址 ──── │ px 副本 │
│ b = 2   │ ←── 同地址 ──── │ py 副本 │
└─────────┘      *px/*py    └─────────┘
```

### 表格精炼总结

| 函数定义 | 拷贝内容 | 能否改外部变量 | 关键操作 |
|----------|----------|----------------|----------|
| `void f(int x)` | `int` **数值副本** | ❌ | 改 `x` 只改副本 |
| `void f(int *px)` | **地址副本**（指针值） | ✅ | `*px = 新值` 改原变量 |
| `void f(int **ppx)` | 二级指针地址副本 | ✅ 改外部指针本身 | `*ppx = new_ptr`（[5.1](./5.1-指针与地址.md)） |

### 易混区分（高频坑）

| 操作 | 效果 |
|------|------|
| `px = &tmp` | 只改 **形参指针副本** 的指向，外部实参 **不变** |
| `*px = 100` | 通过地址写 **原始内存**，外部变量 **同步变化** |

> 口诀：**一级 `*p` 改所指；`p = …` 只改副本；二级 `*pp` 改外部指针**。

## 二、`swap` 逐行拆解

```c
void swap(int *px, int *py)
{
    int t = *px;    /* 读出 px 指向的外部 a 的值 */
    *px = *py;       /* 把 b 的值写入 a 的内存 */
    *py = t;        /* 把原 a 的值写入 b 的内存 */
}

int main(void)
{
    int a = 1, b = 2;
    swap(&a, &b);   /* &a、&b 作为地址实参传给 px、py */
    return 0;
}
```

**执行流程**：

1. `&a` 计算 `a` 的地址，**拷贝** 给形参 `px`；`&b` 拷贝给 `py`；
2. `px`、`py` 是 `swap` 栈帧内的 **局部指针**，与 `main` 里的 `a`、`b` **无直接绑定**，仅 **共享地址**；
3. `*px`、`*py` **穿透副本**，直接读写 `main` 栈上 `a`、`b` 的原始内存；
4. `swap` 返回后 `px`/`py` 栈帧销毁，但 `a`、`b` 的值 **已永久交换**。

| 调用写法 | 含义 |
|----------|------|
| `swap(&a, &b)` | 传 **地址值**，不是传 `a`、`b` 的数值 |
| `*px`、`*py` | 访问 **外面** 的 `a`、`b` |

## 三、三大典型业务场景

### 1. 交换两个独立变量（`swap`）

无指针只能交换副本，函数返回后原值不变——入门最经典案例（[1.8](../ch01-introduction/1.8-参数传值调用.md)）。

### 2. 输出多组结果（输出参数）

C 函数仅 **单返回值**；多结果靠指针 **原地写入**：

```c
void div_mod(int a, int b, int *quot, int *rem)
{
    *quot = a / b;
    *rem  = a % b;
}

int main(void)
{
    int q, r;
    div_mod(7, 3, &q, &r);   /* q=2, r=1 */
    return 0;
}
```

### 3. 底层高性能 / 系统开发

| 场景 | 典型接口 | 指针传参的作用 |
|------|----------|----------------|
| **Linux 内核** | `copy_to_user(void __user *to, const void *from, unsigned long n)` | 操作用户态缓冲区，避免整块拷贝 |
| **硬件驱动** | `volatile uint32_t *reg` | 映射 MMIO，直接读写寄存器 |
| **HFT 环形队列** | `head`/`tail` 指针（或 `int **`）传入 | 原地更新读写位置，降延迟 |
| **订单簿** | `level *` 传入 | 原地改档位，零拷贝 |

**HFT 偏爱指针传参**：只传 8 字节地址；原地更新共享结构；多返回值。代价：调用方须保证 **指针合法、生命周期够长**；内核还须校验用户态地址。

## 四、数组参数补充（衔接 [5.3](./5.3-指针与数组.md)）

数组作函数参数时，编译器 **隐式退化为首元素指针**：

```c
void func(int arr[], int len);   /* 与下面完全等价 */
void func(int *arr, int len);
```

- 传入数组名 ≡ `&arr[0]`，拷贝 **首元素地址**；
- 函数内 `arr[i]` 或 `*(arr+i)` 可 **修改原数组元素**；
- 函数内 **拿不到** 数组真实长度，`sizeof(arr)` = 指针宽度（8）→ **必须另传 `len`**。

```c
void double_arr(int *arr, int len)
{
    for (int i = 0; i < len; i++)
        arr[i] *= 2;    /* 原地修改调用方数组 */
}
```

## 五、易错反例对比

```c
/* ❌ 反例 1：传 int，交换失效 */
void swap_bad(int x, int y)
{
    int t = x; x = y; y = t;
}
/* swap_bad(a, b) 后 a、b 不变 */

/* ✅ 正例：传地址 */
void swap_ok(int *px, int *py) { /* 见 §二 */ }

/* ❌ 反例 2：漏写 &，类型不匹配 */
void f(int *p);
// f(a);      /* 编译错误：传了 int 不是 int* */
// f(*a);     /* 语义错误 */

/* ❌ 反例 3：漏写 *，只改副本 */
void set_bad(int *p)
{
    p = (int *)0x1234;   /* 只改形参副本，外面不变 */
}

void set_ok(int *p)
{
    *p = 100;            /* ✅ 改所指对象 */
}

/* ❌ 反例 4：野指针 / NULL 解引用 */
void crash(int *p)
{
    *p = 1;              /* p 未初始化或 NULL → 段错误 / UB */
}
```

| 错误 | 现象 |
|------|------|
| 传值 `swap(int,int)` | 外面不交换 |
| `f(a)` 漏 `&` | 编译报错 |
| `p = NULL` 忘 `*` | 外面无变化 |
| 野指针解引用 | segfault 或静默写坏内存 |

## 六、与后续衔接 & Bug 预警

### 1. C 指针 ≠ C++ 引用

| | C `int *p` | C++ `int &r` |
|---|-----------|-------------|
| 机制 | 地址 **副本** | 变量 **别名**，无指针副本 |
| 空值 | 可为 `NULL` | 引用必须绑定对象 |
| 底层 | 显式解引用 `*p` | 语法像原变量 `r` |

### 2. 最高频 Bug：`&` / `*` 混淆

- **漏 `*`**：改指针副本，外部无变化；
- **漏 `&`**：传数值非地址，编译报错；
- **野指针 / `NULL`**：解引用 → 段错误（[5.1](./5.1-指针与地址.md)）。

## 背诵要点（精简版）

```
C 只有值传递；传指针 = 拷贝地址，不是拷贝数据
px 与 &a 不同变量、同地址；*px 改原内存
px=&tmp 改副本；*px=100 改外面
swap/div_mod/内核/HFT：输出参数 + 零拷贝 + 原地更新
数组参 → int* + len；C 指针 ≠ C++ 引用
```
