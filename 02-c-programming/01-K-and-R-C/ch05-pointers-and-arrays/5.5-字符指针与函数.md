# 5.5 字符指针与函数

## 本节讲什么

C **没有独立 `string` 类型**（[1.9](../ch01-introduction/1.9-字符数组.md)）：字符串 = **`char` 连续存储 + 终止符 `'\0'`**。库函数 `strlen`、`strcpy`、`strcmp` 及本节 **返回指针的函数**，全部基于 **`char *` 指针遍历** 与 [5.4 地址算术](./5.4-地址算术运算.md)。同时涵盖 **`char *` vs `char[]`** 存储差异与 **生命周期陷阱**。

**前置**：[5.2](./5.2-指针与函数参数.md)、[5.3](./5.3-指针与数组.md)、[5.4](./5.4-地址算术运算.md)、[1.9 字符数组](../ch01-introduction/1.9-字符数组.md)。

## 一、核心：C 字符串底层本质

```
字符串 = char 连续序列 + 末尾 '\0'（ASCII 0）
结束条件：*p == '\0'
```

- 所有 `<string.h>` 函数依赖 **`char *` 自增遍历**，**无内置长度**；
- 忘记 `'\0'` → 库函数 **无限遍历** → 越界 UB。

## 二、两种字符串存储深度对比（必考）

```c
char *s   = "abc";    /* 写法 1：指针 → 只读字面量 */
char buf[] = "abc";   /* 写法 2：数组 → 可写副本 */
```

### 1. `char *s = "abc";`

| 项 | 说明 |
|----|------|
| `"abc\0"` 存哪 | **只读段** `.rodata` / 常量区（[4.3.1](../ch04-functions-and-program-structure/4.3.1-程序内存布局.md)） |
| 相同字面量 | 编译器 **可能合并** 为同一块内存 |
| 变量 `s` | 栈上 **普通指针**，仅存常量区首地址 |
| 修改 | `s[0]='x'` / `*s='x'` → **UB**，通常 **段错误** |
| `sizeof(s)` | **指针大小**（64 位通常 8） |

### 2. `char buf[] = "abc";`

| 项 | 说明 |
|----|------|
| 存储 | 编译时把 `"abc\0"` **拷贝** 到数组：局部在 **栈**，全局在 **静态区** |
| 修改 | `buf[0]='x'` ✅ 合法 |
| `sizeof(buf)` | **4**（`a b c \0`），整个缓冲区大小 |

### 速记口诀

```
指针字面量 → 只读常量区，不可改
数组字符串 → 拷贝副本，可读写
```

对比表见 [5.3 §四](./5.3-指针与数组.md)。

## 三、标准库函数：底层指针实现

全部基于 `char *` 地址算术，依赖 `'\0'` 终止：

### `strlen(const char *s)`

指针自增，计数直到 `*p == '\0'`；返回 **有效字符数**（**不含** `\0`）。

```c
size_t my_strlen(const char *s)
{
    const char *p = s;
    while (*p != '\0')
        p++;
    return (size_t)(p - s);   /* 5.4：同数组内指针相减 */
}
```

### `strcpy(char *dst, const char *src)`

```c
while (*dst++ = *src++)
    ;   /* 逐字节拷贝，含末尾 '\0' */
```

| 缺陷 | 说明 |
|------|------|
| **无边界检查** | `dst` 不够大 → **缓冲区溢出** |
| 工程替代 | `strncpy` / `strlcpy` / `snprintf`；内核/HFT **禁用裸 `strcpy`** |

### `strcmp(const char *s1, const char *s2)`

两指针同步前进，逐字符比；遇不同或任一方 `'\0'` 停止；返回字典序差值。

## 四、返回指针的函数（面试高频）

### 合法可返回

| 来源 | 示例 | 生命周期 |
|------|------|----------|
| **全局 / 文件 static 数组** | `static char g_buf[100];` | 程序全程 |
| **函数内 `static` 局部** | `static char buf[] = "ok";` | 程序全程 |
| **调用方缓冲区**（输出参数） | `void fmt(char *out, ...)` | 由调用方管理 |
| **堆 `malloc`** | `return malloc(n);` | 调用方 `free` |

### 绝对禁止：普通局部自动数组

```c
char *bad(void)
{
    char local[] = "oops";   /* 函数栈帧 */
    return local;            /* ❌ 栈帧销毁 → 野指针 */
}
```

函数返回后 `local` 内存被覆盖；再读 → 乱码 / 崩溃。

### 正确示范

```c
char *good_static(void)
{
    static char buf[] = "safe str";
    return buf;              /* ✅ static 区，程序结束才释放 */
}

void good_buf(char *out, size_t outsz, const char *in)
{
    strncpy(out, in, outsz - 1);
    out[outsz - 1] = '\0';   /* ✅ 写入调用方缓冲区 */
}
```

## 五、示例：`swap` 交换字符串指针（K&R）

```c
void swap(char *v[], int i, int j)
{
    char *temp = v[i];
    v[i] = v[j];
    v[j] = temp;
}
```

| 要点 | 说明 |
|------|------|
| `char *v[]` | 等价 **`char **v`**（[5.6](./5.6-指针数组以及指向指针的指针.md)） |
| 交换内容 | 仅交换 **两个 `char*` 地址**（各 8 字节），**不拷贝** 字符串正文 |
| 用途 | 字符串 **排序** 标准写法；`main` 的 `argv` 排序同理（[5.10](./5.10-命令行参数.md)） |

## 六、高频考点 & 工程坑点

| 坑 | 后果 |
|----|------|
| 改字面量 `*s = 'x'` | 段错误（只读保护） |
| 返回栈局部数组指针 | 野指针 |
| 裸 `strcpy` | 缓冲区溢出；CVE 常客 |
| 缺 `'\0'` | `strlen` 无限遍历 |
| `sizeof(char*)` 当字符串长 | 得到 8 不是字符数 |

**`const char *`**：只读字符串限定，防误写（[2.3 常量](../ch02-types-operators-expressions/2.3-常量.md)）。

## 七、底层工程场景

| 场景 | 用法 |
|------|------|
| **UEFI / `printk`** | `char *` 格式化遍历 |
| **网络协议** | 载荷转 `char *` 解析命令/字段 |
| **HFT 日志** | 预分配 `char[]` 缓冲，禁字面量修改、禁栈指针返回 |
| **`main(argc, argv)`** | `char *argv[]` 指针数组，本节 `swap` 可直接排序参数 |

## 背诵要点（精简版）

```
字符串 = char序列 + '\0'；库函数靠 *p 遍历到 '\0'
char *s="abc" → 只读 rodata，sizeof=8，不可改
char buf[]="abc" → 副本可改，sizeof=4
禁止 return 栈局部数组；可 return static/全局/malloc/调用方buf
strcpy 无界 → strncpy/strlcpy；swap 换指针不换串
```
