# 5.8 指针数组的初始化

## 本节讲什么

**指针数组** 可在定义时用 **字符串字面量列表** 初始化：编译器把各字面量放进只读区，数组元素存 **指向它们的 `char *`**。K&R 月份表、关键字表是经典范例。与 [5.7](./5.7-多维数组.md) **二维 `char` 数组** 形成一组 **笔试高频对照题**。

**前置**：[5.5 字符指针](./5.5-字符指针与函数.md)（字面量只读）、[5.6 指针数组](./5.6-指针数组以及指向指针的指针.md)（`char *argv[]`）。

## 一、核心原理：初始化逻辑

```c
char *month[] = {
    "January", "February", "March"
};
```

| 阶段 | 发生什么 |
|------|----------|
| **编译** | 各 `"January"` 等存入 **只读段 `.rodata`**，自带 `'\0'` |
| **类型** | `month` 是 **指针数组**，元素类型 **`char *`** |
| **初始化** | 每个字面量 **首地址** 赋给 `month[i]` |
| **长度** | 由 `{}` 内元素个数 **自动推断**，无需写 `[3]` |

### 内存结构（分层）

```
栈/全局区                    只读 .rodata（分散）
┌─────────────────┐         ┌──────────┐  ┌───────────┐
│ month[0] ───────┼────────→│ "January"│  │ "February"│ …
│ month[1] ───────┼───┐     └──────────┘  └───────────┘
│ month[2] ───────┼───┘
└─────────────────┘
  只存地址指针              长短不一的字符串正文
```

- **不拷贝** 字符串正文进数组槽位，只存 **8 字节指针**（64 位）；
- 修改 `month[0][0] = 'x'` → 写只读区 → **段错误**（[5.5](./5.5-字符指针与函数.md)）。

## 二、实用技巧：自动计算数组长度

```c
char *month[] = {"Jan", "Feb", "Mar"};
#define MONTH_CNT (sizeof(month) / sizeof(month[0]))   /* 3 */
```

| 位置 | `sizeof(month)` 含义 |
|------|---------------------|
| **定义处**（文件/块作用域数组） | 整个指针数组字节数 → 可算元素个数 |
| **函数形参** `char *tbl[]` | 退化为 `char **`，`sizeof(tbl)` = **8** ❌ |

适用：循环遍历、二分查表、命令关键字匹配。

```c
for (int i = 0; i < MONTH_CNT; i++)
    printf("%s\n", month[i]);
```

## 三、两种字符串表完整对比（面试必考）

### 1. 指针数组 `char *names[]`

```c
char *names[] = {"if", "else", "while"};
```

| 项 | 说明 |
|----|------|
| 行宽 | **可变**，无填充浪费 |
| 布局 | 指针数组 + **分散** 只读字符串 |
| 修改 | `names[0][0]='x'` ❌ 段错误 |
| `sizeof(names)` | 元素数 × `sizeof(char *)` |

### 2. 二维字符数组 `char names[][10]`

```c
char names[][10] = {"if", "else", "while"};
```

| 项 | 说明 |
|----|------|
| 行宽 | **固定 10**，短串后补 `'\0'` + 填充 |
| 布局 | **一整块连续** 可读可写内存 |
| 修改 | `names[0][0]='x'` ✅ |
| `sizeof(names)` | 行数 × 10 |

### 对比速查表

| 对比项 | `char *names[]` | `char names[][10]` |
|--------|-----------------|-------------------|
| 单行长度 | 可变，省空间 | 固定宽，有冗余 |
| 内存 | 指针 + 分散 `.rodata` | 连续块 |
| 改内容 | ❌ 只读 | ✅ 可改 |
| 传参 | `char **names` | `char (*names)[10]` |
| 超长/超短混排 | ✅ 优选 | 固定宽浪费或截断 |

与 [5.7 §四](./5.7-多维数组.md)「真二维 vs 指针数组」同一族对照。

## 四、工程示例：关键字 / 命令表

```c
char *keywords[] = {
    "if", "else", "while", "for", "switch",
    "unsigned long long", "static const"
};
#define KEYWORD_NUM (sizeof(keywords) / sizeof(keywords[0]))

for (int i = 0; i < KEYWORD_NUM; i++)
    printf("%s\n", keywords[i]);
```

长短文本混杂时，指针数组 **最省内存**；若需运行时改词，改用 `char keywords[][N]` 或堆分配。

## 五、底层工程场景

| 场景 | 用法 |
|------|------|
| **内核 syscall 名表 / sysfs** | 静态 `char *sys_call_names[]` + 二分 |
| **Shell / CLI 内建指令** | 指针数组 + 字符串匹配 |
| **HFT 错误码 / 日志字段映射** | 枚举 → 文本查表 |
| **协议 opcode 描述** | 长短描述混排，指针数组节约 RAM |

## 七、高频 Bug 预警

| Bug | 后果 |
|-----|------|
| `keywords[0][0]='a'` | 写 `.rodata`，崩溃 |
| `char *tbl[]` 传给 `char tbl[][N]` 形参 | 类型不匹配 |
| 函数内 `sizeof(tbl)` 当元素个数 | 只得 8 |
| 二维 `char[][5]` 存超长串 | 截断，无 `'\0'` 保护若宽度不够 |

## 背诵要点（精简版）

```
char *m[]={"Jan",...} → 元素是 char*，指向 rodata
初始化 = 赋字面量首地址；长度由 {} 个数决定
定义处 MONTH_CNT = sizeof(m)/sizeof(m[0])
指针数组：可变长、只读串、传 char**
二维 char[][N]：定宽、可改、传 char(*)[N]
超长混排文本 → 优先指针数组
```
