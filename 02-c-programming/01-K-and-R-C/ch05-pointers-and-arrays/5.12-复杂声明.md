# 5.12 复杂声明

## 本节讲什么

综合运用 ch05 全部指针规则，**读懂**（必要时写出）多层指针、数组、函数返回类型的 C 声明。读 Linux、DPDK、UEFI 头文件的必备技能 —— 本章 **总复习与收官**。

**前置**：[2.4 声明围绕标识符](../ch02-types-operators-expressions/2.4-声明.md)、[5.1](./5.1-指针与地址.md)～[5.11](./5.11-指向函数的指针.md)。

## 一、核心逻辑：声明以标识符为中心

C 声明 **不是从左往右念类型名**，而是：

> **以变量/函数名为核心，向外逐层解析 `[]`、`()`、`*`。**

**优先级**（紧贴标识符最高）：

```
[]、()  >  *
括号 () 可强行改变绑定
```

### 通用解读口诀（右左法则根基）

1. **定位中间标识符**；
2. **先向右**：`[]` = 数组，`()` = 函数；
3. **再向左**：`*` = 指针；
4. **括号内优先**解析，再循环向外包裹；
5. 最后拼接基础类型（`int` / `char` / `void` …）。

```
对标识符做什么（[] / () / *），得到的就是其元素类型或指向类型
```

## 二、基础案例逐层拆解

### 1. `int *a[10]` — 指针数组

| 步骤 | 解析 |
|------|------|
| 标识符 | `a` |
| 右 `[10]` | `a` 是 **长度 10 的数组** |
| 左 `*` | 元素类型 **`int *`** |

**读法**：`a` 是数组，含 10 个 `int` 指针。（[5.6](./5.6-指针数组以及指向指针的指针.md)、[5.8](./5.8-指针数组的初始化.md)）

### 2. `int (*b)[10]` — 数组指针

| 步骤 | 解析 |
|------|------|
| 标识符 | `b` |
| `(*b)` | 括号使 `*` 先绑 `b` → **`b` 是指针** |
| 右 `[10]` | 指向 **`int[10]`** |

**读法**：`b` 是指向「含 10 个 `int` 的数组」的指针。（[5.7](./5.7-多维数组.md)、[5.9](./5.9-指针与多维数组.md)）

### 3. `int *f(void)` — 指针函数

| 步骤 | 解析 |
|------|------|
| 标识符 | `f` |
| 右 `(void)` | `f` 是 **无参函数** |
| 左 `*` | 返回 **`int *`** |

**读法**：`f` 是函数，返回 `int*`。（[5.11](./5.11-指向函数的指针.md)）

### 4. `int (*pf)(void)` — 函数指针

| 步骤 | 解析 |
|------|------|
| 标识符 | `pf` |
| `(*pf)` | **`pf` 是指针** |
| 右 `(void)` | 指向「无参、返回 `int`」的函数 |

**读法**：`pf` 是函数指针。

### 5. `char **argv` — 二级字符指针

| 步骤 | 解析 |
|------|------|
| 标识符 | `argv` |
| 左第一层 `*` | `*argv` 是 `char *` |
| 再一层 `*` | `argv` 是 **`char **`** |

> 形参 `char *argv[]` 退化后等价 **`char **argv`**（[5.10](./5.10-命令行参数.md)）。

### 速记对照表

| 声明 | 标识符是 | 口诀 |
|------|----------|------|
| `int *a[10]` | **数组** | 无括号，`[]` 贴名 |
| `int (*b)[10]` | **指针** | `(*名)`，`*` 贴名 |
| `int *f()` | **函数** | `()` 贴名，返回 `int*` |
| `int (*pf)()` | **指针** | `(*名)()`，指向函数 |

## 三、地狱级示例：旧式 `signal` 原型

```c
int (*signal(int sig, void (*handler)(int)))(int);
```

**分层拆解**：

1. 中心标识符 **`signal`**，右侧 `(int, void (*handler)(int))` → **`signal` 是函数**；
2. **参数 1**：`int sig`；
3. **参数 2**：`handler` 是 **函数指针**，指向「入参 `int`、返回 `void`」的函数；
4. 外层包裹 **`int (*)(int)`**：`signal` 的 **返回值** 是指针；
5. 该指针指向：「入参 `int`、返回 `int`」的函数。

**一句话**：`signal` 接收信号编号与处理函数指针，返回 **同签名** 的「`int → int` 函数指针」（旧式 UNIX 原型，教学用）。

**练习建议**：手画括号树；或用 [cdecl](https://cdecl.org) 对照。

## 四、`const` / `volatile` 与指针

`const` **修饰紧邻它的内容**，同样围绕标识符：

| 声明 | `p` 指向 | `p` 本身 | `p = other` | `*p = 'x'` |
|------|----------|----------|-------------|------------|
| `const char *p` | 内容只读 | 可改 | ✅ | ❌ |
| `char * const p` | 内容可写 | 不可改 | ❌ | ✅ |
| `const char * const p` | 只读 | 不可改 | ❌ | ❌ |

```c
const char *p = "a";
p = "b";      /* ✅ 改指向 */
/* *p = 'x';  ❌ */

char buf[] = "a";
char * const q = buf;
*q = 'x';     /* ✅ */
/* q = buf2;  ❌ */
```

见 [2.3 常量](../ch02-types-operators-expressions/2.3-常量.md)、[4.7.1 volatile](../ch04-functions-and-program-structure/4.7.1-volatile限定符.md)。

## 五、致命易错语法坑

### 1. `int *p, q;`

**`*` 只修饰紧跟的标识符**，逗号 **不传播**：

```c
int *p, q;    /* p: int*；q: int */
int *p, *q;   /* 两个都是 int* */
```

[1.6 §8](../ch01-introduction/1.6-数组.md)、[5.1](./5.1-指针与地址.md) —— code review 高频隐藏 bug。

### 2. 数组指针 vs 指针数组

| | 无 `(*名)` | 有 `(*名)` |
|---|-----------|------------|
| 绑定 | `[]` 先贴名 → **数组** | `*` 先贴名 → **指针** |

### 3. 函数指针漏括号

```c
int *pf();      /* ❌ 易误读：指针函数 */
int (*pf)();    /* ✅ 函数指针 */
```

## 六、工程方案：`typedef` 拆解

驱动 / DPDK 头文件标准写法：

```c
typedef void (*callback_t)(void *arg);
typedef int  (*binop_t)(int, int);

callback_t func_table[16];
void reg_cb(callback_t cb);
```

| 优势 | 说明 |
|------|------|
| 隐藏地狱声明 | 业务代码只见到 `callback_t` |
| 改签名一处 | 维护成本低 |
| ch6 结构体 | `ops` 成员批量声明（[6.7](../ch06-structures/6.7-类型定义typedef.md)） |

## 七、右左法则五步法（万能流程）

任意声明固定五步：

1. **定位标识符**；
2. **向右扫**：`[]` → 数组，`()` → 函数，直到不能继续；
3. **向左扫**：`*` → 指针；
4. **遇括号**：先解析括号 **内部** 为整体；
5. **循环向外**，最后接基础类型。

```
例：int (*(*fp)[10])(void)
   标识符 fp → (*fp) 指针 → [10] 指向含10元素…（继续练）
```

```
声明围绕标识符；[] () 优先于 *
```

## 面试题标准答案

1. **`void *p`**：`p` 是指针；`*p` 类型为 `void`（解引用前须转具体类型）。
2. **`int *a[10]`**：`a` 是数组，10 个 `int*`；**`int (*b)[10]`**：`b` 是指针，指向 `int[10]`。
3. **`int *f(void)`**：`f` 是无参函数，返回 `int*`；**`int (*pf)(void)`**：`pf` 是指向「无参返回 int 函数」的指针。
4. **`const char *p`**：可 `p = "other"`，不可 `*p = 'x'`；**`char * const p`**：不可改 `p`，可 `*p = 'x'`。
5. **指向含 5 个 `double` 的数组的指针**：`double (*ptr)[5];`
