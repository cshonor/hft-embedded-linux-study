# 5.11 指向函数的指针

## 本节讲什么

**函数指针** 保存 **函数入口地址**，用于回调、表驱动分发、调度器。必须与 **「返回指针的函数」（指针函数）** 区分 —— **面试第一高频坑**。衔接 [4.8 RPN](./../ch04-functions-and-program-structure/4.8-程序块结构.md) 运算符表、`qsort` 比较器、内核 `file_operations`。

**前置**：[5.6](./5.6-指针数组以及指向指针的指针.md)（`()` 与 `[]` 优先级）、[4.1](../ch04-functions-and-program-structure/4.1-函数的基本知识.md)。

## 一、指针函数 vs 函数指针（必背）

### 1. 指针函数（返回指针的普通函数）

```c
char *strcpy(char *dst, const char *src);
int *f(int x);
```

| 解析 | 含义 |
|------|------|
| `f` **先与 `()` 结合** | `f` 是 **函数** |
| 返回类型 `int *` | 返回值是 **指针** |

本质：普通函数，只是返回地址，**不是**指针变量。

### 2. 函数指针（存函数入口地址的指针变量）

```c
int (*fp)(int, int);
```

**语法拆解**（括号是灵魂）：

| 步骤 | 写法 | 含义 |
|------|------|------|
| 1 | `(*fp)` | `fp` 与 `*` 先结合 → **`fp` 是指针** |
| 2 | `(*fp)(int, int)` | 指向「两 `int` 入参、返回 `int`」的函数 |

### 最简记忆口诀

```
int *f()      无括号包 *f  →  函数，返回 int*（指针函数）
int (*fp)()   *fp 被 () 包  →  指针，指向函数（函数指针）
```

与 [5.6 §五](./5.6-指针数组以及指向指针的指针.md) 同一套「括号优先」规则。

## 二、赋值与调用

### 1. 取函数地址：`&` 可省略

函数名在表达式中 **退化为函数指针**（类似数组名退化）：

```c
int add(int a, int b) { return a + b; }

int (*fp)(int, int);
fp = add;      /* ✅ 推荐 */
fp = &add;     /* ✅ 等价 */
```

### 2. 通过指针调用：`*` 可省略

```c
int res1 = (*fp)(3, 4);   /* 标准解引用调用 */
int res2 = fp(3, 4);      /* ✅ 工程简写，语义相同 */
```

编译器识别函数指针，两种写法 **无性能差别**。

## 三、`typedef` 简化声明

```c
typedef int (*binop_t)(int, int);   /* 「两 int → int」的函数指针类型 */

binop_t fp = add;
binop_t op_table[] = { add, sub, mul };
```

| 优势 | 说明 |
|------|------|
| 可读 | 避免重复 `int (*)(int,int)` |
| 数组 | `binop_t op[]` 直接建函数指针数组 |
| 结构体 | ch6 成员常用（[6.7 typedef](../ch06-structures/6.7-类型定义typedef.md)） |

## 四、函数指针数组（表驱动分发）

替代长 `switch` / `if-else`，按下标跳转（[4.8](./../ch04-functions-and-program-structure/4.8-程序块结构.md)、[3.4 switch](../ch03-control-flow/3.4-switch语句.md)）：

```c
int add(int a, int b) { return a + b; }
int sub(int a, int b) { return a - b; }
int mul(int a, int b) { return a * b; }

int (*op[])(int, int) = { add, sub, mul };
#define OP_CNT (sizeof(op) / sizeof(op[0]))

int calc(int idx, int x, int y)
{
    if (idx >= 0 && idx < OP_CNT)
        return op[idx](x, y);
    return 0;
}
```

| 场景 | 用法 |
|------|------|
| **RPN 计算器** | `op[c](a,b)` 按运算符分发 |
| **Opcode 表** | 指令码 → 处理函数 |
| **状态机** | 状态 × 事件 → 动作函数 |

> `int (*op[])(int,int)`：`op` 是 **指针数组**，元素是 **函数指针**（[5.6](./5.6-指针数组以及指向指针的指针.md)）。

## 五、工程落地场景

| 领域 | 例子 |
|------|------|
| **标准库回调** | `qsort` 比较函数、`signal()` 信号处理 |
| **Linux 内核** | `struct file_operations`（open/read/write）；中断 handler；syscall 表 |
| **DPDK / HFT** | 事件回调、报文/定时器分发；策略函数表动态切换 |
| **嵌入式 / 飞控** | PID 钩子、飞行模式切换；任务入口表 |
| **UEFI** | `EFI_BOOT_SERVICES`、Protocol 结构体全函数指针 |

## 六、必考考点与限制

| 规则 | 说明 |
|------|------|
| **类型严格匹配** | 签名不同不能随意互赋；强转后调用 → **UB / 栈错乱** |
| **`void *` ≠ 函数指针** | 数据指针与函数指针 **不保证** 可互转；不能用 `void*` 通存函数地址 |
| **函数名退化** | 表达式中 `add` → 函数指针；`sizeof(add)` 非法（函数无 `sizeof`） |
| **不支持指针算术** | `fp + 1`、`fp - q` **非法**；函数地址不可像数组那样偏移 |

```c
void *p = (void *)fp;   /* 实现定义/可能非法，标准不推荐 */
fp++;                   /* ❌ 编译错误 */
```

## 八、高频 Bug 预警

| Bug | 后果 |
|-----|------|
| `int *fp(int,int)` 漏括号 | 声明成 **指针函数**，不是函数指针 |
| 混用不同签名 `fp` | 调用栈破坏、崩溃 |
| `fp++` | 编译报错 |
| `void*` 存函数指针 | 标准未定义，部分架构异常 |
| RPN `op[]` 下标未边界检查 | 跳错函数 |

## 背诵要点（精简版）

```
int *f()  指针函数；int (*fp)()  函数指针——括号决定
fp = add; 调用 fp(3,4) 或 (*fp)(3,4)
typedef 简化；op[] 表驱动替代 switch
类型必须匹配；void* 不通配函数指针
函数指针不能做 fp+1 算术
```
