# 5.7 多维数组

## 本节讲什么

二维数组 = **数组嵌套数组**：`int a[3][4]` 表示 `a` 是 **长度为 3 的数组**，每个元素又是 **长度为 4 的 `int` 数组**。内存 **整块连续、行优先（row-major）** 存储。本节吃透 **`a[i][j]` 双层展开**、**退化与 `sizeof`**、与 **指针数组模拟二维** 的本质区别。

**前置**：[5.3](./5.3-指针与数组.md)、[5.4](./5.4-地址算术运算.md)、[5.6](./5.6-指针数组以及指向指针的指针.md)（数组指针 `int (*p)[N]`）。

## 一、核心定义

```c
int a[3][4];
```

| 拆解 | 含义 |
|------|------|
| `a` | 含 **3 个元素** 的数组 |
| 每个元素 | 又是一个 **`int[4]`** 数组 |
| 内存 | **连续**，行优先：`a[0][0]…a[0][3]`, `a[1][0]…`, `a[2][0]…` |
| 无指针跳转 | 不是「行指针表」，就是一块扁平连续内存 |

```
行优先内存（教学用）

a[0][0] a[0][1] a[0][2] a[0][3] | a[1][0] … a[1][3] | a[2][0] … a[2][3]
└────────── 第 0 行 ──────────┘   └──── 第 1 行 ────┘   └──── 第 2 行 ────┘
```

### 下标等价底层推导（必背）

```c
a[i][j]
  = *(a[i] + j)
  = *(*(a + i) + j)
```

**分层拆解**（`int a[3][4]`）：

| 步骤 | 表达式 | 类型 / 含义 |
|------|--------|-------------|
| 1 | `a + i` | `a` 不退化时为 `int (*)[4]`；`+1` **跳过整行** 4 个 `int` |
| 2 | `*(a + i)` | 第 `i` 行 `int[4]`；表达式中退化为 **`&a[i][0]`**（`int *`） |
| 3 | `*(a + i) + j` | 行内偏移 `j` 个 `int` |
| 4 | `*(*(a + i) + j)` | 得到 **`a[i][j]`** 的值 |

> 外层按 **行** 跳（数组指针步长）；内层按 **元素** 跳（`int *` 步长）。见 [5.4](./5.4-地址算术运算.md)。

## 二、关键语法要点

### 1. `sizeof` 计算

```c
int a[3][4];
sizeof(a)       /* 3 × 4 × sizeof(int) = 整表字节 */
sizeof(a[0])    /* 4 × sizeof(int) = 单行 */
sizeof(a[0][0]) /* sizeof(int) = 单元素 */
```

### 2. 数组名退化 & 取地址

| 表达式 | 退化 / 类型 | `+1` 步长 |
|--------|------------|----------|
| `a` | `&a[0]`，`int (*)[4]` | 一整行（4×4=16 字节） |
| `&a` | `int (*)[3][4]` | **整个二维数组**（3 行总字节） |
| `a[i]` | `&a[i][0]`，`int *` | 1 个 `int`（4 字节） |
| `&a[0][0]` | `int *` | 1 个 `int` |

```c
/* 三者地址数值通常相等，类型与步长不同 */
arr == &arr[0] == &arr[0][0]   /* 首元素地址相同 */
```

### 3. 初始化

```c
int a[2][3] = {
    {0, 1, 2},
    {3, 4, 5}
};

int b[2][3] = {0, 1, 2, 3, 4, 5};   /* 平铺，编译器按行填 */
int c[2][3] = {1, 2};               /* 其余自动补 0 */
```

## 三、示例代码完整解析

```c
int a[2][3] = { {0, 1, 2}, {3, 4, 5} };

for (int i = 0; i < 2; i++)
    for (int j = 0; j < 3; j++)
        printf("%d ", a[i][j]);
/* 内存顺序：0, 1, 2, 3, 4, 5 */
```

**扁平指针遍历**（不用 `[i][j]`）：

```c
int *p = &a[0][0];
for (int k = 0; k < 2 * 3; k++)
    printf("%d ", *p++);
```

连续内存下，`&a[0][0]` 起 `p++` 等价按行扫过全部元素（[5.4](./5.4-地址算术运算.md) 同数组内合法）。

## 四、高频对比：多维数组 vs 指针数组模拟二维

| | **`int mat[3][4]`** 真二维数组 | **`int *mat[3]`** 指针数组 |
|---|-------------------------------|---------------------------|
| 内存 | **一块连续** 3×4 个 `int` | 3 个 `int*`，各行可 **分散** |
| 行宽 | 固定 4，编译期确定 | 各行 **可不等长**（锯齿） |
| `sizeof` | `3×4×sizeof(int)` | `3×sizeof(int*)`（只存指针） |
| 传参 | **`int (*row)[4]`** | **`int **mat`** |
| 典型场景 | 矩阵、帧缓冲、固定表 | 字符串表、`argv`、变长行 |

```c
/* 真二维 */
void f1(int (*row)[4], int rows);

/* 指针数组模拟 */
void f2(int **rows, int n);
```

> **面试经典**：内存布局、传参类型、`sizeof`、指针步长 **全部不同**，**不可混用**。`f2` 不能接 `int a[3][4]`。

详见 [5.9](./5.9-指针与多维数组.md) 系统对比。

## 五、底层工程场景

| 场景 | 用法 |
|------|------|
| **帧缓冲 / 显存** | `pixel = base + y * stride + x`；`stride` = 单行字节宽，行优先 |
| **嵌入式矩阵 / 传感器表** | 静态 `float samples[R][C]`，连续内存 cache 友好 |
| **ELF / 内核配置表** | 固定行列二维表，编译期分配，快速 `[i][j]` 索引 |

## 六、章节衔接 & Bug 预警

| 方向 | 内容 |
|------|------|
| 前置 | 5.3 退化、5.4 步长、5.6 `int (*p)[N]` |
| 后置 | [5.9](./5.9-指针与多维数组.md) 传参规范、指针遍历 |

| Bug | 后果 |
|-----|------|
| `int a[M][N]` 用 `int **` 接收 | 类型错，布局完全不同 |
| 混淆 **行步长** 与 **元素步长** | 画面错位、矩阵取值错 |
| `int *p = a` 接二维数组名 | 忽略 `int (*)[4]` 行指针语义 |
| 函数内 `sizeof` 形参当行数 | 得到指针宽 8 |

## 背诵要点（精简版）

```
int a[3][4]：3 个 int[4]，连续行优先
a[i][j] = *(*(a+i)+j)；外跳行、内跳元素
a → int(*)[4]；a[i] → int*；&a → int(*)[3][4]
真二维传 int(*)[N]；指针数组传 int**
&a[0][0] 可扁平 p++ 遍历
```
