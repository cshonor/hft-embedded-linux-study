# 8.5 实例：fopen 和 getc 函数的实现

## 本节讲什么

K&R 用 **fd + read + 用户缓冲** 手写简化版 **`fopen`/`getc`**，揭示第 7 章 stdio 的本质：**在系统调用之上加缓冲与状态**。

## 要点

### 典型 FILE 结构（概念）

```c
struct io_buf {
    int fd;           /* 底层文件描述符 */
    char *buf;        /* 缓冲 */
    char *bufp;       /* 当前读指针 */
    int n;            /* 缓冲中剩余字符数 */
    /* 错误/EOF 标志等 */
};
```

### getc 逻辑

1. 若缓冲有字符 → 直接返回
2. 否则 **`read(fd, buf, BUFSIZ)`**  refill
3. `read` 返回 0 → EOF；`-1` → 错误

### fopen 逻辑

1. **`open(path, flags, mode)`** 得 fd
2. 分配 `FILE` 结构与缓冲
3. 返回指针供 `getc`/`putc` 使用

## stdio vs 原生 read（重难点）

| stdio 带缓冲 | read 无 stdio 缓冲 |
|--------------|-------------------|
| 减少 syscall，吞吐高 | 单次延迟可控 |
| 交互式可能行缓冲 | 适合 HFT、自管 ring |
| `printf` 需 flush | 直接写内核 |
