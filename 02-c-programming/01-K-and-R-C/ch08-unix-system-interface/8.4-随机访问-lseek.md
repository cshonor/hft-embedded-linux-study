# 8.4 随机访问：lseek

## 本节讲什么

**`lseek`** 移动 fd 的**读写偏移**，对应第 7 章 **`fseek`/`ftell`** 的底层实现。随机读 ELF 头、tick 文件某偏移、大镜像分片加载。

## 要点

```c
#include <unistd.h>
off_t lseek(int fd, off_t offset, int whence);
```

| whence | 含义 |
|--------|------|
| `SEEK_SET` | 相对文件开头 |
| `SEEK_CUR` | 相对当前位置 |
| `SEEK_END` | 相对文件末尾 |

- 成功返回**新的绝对偏移**；失败 `-1`
- 普通文件支持随机 seek；**pipe/socket 一般不可 seek**

### 与 ftell/fseek

- stdio 在 `FILE` 结构里维护偏移 + 缓冲；**`fflush` 后** `lseek(fileno(fp), ...)` 才与内核一致
- 大文件：**`lseek64`** / **`off_t` 64 位**

## 示例

```c
lseek(fd, 0, SEEK_END);
off_t size = lseek(fd, 0, SEEK_CUR);
lseek(fd, offsetof(struct Elf64_Ehdr, e_phoff), SEEK_SET);
read(fd, &phoff, sizeof phoff);
```

## 重难点

- **文本 vs 二进制** 与 seek 无关，但 stdio 缓冲可能导致 **`ftell` 不准**
- 并发写同一文件偏移需应用层同步

## 与后续衔接

内存映射 **`mmap`** 是另一种「随机访问」；Miniload 解析 ELF program header 常 `lseek` + `read`。
