# 8.3 open、creat、close 和 unlink

## 本节讲什么

打开/创建/关闭文件，以及 **unlink** 删除目录项。配合 **link** 理解硬链接与引用计数。

## 要点

### open

```c
#include <fcntl.h>
int open(const char *path, int flags, /* mode_t mode */);
```

常用 **flags**（可 `|` 组合）：

| 标志 | 含义 |
|------|------|
| `O_RDONLY` | 只读 |
| `O_WRONLY` | 只写 |
| `O_RDWR` | 读写 |
| `O_CREAT` | 不存在则创建（需 `mode`） |
| `O_TRUNC` | 存在则截断为 0 |
| `O_APPEND` | 追加写 |

- 成功返回 **fd**；失败 **`-1`**

### creat

```c
int creat(const char *path, mode_t mode);
```

- 等价 **`open(path, O_WRONLY | O_CREAT | O_TRUNC, mode)`**

### close

```c
close(fd);
```

- 释放 fd 编号；内核在**引用计数为 0** 时释放文件对象
- 进程 **`exit`** 时关闭所有 fd（除 `FD_CLOEXEC` 等例外）

### unlink

```c
unlink(path);
```

- 删除**目录项**（文件名），**不立刻删磁盘数据**
- 硬链接计数减 1；为 **0** 且无进程打开该 inode 时才释放空间
- 文件仍被某 fd 打开时，unlink 后该 fd 仍可读写

### link（硬链接）

```c
link(oldpath, newpath);
```

- 两个目录项指向**同一 inode**
- 不能跨文件系统；目录一般不可硬链接

## 考点

- **unlink vs 删除数据**：计数与 fd 仍打开
- **O_CREAT** 必须写八进制 **mode**（如 `0644`），还受 **umask** 影响

## 与后续衔接

设备节点 `open("/dev/xxx")`；HFT 日志 `O_APPEND` 追加；ELF/镜像 `O_RDONLY`。
