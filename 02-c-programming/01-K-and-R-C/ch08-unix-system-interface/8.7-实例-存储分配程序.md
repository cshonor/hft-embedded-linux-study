# 8.7 实例：存储分配程序

## 本节讲什么

K&R 用 **`sbrk`/`brk`** 向内核申请堆空间，实现简化版 **`malloc`/`free`**（7.8.5），把 **用户态堆** 与 **系统接口** 连起来。

## 要点

### brk / sbrk

```c
#include <unistd.h>
void *sbrk(intptr_t increment);
```

- 移动 **program break**（堆顶）
- **`malloc`** 经典实现：维护空闲链表，不足时 **`sbrk`** 扩展
- 现代 glibc 用 **`mmap`** 大块分配，但模型仍相关

### 分配器设计要素

- **空闲块链表**（边界标记或链表头）
- **对齐**：返回指针满足 `max_align_t`
- **`free`** 合并相邻空闲块，减少碎片
- **线程安全**：生产 `malloc` 需锁；HFT 热路径用 **预分配池**

### 与第 6、7 章关系

- 6.5 链表节点在堆上
- 7.8.5 `malloc` 标准 API 底层即此类分配器 + 系统调用

## 底层场景

| 环境 | 堆 |
|------|-----|
| Linux 用户态 | brk + mmap |
| 内核 | kmalloc /slab |
| UEFI | AllocatePool |
| 裸机 / OS 实验 | 自管物理页 + 空闲链表 |

## 与后续衔接

理解「堆 = 虚拟地址空间一段 + 系统扩展 break」为读 **mmap**、**内存泄漏**、**ASLR** 打基础；与 fork 的 COW 共享页（章 README）同一内存世界观。
