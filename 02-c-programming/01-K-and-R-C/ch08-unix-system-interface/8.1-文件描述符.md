# 8.1 文件描述符

## 本节讲什么

**文件描述符（fd）** 是内核为进程打开的 I/O 对象分配的**小整数**。第 7 章 `FILE*` 标准流建立在 fd 之上；本章从 stdio **下沉到系统调用**，无用户态缓冲，延迟更低。

## 要点

### 标准 fd（固定）

| fd | 流 | 用途 |
|----|-----|------|
| 0 | stdin | 标准输入 |
| 1 | stdout | 标准输出 |
| 2 | stderr | 标准错误 |

- 进程启动时由 shell 或 init 打开；可 **重定向**（`>` `2>&1`）
- **`FILE *stdin`** 等是对 fd 0/1/2 的 **stdio 封装**（带缓冲）

### fd vs FILE*

| | 系统调用（fd） | stdio（FILE*） |
|---|----------------|----------------|
| 接口 | `read`/`write`/`open` | `fread`/`fprintf`/`fopen` |
| 缓冲 | 内核缓冲，用户态无 stdio 缓冲 | 用户态缓冲 |
| 场景 | HFT、驱动、精确控制 | 可移植工具、格式化 |

### dup / dup2（IO 重定向底层）

```c
dup(oldfd);              /* 复制 fd，返回新编号 */
dup2(oldfd, newfd);      /* 令 newfd 指向与 oldfd 同一打开文件 */
```

- Shell **`cmd > log`**：`dup2(fd, 1)` 把 stdout 绑到文件
- **`2>&1`**：stderr 合并到 stdout
- fork 后子进程**继承** fd 表，配合 pipe 做 IPC（见章 README）

## 底层场景

- **Linux 一切皆文件**：socket、epoll、设备节点也是 fd
- **HFT**：热路径 `read`/`write` 绕过 stdio
- **嵌入式**：`/dev/ttyS0` 打开得 fd，串口读写
