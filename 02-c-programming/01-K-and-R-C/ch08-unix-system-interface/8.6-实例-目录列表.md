# 8.6 实例：目录列表

## 本节讲什么

遍历目录、打印文件名，综合运用 **`open`/`read`/`stat`** 与第 6 章 **struct**。并覆盖 **stat 元数据**、**目录与权限** 等 UNIX 文件模型核心。

## 要点

### stat / fstat

```c
#include <sys/stat.h>
int stat(const char *path, struct stat *st);
int fstat(int fd, struct stat *st);
```

**`struct stat`** 常见字段：

| 字段 | 含义 |
|------|------|
| `st_size` | 字节大小 |
| `st_mode` | 类型 + 权限（`S_ISREG`/`S_ISDIR`/`S_ISCHR`…） |
| `st_mtime` | 修改时间 |
| `st_ino` | inode 号 |

- 配合第 6 章：**对齐、字段类型** 随 POSIX 平台

### 目录操作（用户 8.7）

| 调用 | 作用 |
|------|------|
| `mkdir(path, mode)` | 创建目录 |
| `rmdir(path)` | 删除空目录 |
| `chdir(path)` | 切换进程**当前工作目录** |
| `getcwd(buf, size)` | 读当前路径 |

- 配置加载、相对路径解析依赖 **cwd**

### 权限 chmod（用户 8.6）

```c
chmod(path, mode_t mode);
```

- 修改 permission bits；与 `open` 的 `mode` + **umask** 配合

### 目录读取

- 经典：读 **`read`** 目录 fd（老式）或 **`opendir`/`readdir`**（POSIX，K&R 后扩展）
- K&R **`fstat`** + 目录项结构解析 `ls` 简化版

## 示例逻辑（ls 思想）

```
对每个目录项：
  stat → 打印 mode、大小、名字
  目录则递归或标记
```

## 考点

- **硬链接** 同 inode 多名字（8.3 `link`）
- **设备文件** `S_ISCHR` / `S_ISBLK` —— 嵌入式访问驱动

## 与后续衔接

自制 OS 用户态 `ls`、`pwd`；Linux **`/proc`**、 **`/sys`** 遍历；UEFI 无 POSIX 目录 API，但「元数据 + 枚举」思维相同。
