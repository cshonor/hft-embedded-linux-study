# 8.2 低级 I/O：read 和 write

## 本节讲什么

**`read`/`write`** 是最底层的通用字节 I/O 系统调用，**绕过 stdio 缓冲**。高频交易、驱动、网络栈用户态封装的首选路径。

## 要点

```c
#include <unistd.h>

ssize_t read(int fd, void *buf, size_t n);
ssize_t write(int fd, const void *buf, size_t n);
```

### read

- 从 `fd` 最多读 **`n`** 字节到 **`buf`**
- 返回值：实际读到的字节数；**`0`** 表示 EOF；**`-1`** 出错（查 **`errno`**）
- **可能少于 `n`**：信号中断、管道无数据、接近 EOF 等 —— 循环读直到满足或 EOF

### write

- 写 **`n`** 字节；返回值实际写入数；`-1` 出错
- 大缓冲也需循环写直至写完

### 与 fread/fwrite 对比

| | read/write | fread/fwrite |
|---|------------|--------------|
| 缓冲 | 无 stdio 用户缓冲 | 有 |
| 文本 `\r\n` | 无自动转换 | Windows 文本模式可能转换 |
| 控制 | 每次 syscall 由应用决定 | 块大小与 flush 策略 |

## 示例

```c
char buf[4096];
ssize_t n;
while ((n = read(fd, buf, sizeof buf)) > 0)
    /* 处理 buf[0..n-1] */
if (n < 0)
    perror("read");
```

## 重难点

- **短 read / 短 write** 必须处理，不能假设一次读满
- 与 **第 7 章**性能差异：stdio 减少 syscall 次数；低延迟场景用 `read` + 自管缓冲

## 与后续衔接

socket、pipe、设备文件同一套 `read`/`write`；DPDK 绕过内核直达网卡，但 Linux 用户态 IO 仍从此起步。
