# 3.6 do-while 循环

## 本节讲什么

**先执行循环体，再判断条件**；条件为真则继续。与 `while` 的关键区别：**至少执行一次**。口语里可记 **「先斩后奏」** —— 不管条件一开始是真是假，**先把循环体跑一遍**，再决定要不要继续。适合「无论如何先干一遍，再根据结果决定要不要再来」的逻辑。

**前置**：[3.5 while 与 for](./3.5-while循环与for循环.md)、[3.2 非 0 即真](./3.2-if-else语句.md)。

## 要点

### 1. 语法格式

```c
do {
    循环体;
} while (条件);    /* 末尾分号不能少！ */
```

| 部分 | 说明 |
|------|------|
| **`do`** | 关键字，表示「先进入循环体」 |
| **`{ ... }`** | 循环体；多条语句 **必须用花括号** |
| **`while (条件)`** | 体执行 **完之后** 再判断 |
| **末尾 `;`** | `while (条件)` 之后 **必须有分号** —— 漏写是语法错误 |

单条语句 **也建议写 `{}`**，避免维护时加第二行却不在循环里（同 [3.1](./3.1-语句与程序块.md)）。

```c
do
    x++;
while (x < 10);   /* 合法但可读性差；生产代码多用 {} */
```

### 2. 执行顺序：「先斩后奏」

每轮：

1. **先执行循环体**（不问条件）
2. **再求值条件**
3. 非 0 → 回到步骤 1；0 → 退出

因此 **无论条件一开始是真是假，循环体至少跑 1 次** —— 先干活，后审批。

| 口诀 | 含义 |
|------|------|
| **`while`** | 先奏后斩 —— 条件不过，体 **一次都不执行** |
| **`do-while`** | 先斩后奏 —— **至少执行一次**，再判要不要继续 |

### 3. 与 `while` 对比

| | `while` | `do-while` |
|---|---------|------------|
| 判断时机 | 每次迭代 **之前** | 每次迭代 **之后** |
| 最少执行次数 | **0** | **1** |
| 典型场景 | 可能零次（已 EOF、已满足退出） | **先做一次，再决定要不要继续** |
| 语法 | `while (cond) stmt;` | `do { stmt; } while (cond);` |

```c
/* while：条件一开始就假 → 体零次 */
while (0)
    printf("never\n");

/* do-while：至少打印一次 */
do {
    printf("once\n");
} while (0);
```

### 4. 典型场景：菜单程序（至少显示一次）

用户说的经典用法：**先出菜单让用户选，再判断要不要退出** —— 菜单总得显示至少一遍。

```c
#include <stdio.h>

int choice;

do {
    printf("=== Menu ===\n");
    printf("1. New file\n");
    printf("2. Open file\n");
    printf("0. Quit\n");
    printf("Choice: ");
    scanf("%d", &choice);

    switch (choice) {
    case 1: new_file();  break;
    case 2: open_file(); break;
    case 0: break;
    default:
        printf("Invalid choice, try again.\n");
        break;
    }
} while (choice != 0);
```

若改成 `while (choice != 0) { ... }`，还得在循环 **之前** 先赋初值或先显示一次菜单，否则要么多写重复代码，要么第一次逻辑别扭。**`do-while` 把「至少显示一次」写进结构里。**

### 5. 密码验证：比 `while` 更顺手的例子

验证密码：**先让用户输入，再判断对不对；不对就再输** —— 第一次输入没有理由跳过，典型 **先斩后奏**。

```c
#include <stdio.h>
#include <string.h>

#define PASSWORD "secret123"

char buf[64];
int ok;

do {
    printf("Enter password: ");
    if (scanf("%63s", buf) != 1)
        buf[0] = '\0';
    ok = (strcmp(buf, PASSWORD) == 0);
    if (!ok)
        printf("Wrong password, try again.\n");
} while (!ok);

printf("Access granted.\n");
```

| 写法 | 别扭之处 |
|------|----------|
| **`do-while`** | 自然：读 → 判 → 错则继续 |
| **`while (!ok)`** | 进入循环前 `ok` 未定，得先 **人为** 设 `ok = 0` 或把 **第一次读密码** 写在循环外，逻辑拆成两段 |

```c
/* 用 while 也能写，但要多绕一步 */
int ok = 0;                    /* 仅为让 while 第一次能进体 */
while (!ok) {
    printf("Enter password: ");
    scanf("%63s", buf);
    ok = (strcmp(buf, PASSWORD) == 0);
    if (!ok)
        printf("Wrong password, try again.\n");
}
```

语义相同，但 **`do-while` 更贴「至少试一次」** —— 条件 `!ok` 放在末尾，表示「这次没通过，要不要再来」。

### 6. 还有哪些场景适合 `do-while`？

核心判据：**第一次操作没有理由跳过**。

| 场景 | 为何适合 `do-while` |
|------|---------------------|
| **交互菜单 / CLI 命令行** | 先展示选项，再根据用户输入决定是否继续 |
| **输入校验重试** | 先提示并读入，格式错了再让用户重输 | 
| **猜数 / 问答游戏** | 至少玩一轮，再问「继续吗？」 |
| **密码 / PIN 重试** | 至少尝试一次认证，失败后再循环 |
| **掷骰/抽奖直到满足条件** | 至少抽一次，再判断要不要继续 |
| **`scanf` / `fgets` 读入直到成功** | 先读，失败了（非数字、空行）再重试 |
| **至少处理一个数据单元** | 如链表至少前进一格、至少解析一个 token |
| **硬件/驱动初次探测** | 先发一次命令/读一次寄存器，再根据状态决定是否重试 |
| **游戏主循环（简化版）** | 至少渲染/更新一帧，再根据退出标志循环 |

#### 输入校验（K&R 风格）

```c
do {
    printf("enter a positive number: ");
} while (scanf("%d", &n) != 1 || n <= 0);
/* 至少问一次；scanf 失败或 n<=0 则继续 */
```

#### 猜数游戏

```c
do {
    printf("Guess (1-100): ");
    scanf("%d", &guess);
    if (guess < secret)
        printf("Too low\n");
    else if (guess > secret)
        printf("Too high\n");
} while (guess != secret);
/* 至少猜一次 */
```

#### 继续吗？（Y/N）

```c
char again;
do {
    play_round();
    printf("Play again? (y/n): ");
    scanf(" %c", &again);
} while (again == 'y' || again == 'Y');
```

### 7. 什么时候 **不该** 用 `do-while`？

| 场景 | 更合适 |
|------|--------|
| 文件可能已 EOF，一次都不用读 | `while ((c = getchar()) != EOF)` |
| 条件在循环前就能确定，可能 0 次 | `while` / `for` |
| 纯计数 `0 .. n-1` | `for` |
| 只是懒得写初值，其实可以 0 次 | 别滥用 `do-while` |

**原则**：不是因为「可以少写一行初始化」就用 `do-while`，而是语义上 **确实要先执行一次**。

### 8. 宏里的 `do { ... } while (0)`（了解）

内核/库代码常见 **宏包装多语句**：

```c
#define SAFE_FREE(p) \
    do { free(p); (p) = NULL; } while (0)
```

这里 **不是为了循环**，而是借 `do-while` 语法让宏在 `if/else` 后安全展开成 **单条语句**。与「至少执行一次」的循环语义不同，读到别混淆。

## 易混点

| # | 要点 |
|---|------|
| 1 | `} while (cond);` 的 **`;` 不能漏** |
| 2 | **先斩后奏**：先体后判，最少 **1** 次 |
| 3 | 菜单、**密码验证**、输入重试 = 经典场景 |
| 4 | 与 `while` 互换时常要多写/少一次「第一次」逻辑 |
| 5 | `do { } while (0)` 在宏里 ≠ 业务循环 |

## HFT / 内核提示

- **用户态配置工具、调试 CLI**：菜单、子命令选择适合 `do-while`
- **内核热路径**：极少用 `do-while` 做业务循环；更常见 `while` 或 `for`；`do { } while (0)` 多见于 **宏**
- **重试直到成功的冷路径**（打开设备、等握手）：语义上像 `do-while`，生产代码也常显式写 `while` + 第一次调用在循环外，可读性因团队而异
