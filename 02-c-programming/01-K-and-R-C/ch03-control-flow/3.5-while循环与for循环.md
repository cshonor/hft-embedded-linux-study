# 3.5 while 循环与 for 循环

## 本节讲什么

回顾第 1 章已用过的循环，系统梳理 **`while`** 与 **`for`** 的等价关系、**执行顺序**，以及条件里写 **`i++` / `++i`** 时的差异。遍历、计数、条件驱动三种场景如何选型。

**前置**：[3.1 块与 `{}`](./3.1-语句与程序块.md)、[3.2 非 0 即真](./3.2-if-else语句.md)、[2.8 自增/自减](../ch02-types-operators-expressions/2.8-自增运算符与自减运算符.md)。

## 要点

### 1. `while`：先判断，再执行

```c
while (条件)
    语句;
```

**执行顺序**（每轮）：

1. **先求值「条件」表达式**（含其中的副作用，如 `i++`）
2. 结果为 **非 0** → 执行 **循环体**
3. 回到步骤 1
4. 结果为 **0** → **跳出**，不再执行循环体

条件 **第一次就为假** 时，循环体 **一次都不执行**。

```c
int i = 0;
while (i < 3) {
    printf("%d ", i);
    i++;
}
/* 输出：0 1 2 */
```

### 2. 条件里写 `i++`：先用旧值比，再立刻自增

`while` 的「条件」是一个 **完整表达式**。若写成 `while (i++ < 10)`，**自增发生在「判断」这一步**，不是循环体里。

**后置 `i++` 规则**（见 [2.8](../ch02-types-operators-expressions/2.8-自增运算符与自减运算符.md)）：

1. 表达式 **返回值** = 自增 **前** 的 `i`
2. 整个子表达式求完后，`i` **立刻** +1

#### 逐步示例：`while (i++ < 10)`

设 `i` 当前为 **9**：

| 步骤 | 发生的事 |
|------|----------|
| ① 求 `i++` | 返回值 **9**；副作用 `i` 变为 **10** |
| ② 求 `9 < 10` | **成立** → 进入循环体 |
| ③ 循环体执行完 | 回到条件 |
| ④ 此时 `i` 已是 **10**；再求 `i++` | 返回值 **10**；`i` 变为 **11** |
| ⑤ 求 `10 < 10` | **不成立** → 循环结束 |

退出时 **`i` 为 11**（不是 10）—— 因为 **最后一次** 判断里 `i++` 仍会把 `10` 加成 `11`，尽管比较结果为假。

```c
int i = 0;
while (i++ < 10)
    printf("%d ", i);
/* 打印 1 2 3 ... 10 —— 共 10 次；结束时 i == 11 */
```

**记忆**：`while (i++ < n)` = 「用 **当前** `i` 去比，比完 **马上** `i+1`」；条件成立才进循环体。

### 3. 换成 `++i`：先自增，再判断 —— 逻辑不同

**前置 `++i`**：先 `i+1`，再用 **新值** 参与比较。

```c
int i = 0;
while (++i < 10)
    printf("%d ", i);
/* 打印 1 2 3 ... 9 —— 共 9 次；结束时 i == 10 */
```

| 条件写法 | 首次用于比较的 `i` | 典型循环次数（初值 0，界 10） | 退出时 `i` |
|----------|-------------------|-------------------------------|------------|
| `i++ < 10` | 旧值 0、1、… | **10 次** | **11** |
| `++i < 10` | 新值 1、2、… | **9 次** | **10** |
| `i < 10` + 体内 `i++` | 0、1、… | **10 次** | **10** |

**结论**：条件里塞 `++`/`--` 时，**前置与后置不等价**；工程上更常见 **`for (i = 0; i < n; i++)`** 或 **`while (i < n) { ...; i++; }`**，把「比较」与「步进」拆开，可读性更好。

#### 同一 `i = 9`：进不进循环，取决于写法

自增写在 **`while` 条件里** 时，**直接改变「判断那一刻」的 `i`**：

| 条件（`i` 当前为 9） | 用于比较的值 | 判断后 `i` | 本轮进不进循环体 |
|---------------------|-------------|-----------|-----------------|
| `i++ < 10` | **9**（旧值） | **10** | **进**（`9 < 10` 成立） |
| `++i < 10` | **10**（新值） | **10** | **不进**（`10 < 10` 不成立） |

```c
int i = 9;
while (i++ < 10)   /* 还能再进一轮 */
    body();
/* 判断用 9，i 变 10，进入 body */

int j = 9;
while (++j < 10)   /* 直接结束，体一次都不执行 */
    body();
/* 先 j=10，10<10 假，不进 body */
```

**`while` 把自增塞进条件** → `i++` 与 `++i` **改变判断逻辑、循环次数、退出时 `i`**，必须分开记。

### 4. `for`：自增在循环体**之后** —— `i++` 与 `++i` 无差别

```c
for (初始化; 条件; 步进)
    语句;
```

等价于：

```c
初始化;
while (条件) {
    语句;
    步进;
}
```

| `for` 特点 | 说明 |
|------------|------|
| 三部分 **任意可省略**，分号保留 | `for (;;)` = 无限循环 |
| **先判断、后步进** | 步进在循环体 **之后** 执行 |
| 初始化、步进可用逗号表达式 | `for (i = 0, j = n-1; i < j; i++, j--)` |
| 第三部分只求 **副作用** | `i++` 与 `++i` **等价** —— 判断逻辑与循环次数 **不变** |

`for` 每轮顺序：**判断条件 → 执行体 → 步进**。第三部分 `i++` / `++i` 只负责让 `i` 前进，**返回值不参与条件**，故两种写法 **完全等价**：

```c
for (i = 0; i < 10; i++)   /* 10 次，退出 i == 10 */
for (i = 0; i < 10; ++i)   /* 同上：次数、判断、终值都一样 */
```

| | **`while (i++ < 10)`** | **`for (i=0; i<10; i++)`** |
|--|------------------------|----------------------------|
| 自增发生时机 | **判断时**（条件表达式内） | **体执行完之后**（第三部分） |
| `i++` vs `++i` | **不等价**，改变次数与终值 | **等价**，只关心 +1 副作用 |
| 与 `i < 10` + 体内 `i++` | 不等价（时机不同） | **等价** |

因此 `for (i = 0; i < 10; i++)` 与 `while (i++ < 10)` **不等价**；与 `while (i < 10) { ...; i++; }` 才等价。

```c
/* 字符计数 — while */
while ((c = getchar()) != EOF)
    ++nc;

/* 同上 — for（步进在体后，体可为空语句） */
for (nc = 0; (c = getchar()) != EOF; ++nc)
    ;
```

### 5. 无限循环

```c
while (1) { ... }
for (;;) { ... }    /* 中间、第三部分皆空 */
```

内核线程、行情泵、事件循环常见；退出靠 `break`、`return` 或信号。

### 6. 选用建议

| 场景 | 倾向 | 原因 |
|------|------|------|
| 已知次数、下标遍历 | **`for`** | 初始化/条件/步进一目了然 |
| 读直到 EOF、等事件 | **`while`** | 条件在循环内才明确 |
| 条件含赋值 `while ((c = get()) != EOF)` | **`while`** | K&R 惯用写法 |
| 无限服务循环 | `while (1)` / `for (;;)` | 等价，风格问题 |
| 条件里写 `i++` / `++i` | **慎用** | 易 off-by-one；见 §2、§3 |

## 易混点

| # | 要点 |
|---|------|
| 1 | `while` **先判断**；假则体 **零次** |
| 2 | 条件里的 `i++`：**先用旧值比，比完立刻 +1** |
| 3 | `while (i++ < 10)` 退出时 `i` 常为 **11**（最后一次失败判断仍 `++`） |
| 4 | `i=9` 时：`while (i++<10)` **能进**；`while (++i<10)` **不进** |
| 5 | `for` 步进在 **体之后**；`i++`/`++i` **等价**，判断与次数不变 |
| 6 | `while (i++ < n)` 步进在 **判断时**，与 `for` **不等价** |
| 7 | 无 `{}` 时循环体只有 **紧跟的一条语句**（同 [3.1](./3.1-语句与程序块.md)） |

## HFT / 内核提示

- 热路径遍历：`for (size_t i = 0; i < n; ++i)` —— 边界与步进分离，利于审查与向量化
- `while (read_pkt(&hdr) == 0)`：条件驱动，比硬写 `for` 更贴语义
- 避免 `while (i++ < len)` 处理缓冲区 —— off-by-one 是安全漏洞温床；显式 `i < len` + 体内 `i++`
