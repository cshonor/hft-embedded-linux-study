# 3.2 if-else 语句

## 本节讲什么

根据条件选择执行路径：`if` 单分支、`if-else` 双分支。条件为 **表达式**：**非 0 为真、0 为假**（与 [2.6 关系/逻辑运算](../ch02-types-operators-expressions/2.6-关系运算符与逻辑运算符.md) 一致）。语法简单，但 **`else` 就近匹配**、**无 `{}` 只管住一条** 是经典易错点（见 [3.1 块](./3.1-语句与程序块.md)）。

## 要点

### 1. 形式

```c
if (表达式)
    语句;

if (表达式)
    语句1;
else
    语句2;
```

`if` / `else` 后面各跟 **一条语句** —— 可以是单条表达式语句，也可以是 **`{ 复合语句 }`**（打包多条）。

### 2. 真假规则：非 0 即真，只有 0 为假

`if` 判断的是表达式 **最终的值**，不是「是否为 1」：

| 条件 | 结果 |
|------|------|
| `if (5)`、`if (-3)` | **真** → 进 `if` 分支 |
| `if (0)` | **假** → 跳过 `if`，有则进 `else` |
| `if (x > y)` | 结果为 0/1，0 假 1 真 |

```c
if (5)
    printf("总是执行\n");

if (0)
    printf("永不执行\n");
else
    printf("走 else\n");
```

C89 无 `bool` 类型；C99 可用 `_Bool` / `<stdbool.h>`，但 **`if` 仍看 0 / 非 0**（[2.1.1 C99](../ch02-types-operators-expressions/2.1.1-C99标准与演进.md)）。

### 3. 指针作条件

| 指针 | 当 `if` 条件 |
|------|--------------|
| **非空** | **真** |
| **`NULL` / 空指针** | **假** |

```c
if (p)
    use(p);

if (!p)
    return -1;

if (p != NULL && p->len > 0)   /* 短路判空，见 2.6 */
    parse(p);
```

`if (p)` 与 `if (p != NULL)` 语义相同；团队统一风格即可。**非空 ≠ 指针合法**（野指针、悬垂仍可能非 0）。

### 4. 无花括号：只管住一条

```c
if (ok)
    step1();
    step2();    /* 不在 if 里！ */
```

有 `else` 时同样：只有 **`else` 前面那条** 属于 `if`，`else` 与 **最近未配对的 `if`** 绑定（见下节）。

### 5. `else` 就近匹配（悬空 else）——分步模拟

**规则**：`else` 与 **离它最近、尚未匹配 `else` 的 `if`** 配对；**不是**「视觉上最外层」的 `if`。

#### 5.1 无花括号嵌套：编译器实际看到的结构

```c
if (n > 0)              /* 外层 if —— 记为 IF_OUT */
    if (n < 100)        /* 内层 if —— 记为 IF_IN  */
        return n;
else                     /* 与谁配对？ */
    return -1;
```

**配对过程**（从 `else` 往上扫）：

1. 遇到 `else`，需要找「最近的、还没有 `else` 的 `if`」
2. 往上看：紧挨 `else` 上方是 `return n;`（属于 `IF_IN` 的体）
3. 再往上是 **`if (n < 100)`** → 这是 **IF_IN**，且 **尚未有 `else`**
4. **`else` 绑定 IF_IN**，**不是** `if (n > 0)`

编译器等价解析（缩进表达逻辑，不是写法要求）：

```c
if (n > 0)
    if (n < 100)
        return n;
    else                    /* else 属于内层 if */
        return -1;
/* 外层 if (n > 0) 没有 else！ */
```

#### 5.2 真值表：你以为 vs 实际

| `n` | 程序员常以为 | **实际执行** |
|-----|--------------|--------------|
| `50` | `return 50` | `return 50` ✓ |
| `150` | 可能走外层 `else` → `return -1` | **不进内层** `n<100`；内层 `else` 执行 → **`return -1`** ✓（此例碰巧一致） |
| `-5` | 外层假，跳过？ | **外层假**，不进任何 `if` 体；**`return -1` 不执行** → 函数继续往下走（与「外层 else」预期不同） |

**`n = -5` 是关键**：若你以为 `else` 属于外层「`n>0` 不成立就 -1`」，会错——**根本没有外层 `else`**，`-5` 时 **什么都不 return**。

#### 5.3 再加一层：else 永远找「最近」

```c
if (a)
    if (b)
        if (c)
            f();
        else
            g();    /* 匹配 if (c)，不是 if (a) 或 if (b) */
```

| `else` 绑定 | 不是 |
|-------------|------|
| **`if (c)`** | `if (b)`、`if (a)` |

#### 5.4 加花括号：明确「else 归谁」

意图：**仅当 `n > 0` 且不在 `(0,100)` 内才 -1** vs **只要 `n <= 0` 就 -1** —— 必须用 `{}` 写清：

```c
/* 意图：n<=0 走 else */
if (n > 0) {
    if (n < 100)
        return n;
} else {
    return -1;
}

/* 意图：n>0 但 n>=100 走 -1（内层 else） */
if (n > 0)
    if (n < 100)
        return n;
    else
        return -1;
```

第二种在 **`n > 0` 已成立** 的前提下，由 **内层** `else` 处理 `n >= 100`；**`n <= 0` 仍不会进 `return -1`**（除非外层另有逻辑）。

**工程习惯**：复杂嵌套 **哪怕只有一条语句也加 `{}`**，避免 `else` 挂错层。

#### 5.5 经典 K&R 示例（原文结构）

```c
if (n > 0)
    if (n < 100)
        return n;
else
    return -1;
```

与 §5.1–5.2 相同：**`else` → 内层 `if (n < 100)`**。

**改写**：外层加块，或 [3.3 else-if](./3.3-else-if语句.md) 扁平化：

```c
if (n > 0) {
    if (n < 100)
        return n;
} else {
    return -1;
}
```

### 6. 高频坑

| 坑 | 说明 |
|----|------|
| **`if (x = y)`** | 赋值，非比较；结果非 0 则永远进 `if`（[2.6](../ch02-types-operators-expressions/2.6-关系运算符与逻辑运算符.md) `=` vs `==`） |
| **浮点 `if (a == b)`** | 慎用；用 epsilon（[2.2.1 float.h](../ch02-types-operators-expressions/2.2.1-整型头文件选型总览.md)） |
| **无 `{}` 多行** | 第二行不在分支内 |
| **宏展开** | `if (x) MACRO();` 注意宏内多条语句需 `do { } while(0)` |

### 7. 示例

```c
if (x > y)
    printf("x is greater\n");
else
    printf("x is not greater\n");

if (mbuf != NULL) {
    if (mbuf->data_len > 0)
        decode(mbuf);
} else {
    stats.null_mbuf++;
}
```

## HFT / 内核提示

- 热路径：`if (unlikely(err))` 风格 —— 条件仍是 **非 0 真**
- 协议：`if (flags & RX_OK)` —— 位与结果非 0 为真；测单 bit 用 `(flags & MASK) != 0` 更清晰（[2.9](../ch02-types-operators-expressions/2.9-按位运算符.md)）
- **分支内始终 `{}`**，避免维护时插入第二行掉出 `if`
