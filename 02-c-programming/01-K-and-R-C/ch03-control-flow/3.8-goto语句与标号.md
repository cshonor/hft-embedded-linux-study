# 3.8 goto 语句与标号

## 本节讲什么

C **保留** `goto` 与**标号**（`label:`），可从任意处跳转到同一函数内的标号。K&R 承认其存在，但**不鼓励滥用**；规范场景是**多层循环统一出口**或**错误处理集中清理**。

## 要点

- 形式：`goto 标号;` 与 `标号: 语句`
- 标号作用域为**整个函数**；不能跨函数跳转
- 不能跳进另一个块并跳过变量初始化（C 规则限制，避免未初始化使用）
- 书中推荐用途：
  - 深层嵌套循环**一次跳出多层**
  - 错误路径 **`goto err;`**，在 `err:` 集中释放资源（内核常见模式）

## 合理 vs 滥用

| 合理 | 滥用 |
|------|------|
| 统一错误处理、资源清理 | 替代结构化 if/循环， spaghetti 代码 |
| 从多重循环 breakout | 正常业务分支全靠 goto |
| 有限状态机代码生成、手写汇编对应 | 跳来跳去难以维护 |

## 示例（错误集中处理）

```c
if (setup_a() < 0)
    goto fail;
if (setup_b() < 0)
    goto fail;
/* 正常路径 */
return 0;

fail:
    cleanup();
    return -1;
```

多层循环跳出也可 `goto done;`，比多个 flag + break 更清晰。

## 与后续衔接

Linux 内核大量 `goto out;` / `goto err_free;` 模式；读驱动、网络栈时会天天见到。掌握「标号 + 单向清理」比排斥一切 goto 更实用。
