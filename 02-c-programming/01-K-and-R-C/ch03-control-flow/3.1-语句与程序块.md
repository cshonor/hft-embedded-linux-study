# 3.1 语句与程序块

## 本节讲什么

C 程序由 **语句** 组成；多条语句可用花括号 `{}` **打包** 成一条 **复合语句（块）**。块既是 `if`、循环、函数体里「管多条语句」的手段，也定义 **块作用域** —— 块内声明的变量，出了 `{}` 就不能再用。

**后续**：[3.2 if-else](./3.2-if-else语句.md)、[1.10 作用域](../ch01-introduction/1.10-外部变量与作用域.md)、[2.4 声明](../ch02-types-operators-expressions/2.4-声明.md)。

## 要点

### 1. 语句与分号

- 多数语句以 **`;`** 结束：表达式语句、声明、空语句 `;`
- **控制流**（`if`、`for`、`while` 等）按各自语法，不都在末尾加单独分号代表「整条控制流结束」

### 2. 复合语句：把多条语句「打包」成一条

```c
{
    printf("A\n");
    printf("B\n");
}
```

**语法上 `{ ... }` 整体算一条语句**，因此可以写在只允许 **一条** 子语句的位置（`if`、`else`、`for` 体等）。

| 概念 | 说明 |
|------|------|
| **复合语句** | `{ 声明; 语句; ... }` |
| **作用** | 把多条语句合成 **一条**，交给 `if` / 循环管辖 |
| **无 `{}`** | 控制流只能管 **紧挨着的那一条** 语句 |

### 3. `if` 不加花括号：只管 **紧挨着的一条**

**规则**：`if (条件)` 后面若 **没有 `{`**，则只管辖 **语法上紧跟其后的那一条完整语句**；再下一行起，**已经不在 `if` 里**。

#### 什么叫「一条语句」？

| 算 **一条** | 不算「多行多条」 |
|-------------|------------------|
| `printf("ok\n");` | 两个带 `;` 的表达式语句 = **两条** |
| `return n;` | |
| **`{ ... }` 复合语句** | 块内可有 10 行，**整体仍算一条** |

```c
if (ok) {
    a();
    b();
}   /* 花括号把 a、b 打包成「一条」复合语句 */
```

#### 经典误读：第二行「看起来在 if 里」

```c
if (score >= 60)
    printf("及格\n");
    printf("发证书\n");   /* 不在 if 里！无论真假都会执行 */
```

**编译器分组**（等价写法）：

```c
if (score >= 60)
    printf("及格\n");
printf("发证书\n");        /* 与 if 同级，已脱离 */
```

缩进 **不改变** 语法；只有 **`{}`** 或「确实只有一条语句」才算在分支内。

#### 缩进不是语法（新手高频坑）

C **完全不看缩进**，只认 **语法结构**。第二条语句与 `if` 对齐得再整齐，**没有 `{}` 就不在 `if` 里**：

```c
if (score >= 60)
    printf("及格\n");
    printf("发证书\n");   /* 缩进对齐，仍在 if 外 */
```

| 误区 | 实际 |
|------|------|
| 「两行对齐了，都在 `if` 里」 | **错** —— 只认紧挨的一条 |
| Python 靠缩进表块 | C 靠 **`{}`** 和分号 |
| 多缩进 = 进分支 | **无效** |

**踩坑原因**：把 **缩进当语法**；读 C 要画 **`{}` / 分号** 的语法树，不能靠眼睛对齐。

```c
/* 乱缩进也一样：只有第一条归 if */
if (x)
printf("only this\n");
        printf("always runs\n");
```

#### 加花括号：两条都归 `if` 管

```c
if (score >= 60) {
    printf("及格\n");
    printf("发证书\n");   /* 只有及格才执行 */
}
```

#### `if-else` 时：各自也只搂住一条（无 `{}`）

```c
if (x > 0)
    printf("正\n");
else
    printf("非正\n");
printf("结束\n");          /* 三条分支语句执行完后都会执行 */
```

```c
if (x > 0)
    if (x < 10)
        printf("小正\n");
else
    printf("???\n");       /* else 只绑内层 if，见 3.2 §5 */
```

#### `while` / `for` 同理

```c
while (n-- > 0)
    sum += a[i++];
    i++;                   /* 不在 while 里！循环体实际只有 sum 那一行 */
```

```c
for (i = 0; i < n; i++)
    process(i);
    log(i);                /* 每次循环结束都会 log，不在 for 体内 */
```

**循环**与 **`if` 相同**：无 `{}` → 体部 = **紧挨着的一条**。

#### 工程习惯

| 建议 | 原因 |
|------|------|
| **分支/循环体一律 `{}`** | 防插入第二行掉出控制流 |
| 不依赖缩进表达结构 | 读代码的人、diff、宏展开都靠语法树 |

| | 无 `{}` | 有 `{}` |
|---|---------|---------|
| 管辖范围 | **下一条语句** only | 块内全部 |
| 多行时 | 第 2 行起 **易失控** | 意图明确 |

### 4. 块作用域：出了 `{}` 变量不可用

块内 **声明** 的标识符，从声明点到 **`}`** 可见；离开块后 **名字失效**（对象销毁，见 [1.10](../ch01-introduction/1.10-外部变量与作用域.md) 生命周期）。

```c
{
    int x = 1;
    printf("%d\n", x);
}
/* x 在此不可见，不可再使用 */

if (n > 0) {
    int i;
    for (i = 0; i < n; i++)
        /* ... */
}  /* i 仅在此 if 块内有效 */
```

| 项 | 说明 |
|----|------|
| **可见性** | 块内声明 → 块外 **不能引用** |
| **生命周期** | 进入块创建（若未 `static`），离开 `}` 销毁（自动存储） |
| **隔离** | 不同块可同名变量，互不影响 |

### 5. C89 vs C99 声明位置

| 标准 | 块内声明 |
|------|----------|
| **C89** | 须在块 **第一条语句之前** |
| **C99** | **用到再声明**；`for (int i = 0; …)` 合法（[2.1.1 C99](../ch02-types-operators-expressions/2.1.1-C99标准与演进.md)） |

### 6. 单独 `{}` 限制临时变量

```c
while (read_pkt(&hdr) == 0) {
    {
        uint32_t len = ntohl(hdr.len);  /* 仅本迭代需要 */
        if (len > MAX) break;
        process(len);
    }
}
```

热路径用块 **缩小变量作用域**，利优化与审查。

## HFT / 底层示例

```c
if (pkt_len < HDR_SZ)
    return -1;
    stats.drop++;   /* 坑：不在 if 内，每次循环都可能执行 */

/* 正确 */
if (pkt_len < HDR_SZ) {
    stats.drop++;
    return -1;
}
```
