# 3.4 switch 语句

## 本节讲什么

对 **整型控制表达式** 做 **精准常量匹配** 的多路分支：`switch (表达式)` + 若干 `case 常量:`。匹配后从该 `case` **继续向下执行**（**穿透 fall-through**），直到 **`break`**、`return`、`goto` 或 `switch` 结束。与 [3.3 else-if](./3.3-else-if语句.md) 互补——离散常量用 `switch`，区间/不等式用 `else if`。

**前置**：[3.1 块与 `{}`](./3.1-语句与程序块.md)、[3.2 非 0 即真](./3.2-if-else语句.md)。

## 要点

### 1. 基本形式

```c
switch (表达式) {
case 常量1:
    语句;
    break;
case 常量2:
    语句;
    break;
default:
    语句;
    break;
}
```

- **`switch (表达式)`**：控制表达式须为 **整型**（`char` 会提升为整型比较）
- **不能** 对 `float` / `double` 做 `switch`（C 标准）

### 2. `case` 后必须是整型常量表达式

**核心规则**：`case` 后面只能是 **整型类型的常量** —— 编译期就能确定的一个 **整数标号**。`switch` 做的是 **整型相等跳转**，不是通用多态分发。

| 可以写 | 不可以写 |
|--------|----------|
| **整数字面量** `case 1:`、`case -3:` | **变量** `case n:` |
| **字符常量** `case 'a':`（单引号 → ASCII 等 **整型码值**） | **浮点数** `case 1.0:`、`case 3.14:` |
| **`enum` 常量** `case ST_RUN:` | **字符串** `case "read":` |
| **宏常量** `#define OP 3` → `case OP:` | **运行时表达式** `case a + 1:`、`case x > 0:` |

归纳记忆：

- **数字** → 直接写
- **字符** → 单引号，编译期转成码值整数
- **除此之外**（变量、浮点、字符串、不等式）→ **都不能** 放 `case` 后面

**精准匹配常量值** —— `case` 标签必须是 **编译期可求值的整型常量**，不能是变量或运行时表达式。

```c
#define CMD_READ  1
enum { ST_IDLE, ST_RUN };

switch (op) {
case 'q':              /* 字符常量，按整型值匹配 */
    quit();
    break;
case CMD_READ:         /* 宏常量 */
    read();
    break;
case ST_RUN:           /* 枚举常量 */
    run();
    break;
default:
    break;
}
```

`char` 写 `'A'` 时，与 **提升后的整型值** 匹配（与 `int` 控制表达式一致）。

### 2.1 字符常量：单引号 + 单个字符 → 整型码值

**字符常量**就是用 **单引号** 括起来的 **一个字符**，例如 `'5'`、`'#'`、`'x'`。C 在编译期把它转成该字符在执行字符集里的 **数值**（常见环境即 **ASCII 码**）：

| 写法 | 含义 | 常见 ASCII 值 |
|------|------|---------------|
| `'a'` | 字母 a | **97** |
| `'5'` | **字符** 5（不是整数 `5`） | **53** |
| `'#'` | 井号 | **35** |
| `'\n'` | 换行（**转义字符** 也算字符常量） | **10** |

```c
'a' == 97;     /* 常见 ASCII */
'5' == 53;     /* 字符 '5' ≠ 整数字面量 5 */
'\n' == 10;    /* 换行符的码值 */
```

`getchar()` 读到的是码值（存 `char` 也会提升为 `int`），`switch (c)` 实际比较的是 **整型相等**——`case 'a':` 在 ASCII 下等价于 `case 97:`。**正因为字符常量本质是整数，`switch` 才能用 `case` 匹配字符。**

详见 [2.3 字符常量](../ch02-types-operators-expressions/2.3-常量.md)。

### 2.2 单引号 vs 双引号：`case` 只能用前者

| 写法 | 类型含义 | 能否 `case` |
|------|----------|-------------|
| **`'a'`** 单引号 | **单个字符** 的整型常量（`int` 字符值） | **可以** `case 'a':` |
| **`"a"`** 双引号 | **字符串**（`char` 数组 / `char *`，含 `'\0'`） | **不能** `case "a":` |

```c
switch (c) {
case 'a':              /* OK：字符常量，整型值 97 等 */
    handle_a();
    break;
case '\n':             /* OK：转义字符常量，码值 10 */
    handle_newline();
    break;
/* case "a":           非法：case 标签须整型常量，字符串不是 */
}
```

**记忆**（与 [1.1 入门](../ch01-introduction/1.1-入门.md)、[1.9 字符数组](../ch01-introduction/1.9-字符数组.md)、[2.3 常量](../ch02-types-operators-expressions/2.3-常量.md) 一致）：

- **单引号** `'x'` → **一个字符** 的常量，本质是 **整数**
- **双引号** `"x"` → **字符串**（至少 `x` + `'\0'`），是 **地址 / 数组**，不是编译期整型标号

`switch` 做的是 **整型相等跳转**，不是比字符串内容，故 **`case` 不支持字符串**。

### 2.3 字符串多分支：只能用 `else if` + `strcmp`

对 **命令行参数、协议字段名、配置 key** 等 **字符串** 分支，用 **比较函数** 链：

```c
#include <string.h>

if (strcmp(cmd, "read") == 0) {
    do_read();
} else if (strcmp(cmd, "write") == 0) {
    do_write();
} else if (strcmp(cmd, "quit") == 0) {
    do_quit();
} else {
    fprintf(stderr, "unknown: %s\n", cmd);
}
```

| 方式 | 适用 |
|------|------|
| **`switch` + `case 'q'`** | 单个 **字符**（如 `getchar()`、快捷键） |
| **`else if` + `strcmp`** | **完整字符串** 多路匹配 |
| **哈希表 / 完美哈希**（进阶） | 大量字符串命令，冷路径配置 |

**注意**：`strcmp` 返回 **0 表示相等** —— 要写成 `strcmp(a, b) == 0`，不能 `if (strcmp(a, b))` 当相等（非 0 为真）。

```c
/* 错：相等时 strcmp 返回 0，条件为假 */
if (strcmp(s, "ok"))

/* 对 */
if (strcmp(s, "ok") == 0)
```

### 2.4 C `switch` 限制多 —— 与其他语言对比

C 的 `switch` 是 **整型常量表跳转**，设计年代早、语义窄，**不像很多现代语言** 那样支持多种 `case` 类型或模式匹配：

| 语言 | `switch` / 多分支能力（概览） |
|------|------------------------------|
| **C** | 仅 **整型控制表达式** + **整型常量 `case`**；字符靠码值；字符串/浮点不行 |
| **C++** | 同 C；C++17 起可选 `enum class` 更严，本质仍是整型标号 |
| **Java** | Java 7+ 可 `switch (String)`；`enum`；新版本有 `switch` 表达式 |
| **Go** | `switch` 无表达式也可；`case` 可为 **表达式**；另有 **type switch** |
| **Rust** | 用 **`match` 模式匹配**，支持枚举、范围、绑定等，远比 C `switch` 强 |
| **JavaScript** | `case` 可写 **字符串**；比较用 `===`（仍有限制，但比 C 宽） |
| **C#** | 可 `switch` **string**；新版本 **模式匹配** |

**工程含义**（HFT / 内核仍大量用 C）：

- 协议 **opcode、状态字、`enum`** → C `switch` 很合适
- **命令行子命令、配置 key** 等字符串 → 老老实实 **`else if` + `strcmp`**（或表驱动）
- 需要「按类型分发」→ C 没有 type switch，用 **函数指针表 / 多态**（C++ 另论）

C 的限制不是缺陷，而是 **语义简单、可预测** —— 编译器易生成 **跳转表**，热路径开销低；代价是 **表达能力** 不如 Rust `match` / Go `switch`。

### 3. 穿透（fall-through）：匹配后一路往下

匹配到某个 `case` 后，从 **该标号下第一条语句** 开始执行，**不会自动只执行本 case 就停** —— 必须靠 **`break`** 跳出，否则会 **掉进下一个 case**。

```c
switch (n) {
case 1:
    printf("one\n");    /* 若缺 break */
case 2:
    printf("two\n");    /* n==1 时也会执行到这里 */
    break;
}
```

`n == 1` 输出：

```
one
two
```

| 结束穿透的方式 | 说明 |
|----------------|------|
| **`break;`** | 跳出整个 `switch` |
| **`return` / `goto`** | 离开函数或跳转 |
| **`switch` 右花括号 `}`** | 自然结束 |

**无匹配** 且无 `default`：整个 `switch` 体 **不执行**。

### 4. 故意穿透 vs bug

**刻意合并** 多个 case 共用一段代码（K&R 数字分类）：

```c
switch (c) {
case '0': case '1': case '2': case '3': case '4':
case '5': case '6': case '7': case '8': case '9':
    digit = c - '0';
    break;
case EOF:
    break;
default:
    digit = -1;
    break;
}
```

**忘记 `break`** 在业务代码里多为 **bug**；若故意穿透，**必须注释** `/* fall through */`。

### 5. `default`

```c
default:
    handle_unknown();
    break;
```

- 可选；处理 **所有未列出的值**
- 习惯放 **末尾**（可读性）；语法上位置不限

### 6. `switch` vs `else if`：怎么选？

不是在「谁更高级」之间纠结，而是看 **条件形状**：

| 场景 | 更合适 | 原因 |
|------|--------|------|
| **离散整型常量**（状态码、opcode、`enum`、ASCII） | **`switch`** | 常量表清晰；编译器可优化跳转表 |
| **区间** `n < 0` / `n > 100` | **`else if`** | `case` 不能写不等式 |
| **浮点、字符串比较** | **`else if`** | 不能 `switch` 浮点 / 字符串 |
| **多条件组合** `a > 0 && b < 10` | **`else if`** 或拆 `if` | `case` 只比 **相等** |
| **分支内逻辑差异大** | 皆可 | `switch` 利于扫一眼所有状态码 |
| **分支要按优先级第一个命中** | **`else if`** | 自上而下，第一个真即停 |

```c
/* else if：区间、符号 */
if (n < 0)
    sign = -1;
else if (n == 0)
    sign = 0;
else
    sign = 1;

/* switch：状态机状态字 */
switch (state) {
case ST_IDLE:   idle();   break;
case ST_RUN:    run();    break;
case ST_ERROR:  error();  break;
default:        break;
}
```

**HFT / 协议**：报文 **type 字段**、**msg id** → `switch`；**价格区间分档** → `else if`。

### 7. 与 `if` 相同的 `{}` 习惯

`case` 后若跟多条语句，用 **`{}` 块**（并产生块作用域）：

```c
case OP_RESET: {
    int tmp = read_reg();
    reset(tmp);
    break;
}
```

单条语句可不加 `{}`；多条 **建议加块**，与 [3.1](./3.1-语句与程序块.md) 一致。

## 易混点

| # | 要点 |
|---|------|
| 1 | **`case` = 整型常量**：数字直写、字符单引号转码值；**变量/浮点/字符串都不行** |
| 2 | 匹配后 **默认穿透**，要停须 **`break`** |
| 3 | 字符常量 `'a'`、`'\n'` 本质是 **整型码值**（如 97、10），故可 `case` |
| 4 | 字符 `case 'a':` 合法；**字符串 `case "a":` 非法** |
| 5 | 字符串多分支 → **`else if` + `strcmp`**，见 §2.3 |
| 6 | C `switch` 比 Java/Go/Rust 等 **限制多** —— 见 §2.4 |
| 7 | 无 `default` 且无匹配 → **什么都不做** |
| 8 | 重复 `case` 值 → **编译错误** |
| 9 | 缩进不影响语义（同 3.1） |

## HFT / 内核提示

- **IOCTL / 协议 opcode**：`switch (cmd)` + `default: -EINVAL`
- **故意穿透**：同类事件合并处理，**注释 fall through**
- 热路径状态机：表驱动（函数指针数组）有时比大 `switch` 更 cache 友好——ch05 后再抽象
