# 1.1.2 `printf` 格式、安全与工程规范

## 本节讲什么

`printf` 格式化输出的 **完整注意事项**：语法前提、**`%` 占位符与类型匹配**、转义字符、**缓冲区刷新**、**格式化漏洞**、宽度精度、C99 `%zd`、内核/DPDK 日志习惯。承接 [1.1 入门](./1.1-入门.md) 与 [附录 B.1 `<stdio.h>`](../appendix-b-standard-library/B.1-输入与输出stdio.h.md)。

## 一、基础前提

```c
#include <stdio.h>   /* 必须；否则无 printf 声明 */
```

| 项 | 说明 |
|----|------|
| **原型** | `int printf(const char *format, ...);` |
| **返回值** | 成功输出的 **字符数**；可接收可忽略 |
| **无头文件** | 旧编译器可能仅警告；**C99 起** 无声明使用 → 错误 |

```c
int n = printf("hi\n");   /* n 常为 3 */
```

## 二、格式控制符（`%` 系列 — 极易踩坑）

### 1. 类型必须匹配

| 变量类型 | 正确占位符 | 常见错误 |
|----------|-----------|----------|
| **`int`** | `%d` | `long` 用 `%d` → 异常 |
| **`long`** | `%ld`（`ll` → `%lld`） | `printf("%d", 1L)` |
| **`unsigned`** | `%u` | 负数用 `%u` → 超大正数 |
| **`float`** | **`%f`**（`float` 提升为 `double` 传参） | — |
| **`double`** | `%f` / `%e` / `%g` | **`int` 用 `%f`** → 乱数 |
| **`char`** | `%c` | 勿用 `%s` 打单字符 |
| **字符串 `char*`** | `%s` | 须 **以 `\0` 结尾** 的有效串 |
| **指针** | `%p` | 打印地址 |

```c
int num = 3;
printf("%f", num);   /* 错：类型不匹配，UB/乱码 */
printf("%d", num);   /* 对 */
```

与 [1.2 `5/9`](../1.2-变量与算术表达式.md)、[2.7 类型转换](../ch02-types-operators-expressions/2.7-类型转换.md) 衔接。

### 2. `%s` 高危

| 坑 | 后果 |
|----|------|
| **`printf("%s", NULL)`** | **段错误** |
| 无 **`'\0'`** 的 `char[]` | 越界读，垃圾字符 |
| 未初始化指针 | 同上 |

### 3. 输出字面量 `%` —— 写 `%%`

```c
printf("进度50%%\n");   /* 输出：进度50% */
printf("进度50%\n");    /* 错：格式串不完整 / 缺参数 */
```

## 三、转义字符

| 转义 | 含义 |
|------|------|
| `\n` | 换行（推荐；触发 stdout **行缓冲** 刷新） |
| `\t` | 制表 |
| `\\` | 反斜杠 |
| `\"` | 双引号 |

```c
printf("引号：\"hello\"\n");
```

见 [2.3 字符常量](../ch02-types-operators-expressions/2.3-常量.md)。

## 四、缓冲区刷新（底层高频坑）

**`stdout` 默认行缓冲**（连终端时）：

| 何时输出 | 条件 |
|----------|------|
| 自动刷新 | 遇到 **`'\n'`**、缓冲区满、程序 **正常退出** |
| 不自动 | 无 `\n` 的 `printf` 可能 **暂留缓冲** |
| 强制 | **`fflush(stdout);`** |

```c
printf("等待1秒");   /* 可能暂时看不到 */
fflush(stdout);      /* 立刻显示 */
sleep(1);
```

**`stderr`**：**无缓冲**倾向 —— 报错宜走 `stderr`（见 §八）。

文件重定向、全缓冲场景见 [ch07](../ch07-input-and-output/)。

## 五、安全风险（禁止滥用）

### 1. 格式化字符串漏洞

```c
char buf[128];
/* 假设 buf 来自用户输入 */
printf(buf);         /* 危险！buf 中含 %n %x 可写内存 */
printf("%s", buf);   /* 安全：格式串固定 */
```

**规则**：**格式串** 必须是 **程序员控制的字面量** 或可信常量；**用户数据只做 `%s` 等参数**。

### 2. 长度限制

```c
printf("%.10s", str);   /* 最多打印 10 个字符，防超长 */
```

## 六、宽度、对齐、精度

| 写法 | 效果 | 示例输出 |
|------|------|----------|
| `%5d` | 宽度 5，右对齐，空格填充 | `   12` |
| `%05d` | 宽度 5，不足 **前导 0** | `00012` |
| `%-5d` | **左对齐** | `12   ` |
| `%.2f` | 保留 2 位小数 | `3.14` |

```c
printf("%05d\n", 12);    /* 00012 */
printf("%.2f\n", 3.14159); /* 3.14 */
```

K&R 温度表：`printf("%3d %6.1f\n", fahr, celsius);` — 字段宽度对齐（[1.2](../1.2-变量与算术表达式.md)）。

## 七、C99 / 可移植补充

```c
size_t len = sizeof(int);
printf("%zd\n", len);    /* C99：size_t；勿用 %d */
```

| 类型 | 推荐格式 |
|------|----------|
| `size_t` | `%zu` 或 `%zd`（有符号场景少见） |
| `int32_t` / `int64_t` | [`<inttypes.h>` `PRId32`/`PRId64`](../ch02-types-operators-expressions/2.2.1-整型头文件选型总览.md) |
| `uintptr_t` | `%#" PRIxPTR` 或 `%p`（指针） |

打印 VLA 或串时，**长度变量** 须合法，避免越界 `%.*s` 误用。

## 八、工程规范（内核 / DPDK / HFT）

1. **调试前缀**：`printf("[DEBUG] len=%zd\n", len);` — 便于 grep
2. **热路径少 `printf`** — IO 开销大；正式版用分级日志、环形缓冲、条件编译 `DEBUG`
3. **错误走 stderr**：

```c
fprintf(stderr, "错误：内存分配失败\n");
```

4. 低延迟路径：避免 stdio；用 **ch08 `write(2)`** 或专用日志框架

## 九、常见编译 / 运行问题

| 原因 | 现象 |
|------|------|
| 无 `#include <stdio.h>` | 隐式声明 / 编译错误 |
| 占位符 **个数** ≠ 实参 | UB、警告 |
| 占位符 **类型** 不匹配 | 乱码、崩溃 |
| 格式串缺 `"`、参数 `%` 未写 `%%` | 编译错误 |
| `%s` + NULL / 无 NUL | 运行时段错误 / 乱码 |
