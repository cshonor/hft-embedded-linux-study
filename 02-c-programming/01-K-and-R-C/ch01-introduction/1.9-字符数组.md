# 1.9 字符数组

## 本节讲什么

K&R 第 1 章 **C 风格字符串** 入门。

**核心结论**：C 语言标准 **没有内置 `string` 类型**，不存在单独的字符串关键字；日常说的「字符串」全是 **`char` 字符数组 / `char*` 字符指针** + 末尾 **`'\0'`** 的约定写法，靠 `<string.h>`（ch7）或手动循环完成操作。

核心模型：

> **字符串 = 以 `'\0'`（NUL，空终止符）结尾的 `char` 一维数组**

配套完整示例：**读取标准输入所有行，找出并输出最长一行**。本节约束：

- 仅使用 **固定大小静态字符数组**
- **不使用** 动态内存分配（`malloc`）
- **不引入** `<string.h>` —— 读写、拷贝、长度统计 **全部手动实现**

承接 [1.5 `getchar` 流式 I/O](./1.5-character-io/1.5-字符输入输出.md)；与 [2.3 字符串字面量](../ch02-types-operators-expressions/2.3-常量.md) 衔接。

## 要点

### 1. C 无 `string` 类型 vs C++ `std::string`

| | **C** | **C++** |
|---|-------|---------|
| 原生字符串类型 | **无** | 有 **`std::string`** 封装类 |
| 文本本质 | 以 **`'\0'`** 结尾的连续 **`char`** 字节序列 | 类对象，内部自动管理缓冲与长度 |
| 两种载体 | `char buf[N]`（数组）、`char *p`（指针） | `std::string s = "hello";` |
| 操作方式 | 手动 `'\0'`、循环，或 ch7 `<string.h>` | `.size()`、`+`、`find` 等成员函数 |

```c
char str[20] = "hello";   /* 字符数组：栈上，一般可改 */
char *p = "hello";        /* 指针：常指向只读字面量 */
```

**设计理念**：C 只提供 `char`/`int`/`float` 等基础类型；字符串 = 连续字节，复用 `char` 即可统一处理 **文本、二进制、网络包**（见 §8）。

### 2. C 字符串强制规则：`'\0'` 是唯一识别标准

```c
char arr[] = "abc";   /* 底层：'a' 'b' 'c' '\0' */
```

- 编译器对 **字符串字面量** 自动补 **`'\0'`**
- **无 `'\0'`** → 只是普通字节数组，**不能** 当字符串传给 `printf("%s")`、`strlen` 等 —— **越界乱码**

| 写法 | 是否 C 字符串 |
|------|---------------|
| `char a[] = "abc";` | **是**（4 字节含 NUL） |
| `char a[5] = {'h','e','l','l','o'};` | **否**（无 NUL） |
| `getline` 末尾 `s[i]='\0'` | **强制变成** 合法字符串 |

### 3. 自动补 `\0` vs 手动写 `\0`

#### 一、编译器自动追加（不用手动写）

**场景 1：双引号字面量初始化 `char` 数组**

```c
char arr[] = "abc";   /* 实际：'a' 'b' 'c' '\0'，长度 4 */
```

```c
char buf[10] = "hi";  /* 'h' 'i' '\0' 0 0 … 剩余位填 0 */
```

**场景 2：字符串字面量赋给 `char*`**

```c
char *p = "test";     /* 只读区 "test" 自带 '\0' */
```

**规则**：代码里只要写 **双引号 `"xxx"`**，编译器 **自动** 在字面量末尾加 **`'\0'`**。

#### 二、必须手动补 `\0`（编译器不会自动补）

**场景 1：大括号逐个字符初始化**

```c
char arr1[3] = {'a','b','c'};              /* 错：无 \0，不是字符串 */
char arr2[4] = {'a','b','c','\0'};         /* 对：手动加终止符 */
```

**场景 2：循环 / 下标逐个赋值**

```c
char str[20];
str[0] = 'h'; str[1] = 'e'; str[2] = 'l'; str[3] = 'l'; str[4] = 'o';
str[5] = '\0';   /* 不写 → printf/strlen 越界乱码 */
```

本章 **`getline`** 读入后 **`s[i] = '\0'`** 即此类 —— 从 `getchar` 拼出的字节流 **不会** 自动变字符串。

**场景 3：文件 / 网络 / 键盘读入缓冲**

有效数据末尾须 **自己** 写 `'\0'`，否则 `<string.h>` 与 `printf("%s")` 找不到结束位置。

#### 三、一句话 + 易错对照

| 初始化方式 | 谁补 `\0` |
|------------|-----------|
| **`"xxx"` 双引号** | **编译器自动** |
| **`'x'` 逐个写 / 大括号枚举** | **必须手动** |

```c
char s1[] = "hello";                        /* A：自动 \0，6 字节 */
char s2[] = {'h','e','l','l','o'};            /* B：无 \0，不能 strlen/printf */
char s3[] = {'h','e','l','l','o','\0'};      /* C：手动 \0，等价 A */
```

### 4. 两种载体：`char[]` vs `char*`（本章 vs ch5）

#### `char buf[N] = "xxx";` — 字符数组

- 内存多在 **栈**（或静态区），内容 **可读可写**
- 长度须 **预留 1 字节** 给 `'\0'`

```c
char str[6] = "hello";   /* h e l l o \0 — 刚好 6 字节 */
str[0] = 'H';            /* 合法 */
```

本章最长行 `line[MAXLINE]`、`longest[MAXLINE]` 均属此类。

#### `char *s = "xxx";` — 字符指针

- `"hello"` 在 **只读常量段**（`.rodata`）；`s` 存 **首字符地址**
- **禁止** 改内容：`s[0] = 'H'` → **未定义行为**，常 **段错误**

```c
char *s = "hello";
s[0] = 'H';   /* UB，可能崩溃 */
```

| | `char buf[N]` | `char *p = "..."` |
|---|---------------|-------------------|
| 典型存储 | 栈 / 静态数组 | 指针在栈，串在只读区 |
| 可修改内容 | **是** | **否**（指向字面量时） |
| 本章角色 | **主战场**（`line`、`longest`） | ch5 深入；`argv[]` 预告 |

### 5. C 字符串底层存储规则

字符串字面量 `"hello"` 编译器自动在末尾追加 `'\0'`，内存实际 **6 字节**：

```text
'h'  'e'  'l'  'l'  'o'  '\0'
```

| 规则 | 说明 |
|------|------|
| **`'\0'`** | ASCII 值 **0**，字符串 **结束标记** |
| **遍历/打印** | 读到 `'\0'` 停止 — `printf("%s")`、`copy` 都依赖它 |
| **无 NUL 不是字符串** | `char arr[5] = {'h','e','l','l','o'};` **没有** `'\0'` → `printf("%s", arr)` **越界乱码** |

对比 [2.3](./../ch02-types-operators-expressions/2.3-常量.md)：字面量 `"hello"` 只读；本章 `char line[MAXLINE]` **可读写**。

### 6. 字符数组读写特性

1. **可读写**：`line[i] = 'x'` 自由改单个字符
2. **遍历**：循环直到 `'\0'`、换行 `'\n'` 或 `EOF`（读入时）
3. **边界**：固定长度 `MAXLINE`，超出 → **缓冲区溢出**（本章截断防溢出）

### 7. 最长行程序：算法思路

```text
循环读每一行
  → getline 读入 line[]，返回长度 len
  → 若 len > max：更新 max，copy(line → longest)
输入结束 → 打印 longest
```

| 缓冲区 | 作用 |
|--------|------|
| `line[MAXLINE]` | **临时**，每读一行覆盖 |
| `longest[MAXLINE]` | **持久**，仅出现更长行时覆盖 |

### 8. 完整源码（无 `string.h`、无动态分配）

> **注意**：下文 `getline` 是 **K&R 自定义函数**，不是 POSIX `getline(3)`。

```c
#include <stdio.h>

#define MAXLINE 1000   /* 单行最大字符数（含 '\n'，不含末尾 '\0' 占位逻辑见下） */

int getline(char line[], int lim);
void copy(char to[], char from[]);

int main(void)
{
    int len;
    int max = 0;
    char line[MAXLINE];
    char longest[MAXLINE];

    while ((len = getline(line, MAXLINE)) > 0) {
        if (len > max) {
            max = len;
            copy(longest, line);
        }
    }

    if (max > 0)
        printf("最长行：%s", longest);

    return 0;
}

/* 读一行存入 s，返回该行字符总数（含 '\n' 若存在） */
int getline(char s[], int lim)
{
    int c, i;

    for (i = 0; i < lim - 1 && (c = getchar()) != EOF && c != '\n'; ++i)
        s[i] = c;

    if (c == '\n') {
        s[i] = c;
        ++i;
    }

    s[i] = '\0';
    return i;
}

/* 手动拷贝：逐字符直到源串 '\0'（含 '\0' 一并拷贝） */
void copy(char to[], char from[])
{
    int i = 0;

    while ((to[i] = from[i]) != '\0')
        ++i;
}
```

```bash
gcc -Wall -Wextra -std=c11 -o longest_line longest_line.c
printf "short\nthis is longer\n" | ./longest_line
```

### 9. 关键函数与变量说明

**`getline(char s[], int lim)`**

- 循环上限 **`lim - 1`**：预留 1 字节给 **`'\0'`**，防溢出
- 若读到 **`'\n'`**，**一并存入** 数组（K&R 风格行含换行符）
- 强制 **`s[i] = '\0'`**，保证合法 C 字符串
- **返回值 `i`**：本行总字符数（**含** `'\n'`）
  - 空行仅 `"\n"` → 长度 **1**
  - `EOF` 且无字符 → **0**，`while` 结束

**`copy(char to[], char from[])`**

- **逐字符** `to[i] = from[i]`，直到拷贝到 **`'\0'`**
- 不能写 `to = from` —— 数组名是 **地址常量**，**不能整体赋值**

**`#define MAXLINE 1000`**

- 单行容量上限；超长输入 **截断**，尾部丢失（易错点 §）

### 10. 形参 `char line[]` 是什么？

本章写法 `char line[]`、`char s[]` 表示 **字符数组形参**；与实参 `line[MAXLINE]` 配合时，函数内按数组下标读写。  
**数组与指针的细微差别** 留 [ch05 指针与数组](../ch05-pointers-and-arrays/) —— 现阶段按「固定缓冲 + 下标」理解即可。

### 11. `<string.h>` 预告（ch7，本章不用）

无原生 `string` 类型，标准库用函数操作 **`char` 串**，**全部依赖 `'\0'` 判界**：

| 函数 | 作用 |
|------|------|
| `strlen(s)` | 有效字符数（**不计** `'\0'`） |
| `strcpy` / `strncpy` | 拷贝 |
| `strcat` | 拼接（须保证目标缓冲够大） |
| `strcmp` | 字典序比较 |

本章 **`copy` / `getline`** 即这些函数的 **手动版**，为理解边界与 NUL 铺垫。

### 12. 为何不单独造 `string` 类型？

1. **极简设计**：复杂结构用数组、指针、结构体搭建
2. **统一字节模型**：文本与二进制都是 `char` 序列 — 文件 I/O、网络包、协议解析同一套底层
3. **零开销**：无隐藏分配；HFT/内核可完全掌控缓冲生命周期

**一句话**：C 的「字符串」= **带 `'\0'` 结尾的 `char[]` / `char*`**，不是独立数据类型。

## C 字符数组 vs C++ `std::string`

| 对比维度 | C（本章 `char[]`） | C++ `std::string` |
|----------|-------------------|-------------------|
| 底层表示 | `char buf[N]` + 手动 `'\0'` | 封装类，内部自动管理缓冲 |
| 获取长度 | 手动计数 / 后续 `strlen` | `.size()` / `.length()`，通常 O(1) |
| 内存边界 | 固定大小，越界 → **缓冲区溢出** | 动态扩容，相对安全 |
| 拼接/查找 | 手动循环或 ch7 `strcat`/`strstr` | `+`、`find`、`erase` 等 |
| 内存管理 | 栈上静态数组；堆需 `malloc/free` | RAII 自动释放 |
| 头文件 | `<stdio.h>`；串函数 ch7 `<string.h>` | `<string>` |

## HFT / 底层开发的用处

1. **固定栈缓冲**：热路径日志、临时格式化常用 `char buf[256]`，长度可控、无 malloc
2. **协议字段**：定长 symbol、`memcpy` 前须知 NUL 与可见字符边界
3. **最长行模式**：环形缓冲里「当前最长帧」与 K&R 最长行同构 — 维护 max + 拷贝赢家

## 本节高频易错点

1. **忘记末尾 `'\0'`** → `printf` / `copy` 越界读、乱码
2. **容量算错**：循环应 `< lim - 1`，不是 `< lim`
3. **混淆长度**：`getline` 返回值 **含** `'\n'`；日后 `strlen` **不含** `'\0'`、通常 **不含** `'\n'`（除非串里真有）
4. **超长输入**：超过 `MAXLINE-1` 可见字符被截断，**静默丢数据**
5. **`to = from`**：数组不能整体赋值，必须 **`copy` 或循环**
6. **无 NUL 的 5 字节数组**：不是 C 字符串，不能当 `%s` 打印
7. **`char *p = "hi"; p[0]='H'`**：修改字面量 → UB
8. 以为 C 有 **`string`** 关键字 —— 只有 C++ `std::string`
