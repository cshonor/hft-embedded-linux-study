# 1.8.1 指针形参、`&` 取地址、`*` 解引用

## 本节讲什么

一次性讲清四个常混概念：**普通变量**、**指针形参**、**`&` 取地址**、**`*` 解引用**。承接 [1.8 参数传值调用](./1.8-参数传值调用.md) —— C 永远值传递；形参写 `int *p` 时，实参该传 **数字** 还是 **地址**？何时写 `&`、何时直接传指针名？

## 四个核心概念

| 符号/写法 | 是什么 | 例子 |
|-----------|--------|------|
| **变量 `a`** | 存 **数据** 的内存单元 | `int a = 10;` — `a` 里是 10 |
| **`&a`** | **取地址** — 得到 `a` 所在内存的地址（指针值） | `&a` 类型是 `int *` |
| **`int *p`** | **指针变量** `p`，专门存 **地址** | `p = &a;` |
| **`*p`** | **解引用** — 通过 `p` 里的地址访问那块内存里的 **数据** | `*p = 999;` 改的是 `a` |

```text
  变量 a          指针 p
┌─────────┐      ┌─────────┐
│  10     │ ←──  │ &a 地址 │
└─────────┘      └─────────┘
   数据              存的是「指向 a 的地址」
```

## 一、形参是指针，实参怎么传？

**规则**：形参类型是 **`int *p`**（指针）→ 实参必须传 **地址**，用 **`&变量`** 取出地址。

### 示例 1：普通变量 → 必须 `&num`

```c
#include <stdio.h>

void func(int *p)
{
    *p = 999;   /* 通过地址改外面 num 的值 */
}

int main(void)
{
    int num = 10;

    func(&num);          /* 形参是指针 → 实参传 &num */
    printf("%d\n", num); /* 999 */
    return 0;
}
```

**逻辑拆解**：

| 步骤 | 说明 |
|------|------|
| 1 | `func` 形参 `int *p` → `p` 只能接收 **地址** |
| 2 | `num` 是普通 `int`，存的是 **数字 10**，不是地址 |
| 3 | `func(num)` **类型错误** — 不能把 `int` 当 `int *` 传 |
| 4 | `&num` = `num` 的内存地址 → 类型 `int *`，匹配形参 |
| 5 | 仍是 **值传递**：把 **地址这个值** 拷贝给 `p`；`*p = 999` 改的是 `num` 所在内存 |

### 示例 2：不用指针 — 值传递改不动外面

```c
void func(int x)
{
    x = 999;   /* 只改副本 */
}

int main(void)
{
    int num = 10;
    func(num);           /* 传数字 10 的拷贝 */
    printf("%d\n", num); /* 仍是 10 */
    return 0;
}
```

| 传参方式 | 传进去的是什么 | 函数里改形参 | 外面变量 |
|----------|----------------|-------------|----------|
| `func(num)` | 数值 **拷贝** | 改副本 | **不变** |
| `func(&num)` | **地址** 拷贝给 `int *p` | `*p = …` 改原内存 | **变** |

口语里的「传地址」在 C 标准里仍是 **把地址当作值拷贝给指针形参**，不是 C++ 引用。

## 二、实参已经是指针 — 不用再加 `&`

主函数里 **已有指针变量** `p`，且 `p` 里已经存了地址时，**直接传 `p`**：

```c
void func(int *pt)
{
    *pt = 666;
}

int main(void)
{
    int a = 20;
    int *p = &a;   /* p 里已是 a 的地址 */

    func(p);       /* 传指针变量 p，不要写 func(&p) */
    return 0;
}
```

### 关键区分（必背）

| 实参类型 | 写法 | 原因 |
|----------|------|------|
| **普通变量** `num` | `func(&num)` | 要取出 **变量的地址** |
| **指针变量** `p` | `func(p)` | `p` **本身就是地址值** |
| **错误** | `func(&p)` | 传的是 **指针变量 p 自己的地址**（`int **` 才需要，见 1.8 二级指针） |

```c
int num = 10;
int *p = &num;

func(&num);  /* 对：把 num 的地址给 int * 形参 */
func(p);     /* 对：p 里就是 num 的地址 */
func(&p);    /* 错：除非形参是 int **，否则类型不匹配 */
```

## 三、`&` 和 `*` 在传参里的分工

| 运算符 | 名称 | 在传参场景的作用 |
|--------|------|------------------|
| **`&`** | 取地址 | 从 **普通变量** 生成指针值，匹配 `int *` 形参 |
| **`*`** | 解引用 | 在函数 **内部**，通过指针形参访问/修改 **原变量** 的数据 |

**一句话**：形参是 `int *p` 时 —— 外面用 **`&变量`** 把地址送进来；里面用 **`*p`** 动原来的数据。

## 五、误区 1：定义里的 `*` 不是解引用，是类型修饰符

**`*` 分两种场景，含义完全不同，不能混：**

| 场景 | 写法 | `*` 的含义 |
|------|------|------------|
| **定义变量** | `int *p;` | **类型修饰**：`p` 是指针；`int` 表示 **`*p` 解引用后得到的是 `int` 类型** — 此处 **没有** 解引用动作 |
| **执行表达式** | `*p = 100;` | **解引用运算符**：用 `p` 里的地址去内存读/写数据 |

```c
int a = 10;
int *p = &a;   /* 定义时 *：标记 p 是 int 型指针 */
*p = 20;       /* 表达式里 *：解引用，改 a */
```

| 声明 | 读法 |
|------|------|
| `char *q` | `q` 是 char 指针；`*q` 得到 **char** |
| `float *f` | `f` 是 float 指针；`*f` 得到 **float** |

**极简**：`int *p` 里定义时的 `*` = 类型标记；代码里的 `*p` = 解引用操作 —— **两个不是一回事**。

## 六、误区 2：指针里都存地址，为什么还要 `int*` / `char*` / `float*`？

**对的一半**：所有指针变量存的都是 **内存地址**（一串数字）；64 位下指针变量自身通常都占 **8 字节**，32 位 **4 字节**。

**关键**：地址只是 **起始编号**；编译器不知道从该地址起 **读几个字节**、**按什么规则解释** 二进制。

假设地址 `0x1000` 处字节为：`61 00 00 00`（小端示意）

| 指针类型 | `*p` 行为 |
|----------|-----------|
| `char *p` | 读 **1 字节** → `'a'`（0x61） |
| `int *p` | 读 **4 字节** `0x1000~0x1003` → 整数（如 97） |
| `float *p` | 同样 4 字节，按 **IEEE 754** 解析 → 浮点数（完全不同于 int 97） |

**指针类型决定两件事**：

1. **解引用时读多少字节**（步长 / `sizeof`）
2. **二进制按什么语义解析**（整数 / 字符 / 浮点）

不分类型，编译器只有门牌号，不知道屋里是 1 字节字符还是 4 字节浮点 —— 运算、赋值、传参全乱。

## 七、误区 3：「地址是整数，所以指针都该是 `int*`」

- **指针变量里存的**：地址数值（可想象成整数）
- **指针指向的内存里存的**：类型由 **`int` / `char` / `float`** 决定

**类比**：门牌号都是数字，门后住的东西不同：

| 类型 | 门后（解引用 `*p`） |
|------|---------------------|
| `char*` | 1 个字母 |
| `int*` | 一整块整数（通常 4 字节） |
| `float*` | 按浮点规则解读的数 |

不能把 `char*` 当 `int*` 乱传 —— 地址数字可能一样，**解引用读宽度和语义不同**。

> 底层另有 `void*` 等特殊规则；本章先建立「类型带语义」心智，ch05 / ch02 再讲 `void*`、严格别名。

## 八、串一遍：`&num` 与 `&ch` 为何不能混用

```c
void func_int(int *p)   { *p = 100; }
void func_char(char *p) { *p = 'b'; }

int main(void)
{
    int num = 1;
    char ch = 'a';

    func_int(&num);   /* &num 类型 int*，匹配形参 */
    func_char(&ch);   /* &ch  类型 char*，匹配形参 */
    return 0;
}
```

- `&num` 不只是「一个地址数字」，类型是 **`int *`** —— 告诉编译器：从这里起按 **int** 解读
- `&ch` 类型是 **`char *`** —— 按 **char** 读 1 字节
- 地址 **数值格式** 相似，**类型不同** 就不能 `func_int(&ch)` —— 解引用会读错宽度、错语义

## 九、与 1.8 两层修改对照

```c
void f(int *p)
{
    *p = 100;    /* OK：改指针指向的内容（外面可见） */
    p = NULL;    /* 只改形参 p 的副本，外面 ptr 不变 */
}
```

详见 [1.8 §6–§7](./1.8-参数传值调用.md) 一级指针改不了实参指针本身、二级指针 `int **`。

## 十、极简收尾

1. **`int *p`**：`p` 存地址；`int` 规定 `*p` 取出的是 **整型语义 + int 宽度**
2. 所有指针存的地址都是数字，但 **指向内存的长度与编码规则不同**，必须用 `int`/`char`/`float` 区分
3. 定义里的 `*` = **类型标记**；表达式里的 `*变量` = **解引用** —— 两码事

## HFT / 底层提示

- API 设计：`void update_price(int *ticks)` + 调用 `update_price(&price)` — 模式固定
- 已有指针：`Order *o = …; process(o);` — 不要画蛇添足 `process(&o)`（除非要改 `o` 本身）
- 热路径：弄清传的是 **数据拷贝** 还是 **地址拷贝**，避免隐式大结构体传值
