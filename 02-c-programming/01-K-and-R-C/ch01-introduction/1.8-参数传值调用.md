# 1.8 参数传值调用

## 本节讲什么

讲解 C 语言 **唯一的** 参数传递机制：**传值调用（call by value）** —— 形参是实参值的 **副本**，函数内改形参 **不影响** 外部变量。用 **`swap` 经典反例** 说明「为何普通交换失效」；并 **讲透指针传参**：C 仍全是值传递，传指针 = 拷贝 **地址**，解引用可改原数据，但改形参指针本身无效（需二级指针，ch5 展开）。

承接 [1.7 函数](./1.7-函数.md) 的 `power` 调用；为 [ch05 指针](../ch05-pointers-and-arrays/) 铺路。  
**入门速查**（`&` / `*` / 指针形参怎么传）：[1.8.1 指针形参与取地址](./1.8.1-指针形参与取地址.md)。

## 要点

### 1. 核心结论

1. C **不存在** 传引用（pass by reference）；**所有** 函数参数一律 **传值**
2. 传指针 **不是** 传引用，而是把 **指针变量里存的地址数值** 再拷贝一份给形参
3. 分 **两层** 理解指针传参：
   - **不能** 修改实参 **指针变量本身**（一级指针形参里 `p = &b` 只改副本）
   - **可以** 通过拷贝来的地址 **解引用**，修改指针 **指向的内存**（`*p = 100` 改外面的 `a`）

### 2. 传值调用流程

```text
实参表达式求值 → 得到一份值 → 拷贝给形参 → 形参、实参两块独立内存
```

| 步骤 | 说明 |
|------|------|
| 1 | 先计算实参表达式，得到一个 **值** |
| 2 | 将该值 **拷贝一份** 给函数内部的形参 |
| 3 | 形参、实参 **存储位置不同**，生命周期独立；函数返回后形参销毁 |

在函数内部 **修改形参**，只改 **副本**，**不会影响** 外部原始变量。

### 3. 示例 1：普通 `int`，传值改不动原变量

```c
void func(int x)
{
    x = 100;   /* 只改形参副本 */
}

int main(void)
{
    int a = 10;
    func(a);
    /* a 仍是 10 */
    return 0;
}
```

**原因**：把 `a` 的数值 `10` **复制** 给形参 `x`，两者完全独立。

### 4. `swap` 经典陷阱

```c
void swap(int x, int y)
{
    int temp = x;
    x = y;
    y = temp;   /* 只交换了副本 x、y */
}

int main(void)
{
    int a = 1, b = 2;
    swap(a, b);
    /* a 仍是 1，b 仍是 2 —— 交换失效 */
    return 0;
}
```

- `x`、`y` 只是 `a`、`b` 的 **拷贝**
- 函数内交换副本，返回后副本销毁，**主函数 `a`、`b` 保持原样**
- 要真正交换外部变量，需传 **地址**（指针）——见下文示例 2、ch5

### 5. 示例 2：传一级指针 —— 可修改指向的数据

```c
void func(int *p)
{
    *p = 100;   /* 通过地址改原内存 */
}

int main(void)
{
    int a = 10;
    func(&a);
    /* a 变成 100 */
    return 0;
}
```

**拆解**（仍是值传递）：

1. `&a` 得到 `a` 的 **地址**
2. 把这个 **地址数值** **拷贝** 给形参指针 `p`（不是把 `a` 本体搬进去）
3. `p` 与调用时的 `&a` **存同一个地址**
4. `*p` 访问该地址上的 `int`，修改的就是外面的 `a`

### 6. 重点误区：一级指针改不了实参指针本身

```c
void func(int *p)
{
    int b = 999;
    p = &b;   /* 只改形参 p 副本里存的地址 */
}

int main(void)
{
    int a = 10;
    int *ptr = &a;
    func(ptr);
    /* ptr 依旧指向 a，没有变成指向 b */
    return 0;
}
```

| 操作 | 效果 |
|------|------|
| `*p = 100` | 改 **指针指向的内容**（外面可见） |
| `p = &b` | 改 **形参指针副本** 存的地址（外面 `ptr` 不变） |

`p` 是 `ptr` 的 **副本**；改副本里的地址，**不影响** 原指针 `ptr`。

### 7. 想改指针变量本身：传二级指针

```c
void func(int **pp)
{
    int b = 999;
    *pp = &b;   /* 解引用改「原指针 ptr」里存的地址 */
}

int main(void)
{
    int a = 10;
    int *ptr = &a;
    func(&ptr);
    /* ptr 现在指向 b */
    return 0;
}
```

依旧是值传递：把 **`&ptr`**（指针变量的地址）拷贝给形参 `pp`。  
`*pp` 就是外面的 `ptr`；通过 `*pp = &b` 才能改 **实参指针变量本身**。ch5 会系统讲。

### 8. 跨语言区分

| 语言 | 模型 | 与 C 的差异 |
|------|------|-------------|
| **C** | **纯传值** | 无引用形参；指针 = 拷贝地址 |
| **C++** | 传值 + **`int &x` 引用传参** | 写 C 时要 **摒弃** 引用思维 |
| **Java** | 基本类型传值；对象传 **引用的拷贝** | 对象形参改字段，外面可见；但 **不能** 让外面变量指向新对象（类似 C 一级指针陷阱） |
| **Python** | 名字绑定对象；传的是 **对象引用的拷贝** | 可变对象原地改可见；重新赋值形参不影响外面 |

从 C++/Java 转 C：**「传指针就能改外面」** 只对 **`*p` 改内容** 成立，对 **`p` 改指向** 不成立。

### 9. 数组特例（预告）

数组作实参时会发生 **数组退化（decay）** 为 **指向首元素的指针** —— 表面像「传地址」，本质仍是 **拷贝指针值**（地址）。  
本章统一用 **传值 + 副本隔离** 心智；[ch05](../ch05-pointers-and-arrays/) 详讲。

## 一句话总结

1. C **永远** 值传递，所有参数都是拷贝一份副本  
2. 传指针 = 拷贝 **内存地址** 给形参  
3. **解引用** `*指针` → 可修改原变量指向的 **内容**  
4. **直接赋值** 形参指针 `p = …` → 只改副本，**外部指针不受影响**；要改外部指针需 **`int **`**

## HFT / 底层开发的用处

1. **API 设计**：改调用方数据 → `void update(Order *o)` + `o->price = …`；勿在函数里 `o = &other` 指望外面换对象
2. **swap / 状态机**：理解为何必须 `swap(int *a, int *b)` 或返回新值
3. **内核风格**：`struct sk_buff *skb` 传的是指针 **值的拷贝**；改 `skb` 字段有效，改形参 `skb` 指向无效
