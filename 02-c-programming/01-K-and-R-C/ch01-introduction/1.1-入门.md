# 1.1 入门 Getting Started

## 本节讲什么

从经典 **Hello, world** 出发，建立 C 程序最小骨架：头文件、`main`、标准输出与转义字符；并理清 **编译 → 链接 → 运行** 三阶段。有 C++ 基础时，本节重点是 **甩掉 iostream 习惯**，区分 K&R 老式写法与现代标准 C。

## 要点

### 1. 基础输出与转义字符

- **`printf`**：C 标准格式化输出，声明在 `<stdio.h>`（完整说明见 [附录 B.1 stdio.h](../appendix-b-standard-library/B.1-输入与输出stdio.h.md)；**格式与安全**见 [1.1.2 printf](./1.1.2-printf格式与安全.md)）
- **常用转义符**：

| 转义 | 含义 |
|------|------|
| `\n` | 换行 |
| `\t` | 制表符 |
| `\b` | 退格 |
| `\"` | 双引号字符 |
| `\\` | 反斜杠 |

- **字符串**必须用 **双引号** `"..."`；**单引号** `'a'` 只表示 **单个字符**（`char`），不是字符串

### 2. C 程序完整工作流

```text
  hello.c ──► 编译(cc -c) ──► hello.o ──► 链接(cc / ld) ──► a.out ──► 运行
                  │                              │
            预处理+语法+语义              合并 .o + libc（如 crt 启动代码）
```

1. **编译**：`.c` → 预处理 → 翻译单元 → 目标文件 `.o`（机器码 + 未解析符号）
2. **链接**：多个 `.o` + 系统库（`libc` 等）→ 可执行文件；解析 `printf` 等外部符号
3. **运行**：加载器把二进制映射进内存，从 **`main`** 入口开始执行

与 [C 陷阱 ch04 连接](../../03-C-Traps-and-Pitfalls/ch04-linking/) 、[Expert C ch05 链接](../../04-Expert-C-Programming/ch05-thinking-of-linking/) 后续展开。

### 3. Hello World 示例

**K&R 原书写法（历史遗留，工程勿用）：**

```c
#include <stdio.h>

main()
{
    printf("hello, world\n");
}
```

- 省略 `main` 返回类型时，老式编译器默认 **`int main()`**
- 无 `return` 时，旧实现可能默认返回 0

**现代标准 C（工业代码必须这样写）：**

```c
#include <stdio.h>

int main(void)
{
    printf("hello, world\n");
    return 0;
}
```

- **`int main(void)`**：明确无命令行参数；有参数时用 `int main(int argc, char *argv[])`（见下文 **§4**）
- **`return 0`**：表示成功退出；非 0 常表示错误（脚本/CI 可检测退出码）
- `-Wall -Wextra` 下，`main()` 无类型常触发告警；C99 起 **不允许** 隐式 `int`

```bash
gcc -Wall -Wextra -std=c11 -o hello hello.c
./hello
```

### 4. 为何写 `int main(void)`，不能只写 `int main()`

**C 与 C++ 对空括号 `()` 的含义完全不同**——这是 C++ 转 C 时最容易误判的一点。

#### C 语言规则

| 声明 | 含义 |
|------|------|
| `main()` | **参数未指定**（老式 K&R 遗留）：原型上不约束实参个数与类型 |
| `main(void)` | **明确无参数**：调用时传任何实参 → **编译错误** |

普通函数同理，一眼看懂：

```c
/* C：空括号 = 参数未指定，不是「无参」 */
void func_old()
{
}

void func_new(void)
{
}

void demo(void)
{
    func_old(1, 2, 3);   /* 可能编译通过 —— 隐患 */
    func_new(1);         /* 编译错误 ✓ */
}
```

> C23 之前，`f()` 的「空括号 = 参数未指定」仍广泛存在于老代码；**新代码一律 `(void)`**。

#### C++ 规则（与 C 相反）

在 C++ 中，`int main()` 与 `int main(void)` **等价**，都表示无参数。因此从 C++ 过来时容易误以为 C 里两者也一样——**不是**。

#### 放在 `main` 上的实际意义

- **`int main()`**：编译器**不检查**是否误传参，风格不严谨，移植/静态检查易出问题
- **`int main(void)`**：C99/C11/C17 **规范写法**；显式声明「程序入口不要入参」——内核模块、DPDK、嵌入式工程通用习惯

#### 工业标准完整写法

```c
#include <stdio.h>

int main(void)
{
    printf("hello world\n");
    return 0;   /* int 返回类型必须配套 return */
}
```

#### 带命令行参数时（与无参版本互斥）

需要读取启动参数时才写：

```c
int main(int argc, char *argv[])
{
    /* argc = 参数个数；argv[0] 通常是程序名 */
    return 0;
}
```

不要同时混用 `void` 与 `argc/argv`；选 **无参** 或 **带参** 其一即可。

完整专题见 **[1.1.1 命令行参数](./1.1.1-命令行参数-main-argc-argv.md)**（`argc`/`argv` 拆解、示例、用法模板、demo）。

## 与 C++ 的差异（已有 C++ 基础时）

| 主题 | C++ | C |
|------|-----|---|
| **I/O 头文件** | `#include <iostream>` | `#include <stdio.h>` |
| **输出** | `std::cout << "x\n";` | `printf("x\n");` |
| **输入** | `std::cin >> x;` | `scanf("%d", &x);`（须取地址） |
| **`main` 写法** | `int main()` ≡ `int main(void)`（C++ 不区分） | 规范：**`int main(void)`**；`main()` 在 C 里表示参数未指定 |
| **老式 `main()`** | 部分编译器仍接受 | K&R 兼容；**新工程禁止** |

C **没有** 命名空间、类、RAII；I/O 与内存管理（后续 `malloc`）都是 **显式、底层** 的——这正是内核、DPDK、HFT 数据面使用 C 的原因。

## 学习定位（HFT / 底层路线）

1. **开篇认知**：快速建立 C 语法与工具链，避免把 C++ 的 `cout`、引用、异常带进 C 文件
2. **路线前置**：Linux 内核、DPDK、socket 热路径均为 **标准 C + GNU 扩展**；本节是 `00/01–05` 第一站
3. **规范意识**：笔记与书中可能保留 K&R 老式 `main()` 便于对照原书；**自己写的代码一律 `int main(void)`**，减少告警与跨编译器移植问题
