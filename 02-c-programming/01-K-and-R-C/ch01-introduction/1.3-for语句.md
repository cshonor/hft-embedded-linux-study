# 1.3 for 语句

## 本节讲什么

承接 [1.2 温度转换](./1.2-变量与算术表达式.md)，**同一华氏 → 摄氏案例**分别用 **`while`** 与 **`for`** 实现，对比循环结构：`for` 把 **初始化、终止条件、步进更新** 收拢到一行，代码更集中；`while` 则三段逻辑分散在循环外/内。

## 要点

### 1. `for` 完整语法（C / C++ / Java / JS 同形）

```c
for (初始化表达式; 循环条件表达式; 自增/后置表达式) {
    循环体;
}
```

**核心规则**：

1. **三段表达式全部可选** —— 任意一段、两段、三段都能删掉
2. **分隔三段的两个英文分号 `;` 绝对不能省略** —— 编译器靠分号区分三个槽位，少一个即 **语法错误**

执行顺序：**初始化（一次）→ 判条件 → 循环体 → 步进 → 再判条件 → …**

### 2. 为什么两个分号不能丢？

编译器用 **两个分号** 划分三个固定槽位：

```text
for ( [槽1 初始化] ; [槽2 条件] ; [槽3 步进] )
         ↑              ↑              ↑
      分号1前        两分号之间      分号2后
```

| 槽位 | 作用 | 省略时 |
|------|------|--------|
| 槽1 | 初始化 | 常已在循环外完成 |
| 槽2 | 条件 | **恒真** → 无限循环 |
| 槽3 | 步进 | 常改在循环体内 |

### 3. 各种省略示例

**① 省略第一段（初始化）**

```c
int i = 0;
for (; i < 10; i++) {
}
```

**② 省略第二段（条件 → 无限死循环）**

```c
for (i = 0; ; i++) {
}   /* C99 前：int i 须在 for 外声明 */
```

**③ 省略第三段（步进改在循环体内）**

```c
for (i = 0; i < 10; ) {
    i++;
}
```

**④ 省略两段：初始化 + 条件**

```c
int i = 0;
for (;; i++) {
}
```

**⑤ 省略两段：条件 + 步进**

```c
for (i = 0; ; ) {
    i++;
}
```

**⑥ 三段全省略 —— `for(;;)` 最简死循环**

```c
for (;;) {
}
```

等价 `while (1)`；内核轮询、事件循环常见，**循环体内须有 `break`/`return`**。

汇总表：

| 写法 | 含义 |
|------|------|
| `for (; fahr <= 300; fahr += 20)` | 初始化在循环外 |
| `for (fahr = 0; ; fahr += 20)` | 无显式条件 → 死循环 |
| `for (fahr = 0; fahr <= 300; )` | 步进在循环体末尾 |
| `for (;;)` | 无限循环 |

### 4. 反面错误：丢分号直接编译失败

```c
for (int i = 0 i < 10; i++) { }   /* 错：少第一个 ; */
for (int i = 0; i < 10 i++) { }   /* 错：少第二个 ; */
for (int i = 0 i < 10 i++) { }    /* 错：两个 ; 全丢 */
```

### 5. `for` vs `while` 适用场景

| | **`while`** | **`for`** |
|---|-------------|-----------|
| **典型场景** | 次数不确定、纯条件驱动（等信号、阻塞读、轮询硬件） | 次数已知、数组/表遍历、批量计算 |
| **代码结构** | 初始化在循环**外**，更新在循环**末**，三段分散 | 初始化、条件、步进**同一行** |
| **温度表** | 适合理解循环三要素 | K&R 推荐写法，更紧凑 |

底层经验：

- **`for`**：DPDK mbuf 批量处理、行情数组回放、寄存器表索引、固定长度报文解析
- **`while`**：`read`/`recv` 直到 EAGAIN、事件循环 `while (running)`、无固定上界的队列消费

### 6. 循环体与大括号

- 循环体 **只有一条语句** 时，`{}` **可省略**（K&R 原书常省略）
- **多条语句** 必须用 **`{ ... }`** 组成复合语句
- 工程规范：**一律加 `{}`**，避免 [C 陷阱 dangling else](../03-C-Traps-and-Pitfalls/ch02-syntactic-pitfalls/) 类问题；与 1.2 整数截断、`5.0/9.0` 公式相同，不再重复

## 同案例两种写法对照

### while 版本（初始化、更新分散）

```c
#include <stdio.h>

int main(void)
{
    int fahr = 0;              /* 初始化：循环外 */

    while (fahr <= 300) {      /* 条件 */
        printf("%3d %6.1f\n", fahr, (5.0 / 9.0) * (fahr - 32));
        fahr = fahr + 20;      /* 步进：循环末 */
    }

    return 0;
}
```

### for 版本（三段逻辑收拢）

```c
#include <stdio.h>

int main(void)
{
    int fahr;

    for (fahr = 0; fahr <= 300; fahr += 20) {
        printf("%3d %6.1f\n", fahr, (5.0 / 9.0) * (fahr - 32));
    }

    return 0;
}
```

```bash
gcc -Wall -Wextra -std=c11 -o fahr_for fahr_for.c
./fahr_for
```

两种写法 **输出完全一致**；选哪种取决于可读性与团队规范，表驱动计数更常写 `for`。

## HFT / 嵌入式开发的用处

1. **`for` 是底层遍历高频写法**：报文队列、行情 buffer、DMA 描述符环、内存块按索引扫描
2. **`while` 多用于阻塞/轮询**：等 socket 可读、等 ring 非空、无固定次数的事件监听
3. **按场景选循环**：避免把本适合 `for` 的定长遍历拆成分散的 `while` 变量，减少漏写步进导致的 **死循环或越界**

## 与 C++ 的差异

| 主题 | C++ | C（本节） |
|------|-----|-----------|
| `for` 头内声明 | `for (int i = 0; i < n; ++i)` 常见 | C99 起可 `for (int i = …)`；K&R 风格常外置 `int i` |
| 范围 for | `for (auto x : arr)` | C 无；用下标或指针 `for` |
| 无限循环 | `for (;;)` / `while (true)` | `for (;;)` / `while (1)` |
