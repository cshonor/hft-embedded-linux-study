# 1.6 数组

## 本节讲什么

承接 [1.5 字符 I/O](./1.5-character-io/1.5-字符输入输出.md) 的 `getchar` 循环，引入 **一维数组**：声明、下标访问、初始化；用数组做 **桶计数**（数字 `0`~`9`、空白、其他字符各计一次）。同时厘清 **中括号绑在谁身上**、**`++ndigit[k]` 与前置/后置自增**，并预告 **指针数组 vs 数组指针**（第 5 章展开）。

## 要点

### 1. 一维数组声明与下标

```c
int ndigit[10];   /* ndigit 是含 10 个 int 元素的数组 */
```

| 规则 | 说明 |
|------|------|
| **下标从 0 开始** | 长度 10 → 合法下标 **0～9**，没有 `ndigit[10]` |
| **下标是整型表达式** | 可以是 `c - '0'`、`i`、`f(x)` 等 |
| **越界** | 语言 **不检查**；越界是 **未定义行为**，内核/DPDK/HFT 同样致命 |

### 2. 中括号在变量名右边，不在类型右边

C 里 **`[]` 紧跟变量名**，表示 **这个变量是数组**：

```c
int ndigit[10];     /* 对：ndigit 是 int 数组 */
int[10] ndigit;     /* 错：C 语法不允许 */
```

读法：**`int ndigit[10]`** = 变量名 `ndigit` + 「含 10 个 `int` 的数组」。

这样设计是为了声明 **组合类型** 时，运算符优先级能表达：

- 数组、指针、函数返回类型可以 **叠在一起**
- 读复杂声明时从 **变量名** 出发，见 §6 与 [ch05 指针](../ch05-pointers-and-arrays/)

对比 C++：`int arr[10]` 同样绑在变量名侧；Java `int[] arr` 把 `[]` 写在类型侧 —— **C 不是 Java 写法**。

### 3. 与 `getchar` 结合：数字字符桶计数

K&R 经典模式：读字符流，按类别 **分桶累加**。

```c
int ndigit[10];      /* 桶 0～9：字符 '0'～'9' 各计几次 */
int nwhite, nother;  /* 空白、其他 */

/* 初始化略；静态存储默认 0，局部数组需显式清零 */

while ((c = getchar()) != EOF) {
    if (c >= '0' && c <= '9')
        ++ndigit[c - '0'];
    else if (c == ' ' || c == '\n' || c == '\t')
        ++nwhite;
    else
        ++nother;
}
```

**`c - '0'` 为何能当下标？**

- 字符 `'0'`～`'9'` 在 ASCII 中 **连续编码**
- `'0'` 的码值 + 0 = `'0'`，+ 1 = `'1'`，…
- 故 `'3' - '0'` → 整数 **3**，正好对应 `ndigit[3]`

| 字符 `c` | `c - '0'` | 写入桶 |
|----------|-----------|--------|
| `'0'` | 0 | `ndigit[0]` |
| `'5'` | 5 | `ndigit[5]` |
| `'9'` | 9 | `ndigit[9]` |

必须先 **`if (c >= '0' && c <= '9')`** 再减 `'0'`，否则 `'A'-'0'` 等会得到荒谬下标。

### 4. `++ndigit[k]` 与 `i++` / `++i`

**前置 `++i`**：先把 `i` 加 1，**再用新值**参与运算。  
**后置 `i++`**：先用 `i` **当前值**参与运算，**再**把 `i` 加 1。

对 **单独一条语句** 只关心副作用（把计数加 1）时，二者等价：

```c
++ndigit[k];    /* 前置：ndigit[k] 先 +1 */
ndigit[k]++;    /* 后置：ndigit[k] 也 +1；单独语句效果相同 */
```

**嵌进复杂表达式时差别很大**：

```c
int a[3] = {10, 20, 30};
int i = 0;

/* 后置：先用 ndigit[i] 的旧值参与，再 +1 */
x = ndigit[i]++;   /* x 得旧值，然后 ndigit[i] 加 1 */

/* 前置：先把 ndigit[i] +1，再用新值 */
x = ++ndigit[i];   /* x 得加 1 后的新值 */
```

工程建议：**计数、循环步进写独立语句** 时用 `++ndigit[k]` 或 `ndigit[k]++` 均可；**不要**在一条表达式里混用前置/后置与别的运算 —— HFT 热路径要 **可读、可预测**。

### 5. 完整示例思路（字符分类统计）

```c
#include <stdio.h>

int main(void)
{
    int c, i;
    int ndigit[10];
    int nwhite, nother;

    nwhite = nother = 0;
    for (i = 0; i < 10; ++i)
        ndigit[i] = 0;

    while ((c = getchar()) != EOF) {
        if (c >= '0' && c <= '9')
            ++ndigit[c - '0'];
        else if (c == ' ' || c == '\n' || c == '\t')
            ++nwhite;
        else
            ++nother;
    }

    for (i = 0; i < 10; ++i)
        printf(" %d", ndigit[i]);
    printf(", white space = %d, other = %d\n", nwhite, nother);

    return 0;
}
```

### 6. 指针数组 vs 数组指针（易混，别记反）

两者都用到 `*` 和 `[]`，**括号决定绑定关系** —— 中括号、星号 **跟谁绑**，跟谁就是数组/指针。

| 名称 | 含义 | 写法 | 读法 |
|------|------|------|------|
| **指针数组** | **数组**，元素类型是指针 | `int *arr[10];` | `arr` 是 **10 个元素** 的数组，每个元素是 `int *` |
| **数组指针** | **单个指针**，指向整个数组 | `int (*arr)[10];` | `arr` 是 **指针**，指向「含 10 个 `int` 的数组」 |

```c
int *parr[10];     /* 指针数组：10 个 int* 槽位，常用来存字符串指针 argv 风格 */
int (*parr)[10];   /* 数组指针：一个指针，指向 int[10] 整块 */
```

**记忆口诀**：

- **`[]` 离变量名更近** → 先数组：`int *p[10]` → **指针数组**（很多指针）
- **`()` 把 `*` 和变量名括在一起** → 先指针：`int (*p)[10]` → **数组指针**（指向一整块数组）

本章只用到 `int ndigit[10]` 这种最简单形式；[ch05](../ch05-pointers-and-arrays/) 会系统讲 `[]` 与 `*` 的声明阅读规则。

### 7. 「声明随使用」—— C 没有独立的指针类型写法

**不是**「指针类型无法表述」，而是 C **不提供** Rust 那种单独的类型名：

```rust
// Rust：类型名与变量名分离
let p: *mut [i32; 10];
```

C 里 **「指向 int 数组的指针」** 这种整体类型，**只能** 通过绑定写法体现：

```c
int (*p)[10];   /* p 是指针；*p 使用时是 int[10] */
```

**读声明的核心逻辑（声明随使用 / declaration follows use）**：

1. 从 **变量名** 出发，看 **使用时** 对它做什么操作
2. **操作的结果类型** 反推 **变量本身** 是什么

| 声明 | 使用时 | 反推 |
|------|--------|------|
| `int x;` | `x` 就是 `int` | `x` 是 int |
| `int *p;` | `*p` 是 `int` | `p` 是指向 int 的指针 |
| `int arr[10];` | `arr[i]` 是 `int` | `arr` 是含 10 个 int 的数组 |
| `int (*p)[10];` | `(*p)[i]` 是 `int` | `p` 是指向「int[10]」的指针 |

所以你的理解对：**不是先写一个直观的「指针类型名」再挂变量**，而是 **先写变量名 + 修饰符（`*`、`[]`）**，告诉编译器「对它解引用 / 下标之后得到什么」，**反推** 它自身类型。

`*`、 `[]` 写在 **变量名旁边**，就是在描述 **「用这个变量时，左边/解引用后的类型长什么样」**。  
完整螺旋规则见 [Expert C ch03](../../04-Expert-C-Programming/ch03-analyzing-c-declarations/)；`typedef` 可把复杂声明 **起别名**（如 `typedef int (*arrptr)[10];`），但底层仍是同一套绑定语法。

### 8. 同时声明两个指针：`int *p, *q` 不是 `int *p, q`

`*` **只绑定紧挨它的那个标识符**：

```c
int *p, *q;    /* 对：p、q 都是 int* */
int *p, q;     /* 错：只有 p 是指针，q 是普通 int */
```

**规范**：每行只声明一个指针变量，或每个名字前都写 `*`：

```c
int *p;
int *q;

/* 或 */
int *p, *q;
```

HFT / 内核代码 review 时，`int *p, q` 是常见 **隐藏 bug** 来源。

### 9. 初始化与存储

| 场景 | 未显式初始化时 |
|------|----------------|
| **静态存储**（全局、`static` 局部） | 元素 **默认为 0** |
| **自动存储**（普通局部数组） | 元素 ** indeterminate** —— 用前必须赋值或 `memset` |

- 越界读写：**未定义行为**，无运行时检查
- 桶计数前 **`for` 清零** 或 `static int ndigit[10];` 依赖静态零初始化

## HFT / 底层开发的用处

1. **直方图 / 频次表**：价位 tick 计数、消息类型分布 — 与 `ndigit[c-'0']` 同模式
2. **固定大小 ring / slot 数组**：下标 `0..N-1` 取模，越界即灾难
3. **声明可读性**：热路径 buffer 命名清晰，`int buf[4096]` 而非魔数散落

## 与 C++ 的差异

| 主题 | C（本节） | C++ |
|------|-----------|-----|
| 数组大小 | `[10]` 须编译期常量（VLA 可选扩展） | `std::array<int,10>`、`vector` |
| 边界 | 不检查 | `at()` 可检查；`vector` 更安全 |
| 字符转数字 | `c - '0'` | 仍常用；亦可用 `<cctype>` |
