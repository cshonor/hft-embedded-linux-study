# 1.1.1 命令行参数 `main(int argc, char *argv[])`

## 本节讲什么

C 标准规定的 **带命令行参数** 的 `main` 签名（C89/C99/C11 均支持）。程序从终端 / CMD 启动时，由操作系统把用户输入的分段字符串交给 `argc` / `argv`——`gcc`、`git`、`ls` 等工具都依赖这套机制。

与 [1.1 入门](./1.1-入门.md) **§4** 衔接：无参程序用 `int main(void)`；需要读启动参数时用本节签名，**二者互斥**。

## 代码基础释义

```c
int main(int argc, char *argv[])
{
    /* argc = 参数个数；argv[0] 通常是程序名 */
    return 0;
}
```

### 1. `int argc` — Argument Count

- 整型，**命令行分段参数的总个数**
- **最小值为 1**：可执行文件名本身算第 1 个参数
- 示例：`./test a b c` → 共 4 段，`argc == 4`

### 2. `char *argv[]` — Argument Vector

- 等价写法：**`char **argv`**（数组参数退化为指针）
- 每个元素是 **`char *`**，指向以 **`\0` 结尾** 的 C 字符串
- 索引规则：

| 下标 | 内容 |
|------|------|
| `argv[0]` | 程序名 / 路径（**通常**；见下文注意） |
| `argv[1]` … `argv[argc-1]` | 用户传入的参数 |
| `argv[argc]` | 标准规定为 **`NULL`**（可用于循环兜底） |

### 3. `return 0`

- `main` 的 `int` 返回值交给操作系统 / 调用者（shell、`make`、CI）
- **`0`**：正常退出
- **非 0**：异常或自定义错误码（脚本可 `$?` / `echo %ERRORLEVEL%` 检测）

## 完整示例：打印所有参数

```c
#include <stdio.h>

int main(int argc, char *argv[])
{
    printf("argc = %d\n", argc);
    for (int i = 0; i < argc; i++)
        printf("argv[%d] = %s\n", i, argv[i]);
    return 0;
}
```

### Linux / WSL

```bash
gcc -Wall -Wextra -std=c11 -o test test.c
./test hello 123 -v
```

```text
argc = 4
argv[0] = ./test
argv[1] = hello
argv[2] = 123
argv[3] = -v
```

### Windows CMD（空格自动分段）

```cmd
gcc test.c -o ArgcArgv.exe
ArgcArgv.exe Nice to meet you !
```

shell 以 **空格** 切分；共 **6** 段 → **`argc = 6`**：

```text
Argument 0 is ArgcArgv.exe .
Argument 1 is Nice .
Argument 2 is to .
Argument 3 is meet .
Argument 4 is you .
Argument 5 is ! .
```

对应关系：

| `argv[i]` | 内容 |
|-----------|------|
| `argv[0]` | `ArgcArgv.exe`（程序名） |
| `argv[1]`～`argv[5]` | `Nice` / `to` / `meet` / `you` / `!` |

等价打印代码（与仓库 `demo/demo_argv` 同一逻辑，仅措辞不同）：

```c
for (int i = 0; i < argc; i++)
    printf("Argument %d is %s .\n", i, argv[i]);
```

简单两参数示例：

```cmd
test.exe apple orange
```

```text
argc = 3
argv[0] = test.exe
argv[1] = apple
argv[2] = orange
```

### Windows：`argv[0]` 常为完整路径 + 两种典型传参

在 CMD 中用 **完整路径** 启动时，`argv[0]` 往往是 **exe 全路径**，而不只是文件名：

**示例 A — 开关选项（4 个 flag + 程序路径 → `argc = 5`）**

```cmd
C:\...\Debug\mainFunc.exe -a -b -c -d
```

```text
argc is 5
argv[0] : C:\Users\Administrator\Desktop\mainFunc\Debug\mainFunc.exe
argv[1] : -a
argv[2] : -b
argv[3] : -c
argv[4] : -d
```

**示例 B — 普通词 + 数字字符串（同样 `argc = 5`）**

```cmd
C:\...\Debug\mainFunc.exe age name test 123
```

```text
argc is 5
argv[0] : C:\Users\Administrator\Desktop\mainFunc\Debug\mainFunc.exe
argv[1] : age
argv[2] : name
argv[3] : test
argv[4] : 123
```

要点：

- **每个空格分隔段占一个 `argv[i]`**；`-a` 与 `age` 在 C 眼里都是普通 C 字符串，是否当「选项」由 **你的解析逻辑** 决定（如 `strcmp(argv[i], "-v")`）
- **`123` 仍是字符串 `"123"`**，不是 `int`；计算前必须 `strtol` 等转换
- Linux/WSL 下 `./mainFunc -a -b` 时 `argv[0]` 多为 `./mainFunc`，行为规则相同，仅 `argv[0]` 字符串形式不同

## 关键细节与注意事项

### 1. `argv[0]` 不绝对保证是「短程序名」

标准只保证 **通常是** 调用时的程序名或路径；Windows 常给 **全路径**；`exec` 族还可传入自定义字符串。**用法提示用 `argv[0]` 打印即可，不要假设它一定是 `xxx.exe` 短名**。

### 2. 参数如何被分割

- 默认以 **空格 / 制表符** 分段
- 含空格的整体参数用引号：`test "my file.txt"` → `argv[1]` 为 `my file.txt`（无引号字符）

### 3. 数字参数本质是字符串

`./test 99` → `argv[1]` 是 **`"99"`**，不是 `int`。转数值用 **`atoi` / `strtol` / `strtod`**（优先 **`strtol`** 带错误检查，见 ch05 库函数陷阱）。

### 4. 常见 `main` 签名对比

| 签名 | 用途 |
|------|------|
| `int main(void)` | 不接收命令行参数 |
| `int main(int argc, char *argv[])` | **标准** 接收命令行参数 |
| `int main(int argc, char *argv[], char *envp[])` | 读环境变量（**非标准 C**，POSIX / 部分编译器扩展） |

### 5. 实用模板：参数个数校验

```c
#include <stdio.h>

int main(int argc, char *argv[])
{
    if (argc < 2) {
        fprintf(stderr, "用法: %s <文件名>\n", argv[0]);
        return 1;
    }
    printf("待处理文件: %s\n", argv[1]);
    return 0;
}
```

- 用法信息写 **`stderr`**，正常输出写 **`stdout`**（便于管道重定向）
- **`return 1`**：参数错误的标准惯例

## 应用场景

- **命令行工具**：编译器、包管理、运维脚本入口
- **启动配置**：配置文件路径、`-v` 版本、`-o` 输出路径
- **HFT / 底层**：进程启动传 **网卡名、core mask、配置文件**；DPDK `dpdk-l2fwd` 等即 `argc/argv` 解析

## 与 C++ 的差异

C++ 同样使用 `int main(int argc, char* argv[])`；还可写 `char* argv[]` 或 `char** argv`。C++17 起可选 **`std::vector<std::string>`** 封装（非 `main` 签名本身，而是启动后解析）。
