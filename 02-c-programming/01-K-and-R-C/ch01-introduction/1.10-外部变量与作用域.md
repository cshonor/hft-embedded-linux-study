# 1.10 外部变量与作用域

## 本节讲什么

承接 [1.9 最长行示例](./1.9-字符数组.md)，对比 **自动局部变量** 与 **外部全局变量**；区分三个核心概念：

1. **作用域（scope）**：在 **当前 `.c` 里** 哪段代码 **能看见这个名字**（编译期）
2. **链接（linkage）**：链接器合并多个 `.c` 的 `.o` 时，**同名符号能否关联** —— **`static` = 内部链接**（各编译单元各用各的）；**无 `static` = 外部链接**（可合并成一份）。**编译单元** ≈ **一个 `.c` 文件**（[4.4 §3](../ch04-functions-and-program-structure/4.4-作用域规则.md)、[4.5](../ch04-functions-and-program-structure/4.5-头文件.md)）
3. **存储期 / 生命周期（lifetime）**：内存 **何时分配、何时释放**（运行期）

讲清 **四种作用域**、**`extern` 声明**、**定义 vs 声明**（Define once; declare many times），用 **全局变量版最长行** 做完整示例，铺垫 [ch04 存储类](../ch04-functions-and-program-structure/)（[4.4 四种作用域](../ch04-functions-and-program-structure/4.4-作用域规则.md)、[4.3 外部变量](../ch04-functions-and-program-structure/4.3-外部变量.md)、[4.6 静态变量](../ch04-functions-and-program-structure/4.6-静态变量.md)）、多文件编译、`static`。

## 要点

### 1. 局部自动变量（auto）

**定义位置**：函数或代码块 `{}` 内部；默认存储类 **`auto`**（关键字可省略）。

| 特性 | 说明 |
|------|------|
| **作用域** | 仅当前函数/块内可见；`getline` 的 `i` 与 `copy` 的 `i` **互不干扰** |
| **生命周期** | 函数调用时在 **栈** 分配，返回即释放；多次调用 **不保留** 上次值 |
| **初值** | 未赋值 → **indeterminate**（垃圾值），**不** 自动清零 |

**短板**：函数间无法直接共享数据，只能靠 [1.8 传参/返回值](./1.8-参数传值调用.md)；复杂时参数列表冗长。

### 2. 外部变量与全局变量

**定义位置**：所有函数 **之外**（文件顶层）。

**一句话**：从 **定义开始** 到 **整个程序结束** 都有效，内存 **一直在**（不像局部变量函数结束就销毁）。其中 **无 `static` 的全局变量** 就是 **外部变量**（外部链接），**同一程序** 里有多少个 `.c` 都行：**一处定义**，其他文件 **`extern` 声明** 即可跨文件共用 **同一份** 对象。**`static` 全局** 虽也在全局/静态区，但 **仅限本 `.c`**，不能被他文件 `extern`（见 [4.3 §1.1](../ch04-functions-and-program-structure/4.3-外部变量.md)）。

| 特性 | 说明 |
|------|------|
| **存储** | **静态数据区**；程序启动分配，退出释放 |
| **生命周期** | **启动 → 结束**；函数返回 **不** 销毁全局 |
| **跨文件** | 他文件 `extern int x;` 引用，**不重复分配**（详见 [4.3 §2](../ch04-functions-and-program-structure/4.3-外部变量.md)） |
| **初值** | 未显式初始化 → 编译器 **清零**（0、`'\0'`、`NULL`） |
| **坏处** | 修改点分散、耦合高、难测试、难追踪 |

1. **本文件**：从 **定义行** 到文件末尾，所有函数可见（**文件作用域**）
2. **加 `static`**：作用域 **仅限本 `.c`**，他文件 **不能用**（`extern` 也无效）；**防不同文件间同名冲突**
3. **无 `static`**：**外部链接**；他文件 `extern` 声明后 **可直接用** —— 作用域扩展到整个项目多文件（[4.4 §1②](../ch04-functions-and-program-structure/4.4-作用域规则.md)）
4. **多文件**：一处 **`int num;` 定义**；其余 **`extern`** 或 include 头文件；禁止两文件各定义无 `static` 同名全局

### 3. 定义 vs 声明（本节重中之重）

| | **定义（definition）** | **声明（declaration）** |
|---|------------------------|-------------------------|
| 写法 | 文件顶层 **不带** `extern` | **`extern`** 开头 |
| 内存 | **分配** 存储 | **不分配**，仅引用已有对象 |
| 次数 | 整个程序 **只能一次** | 可 **多次** |

```c
int max;              /* 定义：分配内存 */
extern int max;       /* 声明：不分配，引用别处/本文件前面的定义 */
```

**准则**：**Define once; declare many times**（定义一次，声明多次）。

**`extern` 两种场景**：

1. **同文件**：全局变量定义在函数 **后面** 时，函数内要先 `extern int max;` 才能用
2. **多文件**：A.c 定义，B.c `extern` 引用；更常见是 **`a.h` 里 `extern int global_num;`**，B.c/C.c 只 `#include "a.h"` 即可读写 **同一份** `global_num` —— **谁改谁可见最新值**（见 [4.5 §2.1](../ch04-functions-and-program-structure/4.5-头文件.md)）

> `extern int a;` **不能** 同时做定义式初始化（C 里带初始化的 `extern` 仍是定义）；`int a = 1;` 在文件顶层才是定义并赋初值。

### 4. K&R 示例：用外部变量重写最长行

`getline` 改名为 **`mgetline`**，避免与 POSIX `getline(3)` 冲突。

```c
#include <stdio.h>

#define MAXLINE 1000

/* 外部变量定义：全局，分配内存 */
int max;
char line[MAXLINE];
char longest[MAXLINE];

int mgetline(void);
void copy(void);

int main(void)
{
    int len;

    extern int max;           /* 显式 extern（定义在顶部时可省略） */
    extern char longest[];

    max = 0;
    while ((len = mgetline()) > 0) {
        if (len > max) {
            max = len;
            copy();
        }
    }
    if (max > 0)
        printf("%s", longest);
    return 0;
}

int mgetline(void)
{
    int c, i;

    extern char line[];

    for (i = 0; i < MAXLINE - 1 && (c = getchar()) != EOF && c != '\n'; ++i)
        line[i] = c;
    if (c == '\n') {
        line[i] = c;
        ++i;
    }
    line[i] = '\0';
    return i;
}

void copy(void)
{
    int i = 0;

    extern char line[], longest[];

    while ((longest[i] = line[i]) != '\0')
        ++i;
}
```

```bash
gcc -Wall -Wextra -std=c11 -o longest_global longest_global.c
printf "aa\nbbb\n" | ./longest_global
```

#### 1.9 局部版 vs 1.10 全局版

| | [1.9 局部版](./1.9-字符数组.md) | 1.10 全局版 |
|---|-------------------------------|-------------|
| 数据传递 | `getline(line, lim)`、`copy(to, from)` 传参 | `line`/`longest`/`max` **全局共享** |
| 函数签名 | 带数组参数 | `mgetline(void)`、`copy(void)` |
| 耦合 | **低**，函数可独立测试 | **高**，依赖全局状态 |
| 教学点 | 传值、手动字符串 | 作用域、extern、全局状态代价 |

### 5. 局部自动变量 vs 外部全局变量

| 对比维度 | 局部自动变量 | 外部全局变量 |
|----------|-------------|-------------|
| 定义位置 | `{}` 内 | 函数外，文件顶层 |
| 内存分区 | **栈**（调用时分配） | **静态数据区** |
| 默认初值 | 垃圾值 | **0** |
| 生命周期 | 进入函数创建，退出销毁 | 启动→退出全程 |
| 作用域 | 当前块 | 定义处至文件末；`extern` 跨文件 |
| 同名遮蔽 | 内层局部遮外层 | 被块内同名局部遮蔽 |
| 函数耦合 | 低（参数交互） | 高（共享全局状态） |
| 关键字 | `auto`（默认） | 定义无 `extern`；`static` 限本文件（ch4） |

### 6. 作用域 vs 生命周期（独立概念）

| 概念 | 含义 | 阶段 |
|------|------|------|
| **作用域** | 名字 **可见范围** | **编译期** — 编译器能否识别标识符 |
| **生命周期** | 对象 **存活时长** | **运行期** — 内存是否存在、值是否保留 |

二者 **独立**：

- **`static` 局部**（ch4）：作用域 **仅函数内**，生命周期 **全程**（静态存储）
- **普通全局**：作用域 **全文件/跨文件**，生命周期 **全程**

### 7. 优缺点与工程禁忌

**优势**

1. 多函数共享数据时 **简化参数**（全局缓冲、配置）
2. 调用间 **持久状态**（计数器、最长行 `max`）
3. 多文件共享 **单一资源**（需注意链接与可见性）

**滥用缺陷（重点）**

1. **强耦合**：任意函数可改，逻辑难追踪
2. **难调试**：多处写入，难定位谁改了值
3. **不可重入 / 线程不安全**：竞态；内核/HFT 热路径须 **锁或无锁设计**
4. **单元测试难**：函数依赖全局环境，无法隔离

**工程建议**：大型项目用 **结构体 + 指针传参** 封装状态；全局仅用于 **真正单一实例** 的资源（且加 `static` 或锁）。

## 内核 / HFT 拓展

1. 共享全局状态配 **自旋锁 / 互斥锁**；热路径避免无保护全局写
2. 模块私有：`static` 全局变量 — **文件内可见**，不污染符号表
3. 跨模块：Linux **`EXPORT_SYMBOL` / `EXPORT_SYMBOL_GPL`** 导出；对应「受控的全局接口」
4. 超大缓冲：勿放栈上局部数组（栈溢出），可全局/`static`/堆分配

## 本节高频易错点

1. **定义 vs 声明**：`extern int a;` 不分配内存；`int a = 1;` 才是定义
2. **重复定义**全局变量 → 链接 **multiple definition**
3. **局部遮蔽全局**，误以为改的是全局
4. **`static` 全局**：只 **缩作用域**，不改变静态存储期（ch4）
5. **大数组放栈**：局部 `char buf[1<<20]` → 栈溢出；改全局或 `malloc`
6. **多文件** 未 `extern` → **undefined reference** / 未声明标识符
