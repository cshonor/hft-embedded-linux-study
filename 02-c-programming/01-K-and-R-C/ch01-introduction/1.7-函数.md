# 1.7 函数

## 本节讲什么

如何 **定义、声明、调用** 自定义函数，用 **函数分解** 降低 `main` 复杂度；示例为整数幂 `power(m, n)`。

**核心规则（纠正常见说法）**：不是「除 `main` 外必须把声明写在文件最开头」，而是 **调用出现在定义之前时，必须先让编译器见过声明或完整定义**。声明放文件顶部只是 **工程上最常用**，不是语法硬性要求。`main` 是入口，没有代码会提前调用它，故 **`main` 永远不需要提前声明**。

## 要点

### 1. 为什么需要函数声明（函数原型）

C 编译器 **从上到下逐行编译** 当前翻译单元（`.c` 文件）：

- 调用 `add(1, 2)` 时，若编译器 **还没见过** `add` 的 **定义（函数体）** 或 **声明（原型）**，会 **报错**（C99 起）；旧编译器可能 **隐式假设** `int` 返回、参数类型乱猜 —— **极易出错**
- **函数声明 / 原型** 提前告诉编译器：**叫什么、参数类型是什么、返回什么类型**，调用处才能生成正确的参数传递与返回值处理

```c
int add(int a, int b);   /* 原型：不是函数体，末尾分号 */
```

### 2. 三种合法写法（声明位置不限）

只要满足 **「首次调用前，编译器已见过声明或定义」** 即可。

**方式 1：声明放文件顶部（最常用、工程规范）**

```c
int add(int a, int b);   /* 声明 */

int main(void)
{
    add(1, 2);           /* OK：上面已声明 */
    return 0;
}

int add(int a, int b)    /* 定义在后 */
{
    return a + b;
}
```

**方式 2：定义写在 `main` 前面 —— 无需额外声明**

定义 **本身自带** 原型信息；编译器先看到完整定义，后面调用自然认识：

```c
int add(int a, int b)
{
    return a + b;
}

int main(void)
{
    add(1, 2);           /* OK：add 已在上方定义 */
    return 0;
}
```

**方式 3：声明写在 `main` 内部（合法但极少用）**

块内声明 **只在该块内可见**，其他函数 **看不见**：

```c
int main(void)
{
    int add(int a, int b);   /* 仅 main 内有效 */
    add(1, 2);
    return 0;
}

int add(int a, int b) { return a + b; }
/* 若别的函数也调 add，仍须在文件级或 .h 再声明 */
```

| 方式 | 适用 |
|------|------|
| 顶部声明 + 后部定义 | **默认首选**；原型一览、与 K&R `power` 示例一致 |
| 定义在前 | 小文件、辅助函数紧挨调用者 |
| `main` 内声明 | 教学/临时；**不推荐** 工程代码 |

### 3. 多文件工程：声明进 `.h`

大型项目不会在每个 `.c` 复制一堆原型，而是：

```c
/* math_utils.h */
int add(int a, int b);

/* main.c */
#include "math_utils.h"
/* 或 math_utils.c 里 #include 同一 .h */

/* math_utils.c */
#include "math_utils.h"
int add(int a, int b) { return a + b; }
```

`#include` 把 `.h` 里的声明 **粘贴** 进当前文件 —— 本质仍是 **调用前让编译器看见原型**（ch4 预处理器展开）。

### 4. `main` 为何不需要声明

- **`main`** 由 **链接器 / 运行时** 作为程序入口调用，**没有** 你的 C 代码会在前面 `main(...)` 一下
- 因此 **不需要** 也 **不应依赖** 在文件里写 `int main(void);` 来满足「先声明后调用」（写了通常也无害，但无必要）

### 5. 函数定义 vs 函数原型（声明）

**造零件 vs 贴标签**：

| 比喻 | 含义 |
|------|------|
| **函数定义 = 造零件** | 返回类型、函数名、参数列表 + **函数体** —— 把 **怎么干活** 全写出来 |
| **函数原型 = 贴标签** | 只写返回类型、函数名、参数列表 —— 提前告诉编译器 **叫什么、收什么、返什么**；**无函数体**，末尾 `;` |

```c
int power(int m, int n);              /* 贴标签：原型 */

int power(int base, int n)            /* 造零件：定义 */
{
    int p;
    for (p = 1; n > 0; --n)
        p = p * base;
    return p;
}
```

**调用写在定义前面时**，编译器还没见过这个零件的 **标签** → **报错**。必须先写 **声明（原型）**，或把 **定义** 挪到调用之前。

| 对比 | 原型（贴标签） | 定义（造零件） |
|------|---------------|---------------|
| 函数体 | **无** | **有** |
| 典型位置 | 调用 **之前**、**头文件** | `.c` 源文件 |
| 次数 | 可多次 | 只能 **一次** |

详见 [4.1 §6](../ch04-functions-and-program-structure/4.1-函数的基本知识.md)。

### 6. 函数形式与 `return`

```c
int add(int a, int b)     /* 返回类型 int 写在函数名左边 */
{
    语句;
    return 表达式;        /* 非 void 必须能返回值 */
}
```

- **返回类型** 在函数名 **左侧**：`int add(...)`、`void f(...)` —— 详见 [4.1](../ch04-functions-and-program-structure/4.1-函数的基本知识.md)
- **C89 及以前**：省略返回类型 → 隐式 **`int`**；没 `return` 也当返回 `int`，但值 **未定义**
- **C99 起**：必须显式写返回类型，**不写直接报错**
- **无返回值**：须在左边写 **`void`**，不是省略 `return` 就行
- **`void *`**：通用指针，可指任意类型；**不能直接解引用**，须强转 —— 见 [4.1 §3.1](../ch04-functions-and-program-structure/4.1-函数的基本知识.md)、ch05
- `return 表达式;` — 带返回值；`return;` — `void` 函数
- 形参列表 **`void`** 表示无参：`int f(void);`（与 `int f();` 旧式不同，见 K&R 演进）

### 7. 分解原则

- **`main`**：流程、I/O、调用关系
- **子函数**：单一计算或单一职责（如 `power` 只做乘幂）
- 与内核「小函数、清晰边界」同构；HFT 里热路径函数与非热路径分层

### 8. 与「声明随使用」同一套思维

[1.6](./1.6-数组.md) 讲变量声明：**使用时什么类型，反推变量是什么**。  
函数声明同理：调用处 `power(2, i)` 需要编译器已知 **`power` 接受两个 `int`、返回 `int`**，否则无法生成正确调用指令（参数入栈/寄存器、返回值处理）。

函数指针声明 `int (*fp)(int, int);` 也是 **声明随使用**：`*fp` 调用后得到 `int`，故 `fp` 是指向「接受两 int 返回 int 的函数」的指针 —— 留 [ch05](../ch05-pointers-and-arrays/)。

## 完整示例（方式 1：顶部原型 + 后部定义）

```c
#include <stdio.h>

int power(int m, int n);   /* 文件顶部：原型（非必须在此，但是工程惯例） */

int main(void)
{
    int i;
    for (i = 0; i < 10; ++i)
        printf("%d %d %d\n", i, power(2, i), power(-3, i));
    return 0;
}

int power(int base, int n)
{
    int p;
    for (p = 1; n > 0; --n)
        p = p * base;
    return p;
}
```

```bash
gcc -Wall -Wextra -std=c11 -o power_demo power_demo.c
./power_demo
```

## HFT / 底层开发的用处

1. **头文件接口**：策略引擎 `.h` 声明 API，`.c` 实现 — 改实现不改调用方
2. **编译期检查**：原型错误（参数类型不对）在编译期暴露，而非实盘才崩
3. **`static` 文件内函数**（ch4）：仅本翻译单元可见，减少符号污染

## 与 C++ 的差异

| 主题 | C（本节） | C++ |
|------|-----------|-----|
| 声明位置 | **调用前可见**即可；顶部 / 定义在前 / `.h` 均可 | 同样；另有 overload、namespace |
| 无参 | `f(void)` 明确表示无参 | `f()` 与 `f(void)` 在 C++ 中都是无参 |
| 分解 | 函数 + 文件 | 另加 class、namespace |
