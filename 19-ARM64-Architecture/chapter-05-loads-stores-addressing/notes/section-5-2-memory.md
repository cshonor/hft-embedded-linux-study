## §5.2 内存 — 内存映射 (Memory Map)

> **Ch 5 · 加载、存储与寻址** · [章导读](../README.md)

---

### 内存 = 字节寻址阵列

| 概念 | 说明 |
|------|------|
| **基本单位** | **字节 (8 bit)** — 每个字节一个地址 |
| **连续性** | 逻辑上 **0 … 2³²−1**（32 bit 地址空间）连续编号 |
| **访问** | Load/Store 指定 **宽度**（byte/half/word）+ **起始地址** |

**Ch1/Ch2：** 半字/字对齐要求 — 错宽度或错对齐 → fault 或读错值。

---

### 内存映射 (Memory Map)

SoC 把 **Flash、SRAM、外设** 钉死在地址空间的固定区间 — 程序员用 **同一套 LDR/STR** 访问 RAM 与 **MMIO 寄存器**。

**Tiva 微控制器（书中示例）典型布局：**

```
高地址
    ┌─────────────────┐
    │  外设 (MMIO)     │  UART · Timer · GPIO …
    │  0x40000000 区   │
    ├─────────────────┤
    │  SRAM (数据/栈)   │
    ├─────────────────┤
    │  Flash (代码)    │  复位后 PC 从此执行
低地址 0x00000000
```

| 区域 | 内容 | 访问 |
|------|------|------|
| **Flash/ROM** | 只读代码、常量 | `LDR` 取指；常量池 |
| **SRAM** | 变量、堆、栈 | `LDR`/`STR` 读写 |
| **外设** | 控制/状态寄存器 | **MMIO** — 特定地址 = 硬件（→ **Ch16**） |

---

### 与嵌入式 Linux / 飞控

| MCU 本书 | 应用处理器 |
|----------|------------|
| 查 **数据手册 Memory Map** | **设备树 (DT)** 描述基址/中断（→ [22 DT](../../../22-Device-Tree-Study/)） |
| `EQU UART_BASE 0x4000xxxx` | `ioremap()` / `devm_ioremap_resource()` |
| 4GB 平面地址 | 同上 — **Load/Store 或 MMIO 映射后读写** |

**Cortex-A Linux：** 用户态不直接 `STR` 到 `0x40000000` — 经 **内核驱动** 映射；**指令语义相同**。

---

### 可复述要点

1. 内存 = **按字节编号** 的大数组；Load/Store 选 **宽度 + 地址**。  
2. **Memory Map** 决定 Flash/RAM/外设各在哪段地址。  
3. 驱动 = 对 **正确地址** 做 **正确宽度** 的 Load/Store。
