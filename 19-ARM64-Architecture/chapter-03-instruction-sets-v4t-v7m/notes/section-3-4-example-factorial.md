## §3.4 示例程序 2 — 阶乘 (Factorial)

> **Ch 3 · 指令集简介** · [章导读](../README.md)

---

### 本程序学什么

| 目标 | 内容 |
|------|------|
| **循环** | 递减计数直到条件满足 |
| **比较与标志** | `CMP` 设置 **N/Z/C/V** |
| **条件执行** | ARM7：`MULGT`/`SUBGT` 等 · M4：**IT 块** |

算法：`n!` = n × (n−1) × … × 1，或累乘循环。

---

### CMP 与标志位

```asm
        CMP     r0, #1          ; 伪：r0 - 1，只改标志不改 r0
        ; r0 == 1 → Z=1（EQ）
        ; r0 >  1 → GT（有符号）
        ; r0 <  1 → LT
```

| 标志 | 阶乘里常见用途 |
|------|----------------|
| **Z** | 是否减到 / 等于 1（结束） |
| **N / V** | 有符号 **GT/LT** |
| **C** | 无符号比较（本例多用有符号 GT） |

**CMP ≠ SUB：** `SUB` 改 r0；`CMP` **只刷标志**。系统展开 → [Ch7 §7.3](../../chapter-07-integer-logic-arithmetic/notes/section-7-3-compare.md)

与循环搭配：

```asm
loop
        CMP     r0, #1
        MULGT   r1, r1, r0
        SUBGT   r0, r0, #1
        BGT     loop
```

---

### ARM7TDMI：条件后缀 · MULGT / SUBGT

ARM **32-bit 模式**几乎所有通用指令可加 **条件后缀**：

```
指令助记符 + 条件码
例：MUL + GT → MULGT · SUB + GT → SUBGT · B + GT → BGT
```

| 条件成立 | 执行该指令（写寄存器 / 跳转） |
|----------|------------------------------|
| 不成立 | **当作 NOP** — 不运算、不改任何寄存器 |

条件读的是 **当前 CPSR 的 N/Z/C/V**（通常刚被前面的 `CMP` / `SUBS` / `ADDS` 等更新）。

#### GT 含义

| | |
|--|--|
| **GT** | Greater Than — **有符号大于** |
| **标志条件** | **Z=0 且 N == V** |
| **CMP 后语义** | 若刚做了 `CMP Rn, Op2` → **Rn > Op2**（有符号） |

#### SUBGT

```asm
        CMP     r0, #1
        SUBGT   r0, r0, #1      ; 仅 r0>1 时：r0 = r0 − 1
```

- 基础：`SUB Rd, Rn, Op2`
- **`SUBGT`**：满足 GT 才减；`r0==1` 或 `r0<1` 时本条作废

#### MULGT

```asm
        CMP     r0, #1
        MULGT   r1, r1, r0      ; 仅 r0>1 时：r1 = r1 × r0
```

- 基础：`MUL Rd, Rm, Rs` → `Rd = Rm × Rs`
- **`MULGT`**：不是新指令，只是 **带 GT 的 MUL**

#### 阶乘整段串起来

```asm
        MOV     r0, #5          ; 算 5!
        MOV     r1, #1          ; 乘积初值
loop
        CMP     r0, #1          ; 刷标志
        MULGT   r1, r1, r0      ; r0>1 才乘
        SUBGT   r0, r0, #1      ; r0>1 才减
        BGT     loop            ; r0>1 才跳回
; 结束时 r1 = 120
```

| 阶段 | 行为 |
|------|------|
| r0=5…2 | CMP → GT 成立 → MUL/SUB/B 全执行 |
| r0=1 | CMP 后 **不满足 GT** → MULGT/SUBGT/BGT **全作废** → 循环结束 |

#### 常用条件后缀（背）

| 后缀 | 含义 |
|------|------|
| **EQ** | 相等（Z=1） |
| **NE** | 不等（Z=0） |
| **GT / GE** | 有符号 > / ≥ |
| **LT / LE** | 有符号 < / ≤ |
| **AL** | 永远执行（默认；不加后缀 = AL） |

#### 核心三点

1. 后缀看的是 **已有标志**（多来自上一条 `CMP` 等），不是指令自己比  
2. `MULGT` / `SUBGT` = **条件版** MUL / SUB，不是新 opcode 族  
3. 条件假 → **零副作用**  

**优势：** 短 if 少分支 → 少冲刷流水线（经典 ARM 卖点）。  
系统化 → [Ch8 §8.4](../../chapter-08-branches-loops/notes/section-8-4-conditional.md)

### Cortex-M4：IT 块 (IF-THEN)

Thumb-2 **没有** 所有指令都带条件后缀；用 **IT** 修饰后续 1–4 条：

```asm
        CMP     r0, #1
        IT      GT
        MULGT   r1, r1, r0      ; IT 块内条件执行
        SUBGT   r0, r0, #1
```

| 要点 | 说明 |
|------|------|
| **`IT` 掩码** | `ITTE EQ` 等指定后续几条是否条件执行 |
| **块内指令** | 必须带与 IT 一致的条件后缀 |
| **密度** | 替代 ARM 模式整条指令条件位 |

**易错：** IT 块中间不能插入无关指令；调试时 **单步看清 IT 掩码**。

---

### 与 C 对照

```c
while (n > 1) {
    result *= n;
    n--;
}
```

编译器生成 **CMP + 条件分支**；ARM 条件后缀/IT 是 **把分支折叠进指令** 的手写优化（现代 C 编译器也会选 B.cond）。

---

### 可复述要点

1. **`CMP` 只改标志** — 比较的两操作数之一常用立即数。  
2. **ARM7：指令后缀条件执行**；**M4：IT 块** 包装 Thumb 指令。  
3. 阶乘 = **循环 + 比较 + 乘法** — Ch3 第一个「小算法」。
