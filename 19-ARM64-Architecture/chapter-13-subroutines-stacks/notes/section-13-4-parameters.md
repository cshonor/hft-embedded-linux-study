## §13.4 向子程序传递参数 — 寄存器 · 指针 · 堆栈

> **Ch 13 · 子程序与堆栈** · [章导读](../README.md) · [本章概述](./section-0-本章完整概述.md)

---

### 三种传参方式

| 方式 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| **寄存器** | 调用者 **`MOV r0, arg0`** 再 **`BL`** | **最快** | 寄存器数量有限 |
| **指针（引用）** | **`r0 = &struct`** 或 **`&buffer[0]`** | 大对象、字符串 **不拷贝** | 需约定读写权限 |
| **堆栈** | 调用者 **`PUSH` 参数** 再 **`BL`** | 参数 **任意多** | 慢；要算 **SP 偏移** |

实际 ABI（AAPCS）= **前几个用寄存器，其余用栈**（§13.5）。

---

### 1. 寄存器传参

```asm
    MOV     r0, #1000       ; 延时毫秒
    BL      DelayMs
    ; 返回值可在 r0
```

- 子程序从 **r0–r3**（及浮点 **s0–s15**）读入。  
- 适合 **标量、小整数、指针本身**。

---

### 2. 指针（按引用）

```asm
    LDR     r0, =BigBuffer   ; r0 = 缓冲区首地址
    MOV     r1, #256         ; 长度
    BL      ParseNMEA
```

- 子程序通过 **`LDR/STR [r0, #off]`** 读写内存。  
- **C 里数组、struct *** 就是这种模式。  
- 调用者与子程序须约定：**谁分配、谁释放、是否修改**。

---

### 3. 堆栈传参

**调用者：**

```asm
    MOV     r0, #3
    PUSH    {r0}            ; 第 2 个参数（假设约定栈传）
    MOV     r0, #1          ; 第 1 个参数仍可能在 r0
    BL      AddTwo
    ADD     sp, sp, #4      ; 调用者清理栈（若约定 caller-pops）
```

**被调用者** 在序言后读栈：

```asm
AddTwo
    STMDB   sp!, {lr}
    LDR     r1, [sp, #4]    ; 偏移依序言压栈字数而定
    ADD     r0, r0, r1
    LDMIA   sp!, {pc}
```

**结果写回栈：** 子程序可把 **返回值覆盖** 到栈上参数槽，调用者 **`POP`** 读取 — 较少见，多数用 **r0**。

---

### 偏移计算的坑

子程序 **`STMDB sp!, {…}`** 后，**参数相对 SP 的偏移** 随 **保存寄存器个数** 变化：

```
SP_after_prologue = SP_entry - 4×(saved_regs)
参数在 [SP_entry + 4] 或更高 — 需按帧布局手算或画栈图
```

**口述：** 栈传参 **灵活但易错** — 生产环境优先 **AAPCS 寄存器+固定栈布局**。

---

### 与 AAPCS 的衔接

| 参数序号 | AAPCS (32-bit) |
|----------|----------------|
| 1–4 | **r0–r3** |
| 5+ | **栈**（降序压入，8 字节对齐） |
| 返回值 | **r0**（64 bit 用 **r0+r1**） |

详见 [§13.5](./section-13-5-apcs.md)。

---

### 可复述要点

1. **快 → 寄存器**；**大 → 指针**；**多 → 栈**。  
2. 栈传参要画 **栈帧图** 才算对偏移。  
3. 与 C 互调时 **不要自创传参顺序** — 跟 **AAPCS**。
