## §4.2 汇编语言模块结构

> **Ch 4 · 汇编器规则与伪指令** · [章导读](../README.md)

---

### 标准行格式

一行汇编源程序最多四部分：

```
{标签} {指令|伪指令|伪操作} {操作数} {; 注释}
```

| 规则 | 说明 |
|------|------|
| **标签** | 必须从 **第 1 列** 开始（行首） |
| **指令/伪指令** | 前面必须有 **空格或 Tab** — 不能从第 1 列写，否则被当成标签 |
| **注释** | `;` 起至行尾（ARM/Keil 风格）；GNU 亦常用 `@` 或 `/* */` |

```asm
loop            ; ← 标签（第1列）
    ADD r0, r0, #1    ; ← 指令（缩进）
    B   loop          ; 分支回 loop
```

**链接 Ch3：** 程序 1 的 `start`/`loop` 即标签 — 分支与链接的目标。

---

### 常量表示

| 类型 | 典型写法 | 备注 |
|------|----------|------|
| **十进制** | `42` | 默认 |
| **十六进制** | Keil: `0xFF` · CCS: `0xFFh` 或 `0xFF` | 汇编/调试最常用 |
| **字符** | `'A'` | 常作为 8 bit 数值 |
| **字符串** | `"Hello"` | Keil/CCS 对 **NUL 结尾**、转义 `\"` `\n` 等细节略有差异 |

**C 对照：** `.rodata` 里的 `"Hello\n\0"` ↔ 汇编 `DCB "Hello",0`（Keil）或 `.asciz "Hello"`（GNU）。

---

### 模块 = 一个源文件

| 概念 | 说明 |
|------|------|
| **多文件工程** | 每文件若干 **段 (section)**；链接器合并（→ [Ch1 §1.7](../../chapter-01-overview-computing-systems/notes/section-1-7-tools.md)） |
| **符号可见性** | 标签/`EQU` 可 **LOCAL/EXPORT/GLOBAL** — 跨文件链接靠全局符号 |
| **与 C 混合** | `.c` 编译出 `.o`，`.s` 出 `.o`，链接器统一解析（→ **Ch18**） |

### GNU (gas) 行格式补充

```gas
.global _start
_start:                     @ 标签
    mov     r0, #1          @ 指令
.Lloop:
    b       .Lloop
```

| Keil | GNU |
|------|-----|
| 标签第 1 列 | 同上 |
| `;` 注释 | `@` 或 `#`（取决于语法模式） |

---

### 可复述要点

1. **第 1 列 = 标签；指令必须缩进** — 写错列 = 汇编错误 or  silent bug。  
2. **hex/char/string** 写法随工具链略异，**语义相同**。  
3. 源文件 = **标签 + 指令 + 伪指令** 的序列，由汇编器变成 `.o`。
