## 一、核心：单行汇编四分段标准格式

```
{标签}  {指令/伪指令}  {操作数}  {; 注释}
```

四个字段均为 **可选**；**有指令/伪指令时必须缩进** — ARM 汇编（Keil）与 GNU 汇编通用基础规范。

### 各字段强制规则

| 字段 | 书写规范 | 关键禁忌 |
|------|----------|----------|
| **标签** | 必须写在 **行首（第 1 列）**，无前置空格 | 前面有空格会被当成指令 → 报错；末尾常加 `:`（部分汇编器可省略） |
| **指令/伪指令** | 前方必须有 **空格/Tab 缩进**，不能顶格 | 顶格会被当成 **标签** → 语法错误 |
| **操作数** | 跟在指令后，逗号分隔；无操作数可省略 | — |
| **注释** | **Keil/ARM：** `;` 至行尾<br>**GNU as：** `@` 单行 · `/* */` 块注释 | `;` / `@` **只作用本行**，不跨行 |

**背一句：**

> **顶格 = 标签；缩进 = 指令。**

---

## 二、示例逐行解析

```asm
loop            ; loop 标签：行首无空格，跳转目标
    ADD r0, r0, #1  ; ADD：前置 Tab；r0 = r0 + 1
    B   loop        ; 无条件跳转到 loop
```

| 行 | 作用 |
|----|------|
| `loop` | **标签** — 给当前指令地址起别名；`B`/`BL`/调用可引用 |
| `ADD r0, r0, #1` | 算术指令；必须缩进 |
| `B loop` | 无条件分支 → 标签 `loop` |

## 三、标签（Label）完整讲解

### 1. 本质定义

标签是 **代码/数据行内存地址的文字别名**。

CPU 靠地址取指；手写数字地址易错。标签给位置起名，**汇编器**把名字换成对应 **地址/偏移**。

### 2. 书写硬性规则

1. **必须顶格（第 1 列）**，前不能有空格/Tab：

```asm
loop            ; ✓ 正确：行首无空格
    ADD r0, r0, #1

    loop        ; ✗ 错误：前有空格 → 被当成指令名 → 报错
    ADD r0, r0, #1
```

2. 可加冒号（ARM / GNU 都常见；省略多数也能认）：

```asm
loop:
    ADD r0, r0, #1
```

### 3. 三大核心用途

**① 跳转 / 分支（最常用）**

```asm
loop
    ADD r0, r0, #1
    B   loop        ; 无条件跳到 loop
; BEQ / BNE 等同理，靠标签定位目标
```

**② 函数调用入口**

```asm
main
    BL  func_sum    ; 带返回的跳转

func_sum            ; 函数入口标签
    ADD r0, r1, r2
    MOV pc, lr
```

**③ 数据地址符号**

```asm
str_msg:
    .asciz "hello"
; …
    LDR r0, =str_msg   ; str_msg = 字符串首地址
```

（Keil 侧常见 `DCB "hello",0` + `LDR r0, =str_msg`，语义相同 → [§4.4](./section-4-4-directives.md)）

### 4. 底层原理（汇编两遍直觉）

1. 读到标签 → 记下 **「名字 ↔ 当前偏移/地址」**  
2. 遇到 `B` / `BL` / `LDR =` 引用该名 → **换成数值地址**  

标签本身 **不生成机器指令**，只做记号。

### 5. 易混区分 ⚠

| | 标签 | 指令（ADD / B / LDR…） |
|--|------|------------------------|
| 作用 | 标记 **地址** | 变成可执行 **机器码** |
| 占指令空间？ | **否**（不产生 opcode） | **是** |

### 6. 极简记忆

> **标签 = 代码/数据位置的名字，给跳转、调用、取地址用 — 必须顶格写。**

## 四、注释 · 易错 · 模块意义

### 1. 两种工具链注释

| 工具链 | 注释 |
|--------|------|
| **ARM / Keil MDK** | `;` … 至行尾 |
| **GNU as**（GCC、Linux、裸机） | `@` 单行 · `/* … */` 块 |

### 2. 易错坑 ⚠

1. **指令顶格** → 当成标签 → 未定义符号 / 语法错  
2. **标签前加空格** → 标签失效，无法作跳转目标  
3. **行首先写 `;`** → **整行注释掉**，后面的标签也不会生效  

### 3. 模块结构整体意义

整个 `.s` / `.asm` = 多行四分段格式 + **段伪指令**（Keil `AREA` / GNU `.text` · `.data`）→ 完整可汇编模块。  
→ [§4.4 伪指令](./section-4-4-directives.md)

## 六、模块 = 一个源文件

| 概念 | 说明 |
|------|------|
| **多文件** | 每文件若干 **段**；链接器合并（→ [Ch1 §1.7](../../chapter-01-overview-computing-systems/notes/section-1-7-tools.md)） |
| **符号** | LOCAL / EXPORT / GLOBAL — 跨文件靠全局符号 |
| **与 C 混合** | `.c` + `.s` → 各自 `.o` → 链接（→ **Ch18**） |

### GNU (gas) 行格式示例

```gas
.global _start
_start:                     @ 标签（可带 :）
    mov     r0, #1          @ 指令须缩进
.Lloop:
    b       .Lloop
```

| Keil | GNU |
|------|-----|
| 标签第 1 列 | 同上 |
| `;` | `@` 或 `/* */` |

## 七、记忆要点

1. **顶格 = 标签，缩进 = 指令**  
2. **标签 = 地址别名**；分支 / 调用 / 取数据地址全靠它  
3. 标签 **不产生机器码**；指令才产生  
4. 注释符跟 **工具链**（Keil `;` · GNU `@`）  
5. 源文件 = 标签 + 指令 + 伪指令（段）→ `.o`
