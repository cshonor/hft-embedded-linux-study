## §4.5 宏 (Macros)

> **Ch 4 · 汇编器规则与伪指令** · [章导读](../README.md)

---

### 宏 vs 子程序

| | **宏 (Macro)** | **子程序 (Subroutine)** |
|---|----------------|-------------------------|
| **展开时机** | **汇编时** 文本替换 | **运行时** `BL` 跳转 |
| **代码体积** | 每调用 **复制一份** 指令 | 只有 **一份** 函数体 |
| **速度** | **无** 调用/返回开销 | 有 `BL`/`BX lr` 成本 |
| **参数** | 汇编期 `$` 参数替换 | 运行时寄存器/栈传参（**Ch13**） |

**选用：** 极短、需内联 hot path（如 **读/write 单个 MMIO 位**）→ 宏；复杂逻辑 → 子程序。

---

### Keil 宏语法

```asm
        MACRO
$label  wait_us $count
        ; 展开体 — $count 在汇编时替换
        MOV     r0, $count
        ; ... 延时循环 ...
        MEND

        ; 调用
        wait_us 100
```

| 元素 | 说明 |
|------|------|
| **`MACRO` / `MEND`** | 宏定义起止 |
| **`$参数`** | 形式参数；调用时 **字面替换** |
| **`$label`** | 可选 — 生成 **唯一标签** 避免多次展开冲突 |

**GNU 对照：** C 预处理器 `#define wait_us(n) ...` 或 gas `.macro wait_us count` … `.endm`。

---

### 展开示例（概念）

```asm
; 定义
        MACRO
        set_bit $reg, $bit
        ORR     $reg, $reg, #$bit
        MEND

; 两次调用
        set_bit r1, 4
        set_bit r2, 8
```

汇编后 ≈ 两条独立的 `ORR` 指令 — **不是** `BL set_bit`。

---

### 风险与习惯

| 风险 | 对策 |
|------|------|
| **多次展开膨胀 Flash** | 大宏改子程序 |
| **标签重复** | 用 `$label` 或 `.L` 局部标签（GNU） |
| **副作用** | 宏改寄存器要 **文档化** — 与子程序 ABI 同问题 |

**内核常见：** `macro` 定义 **寄存器保存/恢复**、**TLB/缓存 op** 的重复序列。

---

### 可复述要点

1. **宏 = 汇编期复制粘贴**；子程序 = 运行时跳转。  
2. 参数用 **`$`**（Keil）；GNU 用 `.macro` 参数名。  
3. 省周期换体积 — HFT/中断 handler 里要想清楚。
