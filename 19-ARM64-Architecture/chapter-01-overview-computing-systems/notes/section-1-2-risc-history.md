### RISC 与 CISC 对比

| 特征 | **RISC**（精简指令集） | **CISC**（复杂指令集，如 x86） |
|------|------------------------|--------------------------------|
| 指令长度 | **固定长度**（ARM 常见 32 位；Thumb 16/32 位） | 变长（x86 1–15 字节） |
| 执行模型 | 倾向 **单周期 / 流水线友好** | Historically 微码、复杂寻址 |
| 访存 | **只有 Load/Store 能访问内存**；运算在寄存器间 | 指令可直接 `add [mem], eax` 等 |
| 微代码 | **无微代码**（或极简） | 大量复杂指令靠微码实现 |
| 设计哲学 | 硬件简单、编译器优化空间大 | 单条指令表达力强 |

**汇编直觉：** 写 ARM 汇编时，**算数 ≠ 访存** — 先 `LDR` 进寄存器，算完再 `STR` 写回。这与 [01 CSAPP](../../../01-CSAPP-3rd/) x86 的「内存操作数」习惯不同。

### ARM 处理器演进（书中脉络）

```
ARM1 → … → ARM7TDMI（本书 v4T 代表）
                ↓
         Cortex 家族（按场景分线）
                ↓
    ┌───────────┼───────────┐
    │           │           │
Cortex-A    Cortex-R    Cortex-M
应用处理器   实时/安全     微控制器
(手机/Linux) (汽车/存储)   (本书重点, 如 M4)
```

| 系列 | 典型用途 | 与本书 / 你的路径 |
|------|----------|-------------------|
| **Cortex-A** | 智能手机、树莓派、**无人机应用处理器** | 嵌入式 Linux → [奔跑吧 ARM64](../../arm64-programming-practice/) · [20 构建](../../../20-UBoot-Kernel-Build/) |
| **Cortex-R** | 硬盘控制器、汽车 ECU、硬实时 | 本书仅背景提及 |
| **Cortex-M** | STM32 等 MCU、传感器节点 | **本书正文主战场**（Ch2–18 大量 v7-M 示例） |

### 为什么 Smith 从 ARM7TDMI + Cortex-M 讲起

- **ARM7TDMI**：经典 **v4T** — 理解 ARM/Thumb 双状态、历史向量表。
- **Cortex-M4**：现代 **v7-M** — Thumb-2 为主、NVIC 中断、可选 FPU（Ch9–11 浮点章）。
- **不是 AArch64**：64 位 A 架构语法在 [奔跑吧](../../arm64-programming-practice/OUTLINE.md) 另学；**概念**（Load/Store、栈、异常）可一一映射。

---

### 可复述要点

1. RISC 核心：**固定指令 + Load/Store 访存 + 寄存器运算**。
2. ARM 公司产品线按 **A / R / M** 切场景；本书练 **M**，你的 Linux/飞控退路靠 **A**。
3. ARM7TDMI 与 Cortex-M 是 **两代 ISA 风格**，Ch2 会并排对比寄存器与模式。
