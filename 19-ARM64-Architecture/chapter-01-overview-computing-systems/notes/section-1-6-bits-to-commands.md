## §1.6 将比特翻译为命令 — 助记符与汇编语言

> **Ch 1 · 计算机系统概述** · [章导读](../README.md)

---

### 从比特模式到操作

处理器不做「读 ADD」，只认 **固定宽度的比特模式**（ARM 常见 **32 位**；Thumb **16/32 位**）：

```
32 bit 比特模式  →  译码器  →  控制信号  →  ALU/访存/分支…
```

| 机器层 | 人类可读层 |
|--------|------------|
| `E3A00005`（示例 hex，随 ISA/指令而变） | `MOV r0, #5` |
| 字段拆分：操作码 + 寄存器编号 + 立即数… | **助记符 (Mnemonic)** + 操作数 |

**汇编语言定义：** 用 **助记符** 代替裸比特，由 **汇编器** 翻译回机器码 — 仍是 **ISA 层**，一行（通常）对应一条或几条机器指令。

---

### 为什么是「命令」而不是「数据」

同一串比特，**取决于 PC 指向哪里**：

- 取指路径 → **指令**
- Load 路径 → **数据**

存储程序模型下，**指令和数据都在内存** — 权限与 Harvard/Modified Harvard 是微架构细节（Cortex-M 指令缓存 vs 数据侧）。

---

### 与高级语言的边界

| | 汇编 | C |
|---|------|---|
| 抽象 | 寄存器名、显式 Load/Store | 变量、表达式 |
| 控制流 | `B` / `BL` / 条件码 | `if` / `for` |
| 谁生成机器码 | 汇编器（+ 链接器） | 编译器 **先出汇编或直接出目标码** |

**Ch3 起：** 五个示例程序让你 **单步看寄存器** — 把本节「比特→助记符」变成肌肉记忆。

---

### 可复述要点

1. CPU 只执行 **比特模式**；助记符是给人看的 ISA 文本形式。
2. 汇编器职责：**助记符 → 目标文件中的机器码**（→ §1.7 完整工具链）。
3. 读 listing / 反汇编 = 把机器码 **译回** 助记符 — 调 bug 必备技能。
