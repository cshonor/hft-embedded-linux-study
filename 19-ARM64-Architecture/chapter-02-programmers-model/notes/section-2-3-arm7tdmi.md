## §2.3 ARM7TDMI — 处理器模式 · 寄存器 · 向量表

> **Ch 2 · 程序员模型** · [章导读](../README.md)

---

### 七种处理器模式

| 模式 | 缩写 | 典型触发 | 特权 |
|------|------|----------|------|
| **User** | usr | 正常应用 | **非特权** |
| **FIQ** | fiq | 快速中断 | 特权 |
| **IRQ** | irq | 普通中断 | 特权 |
| **Supervisor** | svc | 软件中断 / OS 调用 | 特权 |
| **Abort** | abt | 内存访问 fault | 特权 |
| **Undefined** | und | 未定义指令 | 特权 |
| **System** | sys | 与 User 同寄存器，但特权 | 特权 |

```
大多数应用跑在 User
        ↓ 中断/异常/fault
六种特权模式之一（自动切换 CPSR 模式位）
        ↓ 处理完毕
返回 User（或 System）
```

**与 Linux 粗对照：** User ≈ **EL0**；特权模式 + 内核代码 ≈ **EL1** 内核态处理异常（AArch64 在 [奔跑吧 Ch1](../../arm64-programming-practice/chapter-01-arm64-fundamentals/) 展开）。

### 寄存器组织 — 37 物理，模式可见 ~17

**任意时刻程序员可见（典型）：**

| 寄存器 | 别名 | 作用 |
|--------|------|------|
| **r0 – r12** | — | 通用 |
| **r13** | **SP** | 堆栈指针（**每种模式可有独立 banked SP**） |
| **r14** | **LR** | 链接寄存器 — 保存 **返回地址**（`BL` / 异常入口） |
| **r15** | **PC** | 程序计数器 — **当前指令地址 + 8**（流水线历史原因，读 PC 需注意） |
| **CPSR** | — | 当前程序状态 — **N Z C V** + 中断屏蔽 + **模式位** |
| **SPSR** | — | 异常时保存 **进入异常前的 CPSR**（User 模式无 SPSR） |

**Banked registers（分组寄存器）：**

- 物理共 **37 个**寄存器；换模式时 **自动换一组 r13/r14/SPSR**（FIQ 还 bank r8–r12）。
- **好处：** 异常响应 **不必立刻把全部寄存器压栈** — 尤其 FIQ 为快中断留了最多私有寄存器。
- **代价：** 程序员要记住「当前模式下 r13 是哪一组 SP」。

**Ch13 预告：** 软件调用仍常用 **PUSH/POP** 保存 r0–r12；bank 解决的是 **硬件自动** 的那一层。

### CPSR 标志位（与 Ch7 联动）

| 位 | 名 | 含义 |
|----|-----|------|
| N | Negative | 结果为负 |
| Z | Zero | 结果为零 |
| C | Carry | 进位/借位/扩展 |
| V | Overflow | 有符号溢出 |

条件执行 `EQ`/`NE`/… 读这些位（→ **Ch8**）。

### 异常向量表 — 存 **跳转指令**

ARM7TDMI 向量表在 **固定地址**（常 `0x00000000` 或 `0xFFFF0000`，由复位引脚决定）：

| 偏移 | 异常 |
|------|------|
| 0x00 | Reset |
| 0x04 | Undefined |
| 0x08 | SWI (SVC) |
| 0x0C | Prefetch Abort |
| 0x10 | Data Abort |
| 0x14 | IRQ |
| 0x18 | FIQ |

**关键：** 每个槽位放的是 **`B handler` 等分支指令**，不是函数指针。CPU 取到异常地址后 **执行该指令** 跳到 handler（→ **Ch14** 详述）。

---

### 可复述要点

1. **7 模式** — 仅 User 非特权；异常自动切特权模式。  
2. **Banked SP/LR/SPSR** — 快响应，FIQ bank 最多。  
3. **r13=SP, r14=LR, r15=PC, CPSR=状态**。  
4. **向量表 = 跳转指令**，不是 C 函数地址表。
