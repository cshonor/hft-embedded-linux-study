## §2.3 ARM7TDMI — 处理器模式 · 寄存器 · 向量表

> **Ch 2 · 程序员模型** · [章导读](../README.md) · [本章概述](./section-0-本章完整概述.md)

---

### 核心概念：什么是处理器模式？

ARM32（ARM7TDMI / **v4T**）用 **CPSR 低 5 位 `M[4:0]`** 区分当前 **7 种模式**。

**本质两件事：**

| 机制 | 作用 |
|------|------|
| **权限隔离** | **User** 唯一 **非特权**；其余 **6 种全特权** — 可访问 NVIC/MMU/外设/关中断等 |
| **异常专用寄存器分组（Bank）** | 各模式可有 **私有 r13/r14、SPSR** — 中断进来 **不必立刻保存全部寄存器**，降低切换开销 |

```
CPSR.M[4:0] 决定「当前模式」
        ↓
换模式 → 换可见的 SP/LR/SPSR 银行（FIQ 还换 r8–r12）
        ↓
异常返回：SPSR → CPSR，LR → PC（→ Ch14 SUBS pc,lr,#n）
```

**与 Cortex-M4 勿混：** M4 只有 **Thread/Handler + 特权位**（[§2.4](./section-2-4-cortex-m4.md)），**没有** 这套 7 模式。

---

### 七种模式总表

| 模式 | 缩写 | 特权 | 典型触发 |
|------|------|------|----------|
| **User** | usr | **非特权** | 应用程序正常运行 |
| **System** | sys | 特权 | 内核 **手动** 写 CPSR 切换 |
| **Supervisor** | svc | 特权/异常 | **Reset** · **`SWI/SVC`** 系统调用 |
| **IRQ** | irq | 特权/异常 | 普通外设中断 |
| **FIQ** | fiq | 特权/异常 | 高速实时中断 |
| **Undefined** | und | 特权/异常 | 无法识别的指令 |
| **Abort** | abt | 特权/异常 | 预取/数据中止（MMU fault） |

---

### 逐条拆解

#### 1. User（usr）— 用户模式

| 项 | 说明 |
|----|------|
| **特权** | **唯一非特权** |
| **用途** | 上层应用、业务代码 |
| **限制** | 不能关中断、配 MMU、直接碰外设；要硬件服务 → **`SVC`/`SWI`** 进内核 |
| **SPSR** | **无**（User 不保存 SPSR） |

#### 2. System（sys）— 系统模式

| 项 | 说明 |
|----|------|
| **特权** | **特权** |
| **用途** | OS **内核后台任务** — 与 User **共用同一套 r0–r12、r13、r14**（相对 User 视图） |
| **特点** | **不切 bank 的 r0–r12**，又有 **完整硬件权限** — 内核线程省上下文切换 |
| **触发** | 内核 **写 CPSR 模式位** 主动进入；**不会** 由硬件异常自动进 System |

#### 3. Supervisor（svc）— 管理模式

| 项 | 说明 |
|----|------|
| **特权** | 特权 / **异常模式** |
| **触发** | ① **上电 Reset**（ARM7 复位进 **SVC**，非 User）② 执行 **`SWI/SVC #imm`** |
| **用途** | 内核主态、驱动 init、**系统调用**处理（open/read/write 底层） |
| **寄存器** | **banked SP_svc、LR_svc、SPSR_svc** |

#### 4. IRQ — 普通中断模式

| 项 | 说明 |
|----|------|
| **特权** | 异常特权 |
| **触发** | 定时器、UART、GPIO 等 **普通优先级** 外设 **IRQ 线** |
| **特点** | 优先级 **低于 FIQ**；**r0–r7 与 User 共用**（不 bank）；**r13/r14 为 IRQ 私有** |
| **软件** | ISR 须 **手动压栈** 会被破坏的 **r0–r12** 等（→ Ch13/Ch14） |

#### 5. FIQ — 快速中断模式

| 项 | 说明 |
|----|------|
| **特权** | 异常特权 |
| **触发** | 高速 ADC、DMA、雷达等 **低延迟** 需求 |
| **核心优势** | **私有 r8–r12、r13、r14**（共 **7 个** 额外 bank）— 进中断 **少保存寄存器**，**延迟极低** |
| **向量** | 表末 **0x18** 后可紧接 handler（→ [Ch14 §14.5](../../chapter-14-exception-handling-arm7tdmi/notes/section-14-5-vector-table.md)） |

#### 6. Undefined（und）— 未定义指令模式

| 项 | 说明 |
|----|------|
| **触发** | CPU **译码失败** — 非法 opcode、Thumb/ARM 状态错 |
| **用途** | ① **崩溃/调试** ② **软件浮点/协处理器仿真**（无 VFP 时 trap 进 handler 用整数库算） |
| **关联** | [Ch14 §14.3](../../chapter-14-exception-handling-arm7tdmi/notes/section-14-3-error-conditions.md) |

#### 7. Abort（abt）— 中止模式

| 项 | 说明 |
|----|------|
| **分两类** | **预取中止**（取指非法）· **数据中止**（Load/Store 越界、MMU 权限、缺页） |
| **用途** | **缺页 fault**、保护违规 — Linux **用户态缺页** 在特权 Abort 路径处理（概念 → [04 LKD](../../../04-Linux-Kernel-Development/)） |
| **返回** | 修复页表后 **`SUBS pc, lr, #4/#8`** 重试（Ch14） |

---

### 极简分类（背诵用）

| 类 | 模式 |
|----|------|
| **普通运行** | **User**（应用）· **System**（内核后台） |
| **内核/复位/ syscall** | **SVC** |
| **中断** | **IRQ**（普通）· **FIQ**（快） |
| **错误** | **UND**（非法指令）· **ABT**（内存/取指 fault） |

### 端到端例子：User 打字 → 按键 IRQ → 保存 → 回到 User

**模式本质 = CPU「身份切换」：** 不同身份 → 不同 **权限** + **专属资源（bank）**。下面用 Smith 教材最常见的 **ARM7 裸机/经典模型** 串一遍（Linux 上按键还会经 **SVC 系统调用** 进内核，那是 Ch14 之后的事；这里先抓 **模式 + bank + 压栈**）。

**七种身份一句话（复习）：**

| 模式 | 身份角色 |
|------|----------|
| **User** | 普通程序 — **权限最低**，不能直接改硬件 |
| **System** | **特权** 但 **与 User 共用 r13/r14** — 跑 **内核线程** 等需要「用户寄存器视图 + 硬件权限」的代码（**不是**「高权限版用户 APP」） |
| **SVC** | 内核主态 — Reset、**系统调用**、特权指令 |
| **IRQ / FIQ** | 中断服务 — 外设打断正常执行 |
| **ABT** | 内存/取指 fault |
| **UND** | 非法指令 trap |

**场景：** User 模式在文档程序里打字 → 按 **保存** 键 → 键盘控制器拉高 **IRQ**。

```
① 异常前（User 模式）
   PC 指向打字代码；CPSR.M = User；SP/LR 用 User 那对 r13/r14

② 硬件自动（进 IRQ 身份）
   CPSR.M[4:0]  → IRQ
   LR_irq       ← 返回地址（被打断处 PC，Ch14 再减 #4 修正流水线）
   SPSR_irq     ← CPSR 快照（含原 User 模式位、N/Z/C/V、I/F…）
   SP 视图      ← 换用 r13_irq（中断专用栈，与 User 栈隔离）

③ 软件（IRQ_Handler 开头）
   r0–r7 与 User 共用 → 若 handler 会改它们，须 STMFD 压进 **IRQ 栈**
   （本例只写 r0–r7；若还用 r8–r12，IRQ 下同样要压 — FIQ 才免 r8–r12）

④ 执行保存文件的中断处理代码（特权、可访问外设/驱动）

⑤ 返回 User（软件 + 一条特殊指令）
   从 IRQ 栈 LDMFD 弹出 r0–r7
   SUBS pc, lr, #4  （或 LDMFD …, {…, pc}^）
   → PC 回到打字处；^ / SUBS 副作用：SPSR_irq → CPSR，模式回到 User
```

**为何比「全手动存」快：** Bank 已替你 **换好中断栈、存好返回地址、快照 CPSR** — 不必再把 **旧 SP、旧 LR、旧 CPSR** 压进内存。软件只需管 **仍共用的 GPR（r0–r7 等）**。

**口述一句：**

> 按保存键 = **User 身份被打断 → 切 IRQ 身份用专属栈和状态副本 → 压共用寄存器 → 干完活 → 弹栈 + 写回 CPSR/PC → 继续打字**。

### 端到端例子 2：刷视频 → 来电 FIQ → 挂电话 → SVC 改设置

**同一套「身份切换」讲法，换成手机日常 — 串起 User、FIQ、SVC 三种身份：**

| 阶段 | 模式 | 在干什么 |
|------|------|----------|
| 刷短视频 | **User** | APP 业务代码；**r0–r12** + **User 的 r13** 当栈 |
| 来电响铃 | **FIQ** | 基带等 **高优先级** 外设打断（教材用 FIQ 讲「快中断」） |
| 挂电话 | **User** | 从 FIQ 返回，继续刷视频 |
| 改后台设置 | **SVC** | APP 执行 **`SVC/SWI`** → 同步 trap 进内核做特权操作 |

> **现实手机：** 多为 **ARM64 + GIC**，来电未必走 **FIQ**（Linux 常把 FIQ 留给安全世界或不用）。此处是 **ARM7 经典模型** 的连贯比喻 — 抓 **FIQ 比 IRQ 少压 r8–r12** 即可。

**① 刷视频（User）**

```
CPSR.M = User
可见：r0–r12，SP = r13（User 栈），PC 在解码/渲染循环里
```

**② 基带拉高 FIQ — 硬件自动**

```
CPSR.M[4:0]  → FIQ
SP 视图      ← r13_fiq（FIQ 专属栈，与用户栈隔离）
SPSR_fiq     ← CPSR（含 User 模式位 — 挂电话后要恢复）
LR_fiq       ← 返回地址（Ch14：返回前 SUBS lr, #4）
```

**③ FIQ_Handler — 软件只压「仍共用」的 r0–r7**

```
r8–r12 已是 r8_fiq–r12_fiq 副本 → 不必压栈（FIQ 快的核心）
r0–r7 仍与 User 共用           → STMFD sp!, {r0-r7}（或更小 live 集）
执行：摘机/振铃/与基带通信等高优先级逻辑
```

**④ 挂电话 — 回到 User 刷视频**

```
LDMFD   sp!, {r0-r7}
SUBS    pc, lr, #4        ; PC 回视频代码，CPSR ← SPSR_fiq → 回 User
```

**不是** 单纯 `LR_fiq → PC` — **`SUBS`/`^` 才把 SPSR_fiq 写回 CPSR**，模式位一并恢复。

**⑤ 系统要改后台设置 — SVC（同步切换，不是外设打断）**

```
User 里 APP 调 settings API
    → 编译器生成 SVC #imm（系统调用号）
    → CPU 同步进 SVC 模式（不是 FIQ 那种异步中断）

硬件：SPSR_svc ← CPSR；LR_svc ← 返回址；SP ← r13_svc（内核栈）
内核：在 SVC 栈上处理 ioctl/write 等特权路径
返回：SUBS pc, lr, #0 或 #4（SVC 流水线偏移与 IRQ 不同 → Ch14）
    → 回到 User，APP 继续跑
```

**与 FIQ 对比（本例要点）：**

| | **FIQ（来电）** | **SVC（改设置）** |
|--|-----------------|-------------------|
| **触发** | 外设 **异步** 中断 | 程序 **主动** `SVC` 指令 |
| **优先级故事** | 高 — 少压栈、快返回 | 同步 trap — 走内核 syscall 路径 |
| **Bank** | r13_fiq, r14_fiq, SPSR_fiq + **r8–r12_fiq** | r13_svc, r14_svc, SPSR_svc |
| **软件压栈** | 通常只 **r0–r7** | 内核按 AAPCS 保存（Ch13） |

**口述一句：**

> **刷视频 = User 身份；来电 = 硬件切 FIQ 身份 + 只压 r0–r7；挂电话 = 弹栈 + SUBS 回 User；改设置 = APP 发 SVC 进内核身份，做完再回 User。** 每次切换，**Bank 替硬件「无缝交接」SP、返回地址、CPSR 快照** — 不必全靠软件逐寄存器搬。

### 副本寄存器（Bank）核心作用

**目标：** **减少中断时的压栈/出栈**，降低 **中断延迟**。

若所有模式 **共用同一套 r13、r14、r8–r12**，一进中断就得把正在用的 **全部通用寄存器** 先 **压栈**，返回再 **弹出** — 大量内存访问拖慢响应。  
硬件为异常模式准备 **副本寄存器**：**切模式 = 换一组硬件实体**，不必立刻写内存。

```
无 Bank（假想）          有 Bank（ARM7）
─────────────────       ─────────────────
异常 → 压满栈保存        异常 → 换 SP/LR/SPSR 视图
     → 慢、易混栈             → FIQ 连 r8–r12 都不用存
```

#### 1. 独立栈空间 — **r13 (SP) 副本**

每种异常模式有 **专属 SP**：`r13_svc`、`r13_irq`、`r13_fiq`、`r13_abt`、`r13_und`；**User/System 共用** `_usr` 栈。

| 场景 | 用的栈 |
|------|--------|
| User 跑应用 | **用户栈**（r13_usr） |
| 进 **IRQ** | 硬件切 **r13_irq** → **中断栈** |

**好处：** 中断 **不篡改用户栈**；栈 **隔离**，避免 ISR 把应用栈打爆；**不用** 进中断前手动改 SP。

→ 启动代码须为 **各模式 SP** 赋初值（[Ch14 Reset](../../chapter-14-exception-handling-arm7tdmi/notes/section-14-8-handler-code.md)）

#### 2. 自动保存返回地址 — **r14 (LR) 副本**

**r14** 存 **返回地址**。异常入口硬件把 **带修正的 PC** 写入 **当前模式的 LR_***。

| | |
|---|---|
| **有副本** | 中断处理完 **`SUBS pc, lr, #n`** 或 **`BX lr`** — LR 已是 **该模式私有** |
| **无副本** | 须先把 PC **压栈** 才能用 LR 做别的事 |

→ [Ch14 §14.4](../../chapter-14-exception-handling-arm7tdmi/notes/section-14-4-exception-sequence.md) 硬件四步

#### 3. 保存异常前 CPU 状态 — **SPSR 副本**

**CPSR** 全局唯一（N/Z/C/V、I/F 屏蔽、**T**、**M 模式位**）。  
每种 **异常模式** 有 **SPSR_***：

```
进入异常：SPSR ← CPSR（快照）
退出异常：CPSR ← SPSR（SUBS 写 PC 时附带）
```

**无 SPSR 副本时** 须 **手动把 CPSR 压栈** — 多指令、易错。  
**User/System 无 SPSR** — 它们不是「从异常返回」的模式。

#### 4. FIQ 专属 **r8–r12** — 高速实时

| 寄存器组 | IRQ / User / SVC / UND / ABT | FIQ |
|----------|------------------------------|-----|
| **r0–r7** | 共用 | 共用 |
| **r8–r12** | **共用一套** | **独立 r8_fiq–r12_fiq** |
| **r13–r14** | 各模式 bank | **FIQ 再 bank** |

**FIQ 场景（ADC/DMA/雷达）：** r0–r7 若够用可 **少压栈**；**r8–r12 完全不用保存** — **FIQ 比 IRQ 快的核心硬件原因**之一（另：更高优先级、向量可紧接 handler，Ch14）。

#### 5. 稳定性 — 上下文硬件隔离

**ABT**（内存 fault）、**UND**（非法指令）各自 **SP/LR/SPSR** 副本 — 崩溃路径 **少污染** 正常业务寄存器；与 **User 栈隔离** 同理，**异常间** 也靠 **不同 bank** 减少互相踩数据。

**注意：** Bank **不替代** 全部保存 — **r0–r7（及 IRQ 的 r8–r12）** ISR 仍常 **PUSH**（Ch13/Ch14）；Bank 是 **硬件已做的那一层**。

#### 6. 压栈分工 — 硬件 Bank vs 软件 PUSH

**切换模式时硬件自动做的（不靠栈）：**

| 动作 | 说明 |
|------|------|
| **CPSR → SPSR_*** | 进异常前状态 **快照** |
| **LR_*** ← 返回地址 | 异常发生点 PC（Ch14 再减 #4/#8） |
| **SP 视图切换** | 用当前模式的 **banked r13** |
| **M[4:0] 改写** | 进入 IRQ/FIQ/SVC/… |

**硬件不会替你保存的 — 若 ISR 会用到，须 **PUSH 到当前模式栈**，返回前 **POP** 恢复：**

| 寄存器 | IRQ / SVC / UND / ABT | FIQ |
|--------|------------------------|-----|
| **r0–r7** | **共用** → **必须手动压栈**（若会破坏） | 同左 |
| **r8–r12** | **共用** → **必须手动压栈** | **r8_fiq–r12_fiq 已 bank** → **不必压栈** |
| **r13 (SP)** | 已换 bank，**不用压** | 同左 |
| **r14 (LR)** | 已换 bank 且存返回址，**不用压旧 LR** | 同左 |
| **CPSR** | 已在 **SPSR**，**不用压** | 同左 |

**口述对照：**

> **Bank 管「换栈、存返回地址、存 CPSR 快照」；栈管「剩下的共用 GPR」。**  
> **FIQ 比 IRQ 快**，部分是因为 **5 个 r8–r12 免压栈** — 少 5 次 Store、少 5 次 Load。

典型 IRQ 入口骨架（Ch13/Ch14 展开）：

```arm
IRQ_Handler
    SUB     lr, lr, #4          ; 修正返回 PC
    STMFD   sp!, {r0-r12, lr}   ; 软件保存共用 GPR + 旧 lr（若用 AAPCS 可能只压子集）
    ; ... 处理中断 ...
    LDMFD   sp!, {r0-r12, pc}^  ; 恢复并 SUBS 写 PC → CPSR ← SPSR
```

FIQ 若 **只用 r0–r7 + r8_fiq–r12_fiq**，可 **只压 r0–r7**（甚至更小子集），handler 更短。

#### 一句话

> **副本寄存器 = 硬件自带的快速上下文缓存** — 用少量额外硅片，换 **少堆栈访问、低延迟中断、栈/返回地址/状态隔离**。

**与 M4 对照：** [§2.4](./section-2-4-cortex-m4.md) **硬件自动压 8-word 栈帧** — 思想类似（硬件帮保存），实现不同（无 7 模式 bank）。

### 寄存器组织 — 37 物理，模式可见 ~17

**程序员任意时刻「看得见」约 17 个**（r0–r15 + CPSR + 当前模式可用的 SPSR）；**硅片里物理实现共 37 个** — 不是每个模式复制整套 r0–r15。

#### 任意时刻可见（典型）

| 寄存器 | 别名 | 作用 |
|--------|------|------|
| **r0 – r12** | — | 通用（**FIQ 模式下 r8–r12 换为 _fiq 银行**） |
| **r13** | **SP** | 堆栈 — **按模式 bank** |
| **r14** | **LR** | 返回地址 — **按模式 bank** |
| **r15** | **PC** | 程序计数器 — 读 PC 常为 **当前+8**（流水线） |
| **CPSR** | — | **N Z C V** + I/F + **T** + **M[4:0]** |
| **SPSR** | — | **仅异常模式** 可见对应 **SPSR_***（User/System **无**） |

#### 37 个物理寄存器 — 分步推导

**前置：** 逻辑上每种模式都像有 r0–r15 + CPSR + SPSR；**硬件只 bank 切换时必须的 SP、LR、SPSR**，FIQ **额外 bank r8–r12**。

**① 全局共享（无副本）**

| 项目 | 数量 |
|------|------|
| **r0–r7** | 8 — 所有模式共用 |
| **CPSR** | 1 — 全局一份 |
| **小计** | **9** |

**② 非 FIQ 的 r8–r12 + User/System 的 SP/LR**

| 项目 | 数量 | 谁用 |
|------|------|------|
| **r8–r12**（_usr 组） | 5 | User、System、IRQ、UND、ABT |
| **r13、r14**（_usr 组） | 2 | **User 与 System 共用** 这一对 |
| **小计** | **7** |

**③ 各异常模式「私有副本」**

| 模式 | 私有物理寄存器 | 数 |
|------|----------------|-----|
| **SVC** | r13_svc, r14_svc, SPSR_svc | 3 |
| **IRQ** | r13_irq, r14_irq, SPSR_irq | 3 |
| **UND** | r13_und, r14_und, SPSR_und | 3 |
| **ABT** | r13_abt, r14_abt, SPSR_abt | 3 |
| **FIQ** | **r8_fiq–r14_fiq**（7 个）+ **SPSR_fiq** | **8** |
| **小计** | | **20** |

> **FIQ 为何是 8 而不是 7：** r8–r14 共 **7** 个 **_fiq** 副本，再加 **SPSR_fiq** 共 **8** 个 FIQ 专用物理寄存器（比其它异常模式多 **5 个 r8–r12** 副本）。

**④ PC — 全局唯一**

| **r15 (PC)** | **1** — 任何模式共用 |

**相加（教科书口径）：**

```
  r0–r7           8
+ CPSR            1
+ r8–r12 (_usr)   5
+ r13,r14 (_usr)  2
+ PC              1
+ SVC/IRQ/UND/ABT 12  (4×3)
+ FIQ 专用        8
─────────────────────
=                37
```

**等价速算（按 bank 种类）：**

| 种类 | 计算 | 小计 |
|------|------|------|
| r0–r7 | 8 | 8 |
| r8–r12 | 5 (_usr) + 5 (_fiq) | 10 |
| r13 (SP) | 1 (_usr/sys) + 5 (svc/irq/abt/und/fiq) | 6 |
| r14 (LR) | 同上 | 6 |
| r15 | 1 | 1 |
| CPSR | 1 | 1 |
| SPSR | 5（svc/irq/fiq/abt/und 各 1） | 5 |
| **合计** | 8+10+6+6+1+1+5 | **37** |

#### 关键结论（口述）

1. **不是** 7 模式 × 16 寄存器 — 只做 **最少 bank**，省面积。  
2. 编程时「换模式 = 换 SP/LR 视图」— **硬件切换副本**，不是软件复制。  
3. **FIQ** 多 **5 个 r8–r12** → 快中断少压栈（→ 上节 FIQ 条）。  
4. **37 = ARMv7-A/ARM7 32-bit 经典数**；**ARM64 无此分组**，用 **x0–x30 + SP/LR/PC + PSTATE**（[奔跑吧 Ch1](../../arm64-programming-practice/chapter-01-arm64-fundamentals/)）。

**通俗一句：** 为中断 **快切换、少保存**，硬件只给异常模式做 **SP/LR/SPSR（+ FIQ 的 r8–r12）** 副本，全部实体加起来 **固定 37 个**。

**Bank 与权限（复习）：**

- **权限：** User 隔离 APP — ≈ OS **用户态/内核态**
- **软件仍要：** 硬件 **不保存全部 r0–r12** — handler **PUSH**（Ch13 · Ch14）

### ARM32 七种模式 vs ARM64（勿混）

| | **ARM32（本节 · ARM7/v4T）** | **ARM64（AArch64）** |
|---|------------------------------|----------------------|
| 划分 | **7 种 CPSR 模式** | **4 级异常级别 EL0–EL3** |
| 用户/内核 | User vs 特权模式 | **EL0 / EL1**（+ EL2 虚拟化、EL3 安全 monitor） |
| 中断 | IRQ/FIQ 两套 | **IRQ @ EL1** · **GIC**（[奔跑吧 Ch13](../../arm64-programming-practice/chapter-13-gic-v2/)） |
| 学习 | Smith **Ch2 §2.3 + Ch14** | [奔跑吧 Ch1/11/13](../../arm64-programming-practice/) |

**口述：** 32 位 **7 模式** 与 64 位 **EL** 是 **两套模型** — 记 **思想**（权限 + 异常上下文），别硬套名字。

### 可复述要点

1. **CPSR.M[4:0]** 定 7 模式；**仅 User 非特权**。  
2. **物理寄存器 37 个** — 分步加总或 **8+10+6+6+1+1+5**；**非** 7×16 全复制。  
3. **副本寄存器**：**SP 栈隔离 · LR 返址 · SPSR 状态 · FIQ 的 r8–r12** — 降延迟。  
4. **Bank SP/LR/SPSR** 后 **r0–r12 仍常需软件 PUSH**（Ch13/Ch14）。  
5. **ARM32 7 模式 ≠ ARM64 EL** — 分开记。
