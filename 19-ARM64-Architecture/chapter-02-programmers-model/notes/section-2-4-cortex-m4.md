> **Ch 2 · 程序员模型** · [章导读](../README.md) · [本章概述](./section-0-本章完整概述.md)

### Cortex-M4 是什么（定位）

**Cortex-M4** 是 ARM 的 **32 位嵌入式处理器内核**（架构族 **ARMv7-M**），主打 **控制类 MCU + 适度数字信号处理**。Smith 本书实验平台之一；与 [§2.3 ARM7TDMI](./section-2-3-arm7tdmi.md) 对照看程序员模型。

| 项 | 说明 |
|----|------|
| **定位** | 高性能嵌入式控制；兼顾 **简单 DSP / 滤波 / 电机算法** |
| **相对 M3** | 同属 Cortex-M；M4 **多 DSP 指令集扩展**（SIMD、饱和、MAC 等），可选 **FPU**（单精度） |
| **典型设备** | 智能手表类穿戴、工业控制板、汽车电子 ECU 节点、电机驱动 |
| **常见 SoC** | **STM32F4** 等系列用 Cortex-M4（常带 FPU，称 **M4F**） |

**M3 ↔ M4 口述对照：**

| | **Cortex-M3** | **Cortex-M4** |
|--|---------------|---------------|
| **架构** | ARMv7-M | ARMv7-M（同源，程序员模型同一套 Thread/Handler） |
| **DSP** | 基础 | **更强** — 硬件乘加、SIMD、饱和算术 |
| **浮点** | 通常无硬件 FPU | **可选 FPU**（带 FPU 的实现叫 **Cortex-M4F**） |
| **用途** | 通用控制、低功耗 | 控制 + **音频/振动/电机** 等需少量浮点/DSP 的场合 |

**注意：**

- 「**比 M3 多了浮点**」——严格说是 **实现可选**；裸 **M4 可不带 FPU**，**M4F** 才有硬件浮点。日常口语里常把 STM32F4 上的 M4 当成「带 FPU 的那种」。
- M4 **不是** ARM7 的直接升级换皮：程序员模型从 **7 模式 Bank** 换成 **Thread/Handler + 硬件压栈**（下文），中断路径差别很大（→ [Ch15](../../chapter-15-exception-handling-cortex-m4/)）。

**口述一句：**

> **M4 = 控制向 32-bit MCU 内核；比 M3 更强 DSP，常配 FPU；手表/工控/车用电子常见；STM32F4 就是这颗内核。**

---

### 模式简化为两种 + 特权级别

| 概念 | 说明 |
|------|------|
| **Thread mode** | 运行普通应用代码 |
| **Handler mode** | 处理异常 / 中断（类似「总在特权异常上下文」） |

**特权划分：**

| 级别 | 能力 |
|------|------|
| **Privileged** | 访问 NVIC、MPU、CONTROL 等系统寄存器 |
| **User (Unprivileged)** | 受限访问 — 配合 RTOS 隔离任务 |

**CONTROL 寄存器** 可切换 Thread 下用 **MSP 还是 PSP**、以及 Thread 是否非特权 — 为小型 OS / 库设计留接口（→ **Ch15**）。

**对比 ARM7：** 不再 7 种模式 bank 一堆寄存器；**硬件自动压栈** + **统一 Handler** 模型，代码更省、中断延迟更可预测。

---

### 寄存器 — 16 个核心 + 可选浮点

**通用视图（与 ARM 经典一致的部分）：**

| 寄存器 | 作用 |
|--------|------|
| **r0 – r12** | 通用（参数/返回值惯例见 **Ch13 APCS**） |
| **r13** | **两个 SP**：**MSP**（主堆栈）· **PSP**（进程堆栈） |
| **r14** | **LR** — 异常时为 **EXC_RETURN**，不是普通返回地址 |
| **r15** | **PC** |

**双堆栈的意义：**

```
内核 / 异常处理  →  MSP
用户任务         →  PSP（RTOS 切换任务时换 PSP）
```

**Cortex-M4F 浮点扩展（本书 Ch9–11，可跳过）：**

- **s0 – s31** — 单精度浮点寄存器  
- **FPSCR** 等 — 浮点状态与控制  

---

### xPSR — 三合一状态寄存器

程序员常通过 **xPSR** 整体访问，逻辑上拆为三视图：

| 视图 | 全称 | 主要内容 |
|------|------|----------|
| **APSR** | Application PSR | **N Z C V** + 饱和标志等 |
| **IPSR** | Interrupt PSR | **当前异常编号**（Handler 中谁在执行） |
| **EPSR** | Execution PSR | Thumb 状态位等 — **M 系只跑 Thumb-2** |

**要点：** 读 `xPSR` 能同时知道 **条件标志 + 是否在异常里 + 哪号异常**。

---

### 异常向量表 — 存 **处理函数地址**

与 ARM7 **根本不同**：

| | ARM7TDMI | Cortex-M4 |
|---|----------|-----------|
| 表项内容 | **`B handler` 指令** | **Handler 函数地址（字）** |
| 类比 | ARM-A 经典 | 8051 中断向量风格 |
| 首项 | Reset 跳转 | **初始 SP** + **Reset_Handler 地址** |

**Thumb 地址规则：** M 系 **仅 Thumb-2**；向量表中的地址 **LSB 必须为 1**（标记 Thumb 状态），硬件取址时清 LSB。

**典型表头两项：**

```
偏移 0x00:  初始 MSP 值
偏移 0x04:  Reset_Handler 地址（LSB=1）
偏移 0x08+:  NMI, HardFault, SVCall, PendSV, SysTick, IRQ0…
```

**Ch15 详述：** NVIC、优先级、硬件压栈帧 — 本书 MCU 中断主线。

---

### ARM7 vs Cortex-M4 对照表

| 主题 | ARM7TDMI | Cortex-M4 |
|------|----------|-----------|
| 模式 | 7 种 + bank | Thread / Handler + 特权位 |
| SP | 按模式 bank | **MSP + PSP** |
| 向量表 | 分支指令 | **函数指针 + 初始 SP** |
| 指令集 | ARM + Thumb | **Thumb-2 only** |
| 对齐 | 严格 | 可配置非对齐 |

---

### 可复述要点

1. **M4 = Thread/Handler + Priv/User**，靠 CONTROL/MPU 做隔离。  
2. **MSP/PSP** 分离内核栈与用户/任务栈。  
3. **向量表放地址，LSB=1**；第一项是 **栈顶** 不是跳转指令。  
4. 本书大部分实验在 **Cortex-M4** 上 — Ch3 示例以 Thumb-2 为主。
