## 1. 数据宽度与对齐（§2.2）

### 知识点

`LDR Rd,[Rn]` 读 **32 位字**。经典 ARM **严格对齐**：地址须为 **4 的倍数**（低 2 位 = `00`，`addr % 4 == 0`）。非对齐直接字访问 → fault / 未定义行为（教材按严格对齐判）。

> M4 可配置允许非对齐；**本考点按 §2.2 / ARM7 严格规则答。**

### 例题

判断能否直接 `LDR r0,[r1]`（字读）：

| 地址 | 计算 | 结论 |
|------|------|------|
| `0x20000000` | 0 % 4 = 0 | **可以** |
| `0x20000004` | 4 % 4 = 0 | **可以** |
| `0x20000001` | 1 % 4 = 1 | **不可以** |
| `0x20000006` | 6 % 4 = 2 | **不可以** |

## 2. 模式与寄存器（§2.3 ARM7TDMI）

### 知识点

- **7 种模式**；其中 **FIQ / IRQ / SVC / Abort / Undefined** 有 **独立 banked SP**
- **User 与 System 共用** 同一组 r13/r14（_usr）
- 异常自动切模式：

| 触发 | 模式 |
|------|------|
| `SWI` / `SVC` | **SVC** |
| 普通外设中断 | **IRQ** |
| 快速中断 | **FIQ** |
| 取指 / 数据访问出错 | **Abort** |
| 非法指令 | **Undefined** |

### 例题

**Q1.** 发生普通外设 IRQ，CPU 进什么模式？该模式 SP 是否与 User 共用？  
**A：** **IRQ**；SP 是 **banked `r13_irq`**，**不与** User 共用。

**Q2.** 哪两种 ARM7 模式共享同一个 SP？  
**A：** **User** 与 **System**。

## 3. CPSR · N/Z/C/V（§2.3）

### 标志定义

| 位 | 含义 |
|----|------|
| **N** | 结果最高位 = 1（有符号负）→ 1 |
| **Z** | 结果全 0 → 1 |
| **C** | 加法：进位；**减法：C = ¬借位**（有借位则 **C=0**） |
| **V** | 有符号溢出 → 1 |

⚠ **ARM 减法 C 标志陷阱：** 「产生借位」≠「C=1」。有借位时 **C 清 0**。

### 例题

`SUBS r0, r0, #1`，r0 初值 `0x00000000`：

1. 运算：`0 - 1 = 0xFFFFFFFF`
2. 标志：

| 位 | 值 | 理由 |
|----|-----|------|
| **N** | **1** | 最高位 1 |
| **Z** | **0** | 结果非 0 |
| **C** | **0** | 发生借位 → ARM 规定 C=0（**不是 1**） |
| **V** | **0** | 有符号 0→−1，无溢出 |

> 易错：把「有借位」写成 C=1。背：**减法有借位 → C=0**。

## 4. 向量表格式对比（§2.3 vs §2.4）

### 例题

**Q1.** ARM7TDMI 向量表 **偏移 0x04** 存放什么？  

**A：** **`B` 到 Undefined 处理**（未定义指令向量），**不是** Reset。

ARM7 表头（存 **跳转指令**）：

| 偏移 | 异常 |
|------|------|
| **0x00** | **Reset** → `B Reset_Handler` |
| **0x04** | **Undefined** |
| 0x08 | SWI |
| … | … |

⚠ 易错：把 0x04 当成 Reset — Reset 在 **0x00**。

**Q2.** Cortex-M4 向量表 **0x00、0x04** 各存什么？函数地址 LSB 为何置 1？  

**A：**

| 偏移 | 内容 |
|------|------|
| **0x00** | 初始 **MSP**（栈顶） |
| **0x04** | **Reset_Handler 地址** |

LSB=1 → 标记 **Thumb**（M 仅 Thumb-2）；硬件取址时 **清 LSB** 得真实函数地址。

## 5. MSP / PSP 双堆栈（§2.4）

### 例题

**Q1.** 什么场景用 MSP？什么场景用 PSP？

| | |
|--|--|
| **MSP** | 复位初始化、**Handler**（中断/异常）、内核代码 |
| **PSP** | **Thread** 下普通用户 / RTOS 业务任务 |

**Q2.** RTOS 切任务口述（双栈视角）：

1. 当前任务触发 **PendSV** → **Handler**，强制 **MSP**  
2. 内核把任务现场保存到 **该任务的 PSP**（硬件已可能压过 8-word 帧；软件再存 R4–R11 等 → [Ch15](../../chapter-15-exception-handling-cortex-m4/)）  
3. OS 改 **PSP** 指向下一就绪任务栈  
4. 异常返回 → **Thread**，新任务用自己的 **PSP** 跑  
5. 内核 / 中断全程靠 **MSP**，任务栈彼此隔离
