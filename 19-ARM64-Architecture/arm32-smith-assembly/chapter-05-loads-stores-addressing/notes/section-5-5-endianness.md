## §5.5 字节序 (Endianness)

> **Ch 5 · 加载、存储与寻址** · [章导读](../README.md)

---

### 问题：一个 Word 占 4 个连续字节，谁在低地址？

32 bit 值 `0x12345678` 在地址 `0x1000` 起：

**小端 Little-Endian（多数 ARM 默认）：**

```
地址:    0x1000  0x1001  0x1002  0x1003
字节:      78      56      34      12
           ↑ LSB 在最低地址
```

**大端 Big-Endian：**

```
地址:    0x1000  0x1001  0x1002  0x1003
字节:      12      34      56      78
           ↑ MSB 在最低地址
```

| 模式 | 记忆 |
|------|------|
| **Little** | **低地址存低位字节** — x86、ARM 默认 |
| **Big** | **低地址存高位字节** — 网络字节序、部分通信协议 |

---

### 对 Load 的影响

| 操作 | 注意 |
|------|------|
| **`LDR` 字** | CPU 按 **当前 endian 设置** 组装 32 bit 到寄存器 |
| **`LDRB`/`LDRH`** | 只读 **指定地址** 的 1/2 字节 — **与字内字节序无关** |
| **跨平台协议** | 线上常 Big；主机常 Little — 需 **显式转换** |

**网络编程对照：** `htonl`/`ntohl` — 与 REV 同类问题。

---

### 字节序转换 — `REV` 等（Cortex-M）

在寄存器内 **翻转字/半字** 字节序，无需多次 `LDRB`/`STRB`：

| 指令 | 作用 |
|------|------|
| **`REV`** | 字内 4 字节反转 |
| **`REV16`** | 各半字内字节反转 |
| **`REVSH`** | 低半字符号扩展 + 字节反转 |

**软件法（无 REV）：** 移位 + `AND`/`ORR` 手动重排 — Ch7 位操作。

**ARM7 时代：** 可能靠 **CP15** 配置 endian 状态；M 系启动后通常 **固定 little**。

---

### 驱动 / HFT 场景

| 场景 | Endian |
|------|--------|
| **MMIO 寄存器** | 按 **SoC 手册** — 勿假设与 CPU 相同 |
| **DMA 缓冲区** | 设备与 CPU 不一致时要 REV 或 swab |
| **网络报文** | Big-endian 字段 — 与 [14 DPDK](../../../14-DPDK-Network-Programming/) 同类 |

---

### 可复述要点

1. **Little = 低地址存 LSB** — ARM 主机默认。  
2. **`LDRB` 读单字节不受「字内序」影响**；**`LDR` 字** 受系统 endian 影响。  
3. **跨 endian 通信用 `REV` 或 htons 类函数**。
