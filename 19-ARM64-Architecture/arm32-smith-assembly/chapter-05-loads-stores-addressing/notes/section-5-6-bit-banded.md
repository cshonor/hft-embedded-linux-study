## §5.6 位带内存 (Bit-Banded Memory) — Cortex-M

> **Ch 5 · 加载、存储与寻址** · [章导读](../README.md)  
> **标签：** **选读** — **Cortex-M3/M4 专用**；Linux/Cortex-A **无** 此机制

---

### 传统问题：改 1 bit = 读-改-写

```asm
        LDR     r0, [GPIO_BASE]       ; 读整字
        ORR     r0, r0, #BIT3         ; 改一位
        STR     r0, [GPIO_BASE]       ; 写回整字
```

| 风险 | 说明 |
|------|------|
| **非原子** | 中断插在 R-M-W 中间 → 丢写 |
| **副作用** | 某些寄存器 **写 1 清 0** — 读回再写会误清其他位 |
| **周期** | 3 条指令 + 2 次总线访问 |

---

### 位带 (Bit-Banding) 思路

硬件把 **SRAM / 外设** 某区域内 **每一个 bit** 映射到 **独立的一个 32 bit 别名地址**：

```
真实 bit 地址空间          别名区（每 bit → 一个 word 地址）
     │                              │
     └──────── 硬件映射 ────────────┘
              STR alias, #1   →  硬件只置位对应那 1 bit
              STR alias, #0   →  硬件只清零那 1 bit
```

**程序员：** 对 **别名地址** 做一次 **`STR` 字** — 等价 **单 bit 写**，无需 R-M-W。

---

### 映射公式（概念）

Cortex-M **位带别名区** 与 **位带位区** 有固定线性关系（见 ARM 文档 / Tiva 手册）：

```
alias_addr = bit_band_base + (byte_offset × 32) + (bit_number × 4)
```

**仅特定区域可位带** — 不是任意地址；需查 **Memory Map**。

---

### 与 Linux / 飞控路径

| MCU 本书 | Cortex-A + Linux |
|----------|-------------------|
| 位带 GPIO | **`set-bit`/`clear-bit` MMIO**、或 **写 1 置位寄存器 (BSRR)** |
| 无 OS | 驱动仍常用 **专用 W1S/W1C 寄存器** 代替 R-M-W |

**结论：** 理解 **R-M-W 问题** 必记；**位带** 为 M3/M4 优化手段，**应用处理器笔记可跳过实现细节**。

---

### 可复述要点

1. **位带 = 1 bit ↔ 1 个 32 bit 别名地址**，单 `STR` 改一位。  
2. 解决 **R-M-W 非原子与 W1C 误写** — M 系特色。  
3. Cortex-A/Linux 用 **别的硬件机制**（BSRR、shadow reg）达同类目的。
