## §8.4 条件执行 — ARM 后缀 vs IT 块

> **Ch 8 · 分支与循环** · [章导读](../README.md)

---

### 动机：少分支，少 flush

**条件执行：** 指令 **进入流水线**，条件不满足时 **当作 NOP 执行**（不写寄存器/不改内存）— 避免 **B 跳过** 整条指令槽。

适合 **短 if 分支**（1–4 条）— 长分支仍用 **B**。

---

### v4T / ARM7 — 指令条件后缀

**32 bit ARM 指令** 高 4 bit = **条件码**（默认 **AL**=Always）：

```asm
        CMP     r0, r1
        SUBGT   r2, r2, #1        ; 仅 GT 时 r2--
        ADDLE   r3, r3, #1        ; 仅 LE 时 r3++
        STRNE   r0, [r4]          ; 仅 NE 时存
```

| 优点 | 限制 |
|------|------|
| 单条指令粒度条件化 | **Thumb 16 bit** 多数无条件 |
| 历史 ARM 性能亮点 | M 系主用 Thumb-2 + IT |

**Ch3 阶乘 ARM 版：** `MULGT` / `SUBGT` — 下面展开 **GT** 规则。

#### 深挖：MULGT / SUBGT（阶乘用的那对）

| 指令 | = | 假时 |
|------|---|------|
| `MULGT Rd, Rm, Rs` | 仅 **有符号 >** 时 `Rd = Rm×Rs` | 不乘、不写 Rd |
| `SUBGT Rd, Rn, Op2` | 仅 GT 时做减法 | 不减、不写 Rd |
| `BGT label` | 仅 GT 时跳转 | 顺序下一条 |

**GT 标志条件：** `Z==0 && N==V`（`CMP a,b` 后 ≈ **有符号 a > b**）。

```asm
        CMP     r0, #1
        MULGT   r1, r1, r0
        SUBGT   r0, r0, #1
        BGT     loop
; r0==1 时三条全“空过”，循环自然结束
```

**勿混：** M4 / Thumb-2 **不能** 像 ARM32 那样每条指令随意加后缀 — 须包在 **IT 块**里（下节）。

---

### v7-M — IT 块 (IF-THEN)

Thumb-2 **16/32 bit 混编** — 不能每条带 4 bit 条件 → 用 **`IT` 修饰接下来 1–4 条**：

```asm
        CMP     r0, r1
        IT      GT
        SUBGT   r2, r2, #1
        IT      LE
        ADDLE   r3, r3, #1
```

**`IT` 掩码字母：**

| 字母 | 含义 |
|------|------|
| **T** (Then) | 条件 **真** 时执行 |
| **E** (Else) | 条件 **假** 时执行 |

**例：`ITTE LT`**

```
条件 LT 成立 → 第 1、2 条执行 (TT)
条件 LT 不成立 → 第 3 条执行 (E，等价 GE)
第 4 条：由掩码第 4 位决定
```

**规则（书中强调）：** **E 的条件 = T 条件的逻辑非** — 如 LT 配 GE，EQ 配 NE。

---

### IT 块约束

| 约束 | 说明 |
|------|------|
| **最多 4 条** | 含 IT 后指令 |
| **块内指令必须带匹配条件后缀** | 与 IT 掩码一致 |
| **块内勿插入分支/IT** | 调试困难 |
| **无 CBZ/CBNZ** | §8.2 |
| **M3+ 调试** | 单步 IT 需看清掩码 |

**Ch3 M4 阶乘** — 已见 **IT GT** + `MULGT`/`SUBGT`。

---

### 何时 IT vs B

| 选 IT / 条件后缀 | 选 B |
|------------------|------|
| **1–4 条** 小分支 | 长 basic block |
| 避免 predict miss 的小代价 | 远距离跳 |
| 热路径 micro-optimization | 清晰控制流优先可读性 |

**HFT：** 极短 critical section 偶见条件 mov；多数仍靠 **无分支算法 + 展开**。

---

### 可复述要点

1. **ARM 模式：每条可带 EQ/GT/… 后缀**。  
2. **M 模式：IT + 最多 4 条条件后缀指令**；**T/E 掩码**。  
3. **目的：减少分支 flush** — 不是取消分支需求。
