## §13.3 子程序

> **Ch 13 · 子程序与堆栈** · [章导读](../README.md) · [本章概述](./section-0-本章完整概述.md)

---

### 调用：`BL`

```asm
    BL      DelayMs        ; LR ← 下一条指令地址；PC ← DelayMs
```

| 寄存器 | 调用后 |
|--------|--------|
| **LR (r14)** | **返回地址**（`BL` 的下一条） |
| **PC** | 子程序入口 |

**返回（简单情形）：** 子程序未再调用他人、未改 LR：

```asm
    BX      lr             ; 或 MOV pc, lr
```

一旦子程序内部还有 **`BL`**，**LR 会被覆盖** — 必须先 **保存 LR**（通常压栈）。

---

### 可重入性 (Reentrant)

**可重入** 子程序满足：

- 可被 **中断** 打断，返回后继续正确  
- 可被 **递归** 或 **嵌套** 调用（同一函数在栈上有多帧）  
- **不依赖固定 RAM 全局** 作唯一状态（或用栈上局部）

**入口铁律：** 在使用前，把 **将要改写的寄存器 + LR** 压栈：

```asm
DelayMs
    STMDB   sp!, {r4-r7, lr}    ; 序言 (prologue)
    ; … 使用 r4-r7，可能 BL 其他函数 …
    LDMIA   sp!, {r4-r7, pc}    ; 尾声 (epilogue) — pc 从栈取原 LR
```

**`pc` 在寄存器列表中：** 硬件把弹出的 **LR 值写入 PC** — **一条指令返回**。

---

### 序言 / 尾声模板

```
调用者 BL Foo
        ↓
Foo:    保存 callee-save + LR
        读参数 (r0-r3 / 栈)
        计算
        恢复寄存器 + PC←LR
        ↓
调用者继续
```

**错误示范：** 在 `Foo` 里用 **r4** 但不保存 — 返回后调用者的 **r4 被破坏**（违反 AAPCS）。

---

### 嵌套调用与 LR

```
main  BL  A      ; LR = 返回 main
A:      保存 LR
        BL  B    ; LR = 返回 A（覆盖 A 的入口 LR！）
B:      … BX lr
A:      恢复 LR，返回 main
```

**口述：** 只要子程序里还有 **`BL`**，**LR 必须进栈**（或换用其他保存策略）。

### 可复述要点

1. **`BL`** 设 **LR**；复杂子程序 **入口压栈 LR + 用到的 r4-r11**。  
2. **`LDMIA sp!, {…, pc}`** = 恢复 + 返回，一条搞定。  
3. **可重入** = 状态在 **栈帧** 上，而非偷偷改调用者寄存器。
