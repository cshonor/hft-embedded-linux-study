## §3.5 示例程序 3 — 交换寄存器 (XOR Swap)

> **Ch 3 · 指令集简介** · [章导读](../README.md)

---

### 本程序学什么

| 目标 | 内容 |
|------|------|
| **无临时寄存器交换** | 三次 `EOR` 互换 rA、rB |
| **大立即数** | `LDR rd, =constant` 伪指令 |

---

### XOR 交换技巧

```asm
        EOR     r0, r0, r1      ; r0 ^= r1
        EOR     r1, r0, r1      ; r1 ^= r0  (原 r0)
        EOR     r0, r0, r1      ; r0 ^= r1  (完成交换)
```

| 性质 | 说明 |
|------|------|
| **无需 r2 / 内存** | 省寄存器 — 极端 constrained 时有用 |
| **前提** | r0、r1 **不是同一物理寄存器**；且地址 distinct |
| **现代实践** | 寄存器够用时 **`MOV` + 临时寄存器** 更清晰 — 编译器常如此 |

**Ch7：** `EOR` 也是通用按位逻辑；与 `AND`/`ORR`/`BIC` 同族。

---

### LDR = 大常数 伪指令

```asm
        LDR     r2, =0x12345678   ; 汇编器展开为 MOVW/MOVT 或 literal pool
```

| | `MOV #imm` | `LDR rd, =expr` |
|---|------------|-----------------|
| 范围 | 小立即数（8 bit 旋转等） | **任意 32 bit 常数/地址** |
| 实现 | 单条（若可编码） | 可能 **PC 相对 load 文字池**（→ **Ch6**） |

**驱动/MMIO：** 外设基址 `0x40000000` 类常数几乎总是 `LDR =` 或 `MOVW/MOVT`。

---

### 可复述要点

1. **三次 XOR** 可交换两寄存器 — 面试/习题经典。  
2. **`LDR rd,=imm`** 是伪指令，汇编器负责展开。  
3. 可读性优先时，**用临时寄存器交换** 往往更好。
