### 作者强调的两条习惯

### 1. 复杂逻辑先画流程图

| 为什么 | 说明 |
|--------|------|
| 汇编 **无结构化语法** | 没有 `if/for` 块，只有标签与分支 |
| 先算法后指令 | 流程图 → 基本块 → 标签名/`B`/`BL` |
| 与 Ch8 衔接 | 循环结构（While/For/Do-While）在 **Ch8** 系统化 |

**HFT 对照：** 热路径手写汇编前同样先 **控制流图** — 与 [16 HFT](../../../16-HFT-Low-Latency/) 优化流程一致。

---

### 2. 高度重视初始化

| 对象 | 风险 |
|------|------|
| **通用寄存器** | 复位后 **未定义** — 第一次使用前 `MOV #0` 或已知值 |
| **RAM 变量** | `.bss` 未清零若链接脚本遗漏 → 随机初值 |
| **内存映射外设寄存器** | 复位后位域可能 **随机/上电默认值** — 直接读改写可能 **锁死硬件**（→ **Ch16 MMIO**） |

**典型启动序列：**

```
设置 SP（向量表/启动代码）
    ↓
清零 BSS / 拷贝 DATA（C 运行时）
    ↓
初始化时钟/GPIO 等外设
    ↓
进入 main 或主循环
```

**Linux 内核/U-Boot：** 同样 **early init** — 时钟、DRAM、串口；概念与 MCU **reset handler** 平行（→ [20 构建](../../../../20-UBoot-Kernel-Build/)）。

### 可复述要点

1. **先流程图，后标签与分支**。  
2. **寄存器、变量、外设** 使用前必须有 **已知初态**。  
3.  uninitialized MMIO = 驱动里 **最难查的 bug** 之一。
