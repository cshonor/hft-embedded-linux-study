## §1.3 计算设备 — 存储程序模型与计算层级

> **Ch 1 · 计算机系统概述** · [章导读](../README.md)

---

### 存储程序计算机（Stored-program computer）

经典模型三要素：

```
        ┌─────────────┐
        │  处理器 CPU  │  取指 → 译码 → 执行
        └──────┬──────┘
               │ 总线 (Address / Data / Control)
        ┌──────┴──────┐
        │    内存 RAM   │  指令 + 数据 同存于地址空间
        └─────────────┘
```

| 组件 | 作用 |
|------|------|
| **处理器** | 按 PC 从内存取指令，修改寄存器与内存 |
| **内存** | 统一存放**程序（指令序列）**与**数据** — 「程序即数据」 |
| **总线** | 地址线选单元、数据线传内容、控制线协调读写时序 |

**与 MikanOS 对照：** [08 MikanOS](../../../../08-system-low-level-hands-on/01-mikan-os/) 里 UEFI 加载 `kernel.elf` 到内存再跳转 — 同样是「内存里先有比特，CPU 从某入口地址开始执行」。

---

### 计算机技术层级（自底向上）

```
物理层     晶体管 → 逻辑门
           ↓
微架构     流水线、缓存、分支预测（[03 Hennessy](../../../../03-Computer-Architecture-6th/) Ch2）
           ↓
ISA        指令集架构 ← 【汇编程序员工作在这一层】
           ↓
高级语言   C / C++（[02 C](../../../../02-c-programming/)）
           ↓
操作系统   Linux / RTOS（[04 LKD](../../../../04-Linux-Kernel-Development/)）
```

| 层级 | 你写什么 | 例子 |
|------|----------|------|
| **ISA** | 汇编 `.S` | `LDR r0, [r1]` · `ADD r2, r0, r3` |
| **高级语言** | `.c` | `x = a + b;` — 编译器生成 ISA |
| **OS** | 系统调用 | `read()` → 内核帮你在用户态/内核态间切换 |

**Ch1 要点：** 汇编不是「比 C 低级一点的语法糖」，而是 **直接描述 ISA 层** — 每条助记符对应（或展开为）确定的比特模式（→ §1.6）。

---

### 「计算设备」在本书语境

- 从 **嵌入式 MCU 板**（Tiva Launchpad 等）到 **通用存储程序机**，硬件形态不同，**模型相同**。
- **外设**（UART、GPIO）在 Cortex-M 上通常 **内存映射** — 特定地址读写即控制硬件（→ **Ch16**）。

---

### 可复述要点

1. 冯·诺依曼：**指令和数据共用内存 + 顺序（或分支）改 PC**。
2. 汇编站在 **ISA 层** — 之上是 C/ OS，之下是微架构与晶体管。
3. 总线连接 CPU 与内存/外设；MMIO 外设也占用地址空间。
