## §1.5 数字与字符的表示 — 整数 · 浮点 · 字符

> **Ch 1 · 计算机系统概述** · [章导读](../README.md)

---

### 整数表示

| 方式 | 要点 | 现代地位 |
|------|------|----------|
| **无符号** | 纯 magnitude，范围 0 … 2ⁿ−1 | 地址、计数、位图 |
| **原码 (Sign-magnitude)** | 最高位符号，其余绝对值 | 历史/概念；±0 问题 |
| **反码 (One's complement)** | 正数不变，负数按位取反 | 几乎仅教学 |
| **补码 (Two's complement)** | 负数 = 对应正数取反 + 1 | **ARM 有符号整数默认** |

**补码为何是核心：**

- **唯一零**；加减可同一套硬件（减法变加法）。
- 溢出时 **N/V/C/Z 标志位** 有明确定义（→ **Ch7**）。
- 位宽固定：ARM 32 位寄存器里 `0xFFFFFFFF` 作为 **有符号** 是 −1。

```
例（8 bit 示意）：+3 = 0000 0011
                  −3 = 1111 1101  （取反+1）
```

**与 C 对照：** `int32_t x = -1;` 在内存里就是全 1 比特模式 — [02 C](../../../../02-c-programming/) 与汇编读同一块内存。

### 浮点数表示（IEEE 754 简介）

解决 **极大/极小** 与 **小数** 的动态范围问题；Ch9–11 整章展开，Ch1 只建立直觉。

**单精度（32 bit）结构：**

```
| S |   指数 E (8)   |        尾数 / 分数 F (23)        |
 1 bit     偏置 127              隐含 leading 1
```

| 字段 | 含义 |
|------|------|
| **S** | 符号 0/1 |
| **E** | 指数（偏置编码，非补码） |
| **F** | 有效数字的小数部分 |

**特殊值（Ch1 预告，Ch9 详述）：** ±0、次正规数、±∞、NaN。

**本书路径：** 嵌入式 Linux / 飞控若用 **Cortex-A + NEON**，浮点主要在 C/NEON；**Cortex-M4F** 才强依赖 Ch9–11 — 可 **跳过** 浮点三章（见 [OUTLINE](../../OUTLINE.md)）。

### 字符表示 — ASCII

| 要点 | 说明 |
|------|------|
| **7 bit ASCII** | `'A'=0x41` · `'0'=0x30` · 可打印字符 + 控制符 |
| **汇编用法** | 数据段 `.byte 'H','i',0` 或 `.ascii` / `.string`（→ **Ch4** 伪指令） |
| **扩展** | UTF-8 等是字节序列上的超集；裸机常先 ASCII |

**调试：** 内存里看到 `48 65 6C 6C 6F` → `Hello` 的 ASCII hex。

附录 [C — ASCII 表](../../appendix-C-ascii-character-codes/) 可速查。

### 可复述要点

1. **有符号整数 = 补码** — 与 ARM 算术、标志位一体。
2. **浮点 = 符号 + 指数 + 尾数** — Ch1 建立框图即可。
3. **字符 = 整数比特模式** — `'A'` 就是 0x41，字符串即字节数组。
