## §1.7 工具 — 源码到可执行文件的流程

> **Ch 1 · 计算机系统概述** · [章导读](../README.md)

---

### 工具链（Tools Flow）

```
汇编源码 (.s / .asm)
    ↓  汇编器 Assembler
目标文件 (.o / .obj)     ← 机器码 + 重定位信息 + 符号表
    ↓  链接器 Linker
可执行映像 (.elf / .axf)  ← 固定入口地址、段布局、外设脚本
    ↓  调试器 / 烧录器
板卡 Flash / QEMU 运行
```

| 工具 | 作用 |
|------|------|
| **汇编器** | 助记符 → 机器码；处理伪指令、符号、段（→ **Ch4**） |
| **链接器** | 合并多个 `.o`，解析符号，按 **链接脚本** 排布 ROM/RAM（→ **Ch9** 奔跑吧 · [20 构建](../../../../20-UBoot-Kernel-Build/)） |
| **编译器** | C → 汇编或目标码；**混合编程**时与汇编 `.o` 一起链接（→ **Ch18**） |
| **调试器** | GDB / IDE 单步、看寄存器与内存 |

**目标文件：** 还不是完整程序 — 含 **未解析的外部符号**（如 `main` 调用的 `delay`），链接后才得到最终地址。

### 与嵌入式 Linux 工具链的对应

| MCU 本书 | 应用处理器（树莓派 / 飞控 SoC） |
|----------|----------------------------------|
| `arm-none-eabi-gcc` 裸机 | **`aarch64-linux-gnu-gcc`** 用户态 / 内核模块 |
| 链接脚本定 SRAM/Flash | U-Boot + **设备树** 定内存与外设（→ [22 DT](../../../22-Device-Tree-Study/)） |
| 烧录 hex/bin | SD 卡分区 + **Bootloader 链**（→ [20](../../../../20-UBoot-Kernel-Build/)） |

概念相同：**汇编/编译 → 链接 → 加载执行**；只是运行环境从「裸机」变成「Bootloader + Linux」。

---

### 可复述要点

1. **汇编器 ≠ 链接器** — 前者做单文件翻译，后者做全局地址与符号解析。
2. Keil/CCS 是 **集成环境**；GNU 命令行是 Linux/嵌入式工业界常态。
3. Ch1 工具流在 **Ch4（伪指令）→ Ch18（C/汇编链接）→ 模块 20（Yocto/内核构建）** 逐层加深。
