## §6.4 文字池 · LTORG · MOVW/MOVT

> **Ch 6 · 常量与文字池** · [章导读](../README.md)

---

### 文字池 (Literal Pool) 是什么

当常数 **无法** 用 §6.2 单条 `MOV/MVN` 编码时，汇编器在 **代码段附近** 插入一块 **只读数据**：

```asm
        LDR     r0, =0x12345678
        ; ... 汇编器在池里生成 ...
        ; Literal pool:
        ;   DCD 0x12345678
        ; 并生成: LDR r0, [pc, #offset_to_pool]
```

| 属性 | 说明 |
|------|------|
| **位置** | 通常 **当前段末尾** 或 **`LTORG` 强制处** |
| **内容** | `DCD` / `.word` 形式的 32 bit 常数 |
| **访问** | **PC 相对 LDR** — offset 有限 |

---

### PC 相对寻址的 ±4KB 限制

ARM 经典 **`LDR [pc, #imm12]`** 偏移范围约 **±4KB**（具体视指令与对齐）。

**问题：** 代码很长时，**默认堆在文件末尾的文字池** 可能离某条 `LDR =` **太远** → 汇编 **报错**（offset 溢出）。

---

### LTORG — 强制就近建池

```asm
        B       skip_pool      ; 无条件跳，避免 CPU 执行池里数据
        LTORG                  ; 在此插入 literal pool
skip_pool
        ; 继续代码...
```

| 习惯位置 | 原因 |
|----------|------|
| **无条件分支之后** | 池不会被当作指令执行 |
| **子程序末尾** | 本函数内常数就近 |
| **长函数中间** | 分段建池 |

**Ch4 回顾：** [§4.4 LTORG](../../chapter-04-assembler-rules-directives/notes/section-4-4-directives.md) · GNU `.ltorg`

### MOVW / MOVT — 无文字池加载任意 32 bit（Thumb-2 / M4）

```asm
        MOVW    r0, #0x5678        ; r0[15:0]  ← 0x5678
        MOVT    r0, #0x1234        ; r0[31:16] ← 0x1234  → r0 = 0x12345678
```

| 指令 | 作用 |
|------|------|
| **`MOVW`** | **Move Wide** — 16 bit 立即数 → 寄存器 **低 16 位**（高 16 清零） |
| **`MOVT`** | **Move Top** — 16 bit 立即数 → **高 16 位**（低 16 保留） |

**优点：** 任意 32 bit 常数 **两条指令**、**无文字池**、无 PC 相对距离问题。  
**代价：** 占 8 字节指令空间 vs 池+单 LDR 可能更省或更费 — 视密度而定。

**ARM7 纯 ARM 状态：** 无 MOVW/MOVT — 更依赖 **`LDR =`** + 池。

**AArch64：** `movz x0, #0x5678` · `movk x0, #0x1234, lsl #16` — 同模式 64 bit。

---

### 三种方式对比

| 方式 | 指令数 | 需要池 | 典型平台 |
|------|--------|--------|----------|
| **MOV/MVN 旋转立即数** | 1 | 否 | 小常数，全架构 |
| **LDR =** → 池 | 1+LDR | 是 | 任意常数，通用 |
| **MOVW+MOVT** | 2 | 否 | Cortex-M / Thumb-2 |

---

### 可复述要点

1. **文字池** = 汇编器放的 **`.word` 常数表** + PC 相对 LDR。  
2. **代码过长用 `LTORG`** 分段建池，避免 ±4KB 溢出。  
3. **MOVW/MOVT** = 两条指令拼任意 32 bit — M4 常用，替代池。
