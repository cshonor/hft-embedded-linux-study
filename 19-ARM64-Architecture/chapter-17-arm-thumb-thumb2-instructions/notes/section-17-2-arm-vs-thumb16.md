## §17.2 ARM 与 16 位 Thumb 指令

> **Ch 17 · ARM、Thumb 和 Thumb-2 指令** · [章导读](../README.md) · [本章概述](./section-0-本章完整概述.md)

---

### 32-bit ARM 指令

| 属性 | 说明 |
|------|------|
| **宽度** | 固定 **32 bit** |
| **条件执行** | **`ADDEQ`/`SUBGT`** 等 — [Ch8 §8.4](../chapter-08-branches-loops/notes/section-8-4-conditional.md) |
| **寄存器** | **r0–r15**（PC 作操作数受限） |
| **立即数** | **8 bit 旋转** 等较宽编码 |
| **标志 S** | **`ADDS`** 才改标志；无 S 则不改 |

**代价：** Flash/RAM 占用大；在 **16-bit 宽内存** 上 **每条指令需两次 fetch**。

---

### 16-bit Thumb 的设计目标

| 目标 | 手段 |
|------|------|
| **代码密度** | 体积约为 ARM 的 **65%–70%** |
| **低成本存储器** | 16-bit 宽 ROM/RAM 上 **fetch 次数减半** |
| **性能** | 在 16-bit 内存系统上 **Thumb 常比 ARM 更快**（fetch 主导） |

---

### 16-bit Thumb 的限制

为塞进 16 bit，架构 **牺牲通用性**：

| 限制 | 例子 |
|------|------|
| **低寄存器** | 多数运算仅 **r0–r7** |
| **同源同宿** | 如 **`ADD Rd, Rd, Rm`** — Rd 兼作源与目的 |
| **小立即数** | 常数范围 **大幅缩小** |
| **无条件执行** | **无 `{cond}` 数据指令** — 仅 **`B{cond}`** |
| **标志位** | 多数 **数据处理默认更新 NZCV** — 不像 ARM 需 **`S` 后缀** |

**口述：** 16-bit Thumb = **压缩格式**；写习惯与 ARM **相反**（标志默认开 vs 默认关）。

---

### 硬件：PLA「解压」

ARM7TDMI **没有** 两套执行单元：

```
16-bit Thumb 取指
        ↓
译码阶段 PLA 扩展为「等价 32-bit ARM 微操作」
        ↓
执行级与 ARM 指令相同流水线
```

**结果：** Thumb **不额外占周期** — 密度赢、语义受限于编码。

---

### ARM vs Thumb 16 速查

| | **ARM** | **Thumb 16** |
|---|---------|--------------|
| 密度 | 低 | **~65–70%** |
| 条件数据指令 | **有** | **无** |
| 标志 | **默认不改**（要 S） | **默认改** |
| 寄存器 | r0–r15 | **多为 r0–r7** |

---

### 可复述要点

1. **Thumb 16** 为 **密度 + 16-bit 总线** 而生，不是为 M4 现代 ISA。  
2. **PLA 解压** → 执行周期与 ARM 同级。  
3. 限制太多 → 引出 **Thumb-2**（§17.3）。
