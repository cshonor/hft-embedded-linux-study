## §6.3 LDR 伪指令 — 加载常量到寄存器

> **Ch 6 · 常量与文字池** · [章导读](../README.md)

---

### 语法（捷径）

```asm
        LDR     r0, =0x40001000        ; 任意 32 bit 常数或表达式
        VLDR    S0, =3.14159           ; 浮点常数（M4F，→ Ch3 §3.7）
```

**这是伪指令** — CPU 不识别 `=`；汇编器 **展开** 为真实指令序列。

---

### 汇编器决策流程

```
LDR Rd, =expr
    ↓
expr 能否用 §6.2 旋转/MVN 单条 MOV/MVN 表示？
    ├─ 是 → 替换为 MOV/MVN（无内存访问，最快）
    └─ 否 → 在文字池放 .word expr
              替换为 PC 相对 LDR Rd, [pc, #offset]（→ §6.4）
```

| 展开结果 | 特征 |
|----------|------|
| **`MOV r0, #imm`** | 单周期、无额外 Flash |
| **`LDR r0, [pc, #n]`** | 多一次内存读；池在附近 |

**Ch5 关联：** 真 `LDR` 从 **文字池地址** 取 32 bit 到寄存器 — 与从 RAM 读变量同一指令。

---

### 常见用途

| 场景 | 示例 |
|------|------|
| **MMIO 基址** | `LDR r1, =UART0_BASE` |
| **位掩码** | `LDR r2, =0x000000FF` |
| **大常数** | `LDR r3, =0x12345678` → 几乎必走文字池 |

**Ch16 驱动：** 基址进寄存器 → `LDR/STR [r1,#off]` 访问寄存器 — **本章 + Ch5** 合体。

---

### 与 MOVW/MOVT 的分工

| 路径 | 何时 |
|------|------|
| **`LDR =`** | 通用；汇编器自动优化 |
| **`MOVW`/`MOVT`** | Thumb-2 **显式** 两条拼 32 bit，**不依赖文字池**（§6.4） |

GNU：`ldr r0, =sym` 同样行为；`movw`/`movt` 或 `mov32` 宏。

---

### 可复述要点

1. **`LDR rd,=cst`** = 伪指令；汇编器 **MOV 或 文字池+LDR**。  
2. 读 listing 区分 **单 MOV** vs **LDR [pc,#…]**。  
3. 外设基址/大掩码 — 嵌入式最高频写法之一。
