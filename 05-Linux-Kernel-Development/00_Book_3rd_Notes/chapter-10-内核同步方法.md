### ① 原子操作 · Atomic Operations

**其他同步机制的基础** — 指令级 **不可中断**：要么 **全做完**，要么 **完全没做**。

#### 原子整数 · `atomic_t` / `atomic64_t`

| 类型 | 宽度 |
|------|------|
| **`atomic_t`** | 32 位 |
| **`atomic64_t`** | 64 位 |

| 用专用类型的原因 | 说明 |
|------------------|------|
| 类型安全 | 原子 API **只接** 这些类型 |
| 防编译器乱优化 | 访问不被错误优化 |
| 屏蔽架构差异 | 统一接口，底层用架构原子指令 |

```c
atomic_inc(&v);
atomic_dec_and_test(&v);
```

#### 原子位操作

| 对比 | 说明 |
|------|------|
| 原子位 op | 对 **通用内存地址（指针）** 某位操作 |
| **`__` 前缀非原子版** | 已持锁保护时可用 — **更快** |

**HFT 对照：** 用户态 `std::atomic` / `__atomic_*` 与内核 `atomic_t` 同一层抽象 — 热路径 **计数、标志位** 优先原子而非锁。

### ② 自旋锁 · Spin Locks

内核 **最常见** 的锁。

| 属性 | 说明 |
|------|------|
| 持有者 | **至多一个** 执行线程 |
| 争用 | **忙等（自旋）** 直到锁可用 |

#### 适用场景

| 适合 | 不适合 |
|------|--------|
| **持有时间极短** | 长时间持锁（浪费 CPU 自旋） |
| **中断上下文**（不能睡眠） | 需要睡眠等锁 |

#### 不可递归

| 规则 | 后果 |
|------|------|
| **同一 spinlock 不能二次 acquire** | **自死锁**（Ch 9） |

#### 与中断配合

ISR 里用自旋锁 → 须 **先关本地中断**，否则：

```
进程持锁 ──► 被中断 ──► ISR 再抢同一把锁 ──► 死锁
```

| API | 作用 |
|-----|------|
| **`spin_lock_irqsave()`** | 关中断 + 加锁；`flags` 保存状态 |
| **`spin_unlock_irqrestore()`** | 解锁 + 恢复中断 |

→ **Ch 7–8** `local_irq_save` · **Ch 8** 常与 `local_bh_disable` 组合

### ③ 读-写自旋锁 · Reader-Writer Spin Locks

使用模式能清晰分为 **读路径 / 写路径** 时用 **rwlock**。

| 锁模式 | 规则 |
|--------|------|
| **读锁** | **多个读者** 可同时持有 |
| **写锁** | **唯一写者** — 且 **无任何读者** |

| 注意 | 说明 |
|------|------|
| Linux rwlock **偏袒读者** | 大量读者可能导致 **写者饥饿** |

| 场景 | 选型 |
|------|------|
| 读 ≫ 写、可容忍写延迟 | rwlock |
| 写不能饿死 | 考虑 **seqlock**（⑧） |

---

### ④ 信号量 · Semaphores

**睡眠锁** — 拿不到锁时 **入等待队列并睡眠**，CPU 去干别的。

| 属性 | 说明 |
|------|------|
| 开销 | 睡眠、唤醒、维护队列 — **比自旋贵** |
| 适用 | **长时间持有** |
| **禁止** | **中断上下文** — 不能睡眠 |

#### 二值 vs 计数

| 类型 | 计数 | 含义 |
|------|------|------|
| **二值 / 互斥信号量** | **1** | 互斥 |
| **计数信号量** | **>1** | 最多 N 个持有者（资源池） |

---

### ⑤ 互斥体 · Mutexes

比「计数为 1 的信号量」更现代的 **睡眠互斥** — **`struct mutex`**。

| 对比信号量 | mutex |
|------------|-------|
| 接口 | **更简单** |
| 性能 | **更高**（典型路径） |
| 规则 | **更严格** + 调试器可自动检查 |

#### 严格规则

| 规则 | 说明 |
|------|------|
| 仅 **一个** 任务持有 | |
| **谁 lock 谁 unlock** | |
| **不可递归** | |
| **不能** 在中断或下半部获取 | softirq/tasklet 也不行 |

#### 作者建议

> **新代码应始终首选 mutex**，除非限制阻碍你（如需计数 >1、需在中断用 — 那时用 spinlock/原子）。

**HFT 用户态类比：** `std::mutex` 规则类似 — 热路径仍倾向 **无锁 / 单生产者单消费者**。

---

### ⑥ 完成变量 · Completion Variables

**两任务同步** — 一方 **等事件**，另一方 **发信号「完成了」**。

| 场景 | 示例 |
|------|------|
| 等待异步工作结束 | **`vfork()`** — 子进程就绪后 completion 唤醒父进程 |

```c
init_completion(&comp);
wait_for_completion(&comp);    /* 等待方 */
complete(&comp);               /* 完成方 */
```

与 mutex 区别：**一次性事件通知**，非保护共享数据临界区。

### ⑦ 大内核锁 · BKL

| 属性 | 说明 |
|------|------|
| 本质 | **全局自旋锁** |
| 历史 | SMP 早期 **粗粒度** 过渡方案 |
| 特殊 | 曾允许睡眠、**可递归** |
| 问题 | **严重阻碍可扩展性** |

| 现状 | **强烈禁止** 在新代码中引入 BKL |

→ **Ch 9** 粗粒度锁害扩展的极端案例

### ⑧ 顺序锁 · Sequential Locks · seqlock

2.6 引入 — 用 **序列计数器** 协调读写。

| 写者 | 读者 |
|------|------|
| 获取锁，写数据，**递增序列号**（变奇数表示写入中） | **读前读序列号 s1** → 读数据 → **读后再读 s2** |
| | 若 **s1 == s2 且为偶数** → 读期间 **无写入**，数据一致 |

| 特点 | 说明 |
|------|------|
| **偏袒写入者** | 读者可重试，**写入者不会被读者饿死** |
| 适合 | **读者极多、写者极少** |

| 例子 | **`jiffies`** 更新 |

| vs rwlock | seqlock 写者优先；rwlock 读者优先 |

---

### ⑨ 禁止抢占 · Preemption Disabling

单 CPU 上 **内核抢占** 也会产生 **伪并发**（Ch 9）。

若只保护 **per-CPU 数据** — 对 **当前 CPU 独占**，不必 spinlock：

```c
preempt_disable();
/* 访问 __get_cpu_var(foo) */
preempt_enable();
```

| API | 作用 |
|-----|------|
| **`preempt_disable()`** | 禁止本任务被抢占 |
| **`preempt_enable()`** | 恢复抢占 |

| 注意 | 临界区须 **短** — 同自旋锁精神 |

→ **Ch 8** per-CPU 与 softirq

### ⑩ 排序和屏障 · Ordering and Barriers

编译器与 CPU 为性能会 **重排** load/store — 与 **设备寄存器** 或 **SMP 可见性** 冲突时需 **屏障**。

| 屏障 | 作用 |
|------|------|
| **`rmb()`** | **读内存屏障** — 屏障前读不与屏障后读乱序 |
| **`wmb()`** | **写内存屏障** — 写顺序 |
| **`mb()`** | **全屏障** — 读写皆不乱序越过 |
| **`barrier()`** | **编译器屏障** — 防编译器重排，**不约束 CPU** |

```
CPU0:  write A ── wmb() ── write B   → 设备看到 A 先于 B
CPU1:  read B  ── rmb() ── read A   → 配对另一端的发布顺序
```

| SMP | 还需 **`smp_mb()`** 等 — 跨核可见性 |

**HFT：** 用户态 **memory_order_acquire/release**、**发布-消费模式** 与内核屏障 **同一类问题** — 无锁结构错误序 = 极难复现的竞态。

→ [01-CSAPP 并发与内存](../../01-CSAPP-3rd/chapter-12-concurrent-programming/)

### 选型速查（Ch 9 + Ch 10）

| 场景 | 首选 |
|------|------|
| 单变量计数/标志 | **atomic_t** |
| 短临界区、中断/下半部（不睡） | **spinlock** + `irqsave` |
| 读多写少、写可饿 | **rwlock** |
| 读极多写极少、写不能饿 | **seqlock** |
| 仅当前 CPU 私有数据 | **`preempt_disable`** |
| 进程上下文、长临界区、互斥 | **`mutex`** |
| 计数资源池、需睡眠 | **semaphore** |
| 等另一任务完成事件 | **completion** |
| 跨核/设备可见顺序 | **mb/rmb/wmb** |

### Ch 10 小结

| 机制 | 睡眠？ | 中断可用？ | 要点 |
|------|--------|------------|------|
| atomic | — | 是 | 基础 |
| spinlock | 否（自旋） | 是（+irqsave） | **不可递归** |
| rwlock | 否 | 是 | **偏袒读者** |
| semaphore | 是 | **否** | 可计数 |
| mutex | 是 | **否** | **新代码首选** |
| completion | 等事件 | 视 API | vfork 等 |
| seqlock | — | 读者无锁快路径 | **偏袒写者** |
| preempt_disable | — | 谨慎 | **per-CPU** |
| barriers | — | 是 | 防重排 |
