### ① Unix 的历史 · History of Unix

| 事实 | 说明 |
|------|------|
| **起源** | **1969** · 贝尔实验室 · **Dennis Ritchie**、**Ken Thompson** |
| **成功因素** | 见下表 |

| Unix 优势 | 含义 |
|-----------|------|
| **设计简洁** | 仅 **~几百个系统调用** — 接口少而稳 |
| **一切皆文件** | 设备、socket、管道… 统一 **open/read/write** |
| **C 语言实现** | **可移植** — 换硬件主要重编译内核 |
| **极快进程创建** | 独特 **`fork()`** — 复制地址空间语义 |
| **稳健 IPC** | 管道、信号等 **简单原语** |

**HFT 对照：** 网关仍活在 **「少 syscall、少拷贝、快 fork/线程」** 的 Unix 遗产里 — 热路径 **`read`/`send`/`mmap`** 皆是「一切皆文件」后代。

### ② Linux 的诞生 · Along Came Linus

| 项 | 内容 |
|----|------|
| **时间** | **1991** · 赫尔辛基大学 · **Linus Torvalds** |
| **动机** | 当时 **缺乏强大且免费的 Unix** |
| **性质** | **非商业** · **互联网协作** 开发 |
| **许可证** | **GNU GPL 2.0** — **自由 / 开源软件** |

### ③ 操作系统与内核概述

#### 内核（Kernel）

**操作系统最内层** — 提供基本服务、**管理硬件**、**分配资源**。

#### 两种地址空间

```
┌─────────────────────────────────────┐
│  user-space   应用程序（策略、网关）   │
│       │ syscall ↑                     │
├───────┼─────────────────────────────┤
│  kernel-space  内核（机制）           │
│       │ ↑ 中断                        │
└───────┼─────────────────────────────┘
        硬件
```

| 空间 | 谁跑 | 权限 |
|------|------|------|
| **kernel-space** | 内核代码 | **完全硬件访问** · 受保护内存 |
| **user-space** | 普通 app | **受限** — 经 **系统调用** 求内核办事 |

| 方向 | 机制 |
|------|------|
| **app → 内核** | **System Calls**（`read`、`write`、`clone`…） |
| **硬件 → 内核** | **Interrupts** → **中断处理程序** |

### ④ Linux 与经典 Unix 内核对比

#### 两派设计

| 类型 | 特点 |
|------|------|
| **宏内核（Monolithic）** | **单地址空间** 内核态 · **性能好** |
| **微内核（Microkernel）** | 服务拆进程 · 消息传递 · 更模块化、理论更「干净」 |

**Linux = 宏内核** — 但 **务实借鉴微内核优点：**

| Linux 特性 | 说明 |
|--------------|------|
| **可加载内核模块** | 动态扩展驱动/功能 |
| **内核线程** | 内核态执行后台工作 |
| **内核抢占（preemption）** | 低延迟调度 · 减少「内核跑太久」 |
| **SMP** | 对称多处理器 |
| **线程 ≈ 进程** | `clone()` — 统一任务模型 |

**HFT：** **绑核、PREEMPT_RT、模块调优网卡驱动** — 都站在「宏内核 + 可抢占 + 模块」这条线上。

### ⑤ Linux 内核版本

| 类型 | 说明 |
|------|------|
| **稳定版（Stable）** | 生产可用 |
| **开发版（Development）** | 新特性实验 |

**传统版本号**（如 **2.6.30.1**）：

| 段 | 含义（经典规则） |
|----|------------------|
| 次版本号 **第二位** | **偶数 → 稳定** · **奇数 → 开发** |

> 注：现行主线已长期 **3.x / 4.x / 5.x / 6.x**，「奇偶」规则 **概念仍常考**，但以 **kernel.org 分支政策** 为准。
