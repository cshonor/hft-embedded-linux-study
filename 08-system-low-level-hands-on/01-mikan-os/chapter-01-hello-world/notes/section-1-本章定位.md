### 四、核心区分：BOOTX64.EFI ≠ 软盘启动（两条线）

| | **软盘启动（BIOS 时代）** | **BOOTX64.EFI（UEFI 时代）** |
|---|---------------------------|------------------------------|
| **时代 / 固件** | **Legacy BIOS** | **UEFI 固件**（BIOS 的现代继任） |
| **介质** | 1.44MB **软盘映像**（`-fda`） | **FAT** U 盘 / 虚拟盘（**不是**「把 EFI 塞进引导扇区」） |
| **入口是什么** | 软盘 **第 0 扇区 512 B** 的 **IPL**（Initial Program Load） | 符合 UEFI 规范的 **PE32+ 可执行文件** `.efi` |
| **CPU 从哪起跑** | **16 位实模式** @ `0x7C00` | **x86-64 长模式**（固件已建好环境） |
| **寻址能力** | 早期仅 **≤1MB** 常规内存，后续需自己切保护/长模式 | 一上来就是 **64 位平坦地址空间** 语境 |
| **典型职责** | 512B 里再读后续扇区 → 逐步拉起 OS | **直接加载并跳转 64 位内核/Loader**，跳过实模式整套老流程 |
| **体积与维护** | 极小、手写汇编可行 | 含 **PE 头、节区** — 用 **C + 工具链** 生成（见 [§2](./section-2-二进制编辑器与BOOTX64.md) · [code/](../code/)） |
| **兼容 vs 效率** | **兼容性强**（老机器都能软盘起）但 **效率极低**、与现代 PC 启动链脱节 | 与 **实体机 / 服务器 / QEMU OVMF** 同款路径，**更贴合 HFT 底层主线** |

**MikanOS 为何选后者：** 绕开「实模式 → 保护模式 → 长模式」那套 **BIOS 软盘启蒙流程**，第一天就在 **现代 64 位 UEFI 环境** 里写可维护的 C/C++ — 与 [02 30 天 OS](../../02-30days-os/) **互补**（理解历史 IPL），但 **不必前置** 才能学 MikanOS。

→ **UEFI 七步 · BIOS 四区别 · 名词表** 完整版 [§5 UEFI 启动流程](./section-5-UEFI启动流程.md)

**HFT 读法：** 生产与回测机上的程序，最终都跑在 **UEFI → OS 加载器 → 长模式内核** 这条链上；搞清 **BOOTX64.EFI 是链上第一个「你的 .efi」**、而 **不是软盘 IPL**，后面读 [04 LKD](../../../04-Linux-Kernel-Development/) · [07 TLPI](../../../07-The-Linux-Programming-Interface/) 启动与内存布局时不会张冠李戴。
