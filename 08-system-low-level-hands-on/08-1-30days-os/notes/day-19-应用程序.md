### ② 改良 type · 换行与制表符

初版 **逐字节原样显示** → 含 **控制符** 的文件 **排版乱**。

| 字节 | 处理 |
|------|------|
| **`0x0A`** | 换行（LF） |
| **`0x0D`** | 回车（CR）— 常与 LF 配合 |
| **`0x09`** | **Tab** — 光标跳到下一制表位 |

**效果：** **`.nas` / 源文件`** 在 Console 里 **可读** — Shell 不只是 hex dump。

### ③ 完全支持 FAT · 大文件与碎片

#### 小文件方案的局限

| 限制 | 原因 |
|------|------|
| **≤512B** | 只读 **一个簇/扇区** |
| **必须连续** | 未查 **下一簇** |

#### FAT（文件分配表）

软盘 **FAT12** 为省空间：**2 个 12 位簇号压进 3 字节** → 需 **解压展开** 成可用表。

**读文件链：**

```
clust = 起始簇
loop:
    读 clust 对应扇区数据
    clust = FAT[clust]   /* 下一簇 */
until clust == 0xFFF (EOC，文件结束标记)
```

| 能力 | 说明 |
|------|------|
| **任意大小** | 多簇接力 |
| **非连续（碎片）** | FAT **链式** — 物理上不邻接也能读 |

**`file_loadfile`（示意名）** — 完整读入 **MEMMAN 分配** 的缓冲区。

→ 对照 **ext4 inode 间接块** — 思想同为 **「表驱动块链」**

### ⑤ 第一个应用程序 · `hlt.hrb`

#### 极简 app

**`hlt.hrb`** — 仅 **3 字节** 级：**HLT + 死循环**（原书示意）。

#### OS 如何运行外部程序

| 步骤 | 说明 |
|------|------|
| **`file_loadfile`** | 从磁盘 **读 .hrb 进内存** |
| **新 GDT 段** | 为 app 分配 **独立段描述符**（如 **1003 号**）— **与内核段隔离** |
| **`farjmp`** | 跳到 **app 段:offset** 执行 |

```
Console: run hlt.hrb
    → file_loadfile → 内存 buffer
    → 注册 GDT 段（用户/app 代码段）
    → farjmp → app 在 Ring 保护下跑（以原书设定为准）
    → HLT…
```

**转折点：** 从 **内置命令** → **加载外部二进制** — **`.hrb` = 纸娃娃的 exe**；Day 20+ **API/syscall** 在此基础上扩展。
