### ① 用 C 写入显存与位运算 · 条纹图案

Day 3 已切 **320×200×8** 图形模式；要 **显示内容** 必须写 **显存 VRAM**。

| 属性 | 值 |
|------|-----|
| **VRAM 起始地址** | **`0xa0000`**（VGA 模式 13h 等常用映射） |
| **像素** | 每字节 **1 个调色板色号**（0–255） |

先用 **`for` 循环** 往 VRAM 写数据 → **整屏纯白**。

再引入 **位运算** 处理像素/掩码：

| 运算 | 典型用途 |
|------|----------|
| **AND `&`** | 清位、取掩码 |
| **OR `\|`** | 置位、合并 |
| **XOR `^`** | 翻转、交替图案 |

→ 在屏上画出 **黑白相间条纹** — 证明 **C 直接驱动 framebuffer**。

**HFT：** 无 GUI，但 **mmap 行情缓冲、位图标志、cache line 对齐** 同样是指针 + 位运算；**少写 VRAM 式乱扫内存**。

### ② 挑战并理解指针

C 写内存 = 必须告诉编译器 **地址** 和 **操作宽度**。

| 汇编 | C |
|------|---|
| `MOV BYTE [xxx], al` | **`char *p`** → **`*p = …`**（1 字节） |
| 明确 BYTE/WORD | **指针类型 + cast** |

```c
char *p = (char *) 0xa0000;   /* VRAM 起点 */
*p = 15;                      /* 写 1 字节色号 */
```

- **`*p`** ≈ 汇编 **内存寻址 `[...]`**
- **指针** = 「存地址的变量 + 元素大小由类型决定」

→ 对照 [01-CSAPP Ch3 指针与数组](../../01-CSAPP-3rd/chapter-03-machine-level-programs/)

### ③ 调色板（Palette）与色号

**8 位模式** 不能直接用 RGB 三元组 — 像素值只是 **0~255 的索引**。

| 概念 | 说明 |
|------|------|
| **调色板** | 把 **256 个索引** 映射到 **实际 RGB（或近似）** |
| **本日目标** | 设定 OS 所需的 **16 种基础色** |

硬件通过 **I/O 端口** 配置（需汇编 **`OUT`**）：

| 端口 | 作用（原书） |
|------|----------------|
| **`0x03c8`** | 调色板地址寄存器 |
| **`0x03c9`** | 调色板数据寄存器 |

流程：**C 调汇编包装** → `OUT` 写端口 → 16 色就绪。

```
像素字节 0~255  ──索引──►  调色板表  ──►  显示器上的颜色
```

---

### ④ EFLAGS 寄存器与中断控制

设调色板要 **连续写多个端口** — 若中途被 **中断打断**，颜色可能错乱。

| 指令 | 作用 |
|------|------|
| **`CLI`** | **Clear Interrupt flag** — **屏蔽可屏蔽中断** |
| **`STI`** | **Set Interrupt flag** — **恢复中断** |

 **`EFLAGS`**（32 位）含 **IF（中断标志）** 等位 — 控制 CPU 是否响应部分中断。

| 栈操作 | 作用 |
|--------|------|
| **`PUSHFD`** | 把 **EFLAGS** 压栈保存 |
| **`POPFD`** | 从栈恢复 **EFLAGS** |

**模式：** `PUSHFD` → `CLI` → 设调色板 → `POPFD`（或 `STI`）— **原子性更好的临界区**。

→ 后续 Day **IDT / 自己的中断处理**；Linux 里对应 **local_irq_save/restore** 一类思路。

### ⑤ 绘制矩形与 OS 雏形

颜色与 **VRAM 地址 ↔ 屏幕坐标 (x, y)** 关系清楚后：

| 函数级能力 | 效果 |
|------------|------|
| **画点** | 单像素色号 |
| **画矩形** | 填充区域 |

在 **屏幕底部** 画一块矩形 → **类似 Windows 任务栏** 的条带 — **纸娃娃 OS 第一次有「界面范儿」**。

```
┌──────────────────────────── 320 ────────────────────────────┐
│                     主显示区（条纹/背景）                      │ 200
│                                                             │
│▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ 任务条矩形 ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│
└─────────────────────────────────────────────────────────────┘
         VRAM: 0xa0000 + y * 320 + x
```

作者强调：此时 **内核约 1.2KB 级** 体量，已能画出 **像样 GUI 底座** — **指针 + 显存模型** 比堆功能名更重要。

### Day 4 小结

| 问题 | 答案 |
|------|------|
| 图形模式像素写哪？ | **VRAM `0xa0000` 起**，每字节 1 色号 |
| 条纹怎么来的？ | **`for` + AND/OR/XOR** 处理像素数据 |
| 指针为何必需？ | 指定 **地址 + 字节宽度**，对应汇编 **BYTE 写内存** |
| 8 位色怎么用 RGB？ | **调色板** 映射；**`0x03c8`/`0x03c9` + `OUT`** |
| 设色为何关中断？ | **`CLI`/`STI`**，**`PUSHFD`/`POPFD`** 保 **EFLAGS** |
| 界面里程碑？ | **画矩形** → 底部 **任务条** 雏形 |

**本章定位：** **GUI 起点** — 为后续窗口、鼠标、文字渲染打 ** framebuffer + 调色板 + 绘图原语** 地基。
