### ① 智能控制光标闪烁

Day 17：**Tab 切焦点** — 非活动窗 **不应继续闪光标**。

**问题：** 主任务 A 与 **`console_task`** **不能直接改对方变量**（不同任务、不同栈）。

**解法：** 复用 **FIFO 任务间通信**（Day 13/17）：

| FIFO 编码（原书） | 含义 |
|-------------------|------|
| **2** | **开始** 光标闪烁 |
| **3** | **停止** 光标闪烁 |

```
Tab 切到 Console → Put(3) 给 A，Put(2) 给 Console
Tab 切回主窗     → 反之
```

各任务 **只处理自己 fifo 里的 2/3** — **解耦** 又 **可靠**。

→ [Day 17 Tab/key_to](./day-17-命令行窗口.md) · [Day 12 定时器驱动光标](./day-12-定时器1.md)

### ② 回车键 · 窗口滚动

#### 回车

| 键 | 编码 | 行为 |
|----|------|------|
| **Enter** | ASCII **10** | 经 fifo 到 console → **`cursor_y++`** |

#### 滚屏 · `cons_newline`

到底仍 Enter → 文字 **出窗** → 需要 **Scroll**。

**实现（原书「粗暴有效」）：**

1. 将图层 **文字区 buffer** 内像素 **整体上移 16 像素**（一行字高）
2. **最底一行** 用 **黑色矩形** 涂满

```
换行前 [行0][行1]…[行N-1] 满
换行后 [行1][行2]…[行N-1][空黑行]
         ↑ memmove 式上移 16px
```

**像终端 scroll** — 无复杂文本布局引擎。

→ [Day 10 sheet 缓冲](./day-10-叠加处理.md)

### ③ mem 与 cls · 引入 strcmp

#### 为何 `strcmp`

命令变多 → **逐字符 if 链** 易错 → **`#include <string.h>`**：

```c
if (strcmp(cmd, "mem") == 0) { … }
else if (strcmp(cmd, "cls") == 0) { … }
else if (strcmp(cmd, "dir") == 0) { … }
```

#### `mem` 命令

| 之前 | 之后 |
|------|------|
| 桌面 **硬编码** 显示内存 | 输入 **`mem`** **按需查询** |

创建 console 时传入 **`memtotal` / `memman` 指针**（Day 15 栈传参同类技巧）→ 打印 **总内存 / 可用内存**。

→ [Day 9 MEMMAN](./day-09-内存管理.md)

#### `cls` 命令

**clear screen：** **黑色矩形** 涂满 Console 文字区 + **光标 y 回顶**。

### ④ dir 命令 · FAT12 根目录

#### 目标

列出 **软盘映像里有哪些文件** — 为 Day 19 **加载 `.hrb` 应用** 铺路。

#### FAT12 根目录（原书布局示意）

| 项 | 值 |
|----|-----|
| **内存地址** | 加载后约 **`0x002600`** 起 |
| **条目数** | 最多 **224** 个 |
| **每项大小** | **32 字节** |

**32 字节目录项（核心字段）：**

| 偏移 | 长度 | 内容 |
|------|------|------|
| 0 | 8 | **文件名**（大写，不足 **空格** 补） |
| 8 | 3 | **扩展名** |
| … | … | **属性**（如 **`0x20`** 普通文件，**`0x01`** 只读） |
| … | 4 | **文件大小**（小端） |

```c
/* 示意 */
for (i = 0; i < 224; i++) {
    p = dir_base + i * 32;
    if (p[0] == 0 || p[0] == 0xE5) continue;  /* 空/已删 */
    /* 解析 name.ext attr size → printf */
}
```

**`dir` 输出：** 文件名 · 扩展名 · 大小 — **交互式外壳第一次读「磁盘结构」**。

→ 对照 [07-TLPI 文件 I/O](../../07-The-Linux-Programming-Interface/) · Linux **VFS/ext4** 是 FAT12 的远房后代
