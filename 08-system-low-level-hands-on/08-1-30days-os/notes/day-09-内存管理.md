### ① 内存容量检查 · 关闭 CPU 缓存

进 32 位后可用 **>1MB** 物理内存 — 先要 **测出到底有多少**。

#### 测试方法（探针）

对候选地址：

1. 写入测试模式（如 **`0xAA55AA55`**、**`0x55AA55AA`**）
2. **按位取反** 再写/读
3. **读回比对** — 一致 → 该地址 **有真实 RAM**；不一致 → 无内存或别名

```
写 pattern → 反转 → 读回 == 期望？
    是 → 这一带算有效物理内存
    否 → 停止向上扩（或跳过洞）
```

#### 必须关 Cache

| 问题 | 原因 |
|------|------|
| 有 **Cache** 时读写可能在 **缓存** 完成 | **没真正打到 DRAM** |
| 测出来 **假阳性** — 以为有内存其实没有 | |

**做法：** 测前改 **`CR0`** 相关位 → **暂时禁用 CPU Cache**，测完再恢复。

→ 对照 [01-CSAPP Ch6 存储层次](../../01-CSAPP-3rd/chapter-06-memory-hierarchy/) · [04-Hennessy Cache](../../04-Computer-Architecture-6th/)

**HFT：** **mmap 大页、NUMA 本地内存、禁止 false sharing** — 都要知道 **CPU 看到的地址 ≠ 每次访问都落 DRAM**。

### ② C 编译器的「过度优化」陷阱

#### 现象

模拟器 **实际 32MB**，用 **C 写的 memtest** 却报 **~3GB**。

#### 根因

打开 **编译器生成的汇编** 发现：**测试循环被整个删掉**。

编译器 **数据流分析** 认为：

> 写入 → 反转两次 → 值没变 → **死代码 / 无副作用** → **优化掉**

对 **普通应用** 这是好事；对 **「探测硬件是否存在」** 这种 **副作用在读写本身** 的代码 — **灾难**。

#### 解法

**用汇编重写 `memtest_sub`** — 编译器 **不会** 把你的 **`MOV [addr], reg`** 当废话删掉 → **容量正确**。

| 教训 | 说明 |
|------|------|
| **`-O2` 双刃剑** | 假设有 **抽象机器语义**；**裸机探针** 常需 **`volatile`、asm、或纯汇编** |
| **看汇编清单** | OS/驱动 **sign-off 前** 对关键路径 **disassemble 一眼** |

→ HFT 里 **`volatile`、memory barrier、禁止 reorder** — 同一类 **「编译器以为安全，硬件语义不对」**。

### ③ 构思内存管理算法

已知 **总可用内存** → 需要 **分配 / 释放**。

#### 位图 / 分配表法

| 做法 | 将内存切成 **固定小块**，每块 **0/1** 标记占用 |
|------|--------------------------------------------------|
| **问题** | 若物理内存 **3GB**，粒度 4KB → 状态表 **~768KB** **仅元数据** — **太浪费** |

```
3GB / 4KB = 786432 块 → 每块 1bit 也要 ~96KB+；更粗粒度则碎片多
位图随 RAM 线性涨 — 小 OS 不想要
```

### ④ MEMMAN · 基于空闲块列表

**只记录「空闲」**，不记录每一块占用。

#### 核心思想

每条记录：**起始地址 + 连续空闲字节数**

```
空闲列表（示意）
[0x00300000, 65536)  [0x00400000, 131072)  …
```

| 操作 | 行为 |
|------|------|
| **alloc(size)** | 遍历列表，找 **≥ size** 的块 → 切分/标记已用 → **更新列表** |
| **free(addr, size)** | 归还 → **插入列表**；若与 **相邻空闲块** 接壤 → **合并成更大块** |

**合并** 防 **外部碎片** 恶化 — 经典 **coalesce**。

#### `MEMMAN` 结构体

封装 **空闲表 + 块数上限** 等字段，对外 **`memman_alloc` / `memman_free`**（名以原书为准）。

```
程序 malloc 式请求
      │
      ▼
 MEMMAN.alloc → 从某 free 块切出
      │
程序 free
      │
      ▼
 MEMMAN.free → 还回 + 与邻居合并
```

→ 对照 [01-CSAPP Ch9 malloc 与碎片](../../01-CSAPP-3rd/chapter-09-virtual-memory/) · Linux **buddy / slab** 是工业级进化 · HFT **对象池 / arena** 常 **预分配、少 free**

### Day 9 小结

| 问题 | 答案 |
|------|------|
| 怎么测内存？ | **写-反转-读** 探针；**关 Cache（CR0）** |
| C 版为何错？ | **优化删掉测试循环** → 假 3GB |
| 正确实现？ | **汇编 `memtest_sub`** |
| 为何不用位图？ | 大 RAM → **管理表本身太大** |
| MEMMAN 思路？ | **只链表记录空闲块** |
| free 还要啥？ | **相邻块合并** 减碎片 |

**后勤保障：** 多任务、窗口图层、更大数据结构 — 都依赖 Day 9 **可信容量 + alloc/free**。
