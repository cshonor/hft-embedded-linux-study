### ① 结构体与指针 · 优化启动信息

Day 4 里 **320×200、VRAM 地址** 等多为 **硬编码** — 换模式就要改多处。

引入 **`struct`** 与 **`->`**（指针访问成员）：

```c
/* 示意：启动时由 IPL/引导代码写入固定内存，内核读取 */
struct BOOTINFO {
    /* 画面模式、显存地址、宽高等 — 以原书为准 */
};
struct BOOTINFO *binfo = (struct BOOTINFO *) 0x0000xxxx;
binfo->screen_x = ...;   /* binfo-> 即 (*binfo). */
```

| 做法 | 好处 |
|------|------|
| **`BOOTINFO` 固定内存布局** | IPL 与 **bootpack** **约定一块区域** 传参 |
| **结构体字段** | 换分辨率只改 **写入端**，C 端 **统一读 struct** |

**HFT 对照：** 像 **共享内存里的行情头结构** — 布局固定、版本字段、生产者写 / 消费者读。

### ② 自力更生显示文字 · 告别 BIOS

32 位保护模式下 **不能再 `INT` 调 16 位 BIOS** 打字符 → **自己画字**。

#### 8×16 点阵

| 要素 | 说明 |
|------|------|
| **字形** | 每个字符 **8 像素宽 × 16 像素高**  bitmap |
| **绘制** | 对每个为 1 的位 → 往 VRAM **写对应色号** |
| **字体数据** | 外部导入 **ASCII 等** 点阵表 |

#### 字符串与格式化输出

- 编写 **整串输出** 函数（逐字符调用点阵绘制）
- 引入 C **`sprintf`** → 支持 **`%d` `%x`** 等

```c
char s[40];
sprintf(s, "mouse=%d", value);
putstr(s, x, y, color);
```

**在无调试器 OS 里，屏幕 print 变量 = 第一调试器。**

→ 对照 [01-CSAPP](../../01-CSAPP-3rd/) · 格式化 I/O 底层仍是 **写内存**；HFT 里类似 **日志 ring + 固定格式**。

### ③ 绘制鼠标指针

与字符相同思路：

| 属性 | 值 |

| **形状** | 经典 **箭头** |

### ④ GDT 与 IDT 初始化 · OS 基石

#### 为何要分段 · **GDT**

| 问题 | 机制 |
|------|------|
| 多程序 / 内核 vs 用户 | 不能把 **4GB** 当一块裸奔 |
| **分段（Segmentation）** | 内存划成 **段**，每段有 **基址、界限、权限** |
| **GDT** | **Global Descriptor Table** — **全局** 存放 **段描述符** 的表 |

CPU 通过 **段选择子 + GDT** 查 **谁能访问哪段内存** — **防重叠、防越权** 的第一道门。

```
逻辑地址 ──► 段号 + 偏移 ──► 查 GDT 描述符 ──► 线性/物理访问（保护检查）
```

→ 后续还有 **分页（页表）**；x86 **分段 + 分页** 并存，GDT 仍必须初始化。

#### 为何要中断 · **IDT**

| 问题 | 机制 |
|------|------|
| CPU 不能 **死循环轮询** 键盘/鼠标 | **硬件有事 → 发中断信号** |
| 中断号 **0~255** | 每种事件一个 **向量号** |
| **IDT** | **Interrupt Descriptor Table** — 记录 **第 n 号中断 → 跳哪个处理函数** |

```
键/鼠动作 ──► 硬件 ──► CPU 查 IDT[n] ──► 跳 ISR ──► 处理完返回
```

**GDT** = **内存怎么分、谁准进**  
**IDT** = **外部事件从哪段代码进内核**

→ 对照 [05-LKD](../../05-Linux-Kernel-Development/) · GDT/IDT 在 Linux 启动早期同样要设；[01-CSAPP Ch8 异常/中断](../../01-CSAPP-3rd/chapter-08-exceptional-control-flow/)

**HFT：** 网卡 **MSI-X / 中断 coalescing**、**绑 IRQ 到核** — 都是 **IDT → 处理函数** 链路的工业版。

### Day 5 小结

| 问题 | 答案 |
|------|------|
| 启动信息怎么灵活？ | **`BOOTINFO` struct** + 固定内存传参 |
| 文字谁画？ | **8×16 点阵** 写 VRAM，**不用 BIOS** |
| 怎么 debug？ | **`sprintf` + 屏幕输出** `%d` / `%x` |
| 鼠标图标？ | **16×16** bitmap + **透明** |
| GDT 干什么？ | **段描述符表** — 内存 **界限与权限** |
| IDT 干什么？ | **0~255 中断 → 处理函数** — **硬件并发** |
