### ① 攻克 __alloca · 大数组栈分配

#### 现象

**`sosu.c`（质数）**、**`winhelo`** 等 — C 里 **>4KB 局部数组** → 编译器生成 **`__alloca` 调用** → 链接 **undefined**。

#### 实现

**汇编实现 `__alloca`** — 运行时 **动态扩展栈** 分配大块。

| 收益 | 说明 |
|------|------|
| **可链接** | 大数组不再必须 **静态进 .hrb** |
| **体积** | 无 **.bss 填 0** — 原书 **`winhelo.hrb` 7664→174B** |

与 Day 23 **API malloc** 互补：**栈上大临时缓冲** vs **堆上长期缓冲**。

→ [Day 23 EDX 9/10](./day-23-图形处理相关.md) · [Day 27 按符号链接](./day-27-LDT与库.md)

### ② 文件操作 API · EDX 21–25

此前 **仅内核** `file_loadfile` / `dir` — app **不能自主读盘**。

| EDX | 功能 |
|-----|------|
| **21** | **open** |
| **22** | **close** |
| **23** | **seek** |
| **24** | **get size** |
| **25** | **read** |

**封装 FAT12**（Day 18–19）— app 经 **INT 0x40** 访问，**不直接摸磁盘**。

**`typeipl.hrb`：** 外部 app **独立读文件并显示** — 文件 I/O **用户态化**。

### ③ 命令行 API · 外部 type（EDX 26）

内核 **`type` 命令** 硬编码 — 要 **完全迁出** 需知 **用户输入整行**。

| EDX | 功能 |
|-----|------|
| **26** | **获取命令行字符串** |

**`type.hrb` 流程：**

```
api_getcmdline(buf)
跳过 "type" 与空格 → 提取文件名 → open/read → 显示
```

**Shell 只负责启动 + 传参** — **逻辑全在 .hrb** — 现代 **`/bin/cat` 是用户程序** 的雏形。

→ [Day 20 cmd_app](./day-20-API.md) · [Day 26 start/ncst](./day-26-为窗口移动提速.md)

### ④ 日文全角 · nihongo.fnt

#### 外置字库

汉字字库 **>100KB** — **不宜塞进内核** → 启动加载 **`nihongo.fnt`**。

#### langmode · TASK

| 值 | 模式 |
|----|------|
| **0** | 英文 |
| **1** | 日文 **Shift-JIS** |
| **2** | 日文 **EUC** |

**`TASK.langmode`** — Console / API 按模式 **解析字节流**。

#### 双字节 → 字模地址

全角 **2 字节** → 按编码 **区号/点号** 公式 → **`nihongo.fnt` 内偏移**。

#### 绘制 · 16×16

| 对比 | 半角 Day 5 | 全角 |
|------|------------|------|
| 尺寸 | **8×16** | **16×16** |
| 绘制 | 一次 blit | 读 **2 字节** 后 **左移 8px** — **左右半拼接** |

#### EDX 27

**获取当前 langmode** — app **自适应** 语言环境。

**国际化基础** — 中文/日文东亚 **双字节** 与 **外置 font** 模式可复用。

→ [Day 5 8×16 点阵](./day-05-结构体文字显示与GDT-IDT.md)
