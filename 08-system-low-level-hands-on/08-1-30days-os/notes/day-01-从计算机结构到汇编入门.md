### ① 先动手操作 · 二进制编辑器写系统

**跳过 C / 汇编**，直接用 **二进制编辑器**（原书示例：**Bz**）输入 **十六进制机器码**，复制粘贴拼出：

| 属性 | 值 |
|------|-----|
| 输出文件 | `helloos.img` |
| 大小 | **1,474,560 字节** = **1440 KB** = **一张标准 1.44MB 软盘** |

把该映像当作 **启动盘** 运行（QEMU / 实机软驱）→ 屏幕出现 **`hello, world`** — 这就是 **第一个极简「操作系统」**（严格说是 **能自举并输出字符的启动程序**）。

**HFT 直觉：** 再复杂的网关/引擎，冷启动时 CPU 读的也是 **磁盘/Flash 上的原始字节** — 没有魔法。

---

### ② 究竟做了些什么 · CPU 只认开关

CPU **不理解**「数字」「图像」「文字」— 只识别 **开（ON）/ 关（OFF）** 电信号，即 **0 和 1** 组成的二进制序列。

我们在编辑器里输入的十六进制，就是在 **直接向 CPU 下发机器语言指令**：

```
你看到的 hello, world
    ↑ 显存/BIOS 字符输出
    ↑ 由若干条机器指令逐字节执行出来
    ↑ 最底层全是 0/1
```

→ 对照 [01-CSAPP Ch3 机器级表示](../../01-CSAPP-3rd/chapter-03-machine-level-programs/)

---

### ③ 初次体验汇编程序

手敲 **数十万** 个 0/1 或 hex：**慢、易错、不可维护** → 引入 **汇编语言**。

```
helloos.nas  ──nask.exe──►  helloos.img   （与手工版一致）
   源码              汇编器           软盘映像
```

| 步骤 | 工具 | 产出 |
|------|------|------|
| 写源码 | 文本编辑器 | `helloos.nas` |
| 汇编 | **`nask.exe`**（原书配套） | 机器码写入映像 |
| 运行 | 模拟器 / 软驱 | 同样 `hello, world` |

**要点：** 汇编是 **机器码的可读别名** — 一条 `MOV` 对应固定字节序列；编译器负责 **编码**，人负责 **逻辑**。

---

### ④ 加工润色 · `$` 与启动区签名

**自动填充：** 汇编里用 **`$`** 让编译器 **自动计算并填充 `0x00`**，把 **第 510–511 字节** 留给启动区魔数 **`55 AA`**（x86 **引导扇区有效标志**）。

```
软盘 / 映像布局（引导扇区，共 512 字节）
┌──────────────────────────────────────┬──────┐
│  IPL 代码 + 数据（nask 用 $ 填 0）    │55 AA │  ← 字节 510–511
└──────────────────────────────────────┴──────┘
  字节 0                              511
```

改输出文字时，只动源码里的字符串，**不必手算** 后面要填多少 `0x00` — 这是 **工具链第一次显身手**。

---

### 关键术语

#### IPL（Initial Program Loader · 启动程序加载器）

- 磁盘 **最开头** 的 **启动区只有 512 字节**，装不下完整 OS。
- 这段小程序的任务：**把 OS 本体再加载进内存** → 叫 **IPL**。
- Day 1 的 `helloos` 已是「能跑的最小 IPL」；后面 Day 会扩展成 **真正的加载器**。

#### Boot（启动 · bootstrap）

- 来自 **bootstrap（靴带）** — 「拽着靴带把自己提起来」的 **逻辑悖论**：
  - 要运行 OS，得先有程序 **从磁盘读 OS 进内存**；
  - 但 **读 OS 的程序本身也在磁盘里** — 谁先来？
- 硬件约定：**上电 → BIOS/UEFI 读引导扇区 → 执行 IPL** — 这条 **自力更生的链** 叫 **bootstrap**，简称 **boot**。

→ 与 Linux 启动链对照：[05-LKD](../../05-Linux-Kernel-Development/) · GRUB / `vmlinuz` 是 **多阶段 IPL** 的工业版。

### Day 1 小结

| 问题 | 答案 |
|------|------|
| 第一天做出了什么？ | **1.44MB 软盘映像**，启动后 **hello, world** |
| 为什么先玩二进制编辑器？ | 证明 **OS 底层就是字节**；消除神秘感 |
| 为什么马上上汇编？ | 机器码 **不可规模维护**；`nask` 生成 **相同 img** |
| 512 字节里最重要的是？ | 代码 + 末尾 **`55 AA`**；更大 OS 靠 **IPL 二段加载** |
| 两个新词？ | **IPL** = 启动加载器；**Boot** = bootstrap 自举 |

**本章目的：** 用 **可见成果** 激发兴趣，从 **机器语言** 跨入 **汇编** — 为后续保护模式、GDT、中断打地基。
