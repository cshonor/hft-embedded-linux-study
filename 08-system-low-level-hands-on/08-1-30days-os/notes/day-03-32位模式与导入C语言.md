### ① 制作真正的 IPL 与读取磁盘

Day 1–2 的 IPL **只显示文字**，**没有装载 OS 本体**。从 Day 3 起，IPL **名副其实**。

#### 软盘物理结构（原书）

| 概念 | 含义 |
|------|------|
| **柱面（Cylinder）** | 盘片上一圈同心磁道 |
| **磁头（Head）** | 上下盘面各一读头 |
| **扇区（Sector）** | 磁道上最小读写单位（常 **512 B**） |

读盘靠 **BIOS `INT 0x13`** — 在 **16 位实模式** 下调用固件例程。

#### 可靠性 · 重试 5 次

软盘 **易读错** → 代码里 **失败重试最多 5 次**，再报错/放弃。

#### 循环读入

用 **循环** 按柱面/扇区读，最终把软盘上 **10 个柱面（约 180KB）** 的数据 **完整读入内存** — 这就是 **bootpack（OS 本体）** 的装载过程。

```
ipl.bin (512B @ 0x7c00)
    │
    │  INT 0x13 循环读盘
    ▼
内存中的 bootpack（~180KB）
```

**HFT 对照：** 像 **从磁盘/NVMe  mmap 策略库** — 启动链上 **「谁读、读多少、读失败怎么办」** 必须显式写清；Linux 里由 bootloader + 内核完成。

### ② 纸娃娃操作系统 · 画面模式切换

系统正式命名：**纸娃娃操作系统（haribote-os / はりぼて）** — 目前 **只有外壳**，但按步骤能 **长成完整 OS**。

#### 如何确认 OS 本体已执行？

用 **显卡 BIOS `INT 0x10`** 切换显示模式：

| 模式 | 参数（原书） | 现象 |
|------|--------------|------|
| **VGA 图形** | **320×200**，**8 位彩色** | 启动后 **整屏全黑** = **OS 已跑起来** |

（文字模式是 Day 1 的 BIOS 输出；图形模式切换证明 **控制权已到 bootpack**。）

### ③ 32 位模式前期准备与导入 C 语言

**本章核心转折。**

#### 为什么要进 32 位？

| 16 位实模式 | 32 位保护模式 |
|-------------|----------------|
| 可用内存 **≈ 1MB 级** 限制 | **远大于 1MB** 的寻址空间 |
| 可直接 **`INT` 调 BIOS** | **不能再调 16 位 BIOS** |
| 汇编为主 | 适合 **C 写复杂逻辑** |

**关键约束：** 进 32 位 **之前**，必须在 16 位阶段做完所有 **BIOS 依赖**：

- 设定 **画面模式**（`INT 0x10`）
- 获取 **键盘状态** 等

做完 → **切换 CPU 模式** → 之后靠 **自己写的代码** 驱动硬件。

#### 引入 C 语言

| 文件 / 符号 | 作用 |
|-------------|------|
| **`bootpack.c`** | OS 本体主要逻辑（C） |
| **`HariMain`** | C 程序 **入口**（类似 `main`） |

**意义：** 复杂功能 **不必再堆汇编** — 为 Day 4+ 图形、内存管理、多任务 **铺路**。

→ 对照 [01-CSAPP](../../01-CSAPP-3rd/) · C 与机器码边界 · [05-LKD](../../05-Linux-Kernel-Development/) 内核 mostly C

### ④ 汇编与 C 的结合 · `io_hlt`

C **不能直接写** 某些 **CPU 专用指令**（如 **`HLT`** — 让 CPU **休眠省电**）。

**做法：** 汇编 **包装**，C **调用**，链接器 **拼在一起**。

```
bootpack.c          naskfunc.nas
    │                    │
    │  调用 io_hlt()     │  io_hlt: HLT; RET
    └────────┬───────────┘
             ▼
        链接 → bootpack 可执行
```

| 层 | 职责 |
|----|------|
| **C** | 业务逻辑、`HariMain`、调用 `io_hlt()` |
| **汇编 `naskfunc.nas`** | 提供 **`io_hlt`** 等 **底层原语** |
| **链接** | `.obj` / `.o` 合并成最终二进制 |

**HFT 模式：** 热路径 **C++** + 少量 **内联汇编 / 独立 .S** — 同一思想：**高级语言写逻辑，汇编只包不可替代的几条指令**。

### Day 3 小结

| 问题 | 答案 |
|------|------|
| IPL 今天真正做了什么？ | **`INT 0x13` 读盘**，**180KB / 10 柱面** 进内存，带 **5 次重试** |
