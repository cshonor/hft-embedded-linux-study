### ② 继续开发 · 深入解析汇编与底层

Day 1 **依葫芦画瓢** 的代码，今天 **逐条拆开**。

#### 基础汇编指令

| 指令 | 含义 |
|------|------|
| **`ORG`** | 告诉汇编器：程序将被装载到内存的 **起始地址** |
| **`JMP`** | **跳转** — 改 IP，下一条指令从别处执行 |
| **`MOV`** | **赋值** — 寄存器 ↔ 寄存器 / 立即数 / 内存 |

#### CPU 寄存器（16 位实模式）

| 类型 | 示例 | 说明 |
|------|------|------|
| **16 位** | AX、CX、DX、BX、SP、BP、SI、DI | 通用 / 栈指针 / 变址 |
| **8 位** | AL/AH（AX 低/高字节）等 | 同一寄存器的 **高低半字** |

**8 个主寄存器加起来 ≈ 16 字节** — CPU **内部存储极小**，大部分数据必须在 **内存** 里。

```
寄存器  ≈  口袋（快、小）
内存    ≈  超大「住宅区」（慢、大）
```

**HFT 直觉：** L1/L2 cache、寄存器分配、**少访存** — 根源就是 **寄存器快、内存慢**（数量级差）。

#### 内存读写 · 方括号 `[]`

```nasm
MOV AL, [SI]    ; 把 SI 指向的 **内存地址** 上的 1 字节读入 AL
```

- **`[]`** = **内存地址**（不是寄存器里的值本身）
- 访问内存 **比访问寄存器慢得多** — 高效程序 = **少碰内存、多复用寄存器**

→ 对照 [01-CSAPP Ch3](../../01-CSAPP-3rd/chapter-03-machine-level-programs/) · [Ch6 存储层次](../../01-CSAPP-3rd/chapter-06-memory-hierarchy/)

#### BIOS 与中断 · `INT`

- **`INT`** = **软件中断**，类似 **函数调用**
- 主板 **出厂固化 BIOS** — 通过指定中断号调用，完成 **显示字符、读键盘** 等基础 I/O
- Day 1 能 **hello, world**，很大程度是 **调 BIOS**，不是 OS 自己驱动显卡

| 概念 | 说明 |
|------|------|
| **BIOS** | 基本输入输出系统，上电后可用 |
| **`INT n`** | 跳到中断向量表第 n 项 — **间接调用** BIOS/固件例程 |

→ 后续 Day 会 **自己写中断处理**；Linux 里对应 **syscall / 驱动** 层。

#### 为什么是 `0x7c00`？

| 问题 | 答案 |
|------|------|
| **`ORG 0x7c00`** 从哪来？ | 早期 **IBM PC + Intel** 约定 |
| 启动时发生什么？ | BIOS 把 **引导扇区 512 字节** 读入内存 **`0x7c00`**，从此处开始执行 |
| 和 Day 1 的关系？ | `helloos.nas` 必须 **ORG 到 0x7c00**，否则跳转、相对地址全错 |

```
磁盘 扇区0 (512B)  ──BIOS 加载──►  物理内存 0x7c00  ──►  CPU 从此执行 IPL
```

### ③ 先制作启动区 · `ipl.bin`

不再让汇编器 **一次吐满 1440KB 软盘**，改为：

```
helloos.nas  ──nask──►  ipl.bin (512 B，纯 IPL)
                            │
                            └── 映像工具 ──►  helloos.img (1440 KB 完整软盘)
```

| 产物 | 大小 | 作用 |
|------|------|------|
| **`ipl.bin`** | **512 B** | 仅 **启动区 / IPL**，迭代快 |
| **`helloos.img`** | 1,474,560 B | 完整可启动软盘（IPL + 其余扇区填 0 等） |

**好处：** 改 IPL 只重编 **512 字节**；整盘用 **磁盘映像管理工具** 拼装 — 为后面 **OS 本体放别的扇区** 铺路。

### ④ Makefile 入门

步骤变多：汇编 → 拼映像 → 开模拟器 … 若全靠多个 **`.bat`**，目录 **又乱又难记**。

引入 **GNU Make** + **`Makefile`**：

```makefile
# 示意（非原书完整文件）
ipl.bin: helloos.nas
	nask helloos.nas helloos.lst ipl.bin

helloos.img: ipl.bin
	# 映像工具把 ipl.bin 写入引导扇区 …

run: helloos.img
	# 启动 QEMU …
```

命令行只打：

```bash
make        # 按依赖自动执行需要更新的步骤
make run    # 构建并运行（若规则已写）
```

| 对比 | `.bat` 堆叠 | **Makefile** |
|------|-------------|--------------|
| 依赖关系 | 手写顺序、易漏 | **目标–依赖–命令**，增量构建 |
| 目录 | 多个脚本 | **一个 Makefile** 收拢 |
| 习惯 | Windows 批处理 | 与 **Linux 内核 / HFT 工程** 同一套 |

**HFT：** 策略引擎、网关 **CMake / Make / Ninja** — 同一思想：**声明依赖，工具链自动增量编译**。
