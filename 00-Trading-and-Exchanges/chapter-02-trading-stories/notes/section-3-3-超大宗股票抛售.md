## 3. 超大宗股票抛售 (A Very Large Block Stock Trade)

Edna 须抛售 **90 万股** 家族企业股票（数千万美元）缴遗产税：

1. 高盛 **大宗经纪 (block broker)** 接手
2. **不能** 直接在公开市场抛售（会 **价格暴跌**）
3. 查 **13F 公开持仓**，定位对该股有兴趣的大型基金
4. 场外 **Upstairs market** 成交：提供 **价格折扣 (discount)** + 说明抛售原因是 **纳税而非基本面恶化**

| 概念 | HFT 关联 |
|------|----------|
| Block / upstairs | 与 HFT 热路径分离的大宗协商市场 |
| Information leakage | 大单意图暴露 = 最大敌人 |
| 13F | 公开信息用于找 counterparty——HFT 不碰此类但需知 institution 行为 |

---

### OTC 折扣 = 「流动性补偿」

Edna 若把 **100 万股** 直接在 **公开 LOB** 砸出去：

- 每吃一档深度，价格就被 **往下打一档**（walk-the-book + **市场冲击**）
- 最终 **成交均价** 远低于开始看到的 Level 1 卖价 — **自己砸崩自己**

所以走 **OTC / Upstairs**：**不经过公开订单簿竞价**，大宗经纪 **直接找对手盘** — 通常是对冲基金、大型资管 — **一次性谈成**：

| 谁 | 得到什么 |
|----|----------|
| **卖方（Edna / 机构）** | 比公开慢慢砸盘 **更高的均价**；快速出清；**不暴露** 在 Level II 上 |
| **买方（对手盘）** | **折扣价**（如比市价低 2%）— 补偿其 **接大单、扛短期波动、占用资金** 的风险 |
| **公开市场** | 价格 **不被大单瞬间冲击** — 信息仍可通过事后披露传导，但无「瀑布式砸盘」 |

**折扣不是「白送便宜货」** — 是买方要求的 **流动性补偿 (liquidity premium)**：

```
公开砸盘损失  ≈  冲击成本 + 暴露意图被 front-run
OTC 折扣      ≈  买方承担 inventory risk 的报价
机构算账      →  若 折扣 < 公开冲击成本，走 OTC 更划算
```

**例子（数字直觉）：**

- Level 1 价 100；公开卖 100 万股可能 **均价跌到 96**
- OTC 谈成 **98 一次性成交** — 卖方比 96 好，买方比 100 便宜 2% 换 **立刻全接 + 扛波动**

### 公开挂单的信息泄露：抢跑 → 砸盘 → 再接货

大资金 **怕在公开簿挂超大限价单**，不只因为 walk-the-book 的 **冲击成本** — 还因为 **意图一露，投机盘会抢跑**：

```
市场看到你挂「超大量限价卖」
  → 投机资金快速做空 / 先砸盘，把价格打下去
  → 你在更低价位被迫成交（或被定点市价买扫走后价格已转弱）
  → 他们反手在低价接货，等价格回升赚差价
```

| 尺度 | 书上哪里讲过 | 机制 |
|------|--------------|------|
| **散户 / 中等挂单** | [Ch 1 §4.2 · 限价单期权](../../chapter-01-introduction-market-microstructure/notes/section-4-4-贯穿全书的关键主题.md#42-期权与外部性-options-and-externalities) | 推价 → **市价买扫你的 Ask** → **市价卖砸 Bid** → 低价接回 |
| **机构超大块** | 本节 Edna | 同上逻辑的 **放大版** + 公开簿 **Level II 全员可见** → 更多 front-run 窗口 |

这就是典型的 **「抢跑砸盘再接货」套利** — 你挂限价单给的 **免费期权**（§4.2），在大单场景下变成 **系统性信息泄露**。

**暗池为什么存在：** 把这条信息链 **掐断** —

| 公开 LOB | 暗池 |
|----------|------|
| 挂单进 Level II，**谁都能看见** | 大单 **不出现在公开 Orderbook** |
| 投机盘有时间 **布局、抢跑** | 通常只有 **撮合双方 + 运营方** 知道 |
| 成交价格被 **公开市场踩踏** 牵动 | 买卖在 **双方认可价** 直接交叉，少一层公开竞价 |

**OTC 折扣再理解一遍（和暗池互补）：**

- **卖方** 愿意 **让一点利（折扣）** → 换 **快速成交 + 不暴露意图**、躲开抢跑与砸盘
- **买方** 拿 **折扣接货** → 补偿 **锁仓、短期波动、不能立刻在公开市场变现** 的风险

```
暗池  = 藏单，在池内撮合，抹平「公开簿可见」的信息差
OTC   = 连池子都 bypass，直接找对手 + 折扣换流动性
§4.2  = 同一套 pick-off 逻辑在散户档位上的慢动作演示
```

### 暗池 vs OTC vs 公开簿：三种大单出路

| | **公开 LOB**（§1 Jennifer） | **暗池**（§2 Bob · POSIT） | **OTC / Upstairs**（本节 Edna） |
|---|---------------------------|---------------------------|----------------------------------|
| **单去哪** | 挂进 **公开 Orderbook** | **藏单** — 不进公开 Level II | **场外** — 不经 LOB 竞价 |
| **怎么找对手** | 价格–时间优先自动撮合 | 暗池内 **保密交叉**（常近 mid） | 经纪 **点名找** 对手（13F、关系网） |
| **价格** | 市场竞价 | 多在 **价差中间** 附近 | **协商折扣** — 流动性补偿 |
| **核心目的** | 散户常规成交 | **藏意图**、减冲击、**防抢跑** | **超大块一次清**、避免砸公开价 |
| **互补关系** | 基础池子 | **藏单** | **直接找对手给优惠价** |

```
机构大单要出
    ├─ 能拆小、能等 → 公开簿 + 算法拆单（§2）
    ├─ 要保密、中等规模 → 暗池 POSIT（§2）
    └─ 超大、必须快 → OTC / Upstairs + 折扣（本节）
```

> **和 go-dex：** 你现在只有 **公开 `Orderbook` + `Match()`**（§1 散户 + MM）。**暗池** = 第二本簿 / internal cross（M5）；**OTC** = **连 LOB 都不进** 的协商成交模块 — 成交价写进系统但不 walk 公开 `Bids`/`Asks`（M5+ 或独立 simulation）。
