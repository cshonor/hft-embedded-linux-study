## 1. 流动性的本质：双边搜索 (Bilateral Search)

### 1.1 定义

**流动性**：在你 **想要交易时**，能够以 **较低成本**、**迅速** 进行 **大宗交易** 的能力。

### 1.2 双边搜索

流动性 = **双边搜索的成功结果**：

```
买方寻找卖方  ↔  卖方寻找买方
        ↓
双方接受条件成交 → 找到流动性
```

| 搜索类型 | 角色 | 行为 |
|----------|------|------|
| **主动搜索** | 缺乏耐心者（**市价单**） | **提取** 流动性 |
| **被动搜索** | 耐心者（**限价单** / 等 broker 询问） | **提供** 流动性 |

**交易所 + 做市商网络** = 降低买卖双方搜索成本的 **「搜索引擎」**。

| HFT 视角 |
|----------|
| **Matching engine + LOB** = 搜索 **基础设施** |
| **Co-lo、direct feed** = 降低 **搜索延迟**（发现对手的速度） |
| **Dark pool / upstairs** = **另类搜索渠道** — [Ch 15](./chapter-15-大宗交易者.md) · [Ch 18](./chapter-18-买方交易者.md) |

## 2. 流动性的维度 (Dimensions of Liquidity)

流动性是 **多维概念** — 讨论时须指明 **哪一维**（避免混淆）。

### 2.1 三大核心 + 一衍生

| 维度 | 英文 | 问什么 | 典型关注者 |
|------|------|--------|------------|
| **即时性** | Immediacy | 给定成本下，安排 **特定规模** 的 **速度** | 急躁小额 taker |
| **宽度** | Width | 给定规模下的 **成本** | 小额交易 |
| | | 体现：**买卖价差** + **佣金** | [Ch 14](./chapter-14-买卖价差.md) |
| **深度** | Depth | 给定成本下能成交的 **规模** | **大资金** |
| **弹性** | Resiliency | 价格因 **不知情大单失衡** 偏离基本面后，**恢复的速度** | 市场质量 / 监管 |

→ 弹性来源 [Ch 16 价值交易者](./chapter-16-价值交易者.md)

### 2.2 维度间的权衡

| 想要… | 通常牺牲… |
|--------|-----------|
| **更大规模（深度）** | 更差价格（**宽度**）或更多时间（**即时性**） |
| **更快（即时性）** | 更宽 spread / 更大 impact |
| **更窄（宽度）** | 排队等待 / 不保证成交 |

```
        Immediacy
           /\
          /  \
         /    \
    Width ——— Depth
         \    /
          \  /
       Resiliency（偏离后恢复）
```

| HFT 视角 |
|----------|
| **BBO spread** → width；**book depth at N ticks** → depth |
| **Recovery time after shock** → resiliency（flash crash 后多久回 mid） |
| Go DEX M3：对同一 `orderbook` 测 **spread、累计深度、大单冲击后 mid 恢复** |

## 3. 谁在提供流动性？(Who Provides Liquidity)

五类主要 **流动性供给者** — 不同 **信息、资本、耐心** → 不同 **生态位**：

### 3.1 做市商 (Market Makers)

| 特点 | 对基本面 **了解不多** |
|------|----------------------|
| **提供** | **狭窄价差** → 不知情 **小额** 交易者的 **即时性** |
| **维度** | **Width（窄）+ Immediacy** |

→ [Ch 13](./chapter-13-做市商.md) · 电子 **HFT MM**

### 3.2 大宗交易做市商 (Block Dealers)

| 特点 | **非常了解客户** |
|------|------------------|
| **提供** | 确信 **不知情** 的大客户 → **深度 (Depth)** |
| **规避** | **逆向选择** — [Ch 15](./chapter-15-大宗交易者.md) |

### 3.3 价值交易者 (Value Traders)

| 特点 | **最了解基本面** |
|------|------------------|
| **角色** | **终极流动性提供者** |
| **时机** | 价格偏离价值、**含做市商在内无人愿交易** 时 **敢于介入** |
| **维度** | **深度** + **弹性 (Resiliency)** 的 **主要来源** |

→ [Ch 16](./chapter-16-价值交易者.md)

### 3.4 预先承诺的交易者 (Precommitted Traders)

| 特点 | **效用型需求**（投资、对冲）提交 **限价单** |
|------|---------------------------------------------|
| **动机** | 为 **更好价格** 而提供流动性 |
| **维度** | 窄价差下提供 **即时性**；通常 **不大规模暴露深度**（防意图泄露） |

→ [Ch 8 效用型](./chapter-08-为什么人们要交易.md) · [Ch 18 暴露](./chapter-18-买方交易者.md)

### 3.5 套利者 (Arbitrageurs)

| 特点 | 了解 **相对价值** |
|------|------------------|
| **角色** | 严格说 **非创造者** — **流动性搬运工 (Porters)** |
| **机制** | 从一市场 **搬** 到另一 **急需流动性** 的市场 → **跨市场增加深度** |

→ [Ch 17](./chapter-17-套利者.md)

## 5. 提取 vs 提供（与全书对照）

| 行为 | 交易者 | 维度消费 |
|------|--------|----------|
| **提取** | 市价 / urgent taker、[Ch 18](./chapter-18-买方交易者.md) 急躁方 | 买 **immediacy**，付 **width** |
| **提供** | MM、limit、value、block dealer | 卖 **immediacy/depth**，收 **spread** 或等 **mispricing** |

| 破坏流动性 | [Ch 11 寄生](./chapter-11-指令预期者.md)、[Ch 12 操纵](./chapter-12-虚张声势者与市场操纵.md) — 提高 **有效 width**、降低 **resiliency** |
|------------|----------------------------------------------------------------------------------------------------------------------------------|

## 6. 与 Ch 9「好市场」

[Ch 9](./chapter-09-好市场.md)：**好市场 = 高流动性 + 信息效率价格**。

本章将 **高流动性** 拆解为：

- **低成本（width）**
- **快速（immediacy）**
- **大规模（depth）**
- **扰动后恢复（resiliency）**

→ 监管与 **market quality metrics** 的 **理论清单**

## 7. 本章总结

| 要点 | 含义 |
|------|------|
| **本质** | 双边搜索成功 — 找到愿成交的对手 |
| **四维** | Immediacy · Width · Depth · Resiliency — **常须权衡** |
| **五类供给** | MM · Block dealer · Value · Precommitted · Arb porter |
| **生态** | 信息/资本/耐心不同 → **分工供给** 不同维度 |
| **基础设施** | 交易所 = **搜索引擎** |

> **HFT 读者 takeaway：** 说「提供流动性」须指明 **哪一维** — HFT MM 主供 **immediacy + 窄 width**；**真 depth + resiliency** 靠 **value capital** 与 **utilitarian limits**。读 `orderbook.go` 时：**best bid/ask** = width 快照；**各档累计量** = depth；**撮合后 mid 多久回位** = resiliency（需事件驱动测）。M3 练手建议把四维 **写成可测指标**。
