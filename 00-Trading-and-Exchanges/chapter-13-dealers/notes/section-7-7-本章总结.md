## 7. 本章总结

| 要点 | 含义 |
|------|------|
| **角色** | 卖 **immediacy**；被动成交 |
| **盈利** | Bid-ask spread；须 **吸引 order flow** |
| **理想** | 出清价附近 **双向 flow** |
| **库存风险** | 可分散（温和）vs **逆向选择**（致命） |
| **调价** | 库存低/高 → **双边同向** 移价 |
| **防 informed** | 快撤、**加宽 adverse selection 成分**、警惕大单/匿名 |
| **盟友** | **价值交易者** 帮 unload；**敌人** bluffer 利用 skew 习惯 |

> **HFT 读者 takeaway：** 做 maker = 本章主角。读 `orderbook.go` 时分清：**matching 规则**（Ch 6）vs **做市商 skew 策略**（本章）— 前者是交易所 spec，后者是你的 **PnL 引擎**。下一章 [Ch 14](../chapter-14-bid-ask-spreads/) 把 spread **拆成可估分量**。
