# 条款 7：永远不要重载逻辑与 &&、逻辑或 ||、逗号运算符

## 本节讲什么

程序员高度依赖 `&&`、`||`、`,` 的**内置语义**：短路求值、从左到右的计算顺序。重载这三个操作符会把它们变成普通函数调用，**无法复现**这些保证，代码变得危险且难懂。操作符重载应让程序更易读写，而非迷惑他人——因此**不要重载** `&&`、`||`、`,`。

## 核心结论

| 操作符 | 内置保证 | 重载后 |
|--------|----------|--------|
| **`&&` / `||`** | 短路求值（右侧可能不执行） | 两侧**都必须**先求值 → 短路丧失 |
| **`,`** | 先算左、再算右，结果为右值 | 参数求值顺序**未定义** → 顺序丧失 |

## 1. 重载 `&&` 和 `||`：丧失短路求值

### 内置行为

对内置类型，C++ 使用**短路求值**：一旦左操作数能决定整个表达式的真假，**右操作数不再执行**。

```cpp
if ((p != nullptr) && (strlen(p) > 10)) {
    // p 为空时，strlen(p) 绝不会被调用 → 避免崩溃
}
```

### 重载后发生什么

重载的 `&&` / `||` 在底层变为函数调用，例如：

```cpp
expression1.operator&&(expression2);
// 或
operator&&(expression1, expression2);
```

带来两处致命变化：

1. **必须计算所有参数**：函数调用前，左右两侧表达式都要求值 → **短路特性荡然无存**。
2. **参数求值顺序未定义**：标准不规定先算左还是先算右 → 行为不可预期。

```cpp
class Widget {
public:
    bool operator&&(const Widget &) const {
        return true;  // 即便写在这里，也无法恢复短路
    }
};

Widget w1, w2;
// if (w1 && expensive(w2)) {}  
// 等价于 w1.operator&&(w2) → expensive(w2) 总会被求值
```

### 典型灾难

若右操作数含副作用或可能抛错/崩溃（如 `strlen(p)`、`p->foo()`），重载后的 `&&` 会在「本可跳过右操作数」时仍执行它。

## 2. 重载逗号 `,`：丧失从左到右顺序

### 内置行为

内置逗号运算符：**先**求值左侧，**再**求值右侧，整个表达式的结果是**右侧**的值。常见于 `for` 循环更新部分：

```cpp
for (int i = 0; i < n; ++i, doSomething(i)) {
    // 保证 ++i 先于 doSomething(i)（在同一逗号表达式内）
}
```

### 重载后发生什么

重载 `,` 同样变成双参数函数调用；**C++ 不保证函数实参的求值顺序**，因此**无法保证**左表达式先于右表达式计算。

```cpp
class Coord {
public:
    Coord operator,(const Coord &) const {
        return *this;  // 无法恢复「先左后右」
    }
};

Coord a, b, c;
// (a, b, c) 若依赖内置顺序，重载后顺序不可依赖
```

## 为什么「努力模拟」也徒劳？

- 在 `operator&&` 内部可以写 `if (!lhs) return false;`，看似短路，但 **`rhs` 作为函数实参在进入函数前往往已被求值**（取决于调用形式与编译器，不能 portable 地依赖）。
- 逗号运算符的「左→右 + 返回右值」与函数参数求值规则根本冲突。

重载的初衷是让类用起来像内建类型；对这三个操作符，**你永远无法**让行为与程序员长期依赖的内置规则一致。

## 正确做法

- **不要**为任何类重载 `&&`、`||`、`,`。
- 需要组合条件时，用**命名成员函数**或**自由函数**，名字表达意图，例如 `both()`、`either()`，调用方清楚两侧都会求值。
- 需要「顺序执行并取后者」时，用语句序列或显式 API，不要假装是逗号运算符。

```cpp
class Safe {
public:
    bool isValid() const;
    bool matches(const Safe &) const;
};

Safe s1, s2;
if (s1.isValid() && s1.matches(s2)) {  // 使用内置 &&，短路成立
    // ...
}

// 不要：
// bool operator&&(const Safe &) const;
```

## 示例对比

```cpp
#include <cstring>

void builtin_short_circuit(const char *p) {
    if (p && std::strlen(p) > 0) {  // p 为 nullptr 时不调用 strlen
        // ...
    }
}
```

## 小结

- **`&&` / `||`**：重载 → 函数调用 → 无短路、参数顺序未定义。
- **`,`**：重载 → 无法保证左先于右。
- 操作符重载应增强可读性；这三个操作符重载只会**打破**程序员对语言的基本假设。
- **明智做法：不要重载 `&&`、`||`、`,`。**
