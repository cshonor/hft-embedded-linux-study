# 条款 1：分清指针（*）和引用（&）的本质区别、使用场景边界

## 本节讲什么

指针（`*`、`->`）与引用（`.`）都能**间接访问**其他对象，但语义与约束不同。选错会带来空指针隐患、多余分支、或违反直觉的运算符重载接口。本条款梳理三处核心差异，并给出「何时用指针、何时用引用」的决策边界。

## 核心差异

### 1. 是否可为空与初始化要求

- **引用**：在任何情况下都不能指向「空值」；必须绑定到某个对象，因此**必须初始化**。
- **指针**：可以为 `nullptr`，也可以处于未初始化状态（未初始化指针本身也是 bug 来源）。

```cpp
int x = 42;
int &rx = x;        // OK：引用必须绑定对象
int *px = &x;       // OK
int *pnull = nullptr; // OK：指针可为空
// int &ry;         // 错误：引用必须初始化
```

### 2. 效率

由于不存在「空引用」，使用引用时通常**不必**在使用前做合法性检查；而指针在语义上可能为空，**应**在解引用前检查（或保证不变量），否则可能产生额外分支与防御性代码。

```cpp
void use_ref(int &r) {
    r = 10;  // 无需 if (&r) ...
}

void use_ptr(int *p) {
    if (p) {  // 指针应检查
        *p = 10;
    }
}
```

### 3. 是否可重新绑定

- **指针**：可随时改指向另一个对象。
- **引用**：初始化后**永远**绑定同一对象；后续赋值改的是**被引用对象**的值，不是引用本身。

```cpp
int a = 1, b = 2;
int *p = &a;
p = &b;       // OK：p 改指向 b

int &r = a;
r = b;        // 不是让 r 绑定 b，而是把 a 赋值为 2
// int &r2 = b; r = r2;  // 同理：改的是 a，不是改绑
```

## 如何决定：指针还是引用？

| 场景 | 更合适的选择 |
|------|----------------|
| 可能「不指向任何对象」 | **指针**（可设为 `nullptr` 表达「无对象」） |
| 需要在不同时刻指向不同对象 | **指针**（可改指向） |
| 始终指向同一对象，且绑定后不再改变 | **引用** |
| 重载 `operator[]` 等需返回可赋值左值的运算符 | **引用**（保持 `v[i] = 10` 的直觉语义） |

**用指针**：存在「无对象」的可能，或需要改指向。

**用引用**：始终有对象、绑定不变；尤其运算符重载要返回「可被直接赋值的目标」时（若返回指针，调用方得写 `*v = 10`，语义丑陋且易错）。

## 示例

```cpp
#include <vector>

class IntVector {
    std::vector<int> data_;
public:
    int &operator[](std::size_t i) { return data_[i]; }  // 返回引用：v[i] = 10
    // int *operator[](std::size_t i);  // 若返回指针：*v[i] = 10，不推荐
};

void maybe_process(int *p) {  // 可能为空 → 指针
    if (!p) return;
    *p = 0;
}

void always_process(int &r) {  // 必有对象 → 引用
    r = 0;
}

int main() {
    int x = 42;
    int &rx = x;
    int *px = &x;

    rx = 10;
    if (px) *px = 20;

    IntVector v;
    v[0] = 10;  // 依赖 operator[] 返回引用
}
```

## 小结

- 引用不能为空、必须初始化、不能改绑；指针三者皆可（后两者是特性也是风险）。
- 引用常略省空检查；指针表达可选性与可变指向更自然。
- 运算符重载需要「可赋值左值」时，优先返回引用而非指针。
