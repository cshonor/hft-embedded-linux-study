# 1.5.2 链接、编译产物与示例

## 本节讲什么

编译链接流程、.o/.exe/.lib、示例代码。

## 链接：编译器怎么找到实现？

```
main.cpp  #include "Sales_item.h"
    ↓ 编译：编译器从头文件知道 isbn() 存在 → 能通过编译
    ↓
Sales_item.cpp  → 单独编译成目标文件
    ↓ 链接：把 item.isbn() 的调用点和 .cpp 里的实现「绑」在一起
    ↓
可执行文件 → 运行时正确执行 isbn() 里的代码
```

你在 `main` 里写 `item.isbn()` 时：
1. **编译阶段**：只看 `.h`，确认 `isbn()` 名字、参数对得上就行
2. **链接阶段**：去 `.cpp` 编译出的模块里找到 `isbn()` 的真正代码并接上

所以：**头文件给编译器看，实现文件给链接器接。**

## 编译产物：从 .cpp 到「成品」

GCC、Clang、MSVC、TCC 等编译器做的事一样：**把人写的代码翻译成机器码**。步骤拆开看：

```
Sales_item.cpp  ──编译──►  Sales_item.o     （目标文件：机器码，还不能单独跑）
main.cpp        ──编译──►  main.o
                              │
                              ▼ 链接（把 .o 和库绑在一起）
                         ┌────┴────┐
                         ▼         ▼
                    book.exe    libSales.a / Sales.dll
                  （可执行程序）  （给别人用的库）
```

| 产物 | 常见后缀 | 是什么 | 给谁用 |
|------|----------|--------|--------|
| **目标文件** | `.o` / `.obj` | 单个 `.cpp` 编译出的机器码片段 | 链接器内部用，一般不单独给别人 |
| **可执行文件** | `.exe`（Windows）/ 无后缀（Linux） | 能直接运行的完整程序 | **终端用户**双击/命令行运行 |
| **静态库** | `.lib`（MSVC）/ `.a`（GCC/Clang） | 实现代码打包进库，链接时拷进你的程序 | **开发者**在自己的项目里链接 |
| **动态库** | `.dll`（Windows）/ `.so`（Linux） | 实现代码在单独文件里，运行时加载 | **开发者**链接 + 运行时带上 dll |

### 给别人当「模块」调用，给什么？

**不是** `.exe` + `.h`。

- **`.exe`**：成品程序，给**用户运行**的，不能当 SDK 让别人链接。
- **给别人调用你的类/函数**，应提供：

| 场景 | 给别人什么 |
|------|------------|
| **静态库** | `.h` + `.lib`（或 `.a`） |
| **动态库** | `.h` + `.lib`（导入库）+ `.dll`（运行时要有） |

对方在自己的项目里 `#include "Sales_item.h"`，链接你的 `.lib`，就能调 `isbn()` 等方法，**全程看不到你的 `.cpp` 源码**。

### Windows 上记一句

```
想让别人「调用你的模块」  →  .h + .lib（+ .dll）
想让别人「直接跑你的程序」  →  .exe（一般不需要给 .h）
```

HFT 里核心策略/行情解析常做成 **静态库或动态库 + 头文件**，交易主程序只链接接口，实现藏在二进制里。

## 头文件长什么样？（`Sales_item.h` 极简版）

```cpp
#pragma once              // 防止头文件被重复包含（也可用 #ifndef 守卫）

#include <string>

class Sales_item {
public:
    std::string isbn() const;   // 声明：有这个方法，具体实现在 .cpp
    // ... 其他 public 接口（>>、<<、+ 等）

private:
    std::string bookNo;         // 数据成员：外部不能直接改
    unsigned units_sold = 0;
    double revenue = 0.0;
};
```

别人 `#include "Sales_item.h"` 之后就知道怎么调 `isbn()`，但**看不到**函数体，也**碰不到** `private` 里的 `bookNo`。

**`Sales_item.cpp`（实现放这里）：**

```cpp
#include "Sales_item.h"

std::string Sales_item::isbn() const {
    return bookNo;              // 具体怎么取书号，写在这里
}
```

## 示例

```cpp
// 需配合书中 Sales_item 头文件使用；此处为用法示意
#include <iostream>
#include "Sales_item.h"

int main() {
    Sales_item item;
    std::cin >> item;                          // 从输入读取一条销售记录
    std::cout << item << std::endl;            // 打印该记录
    std::cout << item.isbn() << std::endl;     // 调用成员函数获取书号

    Sales_item item2;
    std::cin >> item2;
    Sales_item sum = item + item2;             // 两笔销售记录相加
    std::cout << sum << std::endl;
    return 0;
}
```
