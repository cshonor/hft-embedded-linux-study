# 4.12.2 求值顺序与括号规范

## 本节讲什么

**求值顺序**与优先级的区别；未指定顺序的陷阱；教材推荐的括号写法。

## 优先级 ≠ 求值顺序

```cpp
int f(), g();
int h = f() + g();   // 先调 f 还是先调 g？**未指定**
```

编译器可任意选择，同一表达式多次运行结果可能一致，但**不能依赖**调用顺序。

| 有保证的情况 | 说明 |
|--------------|------|
| `&&` `||` | 左侧为假/真时，右侧**不求值**（短路） |
| `?:` | 只求值被选中的分支 |
| `,` 逗号 | 左侧先于右侧 |
| 赋值 `=` | 右侧先于左侧（对操作数求值） |

位运算 `&` `|` **没有**短路顺序保证。

## 未定义行为经典坑

### 同一表达式多次修改

```cpp
int i = 0;
// i = i++;           // ❌ UB
// v[i] = i++;        // ❌ UB（下标与右侧 i 无顺序）
```

### 函数实参

```cpp
// f(++i, ++i);       // ❌ UB：哪个实参先求值未指定
```

**HFT / 工程**：热路径表达式保持简单；需要顺序就拆成多条语句。

## 括号规范（教材建议）

1. **不靠背表**解决复杂式——**多加括号**表达意图  
2. 混合算术、关系、逻辑时括号分组  
3. `?:` 与 `<<`、`=` 混用**必加括号**  
4. 位运算与逻辑混用时分组：`(a & mask) != 0`  

```cpp
// 可读性优先
bool ok = (flags & kReady) != 0 && (flags & kValid);
```

## 背诵极简版

```
优先级：谁跟谁先结合；结合律：同级从左/从右
求值顺序：操作数何时算——f()+g() 顺序不定
&& || ?: , 赋值右侧 有顺序保证；& | 无短路
同一式子多次改同一变量 → UB
复杂表达式加括号，别硬背
```

## 示例

```cpp
#include <iostream>

int main() {
    int a = 2, b = 3;
    std::cout << a + b * 4 << "\n";           // 14

    int grade = 75;
    std::cout << (grade < 60 ? "fail" : "pass") << "\n";

    int x, y, z;
    x = y = z = 0;

    return 0;
}
```

## 课后考点

1. 说出 `*` 与 `+`、`.` 与 `*`、`?:` 与 `<<` 的优先级关系  
2. 赋值、前缀/后缀 `++`、三元运算符的结合方向  
3. **优先级 vs 求值顺序**的区别  
4. `i = i++` 为何 UB  
5. 何时用括号代替记忆优先级
