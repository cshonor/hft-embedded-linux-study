# 4.11.1 隐式类型转换

## 本节讲什么

编译器**无提示、自动完成**的类型转换——发生在运算、赋值、初始化、传参等场景。

## 1. 整型提升（最基础）

小于 `int` 的小整型（`char` / `short` / `bool`）参与运算时，先提升为 **`int`**；若 `int` 放不下则提升为 **`unsigned int`**。

```cpp
char c = 'a';
auto res = c + 10;   // c 先提升为 int 再计算
```

HFT 解析单字节字段时，`char` 参与算术前几乎总会先变 `int`——与 [3.2.5 `unsigned char`](../ch03-strings-vectors-arrays/3.2-library-type-string/3.2.5-cctype字符校验.md) 配合，避免符号扩展坑。

## 2. 常用算术转换（混合运算）

**向范围更大、精度更高的类型转换**，尽量避免精度丢失：

- 有浮点 → 全部转浮点（`int + double` → `int` 转 `double`）
- 全整型：无符号优先级常高于有符号；宽度小的转宽度大的

```cpp
int a = 3;
double b = 3.14;
auto sum = a + b;    // a 隐式转为 double
```

## 3. 数组 → 指针（高频）

绝大多数场景数组名**退化为首元素指针**。

**不退化**例外：`sizeof(arr)`、`&arr`、用字符串字面量初始化 `char` 数组（见 [3.5.3](../ch03-strings-vectors-arrays/3.5-arrays/3.5.3-数组退化与指针.md)）。

```cpp
int arr[5];
int *p = arr;        // 等价 &arr[0]
```

## 4. 指针相关隐式转换

| 转换 | 说明 |
|------|------|
| `0` / `nullptr` → 任意指针 | 空指针 |
| 非 `const` 对象指针 → `void*` | 通用指针 |
| 非 `const` 指针/引用 → 对应 **`const`** 版 | **只加 const，不能反向** |

```cpp
int i = 10;
const int &r = i;    // ✅ 普通引用 → const 引用
// int &r2 = r;      // ❌ 不能去掉 const
```

与 [2.4.1 底层 const](../ch02-variables-and-basic-types/2.4-const-qualifier/2.4.1-const基础与constexpr.md) 一致。

## 5. 布尔转换（条件判断）

标量（数字、指针）在条件上下文转 `bool`：

- `0`、空指针 → `false`
- 非 0、有效指针 → `true`

```cpp
int x = 5;
if (x) { /* true */ }

int *p = nullptr;
while (p) { /* false，不进入 */ }
```

## 6. 赋值 / 初始化 / 传参

右侧、实参自动转为左侧、形参类型：

```cpp
double d = 3.99;
int i = d;           // double → int，**截断小数**，i == 3（非四舍五入）
```

**HFT**：价格 `double` 隐式赋给 `int` 会**静默丢小数**——业务层应显式处理或统一用定点/整数价位。
