# 4.11.3 规范、易错点与示例

## 本节讲什么

教材使用规范、高频易错点、背诵版与完整可运行示例。

## 使用规范

1. **优先隐式转换**——能自动统一类型就不手写强转  
2. **必须强转**时，用四种命名 cast，**拒绝** `(type)expr`  
3. 精度丢失、一般指针转换 → **`static_cast`**；只改 `const` → **`const_cast`**；底层比特重解释 → **`reinterpret_cast`**（慎用）  
4. **浮点 → 整型**截断小数，非四舍五入；编译器常给警告  

## 高频易错点

1. 隐式转换**只能单向加 const**，不能去掉 const  
2. **`static_cast` 无法移除 const**，必须用 `const_cast`  
3. 数组转指针有例外：`sizeof(arr)` 是整个数组大小，不是指针大小  
4. 浮点转整型**直接截断**，不是四舍五入  
5. **`reinterpret_cast` 不做校验**，误用易崩溃 / 未对齐访问  

## 背诵极简版

```
隐式：整型提升、算术转换、数组→指针、加const、→bool、赋值截断
显式：static_cast 算术/void*；const_cast 改const；reinterpret 比特；dynamic 多态
拒绝 (int)d 和 int(d)；优先 static_cast
float→int 截断；static_cast 不能去 const
sizeof(arr) 不退化；&arr 类型是 int(*)[N]
```

## 完整示例

```cpp
#include <iostream>

int main() {
    double d = 3.14;
    int i = static_cast<int>(d);
    std::cout << i << "\n";          // 3

    const int ci = 100;
    const int *cp = &ci;
    int *p = const_cast<int*>(cp);   // 仅演示语法；勿通过 p 改 ci

    short s = 5;
    auto sum = s + 2.5;              // short→int→double
    std::cout << sum << "\n";

    int arr[3] = {1, 2, 3};
    int *ap = arr;                   // 隐式数组→指针
    std::cout << sizeof(arr) << " " << sizeof(ap) << "\n";

    return 0;
}
```

## 课后核心考点

1. 隐式 vs 显式；何时编译器自动转  
2. 整型提升、算术转换、数组退化（及例外）  
3. 四种命名 cast 的**适用边界**  
4. `const` 只能加不能减；`const_cast` 的 UB 风险  
5. 浮点截断、老式强转为何不推荐
