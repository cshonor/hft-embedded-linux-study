# 4.11.2 显式转换与四种 cast

## 本节讲什么

老式 C 风格强转 vs C++ 四种命名强制转换：`static_cast`、`const_cast`、`reinterpret_cast`、`dynamic_cast`。

## 老式转换（不推荐）

```cpp
double d = 3.14;
int i1 = (int)d;     // C 风格
int i2 = int(d);     // 函数式
```

**缺点**：一次写法可同时完成**去 const、指针重解释**等多种语义，难以审查，易埋雷。

## C++ 命名转换（推荐）

统一语法：**`转换名<目标类型>(表达式)`**

### ① `static_cast`（最常用）

**温和、相对安全**的编译期转换：

- 基础算术互转（`double`↔`int`、`float`↔`long`）
- `void*` 与普通指针互转
- 继承体系父子类指针**上行/下行**（编译期检查有限）

**不能**修改 `const` 属性。

```cpp
double d = 3.14;
int i = static_cast<int>(d);   // 3，截断小数
```

### ② `const_cast`（唯一能增删 const/volatile）

仅修改**常量限定**，不改变底层类型。

```cpp
const char *str = "test";
char *p = const_cast<char*>(str);   // 去掉 const（仍须谨慎使用）

const int ci = 100;
const int *cp = &ci;
int *p2 = const_cast<int*>(cp);
// *p2 = 200;  // ❌ 若 ci 原本真是 const，这是 UB
```

**警告**：通过 `const_cast` 修改**原本就是 const** 的对象 → **未定义行为**。合法典型场景：调用只接受非 `const` 指针的遗留 API，而对象本身可变。

### ③ `reinterpret_cast`（底层比特重解释，极高风险）

把内存**二进制位**直接当成另一种类型，**无类型安全检查**：

- 指针与整数互转
- 不相关类型指针互转

```cpp
int num = 0x1234;
char *pc = reinterpret_cast<char*>(&num);
```

**HFT**：解析二进制报文、对齐字段时偶见；日常业务尽量少用，优先 `memcpy` + 明确布局（见 [2.1 基本类型](../ch02-variables-and-basic-types/2.1-基本内置类型.md)）。

### ④ `dynamic_cast`（运行时多态转换）

仅用于**带虚函数**的多态类；基类指针/引用 → 派生类，**运行时**校验，失败指针版返回 `nullptr`。

本章仅提及；详见第 7 / 15 章继承与多态。

## 四种 cast 速查

| cast | 用途 | 风险 |
|------|------|------|
| `static_cast` | 算术、`void*`、继承转换 | 低～中（浮点截断等） |
| `const_cast` | 增删 `const`/`volatile` | 改真 const → UB |
| `reinterpret_cast` | 比特重解释 | **极高** |
| `dynamic_cast` | 多态向下转型 | 低（失败可检测） |
