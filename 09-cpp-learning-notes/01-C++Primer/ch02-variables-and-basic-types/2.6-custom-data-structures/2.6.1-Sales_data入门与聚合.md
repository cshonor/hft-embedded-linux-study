# 2.6.1 Sales_data 入门与聚合

## 本节讲什么

《C++ Primer》入门三成员 `Sales_data`、什么是聚合、花括号与部分赋值。

## 一、`Sales_data` 入门三成员（课本经典）

标准写法（**`struct` 定义末尾必须有分号**）：

```cpp
#include <string>

struct Sales_data
{
    std::string bookNo;              // 书籍 ISBN 编号
    unsigned units_sold = 0;         // C++11 类内默认：销售册数
    double revenue = 0.0;            // C++11 类内默认：总销售额
};  // ← 分号极易漏，漏了编译报错
```

| 成员 | 类型 | 类内初值 | 默认构造后 |
|------|------|----------|------------|
| `bookNo` | `std::string` | 无 | 空串 `""` |
| `units_sold` | `unsigned` | `= 0` | `0` |
| `revenue` | `double` | `= 0.0` | `0.0` |

### 什么是聚合（Aggregate）？

**聚合**是 C++ 对一类特殊 `class` / `struct` 的称呼——本质上就是**只装数据、没有复杂行为**的类型，成员布局简单透明，编译器可以像处理 C 结构体一样处理它。

一句话：**满足聚合条件的类型，可以用花括号 `{}` 按成员声明顺序，直接把各字段一次性初始化完。**

笔记里常说的「**无自定义构造时，合成默认构造 + 聚合初始化**」，拆开理解是：

| 写法 | 发生什么 |
|------|----------|
| `Sales_data item;` | 编译器**合成默认构造函数**，按类内初值 / 默认初始化填成员 |
| `Sales_data data = {"978-7-115-38345-2", 23, 31};` | **聚合初始化**：不调用自定义构造，按顺序给 `bookNo`、`units_sold`、`revenue` 赋值 |

二者都适用于入门版，因为**没有**用户写的构造函数——第一种靠合成默认构造，第二种靠聚合规则。

**成为聚合须同时满足**（C++11/17 入门口径）：

| 条件 | 入门版 `Sales_data` |
|------|---------------------|
| 无 **private / protected** 非 static 数据成员 | ✅ 全 public（`struct` 默认） |
| **无用户提供**的构造 / 析构 / 拷贝或移动赋值函数 | ✅ 只有编译器合成的 |
| **无**私有或 protected 基类 | ✅ 无继承 |
| **无**虚函数 | ✅ |

> **常见误解**：「用了 C++11 类内初始值就不是聚合」——**错**。`units_sold = 0`、`revenue = 0.0` **不妨碍**聚合身份；入门版 **有类内初值且仍是聚合**。真正破坏聚合的是：**用户自定义构造**、**private 成员**、**继承**、**虚函数**等。

入门版三个成员全部满足 → **典型聚合类型**：

```cpp
Sales_data data = {"978-7-115-38345-2", 23, 31};
// bookNo      ← "978-7-115-38345-2"
// units_sold  ← 23
// revenue     ← 31.0（隐式转为 double）
```

等价写法：`Sales_data d{"...", 25, 22.5};` 或 `Sales_data d = {"...", 25, 22.5};`

### 部分赋值（C++11 起）

花括号里的值**不必写满**所有成员；**没写到的成员**走类内默认值（若有）或默认初始化：

```cpp
Sales_data partial = {"book123"};
// bookNo      ← "book123"
// units_sold  ← 0      （类内 = 0）
// revenue     ← 0.0    （类内 = 0.0）
```

**`{}` 与类内初值**：花括号里**已给出**的值覆盖类内默认；**未给出**的成员才用类内 `= 0` / `= 0.0`（`bookNo` 无类内初值则默认空串）。

**HFT 场景**：行情快照、订单委托、成交回报等**纯数据「容器」**，用 `{}` 一次性或**部分**填字段，比调构造函数更简洁，初始化路径也短：

```cpp
struct Order {
    uint64_t order_id = 0;
    double   price    = 0.0;
    int      qty      = 0;
};

Order o{10086, 10.05, 500};   // 全填：按声明顺序
Order o2{10086};              // 部分填：price、qty 走类内 0
```

**一旦「破坏」聚合**——加入自定义构造、private 成员、基类或虚函数（见下文完整版）——就不能再用 `{}` 按顺序填成员，须改用 `Sales_data("...", n, p)` 等构造函数。

```
入门版 Sales_data  ──加自定义构造/私有/虚函数──►  完整版：不再是聚合
     │                                              │
     └─ {} 按顺序初始化                              └─ 构造函数初始化
```
