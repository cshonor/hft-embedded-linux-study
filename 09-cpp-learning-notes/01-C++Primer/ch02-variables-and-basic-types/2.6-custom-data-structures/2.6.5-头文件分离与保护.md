# 2.6.5 头文件分离与保护

## 本节讲什么

struct vs class、.h/.cpp 分离、包含保护、易错点与示例。

## 五、`struct` vs `class`

C++ 中二者**唯一语法差异**是默认访问权限：

| | `struct` | `class` |
|---|----------|---------|
| 默认权限 | **public** | **private** |
| 本示例 | 外部可直接 `item.bookNo` | 成员默认私有，外部不可直接访问 |

教材入门用 `struct` 简化读写；后期改为 `class` 封装数据（见 ch7 类）。

---

## 六、头文件分离式编译（`.h` + `.cpp`）

### 文件分工

| 文件 | 放什么 |
|------|--------|
| **`.h` / `.hpp`** | **声明**：类型定义、函数声明、`extern`、模板、`inline` |
| **`.cpp`** | **定义**：成员函数体、全局变量定义、普通函数实现 |

### 分离示例（完整版骨架）

**Sales_data.h**

```cpp
#ifndef SALES_DATA_H
#define SALES_DATA_H

#include <string>
#include <iostream>

Sales_data add(const Sales_data&, const Sales_data&);
std::ostream& print(std::ostream&, const Sales_data&);
std::istream& read(std::istream&, Sales_data&);

struct Sales_data {
    Sales_data() = default;
    Sales_data(const std::string& s, unsigned n, double p);
    double avg_price() const;
    Sales_data& combine(const Sales_data& rhs);

    std::string bookNo;
    unsigned units_sold = 0;
    double revenue = 0.0;

    friend Sales_data add(const Sales_data&, const Sales_data&);
    friend std::ostream& print(std::ostream&, const Sales_data&);
    friend std::istream& read(std::istream&, Sales_data&);
};

#endif
```

**Sales_data.cpp** — 构造函数与 `avg_price`、`combine`、`add`/`read`/`print` 的实现放此处。

**核心目的**：隐藏实现细节；多 `.cpp` 共用头文件；改 `.cpp` 不必重编全工程（见 [1.5.2 编译产物](../ch01-getting-started/1.5-a-brief-introduction-to-classes/1.5.2-链接编译产物与示例.md)）。

## 七、头文件重复包含保护

### 方案 1：`#ifndef` / `#define` / `#endif`（标准）

```cpp
#ifndef SALES_DATA_H
#define SALES_DATA_H
// ...
#endif // SALES_DATA_H
```

- 全平台兼容；宏名冲突会误屏蔽 → 建议 `项目_模块_文件名_H`

### 方案 2：`#pragma once`（编译器扩展）

```cpp
#pragma once
```

- 一行、无宏冲突；GCC/Clang/MSVC 均支持；非 C++ 标准

| 特性 | `#ifndef` | `#pragma once` |
|------|-----------|----------------|
| 标准 | ✅ | 扩展 |
| 适用 | 跨平台库、嵌入式 | 现代本地项目 |

## 八、高频易错点

1. **`struct` 末尾分号不能漏**
2. **类内初值不能用 `()`**，只能用 `=` 或 `{}`
3. **C++11 之前**不支持类内 `= 0`；内置类型若不初始化，**局部对象**可能是脏值
4. 头文件勿写普通函数实现 / 全局变量定义 → `multiple definition`
5. 加了自定义构造后，**聚合 `{}` 三参数初始化失效**，改用对应构造函数
6. 删掉 struct 前的 `add`/`read`/`print` 原型 → 流构造函数等处 **`read` 未声明**
7. 全局变量在 `.h` 只写 `extern`（见 [2.2.4 extern](../2.2-variables/2.2.4-extern与头文件分工.md)）

---

## 九、背诵极简版

```
friend 友元：外部函数访问 private；struct 前原型 + 内 friend
有源前置：class X; + .cpp 链入，减编译依赖，与 friend 无关
类内初值 ≠ 丧失聚合；自定义构造才破坏聚合
聚合：{} 按顺序/部分赋值；入门 Sales_data 有类内初值仍是聚合
```

## 示例

### 最小可运行（入门三成员，单文件）

复制保存为 `main.cpp`，`g++ -std=c++11 main.cpp -o main` 或 MSVC 等价命令：

```cpp
#include <iostream>
#include <string>

struct Sales_data {
    std::string bookNo;
    unsigned units_sold = 0;
    double revenue = 0.0;
};

int main() {
    Sales_data item1;                                    // "", 0, 0.0
    Sales_data item2{"978-0-201-78345-0", 25, 22.5};    // 聚合初始化

    std::cout << item2.bookNo << " sold " << item2.units_sold
              << ", revenue " << item2.revenue << std::endl;
    return 0;
}
```

### 完整版可运行（单文件，教材接口）

将 [2.6.3 Sales_data 完整版](./2.6.3-Sales_data完整版.md) 全部代码 + 下列 `main` 放在同一 `.cpp`：

```cpp
int main() {
    Sales_data item1;
    Sales_data item2("978123456", 5, 39.9);
    print(std::cout, item2) << std::endl;
    return 0;
}
```

编译：`g++ -std=c++11 sales_demo.cpp -o sales_demo`

### 删掉前置声明必报错（复现 `read` 未声明）

见 [2.6.4 错误示范](./2.6.4-友元与前置声明.md)；去掉 struct 前三行后保留 `Sales_data(std::istream& is) { read(is, *this); }` 即可复现。
