# 2.6.2 类内初始值

## 本节讲什么

C++11 就地成员初始化：语法、优先级、与聚合的关系。

## 二、C++11 类内初始值（in-class initializer）

### 语法限制

只能用 **`=`** 或 **花括号 `{}`**，**不能用圆括号 `()`**（会被当成成员函数声明）：

```cpp
unsigned units_sold{0};   // ✅
unsigned units_sold = 0;  // ✅
unsigned units_sold(0);   // ❌ 编译器误判为成员函数
```

### 初始化优先级

**构造函数初始值列表 > 类内默认值 > 默认初始化**

```cpp
struct S {
    int x = 100;
    S() : x(1) {}   // 最终 x 为 1
};
```

对入门版 `Sales_data`：

- 构造函数没写某成员 → 用类内 `= 0` / `= 0.0`
- `bookNo` 无类内初值 → 默认初始化为空串

### 无自定义构造函数时

编译器生成**合成默认构造函数**，自动应用类内初始值：

```cpp
Sales_data item;
// item.bookNo     == ""
// item.units_sold == 0
// item.revenue    == 0.0
```

### 类内初值 ≠ 丧失聚合（重要纠正）

网上常见说法：「**只要有类内初始值 `{}` / `=`，就不再是聚合**」——**不符合 C++11/17 标准**。

| 因素 | 是否破坏聚合 |
|------|-------------|
| 类内初始值 `units_sold{0}`、`revenue = 0.0` | ❌ **不破坏** |
| 用户自定义构造函数 | ✅ 破坏（完整版 `Sales_data` 因此不是聚合） |
| private / protected 成员 | ✅ 破坏 |
| 基类 / 虚函数 | ✅ 破坏 |

**极简对比**（可复制编译，`g++ -std=c++11`）：

```cpp
#include <string>

// ① 有类内初值 —— 仍是聚合（Primer 入门版）
struct WithDefaults {
    std::string bookNo;
    unsigned    units_sold = 0;
    double      revenue    = 0.0;
};
WithDefaults a = {"978-7", 23, 31};   // ✅ 聚合初始化
WithDefaults b = {"book123"};         // ✅ 部分赋值，其余走类内默认

// ② 无类内初值 —— 也是聚合
struct NoDefaults {
    std::string bookNo;
    unsigned    units_sold;
    double      revenue;
};
NoDefaults c = {"978-7", 23, 31.0};   // ✅ 聚合初始化

// ③ 有自定义构造 —— 不是聚合（与有没有类内初值无关）
struct WithCtor {
    WithCtor() = default;
    WithCtor(std::string s, unsigned n, double r)
        : bookNo(s), units_sold(n), revenue(r) {}
    std::string bookNo;
    unsigned    units_sold = 0;
    double      revenue    = 0.0;
};
// WithCtor x = {"978-7", 23, 31};     // ❌ 不是聚合，不能这样写
WithCtor y("978-7", 23, 31);          // ✅ 用构造函数
```

**结论**：类内写 `变量{值}` **不会**让结构体丧失聚合；**自定义构造函数**才会（完整版即因此改用 `Sales_data(s, n, p)`）。
