# 2.5 处理类型

## 本节讲什么

针对程序中越来越复杂的类型，介绍简化类型管理的几种方法：使用 `typedef` 和 C++11 的 `using` 定义**类型别名**；使用 **`auto`** 让编译器自动推断变量类型；使用 **`decltype`** 根据表达式推断要定义的变量类型，而不实际计算表达式的值。

## 要点

- **类型别名**：`using Str = std::string`（C++11）；`typedef` 为 C 遗留
- **`using` 声明**：`using std::string;` 把名字引入当前作用域（与别名不同）
- **`using` ≠ `#include`**：前者只改名称可见性；后者预编译文本拷贝
- **同名冲突** `A::ABC` / `B::ABC`：禁全局 `using namespace`；双库并存用全限定名
- **`auto`**：必须初始化；默认**丢弃顶层 const 和引用**
- **`auto&`**：保留 const/引用，绑定原对象；**`const auto&`** 只读、零拷贝
- **`decltype(expr)`**：只分析类型，**不执行**表达式；完整保留 const/引用
- **`decltype((var))`**：双层括号 → 左值引用

## 一、类型别名：`typedef` / `using`

### 1. `typedef`（C 语言遗留）

```cpp
typedef int wages;
typedef double WageArr[20];   // WageArr 等价 double[20]
typedef char* CharPtr;
```

缺点：顺序反直觉；**不支持模板别名**；复合 const 易踩坑：

```cpp
typedef char* CharPtr;
const CharPtr p = nullptr;   // 等价 char* const p（常量指针），不是 const char*
```

### 2. `using`（C++11 推荐）

```cpp
using wages = int;
using IntVec = std::vector<int>;
using CharPtr = char*;
```

优势：左右对应直观；**支持模板别名**；数组、指针、const 组合更清晰。

```cpp
template<typename T>
using Vec = std::vector<T>;   // typedef 做不到
```

### 3. `using` 声明：引入命名空间里的名字（C++98 即有）

与上面的**类型别名**不同，这是 **using 声明（using declaration）**——把某个命名空间里的**已有名字**引入当前作用域，省掉长前缀：

```cpp
#include <string>

using std::string;    // 之后可直接写 string，不必 std::string

string name = "book"; // OK
```

```cpp
#include <iostream>

using std::cin;       // 把 std 里的 cin 拉到当前作用域
using std::cout;

int a;
cin >> a;             // 等价 std::cin >> a
cout << a << '\n';    // 等价 std::cout << ...
```

还可一次引入多个（C++17）：

```cpp
using std::cin, std::cout, std::endl;
```

**`using namespace std;`** 是把整个 `std` 里所有名字一次性引入当前作用域——小文件省事，大工程易**命名冲突**（见下节 HFT 习惯）。

### 4. 两种 `using` 对比（本质都是「引入名称」）

| | **类型别名** `using T = U` | **using 声明** `using ns::name` |
|---|---------------------------|--------------------------------|
| 标准 | C++11 | C++98 |
| 作用 | 给类型起**新名字** | 把**已有名字**引入当前作用域 |
| 例子 | `using Str = std::string;` | `using std::string;` |
| 之后写法 | `Str s;` | `string s;`（省略 `std::`） |

Rust 对照里的 `type` 只对应**左边这一列**（类型别名），不对应 `using std::string` 这种声明。

## `using` 与 `#include` 的本质区别

两者常被一起写，但机制完全不同：

| | **`#include`** | **`using` / `using` 声明** |
|---|----------------|----------------------------|
| 阶段 | **预编译**（预处理） | **编译**（作用域 / 类型系统） |
| 做什么 | 把头文件内容**原封不动拷贝**进当前文件（文本替换） | 只声明**名称可见**，**不拷贝**任何实现代码 |
| 例子 | `#include <iostream>` 引入整个 iostream | `using std::cout;` 仅让 `cout` 在当前作用域可用 |
| 重复 | 同一头文件多次包含需守卫（见 [2.6.5](./2.6-custom-data-structures/2.6.5-头文件分离与保护.md)） | 同一 `using` 声明可写多次（有作用域规则） |

典型组合（见 [1.2 初识输入输出](../ch01-getting-started/1.2-初识输入输出.md)）：

```cpp
#include <iostream>      // 预编译：把 iostream 声明/定义文本贴进来
using std::cin;
using std::cout;
using std::endl;

int main() {
    int a;
    cin >> a;
    cout << "hello" << endl;
    return 0;
}
```

`#include` 解决「**有没有这份代码/声明**」；`using` 解决「**名字要不要写全限定名**」。  
与 [2.2.4 `#include` 与 `extern`](./2.2-variables/2.2.4-extern与头文件分工.md) 也不同：`extern` 声明变量实体在别的 `.cpp`，`using` 不涉及跨文件链接。

### HFT / 工程习惯

**原则**：头文件会被多个 `.cpp` `#include`，在头里写 `using` 等于把污染**扩散**到整个工程。

| 写法 | 头文件 `.h` | 源文件 `.cpp` |
|------|-------------|---------------|
| `using namespace std;` | ❌ 几乎禁止 | ⚠️ 仅小工具/单文件 demo |
| `using std::string;` 等**确定不冲突**的声明 | ❌ 不推荐 | ✅ 简化常用名 |
| `std::` 全限定名 | ✅ **最稳**（接口层首选） | ✅ 多库并存时首选 |
| `namespace md = market_data::v2;` | ⚠️ 慎用在头文件 | ✅ 长路径的折中 |

- **稳定性 + 可读性**（HFT 核心）：读代码的人应能**一眼定位符号来自哪个模块**
- **常见做法**：
  1. **头文件 / 对外接口**：全限定名，或仅在 `.cpp` 使用命名空间别名
  2. **`.cpp` 内**：`using std::string;`、`using std::cout;` —— 只引入**本文件确定不会冲突**的名字
  3. **多个第三方库重名**（如 `A::Order` 与 `B::Order`）：**全限定名**，从根源避免冲突
  4. **长命名空间**：`namespace md = market_data::v2;` → 写 `md::Order`，比全路径短，比 `using namespace` / 裸 `using Order` 更清晰
  5. 类型别名 `using OrderId = uint64_t;`（给**类型**起名，不是 using 声明）

```cpp
// ✅ .cpp — 标准库常用名，本文件内基本不会冲突
using std::string;
using std::vector;

string symbol = "IF2406";
```

```cpp
// ✅ .cpp — 两个行情库都有 Order，全限定名最稳
void on_tick() {
    market_data::v2::Order md_order;
    risk_engine::Order     risk_order;
}
```

```cpp
// ✅ .cpp — 长命名空间起短别名（折中）
namespace md = market_data::v2;

void handle(const md::Order& o) {
    md::Book book = o.book();   // 比 market_data::v2::Order 简洁，仍带来源信息
}
```

### 不同命名空间里的同名符号（`A::ABC` vs `B::ABC`）

**命名空间要解决的核心问题**就是：多个库/模块里可能出现**同名**类型、函数、变量。例如 `A::ABC` 与 `B::ABC` 同时存在时，**不能**再写：

```cpp
using namespace A;   // ❌ 与 B 里同名符号大面积冲突
using namespace B;
```

**三种精准控制方式：**

| 方式 | 写法 | 适用 |
|------|------|------|
| **1. 只引入某一个** | `using A::ABC;` | 当前作用域**只用 A 的** `ABC`，且**不要**再 `using B::ABC;` |
| **2. 全限定名** | `A::ABC` / `B::ABC` | 两个都要用 —— **HFT 最常用**，一眼看出来自哪个模块 |
| **3. 命名空间别名** | `namespace A_alias = long_namespace_A;` | 长命名空间缩短前缀，仍保持清晰 |

**方式 1 示例**（本文件/本函数只关心 A 的 `ABC`）：

```cpp
namespace A { struct ABC { /* ... */ }; }
namespace B { struct ABC { /* ... */ }; }

using A::ABC;          // 当前作用域里 ABC = A::ABC

ABC obj;               // 即 A::ABC
// B::ABC b;           // 仍须写 B::ABC，不能裸写 ABC 当 B 的
```

**同一作用域不能同时** `using A::ABC;` **和** `using B::ABC;` —— 两个声明会把同一个裸名 `ABC` 拉进来，编译器报**重复 / 歧义**。需要**同时**用两个时，必须用方式 2：

```cpp
void route() {
    A::ABC a_obj;
    B::ABC b_obj;      // 全限定名区分，复杂策略逻辑里不易搞混
}
```

**方式 3：命名空间别名**（C++11，与类型别名 `using T = U` 不同语法）：

```cpp
namespace market_data::v2 {
    struct Order { /* ... */ };
}

namespace md = market_data::v2;   // 短别名，保留模块信息

md::Order o;                      // 比全路径短；比 using namespace 安全
```

### 工程选型一句话（HFT）

```
单库、常用、不冲突     →  .cpp 里 using std::string;
多第三方库、可能重名   →  A::Order / B::Order 全限定名（最稳）
路径太长但来源单一     →  namespace md = market_data::v2;  再用 md::Order
头文件 / 对外 API      →  全限定名或不在头里写 using
```

**HFT 偏好**：多库并存、策略逻辑复杂时，**默认全限定名**；`using` 声明只在本 `.cpp` 简化**确定无冲突**的常用名；长路径用**命名空间别名**兼顾简洁与可读。

> 详见 ch3 [3.1 命名空间的 using 声明](../ch03-strings-vectors-arrays/3.1-命名空间的using声明.md)、ch18 [18.2 命名空间](../ch18-tools-for-large-programs/18.2-命名空间namespace.md)。

## 二、`auto` 自动类型推导（C++11）

### 基础规则

**必须初始化**；根据**等号右侧**推导；**默认丢弃顶层 const 和引用**。

```cpp
auto i = 42;        // int
auto d = 3.14;      // double
const int ci = 10;
auto x = ci;        // int（顶层 const 被丢弃，拷贝）
```

### `auto&` — 保留引用与底层 const

```cpp
const int ci = 10;
auto &r = ci;       // const int&，保留 const，绑定 ci
int a = 1;
auto &ra = a;       // int&
```

### `auto*` — 指针推导

保留**底层 const**（指向常量的指针）：

```cpp
int val = 5;
auto *p = &val;         // int*
const int cval = 6;
auto *cp = &cval;       // const int*
```

### 三种写法对比

| 写法 | 行为 | 常用场景 |
|------|------|----------|
| `auto` | 值拷贝，丢顶层 const/引用 | 简单局部变量 |
| `auto&` | 绑定左值，保留 const/引用 | 需要改原对象或避免拷贝 |
| `const auto&` | 只读引用，零拷贝 | **范围 for、传参式遍历** |

```cpp
std::vector<std::string> words{"a", "b"};
for (const auto &w : words) { /* 只读，不拷贝 string */ }
```

## 三、`decltype(expr)` 表达式类型推导

**只分析表达式类型，不执行计算**；完整保留 const、引用。

### 规则 1：`decltype(变量名)` — 不加括号

返回变量**声明时的完整类型**：

```cpp
int i = 0;
const int ci = 10;
int &ri = i;

decltype(i) a;       // int
decltype(ci) b;        // const int
decltype(ri) c = i;  // int&（引用必须初始化）
```

### 规则 2：`decltype((变量名))` — 双层括号

`(var)` 是**左值表达式** → 结果为 **`T&`**；原变量有 const 则为 `const T&`：

```cpp
int x = 10;
decltype((x)) rx = x;      // int&

const int cx = 20;
decltype((cx)) rcx = cx;   // const int&
```

### 其他表达式

```cpp
int arr[5];
decltype(arr) arr2;         // int[5]

int num = 1;
decltype(num + 5) res;      // int（算术表达式为右值，无引用）
```

## 四、`auto` vs `decltype` 对比

| 特性 | `auto` | `decltype(expr)` |
|------|--------|------------------|
| 推导依据 | 初始化时的**值** | 任意**表达式** |
| 是否必须初始化 | ✅ 必须 | ❌ 不必（但引用类型须绑定） |
| const/引用 | 默认丢顶层 const、引用 | **完整保留** |
| 括号特殊规则 | 无 | `(var)` → 左值引用 |
| 典型场景 | 迭代器、lambda、简化局部变量 | 模板返回值、精确复刻已有类型 |

## 五、一页速记

```
别名：  using T = ...  优于 typedef
using 声明：using std::x;  引入名字，不是别名
同名 A::ABC/B::ABC：全限定名；不能同时 using A::ABC 和 using B::ABC
namespace 别名：namespace md = market_data::v2;  → md::Order
HFT：多库重名全限定名；.cpp 可 using std::string 等不冲突名
#include 拷贝文本；using 只改名称可见性
auto：  必须初始化；默认拷贝，丢顶层 const
auto&： 绑左值，保留 const/引用
const auto&：只读 + 零拷贝（遍历首选）

decltype(x)   →  x 的声明类型（含 const/引用）
decltype((x))  →  T& 或 const T&（左值表达式）

typedef 坑：const CharPtr → char* const
```

## 综合示例

```cpp
#include <iostream>
#include <vector>

typedef int wages;
using Wage = int;

int main() {
    auto i = 42;

    const int ci = 100;
    auto &r = ci;           // const int&
    auto *p = &i;           // int*

    decltype(ci) c1 = 200;  // const int
    decltype((ci)) cr = ci; // const int&

    std::vector<int> v{1, 2, 3};
    for (const auto &x : v)
        std::cout << x << " ";
    std::cout << std::endl;

    return 0;
}
```

## 与 Rust 对照

`typedef` / `using` 与 Rust 的 `type` 同属**类型别名**——只给已有类型起新名字，不改变内存模型。

| C++ | Rust |
|-----|------|
| `using MyInt = int;` | `type MyInt = i32;` |
| `typedef int MyInt;` | `type MyInt = i32;` |

> **不要混淆**：Rust 的 `type` **不对应** [2.3](./2.3-compound-types/2.3-复合类型.md) 里的指针、引用；那些对应 Rust 的裸指针和借用。

| 概念 | C++ | Rust |
|------|-----|------|
| 类型别名 | `using` / `typedef` | `type` |
| 类型推断 | `auto` / `decltype` | 部分场景编译器推断 + 显式类型 |

## 示例

```cpp
#include <iostream>
#include <vector>
int main() {
    using salary = double;
    salary monthly = 5000.0;

    auto i = 42;
    auto &r = i;
    decltype((i)) k = i;

    std::vector<int> v{1, 2, 3};
    for (auto it = v.begin(); it != v.end(); ++it)
        std::cout << *it << " ";
    std::cout << std::endl;
    return 0;
}
```
