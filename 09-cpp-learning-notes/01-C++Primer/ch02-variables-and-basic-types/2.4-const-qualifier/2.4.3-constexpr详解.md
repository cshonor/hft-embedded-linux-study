# 2.4.3 constexpr 详解

## 本节讲什么

`constexpr`（**constant expression**，常量表达式）C++11 引入，**强制值在编译期计算完成**。可修饰变量、函数、构造函数。与 [3.5.1 数组维度](../ch03-strings-vectors-arrays/3.5-arrays/3.5.1-定义与初始化.md) 高频考点直接相关。

## 核心定义

| | **`const`** | **`constexpr`** |
|---|-------------|-----------------|
| 核心 | **只读**，不可修改 | **只读 + 编译期求值** |
| 值何时确定 | 编译期或运行期均可 | **必须编译期** |
| 典型用途 | 禁止修改的运行期配置 | 数组长度、模板实参、`case` 标签、`static_assert` |

**一句话**：需要**编译期常量**时用 `constexpr`；仅禁止修改用 `const`。

---

## 一、constexpr 修饰变量（数组章节高频）

### 硬性规则

1. 定义时**必须立刻初始化**
2. 初值必须是**纯编译时常量表达式**（字面量、其他 `constexpr` 变量、`constexpr` 函数返回值）
3. 类型须为**字面类型**（`int`、`double`、`char`、数组、`constexpr` 构造的类对象等）
4. 变量**自带 `const` 属性**，不能修改

### 合法 / 非法

```cpp
// ✅ 合法
constexpr int size = 10;
constexpr int total = size * 5;
int arr[size];              // 数组维度须编译常量

// ❌ 非法
int n = 5;
// constexpr int sz = n;   // n 是运行期变量

int getNum() { return 10; }
// constexpr int a = getNum();  // 非 constexpr 函数，不行
```

### const vs constexpr 作数组长度（必考）

```cpp
// 情况 1：const + 字面量 → 可当编译常量（C++ 允许）
const int c1 = 20;
int arr1[c1];               // ✅ 多数编译器合法

// 情况 2：const + 运行期值 → 不是编译常量
const int c2 = rand();
// int arr2[c2];            // ❌ c2 运行期才确定

// constexpr：无论如何必须编译期
constexpr int ce = 30;
int arr3[ce];               // ✅ 永远合法
```

```cpp
// 3.5 典型题：不推荐用运行期 const 当数组长度
int n = 10;
const int sz = n;
// int a[sz];               // 标准 C++ 非法；部分扩展可能放过
constexpr int arr_size = 12;
int arr[arr_size] = {1, 2, 3};  // ✅ 推荐
```

**总结**：数组长度、模板参数 → **直接用 `constexpr`**，别赌 `const` 初值是不是字面量。

---

## 二、constexpr 修饰函数

### 基础特性

1. 参数、返回值一般为**字面类型**
2. **两用**：
   - 传入 **`constexpr` 实参** → **编译期**计算
   - 传入普通变量 → 退化为**普通运行期函数**
3. 函数隐式 **`inline`**

### C++11：函数体极严

通常只能**一条 `return`**，不能 `if`、循环（复杂逻辑不行）：

```cpp
constexpr int square(int x) {
    return x * x;   // C++11 典型：单行 return
}
```

### C++14：大幅放宽

允许 `if`、`for`、局部变量、**递归**：

```cpp
constexpr int fact(int n) {
    if (n <= 1) return 1;
    return n * fact(n - 1);
}
constexpr int f = fact(5);   // 编译期算出 120
```

### 用于数组长度

```cpp
constexpr int calcLen(int a, int b) {
    return a + b;
}
constexpr int len = calcLen(4, 6);
int nums[len];               // ✅
```

---

## 三、constexpr 构造函数（字面类）

自定义类加 `constexpr` 构造，可在编译期创建对象：

```cpp
struct Point {
    int x, y;
    constexpr Point(int a, int b) : x(a), y(b) {}
};
constexpr Point p(2, 3);     // 编译期对象
```

---

## 四、const 与 constexpr 速查表

| 特性 | `const` | `constexpr` |
|------|---------|-------------|
| 核心语义 | 只读 | 编译期可计算 + 只读 |
| 初始化时机 | 编译/运行均可 | **强制编译期** |
| 能否作数组长度 | 仅当初值为字面量等编译常量时 | **一定可以**（若定义合法） |
| 修饰函数 | 成员函数（`this` 只读） | 普通/成员/构造函数 |
| `rand()` 等运行期初始化 | `const int a = rand();` ✅ | ❌ 完全禁止 |
| 隐含属性 | 无 | 变量自带 `const`；函数自带 `inline` |

---

## 五、拓展：C++17 `if constexpr`

编译期分支，不满足条件的分支**不参与编译**（无运行开销）。详见 [09-C++17 ch10 编译期 if](../../09-C++17-The-Complete-Guide/ch10-constexpr-if/)：

```cpp
template<int N>
void func() {
    if constexpr (N > 10) {
        // N<=10 时整段丢弃
    } else {
        // N>10 时丢弃
    }
}
```

---

## 六、使用规范（教材建议）

1. 需要**编译期常量**（数组大小、模板参数、`static_assert`）→ **`constexpr`**
2. 仅禁止修改、不需编译期计算 → 普通 **`const`**
3. 常量数学工具函数 → 写成 **`constexpr`**，编译/运行两用
4. 局部数组尺寸**不要用普通变量**；用 **`constexpr`** 保证标准 C++ 合法（见 [3.5](../../ch03-strings-vectors-arrays/3.5-arrays/3.5-数组.md)）

## 背诵极简版

```
constexpr = 编译期常量；const 只保证只读
数组长度、模板参数 → constexpr，别赌 const
constexpr 变量：定义即初始化，字面类型，自带 const
constexpr 函数：C++11 单行 return；C++14 可 if/递归
const 字面量初始化可当数组长度；const rand() 不行
```

## 示例

```cpp
#include <iostream>

struct Point {
    int x, y;
    constexpr Point(int a, int b) : x(a), y(b) {}
};

constexpr int calcLen(int a, int b) {
    return a + b;
}

int main() {
    constexpr int arr_size = 12;
    int arr[arr_size] = {1, 2, 3};

    constexpr int len = calcLen(4, 6);
    int nums[len]{};

    constexpr Point origin(0, 0);

    std::cout << arr_size << " " << len << "\n";
    return 0;
}
```
