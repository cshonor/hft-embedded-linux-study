# 2.4.1 const 基础与 constexpr

## 本节讲什么

顶层/底层 const、指针、constexpr。

## 一、普通 const 常量

```cpp
const int ci = 0;
// ci = 10;   // ❌ 顶层 const，绝对不能改
// const int ci;  // ❌ 必须初始化
```

- `ci` 是**顶层 const**：变量本身只读
- 仅有 `const` **不保证编译期常量**——若用运行时才知的值初始化，仍是运行时常量：

```cpp
int x = rand();
const int ci = x;   // ci 不可改，但值运行期才确定
```

## 二、常量引用 `const int &r`（底层 const）

1. **底层 const**：不能通过 `r` 修改绑定对象 → `r = 100;` ❌
2. **唯一能绑字面量的引用**：

```cpp
const int &r1 = 10;   // ✅
// int &r2 = 10;      // ❌ 普通引用不能绑临时/字面量
```

3. 只限制「通过 `r` 改」，不限制原变量：

```cpp
int a = 5;
const int &r = a;
a = 10;    // ✅ 直接改 a 可以
// r = 20; // ❌ 通过 r 不行
```

4. 引用没有「顶层 const」说法——带 `const` 的引用**全是底层 const**

## 三、const 修饰指针

### 易混纠正：「指向常量的指针」vs「常量指针」

名字很像，**锁定的对象不同**：

| 写法 | 俗称 | `*p` 能改值？ | `p` 能改指向？ |
|------|------|--------------|--------------|
| `const int *p` | **指向常量的指针** | ❌ 不能通过 `p` 改内容 | ✅ 可以指向别的地址 |
| `int *const p` | **常量指针** | ✅ 可以通过 `p` 改内容 | ❌ 指向不能改 |

```cpp
int a = 1, b = 2;

const int *p1 = &a;   // 指向常量的指针：内容只读，指针能挪
p1 = &b;              // ✅
// *p1 = 99;          // ❌

int *const p2 = &a;   // 常量指针：指向钉死，内容可改
*p2 = 99;             // ✅
// p2 = &b;           // ❌
```

**记法**：`const` **靠近类型**（`const int *`）→ 指向的内容不能改；`const` **靠近变量名**（`int *const p`）→ 指针本身不能改。

与 [3.2.4 string 与 C 风格字符串](../ch03-strings-vectors-arrays/3.2-library-type-string/3.2.4-string与C风格字符串.md) 里 C 风格 `char*` / `const char*` 读旧接口时也会用到。

### 1. 指向常量的指针：`const int *p` / `int const *p`（底层 const）

- 不能 `*p` 改所指对象；**`p` 本身可改指向**

```cpp
int num = 10;
const int *p = &num;
// *p = 20;   // ❌
p = &ci;      // ✅
num = 30;     // ✅ 原变量仍可改
```

### 2. 常量指针：`int *const p`（顶层 const）

- 地址固定，不能换指向；**可通过 `*p` 改值**

```cpp
int num = 10;
int *const p = &num;
// p = &ci;   // ❌
*p = 50;      // ✅
```

### 3. 双重 const：`const int *const p`

- 顶层 + 底层：`p` 不能改指向，`*p` 不能改值

### 记忆口诀（从右往左读）

- `int *const p` → `p` 是 **const 指针**（顶层），指向 `int`
- `const int *p` → `p` 是指针，指向 **const int**（底层）

## 四、顶层 const vs 底层 const

| | **顶层 const** | **底层 const** |
|---|----------------|----------------|
| **修饰谁** | 对象/指针**自身** | 指针/引用**所指/所绑**的对象 |
| **例子** | `const int a`、`int *const p` | `const int *p`、`const int &r` |
| **拷贝时** | 顶层 const **可丢弃** | 底层 const **必须匹配** |

```cpp
const int a = 10;
int b = a;           // ✅ 拷贝时丢弃顶层 const

const int *p1 = &x;
// int *p2 = p1;     // ❌ 不能丢失底层 const
```

**极简**：顶层 = **自己不能变**；底层 = **通过它不能改指向的东西**。

## 五、`constexpr`（编译期常量）

`constexpr` = **constant expression**，强制**编译期求值**，可作数组长度、模板实参。与 `const` 区别：`const` 只保证只读，值可能运行期才确定。

```cpp
constexpr int val = 2 + 3;
int arr[val];        // ✅

int x = rand();
const int c = x;     // const 只读，但 c 是运行期常量
// constexpr int err = x;  // ❌
```

**完整详解**（变量/函数/构造、C++11/14 区别、`if constexpr`、数组例题）：见 **[2.4.3 constexpr 详解](./2.4.3-constexpr详解.md)**。数组维度考点见 [3.5.1](../ch03-strings-vectors-arrays/3.5-arrays/3.5.1-定义与初始化.md)。

## 六、综合示例

```cpp
#include <iostream>

int main() {
    const int ci = 0;
    // ci = 10;

    const int &r = ci;
    // r = 20;

    int num = 100;

    const int *p1 = &num;       // 底层：*p1 不可改
    // *p1 = 200;
    p1 = &ci;

    int *const p2 = &num;       // 顶层：p2 指向不可改
    // p2 = &ci;
    *p2 = 200;

    const int *const p3 = &ci; // 双重
    // p3 = &num; *p3 = 5;

    constexpr int MAX = 1024;
    int buf[MAX];

    return 0;
}
```
