# 2.4.2 const 口诀与声明语法

## 本节讲什么

三类速记、int* vs int *p、HFT 习惯。

## 三类分开记 + 指针口诀（通俗版）

先把三种形态分开，再记指针上 **`const` 在 `*` 左还是右**：

| 形态 | 写法 | 含义 |
|------|------|------|
| **普通 const 变量** | `const int a` | `a` 的值不能改 |
| **常量引用** | `const int& ref` | 不能通过 `ref` 改；绑定固定 |
| **指针** | 见下表 | 看 `const` 相对 `*` 的位置 |

**指针口诀：**

- **`const` 在 `*` 左边** → 指向的**内容**不可改（底层）
- **`const` 在 `*` 右边** → **指针本身**不可改指向（顶层）

### 情况 1：`const int *p`（= `int const *p`）

```cpp
int x = 10, y = 20;
const int *p = &x;
// *p = 100;  // ❌ 不能改指向的值
p = &y;       // ✅ 指针可换指向
```

### 情况 2：`int *const p`

```cpp
int x = 10, y = 20;
int *const p = &x;
*p = 100;     // ✅ 可改值
// p = &y;    // ❌ 不能换指向
```

### 情况 3：`const int *const p`

指针不能换指向，内容也不能改。

### 常量引用 vs 指针（底层直觉）

```cpp
int a = 10;
const int &ref = a;
// ref = 20;  // ❌ 只读
```

概念上接近 **`const int *const`** 的语义：

1. 绑定对象不能换（像 `*` 后面的 const）
2. 不能通过引用改原值（像 `*` 前面的 const）

| | 绑定/指向固定 | 值可改 |
|---|---------------|--------|
| `int& ref` | ✅ | ✅ |
| `const int& ref` | ✅ | ❌（通过 ref） |
| `int *const p` | ✅ | ✅（通过 `*p`） |
| `const int *p` | ❌（p 可换） | ❌（通过 `*p`） |

### HFT 编码习惯

传参优先 **`const Order&`**：不拷贝，又防止函数内意外改订单——等价于「只读 + 绑定固定」的安全用法。

```cpp
void on_book(const OrderBook &book);  // 推荐
```

### 浓缩对比表（四类放一起）

| 写法 | 自身/指针可换指向 | 能通过它改值 | 顶层/底层 |
|------|-------------------|--------------|-----------|
| `const int a` | — | ❌ | 顶层 |
| `const int& ref` | ❌ 不能换绑 | ❌ | 底层（引用） |
| `const int* p` | ✅ | ❌ | 底层 |
| `int* const p` | ❌ | ✅ | 顶层 |
| `const int* const p` | ❌ | ❌ | 顶层+底层 |

## 声明语法：`int*` 还是 `int *p`？（const 混淆的根源）

### 两种理解模式

**模式 1：类型视角（现代写法，看着顺眼）**

```cpp
int* p;   // 把 int* 当成「整型指针」这一整个类型，p 是变量名
```

很多人习惯这么写，会误以为 `*` **依附于类型**。

**模式 2：C 原始声明规则（语法本源，分析 const 必须用这套）**

```c
int *p;
```

含义：**`*p` 是一个 `int`**；`p` 是指针变量——**`*` 修饰的是后面的变量 `p`**，不是 `int`。

一行定义多个名字时就能看懂为什么——**经典坑**：

```c
int *a, *b, c;     // ✅ a、b 是指针，c 是普通 int
char *p, *q;       // ✅ 两个都是 char 指针
```

若写成 `int* a, b;` 或 **`char* p, q;`**，容易误以为两个都是指针，实际上**只有第一个是指针**，第二个仍是普通 `int` / `char`：

```c
char* p, q;        // ❌ 经典迷惑：只有 p 是 char*，q 只是 char！
// 等价于：char *p; char q;

char *p, *q;       // ✅ 规范：星号紧挨变量名，一眼看出谁是指针
```

这就是 C/C++ 里著名的 **「语法糖迷惑点」**：`char*` 看起来像把「指向 char 的指针」当成一个**复合类型**，教材和代码里很常见，**约定俗成**；但语法上 **`*` 始终绑定变量名**，不是绑定 `char`。

| 写法 | 含义 | 风险 |
|------|------|------|
| `char* p` | 单变量时与 `char *p` 相同 | 一行多变量时易误判 |
| `char *p` | `*p` 是 char，`p` 是指针 | **语法本质**，const 分析必用 |
| `char* p, q` | 仅 `p` 是指针 | **高频踩坑** |
| `char *p, *q` | 两个都是指针 | 多变量声明推荐 |

**HFT / 严谨代码习惯**：坚持 **`char *p`**、**`int *p`**（星号靠名字），单声明时也更不易与 `const` 规则搞混；需要两个指针就写 `char *p, *q`，不要写 `char* p, q`。

> 与 [3.2.4 C 风格 `char*`](../ch03-strings-vectors-arrays/3.2-library-type-string/3.2.4-string与C风格字符串.md) 配合：字符串裸指针场景尤其容易在一行声明里写错。

### 套进 const：必须用原始规则

**`int *const p`**

- 基础：`int *p` → `*` 属于 `p`
- `const` 紧贴 `p` → 修饰**指针变量 `p`**
- 结果：不能换指向；`*p` 可改

**`const int *p`**

- `const` 修饰 **`int`** → `*p` 这个 int 是常量
- `p` 本身不受 const 约束 → 可换指向

若把 `int*` 当成整体类型，很难分清 `const int*` 和 `int *const`——**这就是之前混淆的根源**。

### 铁则：`const` 修饰紧靠它的那个

记住：**`const` 只绑定离它最近的标识符**；`*` 是指针标记，分析时跟着变量名走。

**1. `const int *p`**

拆开：`const int` + `*p`

- `const` 紧贴 **`int`** → 修饰 **`*p` 这个 int 值**
- `*p` 不可改；**`p` 本身可换**指向

**2. `int *const p`**

拆开：`int *` + `const p`

- `const` 紧贴 **`p`** → 修饰**指针变量本身**
- **`p` 不可换**指向；`*p` 可改

**等价写法：** `const int *p` ≡ `int const *p`（都是修饰 int，与 `p` 无关）

### 通俗类比：箭头

| 写法 | 箭头（指针） | 箭头戳到的东西（`*p`） |
|------|--------------|------------------------|
| `const int *p` | 可以到处跑 | **不能改** |
| `int *const p` | **钉死在原地** | 可以改 |
| `const int *const p` | 不能动 | 不能改 |

**为什么直觉会反？** 直觉以为「`const` 在后面就约束值」，但语法是 **`const` 绑最近的**——`int *const p` 里 `const` 挨着 `p`，只限指针，管不到 `int`；要限值，必须把 `const` 写在 `int` 前。

### 终极口诀（左/右）

1. **`const` 在 `*` 左边** → 限制**指向的值**
2. **`const` 在 `*` 右边** → 限制**指针本身**
3. 两边都有 → 值不能改、指针也不能改

### 对比常量引用

`const int& ref` 的语义 ≈ **`const int *const`**（概念类比）：

- 绑定不能换（像 `*` 后面的 const）
- 值只读（像 `*` 前面的 const）

普通 `int&` ≈ `int *const`：绑定固定，值可改。

### 再巩固一遍

| 写法 | const 修饰谁 | 结果 |
|------|--------------|------|
| `const int *p` / `int const *p` | `int`（`*p` 的值） | `p` 可换，值不可改 |
| `int *const p` | 指针变量 `p` | `p` 不可换，值可改 |

**一句话：**

- **分析 const / 多变量声明 / HFT 严谨代码**：坚持 **`char *p`**、**`int *p`**（`*` 紧挨变量名）
- **单变量、图省事**：很多人仍写 `char* p`、`int* p`（当复合类型看），能读但**别在一行里写 `char* p, q`**

## 一句话速记

1. `const int ci` → 自身不可改（顶层）
2. `const int&` → 不能通过引用改；可绑字面量（底层）
3. `const int*` → 内容不可改、指针可变（底层）；`int* const` → 地址不可变（顶层）
4. 顶层 = 自己只读；底层 = 指向对象只读
5. `constexpr` = 强制编译期算出

## 示例

```cpp
#include <iostream>
int main() {
    const int ci = 42;
    const int &cr = ci;

    const int *p = &ci;
    int j = 0;
    int *const cp = &j;
    *cp = 100;

    constexpr int size = 64;
    char buf[size];

    std::cout << ci << " " << *cp << std::endl;
    return 0;
}
```
