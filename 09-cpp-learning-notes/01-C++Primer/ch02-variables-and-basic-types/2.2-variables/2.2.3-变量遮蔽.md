# 2.2.3 变量遮蔽

## 本节讲什么

嵌套作用域、shadowing、:: 穿透。

## 变量遮蔽（Variable Shadowing）

**变量遮蔽**：在**嵌套作用域**（代码块、函数、循环）里，内层重新声明与外层**同名**的变量，内层名字会**屏蔽**外层同名变量；在内层范围内，所有对那个名字的引用只指向内层变量，外层同名变量暂时不能直接访问。

**词法作用域查找链（由内向外）：**

1. 先在**当前最内层**找名字，找到就用
2. 找不到再向外层回溯
3. 内层一旦定义同名变量，查找在此终止，**不会**再匹配外层

| 情况 | 结果 |
|------|------|
| **同一 `{}` 内两次 `int a;`** | ❌ 语法错误（重定义） |
| **嵌套 `{}` 内外各一个 `int a;`** | ✅ 合法，内层**遮蔽**外层，两块独立内存 |

### 块级嵌套示例

```cpp
#include <iostream>
int main() {
    int i = 0;
    std::cout << "外层 i = " << i << std::endl;  // 0

    {
        int i = 7;   // 内层 i 遮蔽外层 i
        int j = 9;
        std::cout << "内层 i = " << i << ", j = " << j << std::endl;  // 7, 9
    }   // 内层 i 销毁，遮蔽结束

    std::cout << "外层 i 恢复 = " << i << std::endl;  // 0
    return 0;
}
```

### 四种常见场景

**1. 局部变量遮蔽全局变量**

```cpp
int x = 100;

void func() {
    int x = 50;              // 局部遮蔽全局
    std::cout << x;          // 50
}

int main() {
    func();
    std::cout << x;          // 100
}
```

**2. 内层块遮蔽形参**（同一函数体直接 `int a` 与形参同名会**重定义报错**，须包一层 `{}`）

```cpp
void test(int a) {
    std::cout << a;          // 5（形参）
    {
        int a = 20;        // 内层块遮蔽形参
        std::cout << a;    // 20
    }
    std::cout << a;          // 5（形参恢复可见）
}
```

**3. 循环体内遮蔽循环变量**（C++11 起 `for (int i ...)` 的 `i` 仅在循环作用域内）

```cpp
for (int i = 1; i < 3; ++i) {
    int i = 99;              // 循环体内局部 i 遮蔽循环头 i
    std::cout << i << " ";   // 99 99
}
```

**4. 多层嵌套连续遮蔽**

```cpp
int a = 1;
{
    int a = 2;
    {
        int a = 3;
        std::cout << a;    // 3
    }
    std::cout << a;        // 2
}
std::cout << a;            // 1
```

### C++ 特有：访问被遮蔽的全局变量

用**全局作用域解析符 `::`** 穿透遮蔽：

```cpp
int x = 10;
int main() {
    int x = 20;
    std::cout << x << std::endl;   // 20（内层）
    std::cout << ::x << std::endl; // 10（全局）
}
```

### 工程上的问题与规范

**问题：**

1. 改内层变量，外层同名变量**完全不变**，容易误以为改到了外层
2. 可读性差，难一眼看出用的是哪一层
3. 内层变量出 `{}` 即销毁，生命周期易混淆

**建议：**

1. 避免嵌套作用域用同名变量（如全局 `g_x`、局部 `local_x`）
2. 不要与形参同名；若必须区分，用不同名字而非遮蔽
3. 循环体内不要重复定义与控制变量同名的变量

**一句话**：查找**由内向外**；内层一旦声明同名标识符就**遮蔽**外层，内层范围内只能访问内层那份。
