# 2.2.4 extern 与头文件分工

## 本节讲什么

extern 声明/定义、#include 预处理文本替换、标准库声明与链接。

## `extern` 关键字完整解析

支持**分离式编译**：多个 `.cpp` 分别编译，最后链接成一个程序。`extern` 用来声明「这个名字的定义在别处」。

### 通俗理解：external（外部的）

`extern` = **声明，不是定义**。告诉编译器：

> 这个函数 / 全局变量的**本体不在当前 `.cpp`**，在别的源文件里；编译本文件时先「放行」，**链接阶段**再去别的 `.cpp` 里把它找出来拼上。

| | 定义 | `extern` 声明 |
|---|------|----------------|
| 内存 | ✅ 分配 | ❌ 不分配 |
| 次数 | 全项目**一次** | 可多次 |
| 例子 | `int g_num = 10;` | `extern int g_num;` |

**文件 A.cpp / B.cpp 例子：**

```cpp
// A.cpp
int g_num = 100;   // 定义：内存在这里

// B.cpp
extern int g_num;  // 声明：g_num 在别的文件
#include <iostream>
int main() {
    std::cout << g_num;  // 编译时只知道「有个 int g_num」
    return 0;            // 链接时找到 A.cpp 里的实体，拼在一起
}
```

**函数可以省略 `extern`**：C++ 里函数声明**默认隐式带 `extern`**，所以跨文件调函数一般只写 `void func();` 即可。

**全局变量跨文件必须用 `extern` 声明**（或在头文件里 `extern` 声明后 `#include`），否则编译器不知道去别处找。

### 1. 声明 vs 定义

| | **声明（declaration）** | **定义（definition）** |
|---|-------------------------|------------------------|
| **做什么** | 告诉编译器类型和名字 | 分配内存（变量）或提供函数体 |
| **出现次数** | 可多次 | 整个程序**只能一处** |
| **不写的后果** | — | 多处定义 → 链接报 `multiple definition` |

### 2. `extern` 修饰变量：两种写法

**`extern int i;` —— 纯声明（引用性声明）**

- **不分配内存**
- 表示：`i` 的定义在**其他 `.cpp`** 或本文件更后面
- 可在头文件、多个源文件中重复写，无冲突

```cpp
// a.cpp
int i = 100;        // 唯一定义，分配内存

// b.cpp
extern int i;       // 仅声明，借用 a.cpp 的 i
#include <iostream>
void f() { std::cout << i; }
```

**`extern int i = 0;` —— 带初始化 = 定义（抵消 extern）**

C/C++ 规则：**带显式初始化的 `extern` 一律视为定义**，等价于 `int i = 0;`：

- 会分配全局内存
- 整个项目只能写**一次**
- 违背「引用外部变量」的本意，工程中**极少这么写**

### 3. 易混对比

```cpp
int i;               // 定义（C++ 中尽量避免；全局默认零初始化）
extern int i;        // 仅声明，无内存
int i = 0;           // 定义
extern int i = 0;    // 定义，等价上一行
```

**一句话**：`extern int i;` 只有声明含义；只要 `extern` 变量附带 `= 初值`，就是定义。

### 4. `extern` 修饰函数

函数更简单——**声明默认就是 extern，可省略**：

```cpp
extern void func();  // 声明，无函数体
void func();         // 完全等价
```

**有 `{}` 函数体 = 定义**，与是否写 `extern` 无关：

```cpp
void func() { }              // 定义
extern void func() { }       // 同样是定义，写 extern 无意义
```

### 5. 工程规范（头文件 + 单源文件定义）

**头文件只写 extern 声明：**

```cpp
// global.h
extern int g_val;    // 仅声明，多文件 #include 无冲突
```

**只有一个 `.cpp` 写定义：**

```cpp
// global.cpp
int g_val = 10;      // 唯一定义，分配内存
```

**禁止在头文件写：**

```cpp
extern int g_val = 10;   // ❌ 定义！头文件被多个 .cpp 包含 → 重复定义
int g_val = 10;          // ❌ 同上
```

HFT 多文件项目里：全局配置、共享计数器等用「`.h` 声明 + 一个 `.cpp` 定义」是标准做法。

### 6. 什么时候会「重定义」？怎么规避？

| 错误写法 | 原因 |
|----------|------|
| 两个 `.cpp` 都写 `int g_num = 10;` | 两处定义 → 链接 `multiple definition` |
| 头文件写 `int g_num = 10;` 并被多个 `.cpp` 包含 | 每个 `.cpp` 都生成一份定义 |
| 头文件写 `extern int g_num = 10;` | 带 `=` 仍是定义，同上 |

**规避**：`.h` 里只写 `extern int g_num;`，**恰好一个** `.cpp` 写 `int g_num = 10;`。

### 7. 补充：`extern "C"`（与变量 `extern` 不同）

`extern "C"` 用来**关闭 C++ 名字修饰（name mangling）**，让 C++ 能调用 C 写的函数。DPDK、Linux 内核接口里很常见：

```cpp
extern "C" {
    void dpdk_init();   // 按 C 规则链接，符号名不被 C++ 改写
}
```

这是**链接方式**的声明，和「全局变量 `extern int g_num`」不是同一类语法，后面第 18 章 / 与 C 混编时会再遇到。

## `#include` 的本质：预编译「文本替换」

`#include` 是**预处理指令**，是编译器在**正式编译之前做的第一步**。可以把它理解成：

> **把指定文件的内容，原地复制粘贴到当前代码里**——粘贴位置就是 `#include` 那一行。

比如你写 `#include <string>`，标准库里 `string` 类的声明、成员函数接口等文本会**原封不动插在这行位置**，编译器才知道 `std::string` 是什么、有哪些方法可以调用。

**包含的不是 `.cpp` 实现文件，而是头文件**（`.h` 或 C++ 标准库的 `<xxx>` 无后缀头）。头文件只放「**告诉别人这个东西怎么用**」的**声明**；真正的实现代码在编译器自带的**预编译库**（`.lib` / `.a`）里，**不用你自己编译标准库源码**。

```
编译前预处理（第一步）：
  main.cpp                    展开后（示意）
  ─────────                   ─────────────
  #include <string>    →      [ string 头里全部声明文本 ]
  #include <cctype>    →      [ cctype 头里 isalpha 等声明 ]
  int main() { ... }          int main() { ... }
```

### 标准库例子

```cpp
#include <string>    // 粘贴 string 类声明、operator+ 原型等
#include <cctype>    // 粘贴 isalpha、isdigit、toupper 等函数声明

int main() {
    std::string s = "IF2406";
    for (unsigned char c : s)
        if (!std::isalpha(c)) return 1;   // include 后直接调，不用手写 ASCII 判断
    return 0;
}
```

- 编译器看到 `#include <string>` 后，才知道 **`std::string`**、`size()` 等符号长什么样  
- `#include <cctype>` 之后，才能直接调 **`std::isalpha`**、**`isdigit`** 等（见 [3.2.5 cctype](../ch03-strings-vectors-arrays/3.2-library-type-string/3.2.5-cctype字符校验.md)）

### 声明 vs 实现：为什么要分离？

| 放在头文件（`#include` 进来） | 藏在预编译库里 |
|------------------------------|----------------|
| 类 / 函数**声明**、模板声明 | **函数体**、已优化的机器码 |
| 告诉编译器「能调什么、长什么样」 | 链接阶段才接上，不参与你每次 `#include` 的文本膨胀 |

这种**声明与实现分离**有两层好处：

1. **编译快**：头文件被多次 `#include` 时，不会把庞大实现重复粘贴进每个 `.cpp`  
2. **接口稳定、实现可换**：标准库把核心实现藏起来，只暴露安全的调用接口——HFT 这类依赖稳定、高效底层组件的场景，直接复用标准库即可，不必重复造轮子、也不必碰底层细节

| 阶段 | 谁做什么 |
|------|----------|
| **预处理** | `#include` 复制头文件文本 |
| **编译** | 每个 `.cpp` 单独编译成 `.o`，只验证声明够用 |
| **链接** | 把 `.o` 与标准库 / 你的 `Sales_data.cpp` 等**实现**绑在一起 |

## `#include` 与 `extern`：分工与配合

两者**原理完全不同**，工程里却经常**搭档使用**：

| | **`#include`** | **`extern`** |
|---|----------------|--------------|
| **做什么** | **文本复制**：把头文件内容粘贴进当前 `.cpp` | **一条声明**：变量本体在别的 `.cpp`，不复制实现 |
| **头文件里常放** | 函数声明、`extern` 变量声明、结构体、宏 | 全局变量的 `extern` 声明本身 |

**一句话**：`#include` 负责把 `extern` 声明**导入**本文件；`extern` 负责告诉编译器变量**实体在别的 `.cpp`**。

### 标准组合写法（三文件套路）

**`global.h`**（被 `#include`）

```cpp
extern int g_val;   // 声明全局变量
void test();        // 函数声明（默认 extern，可省略 extern）
```

**`global.cpp`**（唯一定义）

```cpp
int g_val = 100;    // 真正分配内存
void test() { }
```

**`main.cpp`**

```cpp
#include "global.h"   // 把声明拿过来，不用手写 extern

int main() {
    g_val = 200;
    test();
    return 0;
}
```

### 直白区别

1. **函数**：声明写进 `.h`，`#include` 引入即可，一般**不用**写 `extern`（函数声明默认 extern）。
2. **全局变量**：头文件里写 **`extern int g_val;`**，再 `#include` 给其他 `.cpp` 共用。
   - 也可以不在头文件，而在 `main.cpp` 手写 `extern int g_val;`，但项目一大就乱，工程上**统一放头文件**。

### 反面例子（必踩坑）

```cpp
// bad.h  ❌ 千万不要这样
int g_val = 100;    // 这是定义！不是声明

// a.cpp
#include "bad.h"

// b.cpp
#include "bad.h"    // 两个 .cpp 各有一份 g_val 定义 → 链接重定义报错
```

**正确**：头文件只写 `extern int g_val;`，**一个** `global.cpp` 写 `int g_val = 100;`。

## 示例

```cpp
#include <iostream>
#include <limits>

int global_val = 10;

extern int shared_val;  // 声明：在其他 .cpp 中定义

void demo_static() {
    static int call_count = 0;
    ++call_count;
    std::cout << "called " << call_count << " times\n";
}

int main() {
    int ival = 0;
    const int limit = 100;
    constexpr int kMax = 10000;

    demo_static();
    demo_static();  // call_count 累加，不会重置

    std::cout << "int max=" << std::numeric_limits<int>::max() << std::endl;
    std::cout << ival << " " << limit << " " << kMax << std::endl;
    return 0;
}
```
