# 2.2.1 标识符与未初始化

## 本节讲什么

命名规则、局部内置变量 UB。

## 标识符命名规则

**标识符** = 变量、函数、类型等的名字。编译器**强制校验**，违反直接报错。

### 硬性规则（标准规定）

1. **允许字符**：仅 **a–z、A–Z、0–9、下划线 `_`**
2. **首字符**：只能是**字母或 `_`**，**不能以数字开头**
3. **大小写敏感**：`num`、`Num`、`NUM` 是三个不同名字
4. **附加约束**：
   - 不能与**关键字**重名（`int`、`if`、`for`、`return` 等）
   - 内部不能有空格、Tab、换行
   - **`__` 开头、`_` + 大写字母** 等属于保留名，业务代码避免使用

### 合法 / 非法示例

```cpp
// ✅ 合法
age  Age  _age  age123  student_score  MAX_NUM

// ❌ 非法
// 123age         数字开头
// student-score  含减号
// student score  含空格
// int            关键字
// $num           非法字符
```

### 大小写独立

```cpp
int a = 10;
int A = 20;
std::cout << a << " " << A;   // 10 20，完全独立
```

### 工程命名习惯（非语法强制，提升可读性）

| 对象 | 常见风格 | 例子 |
|------|----------|------|
| 局部变量、普通函数 | 小写 + 下划线 | `user_id`、`submit_order` |
| 宏 / 宏常量 | 全大写 | `MAX_LEN` |
| C++ 类 / 结构体 | 大驼峰 | `StudentInfo`、`SalesItem` |
| 循环变量 | 单字母可接受 | `i`、`j`、`k` |

**一句话**：只能由**字母、数字、下划线**组成，**首字符不能是数字**，**区分大小写**，不能用关键字。

## 未初始化局部内置变量（UB）

C/C++ 标准（C11/C17 §6.7.9 等）对**自动存储期**（普通局部变量）与**静态存储期**（全局 / `static` 局部）规则不同：

| 存储期 | 例子 | 未显式初始化时 | 读取 |
|--------|------|----------------|------|
| **自动（auto）** | 函数内 `int a;` | **不确定值**（栈上垃圾） | ❌ **未定义行为（UB）** |
| **静态** | 全局 `int a;`、`static int b;` | **零初始化**（0 / 空指针） | ✅ 安全 |

### 核心概念

- **不确定值（indeterminate value）**：内存里是栈残留数据，每次运行、每次调用可能不同
- **未定义行为（UB）**：标准不约束结果——可能随机输出、逻辑错乱、崩溃，或被编译器优化掉相关代码

### 内存原理（直观）

- **栈上局部变量**：只分配空间，**不清零** → 可能是上次函数留下的垃圾
- **全局 / static**：在 **.bss** 等区域，加载时**整块清零** → 默认为 0

### 代码对比

```cpp
// ❌ UB：局部内置类型未初始化就读取
int main() {
    int a;
    std::cout << a;   // 未定义行为，可能随机数，高优化下 printf 可能被删
}

// ✅ 全局：自动零初始化
int g_a;
int main() {
    std::cout << g_a;  // 稳定输出 0
}

// ✅ static 局部：零初始化
void test() {
    static int b;
    std::cout << b;    // 0
}

// C++ 安全写法
int main() {
    int x;           // 栈局部，未初始化，读 x → UB
    int y{};         // 列表初始化，置 0，安全
    int *p = new int;     // 堆，未初始化，读 *p → UB
    int *q = new int{};   // 堆，零初始化
    delete p; delete q;
}
```

### UB 的危险（不止「随机数字」）

1. 偶现 bug，极难复现
2. 编译器优化：对未初始化值的判断可能被当作永远 false/true
3. 未初始化**指针** → 野指针，段错误
4. 栈残留敏感数据 → 潜在信息泄露（安全场景）

### 编译器检测

- GCC/Clang：`-Wall -Wextra` 报 `uninitialized`；`-Werror` 当错误
- MSVC：C4700；`/SDL` 可更严格
- 静态分析：Clang Static Analyzer、Cppcheck

### 工程规范

```cpp
int a = 0;
float f = 0.0f;
int *p = nullptr;

int a{};      // C++ 统一列表初始化，内置类型置零
double d{};
```

1. 局部内置变量**定义即初始化**
2. C++ 优先 `int x{};` / `int x{0};`
3. `malloc` 后不能直读，需 `calloc` 或手动清零
4. 开警告，别让未初始化变量混进发布版

**一句话**：函数内普通局部内置变量不初始化 → 值不确定；**读取 = UB**。全局、`static` 局部会自动零初始化，读取安全。
