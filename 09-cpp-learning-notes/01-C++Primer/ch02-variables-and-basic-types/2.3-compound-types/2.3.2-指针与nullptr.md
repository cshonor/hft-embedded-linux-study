# 2.3.2 指针与 nullptr

## 本节讲什么

const 指针、nullptr、Rust 对照。

## 指针：`int*` 与 `const`（和引用放一起记）

| 写法 | 读法 | 谁能改 |
|------|------|--------|
| `int*` / `int *p` | 普通指针 | 指针可变指向；可通过 `*p` 改对象 |
| `const int*` / `int const *p` | 指向常量的指针 | **不能**通过 `*p` 改对象；`p` 可改指向 |
| `int* const p` | 常量指针 | 可通过 `*p` 改对象；**不能**改 `p` 的指向 |
| `const int* const p` | 都 const | 指向和对象都不改 |

```cpp
int i = 0, j = 1;
const int *p = &i;   // *p 不可改，p 可指向 j
p = &j;              // ✅
// *p = 2;           // ❌

int *const cp = &i;  // cp 不可改指向，*cp 可改
*cp = 2;             // ✅
// cp = &j;          // ❌
```

**和引用的对应：**

| 引用 | 近似指针含义 |
|------|--------------|
| `int&` | 像绑死的 `int*`，必须有效，可读写 `*p` |
| `const int&` | 像 `const int*`，只读，不能 `*p = ...` |

引用不能重新绑定 ≈ 没有「改指向」这回事；指针可以 `p = &other`。

## `nullptr`：为什么优于 `NULL` / `0`（C++11）

### 三者分别是什么

| | 本质 | 类型 |
|---|------|------|
| **`0`** | 整数零 | `int` |
| **`NULL`** | 宏，通常 `#define NULL 0`（有的编译器 `(void*)0`，不一致） | 仍是**整数语境** |
| **`nullptr`** | **关键字**，专用空指针字面值 | **`std::nullptr_t`** |

```cpp
void func(int);
void func(char*);

func(0);        // 调用 func(int) —— 不是指针！
func(NULL);     // 同上，仍是 int 语境
func(nullptr);  // 调用 func(char*) —— 正确
```

### 核心优势

**① 重载不踩坑**

```cpp
void foo(int)  { std::cout << "int\n"; }
void foo(char*) { std::cout << "pointer\n"; }

foo(0);        // int
foo(NULL);     // int
foo(nullptr);  // pointer ✅
```

**② 模板推导正确** — `0`/`NULL` 推成 `int`；`nullptr` 推成 `std::nullptr_t`，智能指针等接口只认后者。

**③ 类型安全**

```cpp
int a = nullptr;  // ❌ 编译错误
int b = NULL;     // ⚠️ 往往能通过，隐藏 bug
```

**④ 兼容所有指针类型**

```cpp
int *p = nullptr;
void (*fn)() = nullptr;
```

`std::nullptr_t` 可隐式转为任意指针类型，**不能**隐式转为整型——安全根源在此。

### 什么时候用哪个

| 写法 | 用途 |
|------|------|
| **`nullptr`** | **所有**表示空指针的场景（现代 C++ 唯一推荐） |
| **`NULL`** | 老代码遗留，新项目别写 |
| **`0`** | 只表示数字零，**绝不**表示空指针 |

**极简总结**：`0` / `NULL` 是整数，重载和模板易错；`nullptr` 是专门的空指针类型，无歧义、类型安全。

## 与 Rust 对照（横向一览）

| | C++ | Rust |
|---|-----|------|
| **只读间接访问** | `const T&` | `&T`（借用检查） |
| **可变间接访问** | `T&` | `&mut T`（借用检查） |
| **裸指针** | `T*`、`const T*` | `*const T`、`*mut T`（多在 `unsafe`） |
| **编译期安全** | ❌ 靠程序员 + `const` | ✅ 借用规则强制 |

- C++：`T&` / `const T&` ≈ **安全版**用法；`T*` ≈ **自由、风险高**
- Rust：`&T` / `&mut T` ≈ C++ 引用 **再叠加** 编译期借用检查；裸指针脱离规则，在 `unsafe` 里用

**关键区别：**

- Rust：**编译期借用检查**，`&T` 与 `&mut T` 不能同时活跃（同一数据）
- C++：**没有**强制借用检查；`T&` 和 `const T&` 可同时存在——多线程下可能**数据竞争**，HFT 要自己管同步

简单记：**C++ 有引用和裸指针两层；Rust 把引用管得更严，裸指针放进 unsafe。**

## 示例

```cpp
#include <iostream>
#include <string>

void read_only(const std::string &s) {
    std::cout << s << std::endl;
}

void bump(int &n) {
    ++n;
}

int main() {
    int ival = 42;

    int &refVal = ival;
    refVal = 1024;
    std::cout << ival << std::endl;  // 1024

    const int &cref = ival;
    std::cout << cref << std::endl;

    int *p = &ival;
    *p = 0;
    std::cout << ival << std::endl;  // 0

    const int *pc = &ival;           // 指向常量：不能 *pc 改值
    int *const cp = &ival;           // 常量指针：cp 不能改指向

    read_only("hello");
    bump(ival);

    return 0;
}
```
