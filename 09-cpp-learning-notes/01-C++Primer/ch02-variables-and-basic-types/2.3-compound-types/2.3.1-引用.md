# 2.3.1 引用

## 本节讲什么

为什么需要引用、T& 与 const T&。

## 为什么 C++ 还要有引用？

已有指针，再引入引用，是为了解决指针在**业务代码**里的常见痛点。

### 指针的缺点

1. **可为空**（`nullptr`）→ 必须判空，漏了可能崩溃（HFT 里直接宕机）
2. **可随意改指向** → 逻辑容易乱
3. **语法繁琐** → `*p` 解引用、`*`/`&` 混用
4. **可做算术**（`p++`）→ 容易越界

```cpp
int a = 1, b = 2;
int *p = &a;
p = &b;   // 随时换指向
```

### 引用：受约束的「安全版指针」

编译器实现上，引用常类似 **`T* const`**（指向 `T` 的**常量指针**——地址绑死不能改）：

| 特性 | 指针 `int*` | 引用 `int&` |
|------|-------------|-------------|
| 可为空 | ✅ `nullptr` | ❌ 必须绑定对象 |
| 改指向 | ✅ `p = &other` | ❌ 绑定后不能换 |
| 使用语法 | `*p` | 直接当变量用 |
| 指针运算 | ✅ | ❌ |

```cpp
int a = 10;
int &ref = a;
ref = 20;              // 直接改 a，不用 *

// 底层直觉（概念模型，不是你要写的代码）：
// int *const p = &a;  // 地址固定，通过 *p 访问
```

**一句话**：引用 = **受限制、更安全、语法更简洁**的间接访问；日常业务优先引用，底层内存/DPDK 才用裸指针。

### 取舍标准（HFT 编码习惯）

**优先用引用：**

- 函数传参、避免拷贝：`void f(const OrderBook &book);`
- 运算符重载（语法要求）：`operator+`、`operator<<` 等

**必须用指针：**

- 后期要**改变指向**
- 对象**可能不存在**（可选）
- 数组、内核、DPDK 等需要**地址偏移**

## 引用：`T&` 与 `const T&`

### 1. 可变引用（可修改原对象）

```cpp
int a = 10;
int &ref = a;
ref = 20;        // ✅ 通过 ref 修改 a
// a 现在是 20
```

- `int&`：绑定原变量，**可读可写**

### 2. 常量引用（只读）

```cpp
int a = 10;
const int &ref = a;
// ref = 20;     // ❌ 编译错误，不能通过 ref 修改
std::cout << ref; // ✅ 只读
```

- `const int&`：**只能读**，不能通过引用改原对象
- 可绑定到字面量：`const int &r = 42;`（普通 `int&` 不能绑字面量）

### 3. 引用的硬性规则（C++ 独有）

1. **必须初始化**，不能空着
2. **绑定后不能改绑**到其他对象（Rust 的引用可以重新绑定）
3. 不存在「空引用」

### 4. 工程习惯（HFT / 日常 C++）

```cpp
// 传参优先：不拷贝 + 只读
void process(const std::string &s);

// 需要改入参时才用可变引用
void adjust(int &value);
```

| 场景 | 用法 |
|------|------|
| 只读、避免拷贝 | `const T&` |
| 需要修改原对象 | `T&` |
| 可选/可为空 | 用指针或 `std::optional`（指针章节） |
