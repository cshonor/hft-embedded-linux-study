# 小结

## 本节讲什么

本章是 **C++ 标准库学习的起点**。回顾 `string`、`vector`、迭代器与内置数组的关系，并强调：开发者应**优先选用**标准库提供的 `string` 和 `vector`，只在必要时才考虑底层的数组或指针。

## 要点

- **`using` 声明**：简化 `std::` 前缀，头文件中慎用
- **`string`**：可变长字符串，配合 `cctype` 处理字符
- **`vector`**：标准库**动态数组 / 容器**（向量），同类型元素、`push_back` 增容；下标只访问已有元素，**不能越界添加**（HFT 热路径 OOB 写可能静默错价或崩溃）
- **迭代器**：全容器通用遍历；`begin`/`end`（尾后勿解引用）、`cbegin`/`cend`（只读）；`it != end()`；`vector` 扩容后迭代器失效（见 [3.4](./3.4-introducing-iterators/3.4-迭代器介绍.md)）
- **内置数组**：编译期固定、不能拷贝；数组名退化指针；优先 `vector`/`string`（见 [3.5](./3.5-arrays/3.5-数组.md)）
- **多维数组**：本质是数组的数组；可用范围 for 与类型别名简化

## 示例

```cpp
#include <iostream>
#include <string>
#include <vector>

int main() {
    std::vector<std::string> words{"C++", "Primer", "Chapter3"};

    for (const auto &w : words)
        std::cout << w << " ";
    std::cout << std::endl;

    // 标准库类型组合：比裸数组 + 指针更安全、更易用
    return 0;
}
```
