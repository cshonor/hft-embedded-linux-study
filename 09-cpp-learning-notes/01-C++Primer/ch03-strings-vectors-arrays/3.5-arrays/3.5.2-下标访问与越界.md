# 3.5.2 下标访问与越界

## 本节讲什么

数组下标规则、`arr[i]` 与指针运算的关系，以及越界为何是隐蔽 UB。

## 下标规则

1. 下标从 **`0`** 开始：`arr[0]` 首元素，`arr[N-1]` 末元素（`N` 为数组长度）
2. **`arr[i]` 等价 `*(arr + i)`**——下标访问本质是**指针算术**（见 [3.5.3](./3.5.3-数组退化与指针.md)）
3. **无数组边界检查**：下标超出 `0 … N-1` → **未定义行为**，可能静默踩内存或崩溃
4. 对比 `vector`：**`at(i)`** 会越界抛异常；**`operator[]`** 与数组一样**不检查**（见 [3.3.3](../3.3-library-type-vector/3.3.3-下标与增删.md)）

```cpp
int arr[3] = {10, 20, 30};
arr[0] = 1;           // ✅
// arr[3] = 99;       // ❌ UB，无编译/运行报错
```

## HFT 注意

固定 `char buf[N]` 写合约码、订单号时，**必须**保证写入长度 `< N` 且保留 `'\0'`；越界写与 `vector` 一样可能**静默错价**（见 [3.2.4 固定缓冲](../3.2-library-type-string/3.2.4-string与C风格字符串.md)）。
