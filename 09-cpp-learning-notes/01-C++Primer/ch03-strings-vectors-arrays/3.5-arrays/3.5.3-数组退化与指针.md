# 3.5.3 数组退化与指针

## 本节讲什么

**数组名自动退化为指针**的核心规则、三种不退化例外，以及 `&arr` 与 `&arr[0]` 的类型区别。

## 核心规则：数组退化（Decay）

**绝大多数表达式中**，数组名自动转换为**指向首元素的指针**：

```cpp
int arr[5] = {1, 2, 3, 4, 5};
int *p = arr;        // 等价 int *p = &arr[0];

*arr == arr[0];      // true
arr + 1 == &arr[1];  // true
```

因此数组**传参进函数**时常退化为 `int*`，**丢失长度信息**——这是「必须另传 `size`」或改用 `vector` 的根源。

## 不会退化的 3 种例外（必考）

| 场景 | 行为 |
|------|------|
| **`sizeof(arr)`** | 整个数组的字节大小，不是指针大小 |
| **`&arr`** | 取**整个数组**的地址，类型 `int (*)[5]` |
| **用字符串字面量初始化 `char` 数组** | 按数组初始化规则，不退化为指针 |

```cpp
int arr[5] = {1, 2, 3, 4, 5};
int *p = arr;

sizeof(arr);    // 5 * sizeof(int)，如 20（int 为 4 字节）
sizeof(p);      // 指针大小，如 8（64 位）

&arr;           // int (*)[5]——指向「含 5 个 int 的数组」
&arr[0];        // int*——指向单个 int

// &arr 与 &arr[0] 地址**数值**常相同，但**类型不同**，指针算术步长不同：
// &arr + 1 跳过整个数组；&arr[0] + 1 跳过 1 个 int
```

## 与 [3.4 迭代器](../3.4-introducing-iterators/3.4-迭代器介绍.md) 的关系

退化后，`arr` 可当指针遍历；C++11 用 **`std::begin`/`std::end`** 更安全统一（见 [3.5.4](./3.5.4-begin-end与vector对比.md)）。
