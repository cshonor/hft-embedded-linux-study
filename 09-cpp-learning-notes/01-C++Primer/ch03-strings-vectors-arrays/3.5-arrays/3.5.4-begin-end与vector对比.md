# 3.5.4 begin/end 与 vector 对比

## 本节讲什么

C++11 对原生数组的 `begin`/`end`，以及内置数组 vs `std::vector` 的全面对比与现代 C++ 建议。

## std::begin / std::end（C++11）

原生数组**没有成员函数** `begin()`；标准库提供**全局模板**：

```cpp
#include <iterator>

int arr[] = {1, 2, 3};
auto first = std::begin(arr);   // &arr[0]
auto last  = std::end(arr);     // 尾后指针，不可解引用

for (auto it = std::begin(arr); it != std::end(arr); ++it)
    std::cout << *it << " ";
```

与 [3.4.2 `v.begin()`](../3.4-introducing-iterators/3.4.2-begin-end与常量迭代器.md) 语义一致，便于同一套循环写法对接第 10 章算法。

## 内置数组 vs std::vector

| 特性 | 内置数组 | `vector` |
|------|----------|----------|
| **大小** | 编译期固定，不可扩容 | 运行时动态增减 |
| **内存** | 局部多在**栈**（过大易栈溢出） | 元素多在**堆**，较安全 |
| **拷贝/赋值** | ❌ 不支持整体拷贝赋值 | ✅ 支持 `=`、拷贝构造 |
| **长度** | `sizeof(arr)/sizeof(arr[0])`；**传参后丢失** | `.size()` 始终可用 |
| **越界** | 无检查，UB | `.at()` 抛异常；`[]` 仍无检查 |
| **内存管理** | 栈自动释放；`new[]` 须 `delete[]` | RAII 自动析构 |
| **适用** | C 接口、固定尺寸底层、极热路径小缓冲 | **绝大多数业务首选** |

## 教材核心编程建议

1. 元素数量**不确定** → **`vector`**
2. 字符串 → **`std::string`**，不要 `char[]`（除非热路径固定缓冲，见 [3.2.4](../3.2-library-type-string/3.2.4-string与C风格字符串.md)）
3. 避免**裸指针 + 裸数组**组合 → 越界、泄漏、悬垂
4. 必须用数组时 → **`begin`/`end` 遍历**，少手写指针算术
