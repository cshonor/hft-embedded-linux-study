# 3.4.5 易错点、失效与考点

## 本节讲什么

教材重点易错点、迭代器失效、`vector` 与 HFT 热路径的关系，及课后考点总结。

## 教材重点易错点

### 1. 尾后迭代器禁止解引用

```cpp
auto e = v.end();
// *e;   // ❌ 未定义行为，可能崩溃
```

`end()` 只是**结束标记**，不是最后一个元素。最后一个元素是 `*(v.end() - 1)`（非空时）。

### 2. 容器修改导致迭代器失效

| 操作 | `vector` / `string` 常见后果 |
|------|------------------------------|
| **`push_back` 等触发扩容** | 所有迭代器、指针、引用**全部失效** |
| **`erase` 删除元素** | 被删位置及之后的迭代器失效 |
| **`insert` 中间插入** | 插入点及之后可能失效 |

```cpp
std::vector<int> v{1, 2, 3};
auto it = v.begin();
v.push_back(4);        // 若扩容，it 失效
// *it;                // ❌ 危险

// 正确：重新获取
it = v.begin();
```

**HFT**：行情缓冲 `vector` 在热路径 `push_back` 后，**不要**继续用旧的 `begin()`/`end()` 缓存迭代器遍历——应重新取或改用下标 + 已知 `size()`（见 [3.3.3](../3.3-library-type-vector/3.3.3-下标与增删.md)、[3.3.5](../3.3-library-type-vector/3.3.5-拷贝移动与扩容.md)）。

### 3. 只读遍历用 `cbegin` / `cend`

防止循环里误写 `*it = ...` 改坏数据。

### 4. 空容器判断

```cpp
if (v.begin() == v.end())   // 等价 v.empty()
```

### 5. 自增用前置 `++it`

后置 `it++` 多一次临时拷贝，热路径避免。

## 课后核心考点

1. 迭代器 vs 下标：**通用性**，适配全部 STL 容器  
2. `begin` / `end`、`cbegin` / `cend` 的**返回类型**与读写权限  
3. 标准 `for` 遍历模板：`it != end()`、`++it`  
4. `*`、 `->`、`++` / `--`、`==` / `!=`  
5. **随机访问**专属：`+`、`-`、`-` 得距离、`<` 比较  
6. **尾后**特性、空容器、`vector` **失效**基础概念  

## 背诵极简版

```
下标不通用；迭代器全容器通用，string 也支持
begin 首元素；end 尾后勿解引用；空则 begin==end
cbegin/cend 只读；只遍历用 cbegin
遍历 it!=end()，不用 <；++it 优于 it++
*it  it->成员  == !=
vector/string：it+n  end-begin 得个数；list/map 不支持
push_back 扩容、erase 后迭代器失效，勿用旧 it
```

## 完整示例

```cpp
#include <iostream>
#include <vector>
#include <string>

int main() {
    std::vector<std::string> vec = {"apple", "banana", "pear"};

    // 读写迭代器
    for (auto it = vec.begin(); it != vec.end(); ++it) {
        *it = "fruit:" + *it;
        std::cout << *it << "\n";
    }

    // 只读 cbegin/cend
    for (auto cit = vec.cbegin(); cit != vec.cend(); ++cit) {
        std::cout << *cit << " ";
        // *cit = "xxx";  // 编译报错
    }
    std::cout << "\n";

    // 迭代器算术
    auto first = vec.begin();
    auto last  = vec.end();
    std::cout << "元素总数：" << last - first << "\n";

    return 0;
}
```
