# 3.2.5 cctype 字符校验

## 本节讲什么

`<cctype>` 单字符判断与转换；`unsigned char` 参数要求；HFT 报文字段格式校验。

`<cctype>` 是 C++ **继承自 C** 的字符处理头文件（C 里 `<ctype.h>`，C++ 用 `<cctype>` 且函数在 **`std` 命名空间**）。**`#include <cctype>`** 在预处理阶段把函数**声明**粘贴进当前文件（见 [2.2.4 `#include` 本质](../ch02-variables-and-basic-types/2.2-variables/2.2.4-extern与头文件分工.md)）。

| 函数 | 含义 | HFT 示例用途 |
|------|------|--------------|
| `isalpha(c)` | 是否字母 | 合约代码是否全字母段 |
| `isdigit(c)` | 是否数字 | 价格、数量字段是否全数字 |
| `isalnum(c)` | 字母或数字 | 通用 token 校验 |
| `isspace(c)` | 是否空白 | 跳过分隔符、检查非法空白 |
| `toupper(c)` / `tolower(c)` | 大小写转换 | 统一合约码大小写 |

## 参数必须是 `unsigned char` 转成 `int`

标准规定这些函数的参数类型是 **`int`**，且必须是 **`unsigned char` 的值**或 **`EOF`**。若直接把可能为负的 **`char`** 传入，在 `char` 为 signed 且值为 `0x80`～`FF` 的平台上是**未定义行为**。

```cpp
// ❌ 危险：char 可能为 signed
char ch = data[i];
if (std::isdigit(ch)) { /* ... */ }

// ✅ 正确：先转成 unsigned char
unsigned char uch = static_cast<unsigned char>(data[i]);
if (std::isdigit(uch)) { /* ... */ }

// ✅ 遍历 string 时推荐
for (unsigned char c : symbol) {
    if (!std::isalnum(c))
        /* 非法字符 */;
}
```

这是写底层 / 报文校验代码时容易忽略的细节。

## HFT：快速校验报文字段

从订单报文、FIX 字段里取出子串后，用 `<cctype>` 做**轻量格式检查**（正式环境还有完整 schema 校验）：

```cpp
#include <cctype>
#include <string>

bool all_alpha(const std::string& s) {
    for (unsigned char c : s)
        if (!std::isalpha(c)) return false;
    return !s.empty();
}

bool all_digit(const std::string& s) {
    for (unsigned char c : s)
        if (!std::isdigit(c)) return false;
    return !s.empty();
}

// 用法示意
std::string symbol = "IF";      // 合约代码段
std::string price  = "3750";    // 价格字段
// all_alpha(symbol), all_digit(price)
```

比手写 `c >= '0' && c <= '9'` 更易读；实现通常经库优化，且覆盖 locale 相关边界（纯 ASCII 报文场景下与手写等价）。

```cpp
// 统一转大写合约码
std::string upper_symbol = symbol;
for (char& ch : upper_symbol)
    ch = static_cast<char>(std::toupper(static_cast<unsigned char>(ch)));
```
