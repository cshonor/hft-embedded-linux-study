# 3.3.5 拷贝移动与扩容

## 本节讲什么

`vector` 扩容时如何拷贝/移动元素；浅拷贝 vs 深拷贝；Rust `Copy`/`Clone` 对照；HFT 禁用拷贝、POD 选型。

## C++ 如何实现「可复制」

靠 **拷贝构造** `T(const T&)` 和 **拷贝赋值** `operator=(const T&)`：

| 机制 | 何时调用 |
|------|----------|
| 拷贝构造 | 扩容搬元素、`push_back(x)` |
| 拷贝赋值 | 覆盖已有元素 |

**未手写时**编译器隐式生成 **逐成员浅拷贝**：

- 成员全是标量 → 浅拷贝 = 深拷贝，**安全进 `vector`**
- 含 **裸指针 / 动态内存** → 共享堆块 → 扩容拷贝后 **双析构**；须手写深拷贝（三/五法则，见 [13.1](../ch13-copy-control/13.1-拷贝控制成员.md)）

```cpp
struct MarketData {
    double bid{0}, ask{0};
    std::int64_t ts{0};
    MarketData() = default;
    MarketData(const MarketData&) = default;
    MarketData& operator=(const MarketData&) = default;
};

std::vector<MarketData> v;
v.push_back({100.5, 100.6});   // 扩容时逐成员拷贝，便宜
```

## 和 Rust `Copy` / `Clone` 对照

| Rust | C++ |
|------|-----|
| **`Copy`** | trivially copyable / POD 标量 struct |
| **`Clone`** | 用户定义的拷贝构造（可能深拷贝） |
| **移动 / 所有权转移** | 移动构造 + `= delete` 拷贝 |

C++ 无统一 trait，靠 **`= default` / `= delete`** 手动控制。

## HFT：禁用拷贝、只允许移动

```cpp
struct HeavyBook {
    std::vector<double> levels;
    HeavyBook() = default;
    HeavyBook(HeavyBook&&) = default;
    HeavyBook& operator=(HeavyBook&&) = default;
    HeavyBook(const HeavyBook&) = delete;
    HeavyBook& operator=(const HeavyBook&) = delete;
};

std::vector<HeavyBook> books;
books.push_back(HeavyBook{});  // 扩容时移动元素，无深拷贝尖刺
```

不可移动时用 **`vector<unique_ptr<T>>`**，扩容只搬指针。

## 为什么常选 POD 作元素

**POD**（Plain Old Data）：基本类型或只含 POD 成员的 `struct`。

| POD / 轻量 struct | 重对象 |
|-------------------|--------|
| 扩容 **memcpy 级**拷贝 | 深拷贝、分配尖刺 |
| 缓存友好 | cache miss 多 |

工程组合：

1. 热路径 `vector<Tick>`，Tick 全标量
2. 容量已知 → **`reserve(n)`** 只扩一次
3. 大对象 → **`vector<unique_ptr<T>>`**
