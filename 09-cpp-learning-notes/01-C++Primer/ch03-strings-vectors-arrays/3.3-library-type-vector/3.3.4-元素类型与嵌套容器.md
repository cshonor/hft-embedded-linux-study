# 3.3.4 元素类型与嵌套容器

## 本节讲什么

`vector<T>` 的 **`T` 不限于基本类型**——自定义类、指针、嵌套 `vector` 等；HFT 常见元素选型。

只要满足容器要求（默认可**拷贝构造、拷贝赋值、可析构**；C++11 起移动参与增容），**任何类型**都能做元素：

| `T` 的例子 | HFT 场景 |
|------------|----------|
| `int`、`double` | 价格、数量、时间戳 |
| 自定义类 `MarketData` | 一条 tick / 快照 |
| `Tick*`、`std::unique_ptr<Order>` | 指针或智能指针（注意所有权） |
| `std::vector<int>` | **`vector<vector<int>>`**：二维订单簿深度 |

```cpp
struct MarketData {
    double bid{0}, ask{0};
    std::int64_t ts{0};
};

std::vector<MarketData> book;            // 自定义类
std::vector<MarketData*> pending;        // 指针（不拥有对象）
std::vector<std::vector<int>> levels;    // 嵌套：levels[档位][量...]
```

## 扩容与元素类型的关系（预告）

`push_back` 触发扩容时，要把已有元素**搬到新内存**——依赖 `T` 的**拷贝或移动**（详见 [3.3.5](./3.3.5-拷贝移动与扩容.md)）：

- 不可拷贝且不可移动 → 不能进 `vector`
- 拷贝/移动很慢 → 热路径延迟抖动
- **优先 `emplace_back`** 原地构造

## HFT 选型速查

| 场景 | 推荐 |
|------|------|
| 热路径 tick、档位 | `vector<Tick>`，`Tick` 为 POD 风格 struct |
| 大对象、禁拷贝 | `vector<unique_ptr<T>>` 或 `vector<T*>` |
| 二维订单簿 | `vector<vector<int>>` 或扁平 `vector<int>` + 下标换算 |
