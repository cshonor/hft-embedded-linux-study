# 3.3.6 HFT 实践与示例

## 本节讲什么

`vector` 在 HFT 里的工程习惯、综合示例与本章背诵极简版。

## 工程习惯 checklist

| 项 | 做法 |
|----|------|
| 增元素 | **`push_back` / `emplace_back`**，不用下标「添加」 |
| 下标 | `0 <= i < size()`；**`v[v.size()]` 永远错** |
| 初始化 | `v(n)` 要 n 个槽位；`v{a,b}` 只要列表里那些 |
| 扩容 | 容量可估 → **`reserve(n)`**（见 [3.2.3](../3.2-library-type-string/3.2.3-拼接与内存优化.md)） |
| 元素类型 | 热路径 POD；大对象 `unique_ptr` |
| 验证 | `assert(v.size() == expected)` |

## 背诵极简版

```
vector=容器/动态数组，≠ host 宿主
v(10) 十个默认值；v{1,2,3} 三个元素；v{10,5} 只有两个！
下标 0..size()-1；v[v.size()] 越界；追加用 push_back
T 可自定义类/指针/嵌套 vector；扩容靠拷贝或移动
浅拷贝类含指针要深拷贝；HFT 热路径 POD + reserve
禁拷贝留移动：=delete 拷贝；或 vector<unique_ptr<T>>
```

## 示例

```cpp
#include <iostream>
#include <vector>

struct MarketData {
    double bid{0}, ask{0};
};

int main() {
    std::vector<int> v(10);           // 10 个 0
    std::vector<int> v_paren(10, 5);  // 10 个 5
    std::vector<int> v_list{1, 2, 3}; // 3 个元素
    // vector<int> trap{10, 5};       // 坑：size=2

    for (int i = 1; i <= 5; ++i)
        v.push_back(i);

    std::vector<MarketData> ticks;
    ticks.push_back({100.5, 100.6});

    std::vector<std::vector<int>> book(5);  // 5 档
    book[0].push_back(100);

    for (std::size_t i = 0; i != v_list.size(); ++i)
        std::cout << v_list[i] << " ";
    std::cout << std::endl;

    // v[10] = 100;  // ❌ 越界

    return 0;
}
```
