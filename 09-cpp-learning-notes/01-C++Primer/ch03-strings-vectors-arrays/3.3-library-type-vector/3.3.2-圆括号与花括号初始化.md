# 3.3.2 圆括号与花括号初始化

## 本节讲什么

`vector` 的 **`()` 默认值初始化** vs **`{}` 列表初始化**——新手最高频坑；HFT 里 `size()` 错一位会静默读错行情。

这是 **`vector` 最容易混的两套逻辑**：

| | **小括号 `()`** | **花括号 `{}`** |
|---|----------------|-----------------|
| 叫法 | **默认值 / 个数初始化** | **列表初始化**（C++11 统一初始化） |
| 语义 | 「给我 **n 个**元素，每个按规则填初值」 | 「花括号里 **有几个数，容器就有几个元素**」 |
| 典型 | `v(10)` → 10 个 `0` | `v{1,2,3}` → 3 个元素，就是 1、2、3 |
| 自定义类 | 每个元素调 **默认构造函数** | 用列表里的值 **逐个构造**，不多生成默认对象 |

## 圆括号：个数 + 默认值初始化

**`vector<int> v(10)`** = **给我 10 个 `int`，每个都用默认值 `0` 填充**。

| 写法 | 含义 | `size()` |
|------|------|----------|
| `vector<int> v(10);` | **10 个 `int`**，值初始化 → `0` | 10 |
| `vector<int> v(10, 5);` | **10 个 `int`**，每个都是 `5` | 10 |
| `vector<MarketData> v(100);` | **100 个对象**，各调默认构造 | 100 |

```cpp
std::vector<int> v(10);      // 10 个 0
std::vector<int> v2(10, 5);  // 10 个 5

struct MarketData { double bid{0}, ask{0}; };
std::vector<MarketData> book(100);  // 100 个默认构造的 tick 槽位
```

## 花括号：列表初始化（挨个塞初值）

**`vector<int> v{1, 2, 3}`** = 花括号里 **3 个数 → 3 个元素**，值就是 `1、2、3`，**不会**再补默认值。

| 写法 | 含义 | `size()` |
|------|------|----------|
| `vector<int> v{1, 2, 3};` | 恰好 3 个元素：`1, 2, 3` | 3 |
| `vector<int> v{10, 5};` | 恰好 **2** 个元素：`10` 和 `5` | **2**（不是 10 个 5！） |
| `vector<int> v{10};` | 恰好 **1** 个元素：`10` | 1 |

```cpp
std::vector<int> a(10, 5);   // size=10，全是 5
std::vector<int> b{10, 5};   // size=2，元素是 10 和 5  ← 最容易写反
```

## 对照记忆

```
v(10)      → 10 个元素，默认填 0（或默认构造）
v(10, 5)   → 10 个元素，全是 5
v{1,2,3}   → 3 个元素，就是 1、2、3
v{10, 5}   → 2 个元素，就是 10 和 5   （不是 v(10,5)！）
```

## HFT：长度错了 = 静默读错行情

- 预分配 **100 档深度**：`vector<Level>(100)`，**不要** `vector<Level>{100}`（只有 **1** 个元素）
- 写完后 **`assert(v.size() == expected)`** 锁死长度
