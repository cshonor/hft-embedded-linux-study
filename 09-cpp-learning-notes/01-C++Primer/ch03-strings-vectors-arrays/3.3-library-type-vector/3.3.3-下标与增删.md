# 3.3.3 下标与增删

## 本节讲什么

`vector` 下标从 0 开始、为何 `v[v.size()]` 越界，以及**不能用下标添加元素**——须用 `push_back` / `emplace_back`。

## 下标从 0 开始

`vector` 和 C 数组一样，**第一个元素下标是 `0`**，最后一个合法下标是 **`size() - 1`**。

以 **`vector<int> v{1, 2, 3}`** 为例：

| | 下标 `0` | 下标 `1` | 下标 `2` | 下标 `3` |
|---|----------|----------|----------|----------|
| 元素 | `1` | `2` | `3` | ❌ **不存在** |
| | | | `size()-1` | **`size()` 返回 3 → 越界** |

- **`v.size()`** 是**元素个数**，不是「最大合法下标」
- 合法访问：`v[0]` … `v[size()-1]`；循环写 **`i < v.size()`**
- **`v[v.size()]`** 读是 UB，写是 **OOB 写**

```cpp
std::vector<int> v{1, 2, 3};
// v[3] = 4;           // ❌ 合法下标只有 0,1,2
// v[v.size()] = 4;    // ❌ size()==3，下标 3 越界
v.push_back(4);        // ✅ 正确追加
```

## 不能用下标「添加」

`v[10] = 42` 在 `size() <= 10` 时：

- **不会**自动扩容
- 只是往**容器外非法地址**写 → **未定义行为**

新元素一律：

```cpp
v.push_back(x);              // 拷贝/移动进 vector
v.emplace_back(args...);     // 原地构造
```

## HFT：越界写 = 静默灾难

- 可能**不立刻 crash**，先踩坏相邻数据 → 偶发错价、漏单
- 崩溃常在行情突发时 → **错过交易窗口**
- 习惯：`i < v.size()` 断言；增元素只用 **`push_back` / `emplace_back`**

```cpp
std::vector<Tick> ticks;       // 已有 3 条，下标 0..2
// ticks[ticks.size()] = tick; // ❌ OOB 写
ticks.push_back(tick);         // ✅
```
