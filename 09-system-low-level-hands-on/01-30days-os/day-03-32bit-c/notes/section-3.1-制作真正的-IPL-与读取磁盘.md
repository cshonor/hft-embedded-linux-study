## ① 制作真正的 IPL 与读取磁盘

Day 1–2 的 IPL **只显示文字**，**没有装载 OS 本体**，只是「假引导」。从 Day 3 起，IPL **名副其实**——核心职责是调用 **BIOS 中断** 把软盘上的 **bootpack（OS 本体）** 读进内存，完成启动链装载。

> **本节第一次出现三个名字** — 先分清，后面代码才看得懂。

### IPL、bootpack、`INT 0x13` 分别是什么？

**IPL 和 bootpack 是两回事，不是同一个东西。**

| | **IPL** | **bootpack** |
|---|---------|--------------|
| **是什么** | **初始程序加载器**（Initial Program Loader） | **操作系统本体**（后面进 32 位、跑 C 的那一大块） |
| **在软盘上** | **第 1 扇区**（C0-H0-S1），固定 **512 B** | **从第 2 扇区起** 连续很多扇区（本书读 **10 柱面**） |
| **进内存后** | BIOS 放到 **`0x7C00`** 并执行 | IPL 把它放到 **`0x8200` 起**（约 **180 KB**） |
| **Day 1–2** | 只会打印文字，**不读盘** | 还没有 |
| **Day 3 起** | **真 IPL**：读盘，最后 **`JMP 0x8200`** | **被 IPL 搬进内存** 的 OS 数据 |

记两句话：

1. **IPL = 512 字节的「搬运工」** — 自己先被 BIOS 跑起来。  
2. **bootpack = 「货」** — 不是 512 B，是后面那 ~180 KB 的 OS 本体。

**`INT 0x13` 又是什么？** — 它**既不是 IPL，也不是 bootpack**。

- **`INT 0x13`** = **BIOS 磁盘中断**（interrupt **0x13**），固件提供的「读软盘扇区」服务。  
- OS 还没加载、还在 **16 位实模式** 时，代码**不能直接操作磁盘硬件**，只能 **`INT 0x13`** 让 BIOS 代读。  
- Day 3 里 **`AH = 0x02`** 表示读扇区；IPL 里的 **`readloop` 反复调用它**，把 bootpack **一块块（每块 512 B）** 从软盘读进 **`0x8200` 起** 的内存。  
- 读失败时用 **`AH = 0x00`** 复位软驱，**单扇区最多重试 5 次**。

```text
BIOS 读 512B IPL → 0x7C00 执行（搬运工上岗）
       ↓
IPL 循环 INT 0x13（AH=0x02）→ 软盘 S2 起读入 bootpack → 内存 0x8200
       ↓
JMP 0x8200 → bootpack / nasmhead 继续（见 §3.2）
```

**别和硬盘 MBR 搞混：** 网上常把「磁盘最前 512 B + 分区表」也叫引导 — 那是 **PC 硬盘** 说法。本书 Day 3 用的是 **1.44 MB 软盘**：512 B 启动扇区就是 **IPL**，**没有**「先读 MBR 分区表再找系统盘」这一步，直接从 **A 软驱、S2 起** 读 bootpack。

### IPL 是什么

**IPL（Initial Program Loader，初始程序加载器）**

| 要点 | 说明 |
|------|------|
| **载体** | 软盘 **C0-H0-S1**（第 0 柱面、0 磁头、1 扇区），固定 **512 B** |
| **识别** | 末尾两字节必须是 **`0x55AA`**，BIOS 才当作可启动扇区 |
| **加载地址** | BIOS 读入内存 **`0x7C00`** 并从此处执行 |
| **Day 1–2** | 仅屏幕打印，无磁盘加载 |
| **Day 3 起** | **`INT 0x13`** 循环读盘，把 **bootpack** 载入 **`0x8200` 起** |

#### 启动数据流

```text
软盘 C0-H0-S1（ipl.bin 512B）
    → BIOS 载入内存 0x7C00 运行
        ↓ INT 0x13 循环读盘
软盘 C0-H0-S2 ~ C9-H1-S18（10 柱面）
    → 载入内存 0x8200 起（bootpack，约 180KB）
        ↓ JMP 0x8200
nasmhead / bootpack 继续启动（见 [§3.2](./section-3.2-纸娃娃操作系统.md)）
```

### 软盘物理结构（1.44 MB FAT12）

![1.44 MB 软盘柱面 / 扇区 / 磁头（原书插图）](../../assets/floppy-disk-chs-geometry.png)

原书要点（肉眼看不见，但 **`INT 0x13` 必须按这套编号寻址**）：

| 标注 | 含义 |
|------|------|
| **柱面 0 ~ 79** | 共 **80** 个柱面（同心磁道，由外向内编号） |
| **扇区 1 ~ 18** | 每个柱面 **18** 个扇区（径向切分，每扇区 **512 B**） |
| **磁头 0 / 1** | **H0** 正面、**H1** 反面，各对应盘片一面 |

| 术语 | 定义 | 本书参数 |
|------|------|----------|
| **柱面 Cylinder (C)** | 盘片上一组同心磁道，从外向内编号 | 共 80 柱面；IPL 读 **前 10 柱面（C0~C9）** |
| **磁头 Head (H)** | 软盘正反面各 1 个读写头 | **H0** 正面、**H1** 反面 |
| **扇区 Sector (S)** | 单条磁道上最小读写单元 | 每磁道 **18** 扇区，每扇区 **512 B** |

整盘容量：**80 × 2 × 18 × 512 = 1,474,560 B（1.44 MB）**，与 Day 2 的 `helloos.img` 大小一致。

**10 柱面容量：**

```text
10 × 2 磁头 × 18 扇区 × 512 B = 184,320 B ≈ 180 KB  →  bootpack 本体大小
```

读盘顺序（物理寻址）：**同柱面 H0 扇区 1→18 → H1 扇区 1→18 → 柱面 +1**，直到 C9 结束。IPL 自身占 **S1**，OS 数据从 **S2** 开始读。

### 核心 API：BIOS `INT 0x13`（16 位实模式）

#### 读扇区 · `AH = 0x02`（最常用）

| 寄存器 | 含义 | 本书典型值 |
|--------|------|------------|
| **AH** | 子功能 | **`0x02`** = 读扇区 |
| **AL** | 连续读取扇区数 | **`1`**（逐扇区读，便于重试） |
| **CH** | 柱面号 C | **0~9** |
| **CL** | 扇区号 S | **1~18** |
| **DH** | 磁头号 H | **0 / 1** |
| **DL** | 驱动器号 | **`0x00`** = A 软驱 |
| **ES:BX** | 写入内存地址 | **ES=0x0820, BX=0** → 物理地址 **0x8200** |

#### 错误判定

| 结果 | 标志 | 汇编 |
|------|------|------|
| 成功 | **CF = 0** | `JNC next` |
| 失败 | **CF = 1** | `JC error` 或进入重试 |

软盘机械不稳定，标准容错：**单扇区最多重试 5 次**；仍失败则打印 **load error** 并 **`HLT` 停机**。

#### 驱动器复位 · `AH = 0x00`

读取出错后调用，重置磁头、重新寻道，再 **`JMP retry`**。

### 三大核心逻辑（[ipl.asm](../code/ipl.asm)）

#### 1. 失败重试 5 次

```nasm
readloop:
    MOV     SI, 0           ; 本扇区失败计数
retry:
    MOV     AH, 0x02
    MOV     AL, 1
    MOV     BX, 0
    MOV     DL, 0x00
    INT     0x13
    JNC     next            ; 成功 → 下一扇区
    ADD     SI, 1
    CMP     SI, 5
    JAE     error           ; ≥5 次 → 报错停机
    MOV     AH, 0x00        ; 复位软驱
    MOV     DL, 0x00
    INT     0x13
    JMP     retry
```

#### 2. 循环遍历 10 柱面

每读完 **512 B**，**ES 段 +0x20**（因为 512÷16=0x20，x86 没有 `ADD ES, imm` 只好改 AX 再写回 ES）：

```nasm
next:
    MOV     AX, ES
    ADD     AX, 0x0020
    MOV     ES, AX          ; 内存指针下移 512 B
    ADD     CL, 1           ; 扇区 +1
    CMP     CL, 18
    JBE     readloop        ; 本磁道未读完
    MOV     CL, 1           ; 换磁头，扇区归 1
    ADD     DH, 1
    CMP     DH, 2
    JB      readloop        ; H0→H1
    MOV     DH, 0           ; 换柱面
    ADD     CH, 1
    CMP     CH, CYLS        ; CYLS EQU 10
    JB      readloop
```

#### 3. 启动框架 + FAT12 引导参数块（BPB）

IPL 开头除 **`JMP entry`** 外，还要填 **FAT12 磁盘头**（`"HARIBOTE"`、每扇区 512、18 扇区/磁道等），与后面用工具做 **1.44 MB 软盘镜像** 一致：

```nasm
CYLS    EQU     10
ORG     0x7C00
JMP     entry
DB      0x90
DB      "HARIBOTE"
; … BPB 字段 …
RESB    18

entry:
    MOV     AX, 0
    MOV     SS, AX
    MOV     SP, 0x7C00
    MOV     DS, AX
    MOV     AX, 0x0820
    MOV     ES, AX
    MOV     CH, 0
    MOV     DH, 0
    MOV     CL, 2           ; S1 是 IPL 自身，从 S2 读 OS
    ; → readloop / retry / next …
```

完整可编译源码：[code/ipl.asm](../code/ipl.asm) · 构建说明：[code/README.md](../code/README.md)。

### 现代存储对照（HFT 类比）

#### 老式软盘 IPL 链

1. **512 B IPL** 硬编码：**读哪里、读多少、失败重试**；
2. **无内核介入**，实模式 BIOS 直接访问磁盘；
3. 读完后 **`JMP 0x8200`** 进入 32 位保护模式前的 **nasmhead / bootpack**。

#### Linux 现代链（NVMe / SSD / mmap）

1. **Bootloader（GRUB / systemd-boot）** ≈ 老式 IPL：封装读盘、分区寻址、重试、块缓存；用 **EFI Block I/O** 替代 **`INT 0x13`**；
2. 内核启动后：设备驱动接管磁盘，**mmap** 映射块设备；IO 重试、队列由 **块子系统** 统一管理。

| | 相同 | 差异 |
|---|------|------|
| 启动第一阶段 | 必须 **显式定义 IO 策略**（读哪里、读多少、出错怎么办） | 老式：逻辑挤在 **512 B**；现代：**Bootloader + 内核驱动** 分层 |

**HFT：** 像 **从磁盘 / NVMe mmap 策略库** — 启动链上 **「谁读、读多少、读失败怎么办」** 必须写清；Linux 里由 bootloader + 内核完成。

### 关键考点

1. **IPL 本质**：512 B 启动扇区；Day 3 新增 **磁盘加载**，不再只打印文字。
2. **寻址三元组**：**CH** 柱面、**DH** 磁头、**CL** 扇区；单扇区 **512 B**。
3. **读盘 API**：**`INT 0x13`, `AH=0x02`**；用 **CF** 判错。
4. **可靠性**：单扇区 **最多 5 次重试**；失败前 **复位驱动器（`AH=0x00`）**。
5. **加载范围**：**C0~C9** 共 10 柱面 ≈ **180 KB** → 内存 **`0x8200` 起**。
6. **内存步进**：每扇区 **ES += 0x20**（512 字节段偏移）。
