## 5.1–5.3 编译器能力、性能度量与程序示例

### 5.1 优化编译器的能力和局限性

**编译器常能做：**

- 常量折叠、死代码消除、内联、强度削减（`*8` → `<<3`）
- 寄存器分配、循环不变量外提、公共子表达式消除
- 自动向量化、循环展开（`-O3` / `-funroll-loops`）

**编译器不敢/不能做（除非你帮它）：**

| 障碍 | 例子 |
|------|------|
| **指针别名** | `*x` 与 `*y` 是否同一地址未知 → 不敢乱重排 store |
| **全局可见副作用** | 函数可能改内存 → 不敢删「看似多余」调用 |
| **跨编译单元** | 无 LTO 时不知道 `inline` 候选 |
| **违反严格别名** | `int*` 与 `float*` 互转 — UB |

**帮编译器：**

```c
restrict  // C99：承诺指针不重叠
__restrict__
#pragma GCC ivdep   // 谨慎：声称无依赖
static inline + 同文件
-fno-strict-aliasing  // 妥协，非首选
```

**HFT：** 热路径 `.cpp` **单文件或 LTO**；`restrict` 用于明确 buffer；避免热循环里调用 **未知副作用** 的函数。

### 5.2 表示程序性能

- **时钟周期** 比秒更稳（频率变化时）
- **CPE (Cycles Per Element)** — 每处理一个元素平均周期，便于比较不同长度
- **吞吐** — 元素/秒；**延迟** — 单任务完成时间

```
性能 = 工作 / 时间 = (n 元素 × CPE) / 时钟频率
```

**HFT：** 行情 **每 tick CPE**、发单路径 **ns/op**；P99 看 tail，不单看平均 CPE。
