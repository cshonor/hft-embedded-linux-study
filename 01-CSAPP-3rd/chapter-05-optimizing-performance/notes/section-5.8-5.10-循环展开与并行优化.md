## 5.8–5.10 循环展开、提高并行性与结果小结

### 5.8 循环展开 (Loop Unrolling)

**手动或编译器展开：**

```c
// 2 路展开
for (i = 0; i < n-1; i += 2) {
    acc = (acc + data[i]) + data[i+1];
}
// 处理尾部 n%2
```

**收益：**

- 减少 **分支**（每迭代处理多元素）
- 增 **指令级并行** — 多条独立 load/add 可并行发射
- 常与 **多累加器** 联用

**代价：** 代码膨胀、I-cache 压力、寄存器压力（→ 5.11）

```bash
gcc -O3 -funroll-loops ...
```

### 5.9 提高并行性

#### 5.9.1 多个累积变量

```c
// 2 路展开 + 双累加器，打破 loop-carried dep
x0 = ident; x1 = ident;
for (i = 0; i < n-1; i += 2) {
    x0 = x0 + data[i];
    x1 = x1 + data[i+1];
}
acc = (x0 + x1) + ...;
```

`x0` 与 `x1` **无依赖** — CPU 可并行执行。

#### 5.9.2 重新结合变换 (Reassociation)

```c
// 结合律对浮点不严格成立！整数 + 在数学上 OK
// (a + data[i]) + data[i+1]  vs  a + (data[i] + data[i+1])
```

- 编译器在 `-ffast-math` 下对浮点重结合 — **HFT 慎用** 于价格累加
- 整数 tick 求和、checksum 可重结合

### 5.10 优化合并代码的结果小结

原书 `combine` 各版本 CPE 数量级下降 — **要点：**

| 技术 | 典型作用 |
|------|----------|
| 内联/去调用 | 大 |
| 寄存器累加器 | 大 |
| 2× 展开 | 中 |
| 4× 展开 + 多累加器 | 中–大 |
| 8× 展开 | 收益递减 |

**HFT：** 解码 **固定宽度字段** 的循环常自动展开；手写 SIMD（AVX2）是下一档，本书未深讲，见 intrinsics / 编译器 autovec。
