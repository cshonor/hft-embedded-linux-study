## 9.1 物理和虚拟寻址 · 名字辨析

> **Ch9 §9.1** · [章导读](../README.md) · 上节 — · 下节 [§9.2 →](./section-9.2-地址空间.md)

---

| | **物理地址 (PA)** | **虚拟地址 (VA)** |
|--|-------------------|-------------------|
| 指向 | 真实 **DRAM** 字/页 | 进程眼中的逻辑地址 |
| 谁发出 | 翻译完成后的访存 | **程序指令** 里的指针/PC |
| 谁翻译 | — | **MMU + 页表（+ TLB）** |

**为何发明虚拟内存（四作用）：**

1. **隔离** — 每进程独立 VA；互踩难度大  
2. **扩容** — 冷页下 **swap**；可用逻辑空间可大于内存条  
3. **按需加载** — 只驻留用到的页  
4. **简化编程** — 整齐连续 VA；物理碎片透明  

**缓存栈 vs 虚拟内存（别混）：** L1/L2/LLC 缓存 **物理侧** 数据；虚拟内存是覆盖 **DRAM+swap** 的 **OS 抽象**。

```
程序 VA → MMU → PA → [L1→L2→LLC] → DRAM （缺页则 OS 可能从 swap 装入）
```
