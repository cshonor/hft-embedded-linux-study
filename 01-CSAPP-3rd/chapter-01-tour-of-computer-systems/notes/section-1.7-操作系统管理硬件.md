## 1.7 操作系统管理硬件

OS 在 **应用程序与硬件之间** 提供统一抽象，防止进程互相踩踏，并 **更高效** 地使用资源。

### 1.7.1 进程 (Processes)

- **进程 = 程序的一次运行实例** — 独占（逻辑上）虚拟地址空间、文件描述符、权限上下文
- **上下文切换 (context switch)** — CPU 从进程 A 换到 B：保存/恢复寄存器、切换页表、刷新 TLB（有成本）
- **并发假象** — 单核上多进程靠 **时间片轮转**；多核上真并行

**HFT：**

- 热路径进程 **固定 CPU affinity**，减少迁移与 cache 冷启动
- 与 housekeeping（日志、监控）**分进程/分核**（→ [12-HFT ch05](../../../17-HFT-Low-Latency-Practice/chapter-05-操作系统内核极致调优/)）

### 1.7.2 线程 (Threads)

- **线程** — 同一进程内多个执行流，**共享** 地址空间与文件描述符
- 创建/切换通常比进程轻，但引入 **竞态、锁、伪共享**
- 现代 Linux：**N:M 线程模型**（用户线程 ↔ 内核线程）

**HFT：**

- 常见模型：**单进程多线程** — 收包线程 / 策略线程 / 发单线程，用 **无锁队列** 或 **单 writer 原则** 分隔
- 深入 → [Ch 12 并发编程](../../chapter-12-concurrent-programming/)

### 1.7.3 虚拟内存 (Virtual Memory)

> **「虚拟」= 逻辑地址，不是一块名叫「虚拟」的内存条。** 深入 → [Ch 9](../../chapter-09-virtual-memory/)

#### 先分清两个概念

| | **物理内存 (DRAM)** | **虚拟内存** |
|--|---------------------|--------------|
| 是什么 | 主板上真实 **内存条**，有真实物理地址 | **OS + MMU** 造出的 **逻辑地址空间**，不是独立硬件条 |
| 容量 | 有限（如 16G / 32G） | 每进程一套独立的 **虚拟地址**；程序以为自己独占超大连续空间 |
| 谁用直接地址 | 硬件、内核页帧 | **用户程序只看见虚拟地址** |

#### 为何发明（四个核心作用）

| # | 作用 | 白话 |
|---|------|------|
| **① 隔离** | 每进程独立完整 VA 空间 | A 看不见 B；崩溃不易拖垮全系统。无 VM 则大家共物理地址，乱写互相覆盖 |
| **② 扩容 (swap)** | DRAM 不够时把冷页挪到硬盘 **swap / 页面文件** | 程序仍读写原 VA，无感知；可用「逻辑内存」可大于内存条 — 大众最熟的那一档 |
| **③ 按需加载** | **分页**：只用到的页进 DRAM | 程序几百 MB，不必一次装完 |
| **④ 地址统一** | 程序员面对整齐 VA | 碎片物理布局由 **MMU** 做 **VA→PA** 翻译，对软件透明 |

#### 和缓存层级怎么叠（别混）

| 层 | 属性 |
|----|------|
| **L1 / L2 / LLC** | 片上 SRAM，缓存 **物理内存里的数据副本**（硬件） |
| **DRAM** | 真实内存条（硬件） |
| **虚拟内存** | OS **逻辑抽象**，覆盖 **【DRAM + 硬盘 swap】** |

```
CPU 看 VA → MMU(+页表/TLB) → PA → L1/L2/LLC → DRAM（或再经 OS 从 swap 装回）
```

#### 极简例子（物理仅 8G）

1. 浏览器进程拿到「完整」虚拟地址空间，以为独占大内存  
2. 实际可能只用约 1G 物理页  
3. 游戏、聊天各有自己 VA，**互不冲突**  
4. 内存紧时后台页进 **swap**，**VA 不变**，程序无感  

#### 关键术语

| 词 | 含义 |
|----|------|
| **MMU** | CPU 内地址翻译 / 缺页硬件 |
| **缺页 (page fault)** | 要访问的页不在 DRAM → 进内核；可能从 swap/文件装入 |
| **页 (page)** | VM 管理单位，常见 **4KB**（另有 2M/1G 大页） |

**HFT：**

- 热路径数据 **mlock / 大页 / 预 fault**，避免 tick 上 page fault  
- **NUMA：** 分配与访问同节点（→ [Ch 9](../../chapter-09-virtual-memory/)、[06-Linux-Virtual-Memory-Manager](../../../06-Linux-Virtual-Memory-Manager/)）

### 1.7.4 文件 (Files)

- **Unix 哲学：一切皆文件** — 普通文件、目录、设备、socket 都用 **字节 I/O 接口**（`read`/`write`）
- 内核 **VFS** 统一不同文件系统与设备

**HFT：**

- 行情/订单：**socket 也是文件描述符** — `epoll` 等多路复用（→ [Ch 10–11](../../chapter-10-system-io/)、[10-UNP](../../../11-UNP-Vol1/)）
- 配置文件：启动时读一次，不在热路径 `open`

→ OS 专章：[14-Systems-Performance Ch 3](../../../15-Systems-Performance-2nd/chapter-03-operating-systems/)
