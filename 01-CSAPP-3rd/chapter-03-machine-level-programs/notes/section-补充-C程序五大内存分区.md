## 补充 · C 程序五大内存分区（衔接 §3.2.2 栈帧 · HFT）

> [章导读](../README.md) · 栈帧细节 [§3.2.2](./section-3.2.2-栈帧.md) · 段与权限 [Ch2 §2.1.5](../../chapter-02-representing-information/notes/section-2.1.5-代码表示.md) · 加载 [Ch7 §7.9](../../chapter-07-linking/notes/section-7.9-加载可执行目标文件.md) · VA [Ch9](../../chapter-09-virtual-memory/)

> **定位：** 教科书常说「栈 / 堆 / 数据 / 代码」；这里按运行时习惯拆成 **五块**（全局静态再分 `.data` / `.bss`），方便对照第三章栈帧与 HFT 分配策略。

---

### 极简地址空间（高地址 → 低地址）

```
高地址
┌─────────────────────────────┐
│  栈 stack（%rsp 向下长）      │  局部变量 / 返回地址 / 栈帧
│            ↓                │
├─────────────────────────────┤
│  （中间常有 mmap / 共享库）   │  → Ch9；此处略
├─────────────────────────────┤
│            ↑                │
│  堆 heap（brk/mmap 向上长）  │  malloc 等动态内存
├─────────────────────────────┤
│  .bss   零初始化全局/static  │  文件中几乎不占空间
│  .data  已初始化全局/static  │
├─────────────────────────────┤
│  .rodata 只读常量 / 字符串字面量 │
│  .text   机器指令（只读+可执行） │
低地址
└─────────────────────────────┘
```

> 段名是链接器标签；真正硬约束是 **页权限 R/W/X**（→ [§2.1.5](../../chapter-02-representing-information/notes/section-2.1.5-代码表示.md)）。

---

### 1. 栈区 stack（第三章栈帧核心）

| | |
|--|--|
| **放什么** | 局部变量、溢出参数、返回地址、保存的寄存器、`push`/`pop` 临时数据 |
| **谁管** | 进入函数建栈帧，返回即收回 — **不用 `free`** |
| **增长** | x86-64 **向低地址**；主线程默认常约 **8MB** 量级（可调）；递归过深 → 栈溢出 |
| **速度** | 只动 `%rsp` 等，**无系统调用**；临时计算优先栈 / 寄存器 |

**HFT：** 大数组别塞函数局部（易爆栈）→ 堆 / 内存池；热路径 **`inline`** 少建帧（→ [§3.7.7](./section-3.7-过程与栈帧.md)）。

---

### 2. 堆区 heap（`malloc` / `free`）

| | |
|--|--|
| **放什么** | `malloc` / `calloc` / `realloc` 拿到的块 |
| **谁管** | **手动**申请与释放；漏 `free` → 泄漏 |
| **增长** | 向高地址扩张；容量可达 GB 级（受 VA / 物理限制） |
| **速度** | **慢于栈**：分配器锁、碎片、偶发 **`brk`/`mmap` 进内核** |

**纠正：** 不是「每次 `malloc` 都进内核」。glibc 等常在用户态 freelist 里切块；arena 不够才 syscall。对 HFT 仍危险：**热循环里频繁 malloc/free** → 抖动、锁、偶发 syscall。

**HFT：** 启动时 **内存池 / 预分配**；交易路径零动态分配。

---

### 3. 全局 / 静态数据（两段）

#### （1）`.data` — 已初始化

```c
int g_val = 10;       // .data
static int s_a = 5;   // .data
```

加载时已有值；进程结束才释放。

#### （2）`.bss` — 零初始化

```c
int g_empty;          // .bss → 加载时清 0
static int s_empty;   // .bss
```

**不占可执行文件磁盘空间**（只记大小）；加载时内核/加载器统一清零 → 镜像更小。

**HFT：** 行情配置、长期缓存表可放全局/static（`.data`/`.bss`），进程常驻；仍要注意 cache 与多线程 false sharing。

---

### 4. 代码段 `.text`

- 编译后的 **机器指令**（函数体）
- **只读 + 可执行**；运行时改写 → 通常 **segfault**
- 进 **icache**；`-O3` 内联展开后的代码也在这里

**纠正：** 字符串字面量一般在 **`.rodata`**，不在 `.text`。

---

### 5. 常量段 `.rodata`

```c
char *str = "hello";  // "hello" → .rodata；改 str[0] 通常 segfault
```

`const` 全局常量、字符串字面量等；与 `.text` 同属只读映射（权限细节依工具链略有差）。

---

### 与第三章 / HFT 的咬合

| 点 | 含义 |
|----|------|
| **栈帧 = 栈区上的一次调用切片** | `%rsp`/`%rbp`、`call`/`ret` 全在操作栈 → [§3.2.2](./section-3.2.2-栈帧.md) · [§3.7](./section-3.7-过程与栈帧.md) |
| **`-O3`** | 小局部常 **寄存器化**，少栈读写 |
| **延迟优先级（经验）** | **寄存器 > 栈（已在 cache）> 只读热代码/数据 > 堆上冷分配** — 堆贵在 **分配路径**，不在「字节本身更慢」 |
| **全局** | `.data`/`.bss` 适合常驻配置；热路径仍优先寄存器 |

### 口述巩固 · 自测

1. **五块各放什么？** 栈向下、堆向上各什么意思？  
2. **`.data` vs `.bss`？** 为何 `.bss` 让可执行文件更小？  
3. **`"hello"` 在 `.text` 还是 `.rodata`？**  
4. **热循环为何禁频繁 `malloc`？** 每次都进内核吗？  
5. **HFT 临时变量优先放哪？** 大缓冲呢？
