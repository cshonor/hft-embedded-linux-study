## 3.3–3.4 数据格式与访问信息

> **一条线：** 后缀 `b/w/l/q` = 操作多宽；四种操作数写法 = 值从哪来。这是 **x86-64 System V ABI** 在机器码层的底座（类型宽度 + 用寄存器/栈读内存）。

→ ABI 总览：[Ch2 · ABI](../../chapter-02-representing-information/notes/section-2.1.2-abi-application-binary-interface.md) · 传参表：[§3.7](./section-3.7-过程与栈帧.md)

### 3.3 数据格式（x86-64）

ABI / 编译器约定：**C 类型占几字节** ↔ **指令后缀** ↔ **用哪一段寄存器**。

| C 类型 | 字节 | 汇编后缀 | 对应寄存器片段 | 说明 |
|--------|------|----------|----------------|------|
| `char` | 1 | **b** (byte) | `%al`（`%rax` 低 8 位） | 字节操作 |
| `short` | 2 | **w** (word) | `%ax`（低 16 位） | 字操作 |
| `int` | 4 | **l** (long word) | `%eax`（低 32 位） | 双字；32 位整型 |
| `long` / 指针 | 8 | **q** (quad word) | `%rax`、`%rdi` 等完整 64 位 | LP64：指针统一 8 字节 |

**补充：**

1. **`long` 与指针都是 8 字节** — 地址、长整数走同一套 64 位通用寄存器（Linux x86-64 **LP64** ABI 强制）。
2. 后缀告诉 CPU **操作多少位**：`movb` = 1B，`movq` = 8B；后缀与类型对不上 → 半截寄存器/内存被改写。
3. 写 `%eax` 时，硬件常会 **清零 `%rax` 高 32 位**（x86-64 规则）— 读反汇编时别被「只动了 eax」吓到。

```asm
movl $10, %eax      ; l：4 字节 int 立即数 → eax（符合 int 规则）
movq (%rax), %rdi   ; q：间接寻址，读 rax 指向处的 8 字节 → rdi
```

### 3.4.1 操作数指示符（取值从哪来）

汇编操作数常见四类（书后还有基址+偏移、变址；栈/`struct` 读写都靠它们）。

| 类型 | AT&T 示例 | 含义 |
|------|-----------|------|
| **立即数** | `$0x4` | 常数；**必须带 `$`**，编译期固定 |
| **寄存器** | `%rax` | 取寄存器里当前值 |
| **绝对地址** | `0x601000` | 常数当内存地址（全局/老式写法；现代多用 RIP 相对） |
| **间接** | `(%rax)` | 把 `%rax` 里的数当 **地址**，读/写该内存 ≡ C 的 `*p` |

再往上两档（热路径常见）：

| 类型 | 示例 | 含义 |
|------|------|------|
| 基址+偏移 | `8(%rbp)` / `8(%rsp)` | `*(base + 8)` — 栈帧、结构体字段 |
| 变址 | `(%rax,%rcx,4)` | `*(rax + rcx*4)` — 数组 `A[i]` |

**数组：** `A[i]` 常变成 `lea` / 变址寻址 — 热循环里可见。

### 和 ABI 的串联

| ABI 条款 | 在本节约等于 |
|----------|----------------|
| `int` 4B、指针 8B | 指令用 **`l` / `q`** |
| 前 6 个整型/指针参数 → `%rdi,%rsi,…` | 全是 **64 位 q 寄存器** |
| 栈上局部量、多余参数、返回地址 | **`n(%rsp)` / `(%rsp)`** 间接寻址 |

不遵守宽度或偏移 → 读错字段、栈错乱 → 崩。手写 asm 调 C / 解析二进制布局时，**先对后缀，再对寻址**。

### 3.4.2–3.4.3 数据传送指令

- `mov` — 源→目的（**不能** mem→mem）
- `movz` / `movs` — 零扩展 / 符号扩展加载较小类型
- `lea` — **只算地址、不访存**（也常用于 `x + k*scale` 快速算术）

### 3.4.4 压入和弹出栈数据

- **栈向低地址增长** — `%rsp` 指向栈顶
- `pushq %rax`：`rsp -= 8`，写入 `(%rsp)`
- `popq %rax`：读出，`rsp += 8`
- `call` = push 返回地址 + jump；`ret` = pop 到 PC

**HFT：** 深调用栈、大栈帧 → cache miss、页 fault；热路径 **内联、尾调用、少 alloca**。
