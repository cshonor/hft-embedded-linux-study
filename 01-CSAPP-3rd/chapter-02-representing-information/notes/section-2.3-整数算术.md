## 2.3 整数算术（2.3.1–2.3.8）

> **本节一条线：** 固定宽度下算术 = 模 2^w（无符号）或补码语义（有符号）→ 溢出可预期但易错。

---

### 2.3.1 无符号加法

固定 **w 位** 宽度做加法：结果按 **模 \(2^w\)** — 超过 \(2^w\) 就把进位 **丢掉**，**不** 抛异常。

```c
uint32_t a = UINT32_MAX, b = 1;
uint32_t s = a + b;   /* 0 — 绕回；模 2^32，进位丢弃 */
```

| | 含义 |
|--|------|
| `UINT32_MAX + 1` | → **0**（wrap） |
| 检测 wrap（仍在 w 位内） | `sum < a`（`a`、`sum` 均为 unsigned） |

#### 和 ALU 加法器：就是一回事

硬件里的 **ALU 加法器** 做的就是这套 **固定位宽** 加法：算完若产生超出 w 位的进位，硬件直接 **丢掉**（或只留给标志位/`CF` 看一眼），**不会**像高级语言那样替你抛「溢出异常」。

| C 无符号加法（文档规则） | ALU 硬件 |
|--------------------------|----------|
| 固定 w 位 | 加法器位宽固定（如 32/64） |
| 超过 \(2^w\) 丢进位 | 进位输出丢弃 / 仅进标志 |
| 不抛异常 | 没有「异常」这条路径 |
| `UINT32_MAX+1 → 0` | 全 1 + 1 → 全 0（wrap） |

所以语言层的无符号加法和底层 ALU 行为 **严丝合缝**；HFT 吃透序号/checksum wrap，本质是在吃透 **硬件怎么加**。

**HFT：** 序列号、checksum 的 **环形计数** 正好吃这套 wrap；业务累计（成交量等）用 **`uint64`** + 显式上限检查，别 silently 绕回当「还很大」。

---

### 2.3.2 补码加法

- **位级电路与无符号加法相同**；是否「溢出」取决于 **按有符号解释**
- **有符号溢出：** 同号相加得异号

```
  s > 0 && t > 0 && (s+t) < 0   → 正溢出
  s < 0 && t < 0 && (s+t) > 0   → 负溢出
```

**HFT：** 累计成交量、序号用 `uint64`；有符号 tick 差分注意 **INT_MIN** 边界。

---

### 2.3.3 补码的非

- **−t** = **~t + 1**（w 位内）— **所有位**取反再加 1，不是只改符号位  
- 例：`+1` → `00000001` → `~` → `11111110` → `+1` → `11111111`（= −1）→ [§2.2.3](./section-2.2-整数表示与类型转换.md)  
- **INT_MIN** 取负：**仍是自己**（8 位 −128）

```c
/* 危险：-x 当 x == INT_MIN */
-int x = INT_MIN;
```

---

### 2.3.4–2.3.5 无符号与补码乘法

- 乘积 **截断到 w 位** — 无符号等价模 2^w
- 编译器常把 `x * 常数` 优化为 **移位 + 加减**（→ §2.3.6、[Ch 5 优化](../../chapter-05-optimizing-performance/)）

---

### 2.3.6 乘以常数

- `x * (2^k ± c)` → shift/add 组合 — 读汇编时识别 strength reduction

---

### 2.3.7 除以 2 的幂

| 类型 | `>> k` 语义（典型） |
|------|---------------------|
| 无 unsigned | 逻辑右移 ≈ **/ 2^k**（ toward 0） |
| 有 signed | **算术右移**（补符号位）— C 标准对负数不完全保证 |

```c
-8 >> 1;    /* x86 gcc 常为 -4 */
-1 >> 1;    /* x86 gcc 常为 -1 */
```

与 §2.1.9 右移规则一致 — 关键路径用无 unsigned 或显式除法。

---

### 2.3.8 关于整数算术的最终思考

| 原则 | 说明 |
|------|------|
| 机器算术 ≠ 数学算术 | 固定宽度 + 溢出/截断 |
| 无符号 wrap 有时 **故意** | 哈希、checksum、TCP/环形序号 |
| 安全路径 | 更宽类型检查、Rust `checked_*`、UBSan |

**HFT 环形 buffer / 序号：** `uint32_t` 模 2³² 时，不能简单 `a < b` 若跨 wrap — 用 **序列空间 half-range**（同 TCP 序号比较）。
