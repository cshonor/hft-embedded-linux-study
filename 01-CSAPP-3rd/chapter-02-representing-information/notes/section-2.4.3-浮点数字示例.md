## 2.4.3 数字示例

> **Ch2 §2.4** · [章导读](../README.md) · 上节 [§2.4.2 IEEE 754](./section-2.4.2-IEEE754浮点表示.md) · 下节 [§2.4.4 舍入](./section-2.4.4-舍入.md)

> **怎么读例子：** 先写成 `±1.M × 2^E`，再填 **S / EXP(=E+127) / frac(=M)**。字段含义 → [§2.4.2](./section-2.4.2-IEEE754浮点表示.md)

---

### 例 1：`2.0f`（正、规格化、尾数全 0）

```text
2 = 1.0 × 2^1
  → S=0, E=1, EXP=128 = 10000000₂, M=0
```

```text
0 10000000 00000000000000000000000
S   EXP              frac
```

**配合：** 符号正；指数把小数点定在「×2」；尾数没有额外小数位。

---

### 例 2：`0.5f`（E = −1）

```text
0.5 = 1.0 × 2^(-1)
  → EXP = -1 + 127 = 126 = 01111110₂, frac=0
```

```text
0 01111110 00000000000000000000000
```

---

### 例 3：`−5.75f`（负、尾数有内容）

```text
5.75 = 101.11₂ = 1.0111₂ × 2^2
```

| 字段 | 值 |
|------|-----|
| S | `1` |
| E | `2` → EXP=`129`=`10000001` |
| frac | `0111` 后补 0 → `0111000…0` |

```text
1 10000001 01110000000000000000000
S   EXP            frac
```

**验算：** `(-1) × (1 + 1/4 + 1/8 + 1/16) × 2^2 = -1.4375 × 4 = -5.75` ✓

### 例 4：特殊值（对照）

| 值 | 位模式直觉（float） |
|----|---------------------|
| `+0` | `0 00000000 000…0` |
| `-0` | `1 00000000 000…0` |
| `+Inf` | `0 11111111 000…0` |
| `-Inf` | `1 11111111 000…0` |
| NaN | `? 11111111` + frac **非全 0** |

### 例 5：非规格化数——`1.4e-40f` 附近的极小值

当 EXP 全 0 时，规则改变：**没有隐含的 1**，指数固定为 `E = 1 - bias = -126`（float）。

```text
V = (-1)^S × 0.M × 2^(-126)
```

**例：最小正非规格化 float（frac 最低位为 1，其余 0）**

```text
M = 2^(-23)
V = 2^(-23) × 2^(-126) = 2^(-149) ≈ 1.4 × 10^(-45)
```

```text
0 00000000 00000000000000000000001
S   EXP              frac
```

**例：`frac = 000...010`（第 2 位为 1）**

```text
V = 2^(-22) × 2^(-126) = 2^(-148) ≈ 2.8 × 10^(-45)
```

> **非规格化的意义：** 填补了 0 和最小规格化数之间的「空洞」，让浮点能 **渐近下溢 (gradual underflow)** 而不是突然跳到 0。HFT 里算极小概率/极小价差时可能碰到。

**对照：最小正规格化 float**

```text
V = 1.0 × 2^(-126) ≈ 1.18 × 10^(-38)
```

```text
0 00000001 00000000000000000000000
S   EXP              frac
```

EXP=1 → `E = 1 - 127 = -126`，隐含 1 在。这是规格化和非规格化的 **分水岭**。

### 例 6：需要舍入的数——`0.1f` 的真实存储

`0.1` 在二进制里是无限循环 `0.0` + 循环节 `0011`（→ [§2.4.1](./section-2.4.1-分数二进制数.md)），存进 float 必须舍入。

**推导：**

```text
0.1₁₀ = 1.1(0011循环) × 2^(-4)
```

- `E = -4` → EXP = `-4 + 127 = 123 = 01111011₂`
- 尾数无限循环，取前 23 位后看第 24 位决定舍入：

```text
1.10011001100110011001100 | 1100...   （| 后是要舍入的多余位）
  ↑–––– 23 位 frac ––––––↑   ↑第24位=1, 且后面非全0 → 进一
```

舍入后尾数末位进一：

```text
0 01111011 10011001100110011001101
S   EXP              frac
```

> 实际 `0.1f` 的位模式正是 `3DCCCCCD`——注意末尾是 `D`（`1101`）而不是 `C`（`1100`），就是因为 **向偶数舍入进了 1**。这就是 `0.1 + 0.2 != 0.3` 的根因。

**验算反推：** frac = `10011001100110011001101`

```text
M = 1 + 1/2 + 1/16 + 1/32 + … ≈ 1.6000000238…
V ≈ 1.6000000238 × 2^(-4) ≈ 0.10000000149…
```

比真正的 0.1 大了约 `1.5 × 10^(-9)`——这就是 float 的表示误差。

### 例 7：从位模式反推十进制

给定 32 位模式 `0x40490FDB`，还原成十进制 float：

**第一步：转二进制，切三段**

```text
0x40490FDB = 0100 0000 0100 1001 0000 1111 1101 1011

S=0  EXP=10000000  frac=10010010000111111011011
```

**第二步：算指数**

```text
EXP = 128,  E = 128 - 127 = 1
```

**第三步：拼尾数**

```text
M = 1.10010010000111111011011₂
```

**第四步：算值**

```text
V = +1.M × 2^1 = 11.0010010000111111011011₂
```

整数部分 `11` = 3，小数部分约 `0.1415926...` → `V ≈ 3.1415927`——**这就是 `π` 的 float 近似值**。

> 也可以用 `printf` 验证：C 里 `0x40490FDB` 解释为 float 正是 `3.14159274`。注意它 **不等于** 数学上的 π，只是最近的可表示 float。

### 例 8：精度极限——`2^24 + 1` 存不进去

float 尾数 23 位 + 隐含 1 = **24 位有效**。超过 24 位的整数无法精确存储：

```c
int   x = 16777217;    /* 2^24 + 1 = 0x1000001，需要 25 位 */
float f = (float)x;    /* 0x4B800001 → 尾数不够，末位 1 被舍掉 */
int   y = (int)f;      /* y = 16777216，不是 16777217！ */
```

**推导：**

```text
2^24 + 1 = 1.000000000000000000000001₂ × 2^24
```

尾数需要 24 个小数位，但 float 只有 23 个 → 第 24 位 `1` 被舍掉（向偶数，保留位已是 0）→ 存成 `2^24`。

| 整数范围 | float 能精确存？ | 原因 |
|----------|-----------------|------|
| 0 ~ `2^24` (16777216) | **能** | ≤ 24 位有效 |
| `2^24+1` ~ `2^25` | **偶数能，奇数不能** | 每 2 个数丢 1 个 |
| `2^25+1` ~ `2^26` | 每 4 个存 1 个 | 有效位更不够 |
| double: 0 ~ `2^53` | **能** | 52+1=53 位有效 |

> **HFT 含义：** 用 `int32_t` 存订单号/时间戳时安全（≤ `2^31`）；但如果把 `int64` 纳秒时间戳转 `float` 就会丢精度——**必须用 double**（53 位有效，覆盖到 `2^53` ≈ 9×10¹⁵ 纳秒 ≈ 104 天）。

---

### 量级与精度（记数量级）

| | float | double |
|--|-------|--------|
| 有效十进制位（约） | **~7** | **~15–16** |
| 尾数字段 | 23 (+隐含1) | 52 (+隐含1) |
| 精确整数范围 | 0 ~ `2^24` | 0 ~ `2^53` |
| 最大规格化 | ~`3.4 × 10^38` | ~`1.8 × 10^308` |
| 最小正规格化 | ~`1.2 × 10^(-38)` | ~`2.2 × 10^(-308)` |
| 最小正非规格化 | ~`1.4 × 10^(-45)` | ~`4.9 × 10^(-324)` |
| 上溢 / 下溢 | → Inf / →0（或非规格） | 同逻辑 |

超过有效位数 → **不能精确存**（回到 [§2.4.1](./section-2.4.1-分数二进制数.md) 的 `0.1`）。

**HFT：** 看盘价用整数 tick；`float` 只适合对精度不敏感的统计/信号，别当金额真值。`int64` 纳秒时间戳要转浮点时 **必须用 double**。
