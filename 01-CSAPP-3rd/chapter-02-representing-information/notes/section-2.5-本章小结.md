### HFT 铁律汇总

| 铁律 | 原因 | 相关章节 |
|------|------|---------|
| wire 协议用 `int32_t`/`int64_t` 等固定宽度 | 避免跨平台宽度差异 | §2.1.2 |
| 整数字段显式 `htons`/`htonl` | 字节序不能假设 | §2.1.3 |
| 定长字段用 `memcmp`，变长先读 length | 避免 `\0` 和越界问题 | §2.1.4 |
| 价格/PnL 用 **整数 tick**，不用 float/double | 浮点舍入误差会累积成真金白银 | §2.4.4 |
| 禁止 `a == b` 比较浮点 | 舍入让应该相等的值差 1 ULP | §2.4.4 |
| 累加大量小数用 Kahan 求和或整数定点 | 普通连加大数吞小数 | §2.4.5 |
| `int64` 纳秒时间戳转浮点必须用 double | float 只有 24 位有效，存不下 | §2.4.3 |
| 移位前检查移位量范围 | ≥位宽是 UB | §2.1.9 |
| 热路径避免 `imul`，用移位+加减替代 | imul 延迟 3-4 周期 vs lea/add 1 周期 | §2.3 |
| 统一 SSE/AVX，避免 x87 扩展精度 | 防止不同优化级别结果不一致 | §2.4.4 |

### 易错点清单

1. **`sizeof` vs `strlen`：** `sizeof("AB")` = 3（含 `\0`），`strlen("AB")` = 2
2. **`'0'` vs `0`：** 字符 `'0'` = 48，数字 0 = 0
3. **`&` vs `&&`：** 位运算逐位算，逻辑运算整体判短路
4. **`>>` 对负数：** 有符号负数右移是实现定义，别依赖
5. **`<< 32` 是 UB：** 移位量不能 ≥ 类型位宽
6. **`x * -1` 不等于 `-x`：** `INT_MIN * -1` 是 UB，`-INT_MIN` 也是 UB
7. **`0.1 + 0.2 != 0.3`：** 不是 bug，是二进制浮点的表示极限
8. **`2^24 + 1` 存不进 float：** float 只有 24 位有效
9. **`-7 / 2` = -3，`-7 >> 1` = -4：** 向零取整 vs 向下取整不一致
10. **`strncpy` 不保证补 `\0`：** 需手动 `dst[n-1] = '\0'`
