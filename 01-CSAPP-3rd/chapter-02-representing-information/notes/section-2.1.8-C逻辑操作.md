## 2.1.8 C 语言中的逻辑操作

> **Ch2 §2.1** · [章导读](../README.md) · 上节 [§2.1.7 位操作](./section-2.1.7-C位级操作.md) · 下节 [§2.1.9 移位](./section-2.1.9-C移位操作.md)

---

### 三个逻辑运算符

| 运算符 | 名称 | 结果 |
|--------|------|------|
| `&&` | 逻辑与 | 两个都「真」→ `1`，否则 `0` |
| `\|\|` | 逻辑或 | 至少一个「真」→ `1`，否则 `0` |
| `!` | 逻辑非 | 操作数「假」→ `1`，「真」→ `0` |

**关键区别于位运算（→ [§2.1.6](./section-2.1.6-布尔代数简介.md)）：**

| | 位运算 `& \| ~` | 逻辑运算 `&& \|\| !` |
|--|----------------|----------------------|
| 操作对象 | **每一位** 独立运算 | 整个数当 **0 / 非0** 判 |
| 结果 | 任意位模式 | **只能是 0 或 1** |
| 短路？ | **不短路**，两侧都算 | `&& \|\|` **短路** |

---

### C 的「真」与「假」

C 语言没有真正的布尔类型（C99 之前），用整数代替：

- **0 是假**（包括 `0`, `0.0`, `'\0'`, `NULL`）
- **任何非 0 值是真**（包括 `1`, `-1`, `0.5`, `'A'`, 非空指针）

```c
if (x)       /* 等价于 if (x != 0) */
if (!x)      /* 等价于 if (x == 0) */
if (!!x)     /* 把任意非0值「归一化」成 1 */
```

> `!!` 是常见技巧：`!!x` 保证结果是 `0` 或 `1`，不管 `x` 原来是什么非零值。Data Lab 里用来生成标准布尔值。

---

### 短路求值（重点）

`&&` 和 `||` **不总是计算右操作数**——这是逻辑运算和位运算最实用的区别：

| 表达式 | 左操作数为 | 右操作数 | 结果 |
|--------|-----------|----------|------|
| `A && B` | 假 (0) | **不计算** | `0` |
| `A \|\| B` | 真 (非0) | **不计算** | `1` |

**用途 1：防止空指针解引用**

```c
if (ptr != NULL && ptr->value > 0) { ... }
/* 如果 ptr 是 NULL，&& 左边为假，右边 ptr->value 不执行 → 不会崩溃 */
```

如果写成 `if (ptr->value > 0 & ptr != NULL)`（位与，不短路）→ 先解引用 NULL → **segfault**。

**用途 2：避免除零**

```c
if (denom != 0 && num / denom > threshold) { ... }
/* denom 为 0 时不会执行 num/denom → 不会浮点异常 */
```

**用途 3：先检查再访问**

```c
if (i < n && arr[i] == target) { ... }
/* i 越界时不执行 arr[i] → 不会越界读 */
```

> **易错：** 如果右边有 **副作用**（`++`、函数调用、赋值），短路会改变行为：
> ```c
> if (a > 0 && b++ > 5)   /* a<=0 时 b 不自增 */
> if (a > 0 &  b++ > 5)   /* 位与不短路，b 总是自增 */
> ```

---

### `&&` vs `&` 完整对比

```c
int x = 6, y = 3;

x & y    /* 位与：0110 & 0011 = 0010 = 2 */
x && y   /* 逻辑与：6非0 && 3非0 = 1       */

x & 0    /* = 0（所有位清零） */
x && 0   /* = 0（右边假） */

x & 1    /* = 0（看最低位：6 的最低位是 0） */
x && 1   /* = 1（x 非0 且 1 非0） */
```

```c
if (x & y)   /* 位与：结果 2，非0 → 真。检查「有没有公共位为1」 */
if (x && y)  /* 逻辑与：x、y 都非0 → 真。检查「两个都非0」       */
```

> **判断条件用 `&&`，操作位用 `&`。** `if (flag & MASK)` 是位运算的经典用法——「flag 的某些位是否为 1」，这里 `&` 结果非0即真，和 `&&` 语义恰好重合，但 **不要** 因此混用。

---

### `!` 运算符

```c
!0     /* = 1 */
!5     /* = 0 */
!-3    /* = 0（非0都是真，取反得0） */
!(x == 0)  /* 等价于 x != 0 */
```

**常见惯用法：**

```c
if (!done)        /* 还没完成 */
if (!strcmp(a,b)) /* strcmp 返回 0 表示相等 → !0 = 1 → 「两字符串相等」 */
```

> `!strcmp(a, b)` 比 `strcmp(a, b) == 0` 更简洁，老代码常见。

---

### 条件赋值惯用法

逻辑运算的短路特性可以用来做 **条件赋值**（不推荐生产用，但读老代码要会）：

```c
/* 等价于：if (a) x = b; */
a && (x = b);     /* a 为真才执行 x = b */

/* 等价于：if (!a) x = b; */
a || (x = b);     /* a 为假才执行 x = b */
```

> 现代代码用三元 `x = a ? b : x;` 或 `if` 更清晰。但 Data Lab 受限运算符时这类技巧有用。

---

### HFT 视角

- **条件判断优先 `&&`/`||`：** 短路能省一次内存访问（如空指针检查）
- **位掩码判断用 `&`：** `if (flags & READY_MASK)` 比 `if (flags == READY_MASK)` 灵活（只检查特定位）
- **别在热路径用 `!strcmp`：** 字符串比较慢，定长字段用 `memcmp` 或整数比较（→ [§2.1.4](./section-2.1.4-字符串表示.md)）
