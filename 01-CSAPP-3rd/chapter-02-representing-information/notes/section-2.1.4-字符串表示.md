## 2.1.4 字符串表示

### C 字符串的本质

C 字符串 = **字节序列 + 末尾 `'\0'`（0x00）**。`\0` 是 **终止符**，告诉 `strlen`/`printf` 等函数字符串在哪里结束。

```c
char s[] = "AB";
/* 内存：41 42 00    ← 'A' 'B' '\0' */
/* strlen(s) = 2（不算 \0） */
/* sizeof(s) = 3（算 \0） */
```

> **strlen vs sizeof：** `strlen` 数到 `\0` 为止（运行时算）；`sizeof` 是编译时的大小（数组大小包含 `\0`）。指针 `char *p = "AB"; sizeof(p) = 8`（64 位指针大小），不是字符串长度。

---

### ASCII 码

ASCII 是 7 位编码（0-127），覆盖英文大小写、数字、标点、控制字符：

| 字符范围 | ASCII 码 | hex |
|----------|----------|-----|
| `'0'` - `'9'` | 48 - 57 | 0x30 - 0x39 |
| `'A'` - `'Z'` | 65 - 90 | 0x41 - 0x5A |
| `'a'` - `'z'` | 97 - 122 | 0x61 - 0x7A |
| `'\0'` | 0 | 0x00 |
| `'\n'` | 10 | 0x0A |
| `' '` (空格) | 32 | 0x20 |

> **易错：** 字符 `'0'` 的值是 48 (0x30)，**不是** 数字 0。`'0' - '9'` 转数字用 `c - '0'`（= `c - 48`）；数字转字符用 `n + '0'`。

**大小写转换：** 同一个字母大小写差 32 (0x20)：

```c
'A' + 32 == 'a'     /* 65 + 32 = 97 */
'a' - 32 == 'A'
/* 或用位运算（第 5 位控制大小写）*/
'Z' | 0x20 == 'z'   /* 置第5位 → 小写 */
'z' & ~0x20 == 'Z'  /* 清第5位 → 大写 */
```

> Data Lab 的 `toLowerCase` 类题目用 `| 0x20` 比 `+ 32` 更优雅。

---

### UTF-8：变长编码

UTF-8 兼容 ASCII（0-127 的字节和 ASCII 完全一样），但用 **1-4 字节** 编码所有 Unicode：

| 字符范围 | 字节数 | 首字节模式 | 示例 |
|----------|--------|-----------|------|
| U+0000 - U+007F | 1 | `0xxxxxxx` | `'A'` = 41 |
| U+0080 - U+07FF | 2 | `110xxxxx 10xxxxxx` | `'é'` = C3 A9 |
| U+0800 - U+FFFF | 3 | `1110xxxx ...` | `'中'` = E4 B8 AD |
| U+10000+ | 4 | `11110xxx ...` | emoji |

> **好处：** 纯英文的 UTF-8 = ASCII，零迁移成本；`strlen` 数的是 **字节数** 不是字符数（中文一个字占 3 字节）。

**HFT 坑：** 协议里 symbol 如果允许中文/Unicode，`strlen` 给的是字节数不是字符数；定长字段按字节算。

---

### 字符串没有大小端

> **普通 `char` 字符串没有大小端问题。** 每个字符 1 字节，按书写顺序低→高地址存放（`"AB"` → `41 42`），与 CPU 的 LE/BE 无关。
>
> 只有 **多字节整数**（`int`/`short`/`long`）和宽字符（`wchar_t`）才谈 endian。细节 → [§2.1.3 · 字符串无大小端](./section-2.1.3-寻址与字节序.md#字符串没有大小端只有多字节整数才有)

```c
char s[] = "1234";
/* 所有 CPU 上都是：31 32 33 34 00 */
/* 这不是 big-endian —— 字符串天然无大小端 */
```

### 二进制协议中的字符串

HFT 协议（如 binary 行情、FIX binary）通常 **不用 C 字符串**：

| 特征 | C 字符串 | 二进制协议字段 |
|------|---------|---------------|
| 终止符 | `\0` 结尾 | **无** `\0` |
| 长度 | 靠 `\0` 找 | **定长** 或 `length + payload` |
| Symbol/账号 | 不定长 | 常定长 8/16 字节，空格或 `\0` 填充 |
| 字节序 | 无 | 整数字段需 `htons`/`htonl` |

**解析协议字符串的安全做法：**

```c
/* 定长字段：用 memcmp + 已知长度 */
if (memcmp(buf + offset, "AAPL", 4) == 0) { ... }

/* 变长字段：先读 length，再读 payload */
uint16_t len = ntohs(*(uint16_t*)(buf + offset));
char symbol[16];
memcpy(symbol, buf + offset + 2, len);
symbol[len] = '\0';   /* 手动加终止符，才能用 strcmp */
```

> **HFT 铁律：** 用 `strcmp` 前确认字符串以 `\0` 结尾；定长字段用 `memcmp` + 已知长度，更快也更安全。

---

### 常见字符串函数与坑

| 函数 | 作用 | 坑 |
|------|------|----|
| `strlen(s)` | 数到 `\0` 的长度 | 没终止符 → 越界读 |
| `strcmp(a,b)` | 逐字节比较到 `\0` | 慢；没终止符越界 |
| `strcpy(dst,src)` | 复制到 `\0` | 不检查 dst 大小 → 溢出 |
| `strncpy(dst,src,n)` | 复制最多 n 字节 | **不保证** 补 `\0` |
| `memcpy(dst,src,n)` | 复制 n 字节 | 需已知长度，不看 `\0` |
| `memcmp(a,b,n)` | 比较 n 字节 | 定长字段首选 |

> **生产代码：** 用 `strncpy` 后手动补 `\0`；或用 `snprintf`；热路径定长字段直接 `memcmp`。

### 口述巩固 · 自测

1. `"1234"` 在内存里是 `31 32 33 34`，这说明字符串是 big-endian 吗？（答：否 — **字符串无大小端**）
2. 字符 `'0'` 的 ASCII 码是多少？和数字 0 有什么区别？怎么互转？
3. `'A'` 和 `'a'` 的 ASCII 码差多少？用位运算怎么转大小写？
4. `strlen("AB")` 和 `sizeof("AB")` 各是多少？为什么不同？
5. 二进制协议的字符串字段为什么不用 C 字符串？怎么安全解析？
