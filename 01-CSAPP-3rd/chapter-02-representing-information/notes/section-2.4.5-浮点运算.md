## 2.4.5 浮点运算

> **Ch2 §2.4** · [章导读](../README.md) · 上节 [§2.4.4 舍入](./section-2.4.4-舍入.md) · 下节 [§2.4.6 C 浮点](./section-2.4.6-C语言浮点数.md)

> **一条线：** 浮点运算是 **近似算术**（对阶丢位 + 舍入）→ 非结合 → HFT 账本/热路径优先整数。  
> float / double **流程相同**，只是尾数/阶码位数不同。数学形式：\(V = (−1)^s × M × 2^E\)。

---

### 加减法（最复杂 · 核心：对阶）

1. 取出两数的 **符号、尾数、指数**  
2. **对阶：** 指数小的那个，尾数 **右移**，对齐到较大指数（右移会丢低位 → 精度损失）  
3. 尾数做二进制 **加 / 减**（减可化成补码加）  
4. **规格化：** 调小数点，使尾数满足 \(1 ≤ M < 2\)（规格化），同步改指数  
5. **舍入：** 超出 23（float）/ 52（double）的多余位按规则丢掉 → [§2.4.4](./section-2.4.4-舍入.md)  
6. 判定特殊值：0 / Inf / NaN

**对阶直觉：** \(1.0×2^3 + 1.0×2^0\) → 后者右移 3 位成 \(0.001×2^3\)，再同阶相加。

**例：`0.5 + 1.25`**

| | 规格化写法 |
|--|------------|
| 0.5 | \(1.0 × 2^{-1}\) |
| 1.25 | \(1.01 × 2^{0}\) |

对阶：0.5 尾数右移 1 → \(0.10 × 2^{0}\)  
尾数加：\(0.10 + 1.01 = 1.11\)，指数 0  
→ \(1.11 × 2^{0} = 1.75\) ✓

---

### 乘法（不用对阶）

1. 符号 **异或**（同正异负）  
2. 指数 **相加**（再减 bias 等细节由硬件/实现处理）  
3. 尾数 **相乘**  
4. 规格化、舍入；看 Inf/NaN

**例：`2.0 × 4.0`** — \(1.0×2^1\) × \(1.0×2^2\) → 符号正，指数 \(1+2=3\)，尾数 \(1.0\) → \(1.0×2^3=8.0\)

---

### 除法

1. 符号异或  
2. 指数 **相减**（被除数 − 除数）  
3. 尾数相除  
4. 规格化、舍入  

特殊：除数为 0 → **±Inf**；`0/0` → **NaN**（→ [§2.4.2](./section-2.4.2-IEEE754浮点表示.md)）

---

### 非结合 · 大数吞小数

```c
float a = 1e30f, b = -1e30f, c = 1.0f;
(a + b) + c;   /* 0 + 1 = 1 */
a + (b + c);   /* 可能仍 ≈ 0 — 非结合 */
```

- **NaN 传播：** 操作数有 NaN → 结果常为 NaN；比较无序  
- **Inf** 有定义规则，仍易出 bug  

**HFT：** 多 venue partial fill 算 VWAP — 用 **整数分子/分母** 或 Kahan；别裸 `float` 连加 PnL。

3. 为什么 `(a+b)+c` 和 `a+(b+c)` 对 float 可能不同？  
4. 除零与 `0/0` 各得什么？
