## 2.2 整数表示（2.2.1–2.2.8）

### 2.2.1 整数数据类型

- C 标准只规定 **最小范围**，不固定位宽 — 用 `<stdint.h>` 的 `int32_t` / `uint64_t` 写可移植协议与逻辑

### 2.2.2 无符号编码

**w 位无符号：** 取值 0 … 2^w − 1，算术 **模 2^w**（溢出丢弃高位）

### 2.2.3 补码编码 (Two's Complement)

**w 位有符号：** 最高位为符号位，范围 **−2^(w−1) … 2^(w−1) − 1**

| 模式（8 bit 例） | 值 |
|------------------|-----|
| 00000000 | 0 |
| 01111111 | 127 |
| 10000000 | −128 (INT8_MIN) |
| 11111111 | −1 |

- **−x** = **~x + 1**（按 w 位算）
- **唯一不对称：** INT_MIN 没有正数对应（8 位 −128）

### 2.2.4–2.2.5 有符号 / 无符号转换与 C 中的坑

**位模式不变，解释变：**

```c
int x = -1;           // 全 1 位模式
unsigned u = x;       // 大正数 2^32-1（在 32 位 word 上）
if (x < u) ...        // false！x 转成 unsigned 再比
```

**C 规则摘要：**

- 混合运算时 **有符号常转无符号** 很常见 → 负值变巨大正数
- `sizeof` 返回 `size_t`（无符号）— 与 `-1` 比大小陷阱

**HFT：** 循环 `for (int i = n-1; i >= 0; i--)` 若 `n` 为 0 且 `i` 无符号化 → 死循环；订单 count、索引用 **`size_t` 与有符号混用前先 cast 策略**。

### 2.2.6 扩展位表示

| 转换 | 规则 |
|------|------|
| 无符号 **零扩展** | 高位补 0 |
| 有符号 **符号扩展** | 高位补符号位 |

`int8_t → int32_t`：−1 (0xFF) → 0xFFFFFFFF

### 2.2.7 截断

- 宽类型 → 窄类型：**丢弃高位**
- 有符号截断：实现定义多，**别依赖**

**HFT：** 协议 `int16` 数量读入后赋给 `int` 通常安全；反向写入要检查 **范围**。

### 2.2.8 关于有符号与无符号的建议

- **默认用有符号** 表示可能为负的量（价差、库存）
- **无符号** 用于位模式、大小、掩码 — **不要用于「可能减到负数」的算术**
- 需要确定性宽度 → **`int32_t` / `uint64_t`**
