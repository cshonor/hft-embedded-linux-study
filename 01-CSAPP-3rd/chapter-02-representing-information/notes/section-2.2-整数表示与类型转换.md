## 2.2 整数表示（2.2.1–2.2.8）

> **本节一条线：** 同一段 bit 可解释成无符号/有符号 → 转换时 **位不变、含义变** → 扩展/截断规则。

---

### 2.2.1 整数数据类型

- C 标准只规定 **最小范围**，**不** 固定位宽
- 可移植写法：`<stdint.h>` 的 `int8_t` / `int32_t` / `uint64_t` 等（与 §2.1.2 ABI 表配合使用）
- `char` 是否 signed 由实现定义 — 位运算常用 `unsigned char`

---

### 2.2.2 无符号编码

**w 位无符号：** 取值 **0 … 2^w − 1**，算术 **模 2^w**（溢出丢弃高位，无「溢出异常」）

```
8 位: 255 + 1 → 0
```

---

### 2.2.3 补码编码 (Two's Complement)

**w 位有符号：** 最高位为符号位，范围 **−2^(w−1) … 2^(w−1) − 1**

| 8 bit 模式 | 值 |
|------------|-----|
| 00000000 | 0 |
| 01111111 | 127 |
| 10000000 | −128 (INT8_MIN) |
| 11111111 | −1 |

#### 求负：`−x = ~x + 1`（**所有位**取反再加 1）

**不是**「只把符号位翻一下」。硬件/公式就是：**整段位模式按位取反，再加 1**。

**例：从 `+1` 得到补码里的 `−1`（8 位）**

| 步骤 | 位模式 | 说明 |
|------|--------|------|
| 原值 `+1` | `00000001` | |
| `~` 全位取反 | `11111110` | 每一位都翻，不是只翻最高位 |
| `+1` | `11111111` | 正好是补码 **−1**（上表） |

```text
00000001  →  ~  →  11111110  →  +1  →  11111111   (= −1)
```

**易混：原码 → 补码（教材另一条线）**

8 位 **原码** 的 `−1` 常写成 `10000001`（符号 1 + 数值 `1`）。  
转补码的习惯口诀是：**符号位不动，其余位取反得反码，再加 1** → `11111110` → `11111111`。  
若对原码 **8 位全部** `~` 再 `+1`：`10000001` → `01111110` → `01111111`（=+127）— **对不上 −1**，别和「`−x = ~x + 1`」混成同一步。

- **INT_MIN 不对称：** 8 位 −128 取负 **仍是自己**（无对应正数）

**记忆：** `11111111` 是 −1，不是「全 1 的最大无符号数」— **看类型解释**。

---

### 2.2.4–2.2.5 有符号 / 无符号转换与 C 中的坑

**核心：位模式不变，解释变。**

```c
int x = -1;              /* 32 位下位模式 0xFFFFFFFF */
unsigned u = x;          /* u = 4294967295 */
if (x < u) { ... }       /* 假！x 被转成 unsigned 再比较 */
```

**`-1 < 0U` 为假** — 经典笔试/UB 温床。

**C 规则摘要：**

- 混合运算常把有符号 **提升为无符号**（同宽度）→ 负数变巨大正数
- **`sizeof` 的结果类型是无符号的 `size_t`**（标准 C 写死）— 拿它和有符号负数比大小，正好踩中上面这条

#### `sizeof` → `size_t`（无符号）：别拿负数去比

```c
int n = -1;
if (n < sizeof(buf)) {   /* n 被提升成 size_t → 变成超大无符号数 */
    /* 本意「负数小于长度」常为真；实际判断常反过来 / 跑偏 */
}
```

| 你以为 | 实际 |
|--------|------|
| `−1` 很小，一定 `< sizeof(...)` | `−1` 先变成 `SIZE_MAX` 一类巨数，再和 `size_t` 比 |
| 分支按「有符号直觉」走 | 逻辑 **直接跑偏** |

**原则（HFT 尤甚）：有符号数跟无符号数尽量不要混比较、少靠隐式转换。**  
需要比大小时：先统一成同一种符号语义（两端都 `size_t` 且保证非负，或两端都有符号且先校验范围），**别靠编译器悄悄转换**。

经典坑：`−1` 和某个 `size_t`（含 `sizeof`）比 → 判断反过来。热路径里若因此错进/漏进订单分支 → **错单、漏单**，损失可以很大；能不混就别混。

**HFT：**

```c
/* 危险：n 为 0 时 i 无符号下溢 */
for (unsigned i = n - 1; i < n; i--)

/* 订单 count、索引：统一 size_t 或有符号策略，别隐式混用 */
/* 长度比较：先保证 n >= 0，再用 (size_t)n 与 sizeof/len 比 */
```

---

### 2.2.6 扩展位表示

| 转换 | 规则 | 8→32 位例 |
|------|------|-----------|
| 无符号 **零扩展** | 高位补 0 | `0xFF` → `0x000000FF` |
| 有符号 **符号扩展** | 高位补符号位 | `0xFF` (−1) → `0xFFFFFFFF` |

协议解析：`int8_t` 读入后赋给 `int32_t` 会符号扩展 — 通常符合预期；**无符号** 字段别用有符号中间变量。

---

### 2.2.7 截断

- 宽 → 窄：**丢弃高位**，按目标类型 **重新解释**
- 有符号截断：实现定义多 — **别依赖**

```c
int x = 520;           /* 0x00000208 */
char c = x;            /* 截断：常得 8 (0x08) */
```

**HFT：** wire `int16` 数量 → 本地 `int32` 通常安全；写入窄字段前 **检查范围**。

---

### 2.2.8 关于有符号与无符号的建议

| 场景 | 建议 |
|------|------|
| 价差、库存（可负） | 有符号 + 足够宽度 |
| 序号、掩码、size | 无符号 |
| 可能减到负的计数 | **不要** 用 unsigned |
| 与 `sizeof` / `size_t` 比大小 | **先保证非负**，再统一类型；**禁止** 有符号负数直接比 |
| 协议/on-wire | **固定宽度** `int32_t` / `uint64_t` |
