## 2.4.2 IEEE 754 浮点表示

> **Ch2 §2.4** · [章导读](../README.md) · 上节 [§2.4.1 分数二进制](./section-2.4.1-分数二进制数.md) · 下节 [§2.4.3 数字示例](./section-2.4.3-浮点数字示例.md)

---

**float（32 bit）：**

```
| S (1) | exp (8) | frac (23) |
```

**double（64 bit）：** exp 11 位，frac 52 位，bias **1023**

### `exp`：不是补码，也不是「带符号整数」

IEEE 754 的 **阶码字段用无符号偏移编码 (biased / excess)**：

\[
\text{存进去的 } exp = E + bias
\]

| 格式 | `exp` 位数 | bias | 真实指数 \(E\)（规格化时） |
|------|------------|------|---------------------------|
| float 32 | 8 | **127** | \(E = exp - 127\) |
| double 64 | 11 | **1023** | \(E = exp - 1023\) |

- **不是** 补码，也 **不是** 单独再挂一个符号位的有符号数  
- 存的是 **无符号整数**（把真实指数平移到非负区间）  
- **好处：** 浮点大小常可先比符号，再 **直接按无符号整数比 `exp`（再比 `frac`）** — 硬件/软件排序快

规格化数：

\[
V = (−1)^S × 1.M × 2^{E},\quad E = exp - bias
\]

（`exp` 既非全 0 也非全 1）

---

### 特殊值：`Inf` 与 `NaN`

| 类别 | exp | frac | 含义 |
|------|-----|------|------|
| 规格化 | 中间值 | 任意 | 正常数 |
| 非规格化 | 0 | ≠0 | 接近 0 |
| 零 | 0 | 0 | ±0（看符号位） |
| **Inf** | **全 1** | **全 0** | 无穷大 |
| **NaN** | **全 1** | **≠0** | Not a Number（不是数字） |

**Inf（Infinity）：** 溢出、`1.0/0.0` 等；有 **正/负**（看符号位 `S`）：`1.0/0.0` → `+Inf`，`-1.0/0.0` → `−Inf`。  
**NaN：** **不是** 真数值，专门标记 **非法/无意义** 运算：`0/0`、`sqrt(-1)`、`(+Inf)-(+Inf)` 等。  
float / double / 更长 IEEE 浮点 **都有** NaN（不是只出现在某一种宽度里）。口误成「long」时，说的就是 **NaN**。

**位模式怎么区分 NaN 和 Inf：** 都是 `exp` 全 1；`frac` 全 0 → **Inf**；`frac` 非 0 → **NaN**。

---

### 冷知识（CSAPP 常一带而过）

| 现象 | 说明 |
|------|------|
| `NaN == NaN` 为 **假** | NaN 与 **任何** 浮点值比较（含自己）都不相等 → C 里用 `isnan()` 判断 |
| `+Inf + 正数` 仍 `+Inf` | 无穷吸收有限正增量 |
| `(+Inf) - (+Inf)` → **NaN** | 不定式，非法 |

```c
#include <math.h>
if (isnan(x)) { /* … */ }
```

**HFT：** 行情/风控路径遇 NaN 要显式拦截；别靠 `==` 抓 NaN。金额仍优先定点/整数 tick。
