## 2.4.2 IEEE 754 浮点表示

> **Ch2 §2.4** · [章导读](../README.md) · 上节 [§2.4.1 分数二进制](./section-2.4.1-分数二进制数.md) · 下节 [§2.4.3 数字示例](./section-2.4.3-浮点数字示例.md)

---

**float（32 bit）拆分：**

```
| S (1) | EXP / exp (8) | frac (23) |   = 32 bit
```

- **S**：符号  
- **EXP**：专门存 **偏移后的指数** 的 8 个二进制位（下文详表）  
- **frac**：尾数  

**double（64 bit）：** EXP **11** 位，frac 52 位，bias **1023** — 逻辑与 float **一模一样**，只是位数更大、能表示更大范围。

### `EXP`：不是补码，是偏移码（float bias = 127）

\[
\text{真实数学指数 } E = EXP - 127 \quad（规格化时）
\]
\[
\text{存进去的 } EXP = E + 127
\]

- **不是** 补码，也 **不是** 普通带符号整数  
- 8 位 **无符号**，取值 **0…255**  
- **好处：** 硬件可用无符号整数比较逻辑更快判断浮点大小  

| 格式 | EXP 位数 | bias | 规格化时 \(E\) |
|------|----------|------|----------------|
| float 32 | 8 | **127** | \(EXP - 127\) |
| double 64 | 11 | **1023** | \(EXP - 1023\) |

#### 四种 `EXP` 区间 → 浮点四种形态（float）

| EXP（十进制） | 二进制直觉 | 形态 |
|---------------|------------|------|
| **0** | `00000000` | `frac=0` → **±0**；`frac≠0` → **非规格化**（极小值，缓冲下溢） |
| **1…254** | 正常区间 | **规格化数**（日常小数/整数）；\(E = EXP - 127\) |
| **255** | `11111111` | `frac=0` → **Inf**；`frac≠0` → **NaN** |

规格化举例：`EXP=127` → \(E=0\)；`EXP=130` → \(E=3\)。

规格化数值：

\[
V = (−1)^S × 1.M × 2^{E},\quad E = EXP - 127
\]

#### 直观例：`2.0` 的 float

\[
2 = 1.0 × 2^{1} \Rightarrow E=1,\quad EXP = 1+127 = 128 = 10000000_2
\]

规格化隐含整数位 `1.`，尾数 `M` 全 0 → `frac` 23 位全 0：

```text
2.0f:  0 10000000 00000000000000000000000
       S   EXP           frac (23)
```

#### 8 位 EXP 核心作用（三句话）

1. 存偏移后的指数，控制小数点左移/右移（缩放）  
2. 用数值区间区分：**零 / 非规格 / 规格化 / Inf·NaN**  
3. 无符号存储，方便硬件比大小  

### `frac`（23 位尾数）：存有效数字

规格化时数值是 \(1.M × 2^{E}\)，**`M` 就来自这 23 位**（前面的 `1.` 是隐含位，不占存储）。

| 字段 | 干什么 |
|------|--------|
| **S（1）** | 正负 |
| **EXP（8）** | 数量级 / 小数点挪多远（缩放范围） |
| **frac（23）** | **精准描述** 有效数字（尾数细节） |

三者拼在一起才是一个完整 float；double 同理，只是 `frac` 变成 **52** 位（精度更高）。

#### 再举一个：`−5.75` 怎么配合

\[
5.75 = 101.11_2 = 1.0111_2 × 2^{2}
\]

| | |
|--|--|
| S | `1`（负） |
| E | `2` → `EXP = 2+127 = 129 = 10000001₂` |
| M / frac | `0111000…0`（`1.` 后面的 `0111`，其余补 0） |

```text
-5.75f:  1 10000001 01110000000000000000000
         S   EXP            frac
```

→ 更多数值拆解：[§2.4.3](./section-2.4.3-浮点数字示例.md)

### 特殊值：`Inf` 与 `NaN`（float / double 同一套规则）

**通用：** `exp` 全 1 = 特殊值  

1. 尾数（`frac`）**全 0** → **Inf**  
2. 尾数 **不全 0** → **NaN**

| 类别 | exp | frac | 含义 |
|------|-----|------|------|
| 规格化 | 中间值 | 任意 | 正常数 |
| 非规格化 | 0 | ≠0 | 接近 0 |
| 零 | 0 | 0 | ±0（看符号位） |
| **Inf** | **全 1** | **全 0** | 无穷大 |
| **NaN** | **全 1** | **≠0** | Not a Number（不是数字） |

float / double / 更长 IEEE 浮点 **都有**；口误成「long」时，说的就是 **NaN**。

#### float 32 位位模式例

```text
+Inf:  0 11111111 00000000000000000000000
-Inf:  1 11111111 00000000000000000000000
NaN:   0 11111111 00010000000000000000000   /* frac 任意非全 0 */
```

#### Inf 运算例

```c
#include <stdio.h>
#include <math.h>

double pos_inf = 1.0 / 0.0;    /* +Inf */
double neg_inf = -1.0 / 0.0;   /* -Inf */

pos_inf + 100;                 /* +Inf */
pos_inf * 5;                   /* +Inf */
pos_inf - 999;                 /* +Inf */
pos_inf + neg_inf;             /* NaN：无穷±无穷无意义 */
pos_inf * neg_inf;             /* -Inf */
```

#### NaN 产生场景（非法 / 无实数解）

```c
sqrt(-5.0);    /* 负数开方 → NaN */
0.0 / 0.0;     /* → NaN */
pos_inf - pos_inf;  /* Inf−Inf → NaN */
pos_inf * 0.0; /* Inf×0 → NaN */
log(-1.0);     /* → NaN */
```

#### 最反直觉：`NaN != NaN`

```c
double n = sqrt(-1.0);
n == n;        /* 假！NaN 不等于任何值，包括自己 */
isnan(n);      /* 真 — 必须用库函数判断 */
```

#### 实用口诀

1. **除以 0** → 正/负无穷（看分子符号）  
2. **无实数解 / 不定式** → **NaN**  
3. 判断 NaN **不能** `==`，必须 **`isnan`**  
4. Inf 参与常规 ±×÷ 有限数 **仍常是 Inf**；`Inf±Inf`、`Inf×0` 等 → **NaN**

**HFT：** 行情/风控路径遇 NaN 要显式拦截；别靠 `==` 抓 NaN。金额仍优先定点/整数 tick。
