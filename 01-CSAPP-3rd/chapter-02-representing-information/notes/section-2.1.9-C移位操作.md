## 2.1.9 C 语言中的移位操作

### 左移 `<<`

**规则：** 每左移 1 位，高位丢弃，**低位补 0**。

```
0011 0110  << 1  →  0110 1100   （低位补0）
0011 0110  << 2  →  1101 1000   （补两个0）
```

**数学含义：**

- **无符号** `u << k` ≡ \(u \times 2^k \mod 2^w\)（超出宽度截断）
- **有符号** 正数左移可能 **变负**（符号位被挤进来）→ 溢出

```c
unsigned u = 5;   u << 1;   /* 10 */
unsigned v = 5;   v << 3;   /* 40 = 5×8 */
int      s = 5;   s << 1;   /* 10 */
int      t = 0x40000000; t << 1;  /* 符号位变1 → 变负数（溢出） */
```

> **左移对无符号和有符号的补位规则相同**——都是低位补 0。区别只在 **结果是否溢出**。

---

### 右移 `>>`——有两种，看类型

这是 C 移位最坑的地方：**右移行为依赖操作数类型**。

| 类型 | 右移方式 | 高位补什么 | 数学含义 |
|------|----------|-----------|----------|
| **无符号** `unsigned` | **逻辑右移** | 补 **0** | \(\lfloor u / 2^k \rfloor\) |
| **有符号** `int`（正数） | 算术右移 | 补 **0**（符号位是0） | \(\lfloor x / 2^k \rfloor\) |
| **有符号** `int`（负数） | **实现定义** | 补 0 或补 1 | 见下 |

```
逻辑右移（unsigned）：       算术右移（signed 负数）：
1010 0000 >> 2 → 0010 1000   1010 0000 >> 2 → 1110 1000
              ↑↑ 补0                        ↑↑ 补符号位1
```

**C 标准怎么说：** 有符号负数右移是 **实现定义 (implementation-defined)**——编译器可以选逻辑或算术。

**现实：** 几乎所有主流编译器（GCC、Clang、MSVC）在 x86/ARM 上都做 **算术右移**（补符号位），因为 CPU 的 `SAR` 指令就是这么干的。但 **不要依赖这个假设** 写可移植代码。

```c
unsigned u = 16u;   u >> 2;   /* 4，确定 */
int      s = -8;    s >> 1;   /* x86 gcc: -4（算术右移），但标准不保证 */
int      n = -8;    n >> 1;   /* 如果逻辑右移 → 2^31 - 4 = 2147483644 */
```

> **验证：** `INT_MIN >> 1` 在算术右移下仍是负数；逻辑右移下变巨大正数。写代码时如果依赖行为，加 `static_assert` 或注释说明。

### 移位量 ≥ 位宽：未定义行为 (UB)

```c
int x = 1;
x << 32;   /* UB! int 是 32 位，移 32 位超出范围 */
x << 31;   /* OK（虽然可能溢出，但不是 UB） */
x >> 32;   /* UB! */
```

**C 标准：** 移位量 ≥ 类型位宽、或为负数 → **未定义行为**。实际表现可能是：

- x86：取模（`<< 32` 变成 `<< 0`，即不动）→ **静默错误**
- ARM：可能清零
- 编译器优化时可能直接删掉

```c
int shift = 32;
int y = x << shift;   /* UB! 别这么写 */
int z = x << (shift % 32);  /* 安全，但不一定是你想要的语义 */
```

> **HFT 坑：** 从协议字段解析移位量时，**务必检查范围**：
> ```c
> if (shift < 0 || shift >= 32) return error;
> val = raw >> shift;
> ```

### 有符号左移溢出也是 UB

```c
int x = 0x40000000;   /* 2^30 */
x << 1;               /* 2^31 = INT_MIN，符号位翻转 */
x << 2;               /* UB! 左移溢出有符号数 */
```

C99：有符号数左移如果结果 **不可表示** 在该类型中 → UB。无符号数左移溢出则安全（模 \(2^w\)）。

**安全做法：** 需要移位操作时 **优先用无符号**：

```c
unsigned u = 0x40000000u;
u << 2;   /* OK，模 2^32 截断，无 UB */
```

### 移位实现乘除（强度削减）

编译器会把 `×2^k` 和 `/2^k` 优化成移位：

| 表达式 | 编译后 | 说明 |
|--------|--------|------|
| `x * 2` | `x << 1` | 左移1位 |
| `x * 8` | `x << 3` | 左移3位 |
| `x / 4` (unsigned) | `x >> 2` | 逻辑右移 |
| `x / 4` (signed, 正) | `x >> 2` | 算术右移 |
| `x * 7` | `(x << 3) - x` | `8x - x = 7x` |
| `x * 15` | `(x << 4) - x` | `16x - x = 15x` |

> **注意：** 有符号负数的 `/` 是 **向零取整**，而算术右移是 **向下取整**——两者对负数 **不一致**：
> ```c
> -7 / 2    /* = -3（向零取整） */
> -7 >> 1   /* = -4（向下取整，算术右移） */
> ```
> 所以编译器对 `signed / 2^k` 不能简单替换成 `>>`，要加修正。

### HFT 定点缩放实践

价格用 **整数 tick + 定点缩放** 时，移位是核心操作：

```c
/* 价格 = tick_count × tick_size，tick_size = 0.0001 */
/* 用定点：内部存 tick 整数，显示时除以 10000 */

int32_t price_ticks = 123456;        /* 内部存整数 */
double   price_real = price_ticks / 10000.0;  /* 显示用 */

/* 更快：用移位（tick_size = 2^-4 = 0.0625）*/
int32_t price_fixed = 123456;        /* 定点数，4 位小数二进制 */
int32_t dollars = price_fixed >> 4;  /* 取整数部分 */
/* 或用 2 的幂做缩放，避免浮点 */
```

> **关键原则：** 热路径里价格计算 **全用整数 tick**，只在显示/日志时转回浮点。移位代替乘除可以省几条指令的延迟（→ [§2.3 imul 延迟](./section-2.3-整数算术.md)）。

### 移位速查表

| 操作 | 规则 | C |
|------|------|----|
| `<< k` | 高位丢，低位补 0 | `x << k` |
| `>> k` (unsigned) | 低位丢，高位补 0（逻辑） | `u >> k` |
| `>> k` (signed 正) | 低位丢，高位补 0 | `x >> k` |
| `>> k` (signed 负) | 低位丢，高位补符号位（**通常**，不保证） | `x >> k` |
| `<< 0` | 不变 | `x << 0 == x` |
| `<< w` | **UB** | — |
| `<< -1` | **UB** | — |
