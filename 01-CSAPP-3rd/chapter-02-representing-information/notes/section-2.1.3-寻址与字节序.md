## 2.1.3 寻址和字节顺序 (Byte Ordering)

> **Ch2 §2.1 一条线：** [§2.1.1 十六进制](./section-2.1.1-十六进制表示法.md) → [§2.1.2 数据大小](./section-2.1.2-数据大小与sizeof.md) → **§2.1.3 本节**

> **一句话先分清：** 大端/小端只规定 **「多字节数字在内存（或字节流）里的存放顺序」**；  
> 和「数据进来的顺序」**不能直接划等号** — 要分两层：**① 内存怎么排** · **② 网络怎么发/收**。

---

### 寻址 (addressing)

- CPU 通过 **地址总线** 访问内存；x86-64 为 **按字节寻址 (byte-addressable)** — **每个字节** 有唯一地址
- 变量 `int x` 占 **4 个连续字节** 的地址，例如 `0x1000`–`0x1003`
- `&x` = 对象 **起始（最低）字节** 的地址
- 指针算术：`char* p; p+1` 前进 1 字节；`int* q; q+1` 前进 `sizeof(int)` 字节 — 见 [§3.8 指针步长](../../chapter-03-machine-level-programs/notes/section-3.8-指针步长详解.md)

**寻址回答：** 数据在 **哪几个字节地址** 上。

### 字节顺序 (byte order / endianness)

多字节对象占多个连续地址时，**各字节在地址上的排列顺序**。

| | 英文 | 规则 | 典型 |
|--|------|------|------|
| **小端** | Little-Endian | **最低有效字节** 在 **低地址** | **x86-64 原生**（不可切换） |
| **大端** | Big-Endian | **最高有效字节** 在 **低地址** | **网络字节序**、多数 wire 协议 |

> **术语：** byte order / **endianness**

**字节序回答：** 同一数值的多个字节，**谁放在低地址**。

> **易混：** 寻址 ≠ 字节序。先确定占哪段地址，再谈 `0x78` 和 `0x12` 谁先谁后。

### 示例：`int x = 0x12345678`

拆成：高字节 `0x12` · `0x34` · `0x56` · 低字节 `0x78`；地址低→高：`addr0 … addr3`。

| | addr0 | addr1 | addr2 | addr3 |
|--|-------|-------|-------|-------|
| **大端**（网络序） | `0x12` | `0x34` | `0x56` | `0x78` |
| **小端**（x86） | `0x78` | `0x56` | `0x34` | `0x12` |

- 大端：书写顺序 ≈ 地址递增顺序，人读直观。  
- 小端：低位先放低地址 — x86 全系；对硬件加法器友好。

`0x11223344`：**小端** `44 33 22 11`；**大端** `11 22 33 44`。

---

### 「大端 = 数据进来的顺序？」—— 分场景

#### 场景 1：网络包（HFT 重点）— **这里可以这么说**

网络传输 **统一大端（network byte order）**：

- 发送：多字节数字 **先发高字节、后发低字节**  
- 接收：字节流到达顺序 = 大端内存布局  

例：发 `0x12345678` → 线上顺序 `12 → 34 → 56 → 78`。  
→ **在网络层，「数据流进来的顺序」= 大端字节序。**

#### 场景 2：x86 本地内存 — **不能划等号**

x86 **永远小端**。网卡/用户态缓冲里若仍是 **大端字节布局**，用 `*(uint32_t*)buf` 直接当本机 `int` → **数值错**。  
必须 `ntohl` / 显式解码： **wire 大端字节流 ≠ CPU 小端解析顺序**。

#### 场景 3：文件 / 自定义二进制

文件里字节顺序 = **写入方约定**，与「进来」无固定绑定；协议文档写死大端或小端即可。

| 概念 | 固定规则（HFT） |
|------|-----------------|
| **字节流收发顺序** | 网络：高字节先到 = **大端流入** |
| **CPU 内存解析** | x86：低字节在低地址 = **小端** → 收发数字要转换 |

### 和字符串的直觉对比（类比，不是定义）

| | 内存低→高 | 直觉 |
|--|-----------|------|
| `"1234"` | `31 32 33 34` | 书写从左到右，**接近大端直觉** |
| 小端 `0x12345678` | `78 56 34 12` | **低位先放低地址**，与字符串 **相反** |

### 逐字节查看

```c
int a = 0x11223344;
unsigned char *p = (unsigned char *)&a;
/* x86 小端: 44 33 22 11 */
for (size_t i = 0; i < sizeof a; i++)
    printf("%02x ", p[i]);
```

### 网络序转换

```c
#include <arpa/inet.h>   /* Linux */
uint16_t n16 = htons(h16);   /* host → network（小端机上会翻字节） */
uint32_t n32 = htonl(h32);
uint16_t h16 = ntohs(n16);   /* network → host */
uint32_t h32 = ntohl(n32);
```

### 相关：Padding（Ch3）

wire 传 struct：**endian + padding + 类型宽度** 须一起约定 → [§3.9 结构体与对齐](../../chapter-03-machine-level-programs/notes/section-3.9-结构体联合与对齐.md)

### HFT 必记

1. FIX / TCP 二进制行情等：**wire 大端**；本机 x86 **小端** → `hton*`/`ntoh*` 或 schema 生成代码  
2. ARM 可配端序，但 **网络开发一律按大端协议**；x86 **不可切换**  
3. **禁止** `write(fd, &m, sizeof m)` 裸发本地 struct  
4. 同机 mmap：须同 endian、同 ABI、同 padding  

→ [Ch 11 网络编程](../../chapter-11-network-programming/) · [11 UNP](../../../11-UNP-Vol1/)

**极简：** 网络层大端 ≈「流入字节顺序」；本机 x86 小端解析，**中间必须转换**。
