### 为什么用十六进制

二进制太长（`11111111`），十进制看不出位模式。**十六进制 (hex) 每 1 位 = 4 个 bit**，正好把一个字节写成 2 位 hex，读位模式和地址都方便：

| 十进制 | 十六进制 | 二进制（8 bit） |
|--------|----------|---------------|
| 0 | 0x00 | 00000000 |
| 15 | 0x0F | 00001111 |
| 16 | 0x10 | 00010000 |
| 255 | 0xFF | 11111111 |

**hex ↔ bin 速查：**

| hex | bin | hex | bin |
|-----|-----|-----|-----|
| 0 | 0000 | 8 | 1000 |
| 1 | 0001 | 9 | 1001 |
| 2 | 0010 | A | 1010 |
| 3 | 0011 | B | 1011 |
| 4 | 0100 | C | 1100 |
| 5 | 0101 | D | 1101 |
| 6 | 0110 | E | 1110 |
| 7 | 0111 | F | 1111 |

> **背法：** 前 8 个（0-7）和高位 0 对应；后 8 个（8-F）高位 1 对应。`A=10` 是分界，记住 `A=1010` 其余可推。

---

### C 字面量

```c
int a = 0xFF;          /* 255 */
int b = 0xDEADBEEF;    /* 常见魔法值，调试时认出它是 hex */
unsigned long addr = 0x7ffd0000;
```

> 前缀 `0x` 不区分大小写；hex 数字 A-F 也不区分大小写。`0xff` == `0xFF` == `0Xff`。

---

### 三种进制互转

**hex → dec：** 从右往左，每位乘 16 的幂

\[
\text{0x2AF} = 2 \times 16^2 + 10 \times 16^1 + 15 \times 16^0 = 512 + 160 + 15 = 687
\]

**dec → hex：** 反复除 16 取余

```
687 / 16 = 42 余 15 → F
 42 / 16 =  2 余 10 → A
  2 / 16 =  0 余  2 → 2
从下往上读：0x2AF ✓
```

**hex ↔ bin：** 直接逐位展开（最快）

```
0x2AF = 0010 1010 1111
         2    A    F
```

> 实际调试时 **hex ↔ bin 直接逐位转换**，不用走十进制中转。

---

### 2 的幂速查（算对齐/缓冲区时常用）

| 2^n | 十进制 | hex | 用途 |
|-----|--------|-----|------|
| 2^4 | 16 | 0x10 | 对齐粒度 |
| 2^8 | 256 | 0x100 | 一页扇区 |
| 2^10 | 1024 (1K) | 0x400 | KB 边界 |
| 2^12 | 4096 (4K) | 0x1000 | **默认页大小** |
| 2^16 | 65536 (64K) | 0x10000 | 旧 16 位寻址上限 |
| 2^20 | 1M | 0x100000 | MB 边界 |
| 2^32 | 4G | 0x100000000 | 32 位地址空间上限 |
| 2^64 | … | 0x10000000000000000 | 64 位地址空间 |

> **HFT 记住：** `0x1000` = 4096 = 默认页大小（大页是 `0x200000` = 2MB）。内存对齐、`posix_memalign` 的参数都是 2 的幂。

---

### 调试中的 hex 习惯

| 场景 | 为什么要 hex |
|------|-------------|
| 内存 dump | 一字节 = 2 位 hex，直接看位模式 |
| 协议抓包 | wire 格式就是字节序列，hex 最直观 |
| 寄存器值 | CPU 寄存器 32/64 位，hex 紧凑 |
| 地址 | `0x7ffd...` 比 `140726...` 好认 |
| 魔数/签名 | `0xDEADBEEF`、`0xCAFEBABE`(Java class) |

> **GDB：** `x/16xb &buf` 以 hex dump 内存；`p/x var` 以 hex 打印变量。
