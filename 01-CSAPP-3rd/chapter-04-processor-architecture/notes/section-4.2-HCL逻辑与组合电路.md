## 4.2 逻辑设计与 HCL（4.2.1–4.2.5）

### 4.2.1–4.2.3 逻辑门、组合电路、字级表达式

- **组合电路：** 输出仅依赖 **当前输入**，无记忆
- **HCL** — 本书硬件描述语言，类似布尔表达式 + 多路选择

```hcl
bool out = (a && b) || !c;
word out = a ? b : c;   // 字级多路选择器
```

- **ALU、译码器、多路复用器** 都是组合逻辑堆出来的
- **一位全加器** 例：`Sum = A⊕B⊕Cin`，`Cout = (A&B)|(Cin&(A⊕B))` — 不全是异或；**4 个串联 = 4 位加法器 ⊂ ALU**，ALU 还要靠选择电路做减/逻辑 → [§2.3.1](../../chapter-02-representing-information/notes/section-2.3-整数算术.md)

### 4.2.4 集合关系

```hcl
word srcA = [
  icode in { IRRMOVQ, RMMOVQ, ... } : rA;
  icode in { IRMMOVQ, ... } : RNONE;
  1 : RNONE;
];
```

- 用 **集合成员** 写控制信号 — 对应硬件里「opcode 译码表」

### 4.2.5 存储器与时钟

| 类型 | 行为 |
|------|------|
| **组合逻辑** | 电平稳定后输出有效 |
| **时序逻辑（寄存器）** | **时钟上升沿** 采样输入写入 |

- **处理器 = 组合逻辑块 + 流水线寄存器 + 时钟**
- **setup/hold time** — 真芯片时序约束；Y86 模拟器抽象掉

**HFT：** 不必写 HCL；读懂 **「控制信号在时钟边沿锁存」** 即可理解为何流水线要 **插入寄存器**、为何 stall 浪费周期。
