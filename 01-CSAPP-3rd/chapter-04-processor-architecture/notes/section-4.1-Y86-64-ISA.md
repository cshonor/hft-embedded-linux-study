### 核心结论（先钉死）

1. **Y86-64 是纯教学 ISA** — 工业界 / 服务器 / 嵌入式 / **HFT 产品代码没人用**，不会拿来写上线逻辑。  
2. **设计目的：** 砍掉 x86-64 繁杂细节，只留最小核心，专门讲 **CPU 数据通路与流水线**，降低门槛。  
3. **一句话区分：** Y86 = **教具**；x86-64 = **工业实用架构**（HFT 最终写的是后者）。

**教具 ≠ 没用：** 载体（指令编码、模拟器）是虚构的；**底层思想可直接套真机** — 寄存器、栈帧、取指～写回流程、条件码、访存读写。学它是为了 **降低理解 CPU 原理的门槛**，不是为了上线跑 Y86。

同理：[§4.2 HCL](./section-4.2-HCL逻辑与组合电路.md) 语法没人用，但 **MUX / ALU / 时序寄存器** 是真硬件；工业界用 Verilog 写的是同一类电路，只是语法不同。  
→ **载体是教具，背后原理是工业真实逻辑** — 别只看「工具没用」就跳过。

### 4.1.1 程序员可见的状态

| 组件 | Y86-64 | 备注 |
|------|--------|------|
| **寄存器** | **15** 个 64 位：`%rax %rcx %rdx %rbx %rsp %rbp %rsi %rdi %r8`～`%r14` | 命名贴近 x86；**没有 `%r15`**（真 x86-64 有 16 个 GPR） |
| **`%rsp`** | 栈指针 | 规则与真机一致，服务 `push`/`pop`/`call`/`ret` |
| **PC** | 程序计数器 | 下一条指令地址 |
| **CC** | **ZF、SF、OF**（**三个**） | 分支 / `cmov` 用；**无 CF**（Y86 算术集太简，不维护进位旗标） |
| **Stat** | 程序状态 | 正常 / halt / 异常 |
| **内存** | 字节数组，小端 | |

> **纠正口述易混：** 不是「16 个 + CF」。书上是 **15 寄存器 + ZF/SF/OF**。真机才有完整 **16 GPR + CF** 等。

简化目标：**无特权级、无浮点、无向量、无 MMU 特权细节** — 只为讲清数据通路。

### 4.1.2–4.1.3 五大类指令（阉割版）

#### 1. 数据传送

| 指令 | 含义 |
|------|------|
| `irmovq` | 立即数 → 寄存器 |
| `rrmovq` / `cmovXX` | 寄存器 ↔ 寄存器（条件传送） |
| `mrmovq` | 内存 → 寄存器（load） |
| `rmmovq` | 寄存器 → 内存（store） |

#### 2. 算术 / 位运算（仅四条）

`addq` `subq` `andq` `xorq`

**无** 乘、除、移位 — 与真 x86 **最大差别之一**。

#### 3. 分支

`jmp`、`jle`/`jl`/`je`/`jne`/`jge`/`jg` — 看 CC 决定是否改 PC。

#### 4. 栈与调用（呼应 Ch3 栈帧）

`pushq` `popq` `call` `ret` — 靠 `%rsp` 维护栈；原理与 [Ch3 §3.2.2 / §3.7](../../chapter-03-machine-level-programs/notes/section-3.2.2-栈帧.md) **通用**。

#### 5. 停机

`halt` — 终止（另有 `nop`）。

**编码：** 变长（约 1 / 2 / 10 字节等）— **复刻 x86「指令长度不固定」**，便于后面讲取指。

---

### 内存与寻址

- 按 **8 字节字** 做整数访存（`mrmovq`/`rmmovq`）
- 寻址极简：基本是 **「寄存器 + 偏移」**（`D(r)`），无真 x86 那套复杂 scale-index
- 小端

---

### 4.1.4 异常

非法指令、地址错等 → 置 `Stat` 并停。真 CPU 有 `#GP`/`#PF` 等完整异常模型 — 本章不展开。

---

### 为何工业界绝对不用 Y86？

| 原因 | 说明 |
|------|------|
| **功能残缺** | 无乘除/移位/浮点/**SIMD** — HFT 批量行情算都写不全 |
| **无 OS / IO 能力** | 无 syscall、外设模型 — 跑不起真 OS，只能教学小程序 |
| **无真实硅片** | 只有书本 **模拟器**（`ssim`/`psim` 等），没有 Y86 CPU 产品 |

---

### 既然不用，为何还要学 §4.1？（HFT 价值）

1. **屏蔽冗余** — 真 x86 上千条指令；Y86 十几条，先看清 **取指→译码→执行→访存→写回**  
2. **为 §4.3～4.5 铺路** — 流水线 stall、数据依赖、分支冲刷，举例全是 Y86；指令行为不清楚，后面冒险看不懂  
3. **巩固 Ch3** — 栈帧、`call`/`ret`、条件码分支、load/store 逻辑与真汇编 **同一套底层直觉**；Y86 是缩小版演练场  

**HFT 最终写代码用的是 x86-64**；本章学的是 **「一条指令在硬件里变成控制信号 + ALU + 访存」** 的映射 — 读 `objdump` / 想冒险时用得上。

---

### 口述巩固 · 自测

1. **Y86 能上线写交易吗？** — **不能**；纯教具。  
2. **几个 GPR？有没有 `%r15`、CF？** — **15**；**无 `%r15`**；CC 只有 **ZF/SF/OF**。  
3. **算术有哪几条？缺什么？** — add/sub/and/xor；无乘除移位浮点 SIMD。  
4. **为何还要学？** — 为流水线/冒险铺路；巩固 Ch3 调用与访存直觉。
