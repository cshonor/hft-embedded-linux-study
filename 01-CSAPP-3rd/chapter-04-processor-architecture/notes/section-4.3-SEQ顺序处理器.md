## 4.3 Y86-64 的顺序实现 SEQ（4.3.1–4.3.4）

> [章导读](../README.md) · 上节 [§4.2 HCL](./section-4.2-HCL逻辑与组合电路.md) · 下节 [§4.4 流水线](./section-4.4-流水线原理与局限.md)

---

### 本节定位

§4.1（指令）+ §4.2（MUX / ALU / 寄存器零件）铺垫完成 → 本节 **拼出整台完整 CPU：SEQ 单周期顺序处理器**。

**核心规则：** 一条指令把 **五阶段全部走完**，才执行下一条 — **无并行、无流水线重叠**。

> **一句话：** 用 §4.2 的零件搭一台 **串行逐条执行** 的 CPU；为 §4.4 流水线做「慢但清楚」的对照。

---

### 统一的五级流程（重中之重 · 全书通用）

加减、访存、`call`/`ret`/`jXX` — **每条指令都按这 5 步**（教学模型：一个很长的时钟周期内组合逻辑算完，边沿锁存状态）：

| # | 阶段 | 干什么 |
|---|------|--------|
| 1 | **Fetch 取指** | 从 **PC** 读指令字节；拆出操作码、寄存器号、立即数；算 `valP`（顺序下一条） |
| 2 | **Decode 译码** | 从 **寄存器堆** 读源操作数（如 `%rax`、`%rbx`） |
| 3 | **Execute 执行** | **ALU** 做加减/位运算/有效地址计算；更新 **CC（ZF/SF/OF）** — Y86 **无 CF** |
| 4 | **Memory 访存** | 需要时读写数据内存（`mrmovq`/`rmmovq`/`push`/`pop`/`call`/`ret` 等）；纯 `OPq` 则 **空操作** |
| 5 | **Write back 写回** | 把 ALU 结果或内存读值写回目标寄存器 |

五步做完 → 更新 PC → 再取下一条。

```
F  Fetch      — 读指令，算 valP
D  Decode     — 读寄存器
E  Execute    — ALU、有效地址、条件
M  Memory     — 读/写内存（可空）
W  Write back — 写寄存器
（然后锁存新 PC）
```

---

### SEQ 硬件构成（全是 §4.2 零件）

| 模块 | 类型 | 作用 |
|------|------|------|
| **PC** | 时序 | 当前指令地址；时钟边沿更新 |
| **取指 / 译码逻辑** | 组合 + MUX | 拆字段、产生控制信号 `sel` |
| **寄存器堆** | 时序 | Y86：**15** 个 GPR（`%rax`～`%r14`）；双读 + 写端口 |
| **ALU** | 组合 | 加法器 + 逻辑 + **结果 MUX** |
| **数据存储器** | 时序（模型） | load/store |
| **MUX 集群** | 组合 | 每级选数据通路 / 新 PC |
| **CC** | 时序 | ZF/SF/OF，供 `jXX`/`cmov` |
| **Stat** | — | 异常 / halt 汇总 |

> **纠正：** 不是「16 寄存器 + CF」— 与 [§4.1](./section-4.1-Y86-64-ISA.md) 一致。

---

### 例 1：`addq %rax, %rbx`（不访存）

1. **F** — 取到 `addq`，识别 `OPq`  
2. **D** — 读出 `%rax`、`%rbx`  
3. **E** — ALU 相加，更新 CC  
4. **M** — 空操作  
5. **W** — 结果写回 `%rbx`  
6. PC ← `valP`，下一条  

### 例 2：`jle dest`（条件跳转）

1. **F** — 取到分支指令与目标  
2. **D** — 通常无通用寄存器读（看 CC）  
3. **E** — 根据 CC 判是否 ≤  
4. **M** — 空  
5. **W** — 无寄存器写  
6. **PC MUX** — 满足则目标地址，否则 `valP`  

`push`/`pop`/`call`/`ret` 在 M/W 会动栈与返回地址 — 与 [Ch3 栈帧](../../chapter-03-machine-level-programs/notes/section-3.2.2-栈帧.md) 同一套语义，只是落到五级硬件上。

---

### 时序：单周期教学模型

- **一条指令 = 一个极长时钟周期** 内走完五阶段（组合逻辑传播 + 边沿写寄存器/内存/PC）  
- 真 x86：**多周期、流水、乱序、微码** — SEQ 只是清晰起点  

---

### SEQ 最大缺陷（→ §4.4 流水线）

1. **吞吐极低** — 同时只有一条指令占用全机；ALU 算时，取指/内存口常 **闲置**  
2. **时钟迁就最慢级** — 访存往往最慢，整周期必须够长，**无法**按「快阶段」提频  

流水线要解决的正是：**重叠多条指令、提高吞吐**（代价是冒险 — §4.5）。

---

### 学习链路与 HFT

| 点 | 含义 |
|----|------|
| **地基** | 先懂串行五级，才懂流水线为何加速、为何会 stall |
| **↔ Ch3** | `call`/`ret`/栈操作在 SEQ 里逐步落地 |
| **HFT** | 现代 CPU ≈ SEQ 的并行升级；看清 **单条指令吃哪些硬件**，才理解冒险时 **资源冲突 / 依赖** 从哪来 |

**HFT：** SEQ 帮你建立 **单指令 latency** 直觉；高吞吐靠 PIPE/超标量，不是 magic。

---

### 口述巩固 · 自测

1. **SEQ 核心规则？** — 一条走完五级才下一条；无重叠  
2. **五级名字与顺序？** F→D→E→M→W  
3. **`addq` 哪一级空？** M  
4. **`jle` 新 PC 谁选？** PC MUX（看 CC）  
5. **SEQ 两大性能病？** 资源闲置；时钟被最慢级拖死
