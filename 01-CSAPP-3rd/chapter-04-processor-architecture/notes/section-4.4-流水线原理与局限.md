## 4.4 流水线的通用原理（4.4.1–4.4.4）

### 4.4.1 计算流水线

- **目标：** 提高 **吞吐 (throughput)** — 每周期完成一条指令（理想 CPI=1）
- **代价：** 单条指令 **延迟 (latency)** 仍 ≥ 各段延迟之和；只是多条指令重叠

```
无流水线：3 条指令 × 300ps = 900ps
3 段流水线：300 + 2×100 = 500ps（若每段 100ps）— 图示见原书
```

### 4.4.2 流水线操作

- 段间插入 **流水线寄存器** — 保存该段输出供下段使用
- **稳态：** 流水线「填满」后，每拍完成一条

### 4.4.3 流水线的局限性

| 限制 | 说明 |
|------|------|
| **不一致的段延迟** | 最慢段决定时钟周期 |
| **流水线寄存器开销** | 面积与延迟 |
| **冒险 (hazards)** | 下节 PIPE 核心 |

**加速比上界：** 段数 N、指令数 M 很大时，加速比 → N；开头/结尾 **气泡** 浪费

### 4.4.4 带反馈的系统

- **PC 依赖上条指令结果**（跳转、`ret`）— 不能无脑每拍 +1
- **数据反馈** — 写回值下条要用 → **数据冒险**
- 带反馈的系统流水线化需要 **冒险检测 + 转发 + 预测**

**HFT 直觉：**

- **IPC < 1** 常见 — 不全是分支；还有 cache miss、资源冲突
- **微基准「一条指令多快」≠ 程序多快** — 看稳态吞吐与冒险

→ 优化减少冒险：[Ch 5](../../chapter-05-optimizing-performance/)
